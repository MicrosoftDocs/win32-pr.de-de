Dieser Abschnitt enthält eine Liste der empfohlenen Bücher und Ressourcen, die sich auf das Design der Benutzeroberfläche beziehen. (Diese Bücher und Ressourcen sind möglicherweise nicht in einigen Sprachen und Ländern verfügbar.)

## <a name="reading-list"></a>Leseliste

<dl> <dt>

<span id="_Windows_Interface_Guidelines_for_Software_Design"></span><span id="_windows_interface_guidelines_for_software_design"></span><span id="_WINDOWS_INTERFACE_GUIDELINES_FOR_SOFTWARE_DESIGN"></span>*Windows-Schnittstellen Richtlinien für den Software Entwurf*
</dt> <dd>

(Microsoft Press, 1995; ISBN 1-55615-679-0)

In diesem Buch werden alle wichtigen Benutzeroberflächen Regeln erläutert, die Sie befolgen können, und es werden viele Beispiele für das Layout von Dialogfeldern, Assistenten, Symbolleisten und Kontextmenüs erläutert.

</dd> <dt>

<span id="Usability_Engineering_by_Jakob_Nielsen_"></span><span id="usability_engineering_by_jakob_nielsen_"></span><span id="USABILITY_ENGINEERING_BY_JAKOB_NIELSEN_"></span>*Nutzbarkeits Engineering* von Jakob Nielsen 
</dt> <dd>

(AP Professional, 1994; ISBN 0125184069)

Dieses Buch konzentriert sich auf die Techniken zur effektiven Messung der Nutzbarkeit einer Schnittstelle. Es werden viele verschiedene Testverfahren zum Vergleichen verschiedener Entwurfs Ideen erläutert, einschließlich Vorschlägen, mit denen die Kosten für die Verwendbarkeit getestet werden können. Dieses Buch behandelt Analysetechniken, die auf Websites, Webseiten oder beliebige Software angewendet werden können. Es ist hervorragend für die ersten Phasen des Lernens von Benutzern und die mittleren Phasen der Iterations-und Nutz barkeits Tests geeignet.

</dd> <dt>

<span id="Design_of_Everyday_Things_by_Don_Norman"></span><span id="design_of_everyday_things_by_don_norman"></span><span id="DESIGN_OF_EVERYDAY_THINGS_BY_DON_NORMAN"></span>*Entwerfen von alltäglichen Dingen* von Don Norman
</dt> <dd>

(Currency/Doubleday, 1988; ISBN: 0385267746)

Dieses Buch erläutert die psychologischen Prinzipien bei der Arbeit, die die Verwendung von Schnittstellen so schwierig gestalten. Empfohlen für alle Benutzeroberflächen, die ein umfassendes Verständnis der damit verbundenen Probleme wünschen. Beispiele hierfür sind Küchengeräte, Büro Türen und Automaten.

</dd> <dt>

<span id="About_Face_by_Alan_Cooper_"></span><span id="about_face_by_alan_cooper_"></span><span id="ABOUT_FACE_BY_ALAN_COOPER_"></span>*Über Gesicht* von Alan Cooper 
</dt> <dd>

(IDG-Bücher weltweit, 1995; ISBN 1568843224)

Wenn Sie sich über Benutzeroberflächen Lösungen Gedanken machen, sollte ein kurzes durchlaufen dieses Buchs helfen, Ideen zu erhalten.

<span id="The_Elements_of_Friendly_Software_Design_by_Paul_Heckel_"></span><span id="the_elements_of_friendly_software_design_by_paul_heckel_"></span><span id="THE_ELEMENTS_OF_FRIENDLY_SOFTWARE_DESIGN_BY_PAUL_HECKEL_"></span>*Die Elemente des freundlichen Software Entwurfs* von Paul Heckel
