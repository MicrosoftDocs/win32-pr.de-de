<dl> <dt>

<span id="_rras_cyclic_redundancy_check_crc__gly"></span><span id="_RRAS_CYCLIC_REDUNDANCY_CHECK_CRC__GLY"></span>**Zyklische Redundanz Überprüfung (CRC)**
</dt> <dd>

Ein CRC ist ein numerischer Wert, der mit einem Datenblock verknüpft ist. Dieser Wert stellt Informationen bereit, mit denen eine Anwendung ermitteln kann, ob die Daten geändert wurden. Ein CRC ähnelt einer Prüfsumme, wird jedoch als stärkerer Mechanismus zur Fehlerüberprüfung angesehen.

</dd> <dt>

<span id="_rras_compression_control_protocol_ccp__gly"></span><span id="_RRAS_COMPRESSION_CONTROL_PROTOCOL_CCP__GLY"></span>**Compression Control Protocol (CCP)**
</dt> <dd>

Mit dem Compression Control Protocol (CCP) können zwei Computer, die über PPP kommunizieren, kompatible Algorithmen für das Senden und empfangen von komprimierten PPP-Frames aushandeln. Von den beiden Computern wird möglicherweise keine CCP-Verbindung bis zur Netzwerk Steuerungs Protokoll-Phase der PPP-Verbindung verwendet.

Weitere Informationen finden Sie unter [RFC 1962](remote-access-request-for-comments.md). Siehe auch [**PPP- \_ CCP- \_ Informationen**](/windows/desktop/api/Mprapi/ns-mprapi-ppp_ccp_info) und [**raspppccp**](/previous-versions/windows/desktop/legacy/aa377620(v=vs.85)).

</dd> </dl>
