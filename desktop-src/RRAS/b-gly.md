<dl> <dt>

<span id="_rras_best_route_gly"></span><span id="_RRAS_BEST_ROUTE_GLY"></span>**beste Route**
</dt> <dd>

Die beste Route ist eine optimale Route zu einem Netzwerk Ziel, basierend auf den angegebenen Kriterien. Dieses Konzept basiert auf der Tatsache, dass es eine bestimmte "Kosten" für die Durchführung einer Route in einem Netzwerk gibt.

Die beste Route ist die beste Lösung, die auf bestimmten Kriterien basiert. Diese Kriterien können die Anzahl der überquerten Netzwerke, den Typ des überquerten Netzwerks (z. b. öffentlich oder privat) oder eine Währungs-oder Bandbreiten Grenze einschließen.

</dd> <dt>

<span id="_rras_border_gateway_protocol_bgp__gly"></span><span id="_RRAS_BORDER_GATEWAY_PROTOCOL_BGP__GLY"></span>**Border Gateway Protocol (BGP)**
</dt> <dd>

BGP ist ein [*Äußeres gatewayprotokoll*](e-gly.md). BGP leitet Informationen zwischen zwei [*autonomen Systemen*](a-gly.md)weiter. Es ist in [RFC 1267](routing-protocols-request-for-comments.md) und [RFC 1268](routing-protocols-request-for-comments.md)definiert.

</dd> <dt>

<span id="_mpr_brute_force_attack"></span><span id="_MPR_BRUTE_FORCE_ATTACK"></span>**Brute-Force-Angriff**
</dt> <dd>

Ein Versuch, eine Chiffre zu kompromittieren, indem Sie alle möglichen Versuche durchführen.

</dd> </dl>
