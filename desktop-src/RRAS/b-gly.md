<dl> <dt>

<span id="_rras_best_route_gly"></span><span id="_RRAS_BEST_ROUTE_GLY"></span>**Beste Route**
</dt> <dd>

Die beste Route ist eine optimale Route zu einem Netzwerkziel basierend auf einem angegebenen Kriterium. Dieses Konzept basiert auf der Tatsache, dass eine Route über ein Netzwerk mit bestimmten "Kosten" verbunden ist.

Die beste Route ist die Route mit den niedrigsten Kosten basierend auf einem angegebenen Kriterium. Zu diesen Kriterien können die Anzahl der gekreuzten Netzwerke, der Typ des gekreuzten Netzwerks (z. B. öffentlich oder privat) oder ein Monetäres- oder Bandbreitenlimit gehören.

</dd> <dt>

<span id="_rras_border_gateway_protocol_bgp__gly"></span><span id="_RRAS_BORDER_GATEWAY_PROTOCOL_BGP__GLY"></span>**Border Gateway Protocol (BGP)**
</dt> <dd>

BGP ist ein [*äußeres Gatewayprotokoll.*](e-gly.md) BGP gibt Informationen zwischen zwei [*autonomen System weiter.*](a-gly.md) Sie ist in [RFC 1267 und](routing-protocols-request-for-comments.md) [RFC 1268 definiert.](routing-protocols-request-for-comments.md)

</dd> <dt>

<span id="_mpr_brute_force_attack"></span><span id="_MPR_BRUTE_FORCE_ATTACK"></span>**Brute-Force-Angriff**
</dt> <dd>

Ein Versuch, eine Verschlüsselung zu gefährden, indem alles mögliche versucht wird.

</dd> </dl>
