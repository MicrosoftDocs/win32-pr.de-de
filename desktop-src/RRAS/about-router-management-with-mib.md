# <a name="about-router-management-with-mib"></a><span data-ttu-id="a178b-110">Informationen zur Routerverwaltung mit MIB</span><span class="sxs-lookup"><span data-stu-id="a178b-110">About Router Management with MIB</span></span>

<span data-ttu-id="a178b-111">Die MIB-APIs (Management Information Base) für die Routerverwaltung ermöglichen es, die Werte von MIB-Variablen abzufragen und festzulegen, die von einem der routermanager oder einem der Routing Protokolle exportiert werden, die der routermanager-Dienst verwendet.</span><span class="sxs-lookup"><span data-stu-id="a178b-111">The Management Information Base (MIB) APIs for router management makes it possible to query and set the values of MIB variables exported by one of the router managers or any of the routing protocols that the router managers service.</span></span> <span data-ttu-id="a178b-112">Wenn Sie diese API verwenden, unterstützt der Router das Simple Network Management-Protokoll (SNMP).</span><span class="sxs-lookup"><span data-stu-id="a178b-112">By using this API, the router supports the Simple Network Management Protocol (SNMP).</span></span>

<span data-ttu-id="a178b-113">Im SNMP-Framework wird jedes der verwaltbaren Objekte im Router durch eine Variable dargestellt, die über einen eindeutigen Objekt Bezeichner (OID) verfügt.</span><span class="sxs-lookup"><span data-stu-id="a178b-113">In the SNMP framework, each of the manageable objects in the router is represented by a variable that has a unique Object Identifier (OID).</span></span> <span data-ttu-id="a178b-114">Jede OID entspricht einem Wert, der den aktuellen Zustand des Objekts darstellt.</span><span class="sxs-lookup"><span data-stu-id="a178b-114">Corresponding to each OID is a value that represents the current state of the object.</span></span> <span data-ttu-id="a178b-115">Die Auflistung von OIDs und Werten wird als Management Information Base (MIB) bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="a178b-115">The collection of OIDs and values is referred to as a Management Information Base (MIB).</span></span> <span data-ttu-id="a178b-116">Der mpradminmib-Aufruf ermöglicht einem Entwickler, ein Objekt anhand seiner OID anzugeben und den Wert des Objekts entweder abzufragen oder zu schreiben ("Set").</span><span class="sxs-lookup"><span data-stu-id="a178b-116">The MprAdminMib calls allow a developer to specify an object by its OID and either query or write ("Set") the object's value.</span></span>

<span data-ttu-id="a178b-117">Um MIB-Variablen abzufragen und festzulegen, muss das Modul, das die Aufrufe verwendet, eine Reihe von Datenstrukturen definieren.</span><span class="sxs-lookup"><span data-stu-id="a178b-117">To query and set MIB variables, the module that services the calls must define a set of data structures.</span></span> <span data-ttu-id="a178b-118">Diese Datenstrukturen enthalten Strukturen, die als Objekt Bezeichner und Strukturen verwendet werden, die die Werte der MIB-Variablen enthalten, auf die zugegriffen wird.</span><span class="sxs-lookup"><span data-stu-id="a178b-118">These data structures include structures to use as Object Identifiers and structures that hold the values of the MIB variables being accessed.</span></span> <span data-ttu-id="a178b-119">Diese Datenstrukturen sind nur für den Aufrufer der MIB-Funktion und das Modul, das den Aufruf verarbeitet, nicht transparent.</span><span class="sxs-lookup"><span data-stu-id="a178b-119">These data structures are opaque to all but the caller of the MIB function and the module servicing the call.</span></span>

<span data-ttu-id="a178b-120">Das Modul, das den MIB-Befehl verarbeitet, ist ein routermanager oder eines der Routing Protokolle.</span><span class="sxs-lookup"><span data-stu-id="a178b-120">The module servicing the MIB call is a router manager or one of the routing protocols.</span></span> <span data-ttu-id="a178b-121">Der Aufrufer muss einen routermanager angeben, auch wenn der Aufruf von einem der Routing Protokolle verarbeitet wird.</span><span class="sxs-lookup"><span data-stu-id="a178b-121">The caller must specify a router manager even if the call is handled by one of the routing protocol.</span></span> <span data-ttu-id="a178b-123">Wenn z. b. das OSPF (Open kürzeste Path First)-Routing Protokoll den MIB-Aufruf verarbeitet hat, müsste der Aufrufer den IP-routermanager angeben, da OSPF zur IP-Protokollfamilie gehört.</span><span class="sxs-lookup"><span data-stu-id="a178b-123">For example, if the Open Shortest Path First (OSPF) routing protocol were handling the MIB call, *dwTransportId* parameter specifies a router manager, and the *RoutingPid* parameter specifies the routing protocol.</span></span> <span data-ttu-id="a178b-125">Der routermanager verfügt auch über <span data-ttu-id="a178b-126">Die mpradminmib-Funktionen können auf einem anderen Computer als dem verwalteten Computer aufgerufen werden.</span><span class="sxs-lookup"><span data-stu-id="a178b-126">The MprAdminMib functions can be called on a computer other than the one being administered.</span></span> <span data-ttu-id="a178b-127">Die mpradminmib-Funktionen, die Werte Abfragen oder schreiben, nehmen als Parameter ein Handle für den zu verwaltenden Computer an.</span><span class="sxs-lookup"><span data-stu-id="a178b-127">The MprAdminMIB functions that query or write values, take as a parameter a handle to the computer to administer.</span></span> <span data-ttu-id="a178b-128">Verwenden Sie die [**mpradminmibserverconnect**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibserverconnect) -Funktion, um die Verbindung mit dem Remote Computer herzustellen und dieses Handle abzurufen.</span><span class="sxs-lookup"><span data-stu-id="a178b-128">Use the [**MprAdminMIBServerConnect**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibserverconnect) function to establish the connection to the remote computer and obtain this handle.</span></span> <span data-ttu-id="a178b-129">Nachdem Sie die erforderlichen mpradminmib-Funktionen zum Ausführen einer bestimmten administrativen Aufgabe aufgerufen haben, rufen Sie die [**mpradminmibserverdisconnect**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibserverdisconnect) -Funktion auf, um die Verbindung mit dem Remote Computer zu beenden.</span><span class="sxs-lookup"><span data-stu-id="a178b-129">After calling the necessary MprAdminMIB functions to accomplish a particular administrative task, call the [**MprAdminMIBServerDisconnect**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibserverdisconnect) function to terminate the connection to the remote computer.</span></span>

<span data-ttu-id="a178b-130">Die [**mpradminmibentrycreate**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibentrycreate) -und [**mpradminmibentryset**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibentryset) -Funktionen akzeptieren als Parameter eine OID und einen Puffer, der den neuen Wert für das Objekt enthält.</span><span class="sxs-lookup"><span data-stu-id="a178b-130">The [**MprAdminMIBEntryCreate**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibentrycreate) and [**MprAdminMIBEntrySet**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibentryset) functions take as parameters an OID and a buffer which contains the new value for the object.</span></span>

<span data-ttu-id="a178b-134">Die [**mpradminmibentrygetfirst**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibentrygetfirst) -Funktion und die [**mpradminmibentrygetnext**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibentrygetnext) -Funktion ermöglichen einem Entwickler das Ausführen eines *SNMP*-Durchlaufs.</span><span class="sxs-lookup"><span data-stu-id="a178b-134">The [**MprAdminMIBEntryGetFirst**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibentrygetfirst) and [**MprAdminMIBEntryGetNext**](/windows/desktop/api/Mprapi/nf-mprapi-mpradminmibentrygetnext) functions enable a developer to perform an *SNMP walk*.</span></span> <span data-ttu-id="a178b-135">Da die OIDs geordnet sind, verfügt jede OID und somit jedes verwaltbare Objekt über eine *nächste* OID.</span><span class="sxs-lookup"><span data-stu-id="a178b-135">Because the OIDs are ordered, each OID and therefore each manageable object has a *next* OID.</span></span> <span data-ttu-id="a178b-136">Ein SNMP-Walk bezieht sich auf die Durchquerung eines Teils der MIB durch das Lesen oder Schreiben einer OIDs-Sequenz.</span><span class="sxs-lookup"><span data-stu-id="a178b-136">An SNMP walk refers to traversing a portion of the MIB by reading or writing a sequence of OIDs.</span></span>
