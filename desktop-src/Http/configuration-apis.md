# <a name="configuration-apis"></a>Konfiguration-APIs

Die HTTP-Server-APIs bieten Systemadministratoren Funktionen zum Festlegen, Abfragen und Löschen von Konfigurationsparametern. Die Konfigurations-APIs steuern die Einstellungen für alle Anwendungen, die die HTTP-Server-API auf dem System verwenden.

Die unten aufgeführten Abschnitte enthalten Informationen zu Konfigurationseinstellungen.

> [!Note]  
> All dies sind optionale Konfigurationsschritte, die basierend auf den System- und/oder Anwendungsanforderungen angewendet werden.

 

[IP-Abhörliste:](ip-listen-list.md) Beschreibt, wie HTTP für den Betrieb über bestimmte IP-Schnittstellen konfiguriert wird.

[SSL-Zertifikate:](ssl-certificates.md) Hier finden Sie Informationen zu den Verfahren zum Konfigurieren von Zertifikaten mit der HTTP-Server-API.

[Namespacereservierungen, Registrierungen und Routing:](namespace-reservations-registrations-and-routing.md) Beschreibt den Prozess der Reservierung von Teilen des Namespace für alle Anwendungen auf dem Computer mithilfe der HTTP-Server-API. Der Prozess zum Registrieren und Routen von Anforderungen an bestimmte Anwendungen wird ebenfalls beschrieben.

[HttpCfg.exe](httpcfg-exe.md) : Ein Beispielprogramm, das die Konfigurations-APIs verwendet. Eine kompilierte Version dieses Tools wird auch mit dem Betriebssystem unter den Supporttools zusammen mit einer Hilfedatei, die die Verwendung dokumentiert, zusammen.
