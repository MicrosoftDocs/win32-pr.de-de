# <a name="generic-com-error-codes"></a>Generische com-Fehler Codes

Generische com-Fehlercodes drückt den Vorgangs Status aus, der von allen com-Modulen erzeugt wird. Die folgende Tabelle enthält einige Beispiele für derartige Fehlercodes auf einer beliebigen Win32-Plattform.



| Fehlercode     | Hexadezimalwert  | BESCHREIBUNG              |
|----------------|------------|--------------------------|
| E \_ Abbrechen       | 0x80004004 | Der Vorgang wurde abgebrochen.       |
| E \_ fehlschlagen        | 0x80004005 | Unbekannter Fehler.       |
| E \_ nointerface | 0x80004002 | Schnittstelle wird nicht unterstützt. |
| E \_ notimpl     | 0x80004001 | Nicht implementiert.         |
| E- \_ Zeiger     | 0x80004003 | Ungültiger Zeiger.         |
| E \_ unerwartet  | 0x8000FFFF | Schwerwiegender Fehler.    |
