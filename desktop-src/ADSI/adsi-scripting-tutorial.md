# <a name="active-directory-service-interfaces-scripting"></a>Skripterstellung für Active Directory-Dienst Schnittstellen

Die Microsoft Active Directory Service Interfaces (ADSI) stellen in Kombination mit einer Skriptsprache wie Microsoft Visual Basic Scripting Edition oder Microsoft JScript Development Software einen Systemadministrator mit leistungsstarken Tools für die Verwaltung von Benutzern, Diensten, Druck Warteschlangen und anderen Teilen eines Netzwerks von Computern bereit, auf denen Microsoft Windows NT oder Windows 2000-Betriebssysteme ausgeführt werden. Skripts, die das ADSI-Objektmodell verwenden, können sich wiederholende manuelle Aufgaben ersetzen, die normalerweise über Verwaltungs Tools oder die Microsoft Management Console ausgeführt werden. Die ADSI-Skripterstellung kann auch reguläre System Wartungsaufgaben für den praktischen Betrieb automatisieren und bietet eine einfache Schnittstelle zum Schreiben webbasierter System Verwaltungs Tools mithilfe von Active Server Seiten.

Weitere Informationen zum Schreiben von Skripts mit ADSI mit Skript Lernprogrammen, Beispielen und Tools finden Sie unter:

-   Einführung von [ADSI Scripting](https://www.microsoft.com/resources/documentation/windows/2000/server/scriptguide/sas_ads_overview.mspx) auf der Microsoft TechNet-Website
-   [Script Center](https://www.microsoft.com/technet/community/scriptcenter/default.mspx) auf der Microsoft TechNet-Website
