# <a name="active-directory-service-interfaces-scripting"></a>Skripterstellung für Active Directory-Dienstschnittstellen

Die Microsoft Active Directory Service Interfaces (ADSI) stellen in Kombination mit einer Skriptsprache wie Microsoft Visual Basic Scripting Edition oder Microsoft JScript-Entwicklungssoftware einen Systemadministrator mit leistungsstarken Tools zum Verwalten von Benutzern, Diensten, Druckwarteschlangen und anderen Teilen eines Netzwerks von Computern bereit, auf denen Microsoft Windows NT- oder Windows 2000-Betriebssysteme ausgeführt wird. Skripts, die das ADSI-Objektmodell verwenden, können sich wiederholende manuelle Maßnahmen ersetzen, die normalerweise mithilfe von Verwaltungstools oder der Microsoft Management Console durchgeführt werden. ADSI-Skripts können auch reguläre Systemwartungsaufgaben für den freihändigen Betrieb automatisieren und bietet eine einfache Benutzeroberfläche zum Schreiben webbasierter Systemverwaltungstools mit Active Server Pages.

Weitere Informationen zur Skripterstellung mit ADSI mit Skripttutorials, Beispielen und Tools finden Sie unter:

-   [Script Center](https://www.microsoft.com/technet/community/scriptcenter/default.mspx) auf der Microsoft TechNet-Website
