# <a name="provider-minimum-requirements"></a>Mindestanforderungen für den Anbieter

Um einen ADSI-Anbieter zu schreiben, implementieren Sie die Active Directory Dienst Schnittstellen, wie in den folgenden Abschnitten beschrieben.

-   In der [Kern Implementierung](core-implementation.md) werden die Software Elemente, die ein ADSI-Anbieter unterstützen muss, kurz aufgeführt.
-   In der [optionalen Implementierung](optional-implementation.md) werden die Software Elemente, die von einem ADSI-Anbieter dringend unterstützt werden, kurz aufgeführt.
-   [Benutzerdefinierte Eigenschaften](custom-properties.md) sind die Eigenschaften und Schnittstellen, die von den einzelnen Anbietern verfügbar gemacht werden, um die Programmierbarkeit der systemeigenen Features des Verzeichnis Dienstanbieters
-   [Duale Schnittstellen](dual-interfaces.md) beschreiben die ADSI-Konvention zum Bereitstellen von Schnittstellen, die sowohl für die Automatisierung als auch für nicht Automation-kompatible Controller zugänglich sind.
