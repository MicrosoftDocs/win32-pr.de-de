# <a name="provider-minimum-requirements"></a>Mindestanforderungen für Anbieter

Implementieren Sie zum Schreiben eines ADSI-Anbieters die Active Directory-Dienstschnittstellen, wie in den folgenden Abschnitten beschrieben.

-   [Core Implementation](core-implementation.md) listet kurz die Softwareelemente auf, die ein ADSI-Anbieter unterstützen muss.
-   [Die optionale Implementierung](optional-implementation.md) listet kurz die Softwareelemente auf, die ein ADSI-Anbieter dringend unterstützen sollte.
-   [Benutzerdefinierte Eigenschaften](custom-properties.md) sind die Eigenschaften und Schnittstellen, die von den einzelnen Anbietern verfügbar gemacht werden, um die Programmierbarkeit der nativen Features des Verzeichnisdiensts bereitzustellen.
-   [Duale Schnittstellen](dual-interfaces.md) beschreiben die ADSI-Konvention zum Bereitstellen von Schnittstellen, auf die sowohl automation- als auch nicht automatisierungskonforme Controller zugreifen können.
