# <a name="setdefaultttl-method-of-the-win32_networkadapterconfiguration-class"></a>SetDefaultTTL-Methode der Win32 \_ NetworkAdapterConfiguration-Klasse

Die statische **Methode der SetDefaultTTL** [WMI-Klasse](/windows/desktop/WmiSdk/retrieving-a-class) wird verwendet, um den Standardwert für Die Gültigkeitsdauer (Time to Live, TTL) im Header ausgehender IP-Pakete festzulegen.

In diesem Thema wird die MOF-Syntax (Managed Object Format) verwendet. Weitere Informationen zur Verwendung dieser Methode finden Sie unter [Aufrufen einer Methode.](/windows/desktop/WmiSdk/calling-a-method)

## <a name="syntax"></a>Syntax


```mof
uint32 SetDefaultTTL(
  [in] uint8 DefaultTTL
);
```



## <a name="parameters"></a>Parameter

<dl> <dt>

*DefaultTTL* \[ In\]
</dt> <dd>

Der Wert für die Lebzeit, der im Header ausgehender IP-Pakete festgelegt ist. Der Standardwert ist 32. Gültiger Bereich: 1 bis 255

</dd> </dl>

## <a name="remarks"></a>Hinweise

Die Gültigkeitsdauer gibt die Anzahl der Router an, die ein IP-Paket passieren kann, um sein Ziel zu erreichen, bevor es verworfen wird. Jeder Router dekrementiert die TTL-Anzahl eines Pakets um eins und verwirft die Pakete mit einer Gültigkeitsdauer von 0 (null).

## <a name="examples"></a>Beispiele

Im [VBScript-Beispiel Ändern der Standardgültigkeitsdauer für alle Netzwerkadapter](https://Gallery.TechNet.Microsoft.Com/scriptcenter/3a228fb8-5517-4e23-800e-2a15f427d05d) wird **SetDefaultTTL** verwendet, um den Standardwert für die Gültigkeitsdauer im Header ausgehender IP-Pakete auf 64 festzulegen.
