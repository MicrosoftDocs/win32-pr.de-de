## <a name="root-parameters-and-arguments"></a><span data-ttu-id="7b146-111">Stamm Parameter und Argumente</span><span class="sxs-lookup"><span data-stu-id="7b146-111">Root parameters and arguments</span></span>

<span data-ttu-id="7b146-112">Eine Stamm **Signatur** ähnelt einer API-Funktions Signatur, Sie bestimmt die Typen von Daten, die von den Shadern erwartet werden, aber definiert nicht den tatsächlichen Arbeitsspeicher oder die tatsächlichen Daten.</span><span class="sxs-lookup"><span data-stu-id="7b146-112">A **root signature** is similar to an API function signature, it determines the types of data the shaders should expect, but does not define the actual memory or data.</span></span> <span data-ttu-id="7b146-113">Ein **Root-Parameter** ist ein Eintrag in der Stamm Signatur.</span><span class="sxs-lookup"><span data-stu-id="7b146-113">A **root parameter** is one entry in the root signature.</span></span> <span data-ttu-id="7b146-114">Die tatsächlichen Werte der Stamm Parameter, die zur Laufzeit festgelegt und geändert werden, werden als Stamm **Argumente** bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="7b146-114">The actual values of root parameters set and changed at runtime are called **root arguments**.</span></span> <span data-ttu-id="7b146-115">Durch das Ändern der Stamm Argumente werden die von den Shadern gelesenen Daten geändert.</span><span class="sxs-lookup"><span data-stu-id="7b146-115">Changing the root arguments changes the data that the shaders read.</span></span>

## <a name="root-constants-descriptors-and-tables"></a><span data-ttu-id="7b146-116">Stamm Konstanten, Deskriptoren und Tabellen</span><span class="sxs-lookup"><span data-stu-id="7b146-116">Root constants, descriptors and tables</span></span>

<span data-ttu-id="7b146-117">Die Stamm Signatur kann drei Typen von Parametern enthalten. Stamm Konstanten (in den root-Argumenten Inline-Konstanten), Stamm Deskriptoren (in den root-Argumenten Inline Deskriptoren) und deskriptortabellen (Zeiger auf einen Bereich von Deskriptoren im deskriptorheap).</span><span class="sxs-lookup"><span data-stu-id="7b146-117">The root signature can contain three types of parameters; root constants (constants inlined in the root arguments), root descriptors (descriptors inlined in the root arguments), and descriptor tables (pointers to a range of descriptors in the descriptor heap).</span></span>

<span data-ttu-id="7b146-118">Die Stamm Konstanten sind Inline-32-Bit-Werte, die im Shader als konstanter Puffer angezeigt werden.</span><span class="sxs-lookup"><span data-stu-id="7b146-118">The root constants are inline 32-bit values that show up in the shader as a constant buffer.</span></span>

<span data-ttu-id="7b146-119">Die Inline-root-Deskriptoren sollten Deskriptoren enthalten, auf die am häufigsten zugegriffen wird. Sie sind jedoch auf cbvs und RAW-oder strukturierte UAV-oder SRV-Puffer beschränkt.</span><span class="sxs-lookup"><span data-stu-id="7b146-119">The inlined root descriptors should contain descriptors that are accessed most often, though is limited to CBVs, and raw or structured UAV or SRV buffers.</span></span> <span data-ttu-id="7b146-120">Ein komplexerer Typ, z. b. ein 2D-Textur-SRV, kann nicht als Stamm Deskriptor verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="7b146-120">A more complex type, such as a 2D texture SRV, cannot be used as a root descriptor.</span></span> <span data-ttu-id="7b146-121">Stamm Deskriptoren enthalten keine Größenbeschränkung. es kann also keine Überprüfung außerhalb der Grenzen erfolgen, anders als bei Deskriptoren in deskriptorheaps, die eine Größe enthalten.</span><span class="sxs-lookup"><span data-stu-id="7b146-121">Root descriptors do not include a size limit, so there can be no out-of-bounds checking, unlike descriptors in descriptor heaps, which do include a size.</span></span>

<span data-ttu-id="7b146-122">Deskriptortabelleneinträge innerhalb von Stamm Signaturen enthalten den Deskriptor, den HLSL-Shader-Bindungs Namen und das Sichtbarkeits Kennzeichen.</span><span class="sxs-lookup"><span data-stu-id="7b146-122">Descriptor table entries within root signatures contain the descriptor, HLSL shader bind name and visibility flag.</span></span> <span data-ttu-id="7b146-123">Weitere Informationen zu shadernamen finden Sie unter [Shader-Modell 5,1](/windows/desktop/direct3dhlsl/shader-model-5-1) .</span><span class="sxs-lookup"><span data-stu-id="7b146-123">Refer to [Shader Model 5.1](/windows/desktop/direct3dhlsl/shader-model-5-1) for details of shader names.</span></span> <span data-ttu-id="7b146-124">Bei manchen Hardware kann es zu Leistungsvorteilen kommen, wenn Deskriptoren nur für die shaderphasen sichtbar gemacht werden müssen, die Sie benötigen (Weitere Informationen finden Sie unter [**D3D12 \_ Shader \_ Visibility**](/windows/desktop/api/d3d12/ne-d3d12-d3d12_shader_visibility)).</span><span class="sxs-lookup"><span data-stu-id="7b146-124">On some hardware, there can be a performance gain from only making descriptors visible to the shader stages that require them (refer to [**D3D12\_SHADER\_VISIBILITY**](/windows/desktop/api/d3d12/ne-d3d12-d3d12_shader_visibility)).</span></span>

![stammdeskriptortabelleneintrag](images/root-descriptor-table.png)

<span data-ttu-id="7b146-126">Das Layout der Stamm Signatur ist recht flexibel, wobei einige Einschränkungen auf weniger leistungsfähige Hardware angewendet werden.</span><span class="sxs-lookup"><span data-stu-id="7b146-126">The layout of the root signature is quite flexible, with some constraints imposed on less capable hardware.</span></span> <span data-ttu-id="7b146-127">Unabhängig von der Hardwareebene sollten Anwendungen immer versuchen, die Stamm Signatur so klein wie erforderlich zu machen, um eine maximale Effizienz zu erzielen.</span><span class="sxs-lookup"><span data-stu-id="7b146-127">Regardless of the level of hardware, applications should always try to make the root signature as small as needed for maximum efficiency.</span></span> <span data-ttu-id="7b146-128">Anwendungen können mit mehr deskriptortabellen in der Stamm Signatur, aber weniger Platz für Stamm Konstanten oder umgekehrt.</span><span class="sxs-lookup"><span data-stu-id="7b146-128">Applications can trade off having more descriptor tables in the root signature but less room for root constants, or vice versa.</span></span>

<span data-ttu-id="7b146-129">Der Inhalt der Stamm Signatur (die deskriptortabellen, Stamm Konstanten und Stamm Deskriptoren), die von der Anwendung gebunden wurden, wird vom D3D12-Treiber immer dann versioniert, wenn sich ein Teil der Inhalte zwischen Draw (graphics)/Dispatch (Compute)-aufrufen ändert.</span><span class="sxs-lookup"><span data-stu-id="7b146-129">The contents of the root signature (the descriptor tables, root constants and root descriptors) that the application has bound automatically get versioned by the D3D12 driver whenever any part of the contents change between draw (graphics)/dispatch (compute) calls.</span></span> <span data-ttu-id="7b146-130">Jede Zeichnung/Dispatch erhält also einen eindeutigen vollständigen Stamm Signatur Zustand.</span><span class="sxs-lookup"><span data-stu-id="7b146-130">So each draw/dispatch gets a unique full set of root signature state.</span></span>

<span data-ttu-id="7b146-131">Im Idealfall gibt es Gruppen von Pipeline Zustands Objekten (PSOs), die dieselbe Stamm Signatur gemeinsam verwenden.</span><span class="sxs-lookup"><span data-stu-id="7b146-131">Ideally, there are groups of Pipeline State Objects (PSOs) that share the same root signature.</span></span> <span data-ttu-id="7b146-132">Nachdem eine Stamm Signatur in der Pipeline festgelegt wurde, können alle Bindungen, die Sie definiert (deskriptortabellen, Deskriptoren, Konstanten), einzeln festgelegt oder geändert werden, einschließlich Vererbung in Bündel.</span><span class="sxs-lookup"><span data-stu-id="7b146-132">After a root signature is set on the pipeline, all the bindings that it defines (descriptor tables, descriptors, constants) can each be individually set or changed, including inheritance into bundles.</span></span>

<span data-ttu-id="7b146-133">Eine APP kann einen eigenen Kompromiss zwischen der Anzahl der zu lesenden deskriptortabellen in Inline Deskriptoren machen (die mehr Platz beanspruchen, aber eine Dereferenzierung aufheben), die Inline Konstanten (ohne Dereferenzierung) in der Stamm Signatur.</span><span class="sxs-lookup"><span data-stu-id="7b146-133">An app can make its own tradeoff between how many descriptor tables it wants verses inline descriptors (which take more space but remove an indirection) verses inline constants (which have no indirection) they want in the root signature.</span></span> <span data-ttu-id="7b146-134">Anwendungen sollten die Stamm Signatur so sparsam wie möglich verwenden und sich auf den von der Anwendung kontrollierten Arbeitsspeicher verlassen, z. b. Heaps und deskriptorheaps, die auf Sie verweisen, um Massendaten darzustellen.</span><span class="sxs-lookup"><span data-stu-id="7b146-134">Applications should use the root signature as sparingly as possible, relying on application controlled memory such as heaps and descriptor heaps pointing into them to represent bulk data.</span></span>
