# <a name="list-view-control-msaa-ui-element-reference"></a>List-View-Steuer Element (MSAA-Benutzeroberflächen Element-Referenz)

> [!Note]  
> In diesem Thema werden **Listenansicht-Steuer** Element Objekte für den MSAA-Benutzeroberflächen-Element Verweis beschrieben. Das Erstellen von **Listenansicht-Steuer** Element Objekten in verschiedenen Benutzeroberflächen-Frameworks wird hier nicht beschrieben. Weitere Informationen finden Sie in der API-Referenz Dokumentation für das von Ihnen verwendete UI-Framework.

 

Ein Listenansicht-Steuerelement zeigt eine Auflistung von Elementen an, die jeweils aus einem Symbol und einer Bezeichnung bestehen, und bietet mehrere Möglichkeiten, die Elemente anzuzeigen und anzuordnen. Beispielsweise werden in Spalten rechts neben dem Symbol und der Bezeichnung zusätzliche Informationen zu den einzelnen Elementen angezeigt. Windows-Explorer verwendet ein Listenansicht-Steuerelement, um Dateien anzuzeigen.

Der Fenster Klassenname für ein Listenansicht-Steuerelement ist \_ "WC ListView", das in "commctrl. h" als "syslistview" definiert ist.

## <a name="notes"></a>Notizen

-   Microsoft Active Accessibility macht Listen Ansichts Element Text in Anwendungen nicht verfügbar, die eine Rückruffunktion verwenden, um den Text dem Element hinzuzufügen.
-   Listen Ansichts Elemente, für die ein abgeschnitten durchgesetzt wird, verfügen nicht über eine nicht Sicht [**\_ \_ bares Zustands System**](object-state-constants.md) Um festzustellen, ob ein Listenelement abgeschnitten wird, können Clients die von [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) zurückgegebenen Koordinaten des Elements mit dem Ausschneide Bereich des Fensters vergleichen.
-   Microsoft Active Accessibility unterstützt nicht die Verwendung der [**accNavigate**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) -Methode, um zum Header Steuerelement in einem Listenansicht-Steuerelement in der Detailansicht zu navigieren. Clients verwenden jedoch die [**accessibleobjectfrompoint**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfrompoint) -Funktion, um zum Header Steuerelement zu gelangen.
-   Bei nicht rechteckigen Objekten, wie z. b. Listen Ansichts Elementen in der Detailansicht, schlagen die Koordinaten des umgebenden Rechtecks des Elements, das durch den [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) abgerufen wurde, fehl, wenn Sie mit [**accHitTest**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acchittest) getestet werden, da **accHitTest** die Grenzen des Objekts auf Pixel Basis bestimmt.
