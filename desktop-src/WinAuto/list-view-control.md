# <a name="list-view-control-msaa-ui-element-reference"></a>List-View-Steuerelement (REFERENZ ZUM MSAA-UI-Element)

> [!Note]  
> In diesem Thema werden **List-View Control-Objekte** für die MsAA-Benutzeroberflächenelementreferenz beschrieben. Das Erstellen von **List-View-Steuerelementobjekten** in verschiedenen Benutzeroberflächenframeworks wird hier nicht beschrieben. Weitere Informationen finden Sie in der API-Referenzdokumentation für das benutzeroberflächenframework, das Sie verwenden.

 

Ein Listenansicht-Steuerelement zeigt eine Sammlung von Elementen an, die jeweils aus einem Symbol und einer Bezeichnung bestehen, und bietet mehrere Möglichkeiten zum Anzeigen und Anordnen der Elemente. Beispielsweise werden zusätzliche Informationen zu jedem Element in Spalten rechts neben dem Symbol und der Bezeichnung angezeigt. Windows Explorer verwendet ein Listenansicht-Steuerelement, um Dateien anzuzeigen.

Der Fensterklassenname für ein Listenansicht-Steuerelement ist WC \_ LISTVIEW, das in Commctrl.h als "SysListView" definiert ist.

## <a name="notes"></a>Hinweise

-   Microsoft Active Accessibility macht keinen Listenansichtselementtext in Anwendungen verfügbar, die eine Rückruffunktion verwenden, um den Text zum Element hinzuzufügen.
-   Für Elemente in Listenansichten, die abgeschnitten werden, ist [**STATE \_ SYSTEM INVISIBLE \_ nicht**](object-state-constants.md) festgelegt. Um zu bestimmen, ob ein Listenelement abgeschnitten wird, können Clients die von [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) zurückgegebenen Koordinaten des Elements mit dem Ausschneidebereich des Fensters vergleichen.
-   Microsoft Active Accessibility die Verwendung der [**accNavigate-Methode**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) nicht unterstützt, um in einem Listenansicht-Steuerelement in der Detailansicht zum Header-Steuerelement zu navigieren. Clients verwenden jedoch die [**AccessibleObjectFromPoint-Funktion,**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfrompoint) um zum Headersteuerpunkt zu kommen.
-   Bei nicht rechteckigen Objekten wie Listenansichtselementen in der Detailansicht sind die Koordinaten des von [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) abgerufenen begrenzungsbasierten Rechtecks des Elements nicht erfolgreich, wenn sie mit [**accHitTest**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acchittest) getestet werden, da **accHitTest** die Objektgrenzen pixelgenau bestimmt.
