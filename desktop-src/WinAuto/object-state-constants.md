# <a name="object-state-constants"></a>Objekt Zustands Konstanten

In diesem Thema werden die Konstanten Werte beschrieben, die zum Beschreiben des Zustands von Objekten in einer Anwendungs Benutzeroberfläche verwendet werden. Die Zustands Konstanten werden in Oleacc. h definiert.

Ein Objekt ist einem oder mehreren dieser Zustands Werte zu einem beliebigen Zeitpunkt zugeordnet. Die folgenden Objekt Zustands Konstanten werden nicht verwendet: Zustands \_ System \_ Warnung hoch, Zustands System-Warn \_ \_ \_ \_ Mittel, Zustands \_ System \_ Warnung \_ niedrig und Zustands \_ System \_ ist nicht verankert.

Clients rufen den Zustand eines Objekts durch Aufrufen von [**IAccessible:: get \_ accState**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-get_accstate)ab, das eine ganze Zahl zurückgibt, die eine Kombination der folgenden Bitflags ist. Clients rufen [**getstatetext**](/windows/desktop/api/Oleacc/nf-oleacc-getstatetexta) mit dem State-Wert auf, um eine lokalisierte Zeichenfolge abzurufen, die den Zustand des Objekts beschreibt.

Wenn sich der Zustand eines Objekts ändert, sollten die Server [**NotifyWinEvent**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) mit der [**Ereignis Konstante " \_ \_ StateChange" des Ereignis Objekts**](event-constants.md) aufruft. Allerdings weisen Objekte mit dem Status \_ System nicht \_ sichtbar, das Zustands \_ System mit Fokus und das vom \_ Status \_ System \_ ausgewählte Objekt Zustands Konstanten eigene Ereignis Konstanten auf. Verwenden Sie für diese Objekte nicht das **Ereignis \_ Objekt \_ StateChange**. Verwenden Sie stattdessen die individuelle Ereignis Konstante.
