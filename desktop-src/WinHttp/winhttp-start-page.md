# <a name="windows-http-services"></a>Windows HTTP-Dienste

## <a name="purpose"></a>Zweck

Microsoft Windows HTTP Services (WinHTTP) bietet Entwicklern eine HTTP-Clientanwendungsprogrammierschnittstelle (API), um Anforderungen über das HTTP-Protokoll an andere HTTP-Server zu senden.

## <a name="where-applicable"></a>Anwendungsbereich

WinHTTP unterstützt Desktopclientanwendungen, Windows-Dienste und Windows serverbasierte Anwendungen.

Weitere Informationen zur Verwendung von WinHTTP für Anwendungen, die auf microsoft .NET Framework, finden Sie unter [WinHttpHandler-API.](/dotnet/api/system.net.http?view=netframework-4.8)

## <a name="developer-audience"></a>Entwicklergruppe

WinHTTP bietet sowohl eine C/C++-Api (Application Programming Interface) als auch eine com-Automatisierungskomponente (Component Object Model), die für die Verwendung in ASP-basierten Anwendungen (Active Server Pages) geeignet ist.

Ein grundlegendes Verständnis des HTTP-Protokolls ist wichtig, um beide Schnittstellen zu verwenden.

## <a name="run-time-requirements"></a>Laufzeitanforderungen

WinHTTP 5.1 bietet Verbesserungen gegenüber Version 5.0. Sie ist im Betriebssystem enthalten. Weitere Informationen zu neuen Features finden Sie unter Neues [in WinHTTP 5.1](what-s-new-in-winhttp-5-1.md) und Neues [in Windows Server 2008 und Windows Vista.](what-s-new-in-windows-longhorn.md)

> [!IMPORTANT]
> Der WinHTTP 5.0-Download ist nicht mehr verfügbar. Seit dem 1. Oktober 2004 hat Microsoft den WinHTTP 5.0 SDK-Download von MSDN entfernt und den Produktsupport für Version 5.0 beendet.
