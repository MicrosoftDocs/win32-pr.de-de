# <a name="windows-http-services"></a>Windows HTTP-Dienste

## <a name="purpose"></a>Zweck

Die Microsoft Windows HTTP-Dienste (WinHTTP) stellen Entwicklern eine HTTP-Client-API (Application Programming Interface) zum Senden von Anforderungen über das HTTP-Protokoll an andere HTTP-Server zur Verfügung.

## <a name="where-applicable"></a>Anwendungsbereich

WinHTTP unterstützt Desktop Client Anwendungen, Windows-Dienste und Windows Server-basierte Anwendungen.

Weitere Informationen zur Verwendung von WinHTTP für Anwendungen, die auf Microsoft .NET Framework basieren, finden Sie in der [winhttphandler-API](/dotnet/api/system.net.http?view=netframework-4.8) .

## <a name="developer-audience"></a>Entwicklergruppe

WinHTTP bietet sowohl eine C/C++-API (Application Programming Interface) als auch eine Component Object Model (com)-Automatisierungskomponente, die für die Verwendung in ASP-basierten Anwendungen (Active Server Pages) geeignet ist.

Ein grundlegendes Verständnis des HTTP-Protokolls ist wichtig, um beide Schnittstellen zu verwenden.

## <a name="run-time-requirements"></a>Laufzeitanforderungen

WinHTTP 5,1 bietet Verbesserungen gegenüber Version 5,0. Sie ist im Betriebssystem enthalten. Weitere Informationen zu neuen Features finden Sie unter neues [in WinHTTP 5,1](what-s-new-in-winhttp-5-1.md) und Neuerungen [in Windows Server 2008 und Windows Vista](what-s-new-in-windows-longhorn.md).

> [!IMPORTANT]
> Der WinHTTP 5,0-Download ist nicht mehr verfügbar. Seit dem 1. Oktober 2004 hat Microsoft das WinHTTP 5,0 SDK-Download von MSDN entfernt und den Produktsupport für Version 5,0 beendet.
