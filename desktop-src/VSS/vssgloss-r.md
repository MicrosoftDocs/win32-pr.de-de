# <a name="r-volume-shadow-copy-service"></a>R (Volumeschattenkopie-Dienst)

[a](vssgloss-a.md) [B](vssgloss-b.md) [C](vssgloss-c.md) [D](vssgloss-d.md) [E](vssgloss-e.md) [F](vssgloss-f.md) [G](vssgloss-g.md) [H](vssgloss-h.md) [I](vssgloss-i.md) J K [L](vssgloss-l.md) M [N](vssgloss-n.md) [O](vssgloss-o.md) [P](vssgloss-p.md) Q R [S](vssgloss-s.md) [T](vssgloss-t.md) U [V](vssgloss-v.md) [W](vssgloss-w.md) X Y Z

<dl> <dt>

<span id="base.vssgloss_requester"></span><span id="BASE.VSSGLOSS_REQUESTER"></span>**anfordernde Person**
</dt> <dd>

Mindestens ein Prozess, der mit VSS interagiert, um Schatten Kopien und schattenkopiesätze von einem oder mehreren Volumes zu erstellen und zu verwalten.

Normalerweise verwaltet eine anfordernde Person Schatten Kopien, um andere Funktionen zu unterstützen, z. b. Sicherungs-und Wiederherstellungs Vorgänge und Datenträger Spiegelung. Siehe auch [*Schatten Kopie*](vssgloss-s.md), [*Schattenkopiesatz*](vssgloss-s.md).

</dd> <dt>

<span id="base.vssgloss_restore_method"></span><span id="BASE.VSSGLOSS_RESTORE_METHOD"></span>**Restore-Methode**
</dt> <dd>

Eine Spezifikation der Wiederherstellungsprozess Methode. Es steuert solche Probleme, als ob Dateien bei der Wiederherstellung überschrieben werden. Wenn eine Wiederherstellungsmethoden Einstellung mit denen des Wiederherstellungs Ziels in Konflikt steht, hat das Wiederherstellungs Ziel Vorrang. Siehe auch *Restore Target*.

</dd> <dt>

<span id="base.vssgloss_restore_target"></span><span id="BASE.VSSGLOSS_RESTORE_TARGET"></span>**Wiederherstellungs Ziel**
</dt> <dd>

Unter VSS ist ein Wiederherstellungs Ziel eine Angabe, wie ein Anforderer eine Datei wiederherstellen soll, z. b. wenn vorhandene Dateien überschrieben werden sollen.

</dd> </dl>
