# <a name="restores-without-writer-participation"></a><span data-ttu-id="dbebb-103">Wiederherstellungen ohne Writer-Beteiligung</span><span class="sxs-lookup"><span data-stu-id="dbebb-103">Restores without Writer Participation</span></span>

<span data-ttu-id="dbebb-104">Die Writer-Teilnahme an einer VSS-Sicherung ist so konzipiert, dass Anwendungen steuern können, was und wie Ihre Wiederherstellungsdaten verwendet werden sollen.</span><span class="sxs-lookup"><span data-stu-id="dbebb-104">Writer participation in a VSS backup is designed to allow applications to control what and how their restore data is to be used.</span></span>

<span data-ttu-id="dbebb-105">Wenn ein Writer auf einem System verfügbar ist, empfiehlt es sich im Allgemeinen nicht, Daten an Ihrem ursprünglichen Speicherort ohne Writer-Beteiligung wiederherzustellen.</span><span class="sxs-lookup"><span data-stu-id="dbebb-105">In general, if a writer is available on a system, it is never advisable to restore data to its original location without writer participation.</span></span> <span data-ttu-id="dbebb-106">Bei einer solchen Wiederherstellung werden wahrscheinlich gesperrte Zieldateien angezeigt, und es besteht ein erhebliches Risiko, dass Daten beschädigt werden.</span><span class="sxs-lookup"><span data-stu-id="dbebb-106">Such a restore would likely encounter locked destination files and runs a significant risk of corrupting data.</span></span>

<span data-ttu-id="dbebb-107">Es gibt jedoch Gründe, warum eine Sicherungs Anwendung möglicherweise eine VSS-Sicherung ohne Writer-Beteiligung wiederherstellen will oder benötigt:</span><span class="sxs-lookup"><span data-stu-id="dbebb-107">However, there are reasons why a backup application might want or need to restore a VSS backup without writer participation:</span></span>

-   <span data-ttu-id="dbebb-108">Daten werden von VSS-abhängigen Anwendungen verwaltet.</span><span class="sxs-lookup"><span data-stu-id="dbebb-108">Data is managed by VSS-unaware applications.</span></span> <span data-ttu-id="dbebb-109">Fast jedes System verfügt über einige Anwendungen – Texteditoren, e-Mail-Reader, Word-Prozessoren usw. –, die VSS nicht kennen.</span><span class="sxs-lookup"><span data-stu-id="dbebb-109">Almost every system will have some applications—text editors, mail readers, word processors, and so forth—that are VSS unaware.</span></span> <span data-ttu-id="dbebb-110">Diese Daten können nicht mithilfe der Writer-Teilnahme wieder hergestellt werden.</span><span class="sxs-lookup"><span data-stu-id="dbebb-110">This data cannot be restored using writer participation.</span></span>

    <span data-ttu-id="dbebb-111">Im Allgemeinen ist diese Art von Daten nicht System-oder Dienst kritisch, und die Wiederherstellung sollte nicht problematisch oder zumindest bei einer herkömmlichen Wiederherstellung nicht problematisch sein.</span><span class="sxs-lookup"><span data-stu-id="dbebb-111">Generally, this type of data is not system- or service-critical, and restoring it should not be problematic, or at least no more problematic than during a conventional restore.</span></span>

    <span data-ttu-id="dbebb-112">Wie bei der Vorbereitung für konventionelle Wiederherstellungen sollten Wiederherstellungs Operatoren nach Möglichkeit versuchen, solche Anwendungen anzuhalten oder zu beenden, bevor eine VSS-Wiederherstellung gestartet wird.</span><span class="sxs-lookup"><span data-stu-id="dbebb-112">As with preparations for conventional restores, if possible, restore operators should attempt to suspend or terminate such applications prior to starting a VSS restore.</span></span>

-   <span data-ttu-id="dbebb-113">Fehlende VSS-Writer.</span><span class="sxs-lookup"><span data-stu-id="dbebb-113">Missing VSS writers.</span></span> <span data-ttu-id="dbebb-114">Diese Situation kann relativ häufig vorkommen, wenn der Zustand eines beschädigten Systems wieder hergestellt wird.</span><span class="sxs-lookup"><span data-stu-id="dbebb-114">This situation may be fairly common when restoring the state of a damaged system.</span></span> <span data-ttu-id="dbebb-115">Ein Sicherungs Vorgang muss bestimmen, ob es wünschenswert ist, Dateien für fehlende Writer wiederherzustellen.</span><span class="sxs-lookup"><span data-stu-id="dbebb-115">A backup operation must determine whether it is desirable to restore files for missing writers.</span></span> <span data-ttu-id="dbebb-116">Wenn die Wiederherstellung wünschenswert ist, können die Dateien genau wie bei einer herkömmlichen Sicherung wieder hergestellt werden.</span><span class="sxs-lookup"><span data-stu-id="dbebb-116">If restoration is desirable, the files can be restored just as a conventional backup would restore them.</span></span>
-   <span data-ttu-id="dbebb-117">Eine private Wiederherstellung der Daten eines Writers.</span><span class="sxs-lookup"><span data-stu-id="dbebb-117">A private restore of a writer's data.</span></span> <span data-ttu-id="dbebb-118">Ein Anforderer kann sich entscheiden, die Daten eines laufenden Writers an einem privaten Speicherort wiederherzustellen, ohne den Writer zu benachrichtigen.</span><span class="sxs-lookup"><span data-stu-id="dbebb-118">A requester may choose to restore the data of a running writer to some private location without notifying the writer.</span></span> <span data-ttu-id="dbebb-119">Ein Beispiel hierfür ist die Wiederherstellung der Writer-Daten zur Unterstützung des Offline Vergleichs.</span><span class="sxs-lookup"><span data-stu-id="dbebb-119">An example of this might be restoration of the writer's data to support offline comparison.</span></span> <span data-ttu-id="dbebb-120">In dieser Situation möchte ein Anforderer nicht den [*neuen Zielort*](vssgloss-n.md) bei der Wiederherstellung verwenden, da er nicht möchte, dass der Writer auf die Daten zugreift.</span><span class="sxs-lookup"><span data-stu-id="dbebb-120">In this sort of situation, a requester would not want to use the [*new target location*](vssgloss-n.md) while doing the restore, because it does not want the writer to access the data.</span></span>
-   <span data-ttu-id="dbebb-121">Ein Writer möchte bei der Wiederherstellung nicht einbezogen werden.</span><span class="sxs-lookup"><span data-stu-id="dbebb-121">A writer does not want to be involved during restore.</span></span> <span data-ttu-id="dbebb-122">Ein Writer zeigt dies an, indem er VSS \_ WRE \_ Never für den " *Write Restore* "-Parameter von [**ivsskreateschreitermetadata:: "-trestoremethod**](/windows/desktop/api/VsWriter/nf-vswriter-ivsscreatewritermetadata-setrestoremethod)" übergibt.</span><span class="sxs-lookup"><span data-stu-id="dbebb-122">A writer indicates this by passing in VSS\_WRE\_NEVER for the *writerRestore* parameter of [**IVssCreateWriterMetadata::SetRestoreMethod**](/windows/desktop/api/VsWriter/nf-vswriter-ivsscreatewritermetadata-setrestoremethod).</span></span>
-   <span data-ttu-id="dbebb-123">Ein Writer erfordert eine benutzerdefinierte Wiederherstellungsmethode.</span><span class="sxs-lookup"><span data-stu-id="dbebb-123">A writer requires a custom restore method.</span></span> <span data-ttu-id="dbebb-124">Ein Writer gibt an, dass er eine benutzerdefinierte Wiederherstellung erfordert, indem er VSS \_ RME \_ Custom für den *Methoden* Parameter von [**ivsskreateschreitermetadata:: abtrestoremethod**](/windows/desktop/api/VsWriter/nf-vswriter-ivsscreatewritermetadata-setrestoremethod)übergibt.</span><span class="sxs-lookup"><span data-stu-id="dbebb-124">A writer indicates that it requires a custom restore by passing in VSS\_RME\_CUSTOM for the *method* parameter of [**IVssCreateWriterMetadata::SetRestoreMethod**](/windows/desktop/api/VsWriter/nf-vswriter-ivsscreatewritermetadata-setrestoremethod).</span></span> <span data-ttu-id="dbebb-125">In diesem Fall sollte dieser Writer nicht an dem Wiederherstellungs Vorgang beteiligt sein, es sei denn, die Dokumentation zur benutzerdefinierten Wiederherstellung für diesen Writer zeigt andernfalls an.</span><span class="sxs-lookup"><span data-stu-id="dbebb-125">In this case, this writer should not be involved in the restore process unless the custom-restore documentation for that writer indicates otherwise.</span></span>

<span data-ttu-id="dbebb-126">Ein Anforderer bezieht einen Writer in den Wiederherstellungsprozess ein, indem er eine dieser Writer-Komponenten in einem Aufrufen von [**IVssBackupComponents:: setselectedforrestore**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssbackupcomponents-setselectedforrestore)angibt.</span><span class="sxs-lookup"><span data-stu-id="dbebb-126">A requester involves a writer in the restore process by specifying one of that writer's components in a call to [**IVssBackupComponents::SetSelectedForRestore**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssbackupcomponents-setselectedforrestore).</span></span> <span data-ttu-id="dbebb-127">Die Daten eines Writers können wieder hergestellt werden, ohne dass Sie den Writer einbeziehen, indem Sie einfach keine der Komponenten dieses Writers in einem Aufrufen von **IVssBackupComponents:: setselectedforrestore** angeben.</span><span class="sxs-lookup"><span data-stu-id="dbebb-127">A writer's data can be restored without involving the writer by simply not specifying any of that writer's components in a call to **IVssBackupComponents::SetSelectedForRestore**.</span></span> <span data-ttu-id="dbebb-128">Wenn ein Writer keine Wiederherstellungs Ereignisse erwartet, kann dies dazu führen, dass für diesen Writer falsche Fehler gemeldet werden.</span><span class="sxs-lookup"><span data-stu-id="dbebb-128">If a writer does not expect any restore events, involving that writer in the restore process can cause spurious errors to be reported for that writer.</span></span>
