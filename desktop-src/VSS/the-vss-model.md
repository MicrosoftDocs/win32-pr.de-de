# <a name="the-vss-model"></a><span data-ttu-id="44f4e-103">Das VSS-Modell</span><span class="sxs-lookup"><span data-stu-id="44f4e-103">The VSS Model</span></span>

<span data-ttu-id="44f4e-104">VSS dient der Behebung der Probleme, die unter Häufige Probleme mit der [Volumesicherung](common-volume-backup-issues.md)beschrieben werden.</span><span class="sxs-lookup"><span data-stu-id="44f4e-104">VSS is designed to address the problems described in [Common Volume Backup Issues](common-volume-backup-issues.md).</span></span>

<span data-ttu-id="44f4e-105">Das VSS-Modell umfasst Folgendes:</span><span class="sxs-lookup"><span data-stu-id="44f4e-105">The VSS model includes the following:</span></span>

-   <span data-ttu-id="44f4e-106">Der schattenkopiemechanismus.</span><span class="sxs-lookup"><span data-stu-id="44f4e-106">The shadow copy mechanism.</span></span> <span data-ttu-id="44f4e-107">VSS bietet eine schnelle volumeerfassung des Zustands eines Datenträgers zu einem Zeitpunkt – eine [*Schatten Kopie*](vssgloss-s.md) des Volumes.</span><span class="sxs-lookup"><span data-stu-id="44f4e-107">VSS provides fast volume capture of the state of a disk at one instant in time—a [*shadow copy*](vssgloss-s.md) of the volume.</span></span> <span data-ttu-id="44f4e-108">Diese Volumekopie ist parallel zum Live Volume vorhanden und enthält Kopien aller Dateien auf dem Datenträger, die effektiv gespeichert und als separates Gerät verfügbar sind.</span><span class="sxs-lookup"><span data-stu-id="44f4e-108">This volume copy exists side by side with the live volume, and contains copies of all files on disk effectively saved and available as a separate device.</span></span>
-   <span data-ttu-id="44f4e-109">Einheitlicher Datei Zustand über Anwendungs Koordination.</span><span class="sxs-lookup"><span data-stu-id="44f4e-109">Consistent file state via application coordination.</span></span> <span data-ttu-id="44f4e-110">VSS bietet einen com-basierten, ereignisgesteuerten Prozess für prozessübergreifende Kommunikation, mit dem teilnehmende Prozesse den Systemstatus in Bezug auf Sicherungs-, Wiederherstellungs-und schattenkopievorgänge (volumeerfassung) bestimmen können.</span><span class="sxs-lookup"><span data-stu-id="44f4e-110">VSS provides a COM-based, event-driven interprocess communication mechanism that participating processes can use to determine system state with respect to backup, restore, and shadow copy (volume capture) operations.</span></span> <span data-ttu-id="44f4e-111">Diese Ereignisse definieren Phasen, nach denen Anwendungen, die Daten auf einem Datenträger ([*Writer*](vssgloss-w.md)) ändern, alle Ihre Dateien vor der Erstellung der Schatten Kopie in einen konsistenten Zustand bringen können.</span><span class="sxs-lookup"><span data-stu-id="44f4e-111">These events define stages by which applications modifying data on disk ([*writers*](vssgloss-w.md)) can bring all their files into a consistent state prior to the creation of the shadow copy.</span></span>
-   <span data-ttu-id="44f4e-112">Minimierung der Ausfallzeiten von Anwendungen.</span><span class="sxs-lookup"><span data-stu-id="44f4e-112">Minimizing application downtime.</span></span> <span data-ttu-id="44f4e-113">Die VSS-Schatten Kopie ist parallel mit einer Live Kopie des zu sichernden Volumes vorhanden. mit Ausnahme der kurzen Zeitspanne der Vorbereitung und Erstellung des Schatten Kopierens kann eine Anwendung ihre Arbeit fortsetzen.</span><span class="sxs-lookup"><span data-stu-id="44f4e-113">The VSS shadow copy exists in parallel with a live copy of the volume to be backed up, so except for the brief period of the shadow copy's preparation and creation, an application can continue its work.</span></span> <span data-ttu-id="44f4e-114">Die Zeit, die erforderlich ist, um eine Schatten Kopie zu erstellen, die zwischen [*Freeze*](vssgloss-f.md) -und [*Thaw*](vssgloss-t.md) -Ereignissen auftritt, dauert in der Regel ungefähr eine Minute.</span><span class="sxs-lookup"><span data-stu-id="44f4e-114">The time needed to actually create a shadow copy, which occurs between [*Freeze*](vssgloss-f.md) and [*Thaw*](vssgloss-t.md) events, typically takes about one minute.</span></span>

    <span data-ttu-id="44f4e-115">Während der Vorbereitung eines Writers auf eine Schatten Kopie, einschließlich leeren von e/a-Vorgängen und Speichern von Zuständen (siehe [Übersicht über Aufgaben vor der Sicherung](overview-of-pre-backup-tasks.md)), ist möglicherweise nicht trivial, Sie ist deutlich kürzer als die Zeit, die für die eigentliche Sicherung eines Volumes erforderlich ist – für große Volumes kann es mehrere Stunden dauern.</span><span class="sxs-lookup"><span data-stu-id="44f4e-115">While a writer's preparation for a shadow copy, including flushing I/O and saving state (see [Overview of Pre-Backup Tasks](overview-of-pre-backup-tasks.md)), may be nontrivial, it is significantly shorter than the time required to actually back up a volume—which for large volumes may take hours.</span></span>

-   <span data-ttu-id="44f4e-116">Vereinheitlichte Schnittstelle zu VSS.</span><span class="sxs-lookup"><span data-stu-id="44f4e-116">Unified interface to VSS.</span></span> <span data-ttu-id="44f4e-117">VSS abstrahiert die schattenkopiemechanismen innerhalb einer gemeinsamen Oberfläche und ermöglicht einem Hardwarehersteller das Hinzufügen und Verwalten der eindeutigen Features seiner eigenen Anbieter.</span><span class="sxs-lookup"><span data-stu-id="44f4e-117">VSS abstracts the shadow copy mechanisms within a common interface while enabling a hardware vendor to add and manage the unique features of its own providers.</span></span> <span data-ttu-id="44f4e-118">Alle Sicherungs Anwendungen ([*Anforderer*](vssgloss-r.md)) und alle Writer sollten in der Lage sein, auf jedem Datenträger-Speichersystem auszuführen, das die VSS-Schnittstelle unterstützt.</span><span class="sxs-lookup"><span data-stu-id="44f4e-118">Any backup application ([*requester*](vssgloss-r.md)) and any writer should be able to run on any disk storage system that supports the VSS interface.</span></span>
-   <span data-ttu-id="44f4e-119">Multivolumesicherung.</span><span class="sxs-lookup"><span data-stu-id="44f4e-119">Multivolume backup.</span></span> <span data-ttu-id="44f4e-120">VSS unterstützt [*schattenkopiesätze*](vssgloss-s.md), bei denen es sich um Sammlungen von Schatten Kopien handelt, über mehrere Arten von Datenträgervolumes von mehreren Anbietern</span><span class="sxs-lookup"><span data-stu-id="44f4e-120">VSS supports [*shadow copy sets*](vssgloss-s.md), which are collections of shadow copies, across multiple types of disk volumes from multiple vendors.</span></span> <span data-ttu-id="44f4e-121">Alle Schatten Kopien in einem Schattenkopiesatz werden mit dem gleichen Zeitstempel erstellt und weisen für einen Datenträger Status mit mehreren Datenträgern denselben Datenträger Status auf.</span><span class="sxs-lookup"><span data-stu-id="44f4e-121">All shadow copies in a shadow copy set will be created with the same time stamp and will present the same disk state for a multivolume disk state.</span></span>
-   <span data-ttu-id="44f4e-122">Unterstützung für Native Schatten Kopien.</span><span class="sxs-lookup"><span data-stu-id="44f4e-122">Native shadow copy support.</span></span> <span data-ttu-id="44f4e-123">Ab Windows XP ist die Unterstützung von Schatten Kopien über VSS als System eigener Teil des Windows-Betriebssystems verfügbar.</span><span class="sxs-lookup"><span data-stu-id="44f4e-123">Beginning with Windows XP, shadow copy support is available through VSS as a native part of the Windows operating system.</span></span> <span data-ttu-id="44f4e-124">Solange mindestens ein NTFS-Datenträger auf einem System vorhanden ist, können diese Systeme so konfiguriert werden, dass Schatten Kopien aller auf Ihnen bereitgestellten Festplattensysteme unterstützt werden.</span><span class="sxs-lookup"><span data-stu-id="44f4e-124">As long as at least one NTFS disk is present on a system, these systems can be configured to support shadow copies of all disk systems mounted on them.</span></span>
