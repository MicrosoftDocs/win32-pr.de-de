# <a name="about-basic-debugging"></a>Grundlegendes zum Debuggen

Die [Debugfunktionen können](debugging-functions.md) verwendet werden, um einen einfachen, ereignisgesteuerten Debugger zu erstellen. *Ereignisgesteuert bedeutet,* dass der Debugger jedes Mal benachrichtigt wird, wenn bestimmte Ereignisse in dem Prozess auftreten, der debuggt wird. Mit der Benachrichtigung kann der Debugger entsprechende Aktionen als Reaktion auf die Ereignisse ausführen.

Eine Übersicht über Debugbegriffe finden Sie unter [Debugterminologie](debugging-terminology.md).

[Debugunterstützung von Prozess-, Thread-](debug-support-from-process-thread-and-exception-functions.md) und Ausnahmefunktionen beschreibt die debugspezifischen Features bestimmter Prozess-, Thread- und Ausnahmebehandlungsfunktionen.

[Erstellen eines Einfachdebuggers](creating-a-basic-debugger.md) beschreibt die Verwendung der grundlegenden Debugfunktionen zum Erstellen eines einfachen Debuggers. Diese Funktionen ermöglichen es einer Anwendung, auf Debugereignisse zu warten, Breakpointausnahmen zu verursachen, die Ausführungssteuerung an den Debugger zu übertragen und so weiter.

[Debugereignisse](debugging-events.md) beschreibt den Debugereignismechanismus. Debugereignisse bewirken, dass das System den Debugger benachrichtigt.
