# <a name="microsoft-secure-audio-path-deprecated"></a>Microsoft Secure Audio Path (Sicherer Microsoft-Audiopfad) (veraltet)

Auf dieser Seite wird ein Feature dokumentiert, das in zukünftigen Versionen von Windows mit einer anderen technischen Lösung unterstützt wird.

Inhaltsersteller und Verteiler können in einer DRM-Lizenz angeben, dass eine Audiodatei nur auf einem System mit Microsoft Secure Audio Path-Komponenten (SAP) wiedergegeben werden darf. Secure Audio Path bietet einen viel höheren Schutz für Audioinhalte, da es für nicht vertrauenswürdige Anwendungen oder Audiotreiber praktisch unmöglich wird, auf die unverschlüsselten Audiobits zuzugreifen. Sicherer Audiopfad wird auf Microsoft Windows® Me und Windows XP unterstützt. Sie sichert digitale Musik im Betriebssystemkernel. Darüber hinaus macht das Digital Copyright Act das Umgehen von Antimaßnahmen in Software zu einem Diebstahl.

Secure Audio Path wird von der Microsoft DRM-Komponente vollständig automatisch aktiviert und implementiert, wenn eine DRM-Lizenz SAP erfordert. Für Anwendungen, die auf Windows® XP Service Pack 1 ausgeführt werden, können Sie die SAP-Verschlüsselung für alle Audioinhalte außerhalb des Kontexts der Microsoft DRM-Lösung aktivieren, indem Sie das SECURE Audio Path SDK verwenden. Weitere Informationen zum Secure Audio Path SDK finden Sie auf der [Microsoft-Website](/documentation/).
