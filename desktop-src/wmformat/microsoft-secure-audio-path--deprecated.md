# <a name="microsoft-secure-audio-path-deprecated"></a>Microsoft Secure-Audiopfad (veraltet)

Diese Seite dokumentiert eine Funktion, die in zukünftigen Versionen von Windows mit einer anderen technischen Lösung unterstützt wird.

Inhalts Ersteller und-Verteiler können in einer DRM-Lizenz angeben, dass eine Audiodatei nur auf einem System mit Microsoft Secure-Audiopfad-Komponenten (SAP) abgespielt werden darf. Der sichere Audiopfad bietet ein viel höheres Maß an Schutz für Audioinhalte, da nicht vertrauenswürdige Anwendungen oder Audiotreibern nicht auf unverschlüsselte audiobits zugreifen können. Ein sicherer Audiopfad wird unter Microsoft Windows® Me und Windows XP unterstützt. Es sichert digitale Musik im Betriebssystem-Kernel. Außerdem ermöglicht das Digital Millennium-Urheberrechtsgesetz das umgehen von Antipiraterie-Measures in Software a Crime.

Der sichere Audiopfad wird von der Microsoft DRM-Komponente vollständig automatisch aktiviert und implementiert, wenn eine DRM-Lizenz SAP benötigt. Für Anwendungen, die unter Windows® XP Service Pack 1 ausgeführt werden, können Sie die SAP-Verschlüsselung für beliebige Audioinhalte außerhalb des Kontexts der Microsoft DRM-Lösung mit dem Secure-Audiopfad-SDK aktivieren. Weitere Informationen zum Secure audiopath SDK finden Sie auf der [Microsoft-Website](/documentation/).
