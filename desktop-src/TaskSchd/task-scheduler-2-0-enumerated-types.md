# <a name="task-scheduler-20-enumerated-types"></a>Taskplaner 2.0-Enumerationstypen

Beschreibt die enumerationsierten Typen, die die Konstanten definieren, die von den Taskplaner 2.0-APIs verwendet werden.


Die folgenden enumerationsierten Typen werden in Taskplaner 2.0 eingeführt.



| Enumeration                                                                  | Beschreibung                                                                                                      |
|------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------|
| [**\_ \_ TASKAKTIONSTYP**](/windows/desktop/api/taskschd/ne-taskschd-task_action_type)                               | Definiert den Typ der Aktionen, die eine Aufgabe ausführen kann.                                                             |
| [**\_TASKKOMPATIBILITÄT**](/windows/desktop/api/taskschd/ne-taskschd-task_compatibility)                            | Definiert, mit welchen Versionen von Taskplaner oder dem AT-Befehl der Task kompatibel ist.                      |
| [**\_TASKERSTELLUNG**](/windows/desktop/api/taskschd/ne-taskschd-task_creation)                                       | Definiert, wie der Taskplaner Dienst die Aufgabe erstellt, aktualisiert oder deaktiviert.                                   |
| [**\_ \_ TASK-ENUMERATIONSFLAGS**](/windows/desktop/api/taskschd/ne-taskschd-task_enum_flags)                                 | Definiert, wie die Taskplaner registrierte Aufgaben aufzählt.                                              |
| [**\_TASKINSTANZRICHTLINIE \_**](/windows/desktop/api/taskschd/ne-taskschd-task_instances_policy)                     | Definiert, wie der Taskplaner vorhandene Instanzen des Tasks behandelt, wenn eine neue Instanz des Tasks gestartet wird. |
| [**\_TYP DER TASKANMELDUNG**](/windows/desktop/api/taskschd/ne-taskschd-task_logon_type)                                  | Definiert, welche Anmeldetechnik zum Ausführen einer Aufgabe erforderlich ist.                                                          |
| [**TASK \_ PROCESSTOKENSID-TYP \_**](/windows/win32/api/taskschd/ne-taskschd-task_processtokensid_type)             | Definiert die Typen der Prozess-SID, die von Tasks verwendet werden können.                                                      |
| [**TASK \_ RUN \_ FLAGS**](/windows/desktop/api/taskschd/ne-taskschd-task_run_flags)                                   | Definiert, wie eine Aufgabe ausgeführt wird.                                                                                       |
| [**TASK \_ RUNLEVEL \_ TYPE**](/windows/win32/api/taskschd/ne-taskschd-task_runlevel_type)                           | Definiert LUA-Erhöhungsflags, die angeben, mit welcher Berechtigungsstufe der Task ausgeführt wird.                         |
| [**\_ \_ \_ ÄNDERUNGSTYP DES TASKSITZUNGSZUSTANDS \_**](/windows/desktop/api/taskschd/ne-taskschd-task_session_state_change_type) | Definiert, welche Arten von Terminalserver-Sitzungszustandsänderungen Sie verwenden können, um einen Task zum Starten auszulösen.               |
| [**\_TASKSTATUS**](/windows/desktop/api/taskschd/ne-taskschd-task_state)                                            | Definiert die verschiedenen Zustände, in denen sich ein registrierter Task anmelden kann.                                                   |
| [**\_ \_ TASKTRIGGERTYP2**](/windows/desktop/api/taskschd/ne-taskschd-task_trigger_type2)                                  | Definiert den Typ von Triggern, die von Tasks verwendet werden können.                                                          |
