<span data-ttu-id="65753-113">Tracelogging ist auch mit vorhandenen Tools kompatibel.</span><span class="sxs-lookup"><span data-stu-id="65753-113">TraceLogging is also compatible with existing tools.</span></span> <span data-ttu-id="65753-114">Anbieter, die Manifest-basiertes etw verwenden, werden weiterhin unterstützt.</span><span class="sxs-lookup"><span data-stu-id="65753-114">Providers that use manifest-based ETW are still supported.</span></span> <span data-ttu-id="65753-115">Sie müssen keine manifestressourcenbasierten ETW-Anbieter in tracelogging-Anbieter konvertieren, es sei denn, Sie möchten die Einfachheit, die tracelogging bietet, nutzen.</span><span class="sxs-lookup"><span data-stu-id="65753-115">You do not need to convert manifest based ETW providers to TraceLogging providers unless you wish to take advantage of the simplicity that TraceLogging provides.</span></span> <span data-ttu-id="65753-116">Die WPP-Ablauf Verfolgung wird auch weiterhin unterstützt.</span><span class="sxs-lookup"><span data-stu-id="65753-116">WPP tracing is also still supported.</span></span>

-   <span data-ttu-id="65753-119">Ereignisse sind selbst beschreibend und erfordern keine zusätzlichen Binärdateien, die das kompilierte Ereignis Manifest enthalten, um das Ereignis zu interpretieren.</span><span class="sxs-lookup"><span data-stu-id="65753-119">Events are self-describing and do not require any additional binaries containing the compiled event manifest to interpret the event.</span></span> <span data-ttu-id="65753-120">Alle Metadaten über das Ereignis werden im Ereignis aufgezeichnet.</span><span class="sxs-lookup"><span data-stu-id="65753-120">All the metadata about the event is recorded in the event.</span></span>
