# <a name="x509-digital-certification"></a><span data-ttu-id="0cee8-104">Digitale X.509-Zertifizierung</span><span class="sxs-lookup"><span data-stu-id="0cee8-104">X.509 Digital Certification</span></span>

<span data-ttu-id="0cee8-105">Eine primäre Aufgabe eines [*digitalen Zertifikats*](../secgloss/d-gly.md) besteht darin, den Zugriff auf den [*öffentlichen Schlüssel*](../secgloss/p-gly.md)des Antragstellers bereitzustellen.</span><span class="sxs-lookup"><span data-stu-id="0cee8-105">A primary task of a [*digital certificate*](../secgloss/d-gly.md) is to provide access to the subject's [*public key*](../secgloss/p-gly.md).</span></span> <span data-ttu-id="0cee8-106">Das Zertifikat bestätigt auch, dass der öffentliche Schlüssel des Zertifikats zum Betreff des Zertifikats gehört.</span><span class="sxs-lookup"><span data-stu-id="0cee8-106">The certificate also confirms that the certificate's public key belongs to the certificate's subject.</span></span> <span data-ttu-id="0cee8-107">Eine Zertifizierungsstelle ( [*Certification Authority*](../secgloss/c-gly.md) , ca) kann z. b. eine spezielle Nachricht (die Zertifikat Informationen) digital signieren, die den Namen eines Benutzers enthält, z. b. "Alice" und ihren öffentlichen Schlüssel.</span><span class="sxs-lookup"><span data-stu-id="0cee8-107">For example, a [*certification authority*](../secgloss/c-gly.md) (CA) can digitally sign a special message (the certificate information) that contains the name of some user, say "Alice," and her public key.</span></span> <span data-ttu-id="0cee8-108">Dies muss so erfolgen, dass jeder Benutzer überprüfen kann, ob das Zertifikat ausgestellt und von keinem anderen als der Zertifizierungsstelle signiert wurde.</span><span class="sxs-lookup"><span data-stu-id="0cee8-108">This must be done in such a way that anyone can verify that the certificate was issued and signed by no one other than the CA.</span></span> <span data-ttu-id="0cee8-109">Wenn die Zertifizierungsstelle vertrauenswürdig ist und überprüft werden kann, ob das Zertifikat von Alice von dieser Zertifizierungsstelle ausgestellt wurde, kann jeder Empfänger von Alice das Zertifikat für den öffentlichen Schlüssel von Alice von diesem Zertifikat vertrauen.</span><span class="sxs-lookup"><span data-stu-id="0cee8-109">If the CA is trusted and it can be verified that Alice's certificate was issued by that CA, any receiver of Alice's certificate can trust Alice's public key from that certificate.</span></span>

<span data-ttu-id="0cee8-110">Die typische Implementierung der digitalen Zertifizierung umfasst einen Prozess zum Signieren des Zertifikats.</span><span class="sxs-lookup"><span data-stu-id="0cee8-110">The typical implementation of digital certification involves a process for signing the certificate.</span></span>

<span data-ttu-id="0cee8-111">Der Prozess sieht folgendermaßen aus:</span><span class="sxs-lookup"><span data-stu-id="0cee8-111">The process goes like this:</span></span>

1.  <span data-ttu-id="0cee8-112">Alice sendet eine signierte [*Zertifikat Anforderung*](../secgloss/c-gly.md) mit Ihrem Namen, Ihrem öffentlichen Schlüssel und möglicherweise zusätzlichen Informationen an eine Zertifizierungsstelle.</span><span class="sxs-lookup"><span data-stu-id="0cee8-112">Alice sends a signed [*certificate request*](../secgloss/c-gly.md) containing her name, her public key, and perhaps some additional information to a CA.</span></span>
2.  <span data-ttu-id="0cee8-113">Die Zertifizierungsstelle erstellt die Nachricht *m* aus der Anforderung von Alice.</span><span class="sxs-lookup"><span data-stu-id="0cee8-113">The CA creates a message, *m*, from Alice's request.</span></span> <span data-ttu-id="0cee8-114">Die Zertifizierungsstelle signiert die Nachricht mit Ihrem privaten Schlüssel und erstellt eine separate Signatur Nachricht ( *sig*).</span><span class="sxs-lookup"><span data-stu-id="0cee8-114">The CA signs the message with its private key, creating a separate signature message, *sig*.</span></span> <span data-ttu-id="0cee8-115">Die Zertifizierungsstelle gibt die Nachricht, *m* und die Signatur, *sig*, an Alice zurück.</span><span class="sxs-lookup"><span data-stu-id="0cee8-115">The CA returns the message, *m*, and the signature, *sig*, to Alice.</span></span> <span data-ttu-id="0cee8-116">*M* und *sig* bilden das Zertifikat von Alice.</span><span class="sxs-lookup"><span data-stu-id="0cee8-116">Together, *m* and *sig* form Alice's certificate.</span></span>

<span data-ttu-id="0cee8-120">Wie bei jeder [*digitalen Signatur*](../secgloss/d-gly.md)kann jeder Empfänger mit Zugriff auf den öffentlichen Schlüssel der Zertifizierungsstelle ermitteln, ob eine bestimmte Zertifizierungsstelle das Zertifikat signiert hat.</span><span class="sxs-lookup"><span data-stu-id="0cee8-120">As with any [*digital signature*](../secgloss/d-gly.md), any receiver with access to the CA's public key can determine whether a specific CA signed the certificate.</span></span> <span data-ttu-id="0cee8-121">Dieser Prozess erfordert keinen Zugriff auf geheime Informationen.</span><span class="sxs-lookup"><span data-stu-id="0cee8-121">This process requires no access to any secret information.</span></span> <span data-ttu-id="0cee8-122">Das soeben vorgestellte Szenario geht davon aus, dass Bob Zugriff auf den öffentlichen Schlüssel der Zertifizierungsstelle hat.</span><span class="sxs-lookup"><span data-stu-id="0cee8-122">The scenario just presented assumes that Bob has access to the CA's public key.</span></span> <span data-ttu-id="0cee8-123">Bob hätte Zugriff auf diesen Schlüssel, wenn er über eine Kopie des Zertifikats der Zertifizierungsstelle verfügt, das den öffentlichen Schlüssel enthält.</span><span class="sxs-lookup"><span data-stu-id="0cee8-123">Bob would have access to that key if he has a copy of the CA's certificate that contains that public key.</span></span>

<span data-ttu-id="0cee8-124">Digitale [*X. 509*](../secgloss/x-gly.md) -Zertifikate enthalten nicht nur den Namen und den öffentlichen Schlüssel eines Benutzers, sondern auch andere Informationen über den Benutzer.</span><span class="sxs-lookup"><span data-stu-id="0cee8-124">[*X.509*](../secgloss/x-gly.md) digital certificates include not only a user's name and public key, but also other information about the user.</span></span> <span data-ttu-id="0cee8-125">Diese Zertifikate sind mehr als Schritt Steine in einer digitalen Vertrauens Hierarchie.</span><span class="sxs-lookup"><span data-stu-id="0cee8-125">These certificates are more than stepping stones in a digital hierarchy of trust.</span></span> <span data-ttu-id="0cee8-126">Sie ermöglichen es der Zertifizierungsstelle, dem Empfänger eines Zertifikats die Möglichkeit zu geben, nicht nur den öffentlichen Schlüssel des Zertifikats zu vertrauen, sondern auch andere Informationen über den Betreff des Zertifikats.</span><span class="sxs-lookup"><span data-stu-id="0cee8-126">They enable the CA to give a certificate's receiver a means of trusting not only the public key of the certificate's subject, but also that other information about the certificate's subject.</span></span> <span data-ttu-id="0cee8-127">Andere Informationen können z. b. eine e-Mail-Adresse, eine Autorisierung zum Signieren von Dokumenten eines bestimmten Werts oder die Autorisierung sein, um eine Zertifizierungsstelle zu werden und andere Zertifikate zu signieren.</span><span class="sxs-lookup"><span data-stu-id="0cee8-127">That other information can include, among other things, an email address, an authorization to sign documents of a given value, or the authorization to become a CA and sign other certificates.</span></span>
