# <a name="extending-cryptoapi-functionality"></a>Erweitern der kryptoapi-Funktionalität

Die Zukunft der [*Kryptografie*](../secgloss/c-gly.md) und der sicheren Kommunikation kann nicht leicht vorhergesagt werden. Möglicherweise werden neue Zertifikat Typen angezeigt, verschiedene Zertifikat Erweiterungen können häufig verwendet werden, und es können neue Nachrichten Typen eingeführt werden. Daher ist die Erweiterbarkeit Teil des Entwurfs wichtiger [*CryptoAPI*](../secgloss/c-gly.md) -Funktionen.

Die folgenden Abschnitte enthalten Übersichten zur Verwendung von OIDs zum Erweitern von CryptoAPI-Funktionen.

| [Erstellen der neuen Funktionalität](creating-the-new-functionality.md)               | Erstellen von OIDs und Funktionen zum Erweitern der Verwendung vorhandener APIs.                                                                     |
| [Registrieren der neuen Funktionalität](registering-the-new-functionality.md)         | Einrichten neuer OID-bezogener Funktionen.                                                                                               |
| [Installieren der neuen Funktionalität](installing-the-new-functionality.md)           | Installieren von OID-Funktionen im Speicher, um die Leistung zu verbessern                                                                          |
| [Erweitern der certopsstore-Funktionalität](extending-certopenstore-functionality.md) | Erweitern der [**CertOpenStore**](/windows/desktop/api/Wincrypt/nf-wincrypt-certopenstore) -Funktionalität mithilfe installier barer oder registrierter Zertifikat Speicher Anbieter-Funktion. |
