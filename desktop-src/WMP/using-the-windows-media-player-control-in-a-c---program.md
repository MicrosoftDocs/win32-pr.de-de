# <a name="using-the-windows-media-player-control-in-a-c-program"></a>Verwenden des Windows Media Player-Steuer Elements in einem C++-Programm

> [!Note]  
> Die Verwendung von C++ zum Einbetten des Windows Media Player-Steuer Elements wird für Windows Media Player 9-Serie oder höher unterstützt.

 

Es gibt verschiedene Möglichkeiten, das Windows Media Player-Steuerelement in einem C++-Programm zu verwenden. Sie können eine Instanz des-Steuer Elements in einer Konsolenanwendung erstellen, oder Sie können das-Steuerelement in eine Windows-Anwendung einbetten. Außerdem können Sie Schnittstellen implementieren, die es Ihnen ermöglichen, ein eingebettetes Player-Steuerelement im Remote Modus auszuführen. Sie können die Benutzeroberfläche eines eingebetteten Steuer Elements anpassen, indem Sie eine Skin-Definitionsdatei anwenden.

Diese Informationen werden in den folgenden Themen beschrieben.

| [Hosting des Windows Media Player-Steuer Elements in einer Windows-Anwendung](hosting-the-windows-media-player-control-in-a-windows-application.md) | Beschreibt die Verwendung des ATL-ActiveX-Host Fensters zum Einbetten des Windows Media Player-Steuer Elements in ein Windows-Programm.                                                            |

> [!Note]  
> Sie können das Windows Media Player 10 Mobile-Steuerelement in eine Windows CE Anwendung einbetten. Die Verfahren, die Sie hierfür verwenden, ähneln denen, die mit dem Windows-Steuerelement für Desktop-Media Player verwendet werden. Es gibt jedoch Unterschiede zwischen ATL für Windows und ATL für Windows CE. In dieser Dokumentation werden die Unterschiede zwischen diesen Implementierungen, soweit zutreffend, beschrieben.
