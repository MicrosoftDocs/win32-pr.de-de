Nachdem Sie die Ausstehenden und Überprüfungsphasen abgeschlossen haben, veröffentlicht Microsoft Ihren Onlineshop.

## <a name="pending-stage"></a>Ausstehende Phase

In der Phase "Ausstehend" haben Sie die Möglichkeit, Ihren Onlineshop in der Windows Media Player. Es ist auch ein guter Zeitpunkt, um sicherzustellen, dass alle erforderlichen Vereinbarungen und Verträge unterzeichnet werden. Um zu beginnen, senden Sie Microsoft die folgenden Informationen, wie weiter später in diesem Thema definiert:

-   Kontaktinformationen
-   Startinformationen
-   Überprüfungskonten

Nach Erhalt Ihrer Kontakt- und Startinformationen sendet Microsoft Ihnen einen Testschlüssel und einen Produktionsschlüssel. Nachdem Sie den Testschlüssel der Registrierung hinzugefügt und den Player neu gestartet haben, wird Ihr Onlineshop im Player angezeigt, und Sie können ihn testen. Informationen zum Ort, an dem Der Testschlüssel in der Registrierung gespeichert werden soll, finden Sie unter Registrierungsschlüssel und -einträge für einen [Online-Store-Typ 1](registry-keys-and-entries-for-a-type-1-online-store.md) oder Registrierungsschlüssel und -einträge für einen Typ [2 Online Store](registry-keys-and-entries-for-a-type-2-online-store.md).

Sie sollten alle Aspekte Ihres Onlineshops testen, einschließlich seiner Benutzeroberfläche und seines Plug-Ins. Im Rahmen des Testprozesses müssen Sie die tests ausführen, die unter Validierungstests für Typ 2 Online Musik [Stores beschrieben werden.](validation-tests-for-type-2-online-music-stores.md)

> [!Note]  
> Speicher vom Typ 1 müssen alle Validierungstests für Speicher vom Typ 2 sowie einige zusätzliche Tests bestehen, die spezifisch für die Erfahrung mit Typ 1 sind. Informationen zu Validierungstests vom Typ 1 erhalten Sie vom Windows Media Player Services Virtual Team unter mpsvctm@microsoft.com .

## <a name="validation-accounts"></a>Überprüfungskonten

Um den im folgenden Abschnitt Überprüfungsphase beschriebenen Überprüfungsprozess abzuschließen, benötigt Microsoft fünf (5) Überprüfungskonten für jeden Kontotyp, den Ihr Onlineshop anbietet. Wenn Sie beispielsweise Konten anbieten, die zwischen Features und Funktionen unterscheiden, z. B. Basic und Premium, benötigt Microsoft fünf (5) Validierungskonten für jeden Typ, insgesamt zehn (10) Konten.

Senden Sie den Benutzernamen und das Kennwort für jedes Validierungskonto an mpsvctm@microsoft.com . Geben Sie auch den Namen einer Person an, die Microsoft bezüglich Validierungskonten kontaktieren kann. Diese Konten werden für die laufende monatliche Überprüfung verwendet. Schließen Sie daher einen Prozess zum "Aufladen" der Konten ein. Eines der häufigsten Probleme tritt auf, wenn ein Onlineshop nicht genügend Kaufguthaben für Microsoft bereitstellt, um alle Kaufszenarien zu überprüfen. Die Konten müssen nach Abschluss des Onboardingvorgangs aktiv bleiben.

## <a name="validation-stage"></a>Überprüfungsphase

Während der Überprüfungsphase überprüft Microsoft, ob die primären Features Ihres Onlineshops ordnungsgemäß funktionieren. Für alle Filialen (Typ 1 und Typ 2) führt Microsoft die Überprüfungstests aus, die unter [Validierungstests für Typ 2 Online Musik Stores](validation-tests-for-type-2-online-music-stores.md)beschrieben sind. Microsoft führt auch einige zusätzliche Validierungstests für Stores vom Typ 1 aus. Wenn Sie Fragen zur Überprüfungsphase für Speicher vom Typ 1 haben, senden Sie E-Mails an mpsvctm@microsoft.com .

Während der Überprüfungsphase führt Microsoft zwei Validierungsdurchläufe durch. Der erste Durchlauf gilt für Release Candidate 1 (RC1) Ihres Onlineshops. Wenn Ihr Speicher die RC1-Überprüfung besteht, sollten Sie ihn sperren und keine weiteren Änderungen vornehmen. Microsoft führt einen zweiten Überprüfungsdurchlauf für Ihren Store durch, auch wenn Ihr Store die RC1-Überprüfung besteht.

Wenn in Ihrem Speicher ein Teil des RC1-Überprüfungsdurchlaufs fehlschlägt, haben Sie zwei Wochen Zeit, einen zweiten Release Candidate (RC2) zu erstellen, den Microsoft während des RC2-Überprüfungsdurchlaufs überprüft.

Wenn bei Ihrem Speicher ein Teil der RC2-Überprüfung fehlschlägt, müssen Sie bis zum folgenden Start warten.

## <a name="test-and-production-keys"></a>Test- und Produktionsschlüssel

Denken Sie daran, dass Sie während der ausstehenden Phase zwei Sätze von Startinformationen an Microsoft gesendet haben: eine für Ihren Testspeicher und eine für Ihren Produktionsspeicher. Denken Sie auch daran, dass Microsoft Ihnen zwei Schlüssel gesendet hat: einen Testschlüssel und einen Produktionsschlüssel.

Der Testschlüssel ist vollständig für Ihre eigene Verwendung vorgesehen. Wenn sich Ihr Testschlüssel in der Registrierung befindet, verwendet Windows Media Player die ServiceInfo-URL, die Sie für Ihren Testspeicher übermittelt haben.

Wenn sich Ihr Produktionsschlüssel in der Registrierung befindet, verwendet Windows Media Player die ServiceInfo-URL, die Sie für Ihren Produktionsspeicher übermittelt haben. Microsoft verwendet Ihren Produktionsschlüssel während der Überprüfungsphase. Microsoft verwendet Ihren Testschlüssel nie für die Überprüfung.

Wenn Ihr Store live ist, verwendet Windows Media Player die ServiceInfo-URL, die Sie für Ihren Produktionsspeicher übermittelt haben.

Im Allgemeinen sollte Ihr Testspeicher der Ort sein, an dem Sie Ihren Dienst entwickeln und tägliche Änderungen vornehmen. Ihr Produktionsspeicher sollte der Ort sein, an dem Sie ein stabiles Release Ihres Diensts beibehalten.

## <a name="common-on-boarding-problems"></a>Häufige Probleme beim Onboarding

Im Folgenden finden Sie einige häufige Probleme, die dazu führen können, dass Ihr Speicher bei Validierungstests fehlschlägt.

-   Fehler beim Übergang von Testservern zu Produktionsservern. Dies führt zu vielen Problemen, z. B. fehlende Seiten, ungültige IIS- und Sicherheitseinstellungen und Testkonten, die nicht mehr funktionieren.
-   Der Link Kaufen (oder Shoplink) im Bereich Jetzt wiedergeben ist ungültig. Dies kann dazu führen, dass ihre Speicherüberprüfung fehlschlägt, auch wenn alles andere funktioniert.
-   Das Überprüfungsteam von Microsoft testet verschiedene Kaufszenarien von kleinen bis hin zu sehr großen Käufen. Sie müssen ihnen eine angemessene Möglichkeit bieten, als Verbraucher in Ihrem Geschäft zu fungieren. Ihr Geschäft kann nicht überprüft werden, wenn das Validierungsteam nicht über genügend Kaufguthaben verfügt, um all diese Szenarien zu überprüfen.

Eine ausführlichere Liste der allgemeinen Onboardingprobleme und häufig gestellten Fragen, die vom virtuellen Windows Media Player Services-Team kompiliert wurden, finden Sie unter [Allgemeine Onboardingprobleme für Online Musik Stores.](common-on-boarding-problems-for-online-music-stores.md)
