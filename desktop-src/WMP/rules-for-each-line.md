# <a name="rules-for-each-line"></a>Regeln für jede Zeile

Jede Definition in der Skin-Definitionsdatei muss sich in einer separaten Zeile befinden. Mithilfe der folgenden Regeln müssen Sie Text in Zeilen eingeben:

-   Jede Zeile muss mit einer Absatz Rückgabe enden. Einige Zeilen sind möglicherweise länger als die sichtbare Zeilenlänge des Text-Editors, aber die Zeilen werden nicht unterbricht. Möglicherweise möchten Sie den Zeilenumbruch deaktivieren, um sicherzustellen, dass die Zeilen nicht unterbrochen werden.
-   Verwenden Sie nur sichtbare Zeichen gemäß der Definition in der Skin-Referenz.
-   Eine Linie besteht aus einem oder mehreren Elementen, getrennt durch Leerzeichen.
-   Sie können zur besseren Lesbarkeit beliebig viele Leerzeichen zwischen Elementen eingeben.
-   Verwenden Sie keine Tabulator Zeichen an einer beliebigen Stelle in der Datei.
-   Zeilen, die mit zwei aufeinander folgenden Schrägstrichen (//) beginnen, werden ignoriert. Verwenden Sie diese für Kommentare. Leere Zeilen werden ebenfalls ignoriert.
-   Fügen Sie Leerzeichen zwischen den eckigen Klammern und dem Namen des Abschnitts ein.
