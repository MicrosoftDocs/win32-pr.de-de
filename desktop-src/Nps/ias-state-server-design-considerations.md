<span data-ttu-id="534cd-107">Abhängig von Ihrem Entwurf benötigen Sie möglicherweise einen Server, um die Benutzer zu verfolgen, die zurzeit am Netzwerk angemeldet sind.</span><span class="sxs-lookup"><span data-stu-id="534cd-107">Depending on your design, you may need a server to track the users that are currently logged onto the network.</span></span> <span data-ttu-id="534cd-108">Die Hauptaufgabe beim Zustands Server besteht darin, die Informationen in der Zustands Server Datenbank synchron mit der Person zu halten, die gerade angemeldet ist.</span><span class="sxs-lookup"><span data-stu-id="534cd-108">The main challenge with the state server is keeping the information in the state server database in-sync with who is actually logged on.</span></span> <span data-ttu-id="534cd-109">Wenn die Informationen auf dem Zustands Server nicht synchron sind, können Benutzer möglicherweise über mehrere Sitzungen verfügen, wenn Sie dazu nicht berechtigt sind.</span><span class="sxs-lookup"><span data-stu-id="534cd-109">If the information in the state server is out of sync, users may succeed in having multiple sessions when they aren't authorized to do so.</span></span> <span data-ttu-id="534cd-110">Benutzer, die nicht über mehrere Sitzungen verfügen, könnten auch versehentlich benachteiligt werden.</span><span class="sxs-lookup"><span data-stu-id="534cd-110">Also, users who do not have multiple sessions could be inadvertently penalized.</span></span>

<span data-ttu-id="534cd-111">Beim Implementieren des Zustands Servers sollte Folgendes berücksichtigt werden:</span><span class="sxs-lookup"><span data-stu-id="534cd-111">The following should be taken into consideration in implementing the state server:</span></span>

-   <span data-ttu-id="534cd-112">Der Zustands Server muss die Entscheidung innerhalb weniger Sekunden online stellen.</span><span class="sxs-lookup"><span data-stu-id="534cd-112">The state server must make the decision online in a few seconds.</span></span> <span data-ttu-id="534cd-113">Aus diesem Grund benötigt der Zustands Server eine skalierbare Infrastruktur, die viele Updates und Abfragen pro Sekunde unterstützen kann.</span><span class="sxs-lookup"><span data-stu-id="534cd-113">For this reason the state server requires a scaleable infrastructure that can support many updates and queries per second.</span></span> <span data-ttu-id="534cd-114">Relationale Datenbanken eignen sich nicht für solche großen Abfragen mit gleichzeitigen Updates.</span><span class="sxs-lookup"><span data-stu-id="534cd-114">Relational databases are not appropriate for such large queries with simultaneous updates.</span></span> <span data-ttu-id="534cd-115">Relationale Datenbanken werden primär erstellt, um die Daten konsistent zu halten und eine konsistente Sicht der Daten für alle Consumer bereitzustellen.</span><span class="sxs-lookup"><span data-stu-id="534cd-115">Relational databases are primarily built to keep data consistent and to provide a consistent view of the data to all consumers.</span></span> <span data-ttu-id="534cd-116">Sie sind nicht für schnelle Updates konzipiert.</span><span class="sxs-lookup"><span data-stu-id="534cd-116">They are not built for quick updates.</span></span>
-   <span data-ttu-id="534cd-117">Die Transaktions Konsistenz bei Updates zwischen mehreren Objekten ist nicht von Bedeutung.</span><span class="sxs-lookup"><span data-stu-id="534cd-117">Transactional consistency on updates between multiple objects is not important.</span></span> <span data-ttu-id="534cd-118">Dies liegt daran, dass der Zustands Server ein kleines Fenster der Gelegenheit tolerieren kann.</span><span class="sxs-lookup"><span data-stu-id="534cd-118">This is because the state server can tolerate a small window of opportunity.</span></span> <span data-ttu-id="534cd-119">Allerdings ist die Transaktions Konsistenz eines einzelnen Updates wichtig, um die Wahrscheinlichkeit zu verringern, dass der Zustands Server in einem inkonsistenten Zustand belassen wird, wenn einer der RADIUS-Server in der Mitte des Updates heruntergefahren wird.</span><span class="sxs-lookup"><span data-stu-id="534cd-119">However, transactional consistency of a single update is important to reduce the chances of leaving the state server in an inconsistent state if one of the RADIUS servers is shut down in the middle of the update.</span></span>
-   <span data-ttu-id="534cd-120">Persistenz (das Speichern des Zustands des Netzwerks in einem persistenten Speicher) ist nicht wichtig, da die persistenten Informationen schnell mit dem tatsächlichen Status des Netzwerks synchronisiert werden.</span><span class="sxs-lookup"><span data-stu-id="534cd-120">Persistence (saving the state of the network to persistent storage) is not important because the persistent information will quickly fall out of sync with the actual state of the network.</span></span>

<span data-ttu-id="534cd-122">Ein möglicher Entwurf besteht darin, sowohl eine Authentifizierungs Erweiterungs-DLL als auch eine Autorisierungs Erweiterungs-DLL zu implementieren.</span><span class="sxs-lookup"><span data-stu-id="534cd-122">One possible design is to implement both an Authentication Extension DLL and an Authorization Extension DLL.</span></span> <span data-ttu-id="534cd-123">Jede dieser DLLs kann über das Netzwerk mit einer Datenbank kommunizieren.</span><span class="sxs-lookup"><span data-stu-id="534cd-123">Each of these DLLs can communicate over the network with a database.</span></span> <span data-ttu-id="534cd-124">Die Autorisierungs Erweiterungs-DLL kann die Datenbank mit Informationen darüber aktualisieren, wer zurzeit im Netzwerk angemeldet ist.</span><span class="sxs-lookup"><span data-stu-id="534cd-124">The Authorization Extension DLL can update the database with information about who is currently logged onto the network.</span></span> <span data-ttu-id="534cd-125">Die authentifizierungserweiterungs-dll kann die Datenbank nach diesen Informationen Abfragen, um zu entscheiden, ob die Authentifizierungsanforderung eines bestimmten Benutzers angenommen oder abgelehnt werden soll. Wenn der Benutzer bereits angemeldet ist, wird die Anforderung abgelehnt.</span><span class="sxs-lookup"><span data-stu-id="534cd-125">The Authentication Extension DLL can query the database for this information to decide whether to accept or reject a particular user's authentication request; if the user is already logged on, the request is rejected.</span></span>

<span data-ttu-id="534cd-126">Der Vorteil, dass die Autorisierungs Erweiterungs-DLL die State-Server-Datenbank aktualisiert, besteht darin, dass die Autorisierungs Erweiterungs-DLL Zugriff auf Weitere Informationen über den authentifizierten Benutzer hat.</span><span class="sxs-lookup"><span data-stu-id="534cd-126">The advantage of having the Authorization Extension DLL update the state-server database is that the Authorization Extension DLL has access to more information about the authenticated user.</span></span> <span data-ttu-id="534cd-127">Die dll der Autorisierungs Erweiterung hat Zugriff auf Alle Autorisierungs Attribute aus dem NPS-Autorisierungs Mechanismus.</span><span class="sxs-lookup"><span data-stu-id="534cd-127">The Authorization Extension DLL has access to all of the authorization attributes from the NPS Authorization mechanism.</span></span> <span data-ttu-id="534cd-128">Beispielsweise können einige Benutzer über Autorisierungen verfügen, mit denen Sie mehrere Sitzungen haben können.</span><span class="sxs-lookup"><span data-stu-id="534cd-128">For example, some users may have authorizations that allow them to have multiple sessions.</span></span> <span data-ttu-id="534cd-129">Der Zustands Server sollte solche Benutzer als Sonderfall behandeln.</span><span class="sxs-lookup"><span data-stu-id="534cd-129">The state server should treat such users as a special case.</span></span>
