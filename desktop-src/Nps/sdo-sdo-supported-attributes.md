# <a name="sdo-supported-attributes"></a>Unterstützte SDO-Attribute

> [!Note]  
> Der Internet Authentifizierungsdienst (IAS) wurde ab Windows Server 2008 in den Netzwerk Richtlinien Server (Network Policy Server, NPS) umbenannt.

 

Ein NPS-Server Datenobjekt verfügt über eine [**profile**](/windows/desktop/api/sdoias/ne-sdoias-profileproperties) -Eigenschaft, die den Zugriff auf eine Auflistung von Attributen ermöglicht. Die folgenden Attribute werden unterstützt:

-   [Acct-Provider-Name](/windows/desktop/Nps/sdo-acct-provider-name)
-   [Acct-Provider-Type](/windows/desktop/Nps/sdo-acct-provider-type)
-   [Name des Authentifizierungs Anbieters](/windows/desktop/Nps/sdo-auth-provider-name)
-   [Auth-Provider-Type](/windows/desktop/Nps/sdo-auth-provider-type)
-   [Ignore-User-Dialin-Properties](/windows/desktop/Nps/sdo-ignore-user-dialin-properties)
-   [Manipulation: Regel](/windows/desktop/Nps/sdo-manipulation-rule)
-   [Manipulation-Ziel](/windows/desktop/Nps/sdo-manipulation-target)
-   [NP-Allowed-EAP-Typ](/windows/desktop/Nps/sdo-np-allowed-eap-type)
-   [NP-allowed-Port-Typen](/windows/desktop/Nps/sdo-np-allowed-port-types)
-   [NP-Authentication-Type](/windows/desktop/Nps/sdo-np-authentication-type)
-   [NP-aufgerufene Stations-ID](/windows/desktop/Nps/sdo-np-called-station-id)
