<span data-ttu-id="b1539-108">**Sample (Sampler, Location \[ , Offset \[ , Clamp \[ , Feedback \] \] \] )**</span><span class="sxs-lookup"><span data-stu-id="b1539-108">**Sample(sampler, location \[, offset \[, clamp \[, feedback\] \] \])**</span></span>

<span data-ttu-id="b1539-109">Ein Beispiel für eine **Beispiel Methode** ist [**Texture2D. Sample (S, float, int, float, uint)**](/windows/desktop/direct3dhlsl/t2darray-sample-s-float-int-float-uint-).</span><span class="sxs-lookup"><span data-stu-id="b1539-109">An example of a **Sample** method is [**Texture2D.Sample(S,float,int,float,uint)**](/windows/desktop/direct3dhlsl/t2darray-sample-s-float-int-float-uint-).</span></span>

<span data-ttu-id="b1539-110">Die Offset-, Clamp-und Feedback-Parameter sind optional.</span><span class="sxs-lookup"><span data-stu-id="b1539-110">The offset, clamp and feedback parameters are optional.</span></span> <span data-ttu-id="b1539-111">Sie müssen alle optionalen Parameter angeben, die Sie benötigen. Dies entspricht den C++-Regeln für Standard Funktionsargumente.</span><span class="sxs-lookup"><span data-stu-id="b1539-111">You must specify all optional parameters up to the one you need, which is consistent with the C++ rules for default function arguments.</span></span> <span data-ttu-id="b1539-112">Wenn der Feedback Status z. b. erforderlich ist, müssen sowohl Offset-als auch Klammer Parameter explizit für **Stichproben** bereitgestellt werden, auch wenn Sie möglicherweise nicht logisch benötigt werden.</span><span class="sxs-lookup"><span data-stu-id="b1539-112">For example, if the feedback status is needed, both offset and clamp parameters need to be explicitly supplied to **Sample**, even though they may not be logically needed.</span></span>

<span data-ttu-id="b1539-113">Der Klammer Parameter ist ein skalarer float-Wert.</span><span class="sxs-lookup"><span data-stu-id="b1539-113">The clamp parameter is a scalar float value.</span></span> <span data-ttu-id="b1539-114">Der Literalwert von Clamp = 0,0 f gibt an, dass der Klammer Vorgang nicht ausgeführt wird.</span><span class="sxs-lookup"><span data-stu-id="b1539-114">The literal value of clamp=0.0f indicates that the clamp operation isn't performed.</span></span>

<span data-ttu-id="b1539-115">Der Feedback Parameter ist eine **uint** -Variable, die Sie der systeminternen [**checkaccessfullymapping**](/windows/desktop/direct3dhlsl/checkaccessfullymapped) -Funktion für den Speicherzugriff zur Abfrage bereitstellen können.</span><span class="sxs-lookup"><span data-stu-id="b1539-115">The feedback parameter is a **uint** variable that you can supply to the memory-access querying intrinsic [**CheckAccessFullyMapped**](/windows/desktop/direct3dhlsl/checkaccessfullymapped) function.</span></span> <span data-ttu-id="b1539-116">Der Wert des Parameters "Feedback" darf nicht geändert oder interpretiert werden. der Compiler bietet jedoch keine erweiterte Analyse und Diagnose, um zu ermitteln, ob Sie den Wert geändert haben.</span><span class="sxs-lookup"><span data-stu-id="b1539-116">You must not modify or interpret the value of the feedback parameter; but, the compiler doesn't provide any advanced analysis and diagnostics to detect whether you modified the value.</span></span>

<span data-ttu-id="b1539-117">Dies ist die Syntax von [**checkaccessfullymapping**](/windows/desktop/direct3dhlsl/checkaccessfullymapped):</span><span class="sxs-lookup"><span data-stu-id="b1539-117">Here is the syntax of [**CheckAccessFullyMapped**](/windows/desktop/direct3dhlsl/checkaccessfullymapped):</span></span>

<span data-ttu-id="b1539-118">**bool checkaccessfullymapping (in uint feedbackvar);**</span><span class="sxs-lookup"><span data-stu-id="b1539-118">**bool CheckAccessFullyMapped(in uint FeedbackVar);**</span></span>

<span data-ttu-id="b1539-119">[**Checkaccessfullymapping**](/windows/desktop/direct3dhlsl/checkaccessfullymapped) interpretiert den Wert von *feedbackvar* und gibt true zurück, wenn alle Daten, auf die zugegriffen wird, in der Ressource zugeordnet wurden. Andernfalls gibt **checkaccessfullymapping** den Wert false zurück.</span><span class="sxs-lookup"><span data-stu-id="b1539-119">[**CheckAccessFullyMapped**](/windows/desktop/direct3dhlsl/checkaccessfullymapped) interprets the value of *FeedbackVar* and returns true if all data being accessed was mapped in the resource; otherwise, **CheckAccessFullyMapped** returns false.</span></span>

<span data-ttu-id="b1539-120">Wenn entweder der Parameter "Clamp" oder "Feedback" vorhanden ist, gibt der Compiler eine Variante der Grund Anweisung aus.</span><span class="sxs-lookup"><span data-stu-id="b1539-120">If either the clamp or feedback parameter is present, the compiler emits a variant of the basic instruction.</span></span> <span data-ttu-id="b1539-121">Beispielsweise generiert Sample einer gekachelten Ressource die- `sample_cl_s` Anweisung.</span><span class="sxs-lookup"><span data-stu-id="b1539-121">For example, sample of a tiled resource generates the `sample_cl_s` instruction.</span></span> <span data-ttu-id="b1539-122">Wenn weder "Clamp" noch "Feedback" angegeben ist, gibt der Compiler die grundlegende Anweisung aus, sodass es keine Änderung des aktuellen Verhaltens gibt.</span><span class="sxs-lookup"><span data-stu-id="b1539-122">If neither clamp nor feedback is specified, the compiler emits the basic instruction, so that there is no change from the current behavior.</span></span> <span data-ttu-id="b1539-123">Der Klammer Wert von 0,0 f gibt an, dass keine Klammer ausgeführt wird. Daher kann der Treiber Compiler die Anweisung weiter an die Zielhardware anpassen.</span><span class="sxs-lookup"><span data-stu-id="b1539-123">The clamp value of 0.0f indicates that no clamp is performed; thus, the driver compiler can further tailor the instruction to the target hardware.</span></span> <span data-ttu-id="b1539-124">Wenn Feedback ein NULL-Register in einer Anweisung ist, wird das Feedback nicht verwendet. Daher kann der Treiber Compiler die Anweisung weiter an die Zielarchitektur anpassen.</span><span class="sxs-lookup"><span data-stu-id="b1539-124">If feedback is a NULL register in an instruction, the feedback is unused; thus, the driver compiler can further tailor the instruction to the target architecture.</span></span>

<span data-ttu-id="b1539-125">Wenn der HLSL-Compiler diese Klammer als 0,0 f ausgibt und Feedback nicht verwendet wird, gibt der Compiler die entsprechende Grund Anweisung aus (z `sample` . b `sample_cl_s` . anstelle von).</span><span class="sxs-lookup"><span data-stu-id="b1539-125">If the HLSL compiler infers that clamp is 0.0f and feedback is unused, the compiler emits the corresponding basic instruction (for example, `sample` rather than `sample_cl_s`).</span></span>

<span data-ttu-id="b1539-126">Wenn ein gekachelter Ressourcen Zugriff aus mehreren einzelnen Anweisungen für den Byte Code besteht (z. b. bei strukturierten Ressourcen), aggregiert der Compiler einzelne Feedback Werte über den Vorgang oder, um den endgültigen Feedback Wert zu erhalten.</span><span class="sxs-lookup"><span data-stu-id="b1539-126">If a tiled resource access consists of several constituent byte code instructions, for example, for structured resources, the compiler aggregates individual feedback values via the OR operation to produce the final feedback value.</span></span> <span data-ttu-id="b1539-127">Aus diesem Grund wird ein einzelner Feedback Wert für einen solchen komplexen Zugriff angezeigt.</span><span class="sxs-lookup"><span data-stu-id="b1539-127">Therefore, you see a single feedback value for such a complex access.</span></span>
