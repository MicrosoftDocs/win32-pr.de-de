# <a name="effects-direct3d-11"></a><span data-ttu-id="9634e-104">Effekte (Direct3D 11)</span><span class="sxs-lookup"><span data-stu-id="9634e-104">Effects (Direct3D 11)</span></span>

<span data-ttu-id="9634e-105">Ein DirectX-Effekt ist eine Auflistung des Pipelinezustands, die durch in [HLSL](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-reference) geschriebene Ausdrücke und eine syntaxspezifische Syntax festgelegt wird, die für das Effektframework spezifisch ist.</span><span class="sxs-lookup"><span data-stu-id="9634e-105">A DirectX effect is a collection of pipeline state, set by expressions written in [HLSL](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-reference) and some syntax that is specific to the effect framework.</span></span>

<span data-ttu-id="9634e-106">Verwenden Sie nach dem Kompilieren eines Effekts die Effektframework-APIs zum Rendern.</span><span class="sxs-lookup"><span data-stu-id="9634e-106">After compiling an effect, use the effect framework APIs to render.</span></span> <span data-ttu-id="9634e-107">Die Effektfunktionalität kann von einem so einfachen Bereich reichen wie ein Vertex-Shader, der Geometrie und einen Pixel-Shader transformiert, der eine Volltonfarbe ausgibt, bis hin zu einer Renderingtechnik, die mehrere Durchläufe erfordert, jede Phase der Grafikpipeline verwendet und den Shaderzustand sowie den Pipelinezustand bearbeitet, der nicht den programmierbaren Shadern zugeordnet ist.</span><span class="sxs-lookup"><span data-stu-id="9634e-107">Effect functionality can range from something as simple as a vertex shader that transforms geometry and a pixel shader that outputs a solid color, to a rendering technique that requires multiple passes, uses every stage of the graphics pipeline, and manipulates shader state as well as the pipeline state not associated with the programmable shaders.</span></span>

<span data-ttu-id="9634e-108">Der erste Schritt besteht darin, den Zustand zu organisieren, den Sie in einem Effekt steuern möchten.</span><span class="sxs-lookup"><span data-stu-id="9634e-108">The first step is to organize the state you want to control in an effect.</span></span> <span data-ttu-id="9634e-109">Dies schließt den Shaderzustand (Scheitelpunkt, Hülle, Domäne, Geometrie, Pixel- und Compute-Shader), den von den Shadern verwendeten Textur- und Samplerzustand sowie andere nicht programmierbare Pipelinestatus ein.</span><span class="sxs-lookup"><span data-stu-id="9634e-109">This includes shader state (vertex, hull, domain, geometry, pixel and compute shaders), texture and sampler state used by the shaders, and other non-programmable pipeline state.</span></span> <span data-ttu-id="9634e-110">Sie können einen Effekt im Arbeitsspeicher als Textzeichenfolge erstellen, aber in der Regel wird die Größe so groß, dass es praktisch ist, den Effektzustand in einer Effektdatei zu speichern (eine Textdatei, die mit einer FX-Erweiterung endet).</span><span class="sxs-lookup"><span data-stu-id="9634e-110">You can create an effect in memory as a text string, but typically, the size gets large enough that it is handy to store effect state in an effect file (a text file that ends in a .fx extension).</span></span> <span data-ttu-id="9634e-111">Um einen Effekt zu verwenden, müssen Sie ihn kompilieren (um die HLSL-Syntax sowie die Syntax des Effektframeworks zu überprüfen), den Effektzustand über API-Aufrufe initialisieren und Ihre Renderschleife so ändern, dass die Rendering-APIs aufgerufen werden.</span><span class="sxs-lookup"><span data-stu-id="9634e-111">To use an effect, you must compile it (to check HLSL syntax as well as effect framework syntax), initialize effect state through API calls, and modify your render loop to call the rendering APIs.</span></span>

<span data-ttu-id="9634e-112">Ein Effekt kapselt den gesamten Renderzustand, der von einem bestimmten Effekt benötigt wird, in eine einzelne Renderingfunktion, die als Technik bezeichnet wird.</span><span class="sxs-lookup"><span data-stu-id="9634e-112">An effect encapsulates all of the render state required by a particular effect into a single rendering function called a mehrere Durchläufe innerhalb einer Technik.</span><span class="sxs-lookup"><span data-stu-id="9634e-114">To implement a multiple pass rendering effect, implement one or more passes within a technique.</span></span> <span data-ttu-id="9634e-115">Angenommen, Sie möchten einige Geometrien mit einem Satz von Tiefen-/Schablonenpuffern rendern und anschließend einige Sprites darüber zeichnen.</span><span class="sxs-lookup"><span data-stu-id="9634e-115">For example, say you wanted to render some geometry with one set of depth/stencil buffers, and then draw some sprites on top of that.</span></span> <span data-ttu-id="9634e-116">Sie können das Geometrierendering im ersten Durchlauf und die Spritezeichnung im zweiten Durchlauf implementieren.</span><span class="sxs-lookup"><span data-stu-id="9634e-116">You could implement the geometry rendering in the first pass, and the sprite drawing in the second pass.</span></span> <span data-ttu-id="9634e-117">Um den Effekt zu rendern, rendern Sie einfach beide Durchläufe in der Renderschleife.</span><span class="sxs-lookup"><span data-stu-id="9634e-117">To render the effect, you simply render both passes in your render loop.</span></span> <span data-ttu-id="9634e-118">Sie können eine beliebige Anzahl von Techniken in einem Effekt implementieren.</span><span class="sxs-lookup"><span data-stu-id="9634e-118">You can implement any number of techniques in an effect.</span></span> <span data-ttu-id="9634e-119">Je größer die Anzahl der Techniken ist, desto größer ist die Kompilierzeit für den Effekt.</span><span class="sxs-lookup"><span data-stu-id="9634e-119">Of course, the greater the number of techniques, the greater the compile time for the effect.</span></span> <span data-ttu-id="9634e-120">Eine Möglichkeit, diese Funktionalität zu nutzen, besteht darin, Effekte mit Techniken zu erzielen, die für die Ausführung auf unterschiedlicher Hardware konzipiert sind.</span><span class="sxs-lookup"><span data-stu-id="9634e-120">One way to exploit this functionality is to create effects with techniques that are designed to run on different hardware.</span></span> <span data-ttu-id="9634e-121">Dadurch kann eine Anwendung die Leistung basierend auf den erkannten Hardwarefunktionen ordnungsgemäß herabstufen.</span><span class="sxs-lookup"><span data-stu-id="9634e-121">This allows an application to gracefully downgrade performance based on the hardware capabilities detected.</span></span>

<span data-ttu-id="9634e-122">Eine Reihe von Techniken kann in einer Gruppe gruppiert werden (die die Syntax "fxgroup" verwendet).</span><span class="sxs-lookup"><span data-stu-id="9634e-122">A set of techniques can be grouped in a group (which uses the syntax "fxgroup").</span></span> <span data-ttu-id="9634e-123">Techniken können in beliebiger Weise gruppiert werden.</span><span class="sxs-lookup"><span data-stu-id="9634e-123">Techniques can be grouped in any way.</span></span> <span data-ttu-id="9634e-124">Beispielsweise können mehrere Gruppen erstellt werden, eine pro Material. jedes Material kann über eine Technik für jede Hardwareebene verfügen. jede Technik verfügt über eine Reihe von Durchläufen, die das Material auf der jeweiligen Hardware definieren.</span><span class="sxs-lookup"><span data-stu-id="9634e-124">For example, multiple groups could be created, one per material; each material could have a technique for each hardware level; each technique would have a set of passes which define the material on the particular hardware.</span></span>
