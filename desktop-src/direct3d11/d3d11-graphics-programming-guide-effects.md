<span data-ttu-id="417ad-104">Ein DirectX-Effekt ist eine Auflistung von Pipeline Zuständen, die durch in [HLSL](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-reference) geschriebene Ausdrücke und eine bestimmte, für das Effect-Framework spezifische Syntax festgelegt ist.</span><span class="sxs-lookup"><span data-stu-id="417ad-104">A DirectX effect is a collection of pipeline state, set by expressions written in [HLSL](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-reference) and some syntax that is specific to the effect framework.</span></span>

<span data-ttu-id="417ad-105">Verwenden Sie nach dem Kompilieren eines Effekts die Effekt-Framework-APIs zum Rendering.</span><span class="sxs-lookup"><span data-stu-id="417ad-105">After compiling an effect, use the effect framework APIs to render.</span></span> <span data-ttu-id="417ad-106">Die Auswirkung der Funktionalität kann von etwas einfacher sein wie ein Vertexshader, der Geometrie transformiert, und ein Pixelshader, der eine voll Tonfarbe ausgibt, an eine renderingtechnik, die mehrere Durchgänge erfordert, jede Phase der Grafik Pipeline verwendet und den shaderzustand und den Pipeline Zustand, der den programmierbaren Shadern nicht zugeordnet ist, ändert.</span><span class="sxs-lookup"><span data-stu-id="417ad-106">Effect functionality can range from something as simple as a vertex shader that transforms geometry and a pixel shader that outputs a solid color, to a rendering technique that requires multiple passes, uses every stage of the graphics pipeline, and manipulates shader state as well as the pipeline state not associated with the programmable shaders.</span></span>

<span data-ttu-id="417ad-111">Ein Effekt kapselt den gesamten renderzustand, der für einen bestimmten Effekt erforderlich ist, in eine einzelne Renderingfunktion, die als Technik bezeichnet technique.</span></span> <span data-ttu-id="417ad-114">Angenommen, Sie möchten eine Geometrie mit einem Satz von tiefen-/Schablone-Puffern rendern und dann einige Sprites darauf zeichnen.</span><span <span data-ttu-id="417ad-115">Sie könnten das Geometrie Rendering im ersten Durchlauf und die Sprite-Zeichnung im zweiten Durchlauf implementieren.</span><span class="sxs-lookup"><span any number of techniques in an effect.</span></span> <span data-ttu-id="417ad-118">Natürlich ist die Anzahl der Techniken umso größer als die Kompilierzeit für den Effekt.</span><span class="sxs-lookup"><span data-stu-id="417ad-118">Of course, the greater the number of techniques, the greater the compile time for the effect.</span></span> <span data-ttu-id="417ad-119">Eine <span data-ttu-id="417ad-121">Eine Reihe von Techniken kann in einer Gruppe gruppiert werden (die die Syntax "fxgroup" verwendet).</span><span class="sxs-lookup"><span data-stu-id="417ad-121">A set of techniques can be grouped in a group (which uses the syntax "fxgroup").</span></span> <span data-ttu-id="417ad-122">Techniken können in beliebiger Weise gruppiert werden.</span><span class="sxs-lookup"><span data-stu-id="417ad-122">Techniques can be grouped in any way.</span></span> <span data-ttu-id="417ad-123">Beispielsweise können mehrere Gruppen erstellt werden, eine pro Material. jedes Material kann über eine Technik für jede Hardwareebene verfügen. jede Technik hätte eine Reihe von Durchläufen, die das Material auf der jeweiligen Hardware definieren.</span><span class="sxs-lookup"><span data-stu-id="417ad-123">For example, multiple groups could be created, one per material; each material could have a technique for each hardware level; each technique would have a set of passes which define the material on the particular hardware.</span></span>
