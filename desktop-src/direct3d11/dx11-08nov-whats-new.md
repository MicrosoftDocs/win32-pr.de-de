# <a name="whats-new-in-the-nov-2008-direct3d-11-tech-preview"></a>Neues in der Tech Preview vom November 2008 Direct3D 11

Diese Version von Direct3D 11 enthält neue Features, Tools und Dokumentation.




| Element | BESCHREIBUNG | 
|------|-------------|
| <span id="Tessellation"></span><span id="tessellation"></span><span id="TESSELLATION"></span>Tessellation<br /> | Direct3D 11 bietet zusätzliche Pipelinestufen zur Unterstützung des Mosaiks von primitiven Hochordnungstypen in Echtzeit. <a href="direct3d-11-advanced-stages-tessellation.md"></a> Mit umfassenden programmierbaren Funktionen ermöglicht dieses Feature viele verschiedene Methoden zum Auswerten von hochwertigen Oberflächen, einschließlich Unterteilungsoberflächen mit Näherungstechniken, Bézierpatches, adaptivem Mosaik und Verschiebungszuordnung. Dieses Feature ist nur auf Direct3D-Hardware der 11-Klasse verfügbar. Zum Auswerten dieses Features müssen Sie daher den Referenzraster verwenden. Eine Demo des Mosaiks in Aktion finden Sie im <a href="https://msdn.microsoft.com/library/Ee416576(v=VS.85).aspx">SubD11-Beispiel,</a> das über den Beispielbrowser verfügbar ist.<br /> | 
| <span id="Compute_Shader"></span><span id="compute_shader"></span><span id="COMPUTE_SHADER"></span>Compute-Shader<br /> | Der <a href="direct3d-11-advanced-stages-compute-shader.md">Compute-Shader</a> ist eine zusätzliche Stufe, die unabhängig von der Direct3D 11-Pipeline ist, die allgemeines Computing auf der GPU ermöglicht. Zusätzlich zu allen Shaderfunktionen, die vom einheitlichen Shaderkern bereitgestellt werden, unterstützt der Compute-Shader auch punktierte Lese- und Schreibvorgänge in Ressourcen über ungeordnete Zugriffsansichten, einen freigegebenen Speicherpool innerhalb einer Gruppe von ausgeführten Threads, Synchronisierungsprimitiven, atomaren Operatoren und vielen anderen erweiterten datenparallelen Features. In diesem Release wurde eine Variante des Direct3D 11 Compute Shaders aktiviert, die auf Direct3D 10-Hardware betrieben werden kann. Daher ist es möglich, Compute-Shader auf tatsächlicher Hardware zu entwickeln, aber es ist ein aktualisierter Treiber erforderlich. Die vollständige Funktionalität des Direct3D 11 Compute-Shaders ist für die Unterstützung von Direct3D-Hardware der 11-Klasse vorgesehen. Um die vollständige Funktionalität auswerten zu können, müssen Sie den Referenzraster verwenden, bis diese Hardware verfügbar ist. Eine Demo des Compute-Shaders in Aktion finden Sie im <a href="https://msdn.microsoft.com/library/Ee416569(v=VS.85).aspx">Beispiel HDRToneMappingCS11,</a> das über den Beispielbrowser verfügbar ist.<br /> | 
| <span id="Multithreaded_Rendering"></span><span id="multithreaded_rendering"></span><span id="MULTITHREADED_RENDERING"></span>Multithread-Rendering<br /> | Der wichtigste API-Unterschied zu Direct3D 10 in Direct3D 11 ist das Addition von verzögerten <a href="overviews-direct3d-11-render-multi-thread.md">Kontexten,</a>das eine skalierbare Ausführung von Direct3D-Befehlen ermöglicht, die über mehrere Kerne verteilt sind. Ein verzögerter Kontext erfasst und fasst Aktionen wie Zustandsänderungen und Draw-Übermittlungen zusammen, die zu einem späteren Zeitpunkt auf dem tatsächlichen Gerät ausgeführt werden können. Durch die Verwendung von verzögerten Kontexten in mehreren Threads kann eine Anwendung den cpu-Mehraufwand, der in der Direct3D11-Runtime und im Treiber benötigt wird, auf mehrere Kerne verteilen, um eine bessere Verwendung der Computerkonfiguration eines Endbenutzers zu ermöglichen. Dieses Feature ist für die Verwendung auf der aktuellen Direct3D 10-Hardware sowie auf dem Referenzraster verfügbar. Eine Demonstration der API-Nutzung finden Sie im <a href="https://msdn.microsoft.com/library/Ee416570(v=VS.85).aspx">MultithreadedRendering11-Beispiel,</a> das über den Beispielbrowser verfügbar ist.<br /> | 
| <span id="Dynamic_Shader_Linkage_"></span><span id="dynamic_shader_linkage_"></span><span id="DYNAMIC_SHADER_LINKAGE_"></span>Dynamische Shaderverknüpfung <br /> | Um das Problem der kombinatoriellen Explosion zu beheben, das bei der Spezialisierung von Shadern auf Leistung auftritt, führt Direct3D 11 eine eingeschränkte Form der Laufzeit-Shaderverknüpfung ein, die eine nahezu optimale <a href="/windows/desktop/direct3dhlsl/overviews-direct3d-11-hlsl-dynamic-linking">Shaderspezialisierung</a> während der Ausführung einer Anwendung ermöglicht. Dies wird erreicht, indem die Implementierungen bestimmter Funktionen im Shadercode angegeben werden, wenn der Shader der Pipeline zugewiesen wird, sodass der Treiber native Shaderanweisungen schnell inline ausführen kann, anstatt den Treiber zu zwingen, die Zwischensprache mit der neuen Konfiguration in native Anweisungen neu zu kompilieren. Die Shaderentwicklung wird durch die Einführung von Klassen und Schnittstellen in HLSL verfügbar gemacht. Sehen Sie sich das Beispiel <a href="https://msdn.microsoft.com/library/Ee416565(v=VS.85).aspx">Dynamic Shader Linkage 11</a> an, das über den Beispielbrowser verfügbar ist.<br /> | 
| <span id="Windows_Advanced_Rasterizer__WARP_"></span><span id="windows_advanced_rasterizer__warp_"></span><span id="WINDOWS_ADVANCED_RASTERIZER__WARP_"></span>Windows Advanced Rasterizer (WARP)<br /> | WarP ist in diesem SDK über Direct3D 11 und schließlich auch über Direct3D 10.1 verfügbar und ist ein schneller Rasterizer mit mehreren Kernen, der vollständig mit Direct3D 10.1 kompatibel ist. Die Verwendung dieser Technologie ist so einfach wie das Übergeben D3D_DRIVER_TYPE_WARP-Flags bei der Geräteerstellung.<br /> | 
| <span id="Direct3D_10_and_Direct3D_11_on_Direct3D_9_Hardware__D3D10_Level_9_"></span><span id="direct3d_10_and_direct3d_11_on_direct3d_9_hardware__d3d10_level_9_"></span><span id="DIRECT3D_10_AND_DIRECT3D_11_ON_DIRECT3D_9_HARDWARE__D3D10_LEVEL_9_"></span>Direct3D 10 und Direct3D 11 auf Direct3D 9-Hardware (D3D10 Level 9)<br /> | In diesem SDK über Direct3D 11 und schließlich auch über Direct3D 10.1 verfügbar, kann die Direct3D-API die meisten Direct3D 9-Hardware sowie Direct3D 10-, Direct3D 10.1- und Direct3D 11-Hardware als Ziel verwenden. Dies wird erreicht, indem der Mechanismus auf Featureebene zur Verfügung steht, bei dem Hardware je nach Funktionalität in sechs Kategorien unterteilt wird: 9_1, 9_2, 9_3, 10_0, 10_1 und 11_0. Eine Karte erfüllt nur dann eine Funktionsebene, wenn sie vollständig mit dieser Ebene konform ist, und jede Ebene ist eine strenge Oberstufe der unteren Ebene. Die Funktionalität wird minimal emuliert, um sicherzustellen, dass keine unerwarteten Leistungssteigerungen auftreten. Daher sind Features wie Geometry Shader nicht für Direct3D 9-Levelziele verfügbar. <br /> | 
| <span id="Runtime_Binaries"></span><span id="runtime_binaries"></span><span id="RUNTIME_BINARIES"></span>Laufzeitbinärdateien<br /> | Alle Laufzeitbinärdateien, die in der Tech Preview von Direct3D 11 bereitgestellt werden und unter Windows 7 und Windows Vista SP1 verfügbar sind, werden mit dem SDK installiert und als "Beta"-Komponenten (d.h. D3D11_beta.DLL) bezeichnet. Alle Komponenten mit Betabeschriftung sind zeitbeschriftet. Um Projekte zum Auswerten dieser neuen Komponenten zu erstellen, müssen Sie einen Link zu den entsprechenden Importbibliotheken mit Betabeschriftung (d. h. D3D11_beta.lib) erstellen. Wenn Sie über eine PDC-Kopie von Windows 7 verfügen, sind die Header, Bibliotheken und PDBs, die im Windows SDK mit dem Build bereitgestellt werden, für die Entwicklung mithilfe der Direct3D 11-Komponenten geeignet, die in Windows 7 bereitgestellt werden. Reservieren Sie die Verwendung der Header, Bibliotheken und PDBs in diesem SDK für die hier bereitgestellten Betakomponenten.<br /> | 
| <span id="D3DX11"></span><span id="d3dx11"></span>D3DX11<br /> | D3DX11 unterstützt derzeit Texturlade-, Shaderkompilierungs-, Datenlade- und Arbeitsthreadfunktionen für Direct3D 11-Ressourcen. In Zukunft wird diese Komponente mehr der in D3DX10 verfügbaren Technologien bereitstellen. Die Shaderkompilierungsfunktion wird auch direkt über die Direct3D Compiler-Komponente bereitgestellt, die als Nächstes beschrieben wird.<br /> | 
| <span id="HLSL_and_Direct3D_Compiler"></span><span id="hlsl_and_direct3d_compiler"></span><span id="HLSL_AND_DIRECT3D_COMPILER"></span>HLSL- und Direct3D-Compiler<br /> | Der HLSL-Compiler verfügt über mehrere neue Features zur Unterstützung der neuen Technologien, die in Direct3D 11 verfügbar sind. Dies umfasst objektorientierte Programmierung über Schnittstellen und Klassen, eine direkte Indizierungssyntax für Ressourcenlasten und das Schlüsselwort "precise", um sicherzustellen, dass alle Mit einer bestimmten Variablen ausgeführten Vorgänge den strengen Gleitkommaregeln entsprechen. Fast alle neuen linguistischen Features verfügen über gültige Funktionen für vorhandene Shaderziele. Zusätzlich zur Unterstützung aller Direct3D 9-, Direct3D 10-, Direct3D 10.1- und Direct3D 11-Shader unterstützt der HLSL-Compiler auch die speziellen Ziele, die zum Schreiben von Shadern für Direct3D 10 Level 9-Ziele erforderlich sind. Der D3D-Compiler ist jetzt direkt außerhalb von D3DX10 und D3DX11 über D3DCompiler.H und D3DCompiler.lib zugänglich. Bei diesen neuen Dateien muss eine Anwendung keine Verknüpfung mit D3DX erstellen, um die Laufzeitkompilierung durchzuführen, und eine Anwendung muss den Compiler nicht enthalten, wenn nur D3DX-Funktionalität erforderlich ist.<br /> | 
| <span id="D3D11_Reference_Rasterizer"></span><span id="d3d11_reference_rasterizer"></span><span id="D3D11_REFERENCE_RASTERIZER"></span>D3D11 – Referenzraster<br /> | Der Referenzraster bietet eine standardmäßige Gold-Rasterungsimplementierung für die Auswertung von Direct3D 11-Features, die noch nicht in der Hardware verfügbar sind. Der Referenzraster wird auch als Möglichkeit bereitgestellt, die Genauigkeit einer bestimmten Hardwareimplementierung im Rasterungsstandard zu überprüfen. Der Referenzraster ist auf Richtigkeit und nicht auf Leistung ausgelegt. Um ein Referenzgerät zu erstellen, übergeben Sie einfach das D3D_DRIVER_TYPE_REFERENCE-Flag bei der Geräteerstellung. <br /> | 
| <span id="D3D11_SDK_Layers"></span><span id="d3d11_sdk_layers"></span><span id="D3D11_SDK_LAYERS"></span>D3D11 SDK-Ebenen<br /> | Direct3D11 SDK-Ebenen bieten einen Mechanismus zum Nachverfolgen des Betriebs der Direct3D 11-Runtime während der Entwicklung. Derzeit wird dies verwendet, um nützliche Debuginformationen zur Verfügung zu stellen, die nicht nur Fehler bei nicht ordnungsgemäßer Verwendung enthalten, sondern auch Warnungen, die die Verwendung der Laufzeit mit bewährten Methoden empfehlen und häufig ausführliche, nützliche Informationen für das Debuggen bereitstellen. Es wird dringend empfohlen, die Debugausgabe von D3D11 SDK-Ebenen während der Entwicklung jederzeit zu aktivieren, und eine Anwendung generiert während der Ausführung keine Debugausgabe, bevor sie freigegeben oder mit PIX für Windows für die Profilerstellung verwendet wird. Das Aktivieren der Debugebene ist so einfach wie das Übergeben D3D11_CREATE_DEVICE_DEBUG-Flags zum Zeitpunkt der Geräteerstellung. Entwicklern wird dringend empfohlen, Ebenen in Debugbuilds zu verwenden. Ebenen werden nicht für die Verwendung in Profil- oder Release-Builds empfohlen.<br /> | 
| <span id="New_Samples"></span><span id="new_samples"></span><span id="NEW_SAMPLES"></span>Neue Beispiele<br /> | Dieses Release enthält vier neue Beispiele.<br /><ul><li>Das <a href="https://msdn.microsoft.com/library/Ee416565(v=VS.85).aspx">Beispiel Dynamic Shader Linkage 11</a> veranschaulicht die Verwendung von ShaderModell 5-Shaderschnittstellen und Direct3D 11-Unterstützung für das Verknüpfen von Shaderschnittstellenmethoden zur Laufzeit.</li><li>Das <a href="https://msdn.microsoft.com/library/Ee416569(v=VS.85).aspx">HDRToneMappingCS11-Beispiel</a> veranschaulicht das Einrichten und Ausführen des Compute-Shaders (CS für kurze Zeit). Dies ist eines der interessantesten neuen Features von Direct3D 11. Obwohl das Beispiel nur den CS verwendet, um die HDR-Tonzuordnung zu implementieren, sollte das Konzept problemlos auf andere Nachbearbeitungsalgorithmen sowie allgemeinere Berechnungen erweitert werden.</li><li>Im <a href="https://msdn.microsoft.com/library/Ee416570(v=VS.85).aspx">MultithreadedRendering11-Beispiel</a> wird veranschaulicht, wie Sie das Rendering auf mehrere Threads mit sehr geringem Mehraufwand aufteilen.</li><li>Das neue <a href="https://msdn.microsoft.com/library/Ee416576(v=VS.85).aspx">SubD11-Beispiel</a> ähnelt dem SubD10-Beispiel im DirectX SDK. Es wurde jedoch erweitert, um drei neue Direct3D 11-Pipelinestufen zu nutzen: den Hüllen-Shader, den Tessellator und den Domänen-Shader.</li></ul> |
