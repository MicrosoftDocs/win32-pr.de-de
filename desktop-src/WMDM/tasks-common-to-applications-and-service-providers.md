# <a name="tasks-common-to-applications-and-service-providers"></a>Allgemeine Aufgaben für Anwendungen und Dienstanbieter

In diesem Abschnitt werden die folgenden Aufgaben beschrieben, die sowohl für Anwendungen als auch für Dienstanbieter gelten, die auf Windows Media Geräte-Manager.

-   [Beim Kompilieren](compiling-the-idl-files-supplied-with-the-sdk.md) der mit dem SDK bereitgestellten IDL-Dateien wird beschrieben, welche IDL-Dateien verwendet werden, um welche Headerdateien zu erstellen, wenn Sie die mit dem SDK bereitgestellten Headerdateien nicht verwenden möchten.
-   [Aktivieren der Protokollierung](enabling-logging.md) beschreibt ein Protokollierungsobjekt, das von Windows Media Geräte-Manager, um Daten aus einem ausgeführten Objekt zu protokollieren.
-   [Unter Verwenden von sicheren authentifizierten Kanälen](using-secure-authenticated-channels.md) werden sichere authentifizierte Kanäle beschrieben, die für die Verschlüsselung, Entschlüsselung und Komponenten- und Nachrichtenauthentifizierung verwendet werden. Alles, was auf Windows Media Geräte-Manager basiert, muss mindestens in der Lage sein, komponentenbasierte Authentifizierung durchzuführen.
-   [Unter Behandeln geschützter](handling-protected-content.md) Inhalte wird beschrieben, wie Ihre Anwendung Inhalte verarbeiten kann, die durch Windows Media Digital Rights Management (DRM) geschützt werden.
-   [Abrufen und Festlegen von Metadaten und Attributen](getting-and-setting-metadata-and-attributes.md) beschreibt, wie eine Anwendung Attribute und Metadaten für einen Speicher oder ein Gerät abrufen und festlegen kann.
