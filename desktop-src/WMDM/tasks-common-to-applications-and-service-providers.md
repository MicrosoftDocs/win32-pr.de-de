# <a name="tasks-common-to-applications-and-service-providers"></a>Allgemeine Aufgaben für Anwendungen und Dienstanbieter

In diesem Abschnitt werden die folgenden Aufgaben beschrieben, die für Anwendungen und Dienstanbieter, die auf Windows Media Device Manager basieren, gelten.

-   Wenn Sie die im SDK bereitgestellten [IDL-Dateien kompilieren](compiling-the-idl-files-supplied-with-the-sdk.md) , wird beschrieben, welche IDL-Dateien zum Erstellen der Header Dateien verwendet werden, wenn Sie sich entscheiden, die im SDK bereitgestellten Header Dateien nicht zu verwenden.
-   [Aktivieren der Protokollierung](enabling-logging.md) beschreibt ein von Windows Media Device Manager bereitgestelltes Protokollierungs Objekt zum Protokollieren von Daten aus einem laufenden Objekt.
-   [Mithilfe von sicheren authentifizierten Kanälen](using-secure-authenticated-channels.md) werden sichere authentifizierte Kanäle beschrieben, die für die Verschlüsselung, Entschlüsselung und die Authentifizierung von Komponenten und Nachrichten verwendet werden. Alles, was auf Windows Media-Device Manager aufbaut, muss mindestens die Komponenten Authentifizierung durchführen können.
-   [Behandlung geschützter Inhalte](handling-protected-content.md) : Beschreibt, wie Ihre Anwendung Inhalte verarbeiten kann, die von Windows Media Digital Rights Management (DRM) geschützt werden.
-   Das Abrufen [und Festlegen von Metadaten und Attributen](getting-and-setting-metadata-and-attributes.md) beschreibt, wie eine Anwendung Attribute und Metadaten für einen Speicher oder ein Gerät abrufen und festlegen kann.
