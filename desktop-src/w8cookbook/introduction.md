# <a name="introduction-compatibility-cookbook-for-windows"></a>Einführung (Compatibility Cookbook für Windows)

Windows 10, Windows 8, Windows 8.1 und Windows Server 2012 R2 stellen die neuesten Betriebssystemtechnologien und softwareentwicklungsplattformen für die Verwendung durch App-Entwickler und Unternehmen weltweit bereit. Im Rahmen der weiteren Verbesserung der Sicherheit, Zuverlässigkeit, Leistung und des Benutzer Erlebnisses von Windows hat Microsoft viele neue Features eingeführt, vorhandene Features verbessert und andere entfernt.

Obwohl das Ziel von Windows 10, Windows 8, Windows 8.1 und Windows Server 2012 R2 darin besteht, mit den meisten Ihrer Apps, die für bereits veröffentlichte Betriebssysteme geschrieben wurden, eine hohe Kompatibilität zu Gewähr alten, sind einige Kompatibilitäts Umbrüche aufgrund von Innovationen, verschärftes Sicherheit und erhöhter Zuverlässigkeit unvermeidlich. Insgesamt ist die Kompatibilität der neuen Betriebssysteme mit vorhandenen apps hoch.

Dieses *Cookbook* ist in drei Hauptabschnitte unterteilt: Windows 10, Windows 8.1 und Windows 8:

-   Windows 10
-   Windows 8.1 und Windows Server 2012 R2
-   Windows 8 und Windows Server 2012

Die folgenden Themen sind entweder in dieser Version des *Cookbooks* neu oder wurden aktualisiert:

-   Start der Windows-Abbild Datei (wimboot)
-   Zertifizierungskit für Windows-Apps

Wir laden Sie in diese Themen ein, um zu erfahren, wie Sie Ihre apps optimieren und die Vorteile dieser neuesten Version von Windows nutzen können.

Die Windows Compatibility Cookbook-Editionen, die frühere Versionen der Windows-und Windows Server-Betriebssysteme behandeln, finden Sie unter [Windows Vista und Windows Server 2008](/previous-versions/bb757005(v=msdn.10)) und [Windows 7 und Windows Server 2008 R2 Anwendungsqualität Cookbook](/windows/desktop/Win7AppQual/windows-7-application-quality-cookbook) .
