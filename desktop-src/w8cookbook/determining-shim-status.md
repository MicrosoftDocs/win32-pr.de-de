# <a name="determining-shim-status"></a>Bestimmen des Shim-Status

## <a name="platforms"></a>Plattformen

<dl> Clients-Windows 8  
Server-Windows Server 2012  
</dl>

## <a name="description"></a>BESCHREIBUNG

Windows 8 protokolliert Ereignisse, wenn für Apps aus verschiedenen Gründen ein Shim durchlaufen wird. Die einfachste Möglichkeit, um zu ermitteln, ob Ihre APP Shim ist, besteht darin, die Ereignisanzeige zu überprüfen. Wechseln Sie zu Anwendungs-und Dienst Protokolle \\ Microsoft Windows Application-Experience-Programm \\ Telemetrie.

## <a name="mitigation"></a>Minderung

Die beste Möglichkeit, das Shimmen zu erreichen, besteht darin, die ausführbare Datei umzubenennen oder die Hauptversion um 1 zu erhöhen (die ausführbare Datei neu zu kompilieren).
