# <a name="windows-8-and-windows-server-2012-client-and-server-compatibility"></a>Windows 8 und Windows Server 2012 Client- und Serverkompatibilität

In diesem Abschnitt werden Änderungen am Betriebssystem beschrieben, auf die Sie aufgrund der potenziellen Auswirkungen auf vorhandene Apps und der Art und Weise, wie neue Apps auf Clients, Servern oder Clients und Servern entworfen werden sollten, besondere Aufmerksamkeit erhalten sollten. Im Folgenden finden Sie die Liste der Themen, die in diesem Abschnitt behandelt werden, gruppiert nach Funktionsbereich:

**AppCompat**

-   Update der Regeln für die Erkennung von Sicherheits-Apps
-   Bestimmen des Shimstatus
-   Server-Apps müssen ohne die grafische Shell des Servers ausgeführt werden können.
-   RDS-Serverkomponenten werden aus Windows 8
-   Dateityp- und Protokollzuordnungsmodell
-   Desktopaktivitätenmoderator
-   .NET Framework 4.5 ist standard, und .NET Framework 3.5 ist optional.
-   Desktop-Apps sind nach dem Starten des Standardwebbrowsers möglicherweise nicht sichtbar.
-   Modus mit hohem Kontrast
-   App-Manifest (ausführbare Datei)
-   Das aktuelle Modell in der Warteschlange ist veraltet.
-   Szenarien des Programmkompatibilitäts-Assistenten für Windows 8
-   Desktop-Gadgets entfernt

**Storage und Dateisystem**

-   Update der Datenträgerkompatibilität im erweiterten Format (4K)
-   Schlanke Bereitstellung logischer Einheiten
-   Erweiterter Speicher ist jetzt für Windows Preinstallation Environment (WinPE) und Server-SKU optional.
-   Virtual Disk Service wechselt zur Windows Management Instrumentation (WMI)-basierten Windows Storage Verwaltungs-API
-   Benutzeroberfläche früherer Versionen für lokale Volumes entfernt
-   StorAHCI ersetzt MSAHCI.
-   Windows 7 Sicherung und Wiederherstellung veraltet
-   Ausgelagerte Datenübertragungen

**Andere**

-   Desktopfenster-Manager ist immer eingeschaltet
-   Direct2D unterstützt in Internet Explorer 9 kein Rendern in "rich"-Metadateien.
-   Änderungen an der DX9-Legacyhardwareunterstützung
-   MSAA ist nicht für Windows Store-Apps verfügbar.
-   Port 3 ist für NDIS 6.30-Treiber veraltet
