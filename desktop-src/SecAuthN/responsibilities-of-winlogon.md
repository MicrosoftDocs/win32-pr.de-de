<span data-ttu-id="0d3e7-104">[*Winlogon*](../secgloss/w-gly.md) besteht aus den folgenden Zuständigkeiten:</span><span class="sxs-lookup"><span data-stu-id="0d3e7-104">[*Winlogon*](../secgloss/w-gly.md) has the following responsibilities:</span></span>

-   <span data-ttu-id="0d3e7-105">Fenster Station und Desktop Schutz</span><span class="sxs-lookup"><span data-stu-id="0d3e7-105">Window station and desktop protection</span></span>

    <span data-ttu-id="0d3e7-106">Mit Winlogon wird der Schutz der Fenster Station und der entsprechenden Desktops festgelegt, um sicherzustellen, dass jeder ordnungsgemäß verfügbar ist.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-106">Winlogon sets the protection of the window station and corresponding desktops to ensure that each is properly accessible.</span></span> <span data-ttu-id="0d3e7-107">Im Allgemeinen bedeutet dies, dass das lokale System Vollzugriff auf diese Objekte hat und dass ein interaktiv angemeldeter Benutzer über Lesezugriff auf das Fenster Station-Objekt und den vollständigen Zugriff auf das Anwendungs Desktop Objekt verfügt.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-107">In general, this means that the local system will have full access to these objects and that an interactively logged-on user will have read access to the window station object and full access to the application desktop object.</span></span>

-   <span data-ttu-id="0d3e7-108">SAS-Standard Erkennung</span><span class="sxs-lookup"><span data-stu-id="0d3e7-108">Standard SAS recognition</span></span>

    <span data-ttu-id="0d3e7-109">Winlogon verfügt über spezielle Hooks für den User32-Server, mit denen er STRG + ALT + ENTF-SAS-Ereignisse ( [*Secure Attention Sequence*](../secgloss/s-gly.md) ) überwachen kann.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-109">Winlogon has special hooks into the User32 server that allow it to monitor CTRL+ALT+DEL [*secure attention sequence*](../secgloss/s-gly.md) (SAS) events.</span></span> <span data-ttu-id="0d3e7-110">Mit Winlogon werden diese SAS-Ereignis Informationen für Ginas zur Verwendung als SAS oder als Teil ihrer SAS verfügbar gemacht.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-110">Winlogon makes this SAS event information available to GINAs to use as their SAS, or as part of their SAS.</span></span> <span data-ttu-id="0d3e7-111">Im Allgemeinen sollte Ginas SASs selbst überwachen. Allerdings sollte jede [*Gina*](../secgloss/g-gly.md) mit der Standard-SAS (STRG + ALT + ENTF) als einer der von ihr erkannten SASs die Unterstützung für die Windows-Anmeldung verwenden, die zu diesem Zweck bereitgestellt wird.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-111">In general, GINAs should monitor SASs on their own; however, any [*GINA*](../secgloss/g-gly.md) that has the standard CTRL+ALT+DEL SAS as one of the SASs it recognizes should use the Winlogon support provided for this purpose.</span></span>

-   <span data-ttu-id="0d3e7-112">SAS-Routine Verteilung</span><span class="sxs-lookup"><span data-stu-id="0d3e7-112">SAS routine dispatching</span></span>

    <span data-ttu-id="0d3e7-113">Wenn Winlogon ein SAS-Ereignis findet oder eine SAS von der Gina an Winlogon übermittelt wird, wird der Status von Winlogon entsprechend festgelegt, und die Änderungen werden auf den Desktop der Windows-Anmeldung festgelegt, und eine der SAS-Verarbeitungsfunktionen der Gina wird aufgerufen.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-113">When Winlogon encounters a SAS event or when a SAS is delivered to Winlogon by the GINA, Winlogon sets the state accordingly, changes to the Winlogon desktop, and calls one of the SAS processing functions of the GINA.</span></span>

-   <span data-ttu-id="0d3e7-114">Laden von Benutzerprofilen</span><span class="sxs-lookup"><span data-stu-id="0d3e7-114">User profile loading</span></span>

    <span data-ttu-id="0d3e7-115">Wenn sich Benutzer anmelden, werden Ihre Benutzerprofile in die Registrierung geladen.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-115">When users log on, their user profiles are loaded into the registry.</span></span> <span data-ttu-id="0d3e7-116">Auf diese Weise können die Benutzer Prozesse den speziellen Registrierungsschlüssel HKEY \_ Current \_ User verwenden.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-116">In this way, the processes of the user can use the special registry key HKEY\_CURRENT\_USER.</span></span> <span data-ttu-id="0d3e7-117">Winlogon führt dies nach einer erfolgreichen Anmeldung, aber vor der Aktivierung der Shell für den neu angemeldeten Benutzer automatisch durch.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-117">Winlogon does this automatically after a successful logon but before activation of the shell for the newly logged-on user.</span></span>

-   <span data-ttu-id="0d3e7-118">Zuweisung der Sicherheit zur Benutzershell</span><span class="sxs-lookup"><span data-stu-id="0d3e7-118">Assignment of security to user shell</span></span>

    <span data-ttu-id="0d3e7-119">Wenn sich ein Benutzer anmeldet, ist die Gina für das Erstellen eines oder mehrerer anfänglicher Prozesse für diesen Benutzer verantwortlich.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-119">When a user logs on, the GINA is responsible for creating one or more initial processes for that user.</span></span> <span data-ttu-id="0d3e7-120">Winlogon bietet eine Unterstützungsfunktion für die Gina, um die Sicherheit des neu angemeldeten Benutzers auf diese Prozesse anzuwenden.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-120">Winlogon provides a support function for the GINA to apply the security of the newly logged-on user to these processes.</span></span> <span data-ttu-id="0d3e7-121">Die bevorzugte Methode hierfür ist jedoch, dass die Gina die Windows-Funktion " [**deateprocessasuser**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-createprocessasusera)" aufruft und das System den Dienst bereitstellen kann.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-121">However, the preferred way to do this is for the GINA to call the Windows function [**CreateProcessAsUser**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-createprocessasusera), and let the system provide the service.</span></span>

-   <span data-ttu-id="0d3e7-122">Bildschirmschoner-Steuerelement</span><span class="sxs-lookup"><span data-stu-id="0d3e7-122">Screen saver control</span></span>

    <span data-ttu-id="0d3e7-123">Winlogon überwacht die Tastatur-und Maus Aktivität, um zu bestimmen, wann Bildschirmschoner aktiviert werden sollen.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-123">Winlogon monitors keyboard and mouse activity to determine when to activate screen savers.</span></span> <span data-ttu-id="0d3e7-124">Nachdem der Bildschirmschoner aktiviert ist, überwacht Winlogon weiterhin die Tastatur-und Maus Aktivität, um zu bestimmen, wann der Bildschirmschoner beendet werden soll.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-124">After the screen saver is activated, Winlogon continues to monitor keyboard and mouse activity to determine when to terminate the screen saver.</span></span> <span data-ttu-id="0d3e7-125">Wenn der Bildschirmschoner als sicher markiert ist, wird die Arbeitsstation von Winlogon als gesperrt behandelt.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-125">If the screen saver is marked as secure, Winlogon treats the workstation as locked.</span></span> <span data-ttu-id="0d3e7-126">Wenn eine Maus-oder Tastatur Aktivität vorliegt, ruft Winlogon die [**wlxdisplaylockednotice**](/windows/desktop/api/Winwlx/nf-winwlx-wlxdisplaylockednotice) -Funktion des Gina-und gesperrten Arbeitsstations Verhaltens auf.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-126">When there is mouse or keyboard activity, Winlogon invokes the [**WlxDisplayLockedNotice**](/windows/desktop/api/Winwlx/nf-winwlx-wlxdisplaylockednotice) function of the GINA and locked workstation behavior resumes.</span></span> <span data-ttu-id="0d3e7-127">Wenn der Bildschirmschoner nicht sicher ist, beendet jede Tastatur-oder Maus Aktivität den Bildschirmschoner ohne Benachrichtigung an Gina.</span><span class="sxs-lookup"><span data-stu-id="0d3e7-127">If the screen saver is not secure, any keyboard or mouse activity terminates the screen saver without notification to the GINA.</span></span>

-   <span data-ttu-id="0d3e7-128">Unterstützung mehrerer Netzwerkanbieter</span><span class="sxs-lookup"><span data-stu-id="0d3e7-128">Multiple network provider support</span></span>
