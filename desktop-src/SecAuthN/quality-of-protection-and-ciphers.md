# <a name="quality-of-protection-and-ciphers"></a>Qualität von Schutz und Chiffren

Die Qualität des Schutzes bestimmt den Sicherheitsgrad, der beim Senden und empfangen von Nachrichten verwendet wird. Microsoft Digest wählt basierend auf den Anforderungen des Clients und des Servers eine Ebene aus.

Wenn Digest als SASL-Mechanismus verwendet wird, können Anwendungen angeben, dass Nachrichten mithilfe der Verschlüsselung vertraulich aufbewahrt werden. Der Server gibt die Chiffren an, die er für verschlüsselte Nachrichten unterstützt, und der Client wählt eine Chiffre aus dieser Liste aus.

In den folgenden Themen werden diese Sicherheitsmechanismen ausführlich erläutert:

-   [Qualität des Schutzes](quality-of-protection.md)
-   [Ciphers](ciphers.md)
