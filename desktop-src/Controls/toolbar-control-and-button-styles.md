# <a name="toolbar-control-and-button-styles"></a>ToolBar-Steuerelement und Schaltflächen Stile

Die folgenden Fenster Stile sind spezifisch für Symbolleisten. Sie werden mit anderen Fenster Stilen kombiniert, wenn die Symbolleiste erstellt wird.

**Hinweis** Für allgemeine Steuerelemente, [Version 6,00](common-control-versions.md), sind Schaltflächen unabhängig von der Stil Einstellung immer transparent, wenn ein [visueller Stil](themes-overview.md) mit der Symbolleiste verwendet wird. Andernfalls ist das Transparenz Verhalten normal, wie durch die Verwendung des Formats "tbstyle \_ Flat" oder "tbstyle transparent" angegeben \_ .

> [!Note]  
> Comctl32.dll Version 6 ist nicht Verteil Bar, aber Sie ist in Windows enthalten. Wenn Sie Comctl32.dll Version 6 verwenden möchten, geben Sie ihn in einem Manifest an. Weitere Informationen zu Manifesten finden Sie unter [Aktivieren von visuellen Stilen](cookbook-overview.md).

**Hinweis:** Eine Symbolleisten Schaltfläche kann eine Kombination der folgenden Stile aufweisen. Um einen Schaltflächen Stil anzugeben, legen Sie die entsprechenden Flags im **fsstyle** -Member der [**TBBUTTON**](/windows/desktop/api/Commctrl/ns-commctrl-tbbutton) -Struktur der Schaltfläche fest. Es können nicht alle Stile kombiniert werden.

Bei Shell-Version 4,72 und früher verfügten sowohl Fenster-als auch Schaltflächen-stilflags über das Format tbstyle \_ xxx. Wenn Sie eine Anwendung mit der [Version 4,72](common-control-versions.md) oder einer früheren Version von kommctrl. h kompilieren, müssen Sie die tbstyle xxx-Schaltflächen- \_ stilflags verwenden. Bei Version 5,80 und höher haben alle Schaltflächen Stile die Form btns \_ xxx. Alle tbstyle \_ xxx-Werte verfügen über äquivalente btns \_ xxx-Werte mit der gleichen Bedeutung und dem numerischen Wert. Zur einfacheren Verwendung werden beide Formen in der folgenden Tabelle angegeben.
