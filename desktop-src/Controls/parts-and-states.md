# <a name="parts-and-states"></a>Parts and States

In diesem Thema werden die Teile und Zustände beschrieben, mit denen Sie die Darstellung von Steuerelementen ändern, wenn visuelle Stile aktiviert sind. Windows zeichnet bestimmte Teile für Steuerelemente und Fenster, und diese Teile werden in übergeordnete und untergeordnete Teile unterteilt. Die meisten Teile haben einen Zustand, z. B. deaktiviert, aktiv oder ausgewählt. Wenn Sie ein Steuerelement erstellen, entscheiden Sie, auf welche Teile des Steuerelements visuelle Stile angewendet werden. Sie teilen das Steuerelement in einen Hauptteil und null oder mehr benannte untergeordnete Teile auf. Beispiele für Teile sind Textbereiche, gezeichnete Formen, Bilder und Linien. Für jeden Teil können Sie ein oder mehrere Hintergrundbilder in einer einzelnen Bitmapdatei definieren. Die verschiedenen Hintergründe sind in der Regel unterschiedlichen Zuzuständen des Steuerelements zugeordnet, müssen aber nicht sein. Wenn das Steuerelement beispielsweise deaktiviert ist, kann ein schattiertes Hintergrundbild verwendet werden. Zur Laufzeit kann das entsprechende Image ausgewählt werden, indem ein *iStateId-Wert* als 1-basierter Index für das richtige Bild verwendet wird.

> [!Note]  
> Ein Großteil der Design-API ermöglicht es Ihnen, den Wert (0,0) für den Teil und die Status-ID in der Parameterliste zu verwenden, am häufigsten beim Zeichnen des Hintergrunds mit [**DrawThemeBackground**](/windows/desktop/api/Uxtheme/nf-uxtheme-drawthemebackground). Dies gibt an, dass Daten aus einem Element auf höherer Ebene verwendet werden sollen. Teil 0 kann als "Daten, die für alle Teile dieses Elements identisch sind" interpretiert werden, und status 0 als "Daten, die für alle Zustände dieses Teils gleich sind". Verwenden *Sie stateId*=0 nur, wenn keine Zustände definiert sind.
