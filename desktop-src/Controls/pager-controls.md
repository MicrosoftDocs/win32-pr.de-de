# <a name="about-pager-controls"></a>Informationen zu Pager-Steuerelementen

Ein *Pager-Steuer* Element ist ein Fenster Container, der mit einem Fenster verwendet wird, das nicht über genügend Anzeigebereich verfügt, um den gesamten Inhalt anzuzeigen. Mit dem Pager-Steuerelement kann der Benutzer einen Bildlauf zu dem Bereich des Fensters durchführen, der derzeit nicht in der Ansicht angezeigt wird.

-   [Informationen zu Pager-Steuerelementen](#about-pager-controls)

## <a name="about-pager-controls"></a>Informationen zu Pager-Steuerelementen

Microsoft Internet Explorer Version 4,0 (commctrl.dll Version 4,71) führt das Pager-Steuerelement ein. Dieses Steuerelement ist in Situationen nützlich, in denen ein Fenster nicht über genügend Bereich verfügt, um ein untergeordnetes Fenster anzuzeigen. Wenn die Anwendung z. b. eine Symbolleiste hat, die nicht breit genug ist, um alle Elemente anzuzeigen, können Sie die Symbolleiste einem Pager-Steuerelement zuweisen, und Benutzer können einen Bildlauf nach links oder rechts durchführen, um auf alle Elemente zuzugreifen. Sie können auch Pager-Steuerelemente erstellen, die vertikal scrollen.

Ein Fenster, das dem Pager-Steuerelement zugewiesen ist, wird als *enthaltenes Fenster* bezeichnet.

Der folgende Screenshot zeigt eine Symbolleiste, die in einem Pager-Steuerelement enthalten ist. Das Pager-Steuerelement wird rot angezeigt, um anzuzeigen, welche Bereiche des Steuer Elements sichtbar sind.
