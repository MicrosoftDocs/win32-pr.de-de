# <a name="tree-view-control-window-styles"></a>Fenster Stile für Tree-View Steuerelement

In diesem Abschnitt werden die beim Erstellen von Strukturansicht-Steuerelementen verwendeten Fenster Stile aufgelistet.

| <span id="TVS_CHECKBOXES"></span><span id="tvs_checkboxes"></span><dl> <dt>**TV- \_ Kontrollkästchen**</dt> </dl>                | [Version 4,70](common-control-versions.md). Aktiviert Kontrollkästchen für Elemente in einem Strukturansicht-Steuerelement. Ein Kontrollkästchen wird nur angezeigt, wenn dem Element ein Bild zugeordnet ist. Wenn dieses Format festgelegt ist, verwendet das Steuerelement das [**drawframecontrol**](/windows/desktop/api/winuser/nf-winuser-drawframecontrol) -Steuerelement, um eine Status Bild Liste mit zwei Bildern zu erstellen und festzulegen. Status Abbildung 1 ist das Kontrollkästchen deaktiviert, und Status Abbildung 2 ist das Kontrollkästchen. Wenn Sie das Status Bild auf NULL festlegen, wird das Kontrollkästchen vollständig entfernt. Weitere Informationen finden Sie unter [Arbeiten mit Zustands Image Indizes](tree-view-controls.md).<br/> [Version 5,80](common-control-versions.md). Zeigt ein Kontrollkästchen an, auch wenn dem Element kein Bild zugeordnet ist.<br/> Nachdem ein Strukturansicht-Steuerelement mit diesem Stil erstellt wurde, kann der Stil nicht entfernt werden. Stattdessen müssen Sie das Steuerelement zerstören und eine neue erstellen. Durch das zerstören des Strukturansicht-Steuer Elements wird die Bildliste für den Kontrollkästchen Status nicht zerstört. Sie müssen Sie explizit zerstören. Sie können das Handle für die Status Bild Liste aufrufen, indem Sie das Strukturansicht-Steuerelement an eine [**TVM \_ GetImageList**](tvm-getimagelist.md) -Nachricht senden. Zerstören Sie dann die Bildliste mit der [**ImageList- \_ Zerstörung**](/windows/desktop/api/Commctrl/nf-commctrl-imagelist_destroy).<br/> Wenn Sie diesen Stil verwenden möchten, müssen Sie \_ nach dem Erstellen des TreeView-Steuer Elements und vor dem Auffüllen der Struktur den Stil für die TV-Kontrollkästchen mit " [**SetWindowLong**](/windows/desktop/api/winuser/nf-winuser-setwindowlonga) " festlegen. Andernfalls werden die Kontrollkästchen abhängig von den zeitlichen Problemen möglicherweise deaktiviert angezeigt.<br/> |
