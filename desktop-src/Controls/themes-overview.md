# <a name="visual-styles"></a>Visuelle Stile

## <a name="purpose"></a>Zweck

*Visuelle Stile* ändern die Darstellung allgemeiner Steuerelemente basierend auf dem vom Benutzer ausgewählten Design. Vor Windows 8 müssen Sie Ihre Anwendung speziell für die Verwendung visueller Stile konfigurieren. Andernfalls werden die allgemeinen Steuerelemente der Anwendung immer in dem Stil gerendert, der dem Windows klassischen Design zugeordnet ist, unabhängig vom aktuell ausgewählten Design. In Windows 8 können visuelle Stile nicht deaktiviert werden, Windows klassische Modus nicht mehr vorhanden ist und der Modus für hohen Kontrast so geändert wurde, dass er mit visuellen Stilen funktioniert.

Dieser Abschnitt bietet eine Übersicht über visuelle Stile und erläutert, wie Sie Ihre Anwendung für deren Verwendung konfigurieren.

## <a name="developer-audience"></a>Entwicklergruppe

Visuelle Stile sind für die Verwendung durch C/C++-Entwickler und Benutzeroberflächen-Designer konzipiert. Im Allgemeinen benötigen Entwickler ein mittleres Maß an Kenntnissen über Konzepte der Benutzeroberflächenprogrammierung, Windows API-Programmierung und Unicode.

## <a name="run-time-requirements"></a>Laufzeitanforderungen

Windows Vista- und höhere Betriebssysteme.

> [!Note]  
> Visuelle Stile werden im 256-Farbmodus nicht unterstützt.
