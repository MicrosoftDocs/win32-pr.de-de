# <a name="windows-portable-devices"></a>Windows Portable Geräte

Windows Portable Geräte (Portable Devices, WPD) ermöglichen Computern die Kommunikation mit angeschlossenen Medien und Speichergeräten. WPD bietet eine flexible, stabile Möglichkeit für Computer, mit Musikplayern, Speichergeräten, Mobiltelefonen, Kameras und vielen anderen Arten von verbundenen Geräten zu kommunizieren. Dieses System ersetzt sowohl die Windows Media Geräte-Manager als auch Windows Bilderfassung.

Anwendungen, die auf WPD aufgebaut sind, können ein Gerät untersuchen, Inhalte senden und empfangen und sogar das Gerät steuern, z. B. ein Bild oder eine SMS senden. Das System ist so konzipiert, dass es flexibel ist, sodass viele Arten von Geräten untersucht und erweiterbar werden können, sodass Treiberentwickler benutzerdefinierte Eigenschaften und Befehle für benutzerdefinierte Geräte definieren können.

Sie können sowohl Windows anwendungen als auch Webanwendungen mit WPD schreiben. Sie verwenden die WPD-Anwendungsprogrammierschnittstelle zum Erstellen Windows Anwendungen. Diese Anwendungen können in C++, C .NET oder \# Visual Basic .NET geschrieben werden. Sie verwenden das WPD Automation Object Model zum Schreiben von Webanwendungen. Diese Anwendungen werden in JScript HTML geschrieben.

Die WPD-Anwendungsprogrammierschnittstelle wird in den Betriebssystemen Windows 7, Windows Vista und Windows XP unterstützt. Das WPD Automation Object Model wird nur in Windows 7 unterstützt.

Weitere Informationen zum Erstellen einer WPD-Windows finden Sie in der [WpD Application Programming Interface-Dokumentation](./wpd_sdk/wpd-application-programming-interface.md) und in den Beispielen.

Weitere Informationen zum Erstellen einer WPD Automation-Anwendung finden Sie in der [WpD Automation Object Model-Dokumentation](/previous-versions/windows/desktop/legacy/dd389295(v=vs.85)) und in den Beispielen.
