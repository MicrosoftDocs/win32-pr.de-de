# <a name="priority-boosts"></a><span data-ttu-id="22b94-103">Prioritäts Steigerungen</span><span class="sxs-lookup"><span data-stu-id="22b94-103">Priority Boosts</span></span>

<span data-ttu-id="22b94-104">Jeder Thread verfügt über eine *dynamische Priorität*.</span><span class="sxs-lookup"><span data-stu-id="22b94-104">Each thread has a *dynamic priority*.</span></span> <span data-ttu-id="22b94-105">Dies ist die Priorität, die der Planer verwendet, um zu bestimmen, welcher Thread ausgeführt werden soll.</span><span class="sxs-lookup"><span data-stu-id="22b94-105">This is the priority the scheduler uses to determine which thread to execute.</span></span> <span data-ttu-id="22b94-106">Anfänglich ist die dynamische Priorität eines Threads mit der Basis Priorität identisch.</span><span class="sxs-lookup"><span data-stu-id="22b94-106">Initially, a thread's dynamic priority is the same as its base priority.</span></span> <span data-ttu-id="22b94-107">Das System kann die dynamische Priorität erhöhen und verringern, um sicherzustellen, dass Sie reaktionsfähig ist und dass keine Threads zur Prozessorzeit verhungert werden.</span><span class="sxs-lookup"><span data-stu-id="22b94-107">The system can boost and lower the dynamic priority, to ensure that it is responsive and that no threads are starved for processor time.</span></span> <span data-ttu-id="22b94-108">Das System erhöht die Priorität von Threads mit einer Basis Prioritätsstufe zwischen 16 und 31 nicht.</span><span class="sxs-lookup"><span data-stu-id="22b94-108">The system does not boost the priority of threads with a base priority level between 16 and 31.</span></span> <span data-ttu-id="22b94-109">Nur Threads mit einer Basis Priorität zwischen 0 und 15 empfangen dynamische Prioritäts Steigerungen.</span><span class="sxs-lookup"><span data-stu-id="22b94-109">Only threads with a base priority between 0 and 15 receive dynamic priority boosts.</span></span>

<span data-ttu-id="22b94-110">Das System erhöht die dynamische Priorität eines Threads, um die Reaktionsfähigkeit wie folgt zu verbessern.</span><span class="sxs-lookup"><span data-stu-id="22b94-110">The system boosts the dynamic priority of a thread to enhance its responsiveness as follows.</span></span>

-   <span data-ttu-id="22b94-111">Wenn ein Prozess, der die normale \_ Prioritäts \_ Klasse verwendet, in den Vordergrund gestellt wird, erhöht der Planer die Prioritäts Klasse des Prozesses, der dem Vordergrund Fenster zugeordnet ist, sodass er größer als oder gleich der Prioritäts Klasse von Hintergrundprozessen ist.</span><span class="sxs-lookup"><span data-stu-id="22b94-111">When a process that uses NORMAL\_PRIORITY\_CLASS is brought to the foreground, the scheduler boosts the priority class of the process associated with the foreground window, so that it is greater than or equal to the priority class of any background processes.</span></span> <span data-ttu-id="22b94-112">Die Prioritäts Klasse kehrt zur ursprünglichen Einstellung zurück, wenn sich der Prozess nicht mehr im Vordergrund befindet.</span><span class="sxs-lookup"><span data-stu-id="22b94-112">The priority class returns to its original setting when the process is no longer in the foreground.</span></span>
-   <span data-ttu-id="22b94-113">Wenn ein Fenster Eingaben empfängt, wie z. b. Zeit Geber Nachrichten, Maus Meldungen oder Tastatureingaben, erhöht der Planer die Priorität des Threads, der das Fenster besitzt.</span><span class="sxs-lookup"><span data-stu-id="22b94-113">When a window receives input, such as timer messages, mouse messages, or keyboard input, the scheduler boosts the priority of the thread that owns the window.</span></span>
-   <span data-ttu-id="22b94-114">Wenn die Warte Bedingungen für einen blockierten Thread erfüllt sind, erhöht der Scheduler die Priorität des Threads.</span><span class="sxs-lookup"><span data-stu-id="22b94-114">When the wait conditions for a blocked thread are satisfied, the scheduler boosts the priority of the thread.</span></span> <span data-ttu-id="22b94-115">Wenn z. b. ein warte Vorgang mit Datenträger-oder Tastatur-e/a abgeschlossen ist, erhält der Thread eine Prioritäts Erhöhung.</span><span class="sxs-lookup"><span data-stu-id="22b94-115">For example, when a wait operation associated with disk or keyboard I/O finishes, the thread receives a priority boost.</span></span>

    <span data-ttu-id="22b94-116">Sie können das Prioritäts Verstärkungs Feature deaktivieren, indem Sie die Funktion [**setprocesspriorityboost**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-setprocesspriorityboost) oder [**setthreadpriorityboost**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-setthreadpriorityboost) aufrufen.</span><span class="sxs-lookup"><span data-stu-id="22b94-116">You can disable the priority-boosting feature by calling the [**SetProcessPriorityBoost**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-setprocesspriorityboost) or [**SetThreadPriorityBoost**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-setthreadpriorityboost) function.</span></span> <span data-ttu-id="22b94-117">Um zu ermitteln, ob diese Funktion deaktiviert wurde, müssen Sie die [**getprocesspriorityboost**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-getprocesspriorityboost) -oder [**getthreadpriorityboost**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-getthreadpriorityboost) -Funktion aufrufen.</span><span class="sxs-lookup"><span data-stu-id="22b94-117">To determine whether this feature has been disabled, call the [**GetProcessPriorityBoost**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-getprocesspriorityboost) or [**GetThreadPriorityBoost**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-getthreadpriorityboost) function.</span></span>

<span data-ttu-id="22b94-118">Nach dem erhöhen der dynamischen Priorität eines Threads reduziert der Planer diese Priorität um eine Ebene, wenn der Thread einen Zeit Slice abschließt, bis der Thread auf seine Basis Priorität zurückfällt.</span><span class="sxs-lookup"><span data-stu-id="22b94-118">After raising a thread's dynamic priority, the scheduler reduces that priority by one level each time the thread completes a time slice, until the thread drops back to its base priority.</span></span> <span data-ttu-id="22b94-119">Die dynamische Priorität eines Threads ist nie kleiner als die Basis Priorität.</span><span class="sxs-lookup"><span data-stu-id="22b94-119">A thread's dynamic priority is never less than its base priority.</span></span>
