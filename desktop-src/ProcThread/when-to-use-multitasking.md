# <a name="when-to-use-multitasking"></a><span data-ttu-id="ba43f-103">Verwendung von Multitasking</span><span class="sxs-lookup"><span data-stu-id="ba43f-103">When to Use Multitasking</span></span>

<span data-ttu-id="ba43f-104">Multitasking kann auf zwei Arten implementiert werden: als einzelner Prozess mit mehreren Threads oder als mehrere Prozesse mit jeweils einem oder mehreren Threads.</span><span class="sxs-lookup"><span data-stu-id="ba43f-104">There are two ways to implement multitasking: as a single process with multiple threads or as multiple processes, each with one or more threads.</span></span> <span data-ttu-id="ba43f-105">Eine Anwendung kann jeden Thread, der einen privaten Adressraum und private Ressourcen benötigt, in einen eigenen Prozess versetzen, um ihn vor den Aktivitäten anderer Prozessthreads zu schützen.</span><span class="sxs-lookup"><span data-stu-id="ba43f-105">An application can put each thread that requires a private address space and private resources into its own process, to protect it from the activities of other process threads.</span></span>

<span data-ttu-id="ba43f-106">Ein Multithreadprozess kann sich gegenseitig ausschließende Aufgaben mit Threads verwalten, z. b. das Bereitstellen einer Benutzeroberfläche und das Ausführen von Hintergrund Berechnungen.</span><span class="sxs-lookup"><span data-stu-id="ba43f-106">A multithreaded process can manage mutually exclusive tasks with threads, such as providing a user interface and performing background calculations.</span></span> <span data-ttu-id="ba43f-107">Das Erstellen eines Multithread-Prozesses kann auch eine bequeme Möglichkeit zum Strukturieren eines Programms sein, das mehrere ähnliche oder identische Aufgaben gleichzeitig ausführt.</span><span class="sxs-lookup"><span data-stu-id="ba43f-107">Creating a multithreaded process can also be a convenient way to structure a program that performs several similar or identical tasks concurrently.</span></span> <span data-ttu-id="ba43f-108">Beispielsweise kann ein Named Pipe Server einen Thread für jeden Client Prozess erstellen, der an die Pipe angefügt wird.</span><span class="sxs-lookup"><span data-stu-id="ba43f-108">For example, a named pipe server can create a thread for each client process that attaches to the pipe.</span></span> <span data-ttu-id="ba43f-109">Dieser Thread verwaltet die Kommunikation zwischen dem Server und dem Client.</span><span class="sxs-lookup"><span data-stu-id="ba43f-109">This thread manages the communication between the server and the client.</span></span> <span data-ttu-id="ba43f-110">Der Prozess kann mehrere Threads verwenden, um die folgenden Aufgaben auszuführen:</span><span class="sxs-lookup"><span data-stu-id="ba43f-110">Your process could use multiple threads to accomplish the following tasks:</span></span>

-   <span data-ttu-id="ba43f-111">Verwalten von Eingaben für mehrere Fenster.</span><span class="sxs-lookup"><span data-stu-id="ba43f-111">Manage input for multiple windows.</span></span>
-   <span data-ttu-id="ba43f-112">Verwalten von Eingaben über verschiedene Kommunikationsgeräte</span><span class="sxs-lookup"><span data-stu-id="ba43f-112">Manage input from several communications devices.</span></span>
-   <span data-ttu-id="ba43f-113">Unterscheiden von Tasks unterschiedlicher Priorität.</span><span class="sxs-lookup"><span data-stu-id="ba43f-113">Distinguish tasks of varying priority.</span></span> <span data-ttu-id="ba43f-114">Beispielsweise verwaltet ein Thread mit hoher Priorität zeitkritische Tasks, und ein Thread mit niedriger Priorität führt andere Tasks aus.</span><span class="sxs-lookup"><span data-stu-id="ba43f-114">For example, a high-priority thread manages time-critical tasks, and a low-priority thread performs other tasks.</span></span>
-   <span data-ttu-id="ba43f-115">Gewährleisten einer reaktionsschnellen Benutzeroberfläche bei gleichzeitigem Zuweisen von Zeitsegmenten für im Hintergrund ausgeführten Tasks</span><span class="sxs-lookup"><span data-stu-id="ba43f-115">Allow the user interface to remain responsive, while allocating time to background tasks.</span></span>

<span data-ttu-id="ba43f-116">Aus den folgenden Gründen ist es in der Regel effizienter, wenn eine Anwendung Multitasking implementiert, indem ein einzelner Multithread-Prozess erstellt wird, anstatt mehrere Prozesse zu erstellen:</span><span class="sxs-lookup"><span data-stu-id="ba43f-116">It is typically more efficient for an application to implement multitasking by creating a single, multithreaded process, rather than creating multiple processes, for the following reasons:</span></span>

-   <span data-ttu-id="ba43f-117">Das System kann einen Kontextwechsel schneller für Threads ausführen als Prozesse, da ein Prozess mehr Aufwand als ein Thread hat (der Prozess Kontext ist größer als der Thread Kontext).</span><span class="sxs-lookup"><span data-stu-id="ba43f-117">The system can perform a context switch more quickly for threads than processes, because a process has more overhead than a thread does (the process context is larger than the thread context).</span></span>
-   <span data-ttu-id="ba43f-118">Alle Threads eines Prozesses verwenden denselben Adressraum und können auf die globalen Variablen des Prozesses zugreifen, wodurch die Kommunikation zwischen Threads vereinfacht werden kann.</span><span class="sxs-lookup"><span data-stu-id="ba43f-118">All threads of a process share the same address space and can access the process's global variables, which can simplify communication between threads.</span></span>
-   <span data-ttu-id="ba43f-119">Alle Threads eines Prozesses können offene Handles für Ressourcen freigeben, wie z. b. Dateien und Pipes.</span><span class="sxs-lookup"><span data-stu-id="ba43f-119">All threads of a process can share open handles to resources, such as files and pipes.</span></span>

<span data-ttu-id="ba43f-120">Es gibt weitere Techniken, die Sie anstelle von Multithreading verwenden können.</span><span class="sxs-lookup"><span data-stu-id="ba43f-120">There are other techniques you can use in the place of multithreading.</span></span> <span data-ttu-id="ba43f-121">Dies sind die folgenden: asynchrone Eingabe-und Ausgabe Vorgänge (e/a), e/a-Abschlussports, asynchrone Prozedur Aufrufe (APC) und die Möglichkeit, auf mehrere Ereignisse zu warten.</span><span class="sxs-lookup"><span data-stu-id="ba43f-121">The most significant of these are as follows: asynchronous input and output (I/O), I/O completion ports, asynchronous procedure calls (APC), and the ability to wait for multiple events.</span></span>

<span data-ttu-id="ba43f-122">Ein einzelner Thread kann mehrere zeitaufwändige e/a-Anforderungen initiieren, die gleichzeitig mithilfe asynchroner e/a-Vorgänge ausgeführt werden können.</span><span class="sxs-lookup"><span data-stu-id="ba43f-122">A single thread can initiate multiple time-consuming I/O requests that can run concurrently using asynchronous I/O.</span></span> <span data-ttu-id="ba43f-123">Asynchrone e/a-Vorgänge können für Dateien, Pipes und serielle Kommunikationsgeräte ausgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="ba43f-123">Asynchronous I/O can be performed on files, pipes, and serial communication devices.</span></span> <span data-ttu-id="ba43f-124">Weitere Informationen finden Sie unter [Synchronisierung und überlappende Eingabe und Ausgabe](../sync/synchronization-and-overlapped-input-and-output.md).</span><span class="sxs-lookup"><span data-stu-id="ba43f-124">For more information, see [Synchronization and Overlapped Input and Output](../sync/synchronization-and-overlapped-input-and-output.md).</span></span>

<span data-ttu-id="ba43f-125">Ein einzelner Thread kann seine eigene Ausführung blockieren, während darauf gewartet wird, dass ein oder alle von mehreren Ereignissen eintreten.</span><span class="sxs-lookup"><span data-stu-id="ba43f-125">A single thread can block its own execution while waiting for any one or all of several events to occur.</span></span> <span data-ttu-id="ba43f-126">Dies ist effizienter als die Verwendung mehrerer Threads, die jeweils auf ein einzelnes Ereignis warten, und effizienter als die Verwendung eines einzelnen Threads, der Prozessorzeit beansprucht, indem ständig überprüft wird, ob Ereignisse eintreten.</span><span class="sxs-lookup"><span data-stu-id="ba43f-126">This is more efficient than using multiple threads, each waiting for a single event, and more efficient than using a single thread that consumes processor time by continually checking for events to occur.</span></span> <span data-ttu-id="ba43f-127">Weitere Informationen finden Sie unter [Wait-Funktionen](../sync/wait-functions.md).</span><span class="sxs-lookup"><span data-stu-id="ba43f-127">For more information, see [Wait Functions](../sync/wait-functions.md).</span></span>
