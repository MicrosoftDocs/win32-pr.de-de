# <a name="about-processes-and-threads"></a><span data-ttu-id="3d34d-103">Informationen zu Prozessen und Threads</span><span class="sxs-lookup"><span data-stu-id="3d34d-103">About Processes and Threads</span></span>

<span data-ttu-id="3d34d-104">Jeder *Prozess* stellt die Ressourcen bereit, die zum Ausführen eines Programms benötigt werden.</span><span class="sxs-lookup"><span data-stu-id="3d34d-104">Each *process* provides the resources needed to execute a program.</span></span> <span data-ttu-id="3d34d-105">Ein Prozess verfügt über einen virtuellen Adressraum, einen ausführbaren Code, offene Handles für Systemobjekte, einen Sicherheitskontext, eine eindeutige Prozess-ID, Umgebungsvariablen, eine Prioritäts Klasse, minimale und maximale workingsetgrößen und mindestens einen Ausführungs Thread.</span><span class="sxs-lookup"><span data-stu-id="3d34d-105">A process has a virtual address space, executable code, open handles to system objects, a security context, a unique process identifier, environment variables, a priority class, minimum and maximum working set sizes, and at least one thread of execution.</span></span> <span data-ttu-id="3d34d-106">Jeder Prozess wird mit einem einzelnen Thread gestartet, der häufig als *primärer Thread* bezeichnet wird, kann jedoch zusätzliche Threads aus allen Threads erstellen.</span><span class="sxs-lookup"><span data-stu-id="3d34d-106">Each process is started with a single thread, often called the *primary thread*, but can create additional threads from any of its threads.</span></span>

<span data-ttu-id="3d34d-107">Ein *Thread* ist die Entität innerhalb eines Prozesses, die für die Ausführung geplant werden kann.</span><span class="sxs-lookup"><span data-stu-id="3d34d-107">A *thread* is the entity within a process that can be scheduled for execution.</span></span> <span data-ttu-id="3d34d-108">Alle Threads eines Prozesses verwenden den virtuellen Adressraum und die Systemressourcen gemeinsam.</span><span class="sxs-lookup"><span data-stu-id="3d34d-108">All threads of a process share its virtual address space and system resources.</span></span> <span data-ttu-id="3d34d-109">Außerdem werden von jedem Thread Ausnahmehandler, eine Planungs Priorität, ein Thread lokaler Speicher, ein eindeutiger Thread Bezeichner und ein Satz von Strukturen verwaltet, mit denen das System den Thread Kontext so lange speichert, bis er geplant ist.</span><span class="sxs-lookup"><span data-stu-id="3d34d-109">In addition, each thread maintains exception handlers, a scheduling priority, thread local storage, a unique thread identifier, and a set of structures the system will use to save the thread context until it is scheduled.</span></span> <span data-ttu-id="3d34d-110">Der *Thread Kontext* enthält den Thread Satz der Computer Register, den Kernel Stapel, einen Thread Umgebungsblock und einen Benutzer Stapel im Adressraum des Thread Prozesses.</span><span class="sxs-lookup"><span data-stu-id="3d34d-110">The *thread context* includes the thread's set of machine registers, the kernel stack, a thread environment block, and a user stack in the address space of the thread's process.</span></span> <span data-ttu-id="3d34d-111">Threads können auch über einen eigenen Sicherheitskontext verfügen, der zum Annehmen der Identität von Clients verwendet werden kann.</span><span class="sxs-lookup"><span data-stu-id="3d34d-111">Threads can also have their own security context, which can be used for impersonating clients.</span></span>

<span data-ttu-id="3d34d-112">Microsoft Windows unterstützt ein *präemptives Multitasking*, das die Auswirkungen der gleichzeitigen Ausführung mehrerer Threads aus mehreren Prozessen erzeugt.</span><span class="sxs-lookup"><span data-stu-id="3d34d-112">Microsoft Windows supports *preemptive multitasking*, which creates the effect of simultaneous execution of multiple threads from multiple processes.</span></span> <span data-ttu-id="3d34d-113">Auf einem Multiprozessorcomputer kann das System gleichzeitig so viele Threads ausführen, wie Prozessoren auf dem Computer vorhanden sind.</span><span class="sxs-lookup"><span data-stu-id="3d34d-113">On a multiprocessor computer, the system can simultaneously execute as many threads as there are processors on the computer.</span></span>

<span data-ttu-id="3d34d-114">Ein *Auftrags Objekt* ermöglicht das Verwalten von Gruppen von Prozessen als Einheit.</span><span class="sxs-lookup"><span data-stu-id="3d34d-114">A *job object* allows groups of processes to be managed as a unit.</span></span> <span data-ttu-id="3d34d-115">Auftrags Objekte sind namable-, Securable-und freigegeben-Objekte, die Attribute der zugeordneten Prozesse steuern.</span><span class="sxs-lookup"><span data-stu-id="3d34d-115">Job objects are namable, securable, sharable objects that control attributes of the processes associated with them.</span></span> <span data-ttu-id="3d34d-116">Die für das Auftrags Objekt ausgeführten Vorgänge wirken sich auf alle Prozesse aus, die dem Auftrags Objekt zugeordnet sind.</span><span class="sxs-lookup"><span data-stu-id="3d34d-116">Operations performed on the job object affect all processes associated with the job object.</span></span>

<span data-ttu-id="3d34d-117">Eine Anwendung kann den *Thread Pool* verwenden, um die Anzahl der Anwendungsthreads zu verringern und die Verwaltung der Arbeitsthreads zu ermöglichen.</span><span class="sxs-lookup"><span data-stu-id="3d34d-117">An application can use the *thread pool* to reduce the number of application threads and provide management of the worker threads.</span></span> <span data-ttu-id="3d34d-118">Anwendungen können Arbeitselemente in die Warteschlange stellen, Arbeitsaufgaben zuordnen, abnutzbare Handles zuordnen, automatisch basierend auf einem Timer in die Warteschlange eingereiht und mit e/a</span><span class="sxs-lookup"><span data-stu-id="3d34d-118">Applications can queue work items, associate work with waitable handles, automatically queue based on a timer, and bind with I/O.</span></span>

<span data-ttu-id="3d34d-119">Die *benutzermodusplanung* (ums) ist ein schlanker Mechanismus, mit dem Anwendungen ihre eigenen Threads planen können.</span><span class="sxs-lookup"><span data-stu-id="3d34d-119">*User-mode scheduling* (UMS) is a lightweight mechanism that applications can use to schedule their own threads.</span></span> <span data-ttu-id="3d34d-120">Eine Anwendung kann im Benutzermodus zwischen den Verarbeitungsthreads wechseln, ohne den [Systemplaner](scheduling.md) einzubeziehen, und die Steuerung des Prozessors wiederherzustellen, wenn ein ums-Thread im Kernel blockiert wird.</span><span class="sxs-lookup"><span data-stu-id="3d34d-120">An application can switch between UMS threads in user mode without involving the [system scheduler](scheduling.md) and regain control of the processor if a UMS thread blocks in the kernel.</span></span> <span data-ttu-id="3d34d-121">Jeder ums-Thread verfügt über einen eigenen Thread Kontext, anstatt den Thread Kontext eines einzelnen Threads freizugeben.</span><span class="sxs-lookup"><span data-stu-id="3d34d-121">Each UMS thread has its own thread context instead of sharing the thread context of a single thread.</span></span> <span data-ttu-id="3d34d-122">Durch die Möglichkeit, zwischen Threads im Benutzermodus zu wechseln, werden die ums effizienter als Thread Pools für kurze Arbeitsaufgaben, für die nur wenige Systemaufrufe erforderlich sind.</span><span class="sxs-lookup"><span data-stu-id="3d34d-122">The ability to switch between threads in user mode makes UMS more efficient than thread pools for short-duration work items that require few system calls.</span></span>

<span data-ttu-id="3d34d-123">Eine *Fiber* ist eine Ausführungs Einheit, die manuell von der Anwendung geplant werden muss.</span><span class="sxs-lookup"><span data-stu-id="3d34d-123">A *fiber* is a unit of execution that must be manually scheduled by the application.</span></span> <span data-ttu-id="3d34d-124">Fibers werden im Kontext der Threads ausgeführt, die Sie planen.</span><span class="sxs-lookup"><span data-stu-id="3d34d-124">Fibers run in the context of the threads that schedule them.</span></span> <span data-ttu-id="3d34d-125">Jeder Thread kann mehrere Fibers planen.</span><span class="sxs-lookup"><span data-stu-id="3d34d-125">Each thread can schedule multiple fibers.</span></span> <span data-ttu-id="3d34d-126">Im Allgemeinen bieten Fibers keine Vorteile gegenüber einer gut entworfenen Multithread-Anwendung.</span><span class="sxs-lookup"><span data-stu-id="3d34d-126">In general, fibers do not provide advantages over a well-designed multithreaded application.</span></span> <span data-ttu-id="3d34d-127">Die Verwendung von Fibers kann jedoch das Portieren von Anwendungen erleichtern, die zum Planen Ihrer eigenen Threads entworfen wurden.</span><span class="sxs-lookup"><span data-stu-id="3d34d-127">However, using fibers can make it easier to port applications that were designed to schedule their own threads.</span></span>
