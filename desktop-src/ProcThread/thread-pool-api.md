# <a name="thread-pool-api"></a>Thread Pool-API

Die Thread Pool-API (Application Programming Interface) verwendet einen objektbasierten Entwurf. Jedes der folgenden Objekte wird durch eine Benutzermodus-Datenstruktur dargestellt:

-   Ein Pool Objekt ist ein Satz von Arbeitsthreads, die verwendet werden können, um Arbeit auszuführen. Jeder Prozess kann mehrere isolierte Pools mit unterschiedlichen Merkmalen nach Bedarf erstellen. Es gibt auch einen Standard Pool für jeden Prozess.
-   Eine Bereinigungs Gruppe ist einem Satz von Objekten zugeordnet, die das Rückruf Objekt erstellen. Funktionen sind vorhanden, um auf alle Objekte zu warten, die Mitglieder jeder Bereinigungs Gruppe sind. Dadurch wird die Anwendung nicht mehr auf alle Objekte nachverfolgt, die Sie erstellt hat.
-   Ein Arbeits Objekt wird einem Pool und optional einer Bereinigungs Gruppe zugewiesen. Sie kann veröffentlicht werden und bewirkt, dass ein Arbeits Thread aus dem Pool seinen Rückruf ausführt. Ein Arbeits Objekt kann mehrere ausstehende Beiträge aufweisen; jede generiert einen Rückruf. Der Post-Vorgang kann aufgrund fehlender Ressourcen nicht ausgeführt werden.
-   Ein Zeit Geber Objekt steuert die Zeitplanung für Rückrufe. Jedes Mal, wenn ein Timer abläuft, wird der zugehörige Rückruf an seinen WorkerPool gesendet. Das Festlegen eines Timers kann aufgrund fehlender Ressourcen nicht möglich sein.
-   Ein Wait-Objekt bewirkt, dass ein kellnerthread auf ein abnutzbares handle wartet. Nachdem der Warte Vorgang durchgeführt wurde oder der Timeout Zeitraum abgelaufen ist, sendet der Kellner-Thread den Rückruf der Wait-Objekte an den WorkerPool der Warteschlange. Das Festlegen einer Wartezeit kann aufgrund fehlender Ressourcen nicht möglich sein.
-   Ein i/o-Objekt ordnet dem e/a-Abschlussport für den Thread Pool ein Datei Handle zu. Wenn ein asynchroner e/a-Vorgang abgeschlossen ist, übernimmt ein Arbeits Thread den Status des Vorgangs und ruft den Rückruf des I/o-Objekts auf.
