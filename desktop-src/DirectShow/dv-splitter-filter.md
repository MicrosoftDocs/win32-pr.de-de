## <a name="remarks"></a><span data-ttu-id="81a59-128">Bemerkungen</span><span class="sxs-lookup"><span data-stu-id="81a59-128">Remarks</span></span>

<span data-ttu-id="81a59-129">DV-Frames enthalten Audio und Video in demselben Frame.</span><span class="sxs-lookup"><span data-stu-id="81a59-129">DV frames contain audio and video in the same frame.</span></span> <span data-ttu-id="81a59-130">Der DV-Splitterfilter extrahiert die Audiodaten und übermittelt sie als einen oder zwei Audiostreams aus den Audioausgabepins.</span><span class="sxs-lookup"><span data-stu-id="81a59-130">The DV Splitter filter extracts the audio data and delivers it as one or two audio streams, from the audio output pins.</span></span> <span data-ttu-id="81a59-131">Der ursprüngliche DV-Frame wird vom Videoausgabepin als Videoframe übermittelt.</span><span class="sxs-lookup"><span data-stu-id="81a59-131">The original DV frame is delivered from the video output pin, as a video frame.</span></span> <span data-ttu-id="81a59-132">Der Medientyp im Videoframe wird von MEDIATYPE \_ Interleaved in MEDIATYPE \_ Video geändert, andernfalls werden die Daten jedoch nicht geändert.</span><span class="sxs-lookup"><span data-stu-id="81a59-132">The media type on the video frame is changed from MEDIATYPE\_Interleaved to MEDIATYPE\_Video, but otherwise the data is not modified.</span></span> <span data-ttu-id="81a59-133">Der Medientyp wird geändert, um zu signalisieren, dass die Audiodaten im Frame ignoriert werden sollen.</span><span class="sxs-lookup"><span data-stu-id="81a59-133">The media type is changed to signal that the audio data into the frame should be ignored.</span></span> <span data-ttu-id="81a59-134">Der DV-Splitter legt keine Medienzeit für die Ausgabebeispiele fest. <span data-ttu-id="81a59-135">Nur jeweils ein Ausgabepin macht die Schnittstellen [**IMediaPosition**](/windows/desktop/api/Control/nn-control-imediaposition) und [**IMediaSeeking**](/windows/desktop/api/Strmif/nn-strmif-imediaseeking) verfügbar.</span><span class="sxs-lookup"><span data-stu-id="81a59-135">Only one output pin at a time exposes the [**IMediaPosition**](/windows/desktop/api/Control/nn-control-imediaposition) and [**IMediaSeeking**](/windows/desktop/api/Strmif/nn-strmif-imediaseeking) interfaces.</span></span>
