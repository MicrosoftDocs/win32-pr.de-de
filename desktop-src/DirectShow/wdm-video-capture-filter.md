# <a name="wdm-video-capture-filter"></a>WDM-Videoaufnahmefilter

Der WDM Video Capture-Filter steuert analoge Erfassungsgeräte, die WDM-Treiber (Windows-Treibermodell) verwenden.

Dieser Filter ist eigentlich ein KsProxy-Plug-In im Kernelmodus. Sie stellt Eigenschaftenseiten und COM-Schnittstellen für WDM-Treiber bereit, die analoge Erfassungsgeräte (auch als analoge Videodecoder bezeichnet) steuern. Eine Anwendung kann sie einfach als Filter behandeln. Um diesen Filter einem Filterdiagramm hinzuzufügen, verwenden Sie den [Systemgeräte-Enumerator](system-device-enumerator.md). Für jedes Gerät, das dieses Plug-In verwendet, wird ein eindeutiger Moniker zurückgegeben. Weitere Informationen finden Sie unter [Enumerating Devices and Filters und](enumerating-devices-and-filters.md) How Hardware Devices Participate in the Filter [Graph](how-hardware-devices-participate-in-the-filter-graph.md). Wie bei jedem Filter, der auf KsProxy basiert, hängt der Anzeigename des Filters vom zugrunde liegenden Treiber ab.

Nicht alle Geräte, die über den WDM-Videoaufnahmefilter verfügbar gemacht werden, implementieren alle unten aufgeführten Schnittstellen. Die Anwendung kann **QueryInterface** verwenden, um zu bestimmen, welche Schnittstellen von einem bestimmten Gerät unterstützt werden.
