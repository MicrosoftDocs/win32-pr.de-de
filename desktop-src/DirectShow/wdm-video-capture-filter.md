# <a name="wdm-video-capture-filter"></a>WDM-Video Erfassungs Filter

Der WDM-Video Erfassungs Filter steuert die Verwendung von analogen Erfassungs Geräten, die Windows-Treibermodell (WDM)-Treiber verwenden.

Dieser Filter ist tatsächlich ein ksproxy-Plug-in im Kernel Modus. Sie stellt Eigenschaften Seiten und die COM-Schnittstellen für WDM-Treiber bereit, mit denen analoge Erfassungsgeräte (auch als analoge Video Decoder bezeichnet) gesteuert werden. Eine Anwendung kann Sie einfach als Filter behandeln. Verwenden Sie den [Enumerator "System Geräte](system-device-enumerator.md)", um diesen Filter einem Filter Diagramm hinzuzufügen. Es gibt einen eindeutigen Moniker für jedes Gerät zurück, das dieses Plug-in verwendet. Weitere Informationen finden Sie unter Auflisten von [Geräten und Filtern](enumerating-devices-and-filters.md) und [wie Hardware Geräte am Filter Diagramm teilnehmen](how-hardware-devices-participate-in-the-filter-graph.md). Wie bei jedem Filter, der auf ksproxy basiert, hängt der Anzeige Name des Filters vom zugrunde liegenden Treiber ab.

Nicht alle Geräte, die über den WDM-Video Erfassungs Filter verfügbar gemacht werden, implementieren alle unten aufgeführten Schnittstellen. Die Anwendung kann **QueryInterface** verwenden, um zu bestimmen, welche Schnittstellen von einem bestimmten Gerät unterstützt werden.
