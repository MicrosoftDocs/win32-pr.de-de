# <a name="distributed-routing-table-api"></a>Verteiltes Routing Tabellen-API

Mit der DRT-API (Distributed Routing Table) können Anwendungen numerische Schlüssel in einem Peernetzwerk veröffentlichen und auflösen.

Eine Anwendung, die DRT verwendet, kann Folgendes erreichen:

-   Erstellen sie anwendungsspezifische Tabellen für verteiltes Routing.
-   Registrieren Sie Schlüssel, und ordnen Sie diese Schlüssel IP-Adressen und Anwendungsdaten zu.
-   Suchen Sie nach Schlüsseln, und rufen Sie die ihnen zugeordneten IP-Adressen und Anwendungsdaten ab. Diese Funktionalität kann verwendet werden, um verteilte Hashtabellen (Distributed Hash Tables, DHTs), serverlose Namenauflösungssysteme und Überlagerungsnetznetzwerke vieler Topologien zu erstellen.

> [!Note]  
> Die Administratoren und Benutzer von DRT-fähigen Anwendungen sollten die Endbenutzer ihrer Anwendungen auf die veröffentlichten Informationen aufmerksam machen. Microsoft-Server werden von dieser Technologie nicht verwendet, und Informationen werden nicht an Microsoft gesendet.
