# <a name="peer-networking-scenarios"></a>Peer Netzwerk Szenarios

Die Peer-to-Peer-Infrastruktur bietet Ihnen viele Optionen zum Entwickeln von Peer-zu-Peer-Netzwerkanwendungen. Dieses Thema enthält Informationen zu den folgenden Peer-zu-Peer-Netzwerkszenarien:

-   Kollaborative Anwendungen
-   Instant Messaging-oder Chat Anwendungen
-   Anwendungen für die Inhalts Verteilung

## <a name="collaborative-applications"></a>Kollaborative Anwendungen

Mithilfe der Peer-to-Peer-Infrastruktur können Sie kollaborative Anwendungen erstellen, mit denen Benutzer interaktiv Ideen austauschen können, unabhängig davon, ob Sie sich im selben Konferenzraum oder über das Internet befinden. Beispielsweise können Sie eine freigegebene, mit einem Whiteboard ähnliche Anwendung erstellen.

> [!Note]  
> Die Peer-to-Peer-Infrastruktur unterstützt kein Audiostreaming oder Video Streaming.

 

## <a name="instant-messaging-or-chat-applications"></a>Instant Messaging-oder Chat Anwendungen

Instant Messaging-und Chat Anwendungen, die zentrale Server verwenden, können aktualisiert werden, um die Peer-to-Peer-Infrastruktur zu verwenden. Beispielsweise kann eine Chatanwendung die Peer-to-Peer-Infrastruktur als Alternative zu einem zentralisierten Server verwenden. Die Peer-to-Peer-Infrastruktur verbessert die Skalierbarkeit und ermöglicht Benutzern die Teilnahme an Chat-und Messaging Aktivitäten ohne Server.

## <a name="content-distribution-applications"></a>Anwendungen für die Inhalts Verteilung

Sie können die Peer-to-Peer-Infrastruktur als Verteilungsmechanismus für Inhalte verwenden. Beispielsweise können Sie einen Newsletter an Peers verteilen, die einen Dienst abonnieren. Da Daten in der Peer-zu-Peer-Infrastruktur synchronisiert sind, erhalten Abonnenten den Newsletter, auch wenn der Verleger nicht online ist.
