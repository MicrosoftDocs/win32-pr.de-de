# <a name="peer-networking-scenarios"></a>Peernetzwerkszenarien

Die Peer-zu-Peer-Infrastruktur bietet Ihnen viele Optionen für die Entwicklung von Peer-zu-Peer-Netzwerkanwendungen. Dieses Thema enthält Informationen zu den folgenden Peer-zu-Peer-Netzwerkszenarien:

-   Anwendungen für die Zusammenarbeit
-   Instant Messaging- oder Chatanwendungen
-   Inhaltsverteilungsanwendungen

## <a name="collaborative-applications"></a>Zusammenarbeitsanwendungen

Sie können die Peer-zu-Peer-Infrastruktur verwenden, um Anwendungen für die Zusammenarbeit zu erstellen, mit denen Benutzer interaktiv Ideen austauschen können, unabhängig davon, ob sie sich im selben Konferenzraum oder im Internet befinden. Beispielsweise können Sie eine freigegebene Whiteboard-ähnliche Anwendung erstellen.

> [!Note]  
> Die Peer-zu-Peer-Infrastruktur unterstützt kein Audio- oder Videostreaming.

 

## <a name="instant-messaging-or-chat-applications"></a>Instant Messaging- oder Chatanwendungen

Instant Messaging- und Chatanwendungen, die zentrale Server verwenden, können aktualisiert werden, um die Peer-zu-Peer-Infrastruktur zu verwenden. Beispielsweise kann eine Chatanwendung die Peer-zu-Peer-Infrastruktur als Alternative zu einem zentralisierten Server verwenden. Die Peer-zu-Peer-Infrastruktur verbessert die Skalierbarkeit und ermöglicht Benutzern die Teilnahme an Chat- und Messagingaktivitäten ohne Server.

## <a name="content-distribution-applications"></a>Inhaltsverteilungsanwendungen

Sie können die Peer-zu-Peer-Infrastruktur als Verteilungsmechanismus für Inhalte verwenden. Beispielsweise können Sie einen Newsletter an Peers verteilen, die einen Dienst abonnieren. Da Daten in der Peer-zu-Peer-Infrastruktur synchronisiert werden, erhalten Abonnenten den Newsletter auch dann, wenn der Herausgeber nicht online ist.
