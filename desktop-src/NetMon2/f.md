# <a name="f-network-monitor"></a>F (Netzwerkmonitor)

<dl> <dt>

<span id="_netmon_filter_blob_gly"></span><span id="_NETMON_FILTER_BLOB_GLY"></span>**FilterBLOB**
</dt> <dd>

Blob (Binary Large Object), das verwendet wird, um die Suche nach einer Netzwerkschnittstellenkarte (NIC) einzuschränken. Jeder Eintrag im FilterBLOB muss mit einem entsprechenden Eintrag im NPP-BLOB (Network Packet Provider) übereinstimmen, der die NIC darstellt. Beispielsweise kann eine Anwendung nur Ethernet-Karten oder Karten anfordern, die eine bestimmte Bildfrequenz unterstützen. Sie können ein Filterblob zum Aufrufen der **Funktionen GetNPPBlobFromUI** und **GetNPPBlobTable** verwenden.

</dd> <dt>

<span id="_netmon_follow_set_gly"></span><span id="_NETMON_FOLLOW_SET_GLY"></span>**follow set**
</dt> <dd>

Ein Protokollsatz, der einem Protokoll folgt. Netzwerkmonitor verwendet den folgenden Satz eines Protokolls, wenn der Parser das nächste Protokoll aus den Daten in der Erfassung nicht erkennen kann. Siehe auch [ *Handoffset.*](h.md)

</dd> <dt>

<span id="_netmon_frame_gly"></span><span id="_NETMON_FRAME_GLY"></span>**Rahmen**
</dt> <dd>

Ein Datenpaket, das als eine Einheit über ein Netzwerk übertragen wird. Jeder Frame folgt derselben grundlegenden Organisation und enthält Steuerungsinformationen wie das Synchronisieren von Zeichen, Stationsadressen, einen Fehlerüberprüfungswert und eine variable Datenmenge. Wird auch als Paket bezeichnet.

</dd> <dt>

<span id="_netmon_frame_viewer_window_gly"></span><span id="_NETMON_FRAME_VIEWER_WINDOW_GLY"></span>**Frame-Viewer-Fenster**
</dt> <dd>

Ein Netzwerkmonitor, in dem ausführliche Informationen zu einzelnen Frames angezeigt werden. Sie können mehrere Frame-Viewer-Fenster gleichzeitig öffnen.

</dd> </dl>
