# <a name="using-extensible-storage-engine"></a>Verwenden der erweiterbaren Storage-Engine


_**Gilt für:** Windows | Windows Server_

## <a name="using-extensible-storage-engine"></a>Verwenden der erweiterbaren Storage-Engine

Dieser Abschnitt enthält allgemeine Informationen zur Verwendung der folgenden Teile der ESE-API.

  - [Spalten](./columns.md)

  - [Indizierung in der Tabelle](./indexing-in-the-table.md)

  - [Erstellen von Datenbanken](./creating-databases.md)

  - [Transaktionen](./transactions.md)

  - [ESE-Fehler](./extensible-storage-engine-errors.md)

  - [ESE-Dateien](./extensible-storage-engine-files.md)

Ihre Programmquellendateien sollten die Headerdatei esent.h enthalten, um auf Funktionsprototypen und Strukturdefinitionen für die Extensible Storage Engine-API zuzugreifen. Entwickler können die Bibliotheksdatei esent.lib verwenden, um Anwendungen zu erstellen, die die Extensible Storage Engine-API verwenden. Zur Laufzeit sind Anwendungen mit dem esent.dll.
