# <a name="using-extensible-storage-engine"></a>Verwenden der Extensible Storage-Engine


_**Gilt für:** Windows | Windows Server_

## <a name="using-extensible-storage-engine"></a>Verwenden der Extensible Storage-Engine

Dieser Abschnitt enthält allgemeine Informationen zur Verwendung der folgenden Teile der ESE-API.

  - [Spalten](./columns.md)

  - [Indizierung in der Tabelle](./indexing-in-the-table.md)

  - [Erstellen von Datenbanken](./creating-databases.md)

  - [Transaktionen](./transactions.md)

  - [ESE-Fehler](./extensible-storage-engine-errors.md)

  - [ESE-Dateien](./extensible-storage-engine-files.md)

Die Programm Quelldateien sollten die Header Datei "ESENT. h" enthalten, um auf Funktionsprototypen und Strukturdefinitionen für die erweiterbare Speicher-Engine-API zuzugreifen. Entwickler können die Bibliothek Datei ESENT. lib verwenden, um Anwendungen zu erstellen, die die erweiterbare Speicher-Engine-API verwenden. Zur Laufzeit verknüpfen Anwendungen mit dem esent.dll.
