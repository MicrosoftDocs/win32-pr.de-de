# <a name="typeuri-element"></a>TypeUri-Element

Gibt einen Typ an, der aus einer XSD-Datei aufgenommen werden soll.

## <a name="usage"></a>Verbrauch

``` syntax
<typeUri
  type = "character_string"
  uri = "character_string"/>
```

## <a name="attributes"></a>Attribute



| Attribut           | type                         | Erforderlich       | BESCHREIBUNG                                                            |
|---------------------|------------------------------|----------------|------------------------------------------------------------------------|
| **type**<br/> | Zeichen \_ Folge<br/> | Ja<br/> | Der Name des Typs.<br/> <br/>                           |
| **uri**<br/>  | Zeichen \_ Folge<br/> | Ja<br/> | Der Namespace des Typs. Muss ein gültiger URI sein.<br/> <br/> |



## <a name="child-elements"></a>Untergeordnete Elemente

Es gibt keine untergeordneten Elemente.

## <a name="parent-elements"></a>Übergeordnete Elemente



| Element                       | BESCHREIBUNG                                                                       |
|-------------------------------|-----------------------------------------------------------------------------------|
| [**XSD**](xsd.md)<br/> | Gibt eine XSD-Datei an, die für Vertragsinformationen verarbeitet werden soll.<br/> <br/> |
