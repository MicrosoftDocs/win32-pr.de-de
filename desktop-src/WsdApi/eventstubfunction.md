# <a name="eventstubfunction-element"></a>eventStubFunction-Element

Gibt an, ob Stubfunktionsverweise in die Vorgangsstrukturen in den Porttypdefinitionen für Benachrichtigungsvorgänge eingeschlossen werden sollen.

## <a name="usage"></a>Verbrauch

``` syntax
<eventStubFunction/>
```

## <a name="attributes"></a>Attribute

Es gibt keine Attribute.

## <a name="child-elements"></a>Untergeordnete Elemente

Es gibt keine untergeordneten Elemente.

## <a name="parent-elements"></a>Übergeordnete Elemente



| Element                                                       | BESCHREIBUNG                                                  |
|---------------------------------------------------------------|--------------------------------------------------------------|
| [**portTypeDefinitions**](porttypedefinitions.md)<br/> | Generiert C-Konstanten für Porttypen.<br/> <br/> |



## <a name="remarks"></a>Hinweise

Stubfunktionsverweise befinden sich in Clientszenarien für Benachrichtigungsvorgänge (Ereignisse).

Gültige Werte für dieses Element sind 1 (TRUE/Stubfunktionsverweise eingeschlossen) und 0 (FALSE/keine Stubfunktionsverweise enthalten).

|-------------------------------------|---------------|
