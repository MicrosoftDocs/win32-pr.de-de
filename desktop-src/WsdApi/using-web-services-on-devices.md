# <a name="using-web-services-on-devices"></a>Verwenden von Webdiensten auf Geräten

Die Microsoft Web Services on Devices-API (WSDAPI) unterstützt die Implementierung von clientgesteuerten Geräten und Diensten sowie Gerätehosts, die dem [Geräteprofil für Webdienste (Devices Profile for Web Services,](https://specs.xmlsoap.org/ws/2006/02/devprof/) DPWS) entsprechen. WSDAPI kann für die Entwicklung von Client- und Serverimplementierungen (Geräteimplementierungen) verwendet werden.

Verwenden Sie nach Möglichkeit WSDAPI, um einen Windows-basierten WSD-Client oder eine Gerätehostanwendung zu erstellen. Die Verwendung von WSDAPI reduziert die Komplexität der Entwicklung und spart Zeit. Weitere Informationen finden Sie unter [WSD-Anwendungsentwicklung auf Windows](wsd-application-development-on-windows.md) und [WSDAPI-Entwicklungstools.](wsdapi-development-tools.md)

Sie können auch WSD-Geräte erstellen, die nicht Windows ausgeführt werden und mit Windows-basierten WSD-Clients und -Gerätehosts interoperabel sind. Diese Geräte müssen bestimmte Anforderungen erfüllen, um die Interoperabilität aufrechtzuerhalten. Weitere Informationen finden Sie unter [WSD-Geräteentwicklung.](wsd-device-development.md)
