<span data-ttu-id="ff563-104">[WS-Discovery](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf) definiert die logische Adressierung mithilfe von URIs basierend auf dem `urn:uuid:` Format.</span><span class="sxs-lookup"><span data-stu-id="ff563-104">[WS-Discovery](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf) defines logical addressing using URIs based on the `urn:uuid:` format.</span></span> <span data-ttu-id="ff563-105">Der Zweck dieses Adressierungs Schemas besteht darin, die Identität des Geräts von seiner aktuellen IP-Adresse zu unterscheiden.</span><span class="sxs-lookup"><span data-stu-id="ff563-105">The purpose of this addressing scheme is to differentiate the identity of the device from its current IP address.</span></span> <span data-ttu-id="ff563-106">Dieses Schema bietet im Wesentlichen die Funktionalität von DNS-Namen, ohne dass ein Namensserver erforderlich ist.</span><span class="sxs-lookup"><span data-stu-id="ff563-106">This scheme essentially provides the functionality of DNS names without requiring a name server.</span></span> <span data-ttu-id="ff563-107">[Geräte Profil für Webdienste](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS) empfiehlt, dass Geräte dieses Adressierungs Schema verwenden.</span><span class="sxs-lookup"><span data-stu-id="ff563-107">[Devices Profile for Web Services](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS) recommends that devices use this addressing scheme.</span></span>

<span data-ttu-id="ff563-108">Außerdem wird von DPWS empfohlen, dass Dienste physische (auch Transport-) Adressen verwenden.</span><span class="sxs-lookup"><span data-stu-id="ff563-108">DPWS also recommends that services use physical (also called transport) addresses.</span></span> <span data-ttu-id="ff563-109">Dies ermöglicht es Clients, die keine systemeigene Unterstützung für WS-Discovery Adressierungs Mechanismen zur Kommunikation mit den DPWS-Diensten verwenden.</span><span class="sxs-lookup"><span data-stu-id="ff563-109">This enables clients that do not natively support WS-Discovery addressing mechanisms to communicate with DPWS services.</span></span> <span data-ttu-id="ff563-110">Außerdem kann jeder Dienst seine Adressen definieren, was die Adressierung von Transport Ebenen für Geräte Implementierungen ermöglicht, die die Dienst Verteilung auf einer niedrigeren Ebene verwalten.</span><span class="sxs-lookup"><span data-stu-id="ff563-110">Also, each service can define its addresses, which enables transport level addressing for device implementations that manage service dispatch at a lower layer.</span></span> <span data-ttu-id="ff563-111">Zum Schluss maximiert die Verwendung physischer Adressen die Interoperabilität.</span><span class="sxs-lookup"><span data-stu-id="ff563-111">Finally, using physical addresses maximizes interoperability.</span></span>

<span data-ttu-id="ff563-112">Der Nachteil der physischen Adressierung besteht darin, dass die Geräte Implementierungen komplexer werden, da die aktuelle IP-Adresse oder Transport Adresse nachverfolgt werden muss und Geräte Metadaten entsprechend geändert werden müssen.</span><span class="sxs-lookup"><span data-stu-id="ff563-112">The disadvantage of physical addressing is that it adds complexity to device implementations, as the current IP or transport address must be tracked and device metadata must be modified accordingly.</span></span> <span data-ttu-id="ff563-113">Aus diesem Grund ist es für DPWS nicht erforderlich, dass für die Verwendung von Transport Adressen Dienste verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="ff563-113">For this reason, DPWS does not require services to use transport addresses.</span></span>

<span data-ttu-id="ff563-114">Wenn logische Adressen verwendet werden, gibt es einige Szenarien, in denen das Implementierungs Verhalten nicht definiert ist.</span><span class="sxs-lookup"><span data-stu-id="ff563-114">If logical addresses are used, then there are some scenarios where the implementation behavior is undefined.</span></span> <span data-ttu-id="ff563-115">In der WS-Discovery Spezifikation wird nicht beschrieben, was es bedeutet, dass sich ein Dienst bei einer logischen Adresse befinden muss.</span><span class="sxs-lookup"><span data-stu-id="ff563-115">The WS-Discovery specification does not describe what it means for a service to reside at a logical address.</span></span> <span data-ttu-id="ff563-116">R1001 der WS-Discovery Spezifikation empfiehlt die Verwendung von WS-Discovery für gehostete Dienste aufgrund der zugeordneten Netzwerk Chatter.</span><span class="sxs-lookup"><span data-stu-id="ff563-116">R1001 of the WS-Discovery specification recommends against using WS-Discovery on hosted services because of the associated network chatter.</span></span>

<span data-ttu-id="ff563-117">Es wird nicht empfohlen, dass sich Dienste an logischen Adressen befinden, da dadurch die Interoperabilität verringert wird.</span><span class="sxs-lookup"><span data-stu-id="ff563-117">It is not recommended that services reside at logical addresses as this reduces interoperability.</span></span> <span data-ttu-id="ff563-118">Wenn eine Implementierung sich unbedingt an einer logischen Adresse befinden muss, sollte der Dienst dieselbe logische Adresse wie das Gerät verwenden.</span><span class="sxs-lookup"><span data-stu-id="ff563-118">If an implementation absolutely must reside at a logical address, then the service should use the same logical address as the device.</span></span> <span data-ttu-id="ff563-119">Wenn dies dem dispatchmodell auf dem Gerät zu viel Komplexität hinzufügt, empfiehlt es sich, die Dienste mithilfe von Verweis Parametern zu unterscheiden.</span><span class="sxs-lookup"><span data-stu-id="ff563-119">If this adds too much complexity to the dispatch model on the device, then the recommended solution is to use reference parameters to differentiate the services.</span></span> <span data-ttu-id="ff563-120">WSDAPI sendet Nachrichten ordnungsgemäß an-Dienste, wenn Sie dieselbe Endpunkt Adresse wie das Gerät verwenden.</span><span class="sxs-lookup"><span data-stu-id="ff563-120">WSDAPI will correctly send messages to services if they use the same endpoint address as the device.</span></span>
