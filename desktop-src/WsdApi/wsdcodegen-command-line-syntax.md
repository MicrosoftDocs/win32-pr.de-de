# <a name="wsdcodegen-command-line-syntax"></a>WsdCodeGen-Befehlszeilensyntax

WsdCodeGen verfügt über zwei Funktionen: das Generieren von Konfigurationsdateien und das Generieren von Quellcode für WSDAPI-Client- und -Hostanwendungen. In diesem Thema wird die Befehlszeilensyntax angegeben, die zum Ausführen der einzelnen Aufgaben verwendet wird.

## <a name="generating-a-configuration-file"></a>Generieren einer Konfigurationsdatei

### <a name="syntax"></a>Syntax

**WSDCODEGEN.EXE** **/generateconfig:**{**client** \| **host** \| **all**} **\[ /outputfile:**_ConfigFileName_ *_\]_* *WSDLFileNameList*

### <a name="parameters"></a>Parameter

<dl> <dt>

<span id="_generateconfig__client___host___all_"></span><span id="_GENERATECONFIG__CLIENT___HOST___ALL_"></span>**/generateconfig:{client \| host \| all}**
</dt> <dd>

Der Codetyp, den die Ausgabekonfigurationsdatei generiert. **/generateconfig:client** wird verwendet, um eine Konfigurationsdatei zum Generieren von Clientcode zu generieren, **/generateconfig:host** wird verwendet, um eine Konfigurationsdatei zum Generieren von Hostcode zu generieren, und **/generateconfig:all** wird verwendet, um eine Konfigurationsdatei zum Generieren von Client- und Hostcode zu generieren.

</dd> <dt>

<span id="_outputfile_ConfigFileName"></span><span id="_outputfile_configfilename"></span><span id="_OUTPUTFILE_CONFIGFILENAME"></span>**/outputfile:**_ConfigFileName_
</dt> <dd>

Dieser optionale Parameter wird verwendet, um den Dateinamen der Ausgabekonfigurationsdatei anzugeben. Wenn dieser Parameter ausgeschlossen wird, wird der Name der Ausgabekonfigurationsdatei codegen.config.

</dd> <dt>

<span id="_pnpx"></span><span id="_PNPX"></span>*/pnpx*
</dt> <dd>

Schließen Sie eine PnP-X-Vorlage in die Konfigurationsdatei ein.

</dd> <dt>

<span id="WSDLFileNameList"></span><span id="wsdlfilenamelist"></span><span id="WSDLFILENAMELIST"></span>*WSDLFileNameList*
</dt> <dd>

Eine durch Leerzeichen getrennte Liste der WSDL-Dateien, die von WsdCodeGen verarbeitet werden sollen.

</dd> </dl>

## <a name="generating-source-code"></a>Generieren von Quellcode

### <a name="syntax"></a>Syntax

**WSDCODEGEN.EXE** **/generatecode** **\[ /download \]** **\[ /gbc \]** **\[ outputroot:**_path_ *_\]_* **\[ /writeaccess:**_befehl_ *_\]_* *ConfigFileName*

### <a name="parameters"></a>Parameter

<dl> <dt>

<span id="_generatecode"></span><span id="_GENERATECODE"></span>**/generatecode**
</dt> <dd>

Leitet WsdCodeGen an, Quellcode zu generieren. Dies ist der Standardmodus, wenn kein Modus angegeben ist.

</dd> <dt>

<span id="_download"></span><span id="_DOWNLOAD"></span>**/download**
</dt> <dd>

Lädt importierte Dokumente herunter, auf die von der Konfigurationsdatei verwiesen wird. Dieser Parameter ist optional.

</dd> <dt>

<span id="_gbc"></span><span id="_GBC"></span>**/gbc**
</dt> <dd>

Fügt dem Quellcode Kommentare hinzu, die darauf hindeutet, dass der Code generiert wurde. Diesen Kommentaren wird der Ausdruck "Generated by" vorangestellt. Dieser Parameter ist optional.

</dd> <dt>

<span id="_outputroot_path"></span><span id="_OUTPUTROOT_PATH"></span>**/outputroot:**_pfad_
</dt> <dd>

Der Ausgabespeicherort für generierte Dateien. *path* kann ein absoluter oder relativer Pfad sein. Dieser Parameter ist optional.

</dd> <dt>

<span id="_writeaccess_command"></span><span id="_WRITEACCESS_COMMAND"></span>**/writeaccess:**_Befehl_
</dt> <dd>

Leitet WsdCodeGen an, den angegebenen Befehl auszuführen, bevor vorhandene Dateien auf dem Datenträger modifiziert werden. Ausgabedateien, die mit denen auf dem Datenträger identisch sind, erhalten diesen Befehl nicht und werden auch nicht in sie geschrieben. Wenn der Befehl die Sequenz "" enthält, wird diese Sequenz durch den Dateinamen der zu ändernden {0} Datei ersetzt. Wenn dies nicht der Wert ist, wird der Dateiname an den Befehl angefügt.

Beispiele:

**/writeaccess:"attrib -r"**

**/writeaccess:"attrib -r {0} "**

**/writeaccess:"copy {0} .. \\ backup \\ "**

</dd> <dt>

<span id="ConfigFileName"></span><span id="configfilename"></span><span id="CONFIGFILENAME"></span>*ConfigFileName*
</dt> <dd>

Der Name der Konfigurationsdatei, die vor dem Generieren von Code zu verarbeiten ist.

</dd> </dl>

## <a name="formatting-legend"></a>Formatierungslegende



| Format                                                                    | Bedeutung                                                 |
|---------------------------------------------------------------------------|---------------------------------------------------------|
| *Kursiv*                                                                  | Informationen, die der Benutzer bereitstellen muss                  |
| **Fett**                                                                  | Elemente, die der Benutzer genau so eingeben muss, wie sie angezeigt werden       |
| Zwischen eckigen Klammern ( \[ \] )                                                   | Optionale Elemente                                          |
| Zwischen geschweiften Klammern ( {} ); durch die Pipe getrennte Optionen ( \| ). Beispiel: {even \| odd} | Auswahlmöglichkeiten, aus denen der Benutzer nur eine auswählen muss |
