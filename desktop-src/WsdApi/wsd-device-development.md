# <a name="wsd-device-development"></a>WSD-Geräteentwicklung

Gerätehersteller und andere Geräte implementierer können Geräte erstellen, die mit Windows WSDAPI-Clients und -Hosts zusammenarbeiten, ohne WSDAPI zu verwenden.

Zusätzlich zur Entwicklung der Gerätefunktionalität müssen Geräte implementierer das [Geräteprofil für Webdienste](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DEVICE Profile for Web Services, DPWS) implementieren.

Verwenden Sie nach Möglichkeit WSDAPI, um eine Windows WSD-Client- oder Gerätehostanwendung zu erstellen. Die Verwendung von WSDAPI reduziert die Komplexität der Entwicklung und spart Zeit. Weitere Informationen finden Sie unter [WSD-Anwendungsentwicklung auf Windows](wsd-application-development-on-windows.md).
