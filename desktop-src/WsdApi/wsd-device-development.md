# <a name="wsd-device-development"></a>WSD-Geräteentwicklung

Gerätehersteller und andere geräteimplementierer können Geräte erstellen, die mit Windows-basierten WSDAPI-Clients und-Hosts interagieren, ohne WSDAPI zu verwenden.

Zusätzlich zum Entwickeln der Gerätefunktionalität müssen Geräte Implementierungen das [Geräte Profil für Webdienste](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS) implementieren.

Verwenden Sie nach Möglichkeit WSDAPI zum Erstellen einer Windows-basierten WSD-Client-oder Geräte Host Anwendung. Die Verwendung von WSDAPI reduziert die Komplexität der Entwicklung und spart Zeit. Weitere Informationen finden Sie unter [WSD-Anwendungsentwicklung unter Windows](wsd-application-development-on-windows.md).
