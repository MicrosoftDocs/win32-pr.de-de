# <a name="security-principals"></a>Sicherheitsprinzipale

In Windows 2000 ist ein Sicherheits Prinzipal ein Benutzer, eine Gruppe oder ein Computer – eine Entität, die vom Sicherheitssystem erkannt wird. Dies schließt sowohl menschliche Benutzer als auch autonome Prozesse ein. Streng genommen kann das Sicherheitssystem den Unterschied zwischen den angemeldeten Benutzern und Prozessen, die auf dem Computer ausgeführt werden, nicht erkennen. Sie sehen beide als Sicherheits Prinzipale mit Sicherheits Prinzipal Namen.

Benutzer, Gruppen und Computer werden als Objekte in Active Directory Domain Services erstellt und gespeichert. Außerdem gibt es bekannte Sicherheits Prinzipale, die spezielle Identitäten darstellen, die vom Windows 2000-Sicherheitssystem definiert werden, wie z. b. "alle", "Lokales System", "Prinzipal selbst", "authentifizierter Benutzer", "Ersteller-Besitzer Objekte, die die bekannten Sicherheits Prinzipale darstellen, wie z. b. die anonyme Anmeldung, werden im Container WellKnown Security Principals unterhalb des Konfigurations Containers gespeichert.
