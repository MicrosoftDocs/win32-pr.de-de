# <a name="overview-of-change-tracking-techniques"></a><span data-ttu-id="ed208-103">Übersicht über Änderungsnachverfolgung Techniken</span><span class="sxs-lookup"><span data-stu-id="ed208-103">Overview of Change Tracking Techniques</span></span>

<span data-ttu-id="ed208-104">Es gibt mehrere Möglichkeiten, wie sich Änderungs nach Verfolgungs Mechanismen unterscheiden können:</span><span class="sxs-lookup"><span data-stu-id="ed208-104">There are several ways that change tracking mechanisms can differ:</span></span>

-   <span data-ttu-id="ed208-105">Bereich für die Nachverfolgung von Änderungen: eine Anwendung kann Änderungen an einem einzelnen Attribut eines einzelnen Objekts, an allen Objekten in einer Domäne usw. nachverfolgen.</span><span class="sxs-lookup"><span data-stu-id="ed208-105">Scope for tracking changes: An application can track changes to a single attribute of a single object, to all objects in a domain, and so on.</span></span> <span data-ttu-id="ed208-106">Wenn der Mechanismus den Anforderungen der Anwendung entspricht, erhält die Anwendung mindestens irrelevante Daten, wodurch sich die Leistung verbessert.</span><span class="sxs-lookup"><span data-stu-id="ed208-106">If the mechanism matches the requirements of the application, the application receives a minimum of irrelevant data, and thus this enhances performance.</span></span>
-   <span data-ttu-id="ed208-107">Aktualität: eine Anwendung wird bei jeder Änderung benachrichtigt, oder Sie kann über einen Zeitraum von Minuten oder Stunden über den Nettoeffekt von Änderungen benachrichtigt werden.</span><span class="sxs-lookup"><span data-stu-id="ed208-107">Timeliness: An application is notified of every change as it happens, or can be notified of the net effect of changes over a period of minutes or hours.</span></span>

    <span data-ttu-id="ed208-108">Die Verarbeitung weniger zeitgemäßer Daten ist möglicherweise effizienter, da ggf. einige Änderungen reduziert werden.</span><span class="sxs-lookup"><span data-stu-id="ed208-108">Processing less timely data may be more efficient, because several changes may be collapsed into one.</span></span> <span data-ttu-id="ed208-109">Wenn ein Attribut z. b. dreimal innerhalb eines Zeitraums von einer Stunde geändert wird, wird eine Anwendung, die über eine Stunde kumulierte Änderungen benachrichtigt wird, nur über eine Attribut Änderung benachrichtigt, nicht über drei.</span><span class="sxs-lookup"><span data-stu-id="ed208-109">For example, if an attribute changes three times within a one hour interval, an application notified of changes accumulated over an hour will be notified of just one attribute change, not three.</span></span>

    <span data-ttu-id="ed208-110">Beachten Sie bei der Betrachtung der Aktualität die Auswirkung der Replikations Latenz.</span><span class="sxs-lookup"><span data-stu-id="ed208-110">When thinking about timeliness, consider the effect of replication latency.</span></span> <span data-ttu-id="ed208-111">Ein Update, das von einem Domänen Controller stammt, wird nicht sofort auf einen anderen Domänen Controller repliziert.</span><span class="sxs-lookup"><span data-stu-id="ed208-111">An update that originates on one domain controller does not replicate to another domain controller instantly.</span></span> <span data-ttu-id="ed208-112">Das Verlangen der Zeit, dass die Änderungs Nachverfolgung wesentlich besser ist als die erwartete Replikations Wartezeit, bietet der Anwendung häufig keinen echten Vorteil.</span><span class="sxs-lookup"><span data-stu-id="ed208-112">Requiring change-tracking timeliness much better than the expected replication latency often gives no real benefit to the application.</span></span>

-   <span data-ttu-id="ed208-113">Abruf und Benachrichtigung: bei der Abfrage sendet eine Anwendung regelmäßig eine Anforderung an einen Domänen Controller, um Änderungs nach Verfolgungs Daten zu empfangen.</span><span class="sxs-lookup"><span data-stu-id="ed208-113">Polling versus notification: With polling, an application periodically makes a request to a domain controller to receive change tracking data.</span></span> <span data-ttu-id="ed208-114">Bei einer Benachrichtigung sendet der Domänen Controller Änderungen nur dann an die Anwendung, wenn Änderungen auftreten.</span><span class="sxs-lookup"><span data-stu-id="ed208-114">With notification the domain controller sends changes to the application only when changes occur.</span></span>

    <span data-ttu-id="ed208-115">Der Aufwand für den Abruf ist offensichtlich: die Anwendung kann Änderungs nach Verfolgungs Daten anfordern, wenn nichts signifikanter ist.</span><span class="sxs-lookup"><span data-stu-id="ed208-115">The overhead of polling is obvious: The application may request change tracking data when nothing significant has occurred.</span></span> <span data-ttu-id="ed208-116">Der Aufwand für die Benachrichtigung ist sehr gering.</span><span class="sxs-lookup"><span data-stu-id="ed208-116">The overhead of notification is more subtle.</span></span> <span data-ttu-id="ed208-117">Der Server muss Daten zu Benachrichtigungs Anforderungen verwalten und muss diese Daten überprüfen, um zu entscheiden, ob eine Benachrichtigung gesendet werden soll.</span><span class="sxs-lookup"><span data-stu-id="ed208-117">The server must maintain data about notification requests and must consult this data to decide whether or not to send a notification.</span></span> <span data-ttu-id="ed208-118">Dies kann zu normalen Aktualisierungs Anforderungen mehr Aufwand verursachen.</span><span class="sxs-lookup"><span data-stu-id="ed208-118">This can add overhead to normal update requests.</span></span>

-   <span data-ttu-id="ed208-119">Ausdrücken des Wissens der Anwendung: persistent und temporär: jeder Mechanismus für die Änderungs Nachverfolgung muss eine Methode für den Server mit den verfolgten Daten enthalten, um den Zustand des Wissens der Anwendung zu verstehen, sodass die Vorstellung von "ändern" wohl definiert ist.</span><span class="sxs-lookup"><span data-stu-id="ed208-119">Expressing the application's knowledge: persistent versus temporary: Every change tracking mechanism must include some method for the server holding the data tracked to understand the application's state of knowledge, so that the idea of "change" is well defined.</span></span> <span data-ttu-id="ed208-120">Beispielsweise kann der Zustand der Anwendung als "aktualisiert mit allen Änderungen, die vor der Zeit t auf DC d aufgetreten sind" ausgedrückt werden.</span><span class="sxs-lookup"><span data-stu-id="ed208-120">For example, the application's state of knowledge might be expressed as "Updated with all changes that occurred on DC d before time t."</span></span> <span data-ttu-id="ed208-121">Ein Mechanismus, der auf diesem Weg zum Ausdrücken des Wissens Wissens einer Anwendung basiert, bietet eine effiziente Möglichkeit für die Anwendung, Änderungen zu erhalten, die nach einem bestimmten Zeitpunkt aufgetreten sind.</span><span class="sxs-lookup"><span data-stu-id="ed208-121">A mechanism based on this way of expressing an application's state of knowledge would provide an efficient way for the application to obtain changes that have occurred later than a specified time.</span></span>

    <span data-ttu-id="ed208-122">Wenn der Ausdruck des Wissens der Anwendung persistent gespeichert werden kann, d. h., wie in einer Datei oder Datenbank, ist die Anwendungs Neustarts weniger ressourcenintensiv, als wenn dies nicht möglich ist.</span><span class="sxs-lookup"><span data-stu-id="ed208-122">If the expression of the application's knowledge can be persisted, that is, stored recoverably, as in a file or database, application restart is less resource intensive than if it cannot.</span></span> <span data-ttu-id="ed208-123">Im obigen Beispiel kann der Ausdruck des Wissens der Anwendung persistent gespeichert werden, indem der DC d und die Uhrzeit t aufgezeichnet werden.</span><span class="sxs-lookup"><span data-stu-id="ed208-123">In the example above, the expression of the application's knowledge can be persisted by recording the DC d and the time t.</span></span> <span data-ttu-id="ed208-124">Einige Änderungs Benachrichtigungsmechanismen lassen nicht zu, dass diese Daten persistent gespeichert werden.</span><span class="sxs-lookup"><span data-stu-id="ed208-124">Some change notification mechanisms do not allow this data to be persisted.</span></span> <span data-ttu-id="ed208-125">Der Server und die Anwendung müssen mit einem anderen Mechanismus synchronisiert werden, wenn die Anwendung gestartet wird.</span><span class="sxs-lookup"><span data-stu-id="ed208-125">The server and application must synchronize with some other mechanism when the application starts.</span></span> <span data-ttu-id="ed208-126">Dies ist ressourcenintensiv, wenn mehrere Objekte beteiligt sind und eine komplexe Programmierung umfassen können.</span><span class="sxs-lookup"><span data-stu-id="ed208-126">This is resource-intensive if multiple objects are involved, and can involve complex programming.</span></span>

<span data-ttu-id="ed208-127">Verwenden Sie die folgenden Verfahren, um Änderungen in Active Directory Domain Services zu verfolgen:</span><span class="sxs-lookup"><span data-stu-id="ed208-127">Use the following techniques to track changes in Active Directory Domain Services:</span></span>

<span data-ttu-id="ed208-144">Die Dirsync-und die aktualisierbaren Suchtechniken sind für Anwendungen konzipiert, die die Konsistenz zwischen den Daten auf dem Active Directory Server und den entsprechenden Daten in einem anderen Speicher aufrechterhalten.</span><span class="sxs-lookup"><span data-stu-id="ed208-144">The DirSync and USNChanged search techniques are designed for applications that maintain consistency between data on the Active Directory server and corresponding data in some other storage.</span></span> <span data-ttu-id="ed208-145">Diese Techniken werden von Anwendungen verwendet, die regelmäßig Änderungen Abfragen.</span><span class="sxs-lookup"><span data-stu-id="ed208-145">These techniques are used by applications that periodically poll for changes.</span></span> <span data-ttu-id="ed208-146">Die Dirsync-Technik basiert auf einem LDAP-Server Steuerelement, das Sie über ADSI-oder LDAP-APIs verwenden können.</span><span class="sxs-lookup"><span data-stu-id="ed208-146">The DirSync technique is based on an LDAP server control that you can use through ADSI or LDAP APIs.</span></span> <span data-ttu-id="ed208-147">Die Nachteile des Dirsync-Steuer Elements bestehen darin, dass es nur von einem Konto mit hohen Berechtigungen verwendet werden kann, z. b. einem Domänen Administrator.</span><span class="sxs-lookup"><span data-stu-id="ed208-147">The disadvantages of the DirSync control are that it can only be used by a highly privileged account, such as a domain administrator.</span></span> <span data-ttu-id="ed208-148">Im folgenden finden Sie eine Liste der Einschränkungen des Dirsync-Steuer Elements:</span><span class="sxs-lookup"><span data-stu-id="ed208-148">The following is a list of limitations of DirSync control:</span></span>

-   <span data-ttu-id="ed208-150">Das DirSync-Steuerelement kann nur einen gesamten namens Kontext überwachen.</span><span class="sxs-lookup"><span data-stu-id="ed208-150">The DirSync control can only monitor an entire naming context.</span></span> <span data-ttu-id="ed208-151">Sie können den Gültigkeitsbereich einer Dirsync-Suche nicht einschränken, um nur eine bestimmte Unterstruktur, einen Container oder ein Objekt in einem namens Kontext zu überwachen.</span><span class="sxs-lookup"><span data-stu-id="ed208-151">You cannot limit the scope of a DirSync search to monitor only a specific subtree, container, or object in a naming context.</span></span>
