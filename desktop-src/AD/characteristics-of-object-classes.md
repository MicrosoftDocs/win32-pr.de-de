# <a name="characteristics-of-object-classes"></a>Merkmale von Objektklassen

Jede Objektklasse in Active Directory Domain Services wird durch ein **classSchema-Objekt** im Schemacontainer definiert. Die Attribute eines **classSchema-Objekts** geben die Merkmale der Klasse an, z. B.:

-   Klassenbezeichner: Klassen verfügen über mehrere Bezeichner, einschließlich **ldapDisplayName,** die von LDAP-Clients verwendet werden, um die Klasse in Suchfiltern zu identifizieren, und **schemaIDGUID**, die in Sicherheitsdeskriptoren verwendet werden, um den Zugriff auf die Klasse zu steuern.
-   Mögliche Attribute: Eine Objektklassendefinition enthält Listen der obligatorischen und optionalen Attribute, die für eine Instanz der -Klasse festgelegt werden können.
-   Mögliche übergeordnete Objekte: Jede Objektinstanz, mit Ausnahme des Stamms der Verzeichnishierarchie, hat genau ein übergeordnetes Element. Eine Objektklassendefinition enthält Listen möglicher über-Elemente, d.&a0;B. der Objektklassen, die eine Instanz der -Klasse enthalten können.
-   Oberklassen und Hilfsklassen: Jede Objektklasse (mit Ausnahme [*von top)*](/previous-versions/windows/desktop/legacy/ms681941(v=vs.85))wird von einer anderen Klasse abgeleitet. Eine Klasse erbt mögliche Attribute und mögliche übergeordnete Klassen von den klassen darüber in der Klassenhierarchie. Eine Klasse kann auch eine beliebige Anzahl von Hilfsklassen haben, von denen sie Listen möglicher Attribute erbt. Weitere Informationen finden Sie unter [Klassenvererbung im Active Directory-Schema.](class-inheritance-in-the-active-directory-schema.md)

In der folgenden Tabelle sind **der lDAPDisplayName** und die Beschreibung der Schlüsselattribute eines **classSchema-Objekts** aufgeführt. Weitere Informationen und eine vollständige Liste der obligatorischen und optionalen Attribute eines **classSchema-Objekts** finden Sie unter [**classSchema**](/windows/desktop/ADSchema/c-classschema).

Der <strong>cn</strong> der <strong>organizationalUnit-Objektklasse</strong> ist z. B. Organizational-Unit, der in einem Distinguished Name als CN=Organizational-Unit angezeigt wird. Der <strong>cn muss</strong> im Schemacontainer eindeutig sein. |

Der <strong>lDAPDisplayName</strong> einer Klasse muss im Schemacontainer eindeutig sein, was bedeutet, dass er für alle <strong>classSchema-</strong> und <strong>attributeSchema-Objekte eindeutig sein</strong> muss. Weitere Informationen zum Erstellen eines <strong>cn</strong> und eines <strong>lDAPDisplayName</strong> für eine neue Klasse finden Sie unter <a href="naming-attributes-and-classes.md">Naming Attributes and Classes</a>. |

Dieser Anzeigename wird nur verwendet, wenn in der <strong>classDisplayName-Eigenschaft</strong> des Anzeigespezifizierers für die Klasse keine Zuordnung vorhanden ist. Dieser Wert muss unter den <strong>governsIDs</strong> aller <strong>classSchema-Objekte</strong> und den <strong>attributeIDs</strong> aller <strong>attributeSchema-Objekte eindeutig</strong> sein. Weitere Informationen finden Sie unter <a href="object-identifiers.md">Objektbezeichner.</a> | 
| <strong>rDnAttId</strong> | Identifiziert das Benennungsattribut, bei dem es sich um das Attribut handelt, das den RDN für diese Klasse an stellt, wenn es sich vom Standardwert (<strong>cn ) abzieht.</strong> Von der Verwendung eines anderen Benennungsattributs als <strong>cn</strong> wird abgeraten. Benennungsattribute sollten aus der bekannten Gruppe (OU, CN, O, L und DC) abgerufen werden, die von allen LDAP-Clients der Version 3 verstanden wird. Weitere Informationen finden Sie unter <a href="object-names-and-identities.md">Objektnamen und Identitäten</a> und <a href="syntaxes-for-attributes-in-active-directory-domain-services.md">Syntaxen für Attribute in Active Directory Domain Services.</a> Ein Benennungsattribut muss über die Syntax Verzeichniszeichenfolge verfügen. Weitere Informationen finden Sie unter <a href="syntaxes-for-attributes-in-active-directory-domain-services.md">Syntaxes for Attributes in Active Directory Domain Services.</a><br /> | 
| <strong>mustContain</strong>, <strong>systemMustContain</strong> | Ein Paar von mehrwertigen Eigenschaften, die die Attribute angeben, die in Instanzen dieser Klasse vorhanden sein müssen. Dies sind obligatorische Attribute, die während der Erstellung vorhanden sein müssen und nach der Erstellung nicht mehr löschen können. Nach der Erstellung der -Klasse können diese Eigenschaften nicht mehr geändert werden. Der vollständige Satz obligatorischer Attribute für eine Klasse ist die Vereinigung der <strong>systemMustContain-</strong> und <strong>mustContain-Werte</strong> für diese Klasse und alle geerbten Klassen.<br /> | 
| <strong>mayContain</strong>, <strong>systemMayContain</strong> | Ein Paar von mehrwertigen Eigenschaften, die die Attribute angeben, die in Instanzen dieser Klasse vorhanden sein können. Dies sind optionale Attribute, die nicht obligatorisch sind und daher möglicherweise in einer Instanz dieser Klasse vorhanden sind. Sie können <strong>mayContain-Werte</strong> zu einem vorhandenen <strong>ClassSchema-Objekt</strong> der Kategorie 1 oder Kategorie 2 hinzufügen oder daraus entfernen. Bevor Sie einen <strong>mayContain-Wert</strong> aus einem <strong>classSchema-Objekt</strong> entfernen, sollten Sie nach Instanzen der Objektklasse suchen und alle Werte für das Attribut löschen, das Sie entfernen. Nach der Erstellung der -Klasse kann die <strong>systemMayContain-Eigenschaft</strong> nicht geändert werden. Der vollständige Satz optionaler Attribute für eine Klasse ist die Vereinigung der <strong>systemMayContain-</strong> und <strong>mayContain-Werte</strong> für diese Klasse und alle geerbten Klassen.<br /> | 
| <strong>possSuperiors</strong>, <strong>systemPossSuperiors</strong> | Ein Paar von mehrwertigen Eigenschaften, die die strukturellen Klassen angeben, die legale über-Instanzen dieser Klasse sein können. Der vollständige Satz möglicher Vorgesetzter ist die Vereinigung der <strong>SystemPossSuperiors-</strong> und <strong>possSuperiors-Werte</strong> für diese Klasse und aller geerbten strukturellen oder abstrakten Klassen. <strong>systemPossSuperiors-</strong> <strong>und possSuperiors-Werte</strong> werden nicht von Hilfsklassen geerbt. Sie können <strong>possSuperiors-Werte</strong> einem vorhandenen <strong>ClassSchema-Objekt</strong> der Kategorie 1 oder Kategorie 2 hinzufügen oder daraus entfernen. Nach der Erstellung der -Klasse kann <strong>die systemPossSuperiors-Eigenschaft</strong> nicht mehr geändert werden.<br /> | 
| <strong>objectClassCategory</strong> | Ein ganzzahliger Wert, der die Kategorie der -Klasse angibt, die eine der folgenden Sein kann:<ul><li>Strukturell, d. h., sie kann im Verzeichnis instanziiert werden.</li><li>Abstrakt, was bedeutet, dass die -Klasse eine grundlegende Definition einer Klasse bietet, die zum Bilden struktureller Klassen verwendet werden kann.</li><li>Hilfsklasse, d. h. eine Klasse, die verwendet werden kann, um die Definition einer Klasse zu erweitern, die von ihr erbt, aber nicht verwendet werden kann, um selbst eine Klasse zu bilden.</li></ul>Weitere Informationen finden Sie unter <a href="structural-abstract-and-auxiliary-classes.md">Strukturelle, abstrakte und Hilfsklassen</a>.<br /> | 
| <strong>subClassOf</strong> | Eine OID für die unmittelbar übergeordnete Klasse dieser Klasse, d.&a0;B. die Klasse, von der diese Klasse abgeleitet wird. Bei strukturellen Klassen kann <strong>subClassOf</strong> eine strukturelle oder abstrakte Klasse sein.<br /> Bei abstrakten Klassen kann <strong>subClassOf</strong> nur eine abstrakte Klasse sein.<br /> Bei Hilfsklassen kann <strong>subClassOf</strong> eine abstrakte oder eine Hilfsklasse sein.<br /> Wenn Sie eine neue Klasse definieren, stellen Sie sicher, dass die <strong>klasse subClassOf</strong> vorhanden ist oder vorhanden ist, wenn die neue Klasse in das Verzeichnis geschrieben wird. Wenn die Klasse nicht vorhanden ist, wird <strong>das classSchema-Objekt</strong> dem Verzeichnis nicht hinzugefügt.<br /> | 
| <strong>auxiliaryClass</strong>, <strong>systemAuxiliaryClass</strong> | Ein Paar von mehrwertigen Eigenschaften, die die Hilfsklassen angeben, von denen diese Klasse erbt. Der vollständige Satz von Hilfsklassen ist die Vereinigung der <strong>systemAuxiliaryClass-</strong> und <strong>auxiliaryClass-Werte</strong> für diese Klasse und alle geerbten Klassen. Für ein <strong>vorhandenes classSchema-Objekt</strong> können Werte der <strong>auxiliaryClass-Eigenschaft</strong> hinzugefügt, aber nicht entfernt werden. Nach der Erstellung der -Klasse kann <strong>die systemAuxiliaryClass-Eigenschaft</strong> nicht mehr geändert werden.<br /> | 
| <strong>defaultObjectCategory</strong> | Der Distinguished Name dieser Objektklasse oder einer ihrer Übergeordneten. Wenn eine Instanz dieser Objektklasse erstellt wird, legt das System die <strong>objectCategory-Eigenschaft</strong> der neuen Instanz auf den Wert fest, der in der <strong>defaultObjectCategory-Eigenschaft</strong> seiner Objektklasse angegeben ist. Die <strong>objectCategory-Eigenschaft</strong> ist eine indizierte Eigenschaft, die verwendet wird, um die Effizienz von Objektklassensuchen zu erhöhen. Wenn <strong>defaultObjectCategory</strong> beim Erstellen einer Klasse nicht angegeben wird, legt das System sie auf den Distinguished Name (DN) des <strong>classSchema-Objekts</strong> für diese Klasse fest. Wenn Sie beispielsweise eine vordefinierte Klasse (Kategorie 1) als Unterklasse verwenden, wird als bewährte Methode <strong>defaultObjectCategory</strong> auf denselben Wert wie die Übergeordnete Klasse festgelegt. Dadurch kann die Standardbenutzeroberfläche Ihre Unterklasse "suchen".<br /> Weitere Informationen finden Sie unter <a href="object-class-and-object-category.md">Objektklasse und Objektkategorie</a>.<br /> |

Ein Menüelement für die Objektklasse wird im Kontextmenü <strong>Neu</strong> der Administrativen Snap-Ins nicht angezeigt, auch wenn die entsprechenden Eigenschaften des Erstellungs-Assistenten für das <strong>displaySpecifier-Objekt</strong> der Objektklasse festgelegt sind.<br /> Wenn <strong>defaultHidingValue</strong> auf <strong>FALSE</strong>festgelegt ist, werden neue Instanzen des Objekts in den Administrativen Snap-Ins und der Windows Shell angezeigt. Legen Sie diese Eigenschaft auf <strong>FALSE</strong> fest, um Instanzen der -Klasse in den administrativen Snap-Ins und der Shell anzuzeigen und einen Erstellungs-Assistenten und dessen Menüelement im Menü <strong>Neu</strong> der administrativen Snap-Ins zu aktivieren.<br /> Wenn der <strong>defaultHidingValue-Wert</strong> nicht festgelegt ist, ist der Standardwert <strong>TRUE.</strong><br /> | 
| <strong>systemFlags</strong> | Ein ganzzahliger Wert, der Flags enthält, die zusätzliche Eigenschaften der Klasse definieren. Das 0x10 Bit identifiziert eine Klasse der Kategorie 1 (eine Klasse, die Teil des Basisschemas ist, das im System enthalten ist). Sie können dieses Bit nicht festlegen. Das bedeutet, dass das Bit nicht in Klassen der Kategorie 2 festgelegt ist (bei denen es sich um Erweiterungen des Schemas handelt). | 
| <strong>systemOnly</strong> | Ein boolescher Wert, der angibt, ob nur der Active Directory-Server die Klasse ändern kann. Nur-Systemklassen können nur vom Verzeichnissystem-Agent (Directory System Agent, DSA) erstellt oder gelöscht werden. Nur-System-Klassen sind klassen, von denen das System für normale Vorgänge abhängt. | 
| <strong>defaultSecurityDescriptor</strong> | Gibt den Standardsicherheitsdeskriptor für neue Objekte dieser Klasse an. Weitere Informationen finden Sie unter <a href="default-security-descriptor.md">Standardsicherheitsdeskriptor</a> und Festlegen von <a href="how-security-descriptors-are-set-on-new-directory-objects.md">Sicherheitsbeschreibungen für neue Verzeichnisobjekte.</a> | 
| <strong>isDefunct</strong> | Ein boolescher Wert, der angibt, ob die Klasse außer Kraft tritt. Weitere Informationen finden Sie unter <a href="disabling-existing-classes-and-attributes.md">Deaktivieren vorhandener Klassen und Attribute.</a> |

Dabei handelt es sich um die <strong>classSchema-Objektklasse</strong> für alle Klassendefinitionen und die <strong>attributeSchema-Objektklasse</strong> für alle Attributdefinitionen. |
