<span data-ttu-id="19327-104">Windows Server 2008 führt eine neue Art von Domänen Controller ein, den schreibgeschützten Domänen Controller (Read-Only Domain Controller, RODC).</span><span class="sxs-lookup"><span data-stu-id="19327-104">Windows Server 2008 introduces a new type of domain controller, the Read-only Domain Controller (RODC).</span></span> <span data-ttu-id="19327-105">Dadurch wird ein Domänen Controller für die Verwendung in Zweigniederlassungen bereitgestellt, in denen kein vollständiger Domänen Controller platziert werden kann.</span><span class="sxs-lookup"><span data-stu-id="19327-105">This provides a domain controller for use at branch offices where a full domain controller cannot be placed.</span></span> <span data-ttu-id="19327-106">Die Absicht besteht darin, Benutzern in den Zweigstellen zu gestatten, sich zu anmelden und Aufgaben wie die Datei-/Druckerfreigabe auszuführen, auch wenn keine Netzwerk Konnektivität mit Hub-Standorten vorhanden ist.</span><span class="sxs-lookup"><span data-stu-id="19327-106">The intent is to allow users in the branch offices to logon and perform tasks like file/printer sharing even when there is no network connectivity to hub sites.</span></span>

<span data-ttu-id="19327-107">RODC ändert nicht die Art und Weise, in der das Schema verwendet wird.</span><span class="sxs-lookup"><span data-stu-id="19327-107">RODC does not change the way schema is used.</span></span> <span data-ttu-id="19327-108">Es lohnt sich jedoch zu erwähnen, dass das Schema einen Read-Only partiellen Attribut Satz (sspas) unterstützt, der auch als RODC-gefilterter Attribut Satz bezeichnet wird. ## <a name="rodc-filtered-attribute-set"></a><span data-ttu-id="19327-110">Gefilterter RODC-Attribut Satz</span><span class="sxs-lookup"><span data-stu-id="19327-110">RODC filtered attribute set</span></span>

<span data-ttu-id="19327-111">Einige Anwendungen, die [Active Directory Domain Services](active-directory-domain-services.md) als Datenspeicher verwenden, verfügen möglicherweise über Anmelde Informations ähnliche Daten (z. b. Kenn Wörter, Anmelde Informationen oder Verschlüsselungsschlüssel), die nicht auf einem Read-Only Domänen Controller gespeichert werden sollen, falls der RODC gestohlen oder kompromittiert wird.</span><span class="sxs-lookup"><span data-stu-id="19327-111">Some applications that use [Active Directory Domain Services](active-directory-domain-services.md) as a data store may have credential-like data (such as passwords, credentials, or encryption keys) that should not be stored on a Read-Only Domain Controller in case the RODC is stolen or compromised.</span></span> <span data-ttu-id="19327-112">Für diesen Anwendungstyp können Sie das-Attribut zum gefilterten RODC-Attribut hinzufügen, um zu verhindern, dass es in RODCs in der Gesamtstruktur repliziert wird, und das Attribut als vertraulich markieren, wodurch die Fähigkeit zum Lesen der Daten für Mitglieder der Gruppe "authentifizierte Benutzer" (einschließlich aller RODCs) entfällt.</span><span class="sxs-lookup"><span data-stu-id="19327-112">For this type of application, you can add the attribute to the RODC filtered attribute set to prevent it from replicating to RODCs in the forest, and mark the attribute as confidential, which removes the ability to read the data for members of the Authenticated Users group (including any RODCs).</span></span>

<span data-ttu-id="19327-117">Sie können dem RODC-gefilterten Attribut Satz keine systemkritischen Attribute hinzufügen.</span><span class="sxs-lookup"><span data-stu-id="19327-117">You cannot add system-critical attributes to the RODC filtered attribute set.</span></span> <span data-ttu-id="19327-118">Ein Attribut ist System kritisch, wenn es für AD DS, lokale Sicherheits Autorität (Local Security Authority, LSA), Sicherheits Konten Verwaltung (Security Accounts Manager, Sam) und einen Microsoft-spezifischen Sicherheits Dienstanbieter (z. b. das Kerberos-Authentifizierungsprotokoll) erforderlich ist, um ordnungsgemäß zu funktionieren.</span><span class="sxs-lookup"><span data-stu-id="19327-118">An attribute is system critical if it is required for AD DS, Local Security Authority (LSA), Security Accounts Manager (SAM), and any of Microsoft-specific Security Service Providers, such as the Kerberos authentication protocol, to function properly.</span></span> <span data-ttu-id="19327-119">In Versionen von Windows Server 2008 nach Beta 3 weist ein System kritisches Attribut den Wert des schemaFlagsEx-Attributs auf (schemaFlagsEx-Attribut Wert & 0x1 = **true**).</span><span class="sxs-lookup"><span data-stu-id="19327-119">In releases of Windows Server 2008 after Beta 3, a system-critical attribute has a schemaFlagsEx attribute value of (schemaFlagsEx attribute value & 0x1 = **TRUE**).</span></span>

<span data-ttu-id="19327-120">Schritt-für-Schritt-Anweisungen zum Hinzufügen von Attributen zum gefilterten RODC-Attribut Satz finden Sie in [Anhang D der Schritt-für-Schritt-Anleitung für RODCs]( /previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc772331(v=ws.10)).</span><span class="sxs-lookup"><span data-stu-id="19327-120">For step by step instructions to adding attributes to the RODC filtered attribute set, see [Appendix D of the step-by-step guide for RODCs]( /previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc772331(v=ws.10)).</span></span>
