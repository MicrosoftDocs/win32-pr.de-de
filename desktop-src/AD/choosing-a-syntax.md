# <a name="choosing-a-syntax"></a>Auswählen einer Syntax

In Active Directory Domain Services sind 23 Syntaxen definiert. Dieses Thema enthält eine Liste der empfohlenen Syntaxen, die beim Definieren eines neuen Attributs zu verwenden sind. Weitere Informationen finden Sie unter [Syntax für Attribute in Active Directory Domain Services](syntaxes-for-attributes-in-active-directory-domain-services.md).

In der folgenden Tabelle finden Sie eine Liste der Empfehlungen.



| Daten zum Speichern in Attribut      | Zu verwendende Syntax                                                      | Kommentar                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|---------------------------------|--------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Binärdaten                     | [**String(Octet)**](/windows/desktop/ADSchema/s-string-octet)                       | Verwenden Sie zum Speichern von Binärdaten. Dies ist ein Bytearray.                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Binärdaten mit einem DN-Verweis | [**Object(DN-Binary)**](/windows/desktop/ADSchema/s-object-dn-binary)               | Enthält einen binären Wert und einen Distinguished Name (DN). Der Active Directory Server hält den DN auf dem neuesten Stand.                                                                                                                                                                                                                                                                                                                                                                                                     |
| Boolean                         | [**Booleschen**](/windows/desktop/ADSchema/s-boolean)                                  | Verwenden Sie für boolesche Werte.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| DN-Referenz                    | [**Object(DS-DN)**](/windows/desktop/ADSchema/s-object-ds-dn)                       | Verwenden Sie, um Distinguished Names zu speichern, die vom Active Directory Server auf dem neuesten Stand gehalten werden sollen. Wenn ein Attribut der DN-Syntax mit einem gültigen DN erstellt wird, behandelt der Server das Attribut als Verweis auf das Objekt, das durch den festgelegten DN repräsentiert wird. Wenn das Objekt, auf das verwiesen wird, umbenannt oder verschoben wird, stellt der Server sicher, dass das Attribut die Änderung widerspiegelt. Wenn das Attribut mit einem neuen DN zurückgesetzt wird, ist das Attribut ein Verweis auf das Objekt, das durch den neuen DN dargestellt wird.                                |
| Integer                         | [**Integer**](/windows/desktop/ADSchema/s-integer)                                  | Verwenden Sie für ganze Zahlen.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Große ganze Zahl (64-Bit-Werte)   | [**LargeInteger**](/windows/desktop/ADSchema/s-largeinteger)                        | Verwenden Sie für 64-Bit-Werte.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Verknüpfter DN                       | [**Object(DS-DN)**](/windows/desktop/ADSchema/s-object-ds-dn)                       | Diese Zeichen folgen Syntax kann für verknüpften DNS verwendet werden. Backlinks müssen den Syntax-DN aufweisen. Forward-Links können Syntax-DN und Objekt ( [**DN-String)**](/windows/desktop/ADSchema/s-object-dn-string), [**Object (DN-Binary)**](/windows/desktop/ADSchema/s-object-dn-binary), [**Object (Access-Point)**](/windows/desktop/ADSchema/s-object-access-point)oder [**Object (or-Name)**](/windows/desktop/ADSchema/s-object-or-name)sein. Für verknüpfte Attribute muss ein **linkid** definiert sein. Weitere Informationen finden Sie in der Beschreibung von **linkid** in [**Attribut Schema**](/windows/desktop/ADSchema/c-attributeschema) Eigenschaften. |
| Sicherheitsbeschreibung             | [**String(NT-Sec-Desc)**](/windows/desktop/ADSchema/s-string-nt-sec-desc)           | Oktett-Zeichenfolge, die eine Sicherheits Beschreibung enthält.                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Sicherheits-ID (SID)       | [**Zeichenfolge (SID)**](/windows/desktop/ADSchema/s-string-sid)                           | Oktett-Zeichenfolge, die eine Sicherheits-ID (SID) enthält. Verwenden Sie diese Syntax nur, um sid-Werte zu speichern.                                                                                                                                                                                                                                                                                                                                                                                                                  |
| String                          | [**String(Unicode)**](/windows/desktop/ADSchema/s-string-unicode)                   | Verwenden Sie für die meisten Zeichen folgen Attribute. Es unterstützt den Unicode-Zeichensatz. Wenn der Active Directory Server Vergleiche mit Attributen dieser Syntax ausführt (z. b. das Auswerten einer Abfrage), werden Vergleiche ohne Berücksichtigung der Groß-/Kleinschreibung durchführen. Verwenden Sie die andere Zeichen folgen Syntax ([**String (IA5)**](/windows/desktop/ADSchema/s-string-ia5), [**String (numeric)**](/windows/desktop/ADSchema/s-string-numeric)usw.) zum Speichern von Zeichen folgen, die nur die von der Syntax unterstützten Zeichensätze enthalten sollen.<br/>                          |
| Zeichen folgen Daten mit einem DN-Verweis | [**Object(DN-String)**](/windows/desktop/ADSchema/s-object-dn-string)               | Zeichenfolge, die einen Zeichen folgen Wert und einen Distinguished Name (DN) enthält. Der Active Directory Server hält den DN auf dem neuesten Stand.                                                                                                                                                                                                                                                                                                                                                                                            |
| Time                            | [**String(Generalized-Time)**](/windows/desktop/ADSchema/s-string-generalized-time) | Verwenden Sie die Syntax der [**Zeichenfolge (generalisierte Zeit)**](/windows/desktop/ADSchema/s-string-generalized-time) , um Zeit Werte anstelle der Zeichen folgen Syntax ( [**UTC-Zeit)**](/windows/desktop/ADSchema/s-string-utc-time) zu speichern, da die **Zeichenfolge (generalisierte Zeit)** vier Zeichen für das Jahr verwendet und die **Zeichenfolge (UTC-Zeit)** nur zwei Zeichen verwendet.                                                                                                                                                                                                                 |
