## <a name="replicable-services"></a><span data-ttu-id="83ac0-109">Replizierungsdienste</span><span class="sxs-lookup"><span data-stu-id="83ac0-109">Replicable services</span></span>

<span data-ttu-id="83ac0-110">Bei einem replizierbaren Dienst können eine oder mehrere Instanzen oder Replikate des Dienstanbieter vorhanden sein, und Clients sind nicht darauf achten, dass das Replikat, mit dem Sie verbunden sind, den gleichen Dienst bereitstellt.</span><span class="sxs-lookup"><span data-stu-id="83ac0-110">For a replicable service there can be one or many instances, or replicas, of the service, and clients do not care which replica they connect to because each provides the same service.</span></span> <span data-ttu-id="83ac0-111">Active Directory Domain Services Beispiele für replizierte Dienste: alle Domänen Controller für eine bestimmte Domäne enthalten identische Daten, unterliegen der Replikations Latenz und stellen identische Dienste bereit.</span><span class="sxs-lookup"><span data-stu-id="83ac0-111">Active Directory Domain Services are examples of replicated services: all domain controllers for a given domain hold identical data, subject to replication latency, and provide identical services.</span></span>

<span data-ttu-id="83ac0-112">Replizierungsdienste können SCPs und andere Dienst spezifische Objekte für mehrere Replikate in einem einzelnen Container speichern.</span><span class="sxs-lookup"><span data-stu-id="83ac0-112">Replicable services can store the SCPs and other service-specific objects for multiple replicas in a single container.</span></span> <span data-ttu-id="83ac0-113">Die Setup Anwendung für das erste Replikat kann den Container als untergeordnetes Element des System Containers der lokalen Domäne erstellen.</span><span class="sxs-lookup"><span data-stu-id="83ac0-113">The setup application for the first replica can create the container as a child of the local domain's System container.</span></span> <span data-ttu-id="83ac0-114">Weitere Informationen finden Sie unter [Veröffentlichen in einem Domänen System Container](publishing-in-a-domain-system-container.md).</span><span class="sxs-lookup"><span data-stu-id="83ac0-114">For more information, see [Publishing in a Domain System Container](publishing-in-a-domain-system-container.md).</span></span> <span data-ttu-id="83ac0-115">Stellen Sie sicher, dass die Sicherheits Beschreibung für den Container es den Setup Programmen für nachfolgende Replikate ermöglicht, Ihre Objekte im gleichen Container zu erstellen.</span><span class="sxs-lookup"><span data-stu-id="83ac0-115">Ensure that the security descriptor for your container allows the setup programs for subsequent replicas to create their objects in the same container.</span></span> <span data-ttu-id="83ac0-116">Erteilen Sie Berechtigungen für den Installations Administrator, um die Benutzer oder Gruppen anzugeben, die Objekte im Container erstellen oder ändern können.</span><span class="sxs-lookup"><span data-stu-id="83ac0-116">Grant permissions for the installing administrator to specify the users or groups that can create or modify objects in the container.</span></span>

<span data-ttu-id="83ac0-117">Eine Strategie für einen replizierbaren Dienst besteht darin, einen Dienst Verbindungspunkt für jedes Replikat zu erstellen.</span><span class="sxs-lookup"><span data-stu-id="83ac0-117">One strategy for a replicable service is to create an SCP for each replica.</span></span> <span data-ttu-id="83ac0-118">Wenn ein Client nach der Produkt-GUID oder einem anderen identifizierenden Schlüsselwort fragt, sucht es nach den SCP-Objekten für alle Replikate und wählt eine nach dem Zufallsprinzip oder mithilfe eines Lasten Ausgleichs Algorithmus aus.</span><span class="sxs-lookup"><span data-stu-id="83ac0-118">When a client queries for the service's product GUID or other identifying keyword, it finds the SCP objects for all replicas and selects one at random or using some load-balancing algorithm.</span></span> <span data-ttu-id="83ac0-119">Ein Administrator kann z. b. für jedes Replikat, ähnlich den Feldern Priorität und Gewichtung eines DNS-SRV-Einsatzes, Prioritäts-und Lasten Ausgleichs Daten angeben.</span><span class="sxs-lookup"><span data-stu-id="83ac0-119">For example, an administrator could specify priority and load-balancing data for each replica, similar to the priority and weight fields of a DNS SRV record.</span></span> <span data-ttu-id="83ac0-120">Die Setup Anwendung des Dienstanbieter kann diese Daten im **serviceBindingInformation** -Attribut jedes Replikat-Dienst Prinzipals speichern.</span><span class="sxs-lookup"><span data-stu-id="83ac0-120">The service's setup application can store this data in the **serviceBindingInformation** attribute of each replica's SCP.</span></span> <span data-ttu-id="83ac0-121">Clients rufen die Daten von jedem SCP ab und verwenden Sie zum Auswählen eines Replikats.</span><span class="sxs-lookup"><span data-stu-id="83ac0-121">Clients retrieve the data from each SCP and use it to select a replica.</span></span>

<span data-ttu-id="83ac0-122">Eine andere Strategie ist die Erstellung eines einzelnen Dienst Verbindungs Punkts für alle Replikate und das Festlegen des SCP **servicednsname** -Attributs auf den Namen eines DNS-SRV-Einsatzes.</span><span class="sxs-lookup"><span data-stu-id="83ac0-122">Another strategy is to create a single SCP for all replicas and set the SCP **serviceDNSName** attribute to the name of a DNS SRV record.</span></span> <span data-ttu-id="83ac0-123">Anschließend registriert die Setup Anwendung für jedes Replikat einen SRV-Datensatz mit diesem Namen.</span><span class="sxs-lookup"><span data-stu-id="83ac0-123">Then the setup application for each replica registers a SRV record with that name.</span></span> <span data-ttu-id="83ac0-124">Wenn ein Client den Lone SCP des dienstanders identifiziert, ruft der Client den Namen des SRV-Eintrags ab und verwendet die [**DNSQuery**](/windows/desktop/api/windns/nf-windns-dnsquery_a) -Funktion, um das Array der SRV-Einträge für die Replikate abzurufen.</span><span class="sxs-lookup"><span data-stu-id="83ac0-124">When a client identifies the service's lone SCP, the client retrieves the name of the SRV record and uses the [**DnsQuery**](/windows/desktop/api/windns/nf-windns-dnsquery_a) function to retrieve the array of SRV records for the replicas.</span></span> <span data-ttu-id="83ac0-125">Jeder SRV-Datensatz enthält den Namen eines Host Computers und zusätzliche Daten, die der Client zum Auswählen eines Replikats verwenden kann.</span><span class="sxs-lookup"><span data-stu-id="83ac0-125">Each SRV record contains the name of a host computer and additional data that the client can use to select a replica.</span></span>

<span data-ttu-id="83ac0-127">Unterschiedliche Instanzen eines Daten Bank Dienstanbieter enthalten möglicherweise ganz andere Daten, auch wenn es sich um dieselbe Art von Dienst handelt, die in der Regel als Dienstklasse bezeichnet wird.</span><span class="sxs-lookup"><span data-stu-id="83ac0-127">Different instances of a database service may contain entirely different data, even though they are all the same kind of service, usually called service class.</span></span> <span data-ttu-id="83ac0-128">Um diese Art von Dienst zu veröffentlichen, kann das **Keywords** -Attribut des Dienst Verbindungs Punkts sowohl die Dienstklasse als auch die bestimmte Datenbank identifizieren.</span><span class="sxs-lookup"><span data-stu-id="83ac0-128">To publish this kind of service, the **keywords** attribute of the SCP can identify both the service class and the specific database.</span></span> <span data-ttu-id="83ac0-129">Ein allgemeiner Client, der nur die GUID der Dienstklasse kennt, kann Abfragen für alle Datenbanken durch diese Dienstklasse durchsuchen und dann eine Benutzeroberfläche präsentieren, die dem Benutzer ermöglicht, einen auszuwählen.</span><span class="sxs-lookup"><span data-stu-id="83ac0-129">A general-purpose client that knows only the GUID of the service class can query for all databases published by that service class, and then present a user interface to allow the user to select one.</span></span> <span data-ttu-id="83ac0-130">Für einen Client, der speziell für die Zieldatenbank entworfen wurde, können Sie die Datenbank-GUID im Client Code hart codieren.</span><span class="sxs-lookup"><span data-stu-id="83ac0-130">For a client that is designed specifically for the target database, you can hard-code the database GUID into the client code.</span></span>
