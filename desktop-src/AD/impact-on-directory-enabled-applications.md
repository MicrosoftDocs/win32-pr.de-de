## <a name="collisions"></a><span data-ttu-id="472c7-136">Kollisionen</span><span class="sxs-lookup"><span data-stu-id="472c7-136">Collisions</span></span>

<span data-ttu-id="472c7-139">Ein einfaches Beispiel sind Benutzer Adressinformationen: ein Benutzer ändert eine e-Mail-Adresse auf dem Replikat R1, und ein Administrator ändert dieselbe Mailingadresse bei Replikat R2.</span><span class="sxs-lookup"><span data-stu-id="472c7-139">A simple example is user address information: A user changes a mailing address at replica R1 and an administrator changes the same mailing address at replica R2.</span></span> <span data-ttu-id="472c7-140">Ein geschriebener Wert wird weitergegeben, bis ein anderer Wert durch den Konflikt Abstimmungsmechanismus über diesem Wert ausgewählt wird.</span><span class="sxs-lookup"><span data-stu-id="472c7-140">A written value propagates until another value is selected over that value by the collision reconciliation mechanism.</span></span> <span data-ttu-id="472c7-141">Solange ein Wert für andere Werte im Konflikt Auflösungsprozess weiterhin "gewinnt", wird der Wert weiter verteilt.</span><span class="sxs-lookup"><span data-stu-id="472c7-141">As long as a value continues to "win" against other values in the collision resolution process, the value continues to propagate.</span></span> <span data-ttu-id="472c7-142">Letztendlich wird dieser "gewinnende" Wert an R1, R2 und alle anderen Replikate weitergegeben, wenn keine anderen Änderungen vorgenommen werden.</span><span class="sxs-lookup"><span data-stu-id="472c7-142">Ultimately, this "winning" value will be propagated to R1,R2, and all other replicas if no other changes are made.</span></span>
