## <a name="remarks"></a>Bemerkungen

Wenn Sie ein Zeichen mithilfe des HTTP-Protokolls (ein-Zeichen) laden. ACF-Datei) verwenden Sie die **Get** -Methode zum Abrufen von Animationsdaten, bevor Sie die Animation wiedergeben können. Sie verwenden diese Methode nicht, wenn Sie das Zeichen mithilfe des UNC-Protokolls (eines geladen haben. ACS-Datei). Mithilfe von **Get** können Sie auch keine HTTP-Daten für ein Zeichen abrufen, wenn Sie dieses Zeichen mithilfe des UNC-Protokolls () geladen haben. ACS-Zeichen Datei).

Wenn Sie einen Objekt Verweis deklarieren und diesen auf diese Methode festlegen, wird ein [**Anforderungs**](/windows/desktop/lwef/the-request-object) Objekt zurückgegeben. Wenn die zugeordnete Animation nicht geladen werden kann, legt der Server die [**Status**](status-property.md) -Eigenschaft des **Anforderungs** Objekts auf "failed" (Fehler) mit einer entsprechenden Fehlernummer fest. Sie können das [**requestcomplete**](requestcomplete-event.md) -Ereignis verwenden, um den Status zu überprüfen und zu bestimmen, welche Aktion ausgeführt werden soll.

Animationen oder Audiodaten, die mit der **Get** -Methode abgerufen werden, werden im Cache des Browsers gespeichert. Bei nachfolgenden Aufrufen wird der Cache überprüft. wenn die Animationsdaten bereits vorhanden sind, lädt das-Steuerelement die Daten direkt aus dem Cache. Nach dem Laden können die Animations-oder Audiodaten mit den Methoden " [**abspielen**](play-method.md) " und " [**Sprache**](speak-method.md) " wiedergegeben werden.
