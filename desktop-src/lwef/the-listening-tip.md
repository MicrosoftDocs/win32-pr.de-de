# <a name="the-listening-tip"></a>Der Abhör Tipp

\[Der Microsoft-Agent ist ab Windows 7 veraltet und in nachfolgenden Versionen von Windows möglicherweise nicht verfügbar.\]

Der Abhör Tipp ist ein weiterer Spracheingabe Dienst, der vom Microsoft-Agent bereitgestellt wird. Wenn die Spracheingabe installiert ist, enthält der-Agent ein spezielles QuickInfo-Fenster, das angezeigt wird, wenn der Benutzer den lauschenden Hotkey drückt oder die Abhör Methode aufruft. Der Abhör Tipp wird nur angezeigt, wenn die Sprachdienste verfügbar sind. Wenn kein Client einen Sprachbefehl verfasst hat oder eine Sprach-Engine erfolgreich lädt, wird der lausch Tipp nicht angezeigt. Außerdem müssen sowohl die Spracheingabe als auch die Option zum Anzeigen von Tipps in den erweiterten Zeichen Optionen aktiviert sein, damit der Tipp angezeigt wird.

Der lausch Tipp gibt nach der Präsentation automatisch ein Timeout aus. Wenn der "hörende" Text Timeout abgeschlossen ist, während der Benutzer den Hotkey immer noch gedrückt hält, wird die QuickInfo auf den "lauschen"-Text zurückgesetzt, es sei denn, der Server erhält eine andere übereinstimmende Äußerung. In diesem Fall zeigt der Tipp den neuen "Heard"-Text an und beginnt den Timeout Wert für den trinktext. Wenn der Benutzer den Hotkey freigibt und der Server den "Heard"-Text anzeigt, wird das Timeout fortgesetzt, und das Fenster für das lauschen von Tipps wird ausgeblendet, wenn das Timeout Intervall abläuft.

Wenn der Server noch nicht versucht hat, eine Spracherkennungs-Engine zu laden, wird der lausch Tipp nicht angezeigt. Ebenso wird, wenn der Benutzer die Anzeige der sprechenden oder deaktivierten Spracheingabe in erweiterten Zeichen Optionen deaktiviert hat, der lausch Tipp nicht angezeigt.

Der Abhör Tipp wird nicht angezeigt, wenn sich der Mauszeiger über dem Task leisten Symbol des Zeichens befindet. Stattdessen wird das standardmäßige Benachrichtigungs Tipp Fenster angezeigt, und der Name des Zeichens wird angezeigt.

Client Anwendungen können nicht direkt in den Abhör Tipp schreiben, Sie können jedoch alternativen Text angeben, der vom Server bei der Erkennung eines übereinstimmenden sprach Befehls angezeigt wird. Legen Sie dazu die Eigenschaft " [**Vertrauen**](confidence-property.md) " und die neue Eigenschaft " [**conficetext**](confidencetext-property.md)" für den Befehl fest. Wenn eine gesprochene Eingabe mit dem Befehl übereinstimmt, die beste Übereinstimmung jedoch nicht die Vertrauens Einstellung überschreitet, verwendet der Server den Text, der in der Eigenschaft **conficetext** im Tip-Fenster festgelegt ist. Wenn der Client diesen Wert nicht bereitstellt, zeigt der Server den Text (die Grammatik) an, mit dem er übereinstimmt.

Der lausch Text wird in der Sprache auf der Grundlage der Zeichen Sprach-ID-Einstellung des Eingabe aktiven Clients angezeigt, unabhängig davon, ob eine sprach kompatible sprach Erkennungs-Engine verfügbar ist.
