# <a name="input-active-client"></a><span data-ttu-id="2f85b-103">Input-Active-Client</span><span class="sxs-lookup"><span data-stu-id="2f85b-103">Input-Active Client</span></span>

<span data-ttu-id="2f85b-105">Da mehrere Client Anwendungen dasselbe Zeichen gemeinsam verwenden können und mehrere Clients gleichzeitig verschiedene Zeichen verwenden können, legt der Server einen Client als *Eingabe aktiven* Client fest und sendet Maus-und Spracheingaben nur an diese Client Anwendung.</span><span class="sxs-lookup"><span data-stu-id="2f85b-105">Because multiple client applications can share the same character and because multiple clients can use different characters at the same time, the server designates one client as the *input-active* client and sends mouse and voice input only to that client application.</span></span> <span data-ttu-id="2f85b-106">Dadurch wird die ordnungsgemäße Verwaltung von Benutzereingaben aufrechterhalten, sodass ein entsprechender Client auf die Eingabe antwortet.</span><span class="sxs-lookup"><span data-stu-id="2f85b-106">This maintains the orderly management of user input, so that an appropriate client responds to the input.</span></span>

<span data-ttu-id="2f85b-107">In der Regel bestimmt die Benutzerinteraktion, welche Client Anwendung als Eingabe aktiv wird.</span><span class="sxs-lookup"><span data-stu-id="2f85b-107">Typically, user interaction determines which client application becomes input-active.</span></span> <span data-ttu-id="2f85b-108">Wenn der Benutzer z. b. auf ein Zeichen klickt, wird die Client Anwendung dieses Zeichens als Eingabe aktiv.</span><span class="sxs-lookup"><span data-stu-id="2f85b-108">For example, if the user clicks a character, that character's client application becomes input-active.</span></span> <span data-ttu-id="2f85b-109">Ebenso gilt: Wenn ein Benutzer den Namen eines Zeichens spricht, wird er als Eingabe aktiv.</span><span class="sxs-lookup"><span data-stu-id="2f85b-109">Similarly, if a user speaks the name of a character, it becomes input-active.</span></span> <span data-ttu-id="2f85b-110">Wenn der Server außerdem die [**Show**](show-method.md) -Methode eines Zeichens verarbeitet, wird der Client dieses Zeichens als Eingabe aktiv.</span><span class="sxs-lookup"><span data-stu-id="2f85b-110">Also, when the server processes a character's [**Show**](show-method.md) method, the client of that character becomes input-active.</span></span>

<span data-ttu-id="2f85b-111">Wenn ein Zeichen ausgeblendet wird, ist der Client dieses Zeichens für dieses Zeichen nicht mehr als Eingabe aktiv.</span><span class="sxs-lookup"><span data-stu-id="2f85b-111">When a character is hidden, the client of that character will no longer be input-active for that character.</span></span> <span data-ttu-id="2f85b-112">Der Server stellt den aktiven Client automatisch für alle verbleibenden Zeichen (s) ein.</span><span class="sxs-lookup"><span data-stu-id="2f85b-112">The server automatically makes the active client of any remaining character(s) input-active.</span></span> <span data-ttu-id="2f85b-113">Wenn alle Zeichen ausgeblendet sind, ist kein Client Eingabe aktiv.</span><span class="sxs-lookup"><span data-stu-id="2f85b-113">When all characters are hidden, no client is input-active.</span></span> <span data-ttu-id="2f85b-114">Wenn der Benutzer jedoch in dieser Situation den abzurufenden Hotkey drückt, lauscht der-Agent weiterhin auf seine Befehle (mithilfe der sprach Erkennungs-Engine, die mit dem obersten Zeichen des letzten Eingabe aktiven Clients übereinstimmt).</span><span class="sxs-lookup"><span data-stu-id="2f85b-114">However, in this situation, if the user presses the Listening hotkey, Agent will continue to listen for its commands (using the speech recognition engine matching the topmost character of the last input-active client).</span></span>

<span data-ttu-id="2f85b-115">Wenn mehrere Clients dasselbe Zeichen gemeinsam nutzen, wird der *aktive Client* vom Server als aktiver Eingabe Client bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="2f85b-115">If multiple clients are sharing the same character, the server will designate its *active client* as input-active client.</span></span> <span data-ttu-id="2f85b-116">Das aktive Zeichen ist die oberste Position in der Client Reihenfolge.</span><span class="sxs-lookup"><span data-stu-id="2f85b-116">The active character is the topmost in the client order.</span></span> <span data-ttu-id="2f85b-117">Mithilfe der Methode [**aktivieren**](activate-method.md) können Sie den Client als aktiven oder nicht aktiven Client festlegen.</span><span class="sxs-lookup"><span data-stu-id="2f85b-117">You can set your client to be the active or not-active client using the [**Activate**](activate-method.md) method.</span></span> <span data-ttu-id="2f85b-118">Sie können auch die Methode **aktivieren** verwenden, um die Client Eingabe-aktiv zu machen. um die Unterbrechung anderer Clients des Zeichens zu vermeiden, sollten Sie dies jedoch nur dann tun, wenn die Client Anwendung aktiv ist.</span><span class="sxs-lookup"><span data-stu-id="2f85b-118">You can also use the **Activate** method to explicitly make your client input-active; but to avoid disrupting other clients of the character, you should do so only when your client application is active.</span></span> <span data-ttu-id="2f85b-119">Wenn der Benutzer z. b. auf das Fenster der Anwendung klickt und die Anwendung aktiviert, können Sie die **Aktivierungs** Methode aufrufen, um die Maus-und Spracheingaben zu empfangen und zu verarbeiten, die an das Zeichen weitergeleitet werden.</span><span class="sxs-lookup"><span data-stu-id="2f85b-119">For example, if the user clicks your application's window, activating your application, you can call the **Activate** method to receive and process mouse and speech input directed to the character.</span></span>
