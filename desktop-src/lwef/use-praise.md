# <a name="use-praise"></a>Verwendung von Lob

\[Der Microsoft-Agent ist ab Windows 7 veraltet und in nachfolgenden Versionen von Windows möglicherweise nicht verfügbar.\]

Die Menschen reagieren besser, wenn Sie sich als Kritik erweisen, auch wenn das Lob möglicherweise nicht gerechtfertigt ist. Obwohl es bei den meisten Software Schnittstellen bestrebt ist, evaluierender Feedback zu vermeiden, ist das loben nachweislich effektiver als die Bereitstellung überhaupt keiner Evaluierung. Darüber hinaus werden viele Schnittstellen, die als neutral entwickelt wurden, oft von Benutzern als kritisch angesehen, da Sie selten positives Feedback erhalten, wenn Sie normal arbeiten, und Fehlermeldungen, wenn etwas schief geht. Entsprechend wird der Benutzer mit einem Wortlaut, der dem Benutzer mitteilt, dass es eine bessere Möglichkeit zum Ausführen einer Aufgabe gibt, den Benutzer implizit kritisieren.

Da Zeichen einen sozialen Kontext bilden, ist die sorgfältige Verwendung von Lob und Kritik noch wichtiger als bei einer herkömmlichen Benutzeroberfläche. Obwohl nur wenige Benutzer das Verhalten von sycophantic genießen, sind die Grenzwerte für die liberale Verwendung des Lobes noch nicht zu veranschaulichen. Das Lob ist in Situationen, in denen Benutzer weniger sicher sind, hinsichtlich der Leistung einer Aufgabe, besonders effektiv. Andererseits sollte die Kritik sparsam verwendet werden. Selbst wenn Sie der Ansicht sind, dass eine entsprechende Kritik angebracht ist, sollten Sie beachten, dass Menschen eine Tendenz haben, Sie zu verwerfen oder Sie zurück an Ihre Quelle umzuleiten

Vermeiden Sie jedoch das selbst loben, es sei denn, es ist ein humorvoller Teil der Persönlichkeit der Zeichen Projekte. Wir neigen dazu, das Self-conceit mit Skepsis zu beurteilen. Wenn Sie die approbe an ein Zeichen weiterleiten möchten, lassen Sie Sie von einer anderen Quelle, z. b. einem anderen oder einer beschreibenden Erläuterung, weiterleiten.
