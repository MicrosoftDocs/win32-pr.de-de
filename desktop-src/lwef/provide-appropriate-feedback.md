# <a name="provide-appropriate-feedback"></a>Entsprechendes Feedback bereitstellen

\[Der Microsoft-Agent ist ab Windows 7 veraltet und in nachfolgenden Versionen von Windows möglicherweise nicht verfügbar.\]

Qualität, Eignung und zeitliche Steuerung sind wichtige Faktoren, die bei der Bereitstellung von Feedback in einem beliebigen Schnittstellen Entwurf zu berücksichtigen sind. Wenn Sie interaktive Zeichen integrieren, erhöhen sich die Möglichkeiten für natürliche Formen des Feedbacks, ebenso wie die Erwartung, dass das Feedback der entsprechenden sozialen Interaktion entspricht. Ein Zeichen kann zur Bereitstellung von verbalen und nicht verbalen konverterungen zusätzlich zur gesprochenen Audioausgabe entworfen werden. Verwenden Sie Gesten oder Gesichtsausdrücke, um Informationen über Ihre Stimmung oder Absicht zu vermitteln. Das Gesicht ist insbesondere bei der Kommunikation wichtig, daher sollten Sie immer den Gesichtsausdruck des Zeichens in Erwägung gezogen. Beachten Sie, dass *es sich* bei keinem Gesichtsausdruck um einen Gesichtsausdruck handelt.

Wir haben einen orientierende Reflex, mit dem wir Änderungen an unserer Umgebung, insbesondere Änderungen an der Bewegung, dem Volume oder dem Kontrast, Unternehmen. Daher sollten die Zeichen Animation und Soundeffekte minimal beibehalten werden, um das ablenken von Benutzern zu vermeiden, wenn Sie nicht direkt mit dem Zeichen interagieren. Dies bedeutet nicht, dass das Zeichen eingefroren werden muss, aber das Verhalten natürlicher Leerlauf, wie z. b. das Atmen oder das Durchsuchen, ist einer größeren Bewegung vorzuziehen Das idk-Verhalten bewahrt die Illusion von sozialem Kontext und Verfügbarkeit des Zeichens auf, ohne den Benutzer zu beeinträchtigen. Sie sollten das Zeichen auch entfernen, wenn der Benutzer für einen festgelegten Zeitraum nicht mit ihm interagiert hat, aber sicherstellen muss, dass der Benutzer versteht, warum das Zeichen entfernt wird.

Umgekehrt ist eine große Bewegung, eine ungewöhnliche Bewegung oder eine hochgradig aktive Animation sehr effektiv, wenn Sie die Aufmerksamkeit des Benutzers erfassen möchten, insbesondere, wenn die Animation außerhalb des aktuellen Fokus des Benutzers erfolgt. Beachten Sie auch, dass die Bewegung für den Benutzer die Aufmerksamkeit des Benutzers effektiv erreichen kann.

Die Platzierung und Bewegung des Zeichens sollte für die Teilnahme an der aktuellen Aufgabe des Benutzers geeignet sein. Wenn die aktuelle Aufgabe das Zeichen einschließt, kann das Zeichen am Punkt des Fokus platziert werden. Wenn der Benutzer nicht mit dem Zeichen interagiert, verschieben Sie ihn an einen konsistenten "standbyspeicherort" oder an den Speicherort, an dem der Benutzer nicht die Aufgaben beeinträchtigt oder den Benutzer abweist. Stellen Sie immer eine Begründung dafür bereit, wie das Zeichen von einem Speicherort zu einem anderen gelangt. Ebenso fühlen sich die Benutzer besonders gut, wenn das Zeichen an derselben Bildschirmposition angezeigt wird, von der es Abweichern.
