# <a name="about-vds"></a><span data-ttu-id="a99db-103">Informationen zu VDS</span><span class="sxs-lookup"><span data-stu-id="a99db-103">About VDS</span></span>

<span data-ttu-id="a99db-104">\[Ab Windows 8 und Windows Server 2012 wird die COM-Schnittstelle des [virtuellen Festplatten Dienstanbieter](virtual-disk-service-portal.md) durch die [Windows-Speicherverwaltungs-API](/previous-versions/windows/desktop/stormgmt/windows-storage-management-api-portal)ersetzt.\]</span><span class="sxs-lookup"><span data-stu-id="a99db-104">\[Beginning with Windows 8 and Windows Server 2012, the [Virtual Disk Service](virtual-disk-service-portal.md) COM interface is superseded by the [Windows Storage Management API](/previous-versions/windows/desktop/stormgmt/windows-storage-management-api-portal).\]</span></span>

<span data-ttu-id="a99db-105">Der Dienst für virtuelle Datenträger ist ein Microsoft Windows-Dienst, der Abfrage-und Konfigurations Vorgänge bei der Anforderung von Endbenutzern, Skripts und Anwendungen ausführt.</span><span class="sxs-lookup"><span data-stu-id="a99db-105">Virtual Disk Service is a Microsoft Windows service that performs query and configuration operations at the request of end users, scripts, and applications.</span></span> <span data-ttu-id="a99db-106">Der-Dienst erweitert die vorhandenen Speicherfunktionen von Windows Server-Betriebssystemen wie folgt:</span><span class="sxs-lookup"><span data-stu-id="a99db-106">The service extends the existing storage capabilities of Windows Server operating systems in the following ways:</span></span>

<span data-ttu-id="a99db-114">VDS definiert drei Schnittstellen: eine einzelne Schnittstelle zwischen der Anwendungsebene und dem Dienst und zwei Schnittstellen zwischen dem Dienst und den Anbieter Programmen in der Datenschicht.</span><span class="sxs-lookup"><span data-stu-id="a99db-114">VDS defines three interfaces: a single interface between the application layer and the service, and two interfaces between the service and provider programs in the data layer.</span></span> <span data-ttu-id="a99db-115">Die folgende Abbildung zeigt die Grenze für die Anwendungs-zu-Dienst-Grenze und die Dienst-zu-Anbieter-Grenze.</span><span class="sxs-lookup"><span data-stu-id="a99db-115">The following illustration shows the application-to-service boundary and the service-to-provider boundary.</span></span>

![Das Diagramm zeigt die Dienst Architektur, die in den Abschnitten "Anwendungen", "virtueller Datenträger Dienst" und "VDS-Anbieter" unterteilt ist.](images/vdsoverview.png)

<span data-ttu-id="a99db-117">Die N-Tier-Architektur ermöglicht VDS das koordinieren mit den Dateisystemfunktionen, das Synchronisieren von Anbieter Aktivitäten und das zwischen Stellen von Anwendungen.</span><span class="sxs-lookup"><span data-stu-id="a99db-117">N-tier architecture enables VDS to coordinate with the file-system functions, synchronize provider activities, and arbitrate between applications.</span></span> <span data-ttu-id="a99db-118">VDS besteht zwischen der Anwendung und dem Anbieter und bietet eine einheitliche Funktionalität für Anwendungen, auch wenn einige der zugrunde liegenden Anbieter eine solche Einheitlichkeit aufweisen können.</span><span class="sxs-lookup"><span data-stu-id="a99db-118">Being between the application and provider, VDS presents uniform functionality to applications even though some the underlying providers might lack such uniformity.</span></span>

<span data-ttu-id="a99db-119">Der Dienst implementiert allgemeine Funktionen: Formatieren von Volumes, hinzufügen und Entfernen von Laufwerk Buchstaben oder bereitgestellten Ordnern sowie verwalten nicht zugewiesener Datenträger – Datenträger, die keine Partitionsinformationen aufweisen.</span><span class="sxs-lookup"><span data-stu-id="a99db-119">The service implements common functionality: formatting volumes, adding and removing drive letters or mounted folders, as well as managing unallocated disks—disks having no partition information.</span></span> <span data-ttu-id="a99db-120">VDS gibt auch Ereignis Benachrichtigungen an registrierte Anwendungen zurück.</span><span class="sxs-lookup"><span data-stu-id="a99db-120">VDS also returns event notifications to registered applications.</span></span> <span data-ttu-id="a99db-121">Weitere Informationen finden Sie unter [VDS-Benachrichtigungen](vds-notification-model.md).</span><span class="sxs-lookup"><span data-stu-id="a99db-121">For details, see [VDS Notifications](vds-notification-model.md).</span></span>

## <a name="role-of-providers"></a><span data-ttu-id="a99db-122">Rolle von Anbietern</span><span class="sxs-lookup"><span data-stu-id="a99db-122">Role of Providers</span></span>

<span data-ttu-id="a99db-123">VDS definiert zwei Anbieter Schnittstellen, eine für einen Softwareanbieter und eine für einen Hardware Anbieter.</span><span class="sxs-lookup"><span data-stu-id="a99db-123">VDS defines two provider interfaces, one for a software provider and one for a hardware provider.</span></span> <span data-ttu-id="a99db-124">Jeder Anbieter implementiert einen anderen Teil der API, der von VDS definiert wird:</span><span class="sxs-lookup"><span data-stu-id="a99db-124">Each provider implements a different portion of the API defined by VDS:</span></span>

-   <span data-ttu-id="a99db-125">Ein *Softwareanbieter* ist ein Host basiertes Programm, das von einem Kernelmodustreiber im Speicher-e/a-Stapel unterstützt wird.</span><span class="sxs-lookup"><span data-stu-id="a99db-125">A *software provider* is a host-based program that is supported by a kernel-mode driver in the storage I/O stack.</span></span> <span data-ttu-id="a99db-126">Die Anbieter-Kernel-Laufzeit interagiert zum Zeitpunkt der Startzeit mit dem Mount Manager, oder der Plug & Play (PNP) Manager zum Zeitpunkt der Ermittlung, um die einzelnen Datenträger zu beanspruchen.</span><span class="sxs-lookup"><span data-stu-id="a99db-126">The provider-kernel runtime interacts with the Mount Manager at boot time or the Plug and Play (PnP) Manager at discovery time to claim each disk.</span></span> <span data-ttu-id="a99db-127">Software Anbieter arbeiten auf Volumes, Datenträgern und Datenträger Partitionen.</span><span class="sxs-lookup"><span data-stu-id="a99db-127">Software providers operate on volumes, disks, and disk partitions.</span></span>

    <span data-ttu-id="a99db-128">VDS enthält zwei Anbieter Typen.</span><span class="sxs-lookup"><span data-stu-id="a99db-128">VDS includes two provider types.</span></span> <span data-ttu-id="a99db-129">Der Basic-Softwareanbieter verwaltet Basis Datenträger und bietet keine fehlertolerante Bindung.</span><span class="sxs-lookup"><span data-stu-id="a99db-129">The basic software provider manages basic disks and offers no fault-tolerant binding.</span></span> <span data-ttu-id="a99db-130">Der dynamische Softwareanbieter verwaltet dynamische Datenträger und bietet ggf. die Fehler Verwaltung.</span><span class="sxs-lookup"><span data-stu-id="a99db-130">The dynamic software provider manages dynamic disks and offers fault management where applicable.</span></span> <span data-ttu-id="a99db-131">Das Verhalten des Software Anbieters ist mit dem Verhalten von grundlegenden und dynamischen Datenträgern auf dem Host konsistent.</span><span class="sxs-lookup"><span data-stu-id="a99db-131">Software-provider behavior is consistent with the behavior of basic and dynamic disks on the host.</span></span> <span data-ttu-id="a99db-132">Wenn das Betriebssystem eines bestimmten Hosts z. b. fehlertolerante dynamische Datenträger unterstützt, unterstützt VDS auch dieses Verhalten auf dem Host.</span><span class="sxs-lookup"><span data-stu-id="a99db-132">For example, if the operating system of a given host supports fault-tolerant dynamic disks, VDS also supports this behavior on the host.</span></span>

-   <span data-ttu-id="a99db-133">Ein *Hardware Anbieter* implementiert die Methoden, die zum Verwalten eines Speicher Subsystems verwendet werden – ein Hardware Festplatten Array oder eine Adapterkarte, mit dem logische Datenträger erstellt werden können, die zur Verbesserung der Leistung, der Datenverfügbarkeit oder der Datenwiederherstellung konfiguriert sind.</span><span class="sxs-lookup"><span data-stu-id="a99db-133">A *hardware provider* implements the methods that are used to manage a storage subsystem—a hardware disk array or adapter card that enables the creation of logical disks configured to enhance performance, data availability, or data recovery.</span></span> <span data-ttu-id="a99db-134">Viele große RAID-CAB-Hersteller haben einen Hardware Anbieter erstellt, der für die Verwendung mit VDS konzipiert ist.</span><span class="sxs-lookup"><span data-stu-id="a99db-134">Many major RAID cabinet manufacturers have produced a hardware provider that is designed for use with VDS.</span></span> <span data-ttu-id="a99db-135">Dienstconsumer müssen einen Hardware Anbieter und zugehörige Hardware vom Hersteller erhalten.</span><span class="sxs-lookup"><span data-stu-id="a99db-135">Service consumers must obtain a hardware provider and associated hardware from the manufacturer.</span></span>

    <span data-ttu-id="a99db-136">Die Funktionen eines Hardware Anbieters hängen von den Funktionen der zugrunde liegenden Hardware ab.</span><span class="sxs-lookup"><span data-stu-id="a99db-136">The capabilities of a hardware provider depend on the capabilities of the underlying hardware.</span></span> <span data-ttu-id="a99db-137">Folglich kann der Grad, in dem die einzelnen Hersteller die API implementieren, variieren.</span><span class="sxs-lookup"><span data-stu-id="a99db-137">Consequently, the degree to which each manufacturer implements the API can vary.</span></span> <span data-ttu-id="a99db-138">Hersteller können z. b. zusätzliche Methoden zum Optimieren von Konfigurationen, zum Überwachen und zum dynamischen Optimieren der Leistung, zur Automatisierung der Fehler Verwaltung oder zur Bereitstellung anderer nützlicher Funktionen einschließen.</span><span class="sxs-lookup"><span data-stu-id="a99db-138">For example, manufacturers can include additional methods to optimize configurations, monitor and dynamically tune performance, automate fault management, or provide other beneficial functionality.</span></span>

    <span data-ttu-id="a99db-139">Hardware Anbieter bieten verschiedene Konfigurationsoptionen, die für die Softwareanbieter nicht verfügbar sind.</span><span class="sxs-lookup"><span data-stu-id="a99db-139">Hardware providers offer several configuration options that are not available to the software providers.</span></span> <span data-ttu-id="a99db-140">Besonders erwähnenswert ist das automagingkonfigurationsmodell, das eine Attribut basierte Ansicht des Speichers für jede Anwendung darstellt.</span><span class="sxs-lookup"><span data-stu-id="a99db-140">Most notable is the automagic configuration model, which presents an attribute-based view of storage to each application.</span></span> <span data-ttu-id="a99db-141">Bindungs Hinweise, wie z. b. "überwiegend Lesevorgänge" oder "schnelle Absturz Wiederherstellung erforderlich" ersetzen die Komplexität der Bindung von physischem Speicher in den virtuellen Speicher.</span><span class="sxs-lookup"><span data-stu-id="a99db-141">Binding hints, such as "mostly reads" or "fast crash recovery required" replace the complexity of binding physical storage into virtual storage.</span></span> <span data-ttu-id="a99db-142">Jeder Hardware Anbieter führt eine Block Zuordnung, eine Speicherplatz Zuordnung und eine Bindungstyp Auswahl basierend auf den Hinweisen aus, die von einer Anwendung übermittelt werden.</span><span class="sxs-lookup"><span data-stu-id="a99db-142">Each hardware provider performs extent mapping, space allocation, and binding-type selection based on the hints that are submitted by an application.</span></span> <span data-ttu-id="a99db-143">Die Beschreibung des kompletten Hardware Anbieters, einschließlich der Konfigurationsoptionen, finden Sie in der Dokumentation des Subsystem-Herstellers.</span><span class="sxs-lookup"><span data-stu-id="a99db-143">For the complete hardware provider description, including the configuration options, see the documentation that is supplied by the subsystem manufacturer.</span></span>
