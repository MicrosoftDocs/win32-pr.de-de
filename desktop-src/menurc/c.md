<dl> <dt>

<span id="tools.c_1_gly"></span><span id="TOOLS.C_1_GLY"></span>**Zeichen folgen Konstanten können nicht wieder verwendet werden.**
</dt> <dd>

Der gleiche Wert wird in einer [**STRINGTABLE**](stringtable-resource.md) -Anweisung zweimal verwendet. Stellen Sie sicher, dass sich überlappende Dezimal-und hexadezimale Werte nicht gemischt befinden.

</dd> <dt>

<span id="tools.c_2_gly"></span><span id="TOOLS.C_2_GLY"></span>**Steuerzeichen außerhalb des gültigen Bereichs \[ A? Z\]**
</dt> <dd>

Ein Steuerzeichen in der [**Accelerators**](accelerators-resource.md) -Anweisung ist ungültig. Das Zeichen, das auf das Caretzeichen (^) folgt, muss im Bereich von A bis Z liegen.

</dd> <dt>

<span id="tools.c_3_gly"></span><span id="TOOLS.C_3_GLY"></span>**Openin-File-Name konnte nicht geöffnet werden.**
</dt> <dd>

Der Microsoft Windows-Ressourcen Compiler (RC) konnte die angegebene Datei nicht öffnen. Stellen Sie sicher, dass die Datei vorhanden ist und dass Sie den Dateinamen ordnungsgemäß eingegeben haben.

</dd> <dt>

<span id="tools.c_4_gly"></span><span id="TOOLS.C_4_GLY"></span>**Openresource-Name konnte nicht gefunden werden**
</dt> <dd>

Der Microsoft Windows-Ressourcen Compiler (RC) konnte die angegebene Datei nicht öffnen. Stellen Sie sicher, dass die Datei vorhanden ist und dass Sie den Dateinamen ordnungsgemäß eingegeben haben.

</dd> <dt>

<span id="tools.c_5_gly"></span><span id="TOOLS.C_5_GLY"></span>**Name der Datenquelle (Name)**
</dt> <dd>

(V) der Microsoft Windows-Ressourcen Compiler (RC) erstellt eine neue binäre Ressourcen Datei (. res).

</dd> </dl>
