# <a name="standards"></a>Standards

Windows Internet Explorer 8 enthält eine neue Rendering- und Layout-Engine, die CSS 2.1 vollständig implementiert und zusätzliche HTML 4.0-Kompatibilität enthält (im Gegensatz zu früheren Versionen von Windows Internet Explorer). Die umfassendere Unterstützung von Webstandards in Internet Explorer 8 kann dazu führen, dass einige Webanwendungen falsch funktionieren, da sie nicht für Microsoft Internet Explorer 6 oder nicht standardkonform geschrieben sind. Anwendungen können auf unterschiedliche Weise fehlschlagen, z. B. Bilder oder Text, die an verschiedenen Stellen angezeigt werden, oder ganze Anwendungen, die auf eine Weise gerendert werden, die die Anwendung unbrauchbar macht.

Internet Explorer 8 enthält auch eine verbesserte JavaScript-Engine. Diese neue Skript-Engine verbessert die Leistung, unterstützt neue Methoden nativ, bietet erhöhte Sicherheit und ändert einige Funktionen so, dass sie standardkonformer funktionieren. Die verbesserte Leistung führt nicht zu Problemen bei der Anwendungskompatibilität, aber andere Änderungen können dazu führen, dass einige Webanwendungen nicht mehr wie erwartet funktionieren. Weitere Informationen zu Kompatibilitätsproblemen im Zusammenhang mit JavaScript-Änderungen finden Sie unter [Sitekompatibilität und Internet Explorer 8.](/archive/blogs/ie/site-compatibility-and-ie8)
