# <a name="standards"></a>Standards

Windows Internet Explorer 8 enthält eine neue Rendering-und LayoutEngine, die CSS 2,1 vollständig implementiert und eine zusätzliche HTML-4,0-Konformität umfasst (im Gegensatz zu früheren Versionen von Windows Internet Explorer). Die umfassendere Unterstützung von Webstandards in Internet Explorer 8 kann dazu führen, dass einige Webanwendungen unkorrekt funktionieren, da Sie nicht für Microsoft Internet Explorer 6 oder nicht standardkonform geschrieben wurden. Anwendungen können auf unterschiedliche Weise fehlschlagen, z.b. Bilder oder Text, die an unterschiedlichen Speicherorten oder in ganzen Anwendungen angezeigt werden, die auf eine Weise dargestellt werden, mit der die Anwendung unbrauchbar wird.

Internet Explorer 8 enthält auch ein verbessertes JavaScript-Modul. Mit dieser neuen Skript-Engine wird die Leistung verbessert, neue Methoden nativ unterstützt, höhere Sicherheit erzielt und einige Funktionen so geändert, dass Sie auf eine besser kompatible Weise funktionieren. Die verbesserte Leistung führt nicht zu Anwendungs Kompatibilitätsproblemen, aber andere Änderungen können dazu führen, dass einige Webanwendungen nicht mehr wie erwartet funktionieren. Weitere Informationen zu Kompatibilitätsproblemen im Zusammenhang mit JavaScript-Änderungen finden Sie unter [Website Kompatibilität und Internet Explorer 8](/archive/blogs/ie/site-compatibility-and-ie8).
