# <a name="standards"></a>Standards

Windows Internet Explorer 8 enthält eine neue Rendering- und Layout-Engine, die CSS 2.1 vollständig implementiert und zusätzliche HTML 4.0-Kompatibilität enthält (im Gegensatz zu früheren Versionen von Windows Internet Explorer). Die umfassendere Unterstützung von Webstandards in Internet Explorer 8 kann dazu führen, dass einige Webanwendungen unkorrekt ausgeführt werden, da sie nicht für Microsoft Internet Explorer 6 oder auf nicht konforme Weise geschrieben werden. Anwendungen können auf unterschiedliche Weise fehlschlagen, z. B. Bilder oder Text, die an unterschiedlichen Speicherorten angezeigt werden, oder ganze Anwendungen, die so gerendert werden, dass die Anwendung nicht mehr verwendet werden kann.

Internet Explorer 8 enthält auch eine verbesserte JavaScript-Engine. Diese neue Skript-Engine verbessert die Leistung, unterstützt neue Methoden nativ, bietet mehr Sicherheit und ändert einige Funktionen, um standardskonformer zu arbeiten. Die verbesserte Leistung führt nicht zu Problemen mit der Anwendungskompatibilität, aber andere Änderungen können dazu führen, dass einige Webanwendungen nicht mehr wie erwartet funktionieren. Weitere Informationen zu Kompatibilitätsproblemen im Zusammenhang mit JavaScript-Änderungen finden Sie unter [Websitekompatibilität und Internet Explorer 8.](/archive/blogs/ie/site-compatibility-and-ie8)
