# <a name="windows-troubleshooting"></a>Windows-Problembehandlung

## <a name="affected-platforms"></a>Betroffene Plattformen

**Clients** – Windows 7  
**Server** – Windows Server 2008 R2  









## <a name="description"></a>BESCHREIBUNG

Ein neues Windows 7 Action Center-Feature (früher als Lösungscenter bezeichnet), Windows-Problembehandlung, bietet dem Benutzer eine systematische Problembehandlung. Das Aktionscenter ist eines der fünf Symbole, die in der Systray angeheftet sind. Windows-Problembehandlung können Sie nach in-box-Problembehandlungspaketen und problembehebungsbasierten Paketen suchen, die auf einem Microsoft-Server im Internet gespeichert sind – eine bessere BwC-Erfahrung (Better When Connected). Sie können ein Problembehandlungspaket auswählen und ausführen, um zu versuchen, das Problem zu beheben. Wenn Sie keine Lösung für das Problem identifizieren können, haben Sie die Möglichkeit, Hilfe, Communityinhalte und Supportartikel oder andere Problembehandlungspakete nach relevanten verwandten Inhalten zu durchsuchen. Wenn dies keine Lösung für das Problem bietet, können Sie den PC auf einen Zeitpunkt vor dem Problem wiederherstellen oder Hilfe über Remoteunterstützung erhalten. Die Absicht besteht in der einfachen und schnellen Suche nach einer Lösung für das Problem.

## <a name="usage"></a>Verbrauch

Das Aktionscenter ist deutlich sichtbar und von mehreren Standorten aus verfügbar. Sie können sie über das Kontextmenü des Aktionscenters in der Systray, über die Systemsteuerung als Verknüpfungslink unter System und Wartung, über die Hauptseite des Aktionscenters und über Hilfeinhalte starten.

Problembehandlungspakete basieren auf PowerShell-Skripts. Der Prozess der Erstellung und Veröffentlichung eines Pakets zur Problembehandlung wird öffentlich gemacht, damit OEMs, IHVs, ISVs und IT-Experten eigene Problembehandlungsinhalte entwickeln und versenden können.

Um eine konsistente Benutzererfahrung zu gewährleisten, befolgen Sie beim Entwerfen eigener Problembehandlungspakete die bewährten Methoden und Richtlinien, die im Windows-Toolkit zur Problembehandlung beschrieben sind.
