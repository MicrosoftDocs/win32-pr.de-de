# <a name="compatibility-administrator"></a>Kompatibilitätsadministrator

## <a name="affected-platforms"></a>Betroffene Plattformen

 **Clients:** Windows 2000, Windows XP, Windows Vista, Windows 7  
**Server:** Windows Server 2003, Windows Server 2008, Windows Server 2008 R2  


## <a name="description"></a>BESCHREIBUNG

Mit dem Kompatibilitätsadministratortool, das vom Application Compatibility Toolkit (ACT) bereitgestellt wird, können Sie viele Ihrer potenziellen Probleme mit der Anwendungskompatibilität beheben, bevor Sie eine neue Version von Windows in Ihrer Organisation bereitstellen, indem Sie:

-   Bereitstellen einzelner Kompatibilitätskorrekturen, Kompatibilitätsmodi und AppHelp-Meldungen, mit denen Sie bestimmte Kompatibilitätsprobleme beheben können
-   Sie können benutzerdefinierte Kompatibilitätskorrekturen, Kompatibilitätsmodi, AppHelp-Meldungen und Kompatibilitätsdatenbanken erstellen.
-   Bereitstellen eines Abfragetools, mit dem Sie nach installierten Fehlerbehebungen auf Ihren lokalen Computern suchen können

## <a name="usage"></a>Verbrauch

Das folgende Flussdiagramm veranschaulicht die Schritte, die bei der Verwendung des Kompatibilitätsadministrators erforderlich sind, um Ihre Kompatibilitätskorrekturen, Kompatibilitätsmodi und Ihre AppHelp-Nachrichten zu erstellen.



|                                            |          |                                                                                            |          |                                                     |          |                                                                             |
|--------------------------------------------|----------|--------------------------------------------------------------------------------------------|----------|-----------------------------------------------------|----------|-----------------------------------------------------------------------------|
| Erstellen einer neuen Kompatibilitätsdatenbank (.sdb) | **>** | Wählen Sie die Anwendung aus, und wählen Sie dann die Kompatibilitätskorrekturen aus, die auf die Anwendung angewendet werden sollen. | **>** | Testen der Anwendung mit dem neuen Kompatibilitätsfix | **>** | Speichern Sie die Kompatibilitätsdatenbank, und stellen Sie die Korrektur für Ihr Unternehmen bereit. |
