# <a name="windows-7-and-windows-server-2008-r2-application-quality-cookbook"></a>Windows 7 und Windows Server 2008 R2 Application Quality Cookbook

Das *Windows 7 und Windows Server 2008 R2 Application Quality Cookbook* vertrautt Anwendungsentwickler mit der Überprüfung der Kompatibilität ihrer Anwendungen mit dem neuen Betriebssystem und bietet eine Übersicht über die wenigen bekannten Probleme mit der Anwendungskompatibilität in Windows 7 und Windows Server 2008 R2. Außerdem werden Unterschiede in Bezug auf Leistung, Zuverlässigkeit und Benutzerfreundlichkeit sowie Links zu ausführlichen Whitepapers und anderen Entwicklerleitfäden beschrieben.

## <a name="overview"></a>Übersicht

Windows 7 und Windows Server 2008 R2 wird die neueste Betriebssystemtechnologie und Softwareentwicklungsplattform eingeführt, die von Anwendungsentwicklern und Unternehmen weltweit verwendet werden kann. Im Rahmen der weiteren Verbesserung der Sicherheit, Zuverlässigkeit, Leistung und Benutzererfahrung von Windows wurden viele neue Features eingeführt, vorhandene Features wurden verbessert, und einige Features wurden entfernt.

Während Windows 7 und Windows Server 2008 R2 mit den meisten ihrer jeweiligen Anwendungen, die für Windows XP, Windows Server 2003, Windows Vista, Windows Server 2008, Windows Server 2008 R2 und ihre Service Packs geschrieben wurden, hochkompatibel sind, sind einige Kompatibilitätsunterbrechungen aufgrund von Innovationen, Sicherheitsverschärfungen und höherer Zuverlässigkeit unvermeidlich. Insgesamt ist die Kompatibilität von Windows 7 und Windows Server 2008 R2 mit vorhandenen Anwendungen hoch.

Dieses Dokument baut auf den Konzepten auf, die im [Cookbook zur Anwendungskompatibilität von Windows Vista und Windows Server 2008](/previous-versions/bb757005(v=msdn.10))enthalten sind. (Diese Ressource ist in einigen Sprachen und Ländern/Regionen möglicherweise nicht verfügbar.) Dieses Dokument bietet Ihnen die Möglichkeit, sich mit der Überprüfung der Kompatibilität Ihrer Anwendungen mit dem neuen Betriebssystem vertraut zu machen, und bietet eine Übersicht über die wenigen bekannten Inkompatibilitätsprobleme bei Anwendungen in Windows 7 und Windows Server 2008 R2. Darüber hinaus werden jedoch auch Unterschiede in Bezug auf Leistung, Zuverlässigkeit und Benutzerfreundlichkeit sowie Links zu ausführlichen Whitepapers und anderen Anleitungen für Entwickler beschrieben.

Darüber hinaus investiert Microsoft in verschiedene neue und erweiterte Features und Tools, mit denen Sie qualitativ hochwertigere Anwendungen erstellen und Probleme beheben können, wenn Anwendungen auf Windows 7 und Windows Server 2008 R2 nicht ordnungsgemäß funktionieren. Diese Tools enthalten Anweisungen zum Qualifizieren Ihrer Client- und Serveranwendungen für die Teilnahme am Windows Logo-Programm.

Dieses *Cookbook* enthält mehr als drei Dutzend Themen, die in drei Hauptabschnitte unterteilt sind:

-   [Kompatibilität](compatibility.md) (sowohl allgemein als auch serverspezifisch)
-   [Neue Features und Verbesserungen](new-features-and-enhancements.md)
-   [Tools, bewährte Methoden und Anleitungen](tools--resources--and-guidance.md)

Wir laden Sie ein, diese Themen zu lesen, um zu erfahren, wie Sie Ihre Anwendung optimieren können, um die Vorteile von Windows 7 zu nutzen.

Wir nehmen Ihre Vorschläge für Verbesserungen an diesem Dokument sehr wichtig.
