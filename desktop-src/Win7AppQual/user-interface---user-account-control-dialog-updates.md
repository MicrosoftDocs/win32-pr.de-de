# <a name="user-interface---user-account-control-dialog-updates"></a>Benutzeroberfläche : Updates des Dialogfelds "Benutzerkontensteuerung"

## <a name="affected-platforms"></a>Betroffene Plattformen

**Clients** – Windows 7  
**Server** – Windows Server 2008 R2  









## <a name="feature-impact"></a>Auswirkungen auf Das Feature

**Schweregrad –** Niedrig  
**Frequency** - Medium  











## <a name="description"></a>BESCHREIBUNG

In Windows 7 haben wir die Optionen des UAC-Dialogfelds standardisiert. Bisher mussten Benutzer aus mehreren Optionen auswählen, z. B. "Weiter", "Abbrechen" usw. Jetzt erhalten Benutzer in allen Dialogfeldern ein einfaches "Ja" oder "Nein". Das Dialogfeldlayout zeigt nun auch den Programmnamen, Herausgeber und Ursprung deutlich an.

## <a name="leveraging-feature-capabilities"></a>Nutzen von Featurefunktionen

Die Anforderungen an die Anwendungsentwicklung in Windows 7 für UAC-Kompatibilität sind mit denen in Windows Vista identisch. Windows Vista-kompatible Anwendungen interagieren ohne Änderungen mit der UAC in Windows 7. Informationen dazu, wie Windows XP-Anwendungen unter Windows 7 ordnungsgemäß funktionieren, finden Sie in den Themen zur Benutzerkontensteuerung im Cookbook zur *Windows Vista-Anwendungskompatibilität.* Die UAC-Verbesserungen für Windows 7 wirken sich zwar auf die Benutzerfreundlichkeit aus, wirken sich jedoch nicht auf die Anwendungsschnittstelle aus. Wenn jedoch Hilfeinhalte mit den UAC-Dialogen verknüpft sind, müssen Sie die Screenshots möglicherweise aktualisieren.
