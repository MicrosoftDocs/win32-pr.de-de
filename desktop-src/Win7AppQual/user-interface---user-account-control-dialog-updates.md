# <a name="user-interface---user-account-control-dialog-updates"></a>Benutzeroberfläche – Dialogfeld "Benutzerkontensteuerung" aktualisiert

## <a name="affected-platforms"></a>Betroffene Plattformen

**Clients** – Windows 7  
**Server** – Windows Server 2008 R2  









## <a name="feature-impact"></a>Auswirkung von Features

**Schweregrad:** Niedrig  
**Häufigkeit** – Mittel  











## <a name="description"></a>Beschreibung

In Windows 7 haben wir die Optionen für UAC-Dialogfelder standardisiert. Zuvor mussten Benutzer aus mehreren Optionen auswählen, z. B. "Weiter", "Abbrechen" und so weiter. Jetzt erhalten Benutzer in allen Dialogfeldern ein einfaches "Ja" oder "Nein". Das Dialoglayout zeigt nun auch den Programmnamen, den Herausgeber und den Ursprung an.

## <a name="leveraging-feature-capabilities"></a>Nutzen von Featurefunktionen

Die Anwendungsentwicklungsanforderungen in Windows 7 für UAC-Kompatibilität sind die gleichen wie in Windows Vista. Windows Vista-kompatible Anwendungen interagieren ohne Änderungen mit UAC in Windows 7. In den Themen zur Benutzerkontensteuerung im Windows Vista Application *Compatibility Cookbook* finden Sie Informationen dazu, wie Sie sicherstellen, dass Windows XP-Anwendungen auf Windows 7 ordnungsgemäß funktionieren. Die UAC-Verbesserungen für Windows 7 wirken sich zwar auf die Benutzerfreundlichkeit aus, wirken sich jedoch nicht auf die Anwendungsschnittstelle aus. Wenn jedoch Hilfeinhalte mit den UAC-Dialogen verknüpft sind, müssen Sie die Screenshots möglicherweise aktualisieren.
