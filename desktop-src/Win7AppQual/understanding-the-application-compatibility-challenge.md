# <a name="understanding-the-application-compatibility-challenge"></a>Grundlegendes zu Herausforderungen der Anwendungskompatibilität

Windows Internet Explorer 8 ist der Standardbrowser im Windows 7-Betriebssystem. Internet Explorer 8 verbessert viele Features, einschließlich einer erweiterten Plattformarchitektur, die das Web sicherer und schneller für Benutzer und einfacher für IT-Experten konfigurieren und verwalten sowie innovative Inhalte für Webentwickler erstellen soll.

Einige dieser Features verwenden neue Rendering- und Skript-Engines. Dies sind die Kernelemente, die Windows Internet Explorer zum Anzeigen von HTML- und CSS-Inhalten und Ausführen von Skripts verwendet. Die Architekturänderungen und Plattformverbesserungen führen zu Anwendungskompatibilitätsproblemen, die auftreten können, wenn Sie Anwendungen aus einem vorherigen Browser zu Internet Explorer 8 migrieren.

Internet Explorer 8 enthält eine gewisse Abwärtskompatibilität, aber es gibt einige Änderungen (einschließlich Unterstützung für Die Unterstützung von Webstandards und Sicherheitsfeatures), die möglicherweise ein Upgrade von Webanwendungen erfordern, die für Microsoft Internet Explorer 6 oder frühere Versionen erstellt wurden.

Weitere Informationen zum Testen und Beheben von Kompatibilitätsproblemen für Internet Explorer 8 finden Sie unter [Addressing Application Compatibility When Migrating to Internet Explorer 8 – Information for Information Technology Professionals](/previous-versions/windows/it-pro/windows-7/ff955640(v=ws.10)).
