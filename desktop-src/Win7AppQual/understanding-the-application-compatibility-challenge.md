# <a name="understanding-the-application-compatibility-challenge"></a>Grundlegendes zu Herausforderungen der Anwendungskompatibilität

Windows Internet Explorer 8 ist der Standardbrowser im Windows 7-Betriebssystem. Internet Explorer 8 verbessert viele Features, einschließlich einer verbesserten Plattformarchitektur, die dafür konzipiert ist, das Web für Benutzer sicherer und schneller zu machen und IT-Experten einfacher zu konfigurieren und zu verwalten sowie innovative Inhalte für Webentwickler zu erstellen.

Einige dieser Features verwenden neue Rendering- und Skript-Engines. Dabei handelt es sich um die Kernelemente, die Windows Internet Explorer zum Anzeigen von HTML- und CSS-Inhalten und zum Ausführen von Skripts verwenden. Die Architekturänderungen und Plattformverbesserungen führen zu Problemen bei der Anwendungskompatibilität, die bei der Migration von Anwendungen zu Internet Explorer 8 aus einem vorherigen Browser möglicherweise auft kommen.

Internet Explorer 8 enthält abwärtskompatibel, aber es gibt einige Änderungen (einschließlich Unterstützung für Unterstützung von Webstandards und Sicherheitsfeatures), die möglicherweise ein Upgrade von Webanwendungen erfordern, die für Microsoft Internet Explorer 6 oder frühere Versionen erstellt wurden.

Weitere Informationen zum Testen und Beheben von Kompatibilitätsproblemen für Internet Explorer 8 finden Sie unter [Addressing Application Compatibility When Migrating to Internet Explorer 8](/previous-versions/windows/it-pro/windows-7/ff955640(v=ws.10))– Information for Information Technology Professionals .
