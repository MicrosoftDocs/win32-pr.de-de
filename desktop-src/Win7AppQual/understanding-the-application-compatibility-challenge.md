# <a name="understanding-the-application-compatibility-challenge"></a>Grundlegendes zu Herausforderungen der Anwendungskompatibilität

Windows Internet Explorer 8 ist der Standardbrowser im Windows 7-Betriebssystem. Internet Explorer 8 verbessert viele Features, darunter eine erweiterte Plattformarchitektur, die entwickelt wurde, um das Web für Benutzer sicherer und schneller zu gestalten und die Konfiguration und Verwaltung von IT-Experten zu vereinfachen und innovative Inhalte für Webentwickler zu erstellen.

Für einige dieser Features werden neue renderingmodule und Skript Module verwendet, die von Windows Internet Explorer zum Anzeigen von HTML-und CSS-Inhalten und Ausführen von Skripts verwendet werden. Die Architektur Änderungen und Platt Form Erweiterungen führen zu Anwendungs Kompatibilitätsproblemen, die beim Migrieren von Anwendungen zu Internet Explorer 8 aus einem vorherigen Browser auftreten können.

Internet Explorer 8 enthält eine gewisse Abwärtskompatibilität, aber es gibt einige Änderungen (einschließlich der Unterstützung für Webstandards-Unterstützung und Sicherheitsfeatures), für die Sie möglicherweise Webanwendungen aktualisieren müssen, die für Microsoft Internet Explorer 6 oder frühere Versionen erstellt wurden.

Weitere Informationen zum Testen und behandeln von Kompatibilitätsproblemen für Internet Explorer 8 finden Sie unter [adressieren der Anwendungs Kompatibilität bei der Migration zu Internet Explorer 8 – Informationen für IT-Experten](/previous-versions/windows/it-pro/windows-7/ff955640(v=ws.10)).
