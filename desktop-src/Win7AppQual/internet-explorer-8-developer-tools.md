# <a name="internet-explorer-8-developer-tools"></a>Internet Explorer 8-Entwicklertools

Windows Internet Explorer 8 enthält das Entwicklertools Feature. Mit diesem Feature können Sie Microsoft JScript schnell debuggen, ein Windows Internet Explorer spezifisches Verhalten untersuchen oder schnell einen Prototyp erstellen oder eine neue Entwurfslösung testen.

In den folgenden Abschnitten werden die wichtigen Merkmale der Entwicklertools in Internet Explorer 8 beschrieben.

## <a name="integrated-and-simple-to-use"></a>Integriert und einfach zu verwenden

Die Entwicklertools sind in jeder Installation von Internet Explorer 8 enthalten, sodass Sie Probleme auf jedem Computer debuggen können, auf dem Internet Explorer 8 ausgeführt wird. Darüber hinaus werden die Tools nur geladen, wenn Sie sie benötigen, um die Auswirkungen auf die Leistung des Browsers zu beschränken. Mithilfe der Entwicklertools können Sie schnell nur den aktuellen Internet Explorer debuggen, anstatt für alle Internet Explorer.

## <a name="provide-a-visual-interface-to-the-platform"></a>Bereitstellen einer grafischen Benutzeroberfläche für die Plattform

Anstatt reverse engineering how your website works or modifying your site to output debug information, the Entwicklertools let you look into Internet Explorer to view its representation of your site. Diese Ansicht reduziert die Zeit, die Sie mit dem Debuggen dynamischer Websites verbringen, bei denen die Quelluntersuchung nicht nützlich ist, oder das Untersuchen eines Internet Explorer-spezifischen Verhaltens, bei dem ein generisches Erstellungstool nicht helfen kann.

## <a name="enable-fast-experimentation"></a>Schnelles Experimentieren aktivieren

Wenn Sie in früheren Versionen von Internet Explorer eine Prototyperstellung für einen neuen Entwurf erstellen oder Fixes testen, bearbeiten Sie wahrscheinlich Ihre Quelle, speichern sie, aktualisieren Ihre Seite im Browser und wiederholen sie. Die Entwicklertools dieses Szenario optimieren, indem Sie Ihre Website im Browser bearbeiten und Änderungen sofort anzeigen können.

## <a name="optimize-application-performance"></a>Optimieren der Anwendungsleistung

Um Leistungsprobleme zu identifizieren und zu beheben, iterieren Sie wahrscheinlich, indem Sie sich auf ein Szenario nach dem anderen konzentrieren. Mithilfe des Skriptprofilers in der Entwicklertools können Sie Statistiken sammeln, einschließlich der Ausführungszeit und der Anzahl, wie oft eine JavaScript-Funktion aufgerufen wird. Sie können diese Informationen verwenden, wenn Sie Ihre Anwendung testen und den Profilbericht verwenden, um Leistungsengpässe schnell zu identifizieren und zu beheben.

Um auf die Entwicklertools zugreifen zu können, drücken Sie F12, während Sie eine Webseite anzeigen, oder klicken **Sie Entwicklertools** Menü **Extras auf Klicken.**
