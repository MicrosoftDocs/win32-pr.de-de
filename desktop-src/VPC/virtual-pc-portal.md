# <a name="windows-virtual-pc"></a>Windows Virtual PC

\[Windows Virtual PC ist nicht mehr für die Verwendung ab Windows 8 verfügbar. Verwenden Sie stattdessen den [Hyper-V-WMI-Anbieter (v2)](../hyperv_v2/windows-virtualization-portal.md).\]

## <a name="purpose"></a>Zweck

Diese Dokumentation enthält Informationen zur Windows Virtual PC-COM-Schnittstelle.

## <a name="where-applicable"></a>Anwendungsbereich

Windows Virtual PC ist die neueste Microsoft-Virtualisierungstechnologie. mit diesem Dienst können Sie viele Produktivitätsanwendungen in einer virtuellen Windows-Umgebung mit einem einzigen Mausklick direkt von Windows 7 ausführen. Sie können separate virtuelle Computer zusätzlich zu Ihrem Windows 7-Desktop erstellen. Jeder virtuelle Computer emuliert ein komplett Hardwaresystem – vom Prozessor bis zur Netzwerkkarte – in einer eigenständigen, isolierten Softwareumgebung, die den gleichzeitigen Betrieb von anderweitig inkompatiblen Systemen ermöglicht.

## <a name="developer-audience"></a>Entwicklergruppe

Die Windows Virtual PC-COM-Schnittstellen sind für Entwickler gedacht, die Client Anwendungen erstellen, die die Bereitstellung und den Betrieb von virtuellen Computern automatisieren.

## <a name="run-time-requirements"></a>Laufzeitanforderungen

Windows Virtual PC erfordert eine der folgenden Windows 7-Editionen: Home Basic, Home Premium, Professional, Ultimate oder Enterprise Edition.

Zum Abrufen der zugehörigen Header-und Bibliotheksdateien installieren Sie den Windows SDK für Windows 7 aus dem [Microsoft Download Center](https://www.microsoft.com/download/details.aspx?id=8279).
