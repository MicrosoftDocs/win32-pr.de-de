# <a name="registry-key-security-and-access-rights"></a>Sicherheit und Zugriffsrechte für den Registrierungsschlüssel

Das Windows-Sicherheitsmodell ermöglicht es Ihnen, den Zugriff auf Registrierungsschlüssel zu steuern. Weitere Informationen zur Sicherheit finden Sie unter [Zugriffs Steuerungsmodell](/windows/desktop/SecAuthZ/access-control-model).

Sie können eine [Sicherheits Beschreibung](/windows/desktop/SecAuthZ/security-descriptors) für einen Registrierungsschlüssel angeben, wenn Sie die [**regupatekeyex**](/windows/desktop/api/Winreg/nf-winreg-regcreatekeyexa) -oder [**regsetkeysecurity**](/windows/desktop/api/winreg/nf-winreg-regsetkeysecurity) -Funktion aufrufen. Wenn Sie **null** angeben, erhält der Schlüssel eine Standard Sicherheits Beschreibung. Die ACLs in einer Standard Sicherheits Beschreibung für einen Schlüssel werden von seinem direkten übergeordneten Schlüssel geerbt.

Um die Sicherheits Beschreibung eines Registrierungsschlüssels abzurufen, müssen Sie die [**reggetkeysecurity**](/windows/desktop/api/winreg/nf-winreg-reggetkeysecurity)-, [**GetNamedSecurityInfo**](/windows/desktop/api/aclapi/nf-aclapi-getnamedsecurityinfoa)-oder [**GetSecurityInfo**](/windows/desktop/api/aclapi/nf-aclapi-getsecurityinfo) -Funktion aufrufen.

Zu den gültigen Zugriffsrechten für Registrierungsschlüssel zählen die \_ Standard Zugriffsrechte DELETE, Read Control, Write \_ DAC und Write \_ Owner. [](/windows/desktop/SecAuthZ/standard-access-rights) Die Standard Zugriffsrechte für die Registrierung werden von Registrierungs Schlüsseln nicht unterstützt.

Wenn Sie die [**RegOpenKeyEx**](/windows/desktop/api/Winreg/nf-winreg-regopenkeyexa) -Funktion aufrufen, prüft das System die angeforderten Zugriffsrechte auf die Sicherheits Beschreibung des Schlüssels. Wenn der Benutzer nicht über den richtigen Zugriff auf den Registrierungsschlüssel verfügt, schlägt der Öffnungsvorgang fehl. Wenn ein Administrator Zugriff auf den Schlüssel benötigt, besteht die Lösung darin, die \_ Berechtigung zum Übernehmen des \_ Besitz Eigentums namens zu aktivieren \_ und den Registrierungsschlüssel mit Schreibzugriff zu öffnen \_ . Weitere Informationen finden Sie unter [aktivieren und Deaktivieren von Berechtigungen](/windows/desktop/SecAuthZ/enabling-and-disabling-privileges-in-c--).

Sie können das Zugriffs \_ System \_ -Sicherheits Zugriffsrecht auf einen Registrierungsschlüssel anfordern, wenn Sie die System Zugriffs Steuerungs Liste (SACL) des Schlüssels lesen oder schreiben möchten. Weitere Informationen finden Sie unter [Zugriffs Steuerungs Listen (Access Control Lists, ACLs)](/windows/desktop/SecAuthZ/access-control-lists) und [SACL-Zugriffsrecht](/windows/desktop/SecAuthZ/sacl-access-right).

Um die aktuellen Zugriffsrechte für einen Schlüssel, einschließlich der vordefinierten Schlüssel, anzuzeigen, verwenden Sie den Registrierungs-Editor (Regedt32.exe). Wechseln Sie nach dem Navigieren zum gewünschten Schlüssel zum Menü **Bearbeiten** , und wählen Sie **Berechtigungen** aus.
