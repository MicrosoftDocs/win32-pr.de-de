# <a name="predefined-keys"></a>Vordefinierte Schlüssel

Eine Anwendung muss einen Schlüssel öffnen, bevor Sie der Registrierung Daten hinzufügen kann. Um einen Schlüssel zu öffnen, muss eine Anwendung ein Handle für einen anderen Schlüssel in der Registrierung bereitstellen, der bereits geöffnet ist. Das System definiert vordefinierte Schlüssel, die immer geöffnet sind. Vordefinierte Schlüssel helfen einer Anwendung, in der Registrierung zu navigieren und es ermöglicht, Tools zu entwickeln, mit denen ein Systemadministrator Datenkategorien bearbeiten kann. Anwendungen, die Daten zur Registrierung hinzufügen, sollten immer innerhalb des Frameworks von vordefinierten Schlüsseln funktionieren, sodass die neuen Daten von Verwaltungs Tools gefunden und verwendet werden können.

Eine Anwendung kann Handles für diese Schlüssel als Einstiegspunkte für die Registrierung verwenden. Diese Handles sind für alle Implementierungen der Registrierung gültig, obwohl die Verwendung der Handles von Plattform zu Plattform abweichen kann. Mit der [**regoverridepredefkey**](/windows/desktop/api/Winreg/nf-winreg-regoverridepredefkey) -Funktion können Sie einem angegebenen Schlüssel in der Registrierung einen vordefinierten Registrierungsschlüssel zuordnen. Beispielsweise kann ein Software Installationsprogramm vor der Installation einer DLL-Komponente einen vordefinierten Schlüssel neu zuordnen. Dadurch kann das Installationsprogramm problemlos die Informationen untersuchen, die von der Installationsprozedur der dll in den vordefinierten Schlüssel geschrieben werden.

Die Funktionen " [**regdisablepredefinedcache**](/windows/desktop/api/Winreg/nf-winreg-regdisablepredefinedcache) " und " [**regdisablepredefinedcacheex**](/windows/desktop/api/Winreg/nf-winreg-regdisablepredefinedcacheex) " deaktivieren die Handle-Zwischenspeicherung für vordefinierte Registrierungs Handles. Dienste, die den Identitätswechsel verwenden, sollten **regdisablepredefinedcacheex** aufrufen, bevor vordefinierte Registrierungs Handles verwendet werden.

Die vordefinierten Handles sind nicht Thread sicher. Das Schließen eines vordefinierten Handles in einem Thread wirkt sich auf alle anderen Threads aus, die das Handle verwenden.
