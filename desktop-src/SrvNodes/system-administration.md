# <a name="system-administration"></a>Systemverwaltung

> [!Note]  
> Diese Themen sind für Softwareentwickler vorgesehen. Informationen zu Kundensupportproblemen finden Sie unter [Support – Windows Hilfe.](https://windows.microsoft.com/windows/support#1TC=windows-10)

 

## <a name="in-this-section"></a>In diesem Abschnitt

<dl> <dt>

[Startkonfigurationsdaten WMI-Anbieter](/previous-versions/windows/desktop/bcd/boot-configuration-data-portal)
</dt> <dd>

Der anbieter Startkonfigurationsdaten (BCD) Windows Management Instrumentation (WMI) bietet programmgesteuerten Zugriff auf BCD-Speicher, die Startanwendungen und Startanwendungseinstellungen beschreiben.

</dd> <dt>

[Gruppenrichtlinie](group-policy.md)
</dt> <dd>

Gruppenrichtlinie ermöglicht die richtlinienbasierte Verwaltung mithilfe von Microsoft Active Directory-Verzeichnisdiensten. Gruppenrichtlinie verwendet Verzeichnisdienste und die Mitgliedschaft in Sicherheitsgruppen, um Flexibilität zu bieten und umfassende Konfigurationsinformationen zu unterstützen.

</dd> <dt>

[Microsoft Management Console (MMC)](microsoft-management-console--mmc-.md)
</dt> <dd>

Microsoft Management Console (MMC) bietet eine einfache, konsistente und integrierte Benutzeroberfläche und ein integriertes Verwaltungsmodell.

</dd> <dt>

[Mobile Geräteverwaltung Application Provider](/previous-versions/windows/desktop/mdmappprov/mobile-device-management-application-provider-portal)
</dt> <dd>

Der Mobile Geräteverwaltung(MDM)-Anwendungsanbieter verwaltet Anwendungen auf Geräten, die einen MDM-Dienst abonniert haben.

</dd> <dt>

[Mobile Geräteverwaltung-Registrierung](/windows/desktop/MDMReg/mobile-device-management-registration-portal)
</dt> <dd>

Die MDM-Registrierung registriert Geräte bei einem MDM-Dienst.

</dd> <dt>

[Mobile Geräteverwaltung Einstellungen Provider](/previous-versions/windows/desktop/mdmsettingsprov/mobile-device-management-settings-provider-portal)
</dt> <dd>

Der MDM Einstellungen-Anbieter ermöglicht die Verwaltung von Einstellungen auf Geräten, die einen MDM-Dienst abonniert haben.

</dd> <dt>

[NetShell](/previous-versions/windows/desktop/netshell/netshell-start-page)
</dt> <dd>

NetShell ist ein befehlszeilenbasiertes Tool, mit dem Administratoren wichtige Netzwerkdienste remote verwalten und konfigurieren können.

</dd> <dt>

[Einstellungen Verwaltungsinfrastruktur](/previous-versions/windows/desktop/smi/settings-management-infrastructure--smi-)
</dt> <dd>

Einstellungen Die Verwaltungsinfrastruktur (Management Infrastructure, SMI) bietet eine standardisierte Infrastruktur für den Zugriff auf und die Bearbeitung von Einstellungen, die von Benutzern und Anwendungen geändert werden können.

</dd> <dt>

[Softwareinventurprotokollierung](/previous-versions/windows/desktop/sil/software-inventory-logging-portal)
</dt> <dd>

Die Protokollierung des Softwarebestands sammelt Lizenzierungsdaten zu Software, die auf einem Windows Server installiert ist, und bietet Remotezugriff auf die Daten, sodass sie problemlos von einem Rechenzentrum aggregiert werden können.

</dd> <dt>

[Softwarelizenzierungs-API](/previous-versions/windows/desktop/secslapi/software-licensing-api-portal)
</dt> <dd>

Die Softwarelizenzierungs-API (SLAPI) kann verwendet werden, um eine Originalinstallation von Microsoft Windows zu bestimmen, eine Asset Management-Lizenz zu installieren und zu protokollieren und Informationen zur Lizenzierungsrichtlinie einer Softwarekomponente abzurufen.

</dd> <dt>

[Systemwiederherstellung](/windows/desktop/sr/system-restore-portal)
</dt> <dd>

Die Systemwiederherstellung überwacht und zeichnet wichtige Systemänderungen automatisch auf dem Computer eines Benutzers auf. Es wurde entwickelt, um Supportkosten zu senken und die Kundenzufriedenheit zu erhöhen, indem es einem Benutzer ermöglicht wird, eine Änderung rückgängig zu machen, die möglicherweise ein Problem mit dem System verursacht hat, oder auf einen Tag zurückzukehren, an dem das System optimal funktioniert hat.

</dd> <dt>

[Herunterfahren des Systems](/windows/desktop/Shutdown/system-shutdown)
</dt> <dd>

Mit den Funktionen und Meldungen zum Herunterfahren des Systems können anwendungen den aktuellen Benutzer abmelden, das System herunterfahren oder die Arbeitsstation sperren.

</dd> <dt>

[Aufgabenplanung](/windows/desktop/TaskSchd/task-scheduler-start-page)
</dt> <dd>

Mit dem Taskplaner können Sie automatisch Routineaufgaben auf einem ausgewählten Computer ausführen. Der Taskplaner überwacht dabei die Kriterien, die Sie zum Initiieren der Aufgaben (als Trigger bezeichnet) auswählen, und führt die Aufgaben dann aus, wenn die Kriterien erfüllt sind.

</dd> <dt>

[Benutzerzugriffsprotokollierung](/previous-versions/windows/desktop/ual/user-access-logging)
</dt> <dd>

Die Benutzerzugriffsprotokollierung (User Access Logging, UAL) ist ein gängiges Framework für Windows Serverrollen, um ihre jeweiligen Nutzungsmetriken zu melden. Dieses UAL-Framework ist eine grundlegende und wichtige Komponente der größeren Lösung für die Lizenzierungsverwaltung.

</dd> <dt>

[Windows Bereitstellungsdienste](/windows/desktop/Wds/windows-deployment-services-portal)
</dt> <dd>

Windows Bereitstellungsdienste (Deployment Services, WDS) ist die überarbeitete Version der Remoteinstallationsdienste (REMOTE Installation Services, RIS). WDS ermöglicht die Bereitstellung von Windows Betriebssystemen. Sie können WDS verwenden, um neue Clients mit einer netzwerkbasierten Installation einzurichten, ohne dass Administratoren jeden Computer besuchen oder direkt von CD- oder DVD-Medien installieren müssen.

</dd> <dt>

[Windows Originalvorteil-API](/previous-versions/windows/desktop/wingen/windows-genuine-advantage-api-portal)
</dt> <dd>

Die Windows Originalvorteil-API wird verwendet, um zu bestimmen, ob es sich bei dem Windows Betriebssystem, das auf dem aktuellen System ausgeführt wird, um eine Originalkopie handelt.

</dd> <dt>

[Windows-Verwaltungsinstrumentation](/windows/desktop/WmiSdk/wmi-start-page)
</dt> <dd>

Windows Die Verwaltungsinstrumentation (Management Instrumentation, WMI) ist die Infrastruktur für Verwaltungsdaten und -vorgänge auf Windows-basierten Betriebssystemen. Sie können WMI-Skripts oder -Anwendungen schreiben, um administrative Aufgaben auf Remotecomputern zu automatisieren.

</dd> <dt>

[Windows Defender WMIv2-APIs](/previous-versions/windows/desktop/defender/windows-defender-wmiv2-apis-portal)
</dt> <dd>

Windows Defender WMI-APIs können verwendet werden, um den Schutz vor Schadsoftware über Skripts oder Anwendungen zu verwalten.

</dd> <dt>

[Windows PowerShell](https://msdn.microsoft.com/library/Dd835506(v=VS.85).aspx)
</dt> <dd>

Windows PowerShell ist eine aufgabenbasierte Befehlszeilenshell und Skriptsprache, die speziell für die Systemverwaltung entwickelt wurde. Windows PowerShell basiert auf .NET Framework und unterstützt IT-Experten und erfahrene Benutzer beim Steuern und Automatisieren der Verwaltung von Windows-Betriebssystemen sowie von Anwendungen, die unter Windows ausgeführt werden.

</dd> <dt>

[Windows-Remoteverwaltung](/windows/desktop/WinRM/portal)
</dt> <dd>

Die Windows-Remoteverwaltung (WinRM) ist die Microsoft-Implementierung von WS-Management-Protokoll, einem standardmäßigen SOAP-basierten (Simple Object Access Protocol)-basierten, firewallfreundlichen Protokoll, mit dem Hardware und Betriebssysteme verschiedener Anbieter zusammenarbeiten können. Die WS-Management-Protokollspezifikation stellt eine allgemeine Möglichkeit für Systeme bereit, auf Verwaltungsinformationen in einer IT-Infrastruktur zuzugreifen und diese auszutauschen.

</dd> <dt>

[Windows Ressourcenschutz](/windows/desktop/Wfp/windows-resource-protection-portal)
</dt> <dd>

Windows Resource Protection (WRP) verhindert den Austausch wichtiger Systemdateien, Ordner und Registrierungsschlüssel, die als Teil des Betriebssystems installiert werden.

</dd> <dt>

[Windows Server Update Services](windows-server-update-services.md)
</dt> <dd>

Systemadministratoren können die Windows Server Update Services-API (WSUS) verwenden, um zu bestimmen, welche Updates für einen Computer oder eine Gruppe von Computern gelten, diese Updates herunterzuladen und sie mit nur geringem oder gar keinem Benutzereingriff zu installieren.

Unabhängige Softwarehersteller und Endbenutzerentwickler können WSUS-Features in Die Computerverwaltungs- oder Updateverwaltungssoftware integrieren, um eine nahtlose Betriebsumgebung bereitzustellen.

</dd> <dt>

[Windows Systembewertungstool](/windows/desktop/WinSAT/windows-system-assessment-tool-portal)
</dt> <dd>

Verwenden von WinSAT zum Bewerten der Leistungsmerkmale und -funktionen eines Computers

</dd> <dt>

[Arbeitsordner](/previous-versions/windows/desktop/syncshareservermgmt/sync-share-server-management-portal)
</dt> <dd>

Arbeitsordner ermöglicht Es Benutzern, Arbeitsdateien auf PCs und Geräten zu speichern und darauf zuzugreifen.

</dd> </dl>
