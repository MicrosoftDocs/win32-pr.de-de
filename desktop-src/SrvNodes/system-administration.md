# <a name="system-administration"></a>Systemverwaltung

> [!Note]  
> Diese Themen gelten für Softwareentwickler. Informationen zu Problemen mit dem Kundensupport finden Sie [unter Support-Windows-Hilfe](https://windows.microsoft.com/windows/support#1TC=windows-10).

 

## <a name="in-this-section"></a>In diesem Abschnitt

<dl> <dt>

[WMI-Anbieter Startkonfigurationsdaten](/previous-versions/windows/desktop/bcd/boot-configuration-data-portal)
</dt> <dd>

Der Startkonfigurationsdaten (BCD) Windows-Verwaltungsinstrumentation (WMI)-Anbieter ermöglicht den programmgesteuerten Zugriff auf BCD-Speicher, in dem Start Anwendungen und Start Anwendungseinstellungen beschrieben werden.

</dd> <dt>

[Gruppenrichtlinie](group-policy.md)
</dt> <dd>

Gruppenrichtlinie ermöglicht die Richtlinien basierte Verwaltung mithilfe von Microsoft Active Directory Directory-Diensten. Gruppenrichtlinie verwendet Verzeichnisdienste und die Mitgliedschaft in Sicherheitsgruppen, um Flexibilität zu bieten und umfassende Konfigurationsinformationen zu unterstützen.

</dd> <dt>

[Microsoft Management Console (MMC)](microsoft-management-console--mmc-.md)
</dt> <dd>

Microsoft Management Console (MMC) bietet eine einfache, konsistente und integrierte Verwaltungs Benutzeroberfläche und ein Verwaltungsmodell.

</dd> <dt>

[Anwendungsanbieter für die Verwaltung mobiler Geräte](/previous-versions/windows/desktop/mdmappprov/mobile-device-management-application-provider-portal)
</dt> <dd>

Der MDM-Anwendungsanbieter (Mobile Device Management, Verwaltung mobiler Geräte) verwaltet Anwendungen auf Geräten, die einen MDM-Dienst abonniert haben.

</dd> <dt>

[Registrierung für die Verwaltung mobiler Geräte](/windows/desktop/MDMReg/mobile-device-management-registration-portal)
</dt> <dd>

Bei der MDM-Registrierung werden Geräte in einem MDM-Dienst registriert.

</dd> <dt>

[Anbieter für die Verwaltung mobiler Geräte](/previous-versions/windows/desktop/mdmsettingsprov/mobile-device-management-settings-provider-portal)
</dt> <dd>

Der Anbieter für MDM-Einstellungen ermöglicht die Verwaltung von Einstellungen auf Geräten, die einen MDM-Dienst abonniert haben.

</dd> <dt>

[NetShell](/previous-versions/windows/desktop/netshell/netshell-start-page)
</dt> <dd>

NetShell ist ein Befehlszeilen basiertes Tool, mit dem Administratoren kritische Netzwerkdienste remote verwalten und konfigurieren können.

</dd> <dt>

[Verwaltungsinfrastruktur für Einstellungen](/previous-versions/windows/desktop/smi/settings-management-infrastructure--smi-)
</dt> <dd>

Die Einstellungs Verwaltungsinfrastruktur (SMI) bietet eine standardisierte Infrastruktur zum Zugreifen auf und Ändern von Einstellungen, die von Benutzern und Anwendungen geändert werden können.

</dd> <dt>

[Softwareinventurprotokollierung](/previous-versions/windows/desktop/sil/software-inventory-logging-portal)
</dt> <dd>

Bei der Protokollierung des Software Bestands werden Lizenzierungs Daten zu Software gesammelt, die auf einem Windows-Server installiert ist, und Remote Zugriff auf die Daten ermöglicht, sodass Sie problemlos von einem Rechenzentrum aggregiert werden können.

</dd> <dt>

[Software Lizenzierungs-API](/previous-versions/windows/desktop/secslapi/software-licensing-api-portal)
</dt> <dd>

Die Software Licensing API (slapi) kann verwendet werden, um eine echte Microsoft Windows-Installation zu bestimmen, eine Lizenz für die Asset-Verwaltung zu installieren und zu protokollieren und Informationen über die Lizenzierungs Richtlinie einer Software Komponente abzurufen.

</dd> <dt>

[Systemwiederherstellung](/windows/desktop/sr/system-restore-portal)
</dt> <dd>

Bei der Systemwiederherstellung werden wichtige Systemänderungen auf dem Computer eines Benutzers automatisch überwacht und aufgezeichnet. Es wurde entwickelt, um Supportkosten zu senken und die Kundenzufriedenheit zu erhöhen, indem ein Benutzer eine Änderung rückgängig machen kann, die möglicherweise ein Problem mit dem System verursacht hat, oder bis zu einem Tag zurückgekehrt ist, an dem das System optimal funktioniert.

</dd> <dt>

[Herunterfahren des Systems](/windows/desktop/Shutdown/system-shutdown)
</dt> <dd>

Mit den Funktionen und Nachrichten für das Herunterfahren des Systems können sich Anwendungen den aktuellen Benutzer abmelden, das System Herunterfahren oder die Arbeitsstation sperren.

</dd> <dt>

[Aufgabenplanung](/windows/desktop/TaskSchd/task-scheduler-start-page)
</dt> <dd>

Der Taskplaner ermöglicht Ihnen das automatische Ausführen von Routineaufgaben auf einem ausgewählten Computer. Dies erfolgt durch Taskplaner das Überwachen der Kriterien, die Sie zum Initiieren der Aufgaben auswählen (als Trigger bezeichnet), und zum anschließenden Ausführen der Aufgaben, wenn die Kriterien erfüllt sind.

</dd> <dt>

[Benutzerzugriffsprotokollierung](/previous-versions/windows/desktop/ual/user-access-logging)
</dt> <dd>

Die Benutzer Zugriffs Protokollierung (User Access Logging, UAL) ist ein gängiges Framework für Windows Server-Rollen, mit dem ihre jeweiligen Nutzungs Dieses UAL-Framework ist eine grundlegende und wichtige Komponente der größeren Lizenzierungs Verwaltungs Lösung.

</dd> <dt>

[Windows-Bereitstellungs Dienste](/windows/desktop/Wds/windows-deployment-services-portal)
</dt> <dd>

Die Windows-Bereitstellungs Dienste (Windows Deployment Services, WDS) sind die überarbeitete Version der Remoteinstallations Dienste WDS ermöglicht die Bereitstellung von Windows-Betriebssystemen. Sie können WDS verwenden, um neue Clients mit einer netzwerkbasierten Installation einzurichten, ohne dass Administratoren jeden Computer besuchen oder direkt von CD-oder DVD-Medien installieren müssen.

</dd> <dt>

[Windows-Original-Vorteil-API](/previous-versions/windows/desktop/wingen/windows-genuine-advantage-api-portal)
</dt> <dd>

Mithilfe der Windows authentiadvantage-API können Sie feststellen, ob das Windows-Betriebssystem, das auf dem aktuellen System ausgeführt wird, eine echte Kopie ist.

</dd> <dt>

[Windows-Verwaltungsinstrumentation](/windows/desktop/WmiSdk/wmi-start-page)
</dt> <dd>

Windows-Verwaltungsinstrumentation (WMI) ist die Infrastruktur für Verwaltungsdaten und Vorgänge auf Windows-basierten Betriebssystemen. Sie können WMI-Skripts oder-Anwendungen schreiben, um administrative Aufgaben auf Remote Computern zu automatisieren.

</dd> <dt>

[Windows Defender WMIv2-APIs](/previous-versions/windows/desktop/defender/windows-defender-wmiv2-apis-portal)
</dt> <dd>

Mithilfe von Windows Defender WMI-APIs können Sie den Malware Schutz mithilfe von Skripts oder Anwendungen verwalten.

</dd> <dt>

[Windows PowerShell](https://msdn.microsoft.com/library/Dd835506(v=VS.85).aspx)
</dt> <dd>

Windows PowerShell ist eine aufgabenbasierte Befehlszeilenshell und Skriptsprache, die speziell für die Systemverwaltung entwickelt wurde. Windows PowerShell basiert auf .NET Framework und unterstützt IT-Experten und erfahrene Benutzer beim Steuern und Automatisieren der Verwaltung von Windows-Betriebssystemen sowie von Anwendungen, die unter Windows ausgeführt werden.

</dd> <dt>

[Windows-Remoteverwaltung](/windows/desktop/WinRM/portal)
</dt> <dd>

Der Windows-Remoteverwaltung (WinRM) ist die Microsoft-Implementierung des WS-Management-Protokolls, ein Standardmäßiges SOAP (Simple Object Access Protocol)-basiertes, firewallfreundliches Protokoll, das die Interoperabilität von Hardware und Betriebssystemen von unterschiedlichen Anbietern ermöglicht. Die WS-Management-Protokollspezifikation bietet eine gängige Möglichkeit für Systeme, auf Verwaltungsinformationen in einer IT-Infrastruktur zuzugreifen und diese auszutauschen.

</dd> <dt>

[Windows-Ressourcenschutz](/windows/desktop/Wfp/windows-resource-protection-portal)
</dt> <dd>

Windows-Ressourcenschutz (WRP) verhindert, dass wichtige Systemdateien, Ordner und Registrierungsschlüssel ersetzt werden, die als Teil des Betriebssystems installiert werden.

</dd> <dt>

[Windows Server Update Services](windows-server-update-services.md)
</dt> <dd>

System Administratoren können die WSUS-API (Windows Server Update Services) verwenden, um zu bestimmen, welche Updates für einen Computer oder eine Gruppe von Computern gelten, welche Updates heruntergeladen werden und mit geringem oder ohne Benutzereingriff installiert werden.

Unabhängige Softwarehersteller und Entwickler von Endbenutzern können WSUS-Features in die Computerverwaltungs-oder Update Verwaltungssoftware integrieren, um eine nahtlose Betriebsumgebung bereitzustellen.

</dd> <dt>

[Windows-System Bewertungs Tool](/windows/desktop/WinSAT/windows-system-assessment-tool-portal)
</dt> <dd>

Verwenden von WinSAT zum Bewerten der Leistungsmerkmale und-Funktionen eines Computers.

</dd> <dt>

[Arbeitsordner](/previous-versions/windows/desktop/syncshareservermgmt/sync-share-server-management-portal)
</dt> <dd>

Mithilfe von Arbeits Ordnern können Benutzer Arbeitsdateien auf PCs und Geräten speichern und auf diese zugreifen.

</dd> </dl>
