# <a name="windows-windows-and-messages"></a>Windows (Fenster und Meldungen)

In einer grafischen Windows-basierten Anwendung ist ein Fenster ein rechteckiger Bereich des Bildschirms, in dem die Anwendung die Ausgabe anzeigt und Eingaben vom Benutzer empfängt. Daher besteht eine der ersten Aufgaben einer grafischen Windows-basierten Anwendung darin, ein Fenster zu erstellen.

Ein Fenster teilt den Bildschirm mit anderen Fenstern, einschließlich der Fenster aus anderen Anwendungen. Nur jeweils ein Fenster kann Eingaben vom Benutzer empfangen. Der Benutzer kann die Maus, die Tastatur oder ein anderes Eingabegerät verwenden, um mit diesem Fenster und der Anwendung zu interagieren, die es besitzt.

| [**WM \_ NCDESTROY**](wm-ncdestroy.md)                           | Informiert ein Fenster darüber, dass sein Nicht-Clientbereich zerstört wird. Die [**DestroyWindow-Funktion**](/windows/win32/api/winuser/nf-winuser-destroywindow) sendet die [**WM \_ NCDESTROY-Nachricht**](wm-ncdestroy.md) an das Fenster nach der [**WM \_ DESTROY-Nachricht.**](wm-destroy.md) **WM \_ DESTROY** wird verwendet, um das zugeordnete Speicherobjekt frei zu geben, das dem Fenster zugeordnet ist. <br/> Die [**WM \_ NCDESTROY-Nachricht**](wm-ncdestroy.md) wird gesendet, nachdem die untergeordneten Fenster zerstört wurden. Im Gegensatz dazu wird [**WM \_ DESTROY**](wm-destroy.md) gesendet, bevor die untergeordneten Fenster zerstört werden.<br/>                                                                                      |
| [**WM \_ NULL**](wm-null.md)                                     | Führt keinen Vorgang aus. Eine Anwendung sendet die [**WM \_ NULL-Nachricht,**](wm-null.md) wenn sie eine Nachricht veröffentlichen möchte, die das Empfängerfenster ignoriert.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| [**WM \_ PARENTNOTIFY**](/previous-versions//ms632638(v=vs.85))                  | Wird an das übergeordnete Element eines untergeordneten Fensters gesendet, wenn das untergeordnete Fenster erstellt oder zerstört wird oder wenn der Benutzer auf eine Maustaste klickt, während sich der Cursor über dem untergeordneten Fenster befindet. Wenn das untergeordnete Fenster erstellt wird, sendet das System [**WM \_ PARENTNOTIFY**](/previous-versions//ms632638(v=vs.85)) direkt vor der [**Funktion CreateWindow**](/windows/win32/api/winuser/nf-winuser-createwindowa) oder [**CreateWindowEx,**](/windows/win32/api/winuser/nf-winuser-createwindowexa) die das Fenster erstellt. Das Fenster wird vom Benutzer gezogen, aber es ist kein Symbol für seine Klasse definiert. Eine Anwendung kann ein Handle an ein Symbol oder einen Cursor zurückgeben. Das System zeigt diesen Cursor oder dieses Symbol an, während der Benutzer das Symbol zieht.<br/>                                                                                                                                                                                                                                                                                                                                                                            |
| [**WM \_ QUERYOPEN**](wm-queryopen.md)                           | Wird an ein Symbol gesendet, wenn der Benutzer anfordert, dass das Fenster in seiner vorherigen Größe und Position wiederhergestellt wird.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |

| [**\_WM-GRÖßE**](wm-size.md)                                     | Wird an ein Fenster gesendet, nachdem seine Größe geändert wurde.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| [**GRÖßE DER WM \_**](wm-sizing.md)                                 | Wird an ein Fenster gesendet, in dem die Größe des Benutzers geändert wird. Durch die Verarbeitung dieser Nachricht kann eine Anwendung die Größe und Position des Ziehrechtecks überwachen und bei Bedarf die Größe oder Position ändern. <br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| [**WM \_ STYLECHANGED**](wm-stylechanged.md)                     | Wechseln Sie zu einem Fenster, nachdem die [**SetWindowLong-Funktion**](/windows/win32/api/winuser/nf-winuser-setwindowlonga) einen oder mehrere Stile des Fensters geändert hat.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| [**WM \_ STYLECHANGING**](wm-stylechanging.md)                   | Wird an ein Fenster gesendet, wenn die [**SetWindowLong-Funktion**](/windows/win32/api/winuser/nf-winuser-setwindowlonga) einen oder mehrere Stile des Fensters ändern soll.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| [**WM \_ THEMECHANGED**](wm-themechanged.md)                     | Wird nach einem Designänderungsereignis an jedes Fenster gesendet. Beispiele für Designänderungsereignisse sind die Aktivierung eines Designs, die Deaktivierung eines Designs oder ein Übergang von einem Design zu einem anderen.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| [**WM \_ USERCHANGED**](wm-userchanged.md)                       | Wird an alle Fenster gesendet, nachdem sich der Benutzer angemeldet oder deaktiviert hat. Wenn sich der Benutzer anmeldet oder deaktiviert, aktualisiert das System die benutzerspezifischen Einstellungen. Das System sendet diese Nachricht sofort nach dem Aktualisieren der Einstellungen.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                   |

| [**\_WM-FENSTERPOSCHANGING**](wm-windowposchanging.md)           | Wird an ein Fenster gesendet, dessen Größe, Position oder Position in der Z-Reihenfolge aufgrund eines Aufrufs der [**SetWindowPos-Funktion**](/windows/win32/api/winuser/nf-winuser-setwindowpos) oder einer anderen Fensterverwaltungsfunktion geändert werden soll.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                            |
