# <a name="data-proxy-event-flow"></a>Ereignis Fluss des Daten Proxys

In diesem Thema werden die Details von Ereignissen bei der Verwendung der Daten Proxy Features für die frei Hand Analyse erläutert.

## <a name="data-proxy-event-flow-overview"></a>Übersicht über den Daten Proxy-Ereignis Fluss

In der Daten Proxy Verwendung von [**InkAnalyzer**](inkanalyzer.md)wird davon ausgegangen, dass die Anwendung, die den InkAnalyzer integriert, bereits über ein vorhandenes Dokument Modell verfügt, in dem Sie die Ergebnisse der Analyse als Proxy verwenden möchten. Außerdem wird davon ausgegangen, dass die Anwendung Ergebnisse aus einem vorherigen Analyse Vorgang enthält, den Sie erstellen möchten, wenn Sie in Ihrem Dokument Modell gespeichert werden. Möglicherweise gibt es auch nicht-Ink-Kontext, der dem **InkAnalyzer** in Form eines **ImageNode** -oder **TextWordNode**-[**ContextNode**](icontextnode.md) hinzugefügt werden kann, um potenziell mit frei Hand Eingaben versehen zu werden.

Der Schlüssel für das Daten Proxy System besteht darin, dass die Anwendung den [**ContextNode**](icontextnode.md) als "teilweise aufgefüllt" kennzeichnen muss, indem " [**partiallyaufgefüllt**](icontextnode-setpartiallypopulated.md)" verwendet wird. Wenn dieses Flag true ist, geht der [**InkAnalyzer**](inkanalyzer.md) davon aus, dass es drei Dinge zu diesem **ContextNode** gibt:

-   Der Speicherort von [**ContextNode**](icontextnode.md) ist richtig.
-   Der Typ von [**ContextNode**](icontextnode.md) ist richtig.
-   Alle anderen Informationen zu diesem [**ContextNode**](icontextnode.md) werden an anderer Stelle gespeichert.

Basierend auf diesen drei Regeln, wenn der [**InkAnalyzer**](inkanalyzer.md) zusätzliche Informationen zu einem [**ContextNode**](icontextnode.md) benötigt, wird das Ereignis [**PopulateContextNode**](-ianalysisproxyevents-populatecontextnode.md) ausgegeben, und es wird auf den fraglichen **ContextNode** verwiesen. Dadurch erhält die Anwendung die Möglichkeit, alle bekannten Informationen auf diesem **ContextNode** festzulegen, bevor Sie von **InkAnalyzer** ausführlicher betrachtet werden. Nach der Behandlung eines auffüllungs Ereignis auffüllungs-Ereignisses muss der fragliche **ContextNode** über eine gültige Location-Eigenschaft verfügen, die richtige Anzahl von untergeordneten Knoten, die festgelegt ist, wenn es sich **um einen Container** **ContextNode** handelt oder die richtigen Striche festgelegt sind, mithilfe von [**setstrokes**](icontextnode-setstrokes.md), wenn es sich um ein frei Hand Blatt **ContextNode** handelt Fehler beim ordnungsgemäßen Festlegen dieses Speicher Orts und der untergeordneten Knoten-oder Strich Informationen führen zu einer **InvalidOperation** -Ausnahme. Die untergeordneten Knoten für einen Container " **ContextNode** " können selbst als teilweise aufgefüllt festgelegt werden. in diesem Fall werden weitere " **PopulateContextNode** "-Ereignisse ausgelöst, wenn der **InkAnalyzer** feststellt, dass Sie für den aktuellen Analyse Vorgang benötigt werden.

In den folgenden Tabellen wird beschrieben, wann das Ereignis " [**PopulateContextNode**](-ianalysisproxyevents-populatecontextnode.md) " während der Verwendung von " [**InkAnalyzer**](inkanalyzer.md)" ausgelöst wird.

Nachdem der [**InkAnalyzer**](inkanalyzer.md) einige restuls berechnet hat, wird die Anwendung zurück gesucht, um die Ergebnisse zu aktualisieren. Das erste Ereignis, das ausgelöst wird, ist das [**InkAnalyzerStateChanging**](-ianalysisproxyevents-inkanalyzerstatechanging.md) -Ereignis. Dieses Ereignis deutet einfach auf die Anwendung hin, die der Struktur Zustand des **InkAnalyzer** -Objekts ändert. Dadurch haben Anwendungen die Möglichkeit, das Flag " [**partiallyaufgefüllt**](icontextnode-setpartiallypopulated.md) " für alle [**ContextNodes**](icontextnodes.md) festzulegen, deren Zustand an anderer Stelle gespeichert ist. Der **InkAnalyzer** führt dann eine Reihe von [**auffüllungsecontextnode**](-ianalysisproxyevents-populatecontextnode.md) -Ereignissen aus, um den aktuellen Zustand der Daten zu deaktivieren. Sobald dies determiniert ist, wird ein Abstimmungs Vorgang komprimiert, um zu bestimmen, welche Hintergrund Ergebnisse noch angewendet werden können.

Um Ergebnisse auf den [**InkAnalyzer**](inkanalyzer.md) anzuwenden, werden eine Reihe von Struktur Änderungs Ereignissen ausgelöst. Die Struktur Änderungs Ereignisse beschreiben schrittweise alle Änderungen, die zum Aktualisieren der Ergebnisse erforderlich sind. Diese Ereignisse sind für die Verarbeitung in der Durchsicht ohne Interoperabilität oder Abbruch vorgesehen. Wenn der Analyse Vorgang (über die [**Abort**](iinkanalyzer-abort.md) -Methode) während der Verarbeitung der Struktur Änderungs Ereignisse abgebrochen wird, ist der Status des **inkanalyzers** ungültig, und das gesamte Dokument muss möglicherweise erneut analysiert werden.

In den folgenden Tabellen wird beschrieben, wann die Struktur Änderungs Ereignisse während der Verwendung von InkAnalyzer ausgelöst werden. Die Tabellen beziehen sich auf die nach dem Ereignis Flussdiagramm gezeigten Zeitstempel.
