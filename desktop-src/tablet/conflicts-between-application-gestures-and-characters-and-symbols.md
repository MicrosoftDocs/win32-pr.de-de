# <a name="conflicts-between-application-gestures-and-characters-and-symbols"></a>Konflikte zwischen Anwendungs Gesten und Zeichen und Symbolen

Einige Anwendungs Gesten können mit Zeichen, Zeichenkombinationen, Symbolen, Zeichenkombinationen und Symbolen oder ganzen Wörtern in einem einzelnen Strich in Konflikt stehen. Die meisten dieser Konflikte lassen sich vermeiden, indem Sie einen detaillierten Einblick in den Kontext haben, in dem die Anwendungs Geste ausgeführt wird.

Um z. b. eine Rechte Geste von einem Bindestrich (-) oder einem Unterstrich () zu unterscheiden \_ , kann eine Anwendung filtern, wie die Bewegung für benachbarte Zeichen, die relative Größe im Vergleich zu Zeichen oder andere in einem Paket enthaltene Informationen ist. Die zeitliche Steuerung der Anwendungs Geste kann auch ein entscheidender Faktor bei der Entscheidung zwischen Gesten und Zeichen sein. Vor dem Anwenden einer Anwendungs Bewegung, die vor dem Einfügen von Zeichen steht, ist möglicherweise mehr Pause vorhanden. Wenn ein Benutzer eine Reihe von Rückgängig-oder rückgängigbewegungen ausführt, kann es sein, dass nach einer Bewegung auch mehr eine Pause als ein Zeichen besteht.
