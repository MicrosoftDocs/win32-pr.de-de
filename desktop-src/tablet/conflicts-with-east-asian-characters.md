# <a name="conflicts-with-east-asian-characters"></a>Konflikte mit East-Asian Zeichen

Einige *Anwendungs Gesten* können mit ostasiatischen Zeichen oder Kombinationen aus ostasiatischen Zeichen in Konflikt stehen. In der folgenden Tabelle werden mögliche Konflikte zwischen Anwendungs Gesten und Zeichen in Anwendungen aufgelistet, die eine bestimmte ostasiatische Zeicheneingabe aufweisen.



| Geste                 | Chinesisch (vereinfacht) | Chinesisch (traditionell) | Japanisch     | Koreanisch       |
|-------------------------|----------------------|-----------------------|--------------|--------------|
| Nach unten<br/>         | X<br/>         | X<br/>          | X<br/> | X<br/> |
| Rechts<br/>        | X<br/>         | X<br/>          | X<br/> | X<br/> |
| DownRight<br/>    | -<br/>         | X<br/>          | -<br/> | X<br/> |
| Rechts unten<br/>    | -<br/>         | -<br/>          | X<br/> | X<br/> |
| Circle<br/>       | -<br/>         | -<br/>          | -<br/> | X<br/> |
| ChevronRight<br/> | -<br/>         | -<br/>          | -<br/> | X<br/> |



 

Microsoft ist weiterhin bestrebt, *Gesten* zu entwickeln. Microsoft erkennt beim Entwickeln von Anwendungen, dass unabhängige Softwareanbieter (Independent Softwareanbieters, ISVs) wissen müssen, welche Aktionen Gesten zugeordnet werden. [Nicht implementierte Symbole](unimplemented-glyphs.md) listet einen Satz von Stift Aktionen auf, die Microsoft plant, Gesten zuzuordnen, sowie Gesten, die für Aktionen reserviert werden. Diese Informationen werden bereitgestellt, damit Sie wissen, welche Aktionen und Gesten in Zukunft bereitgestellt werden.

Zusätzlich zur Verwendung von Gesten, die in Microsoft Windows XP Tablet PC Edition bereitgestellt werden, können Anwendungen ihre eigenen Gesten erstellen. Diese Gesten können dann von einer *Microsoft Gestenerkennung* erkannt werden, die der Anwendungsentwickler erstellt. Wenn Sie Ihre eigenen Gesten implementieren möchten, sollten Sie sich [nicht implementierte Symbole](unimplemented-glyphs.md) ansehen, um zu vermeiden, dass sich mit Gesten, die Microsoft implementieren soll, überlappen.
