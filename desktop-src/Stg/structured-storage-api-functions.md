# <a name="structured-storage-api-functions"></a>Strukturierte Speicher-API-Funktionen

Einige der API-Funktionen für die strukturierte Speicherung sind Hilfsfunktionen, die eine Sequenz von Aufrufen anderer API-Funktionen und Schnittstellen Methoden ausführen. Diese Hilfsfunktionen können als kurze Kürzungen verwendet werden.

Andere API-Funktionen ermöglichen den Zugriff auf die com-Implementierung strukturierter Speicher, Verbund Dateien.

Ein weiterer Satz von API-Funktionen ermöglicht es Ihnen, OLE 1-Objekte in strukturierte Speicher zu konvertieren. Sie können diese Funktionen verwenden, um zu bestimmen, ob eine Objektklasse von OLE 1 ist, und um Objekte zwischen OLE 1 und aktuellen com-Speicher Formaten zu konvertieren.

Schließlich gibt es API-Funktionen zum umrechnen und emuinieren anderer Objekte. Diese Funktionen stellen Dienste für eine Serveranwendung bereit, um mit Daten aus einer anderen Anwendung zu arbeiten. Die Daten können in das systemeigene Format der Serveranwendung konvertiert werden, indem die Daten aus dem ursprünglichen Format gelesen, aber im systemeigenen Format der Objekt Anwendung geschrieben werden. Oder die Daten können im ursprünglichen Format verbleiben, während die Serveranwendung die Daten im ursprünglichen Format liest und schreibt.

Weitere Informationen finden Sie im folgenden Abschnitt [strukturierte Speicherzugriffs Modi](structured-storage-access-modes.md).
