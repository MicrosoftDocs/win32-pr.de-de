# <a name="the-evolution-of-file-systems"></a><span data-ttu-id="830c0-103">Die Entwicklung von Dateisystemen</span><span class="sxs-lookup"><span data-stu-id="830c0-103">The Evolution of File Systems</span></span>

<span data-ttu-id="830c0-104">Bevor Computer für Betriebssysteme entwickelt wurden, wurde jeder Computer erstellt, um eine einzelne, proprietäre Anwendung auszuführen, die vollständige und exklusive Kontrolle über den gesamten Computer hatte.</span><span class="sxs-lookup"><span data-stu-id="830c0-104">Before computers were developed to function on disk operating systems, each computer was built to run a single, proprietary application, which had complete and exclusive control of the entire machine.</span></span> <span data-ttu-id="830c0-105">Die Anwendung schreibt die persistenten Daten direkt auf einen Datenträger oder eine Trommel, indem Befehle direkt an den Datenträger Controller gesendet werden.</span><span class="sxs-lookup"><span data-stu-id="830c0-105">The application would write its persistent data directly to a disk, or drum, by sending commands directly to the disk controller.</span></span> <span data-ttu-id="830c0-106">Die Anwendung ist dafür verantwortlich, die absoluten Speicherorte der Daten auf dem Datenträger zu verwalten und sicherzustellen, dass Sie bereits vorhandene Daten nicht überschrieben haben.</span><span class="sxs-lookup"><span data-stu-id="830c0-106">The application was responsible for managing the absolute locations of data on the disk, making sure that it was not overwriting already-existing data.</span></span> <span data-ttu-id="830c0-107">Da immer nur eine Anwendung auf dem Computer ausgeführt wurde, war diese Aufgabe nicht zu schwierig.</span><span class="sxs-lookup"><span data-stu-id="830c0-107">Since only one application was running on the computer at any time, this task was not too difficult.</span></span>

<span data-ttu-id="830c0-108">Die Einführung von Computersystemen, die mehrere Anwendungen ausführen konnten, erforderte einen Mechanismus, mit dem sichergestellt werden kann, dass die Anwendungen nicht über die Daten der anderen Anwendungen geschrieben wurden.</span><span class="sxs-lookup"><span data-stu-id="830c0-108">The advent of computer systems that could run more than one application einzelnen Standard für die Unterscheidung von Datenträger Sektoren verwenden, die freigegeben wurden, indem Sie Sie entsprechend markieren.</span><span class="sxs-lookup"><span data-stu-id="830c0-109">Application developers addressed this problem by adopting a single standard for distinguishing disk sectors in use from those that were free by marking them accordingly.</span></span> <span data-ttu-id="830c0-110">In der Zeit wurden diese Standards zu einem Betriebssystem zusammengestellt, das verschiedene Dienste für die Anwendungen bereitstellte, einschließlich eines Dateisystems für die Verwaltung von persistenten Speicher.</span><span class="sxs-lookup"><span data-stu-id="830c0-110">In time, these standards coalesced to become a disk operating system, which provided various services to the applications, including a file system for managing persistent storage.</span></span> <span data-ttu-id="830c0-111">Bei der Einführung eines Dateisystems mussten Anwendungen nicht mehr direkt mit dem physischen Speichermedium umgehen.</span><span class="sxs-lookup"><span data-stu-id="830c0-111">With the advent of a file system, <span data-ttu-id="830c0-115">Das Dateisystem hat eine einzelne Dereferenzierungsebene zwischen Anwendungen und dem Datenträger bereitgestellt, und das Ergebnis war, dass jede Anwendung eine Datei als einzelner zusammenhängender Stream von Bytes auf dem Datenträger sah, obwohl das Dateisystem tatsächlich die Datei in separaten Sektoren gespeichert hat.</span><span class="sxs-lookup"><span data-stu-id="830c0-115">The file system provided a single level of indirection between applications and the disk, and the result was that every application saw a file as a single contiguous stream of bytes on the disk even though the file system was actually storing the file in discontiguous sectors.</span></span> <span data-ttu-id="830c0-116">Die Dereferenzierung gab an, dass die Anwendungen die absolute Position von Daten auf einem Speichergerät nicht verfolgen mussten.</span><span class="sxs-lookup"><span data-stu-id="830c0-116">The indirection freed the applications from having to track the absolute position of data on a storage device.</span></span>

<span data-ttu-id="830c0-117">Heute bieten praktisch alle System-APIs für die Dateieingabe und-Ausgabe Anwendungen zum Schreiben von Informationen in eine Flatfile.</span><span class="sxs-lookup"><span data-stu-id="830c0-117">Today, virtually all system APIs for file input and output provide applications for writing information into a flat file.</span></span> <span data-ttu-id="830c0-118">Anwendungen sehen diese Datei als einen einzelnen Datenstrom, der so groß wie erforderlich vergrößert werden kann, bis der Datenträger voll ist.</span><span class="sxs-lookup"><span data-stu-id="830c0-118">Applications see this file as a single stream of bytes that can grow as large as necessary until the disk is full.</span></span> <span data-ttu-id="830c0-119">Lange Zeit waren diese APIs für Anwendungen ausreichend, um Ihre permanenten Informationen zu speichern.</span><span class="sxs-lookup"><span data-stu-id="830c0-119">For a long time these APIs have been sufficient for applications to store their persistent information.</span></span> <span data-ttu-id="830c0-120">Anwendungen haben bedeutende Neuerungen in Bezug auf den Umgang mit einem einzelnen Informationsdaten Strom hergestellt, um Features wie inkrementelle "schnelle" Speicherungen bereitzustellen.</span><span class="sxs-lookup"><span data-stu-id="830c0-120">Applications have made significant innovations in how they deal with a single stream of information to provide features like incremental "fast" saves.</span></span>

<span data-ttu-id="830c0-121">In einer Welt von Komponenten Objekten ist das Speichern von Daten in einer einzelnen Flatfile jedoch nicht mehr effizient.</span><span class="sxs-lookup"><span data-stu-id="830c0-121">However, in a world of component objects, storing data in a single flat file is no longer efficient.</span></span> <span data-ttu-id="830c0-122">Ebenso wie es bei Dateisystemen nicht mehr erforderlich ist, dass mehrere Anwendungen dasselbe Speichermedium gemeinsam verwenden, benötigen Komponenten Objekte jetzt ein System, das es Ihnen ermöglicht, Speicher innerhalb des konzeptionellen Frameworks einer einzelnen Datei freizugeben.</span><span class="sxs-lookup"><span data-stu-id="830c0-122">Just as file systems arose out of the need for multiple applications to share the same storage medium, so, now, do component objects require a system that allows them to share storage within the conceptual framework of a single file.</span></span> <span data-ttu-id="830c0-123">Obwohl es möglich ist, die separaten Objekte mithilfe von herkömmlichem Flatfile-Speicher zu speichern, wenn sich eines der Objekte vergrößert, oder wenn Sie einfach ein anderes Objekt hinzufügen, ist es erforderlich, die gesamte Datei in den Arbeitsspeicher zu laden, das neue Objekt einzufügen und die gesamte Datei zu speichern.</span><span class="sxs-lookup"><span data-stu-id="830c0-123">Even though it is possible to store the separate objects using conventional flat-file storage, if one of the objects increases in size, or if you simply add another object, it becomes necessary to load the entire file into memory, insert the new object, and then save the whole file.</span></span> <span data-ttu-id="830c0-124">Dieser Prozess kann sehr zeitaufwändig sein.</span><span class="sxs-lookup"><span data-stu-id="830c0-124">This process can be extremely time-consuming.</span></span>
