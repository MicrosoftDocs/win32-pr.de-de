# <a name="storage-creation-flags"></a>Speichererstellungsflags

Speicherstellungsflags geben an, was com tun soll, wenn ein vorhandenes Speicher-, Datenstrom-oder LockBytes-Objekt den gleichen Namen wie ein neues Speicher-oder Streamobjekt hat, das Sie erstellen. Standardmäßig wird eine Fehlermeldung zurückgegeben, und es wird nicht das neue-Objekt erstellt. Sie können nur eines dieser Flags in einem angegebenen Erstellungs Befehl verwenden.
