# <a name="about-structured-storage"></a>Informationen zu strukturierten Storage

Herkömmliche Dateisysteme stoßen auf Herausforderungen, wenn sie versuchen, mehrere Arten von Objekten effizient in einem Dokument zu speichern. COM stellt eine Lösung bereit: ein Dateisystem innerhalb einer Datei. Mit strukturiertem COM-Speicher wird definiert, wie eine einzelne Dateientität als strukturierte Sammlung von zwei Objekttypen (Speicher und Datenströme) behandelt wird, die wie Verzeichnisse und Dateien ausgeführt werden. Dieses Schema wird als *strukturierter Speicher* bezeichnet. Der Zweck des strukturierten Speichers besteht darin, die Leistungseinbußen und den Mehraufwand im Zusammenhang mit der Speicherung separater Objekte in einer Flatfile zu reduzieren.

Dieser Abschnitt enthält eine Übersicht über die Vorteile und Grundlagen des strukturierten Speichers, Eigenschaftensätze und asynchronen strukturierten Speicher sowie einen historischen Blick auf Dateisysteme:

-   Verbunddateiimplementierungen
-   NTFS-Dateisystemimplementierung
-   Eigenständige Implementierung
