# <a name="about-structured-storage"></a>Informationen über strukturierte Speicherung

Herkömmliche Dateisysteme stoßen auf Herausforderungen, wenn Sie versuchen, in einem Dokument effizient mehrere Arten von Objekten zu speichern. COM bietet eine Lösung: ein Dateisystem in einer Datei. Der strukturierte com-Speicher definiert, wie eine einzelne Datei Entität als strukturierte Auflistung von zwei Typen von Objekten behandelt wird – Storages und Streams –, die wie Verzeichnisse und Dateien funktionieren. Dieses Schema wird als *strukturierter Speicher* bezeichnet. Der Zweck der strukturierten Speicherung besteht darin, die Leistungseinbußen und den Aufwand für das Speichern von separaten Objekten in einer Flatfile zu verringern.

Dieser Abschnitt enthält eine Übersicht über die Vorteile und Grundlagen der strukturierten Speicherung, die Eigenschaften Sätze und die asynchrone strukturierte Speicherung sowie eine historische Betrachtung der Dateisysteme:

-   Implementierung der Verbund Datei
-   NTFS-Dateisystem Implementierung
-   Eigenständige Implementierung

Dieser Abschnitt enthält Links zur [Entwicklung von Dateisystemen](the-evolution-of-file-systems.md), [Storages und Streams](storages-and-streams.md), [Verbund Dateien](compound-files.md), [Grundlagen strukturierter Speicherung](structured-storage-fundamentals.md)und [strukturierten Speicher Beispielen](using-structured-storage.md).
