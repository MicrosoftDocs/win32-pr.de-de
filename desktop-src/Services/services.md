# <a name="services-services"></a>Dienste (Dienste)

Eine- *Dienst Anwendung* entspricht den Schnittstellen Regeln des Dienststeuerungs-Managers (SCM). Sie kann automatisch beim Systemstart, von einem Benutzer über das System Steuerungs Applet der Dienste oder durch eine Anwendung gestartet werden, die die [Dienstfunktionen](service-functions.md)verwendet. Dienste können auch dann ausgeführt werden, wenn kein Benutzer am System angemeldet ist.

Ein *Treiber Dienst* entspricht den Gerätetreiber Protokollen. Sie ähnelt einer-Dienst Anwendung, aber Sie interagiert nicht mit dem SCM. Der Einfachheit halber bezieht sich der Begriff *Dienst* auf eine *Dienst Anwendung* in dieser Übersicht.

Trigger können jetzt zum Steuern des Dienst Starts verwendet werden. Weitere Informationen finden Sie unter [Dienst Konfiguration](service-configuration.md).
