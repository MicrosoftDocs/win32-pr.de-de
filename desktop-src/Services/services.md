# <a name="services-services"></a>Dienste (Dienste)

Eine *Dienstanwendung* entspricht den Schnittstellenregeln des Dienststeuerungs-Managers (Service Control Manager, SCM). Sie kann automatisch beim Systemstart, von einem Benutzer über das Systemsteuerungs-Applet Dienste oder von einer Anwendung gestartet werden, die die [Dienstfunktionen](service-functions.md)verwendet. Dienste können auch dann ausgeführt werden, wenn kein Benutzer am System angemeldet ist.

Ein *Treiberdienst* entspricht den Gerätetreiberprotokollen. Sie ähnelt einer Dienstanwendung, interagiert jedoch nicht mit dem SCM. Der Einfachheit halber bezieht sich der Begriff *Dienst* in dieser Übersicht auf eine *Dienstanwendung.*

Trigger können jetzt verwendet werden, um den Dienststart zu steuern. Weitere Informationen finden Sie unter [Dienstkonfiguration.](service-configuration.md)
