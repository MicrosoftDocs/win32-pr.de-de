<span data-ttu-id="42f87-108">Sie können IIS so konfigurieren, dass die Kommunikation zwischen dem XML-Webdienst und-Clients mithilfe von SSL-oder TLS-Protokollen für öffentliche Schlüssel verschlüsselt wird.</span><span class="sxs-lookup"><span data-stu-id="42f87-108">You can configure IIS to encrypt your XML web service communications with clients by using public key encryption SSL or TLS protocols.</span></span> <span data-ttu-id="42f87-109">Wenn Sie die SOAP-Kommunikation nicht verschlüsseln, können die zwischen einem Client und einem Server ausgetauschten Daten von einem Drittanbieter mit Zugriff auf ein beliebiges Netzwerk, über das die SOAP-Kommunikation übertragen wird, beobachtet werden. abhängig von der Netzwerktopologie kann es sich hierbei um ein kleines LAN oder um das Internet handeln.</span><span class="sxs-lookup"><span data-stu-id="42f87-109">If you do not encrypt SOAP communications, data exchanged between a client and server could be observed by a third party with access to any network over which the SOAP communication travels; depending on network topology, this might be a small LAN or it might be the Internet.</span></span>

<span data-ttu-id="42f87-110">Standardmäßig werden unverschlüsselte SOAP-Kommunikationen am HTTP-Port (80) empfangen, und verschlüsselte SOAP-Kommunikationen werden am HTTPS-Port (443) empfangen.</span><span class="sxs-lookup"><span data-stu-id="42f87-110">By default, unencrypted SOAP communications are received at the HTTP port (80) and encrypted SOAP communications are received at the HTTPS port (443).</span></span> <span data-ttu-id="42f87-111">Damit ein Client erfolgreich auf einen XML-Webdienst zugreifen kann, müssen alle Firewalls zwischen dem Client und dem Server so konfiguriert werden, dass TCP-SYN-Pakete den entsprechenden Serverport erreichen können.</span><span class="sxs-lookup"><span data-stu-id="42f87-111">For a client to successfully access an XML web service, any firewalls between the client and the server must be configured to allow TCP SYN packets to reach the appropriate server port.</span></span> <span data-ttu-id="42f87-112">Umgekehrt kann ein Firewalladministrator diese Ports schließen, um den Zugriff auf XML-Webdienste einzuschränken.</span><span class="sxs-lookup"><span data-stu-id="42f87-112">Conversely, to limit access to XML web services, a firewall administrator may choose to close these ports.</span></span>

<span data-ttu-id="42f87-113">Die Standard Sicherheitseinstellungen für eine COM-Komponente, die als XML-Webdienst verfügbar gemacht wird, unterscheiden sich abhängig davon, welche Version von Microsoft .NET Framework installiert ist.</span><span class="sxs-lookup"><span data-stu-id="42f87-113">The default security settings for a COM component exposed as an XML web service differ depending on which version of the Microsoft .NET Framework is installed.</span></span> <span data-ttu-id="42f87-114">Wenn Version 1,0 installiert ist, sind XML-Webdienste standardmäßig nicht sicher. alle Aufrufe werden akzeptiert, und es wird keine Verschlüsselung verwendet.</span><span class="sxs-lookup"><span data-stu-id="42f87-114">If version 1.0 is installed, XML web services are nonsecure by default; all calls are accepted and no encryption is used.</span></span> <span data-ttu-id="42f87-115">Wenn Version 1,1 oder höher installiert ist, sind XML-Webdienste standardmäßig sicher. <span data-ttu-id="42f87-116">Ein geschützter XML-Webdienst unterstützt den WKO-Zugriff über WSDL nicht.</span><span class="sxs-lookup"><span data-stu-id="42f87-116">A secured XML web service does not support WKO access via WSDL.</span></span> <span data-ttu-id="42f87-117">Stattdessen können Clients, auf denen die .NET Framework Version 1,1 installiert ist, im Modus "Cao" aufgerufen werden.</span><span class="sxs-lookup"><span data-stu-id="42f87-117">Instead, clients that have installed the .NET Framework version 1.1 can call it in CAO mode.</span></span> <span data-ttu-id="42f87-118">Wenn Clients von Drittanbietern über WSDL auf Ihren XML-Webdienst zugreifen müssen, müssen Sie den anonymen Zugriff zulassen.</span><span class="sxs-lookup"><span data-stu-id="42f87-118">If third-party clients need to access your XML web service via WSDL, you must allow anonymous access.</span></span>

<span data-ttu-id="42f87-119">Eine COM-Komponente, die als XML-Webdienst verfügbar gemacht wird, wird standardmäßig mit den Berechtigungen des anonymen Benutzers (nicht mit den Berechtigungen des Aufrufers) ausgeführt.</span><span class="sxs-lookup"><span data-stu-id="42f87-119">A COM component exposed as an XML web service runs by default with the permissions of the anonymous user (not with the permissions of its caller).</span></span> <span data-ttu-id="42f87-120">Sie können IIS so konfigurieren, dass der XML-Webdienst als anderer Benutzer ausgeführt wird.
