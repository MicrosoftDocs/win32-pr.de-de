<span data-ttu-id="86fff-182">Ein wichtiger Entwurfspunkt ist die einfache Verwendung.</span><span class="sxs-lookup"><span data-stu-id="86fff-182">One important design point is ease of use.</span></span> <span data-ttu-id="86fff-183">In dem Moment, in dem Sie ADO-Recordsets verwenden, um einen strukturierten Satz von Details an die Präsentationsebene zurückzuleiten, haben Sie ein Komplexitäts Problem eingeführt.</span><span class="sxs-lookup"><span data-stu-id="86fff-183">The moment you decided to use ADO recordsets to pass a structured set of details back to the presentation tier, you introduced a complexity problem.</span></span> <span data-ttu-id="86fff-184">Programmierer der Präsentationsebene müssen die Struktur dieser Recordsets verstehen.</span><span class="sxs-lookup"><span data-stu-id="86fff-184">Presentation tier programmers need to understand the structure of those recordsets.</span></span> <span data-ttu-id="86fff-185">Ein größeres Problem kann auftreten, wenn Sie mehrere Details für einen Datensatz benötigen, der in einem Recordset übermittelt werden soll.</span><span class="sxs-lookup"><span data-stu-id="86fff-185">A greater complexity issue can arise when you require multiple details for a set of data to be passed in a recordset.</span></span> <span data-ttu-id="86fff-186">Um die Dinge zu vereinfachen, sollten Sie eine Recordset Factory-Methode erstellen, wenn Sie beabsichtigen, Daten in strukturierten ADO-Recordsets zu übernehmen.</span><span class="sxs-lookup"><span data-stu-id="86fff-186">To simplify matters, you should consider creating a recordset factory method whenever you intend to require data to be passed in structured ADO recordsets.</span></span>

<span data-ttu-id="86fff-187">Die recordsetfactory sollte ein leeres, aber beschreibbares, nicht mit dem Aufrufer getrennte Recordset zurückgeben.</span><span class="sxs-lookup"><span data-stu-id="86fff-187">The recordset factory should return an empty but writable disconnected recordset to the caller.</span></span> <span data-ttu-id="86fff-188">Für dieses Recordset sollten die richtigen Felder (Namen und Datentypen) bereits konfiguriert sein, damit der aufrufenden Client nicht wissen muss, wie das Recordset hergestellt werden muss.</span><span class="sxs-lookup"><span data-stu-id="86fff-188">This recordset should have the proper fields (names and data types) already configured so that the calling client does not need to know how to manufacture the recordset.</span></span> <span data-ttu-id="86fff-189">Diese Vorgehensweise wird häufig als *Factorymuster* bezeichnet und ist ein gängiges Lösungsmuster, das das Erstellen eines Objekts einer bestimmten Komplexität vereinfacht, ohne die Feinheiten der Erstellung kennen zu müssen.</span><span class="sxs-lookup"><span data-stu-id="86fff-189">This approach is often referred to as the *factory pattern* and is a common solution pattern that solves the need to create an object of a given complexity without having to know the nuances of creating it.</span></span>

<span data-ttu-id="86fff-190">Einfache Entwurfs Optionen, wie z. b. die Auswahl desselben Parameter namens für die gleichen Daten in jeder Methode, in der diese Informationen übergeben werden, erleichtern es, eine Methode zu betrachten und zu wissen, was erwartet wird.</span><span class="sxs-lookup"><span data-stu-id="86fff-190">Simple design choices such as choosing the same parameter name for the same piece of data across every method where that information is passed makes it easy to look at a method and know what is expected.</span></span> <span data-ttu-id="86fff-191">Sicherzustellen, dass die Reihenfolge, in der like-Argumente übergangen werden, über eine gesamte Methoden Familie hinweg konsistent ist, stellt einen Komfort Punkt dar, der ein konsistentes Modell erzwingt.</span><span class="sxs-lookup"><span data-stu-id="86fff-191">Making sure that the order in which like arguments are passed is consistent across an entire family of methods provides a comfort point that enforces a consistent model.</span></span>
