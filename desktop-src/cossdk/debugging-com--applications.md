# <a name="debugging-com-applications"></a>Debuggen von COM+-Anwendungen

Debugverfahren für COM+-Anwendungen hängen von der Sprache ab, in der Sie ihre Komponente schreiben möchten.

Wenn Sie in Microsoft Visual C++ coden, können Sie den Debugger in C++ starten oder mit einem Remoteclient debuggen, indem Sie die OLE-Remoteprozedursteuerung (REMOTE Procedure Control, RPC) und just-in-time (JIT)-Debuggen verwenden. Sie können die Komponente jederzeit mithilfe des Verwaltungstool Komponentendienste debuggen, indem Sie die Einstellung COM+ **Starten** im Debugger auf der Registerkarte Erweitert des Dialogfelds Eigenschaften der COM+-Anwendung verwenden.   Weitere Informationen zum Debuggen von Komponenten, die in C++ codiert sind, finden Sie unter [Debugging Components Written in Visual C++.](debugging-components-written-in-visual-c--.md)

Sofern Sie derzeit nicht Multithreading, Komponentennachverfolgung, Remoteaufrufe oder Prozessisolation debuggen, können Sie die Microsoft Visual Basic-Umgebung zu Debugzwecken verwenden. [Debugging Components Written in Visual Basic](debugging-components-written-in-visual-basic.md) beschreibt den Debugprozess innerhalb einer Visual Basic Umgebung.

Das Thema [Debuggen in der Visual Basic-IDE](debugging-in-the-visual-basic-ide.md) bietet eine allgemeine Übersicht mit Richtlinien, Tipps und einem Verfahren zum Debuggen in der integrierten Entwicklungsumgebung (IDE).

Weitere Informationen zum Debuggen erweiterter Prozesse finden Sie unter [Debuggen kompilierter Visual Basic Komponenten.](debugging-compiled-visual-basic-components.md)

> [!Note]  
> Mehrere Einschränkungen im Zusammenhang mit der Verwendung Visual Basic-Umgebung zum Debuggen von Komponenten mit MTS wurden für COM+ behoben. Weitere Informationen finden Sie unter [COM+-Visual Basic Debugunterstützung im Gegensatz zu MTS.](com--visual-basic-debugging-support-contrasted-with-mts.md)
