# <a name="com-services-without-components-concepts"></a><span data-ttu-id="524c7-103">Konzepte der com+-Dienste ohne Komponenten</span><span class="sxs-lookup"><span data-stu-id="524c7-103">COM+ Services Without Components Concepts</span></span>

<span data-ttu-id="524c7-104">Mit com+ 1,5 wird die Möglichkeit eingeführt, com+-Dienste ohne Komponenten zu verwenden.</span><span class="sxs-lookup"><span data-stu-id="524c7-104">COM+ 1.5 introduces the ability to use COM+ services without components.</span></span> <span data-ttu-id="524c7-105">Dadurch werden die Leistungskosten erheblich reduziert, wenn com+-Dienste aus einer Umgebung verwendet werden, in der keine Komponenten verwendet werden, und die Komplexität der Verwendung dieser Dienste entfällt.</span><span class="sxs-lookup"><span data-stu-id="524c7-105">This significantly reduces the performance costs when using COM+ services from an environment that doesn't use components, and it also eliminates the complexity of using these services.</span></span> <span data-ttu-id="524c7-106">Ab IIS 6,0 nutzen IIS und ASP die Verwendung von COM+-Diensten ohne Komponenten.</span><span class="sxs-lookup"><span data-stu-id="524c7-106">Beginning with IIS 6.0, IIS and ASP take advantage of using COM+ services without components.</span></span>

<span data-ttu-id="524c7-107">Com+-Dienste wurden ursprünglich für die Verwendung mit COM+-Komponenten entwickelt.</span><span class="sxs-lookup"><span data-stu-id="524c7-107">COM+ services were originally designed to be used with COM+ components.</span></span> <span data-ttu-id="524c7-108">Einige Programmierumgebungen sind jedoch nicht Komponenten basiert und erfordern daher einen beträchtlichen Verwaltungsaufwand für die Verwendung von COM+-Diensten.</span><span class="sxs-lookup"><span data-stu-id="524c7-108">However, some programming environments are not component-based and therefore required substantial overhead to use COM+ services.</span></span> <span data-ttu-id="524c7-109">Vor der Veröffentlichung von com+ 1,5 musste IIS beispielsweise ausschließlich Shim-Objekte erstellen, um com+-Transaktionsdienste in ASP-Seiten verwenden zu können.</span><span class="sxs-lookup"><span data-stu-id="524c7-109">For example, before the release of COM+ 1.5, IIS had to create shim objects solely to be able to use COM+ transaction services in ASP pages.</span></span> <span data-ttu-id="524c7-110">Zu den Leistungseinbußen bei der Erstellung dieser Objekte gehören das Speichern der Konfigurationsdaten in der IIS-Metabasis und der COM+-Registrierungsdatenbank (RegDB) sowie die zusätzliche Kommunikation zwischen der IIS-Metabase und der com+-RegDB, die zur effektiven Verwaltung der Konfigurationsdaten benötigt wird.</span><span class="sxs-lookup"><span data-stu-id="524c7-110">The performance costs that come from creating those objects include storing the configuration data in both the IIS metabase and the COM+ registration database (RegDB) as well as the extra communication between the IIS metabase and the COM+ RegDB that is needed to effectively manage the configuration data.</span></span>

<span data-ttu-id="524c7-111">Wenn IIS einen zweiten com+-Dienst verwenden muss, z. b. die Synchronisierung, musste hierfür ein vollständig anderes Shim-Objekt erstellt werden.</span><span class="sxs-lookup"><span data-stu-id="524c7-111">If IIS needed to use a second COM+ service, such as synchronization, it had to create a completely different shim object to do so.</span></span> <span data-ttu-id="524c7-112">Um com+-Transaktionen und die Synchronisierung zu verwenden, wäre ein drittes shimobjekttyp erforderlich.</span><span class="sxs-lookup"><span data-stu-id="524c7-112">To use both COM+ transactions and synchronization, a third type of shim object would be needed.</span></span> <span data-ttu-id="524c7-113">Die Komplexität dieses Ansatzes wird als O (N2) skaliert, sodass die Implementierung neuer Dienste sehr schwierig ist.</span><span class="sxs-lookup"><span data-stu-id="524c7-113">The complexity of this approach scales as O(n2), making the implementation of new services extremely difficult.</span></span>

<span data-ttu-id="524c7-114">Durch die Einführung von COM+-Diensten ohne Komponenten werden die benötigten Dienste über ein Objekt konfiguriert, das von der-Klasse instanziiert wird.</span><span class="sxs-lookup"><span data-stu-id="524c7-114">With the introduction of COM+ services without components, the services that are needed are configured through an object instantiated from the class.</span></span> <span data-ttu-id="524c7-115">Die [**cserviceconfig**](cserviceconfig.md) -Klasse implementiert die zum Konfigurieren der verschiedenen Dienste erforderlichen Schnittstellen und bietet gleichzeitig die Flexibilität, mehrere Dienste gleichzeitig zu unterstützen, sowie die Möglichkeit, neue Dienste in Zukunft zu unterstützen.</span><span class="sxs-lookup"><span data-stu-id="524c7-115">The [**CServiceConfig**](cserviceconfig.md) class implements the interfaces needed to configure the different services while providing the flexibility to support multiple services at the same time and the ability to support new services in the future.</span></span>

<span data-ttu-id="524c7-116">Die konfigurierten Dienste können dann über zwei verschiedene Mechanismen verwendet werden: Sie können mithilfe der [**cokreateactivity**](/windows/desktop/api/ComSvcs/nf-comsvcs-cocreateactivity) -Funktion verwendet werden, die die Dienste auf alle über die von der-Funktion erstellten-Aktivitäten übermittelt, und Sie können auch verwendet werden, indem Sie die Arbeit einbetten, die die Dienste zwischen Aufrufen der [**coenterservicedomain**](/windows/desktop/api/ComSvcs/nf-comsvcs-coenterservicedomain) -und [**coleaveservicedomain**](/windows/desktop/api/ComSvcs/nf-comsvcs-coleaveservicedomain) -Funktionen verwendet.</span><span class="sxs-lookup"><span data-stu-id="524c7-116">The configured services can then be used through two different mechanisms: They can be used through the [**CoCreateActivity**](/windows/desktop/api/ComSvcs/nf-comsvcs-cocreateactivity) function, which applies the services to all of the work submitted via the activity that is created by the function, and they can also be used by embedding the work that uses the services between calls to the [**CoEnterServiceDomain**](/windows/desktop/api/ComSvcs/nf-comsvcs-coenterservicedomain) and [**CoLeaveServiceDomain**](/windows/desktop/api/ComSvcs/nf-comsvcs-coleaveservicedomain) functions.</span></span> <span data-ttu-id="524c7-117">Keine dieser Funktionen erfordert die Erstellung neuer Komponenten, damit die com+-Dienste verwendet werden können. nur das [**cserviceconfig**](cserviceconfig.md) -Objekt ist erforderlich.</span><span class="sxs-lookup"><span data-stu-id="524c7-117">None of these functions requires the creation of any new components to be able to use the COM+ services; only the [**CServiceConfig**](cserviceconfig.md) object is needed.</span></span>
