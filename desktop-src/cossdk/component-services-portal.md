# <a name="com-component-services"></a>COM+ (Komponentendienste)

## <a name="purpose"></a>Zweck

COM+ ist eine Weiterentwicklung von Microsoft Component Object Model (COM) und Microsoft Transaction Server (MTS). COM+ baut auf Anwendungen auf, die mit COM, MTS und anderen COM-basierten Technologien geschrieben wurden, und erweitert sie. COM+ übernimmt viele der Ressourcenverwaltungsaufgaben, die Sie zuvor selbst programmieren mussten, z. B. Threadzuordnung und Sicherheit. COM+ macht Ihre Anwendungen auch skalierbarer, indem Threadpooling, Objektpooling und Just-In-Time-Objektaktivierung zur Verfügung gestellt werden. COM+ trägt auch zum Schutz der Integrität Ihrer Daten bei, indem Transaktionsunterstützung geboten wird, auch wenn sich eine Transaktion über mehrere Datenbanken über ein Netzwerk erstreckt.

## <a name="where-applicable"></a>Anwendungsbereich

COM+ kann verwendet werden, um unternehmensweite, unternehmenskritische, verteilte Anwendungen für Windows zu entwickeln.

Als Systemadministrator installieren, bereitstellen und konfigurieren Sie COM+-Anwendungen und deren Komponenten. Wenn Sie ein Anwendungsprogrammierer sind, schreiben Sie Komponenten und integrieren sie als Anwendungen. Wenn Sie ein Toolshersteller sind, entwickeln oder ändern Sie Tools, um in der COM+-Umgebung zu funktionieren.

## <a name="developer-audience"></a>Entwicklergruppe

COM+ ist in erster Linie für Microsoft Visual C++- und Microsoft Visual Basic-Entwickler konzipiert.

## <a name="run-time-requirements"></a>Laufzeitanforderungen

COM+ Version 1.5 ist ab Windows XP und Windows Server 2003 in Windows enthalten. COM+ Version 1.0 ist in Windows 2000 enthalten.
