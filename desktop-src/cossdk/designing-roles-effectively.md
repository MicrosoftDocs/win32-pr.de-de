# <a name="designing-roles-effectively"></a><span data-ttu-id="9e20f-103">Effektives Entwerfen von Rollen</span><span class="sxs-lookup"><span data-stu-id="9e20f-103">Designing Roles Effectively</span></span>

## <a name="user-and-data-characteristics-and-the-suitability-of-roles"></a><span data-ttu-id="9e20f-106">Benutzer-und Daten Merkmale und die Eignung von Rollen</span><span class="sxs-lookup"><span data-stu-id="9e20f-106">User and Data Characteristics and the Suitability of Roles</span></span>

<span data-ttu-id="9e20f-107">Rollen funktionieren sehr gut, um Gruppen von Benutzern zu bezeichnen und auf dieser Grundlage die Aktionen zu filtern, die diese Benutzer ausführen können.</span><span class="sxs-lookup"><span data-stu-id="9e20f-107">Roles work very well to characterize groups of users and, on that basis, filter what actions those users can perform.</span></span> <span data-ttu-id="9e20f-108">Dies wird häufig in der Praxis durch das Erstellen von Rollen durchgeführt, die die Position eines Benutzers innerhalb einer Organisation widerspiegeln, z. –. die Rollen "Manager" und "Kassierer", "Administratoren" und "Leser", "Projekt ein Mitarbeiter" und "Projekt zwei Mitarbeiter".</span><span class="sxs-lookup"><span data-stu-id="9e20f-108">Often, this is carried out in practice by creating roles that reflect a user's place within an organization—for example, the roles "Managers" and "Tellers," "Administrators" and "Readers," "Project One Employees" and "Project Two Employees."</span></span> <span data-ttu-id="9e20f-109">In solchen Fällen, in denen die Daten, auf die zugegriffen wird, relativ zu den Benutzern relativ generisch sind, können Rollen effizient verwendet werden, um Geschäftsregeln wie die folgenden zu erzwingen:</span><span class="sxs-lookup"><span data-stu-id="9e20f-109">In such cases, where the data being accessed is fairly generic relative to the users, roles can be used efficiently to enforce business rules such as the following:</span></span>

-   <span data-ttu-id="9e20f-110">"Manager können jede beliebige Menge an Geld übertragen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-110">"Managers can transfer any amount of money.</span></span> <span data-ttu-id="9e20f-111">Der Kassierer kann nur bis zu $10.000 übertragen. "</span><span class="sxs-lookup"><span data-stu-id="9e20f-111">Tellers can transfer only up to $10,000."</span></span>

-   <span data-ttu-id="9e20f-112">"Administratoren können beliebige Änderungen vornehmen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-112">"Administrators can change anything.</span></span> <span data-ttu-id="9e20f-113">Alle anderen Personen können nur lesen. "</span><span class="sxs-lookup"><span data-stu-id="9e20f-113">Everyone else can only read."</span></span>

-   <span data-ttu-id="9e20f-114">"Projekt ein Mitarbeiter kann auf eine bestimmte Datenbank zugreifen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-114">"Project One employees can access a certain database.</span></span> <span data-ttu-id="9e20f-115">Das ist nicht möglich. "</span><span class="sxs-lookup"><span data-stu-id="9e20f-115">No one else can."</span></span>

<span data-ttu-id="9e20f-116">Abhängig davon, wie die Rollen die Organisationsstruktur eines Unternehmens modellieren, können Benutzer natürlich auf mehrere Rollen zugreifen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-116">Users might naturally fall into multiple roles, depending on how the roles model the organizational structure of a business.</span></span>

<span data-ttu-id="9e20f-117">Rollen funktionieren jedoch nicht sehr gut, wenn eine Entscheidung über den Sicherheits Zugriff auf die Merkmale eines bestimmten Daten Abschnitts trifft.</span><span class="sxs-lookup"><span data-stu-id="9e20f-117">However, roles don't work very well when a security access decision rests on the characteristics of a particular piece of data.</span></span> <span data-ttu-id="9e20f-118">Beispielsweise wäre es schwierig, Rollen zu verwenden, um komplizierte Benutzerdaten Beziehungen wie die folgenden widerzuspiegeln:</span><span class="sxs-lookup"><span data-stu-id="9e20f-118">For example, it would be difficult to use roles to reflect complicated user-data relationships such as the following:</span></span>

-   <span data-ttu-id="9e20f-119">"Ein bestimmter Manager kann nur für Ihre Berichte auf Personaldaten zugreifen."</span><span class="sxs-lookup"><span data-stu-id="9e20f-119">"A particular manager can access personnel data for only her reports."</span></span>

-   <span data-ttu-id="9e20f-120">"Joe Consumer kann seinen Konto Saldo nachschlagen."</span><span class="sxs-lookup"><span data-stu-id="9e20f-120">"Joe Consumer can look up his account balance."</span></span>

<span data-ttu-id="9e20f-121">In solchen Fällen muss die Sicherheitsüberprüfung häufig in der Datenbank selbst durchgeführt werden, bei der es einfacher ist, die angeborenen Merkmale der Daten zu berücksichtigen, auf die zugegriffen wird.</span><span class="sxs-lookup"><span data-stu-id="9e20f-121">In such cases, security checking must often be done at the database itself, where it is easier to take into account innate characteristics of the data being accessed.</span></span> <span data-ttu-id="9e20f-122">Dies bedeutet, dass Sie Identitäts *Wechsel verwenden müssen* , um die Client Identität an die Datenbank zu übergeben und die Datenbank überprüfen zu lassen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-122">This means that you must use *impersonation* to pass the client identity along to the database and let the database validate the request.</span></span> <span data-ttu-id="9e20f-123">Dies kann die Leistung beeinträchtigen und den Anwendungs Entwurf beeinflussen – beispielsweise funktioniert das Verbindungspooling möglicherweise nicht, wenn eine Verbindung unter einer bestimmten Benutzeridentität geöffnet wird.</span><span class="sxs-lookup"><span data-stu-id="9e20f-123">This can affect performance and can affect the application's design—for example, connection pooling might not work when a connection is opened under a particular user identity.</span></span> <span data-ttu-id="9e20f-124">Eine Erläuterung der betroffenen Probleme finden Sie unter [mehrstufige Anwendungssicherheit](multi-tier-application-security.md) und Client Identitätswechsel [und Delegierung](client-impersonation-and-delegation.md).</span><span class="sxs-lookup"><span data-stu-id="9e20f-124">For a discussion of issues involved, see [Multi-Tier Application Security](multi-tier-application-security.md) and [Client Impersonation and Delegation](client-impersonation-and-delegation.md).</span></span>

## <a name="complexity-and-scalability-of-a-role-based-authorization-policy"></a><span data-ttu-id="9e20f-125">Komplexität und Skalierbarkeit einer Role-Based Autorisierungs Richtlinie</span><span class="sxs-lookup"><span data-stu-id="9e20f-125">Complexity and Scalability of a Role-Based Authorization Policy</span></span>

<span data-ttu-id="9e20f-126">Jede Sicherheitsrichtlinie, die Sie bereitstellen, ist nur so effektiv wie die Implementierung durch die Systemadministratoren, die Ihre Anwendung bereitstellen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-126">Whatever security policy you put in place will be only as effective as its implementation by the system administrators deploying your application.</span></span> <span data-ttu-id="9e20f-127">Wenn Administratoren Fehler beim Zuweisen von Benutzern zu den richtigen Rollen gemäß Ihrer Sicherheitsrichtlinie leisten, funktioniert Ihre Richtlinie nicht wie vorgesehen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-127">If administrators make mistakes in assigning users to the correct roles according to your security policy, your policy won't work as intended.</span></span> <span data-ttu-id="9e20f-128">Probleme treten wahrscheinlich unter den folgenden Umständen auf:</span><span class="sxs-lookup"><span data-stu-id="9e20f-128">Problems are most likely to occur under the following circumstances:</span></span>

-   <span data-ttu-id="9e20f-129">Sie haben eine sehr komplizierte rollenbasierte Richtlinie erstellt, mit vielen Rollen und Benutzern, die zahlreichen Rollen zuordnet.</span><span class="sxs-lookup"><span data-stu-id="9e20f-129">You have made a very intricate role-based policy, with many roles and users mapping to numerous roles.</span></span>
-   <span data-ttu-id="9e20f-130">Sie erstellen Rollen mit mehrdeutigen Kriterien für die Benutzer, die zu Ihnen gehören sollen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-130">You create roles with ambiguous criteria for who should belong to them.</span></span>
-   <span data-ttu-id="9e20f-131">An dem Standort, an dem die Anwendung installiert ist, gibt es viele Benutzer, die sich häufig innerhalb der Organisation bewegen und sich in Bezug auf die von Ihnen erstellten Rollen ändern.</span><span class="sxs-lookup"><span data-stu-id="9e20f-131">There are many users at the site where the application is installed, and users are frequently moving within the organization, changing with respect to the roles you have created.</span></span>
-   <span data-ttu-id="9e20f-132">An dem Standort, an dem die Anwendung installiert ist, gibt es viele Administratoren mit einer Division von Zuständigkeiten, sodass ein Administrator, der mit den Sicherheitsanforderungen Ihrer Anwendung vertraut ist, nicht mit den großen Gruppen von Benutzern vertraut ist, die ihn verwenden müssen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-132">There are many administrators at the site where the application is installed, with division of responsibilities, so that an administrator familiar with the security requirements of your application is potentially unfamiliar with the large groups of users that must use it.</span></span> <span data-ttu-id="9e20f-133">Dies ist insbesondere dann von Bedeutung, wenn Benutzer innerhalb des Unternehmens wechseln – diese Informationen müssen gut kommuniziert werden.</span><span class="sxs-lookup"><span data-stu-id="9e20f-133">This is of particular concern as users move within the organization—such information needs to be well communicated.</span></span>

<span data-ttu-id="9e20f-134">Wenn die Anzahl der Rollen, die einer Anwendung zugewiesen sind, zunimmt, bedeutet dies, dass die Zeitspanne, die com+ für die Suche nach der Aufrufer-Mitgliedschaft in diesen Rollen aufwenden muss, mit einer wahrscheinlichen Beeinträchtigung der Leistung.</span><span class="sxs-lookup"><span data-stu-id="9e20f-134">Additionally, as the number of roles assigned to an application increases, so does the amount of time COM+ must spend looking up caller membership in those roles, with a likely degradation in performance.</span></span>

<span data-ttu-id="9e20f-135">Um die Komplexität zu verwalten und diese Probleme zu mindern, können Sie die folgenden Richtlinien verwenden:</span><span class="sxs-lookup"><span data-stu-id="9e20f-135">To manage complexity and mitigate these concerns, you can use the following guidelines:</span></span>

-   <span data-ttu-id="9e20f-136">Wählen Sie Rollennamen aus, die selbst beschreibend sind.</span><span class="sxs-lookup"><span data-stu-id="9e20f-136">Choose role names that are self-descriptive.</span></span> <span data-ttu-id="9e20f-137">Legen Sie fest, wie möglich die Benutzer zu welchen Rollen gehören sollen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-137">Make it as obvious as possible which users should belong to which roles.</span></span>
-   <span data-ttu-id="9e20f-138">Machen Sie Ihre rollenbasierte Richtlinie so einfach wie möglich (und nicht einfacher).</span><span class="sxs-lookup"><span data-stu-id="9e20f-138">Make your role-based policy as simple as possible (and no simpler).</span></span> <span data-ttu-id="9e20f-139">Wählen Sie möglichst wenige Rollen aus, während Sie eine effektive Richtlinie beibehalten.</span><span class="sxs-lookup"><span data-stu-id="9e20f-139">Choose as few roles as you can, while maintaining an effective policy.</span></span>
-   <span data-ttu-id="9e20f-140">Dokumentieren Sie eindeutig die rollenbasierte Richtlinie, die Sie für Administratoren erstellen, unabhängig davon, ob die Rollen Mitgliedschaft offensichtlich ist (für Sie).</span><span class="sxs-lookup"><span data-stu-id="9e20f-140">Clearly document the role-based policy that you construct for administrators, whether role membership is obvious (to you).</span></span> <span data-ttu-id="9e20f-141">Verwenden Sie insbesondere das Beschreibungsfeld, das für jede Rolle verfügbar ist, um die Absicht der Rolle zu beschreiben.</span><span class="sxs-lookup"><span data-stu-id="9e20f-141">In particular, use the description field available for each role to describe the intent of the role.</span></span> <span data-ttu-id="9e20f-142">Wenn Sie nicht in der Regel beschreiben können, wer in einigen Sätzen der Rolle angehören soll, ist dies wahrscheinlich zu kompliziert.</span><span class="sxs-lookup"><span data-stu-id="9e20f-142">If you can't describe generally who should belong to the role in a couple of sentences, it is probably too complicated.</span></span>
-   <span data-ttu-id="9e20f-143">Legen Sie nachdrücklich fest, dass die Administratoren Ihrer Anwendung Rollen mit Benutzergruppen anstelle einzelner Benutzer auffüllen.</span><span class="sxs-lookup"><span data-stu-id="9e20f-143">Strongly suggest that administrators of your application populate roles with user groups instead of individual users.</span></span> <span data-ttu-id="9e20f-144">Dies ist eine viel besser skalierbare Lösung.</span><span class="sxs-lookup"><span data-stu-id="9e20f-144">This is a much more scalable solution.</span></span> <span data-ttu-id="9e20f-145">Es vereinfacht das Neuzuweisen oder Entfernen von Benutzern, wenn Sie sich innerhalb der Organisation bewegen, und isoliert Administratoren von einem gewissen Maß an Überwachung und Problemen bei der Kommunikation.</span><span class="sxs-lookup"><span data-stu-id="9e20f-145">It makes it easier to reassign or remove users as they move within the organization and insulates administrators from a certain amount of oversight and problems in communication.</span></span>
