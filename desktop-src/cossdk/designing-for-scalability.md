<span data-ttu-id="0d93f-104">Skalierbarkeit ist die Fähigkeit einer Anwendung, eine zusätzliche Last mit einem linearen Anstieg der Ressourcennutzung zu bedienen.</span><span class="sxs-lookup"><span data-stu-id="0d93f-104">Scalability is the ability of an application to service an additional load with a linear increase in resource usage.</span></span> <span data-ttu-id="0d93f-105">Skalierbarkeit ist in allen verteilten Anwendungen wichtig.</span><span class="sxs-lookup"><span data-stu-id="0d93f-105">Scalability is important in any distributed application.</span></span> <span data-ttu-id="0d93f-106">Einschränkungen der Skalierbarkeit liegen in der Regel im Zusammenhang mit der Ressourcenverwendung und Abhängigkeiten, die versehentlich im Anwendungs Entwurf erstellt wurden.</span><span class="sxs-lookup"><span data-stu-id="0d93f-106">Limits to scalability usually center around resource use and dependencies inadvertently created in the application's design.</span></span>

<span data-ttu-id="0d93f-107">In der folgenden Liste werden Skalierbarkeits Probleme beschrieben und Lösungen vorgeschlagen:</span><span class="sxs-lookup"><span data-stu-id="0d93f-107">The following list describes scalability issues and suggests solutions:</span></span>

-   <span data-ttu-id="0d93f-108">Ressourcen innerhalb des Computers.</span><span class="sxs-lookup"><span data-stu-id="0d93f-108">Intra-computer resources.</span></span> <span data-ttu-id="0d93f-109">Die Anzahl der Threads und der verfügbaren Arbeitsspeicher kann die Skalierbarkeit einschränken.</span><span class="sxs-lookup"><span data-stu-id="0d93f-109">The number of threads and memory available can limit scalability.</span></span> <span data-ttu-id="0d93f-110">Verwenden Sie ein Threading Modell, das für Ihre Anwendung am effizientesten ist.</span><span class="sxs-lookup"><span data-stu-id="0d93f-110">Use a threading model that is most efficient for your application.</span></span>
-   <span data-ttu-id="0d93f-111">Computer übergreifende Ressourcen.</span><span class="sxs-lookup"><span data-stu-id="0d93f-111">Inter-computer resources.</span></span> <span data-ttu-id="0d93f-112">Die Anzahl der verfügbaren Computer zum Verteilen der Anwendungs Arbeitsauslastung kann die Skalierbarkeit beeinträchtigen.</span><span class="sxs-lookup"><span data-stu-id="0d93f-112">The number of available computers to distribute the application workload can affect scalability.</span></span>
-   <span data-ttu-id="0d93f-113">Client Affinität.</span><span class="sxs-lookup"><span data-stu-id="0d93f-113">Client affinity.</span></span> <span data-ttu-id="0d93f-114">Zwei Affinitäts Situationen können versehentlich von einer Anwendung erstellt werden: eine Situation, in der die Anwendung vom Zustand aus den Daten abhängt, die der Client mit der Anforderung sendet. und eine Situation, in der die Anwendung einen Client spezifischen Zustand erfordert.</span><span class="sxs-lookup"><span data-stu-id="0d93f-114">Two situations of affinity can be inadvertently created by an application: a situation where the application depends on state from data the client sends with its request; and a situation where the application requires a client-specific state.</span></span> <span data-ttu-id="0d93f-115">Vermeiden Sie das Entwerfen von Zustands Abhängigkeiten zwischen dem Client und der Anwendung.</span><span class="sxs-lookup"><span data-stu-id="0d93f-115">Avoid designing state dependency between the client and the application.</span></span>
-   <span data-ttu-id="0d93f-116">Server Affinität.</span><span class="sxs-lookup"><span data-stu-id="0d93f-116">Server affinity.</span></span> <span data-ttu-id="0d93f-117">Eine COM+-Anwendung kann die Skalierbarkeit einschränken, indem Sie eine Server Affinität erstellt, bei der die Anwendung von Informationen zu einem bestimmten Server Computer abhängig ist.</span><span class="sxs-lookup"><span data-stu-id="0d93f-117">A COM+ application can limit its scalability by creating a server affinity, where the application depends on going to a particular server computer for information.</span></span> <span data-ttu-id="0d93f-118">Diese Affinität kann bei vielen Daten Bank orientierten Anwendungen auftreten.</span><span class="sxs-lookup"><span data-stu-id="0d93f-118">This affinity can occur with many database-oriented applications.</span></span> <span data-ttu-id="0d93f-119">Die beste Möglichkeit, einen Engpass bei der Server Affinität zu vermeiden, ist das Partitionieren von Daten auf verschiedenen Server Computern.</span><span class="sxs-lookup"><span data-stu-id="0d93f-119">The best way to avoid a server-affinity bottleneck is to partition data onto various server computers.</span></span> <span data-ttu-id="0d93f-120">Unterteilen Sie Kundendaten z. b. durch den am häufigsten verwendeten Schlüssel, oder Spannen Sie eine Kundendatenbank über mehrere Server mit dem Nachnamen des Kunden (z. b. Server1: a-f, Server2: g-m, Server3: n-z).</span><span class="sxs-lookup"><span data-stu-id="0d93f-120">For example, divide customer data among servers by the most frequently accessed key, or span a customer database over several servers using the customer's last name (for example, Server1: a-f, Server2: g-m, Server3: n-z).</span></span>
    > [!Note]  
    > <span data-ttu-id="0d93f-121">Die Daten Partitionierung kann der Programmierlogik eine große Komplexität verleihen und sollte erst ausgeführt werden, nachdem andere Optionen zum Erhöhen der Skalierbarkeit ausprobiert wurden.</span><span class="sxs-lookup"><span data-stu-id="0d93f-121">Data partitioning can add a great deal of complexity to the programming logic and should be done only after other options to increase scalability have been tried.</span></span>

     

-   <span data-ttu-id="0d93f-122">Objekt Lebensdauer.</span><span class="sxs-lookup"><span data-stu-id="0d93f-122">Object lifetime.</span></span> <span data-ttu-id="0d93f-123">Um skalierbar zu sein, muss eine COM+-Anwendung genau auf die Lebensdauer von Objekten achten.</span><span class="sxs-lookup"><span data-stu-id="0d93f-123">To be scalable, a COM+ application must pay close attention to the object exists, it is consuming resources.</span></span> <span data-ttu-id="0d93f-125">Es ist wichtig, sicherzustellen, dass die Lebensdauer von Objekten, die zu teuren Ressourcen gehören, sorgfältig verwaltet wird.</span><span class="sxs-lookup"><span data-stu-id="0d93f-125">It is important to make sure that the lifetimes of objects that hold onto expensive resources are carefully managed.</span></span> <span data-ttu-id="0d93f-126">Bei hochleistungsfähigen Objekten, die keine teuren Ressourcen belegen, kann das [com+-Objekt Pooling](com--object-pooling.md) die Skalierbarkeit erhöhen, da Sie die Pooling-Werte administrativ anpassen können, um die Vorteile der Hardware zu nutzen, die Sie möglicherweise haben.</span><span class="sxs-lookup"><span data-stu-id="0d93f-126">For high-demand objects that don't consume expensive resources, [COM+ object pooling](com--object-pooling.md) can increase scalability because you can administratively adjust the pooling values to take advantage of whatever hardware you might have.</span></span> <span data-ttu-id="0d93f-127">Und das ist eine natürliche Möglichkeit zum Steuern von Verbindungen: Wenn Sie z. b. über eine Lizenz für 20 SQL-Verbindungen verfügen, können Sie dies mit der Einstellung "Max Pool" vorschreiben.</span><span class="sxs-lookup"><span data-stu-id="0d93f-127">And it's a natural way to govern connections: For example, if you have a license for 20 SQL connections, you can dictate that with the Max Pool setting.</span></span>
-   <span data-ttu-id="0d93f-128">Gruppieren von Anwendungskomponenten.</span><span class="sxs-lookup"><span data-stu-id="0d93f-128">Application component grouping.</span></span> <span data-ttu-id="0d93f-129">Um die Skalierbarkeit einer COM+-Anwendung zu verbessern, sollten die Komponenten der mittleren Ebene in zeitabhängige und zeitunabhängige Dienste aufgeteilt werden.</span><span class="sxs-lookup"><span data-stu-id="0d93f-129">To enhance the scalability of a COM+ application, the middle-tier components should be divided into time-dependent and time-independent services.</span></span> <span data-ttu-id="0d93f-130">Dies ermöglicht es Ihnen, sich auf die Verwendung eines Microsoft Windows-Dienstanbieter zu konzentrieren, um eine erforderliche Komponenten Aktion zu implementieren.</span><span class="sxs-lookup"><span data-stu-id="0d93f-130">This allows you to focus on possibly using a Microsoft Windows service to implement a required component action.</span></span> <span data-ttu-id="0d93f-131">Beispielsweise können Sie einen Dienst wie [Message Queuing](/previous-versions/windows/desktop/legacy/ms711472(v=vs.85)) -oder [com+-Komponenten in der Warteschlange](com--queued-components.md) verwenden, um zeitunabhängige, asynchrone Aufgaben zu verarbeiten.</span><span class="sxs-lookup"><span data-stu-id="0d93f-131">For example, you could elect to use a service such as [Message Queuing](/previous-versions/windows/desktop/legacy/ms711472(v=vs.85)) or [COM+ queued components](com--queued-components.md) to handle time-independent, asynchronous tasks.</span></span>
