# <a name="com-queued-components"></a>Com+-Komponenten in der Warteschlange

Mit dem Feature "com+-Komponenten in der Warteschlange" kann ein Client einen Dienst von einer COM+-Serveranwendung anfordern, auch wenn die Serveranwendung vorübergehend nicht verfügbar ist. Die Client Anforderung wird in einer Warteschlange gespeichert und verarbeitet, wenn der Server verfügbar wird.

> [!Note]  
> Wenn Sie den com+-Dienst in der Warteschlange verwenden möchten, müssen [Message Queuing](/previous-versions/windows/desktop/legacy/ms711472(v=vs.85)) auf dem Computer installiert sein.
