# <a name="com-partitions-concepts"></a>COM+-Partitionskonzepte

In Computingumgebungen, in denen verschiedene Versionen oder Konfigurationen einer Anwendung auf demselben Computer verwendet werden müssen, können Konflikte auftreten, wenn eine Version der Anwendung andere Komponenten als die andere erfordert oder wenn eine Version eine Komponente auf dem Computer erfordert, die mit der anderen Version der Anwendung nicht kompatibel ist. In der Vergangenheit war die einzige Möglichkeit, dieses Problem zu lösen, die Wartung mehrerer Computer und das Ausführen einer anderen Version der Anwendung auf jedem Computer. Ein solcher Ansatz ist kostspielig und ineffizient, da er die Hardwarekosten und den Verwaltungsaufwand erhöht.

COM+ löst dieses Problem, indem das Com+-Partitionsfeature zur Verfügung stellt. Sie können COM+-Partitionen verwenden, um verschiedene Versionen einer COM+-Anwendung auf einem Computer zu installieren und gleichzeitig ausführen.

-   [Komponenten und Partitionen in der COM+-Warteschlange](com--queued-components-and-partitions.md)
