# <a name="designing-the-com-application-using-uml"></a>Entwerfen der com+-Anwendung mithilfe von UML

Zum Entwickeln einer erfolgreichen com+-Anwendung ist ein vorab Anwendungsarchitektur Entwurf erforderlich. Der Unified Modeling Language (UML) ist entscheidend für diese Entwurfsentwicklung. UML ist eine Modellierungs Notation für Anwendungsdaten und Prozesse, die die bewährten Methoden der Softwarebranche kombiniert. Da die UML-Anwendung die Anwendung in drei Ansichten unterteilt, die die Anwendung sowie deren Verpackung und Implementierung widerspiegeln, ist die Modellierungs Notation gut für die Unterstützung von Enterprise Modeling.

Die UML behandelt drei Ansichten der Anwendung wie folgt:

-   Die statische Ansicht, die anhand von Informationen modelliert wird, die aus Benutzer Szenarien und Klassendiagrammen entnommen wurden.
-   Die dynamische Ansicht, die mithilfe von Sequenz-, Zusammenarbeits-und Zustands Übergangs Diagrammen modelliert wird.
-   Die funktionale Ansicht, bei der es sich um die herkömmliche beschreibende Darstellung mit Pseudo Code und Spezifikationen handelt.

Die Informationen für diese Sichten können durch die folgenden drei Entwurfs Schritte erfasst werden, die für die UML-Arbeit geeignet sind. Bevor Sie eine einzelne Codezeile schreiben, müssen Sie die folgenden Modelle erstellen:

<dl> <dt>

<span id="Conceptual_model"></span><span id="conceptual_model"></span><span id="CONCEPTUAL_MODEL"></span>Konzeptionelles Modell
</dt> <dd>

Entscheiden Sie, welche Komponenten und Dienste erforderlich sind.

</dd> <dt>

<span id="Logical_model"></span><span id="logical_model"></span><span id="LOGICAL_MODEL"></span>Logisches Modell
</dt> <dd>

Bestimmen Sie, welcher logische Entwurfs Ebene Sie angehören.

</dd> <dt>

<span id="Physical_model"></span><span id="physical_model"></span><span id="PHYSICAL_MODEL"></span>Physisches Modell
</dt> <dd>

Legen Sie fest, wo sich die Komponenten physisch befinden und wie Sie codiert werden sollen.

</dd> </dl>

Diese Modelle können dann mit UML-basierten Case-Tools verwendet werden. Weitere Informationen zu diesen drei Entwurfs Modellen finden Sie in den folgenden Themen in diesem Abschnitt:

-   [Das konzeptionelle Modell: Anwendungsanforderungen](the-conceptual-model--application-requirements.md)
-   [Das logische Modell: Anwendungs Definition und-Planung](the-logical-model--application-definition-and-planning.md)
-   [Das physische Modell: Anwendungsarchitektur](the-physical-model--application-architecture.md)
