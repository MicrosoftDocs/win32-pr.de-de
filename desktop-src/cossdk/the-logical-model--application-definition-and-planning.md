# <a name="the-logical-model-application-definition-and-planning"></a><span data-ttu-id="5bb35-104">Das logische Modell: Anwendungs Definition und-Planung</span><span class="sxs-lookup"><span data-stu-id="5bb35-104">The Logical Model: Application Definition and Planning</span></span>

<span data-ttu-id="5bb35-105">Der zweite Schritt in einem com+-Anwendungs Entwurfsprozess besteht darin, das konzeptionelle Modell in die logischen Ebenen der dreistufigen Architektur zu unterbrechen: die *Präsentationsebene* oder die Benutzerdienste. die *mittlere Ebene* oder Unternehmens Dienste; und die *Datenebene* oder Datendienste.</span><span class="sxs-lookup"><span data-stu-id="5bb35-105">The second step in a COM+ application design process is to break the conceptual model out into the logical tiers of the three-tier architecture: the *presentation tier*, or user services; the *middle tier*, or business services; and the *data tier*, or data services.</span></span> <span data-ttu-id="5bb35-106">Wenn Sie mit der dreistufigen Architektur nicht vertraut sind, finden Sie unter [Verwenden eines Three-Tier Architekturmodells](using-a-three-tier-architecture-model.md)Weitere Informationen.</span><span class="sxs-lookup"><span data-stu-id="5bb35-106">If you are unfamiliar with three-tier architecture, see [Using a Three-Tier Architecture Model](using-a-three-tier-architecture-model.md).</span></span>

<span data-ttu-id="5bb35-107">Beginnen Sie den Prozess, indem Sie das konzeptionelle Modell nach Substantiven und Verben betrachten.</span><span class="sxs-lookup"><span data-stu-id="5bb35-107">Begin this process by looking through the conceptual model for nouns and verbs.</span></span> <span data-ttu-id="5bb35-108">Die Nomen können häufig in Geschäftsobjekte (Klassen) übersetzt werden, und die zugeordneten Verben können in Aktionen übersetzt werden (Methoden der Klassen).</span><span class="sxs-lookup"><span data-stu-id="5bb35-108">The nouns can often translate into business objects (classes), and the verbs associated with them can translate into actions (methods of the classes).</span></span> <span data-ttu-id="5bb35-109">Obwohl es schwierig sein kann, alle Nomen als Geschäftsobjekte zu definieren, werden Ausfälle oder Fehler durch den Zeitpunkt der Fertigstellung des logischen Modells offensichtlich.</span><span class="sxs-lookup"><span data-stu-id="5bb35-109">While it can be difficult to define all the nouns as business objects, omissions or mistakes will become obvious by the time you complete the logical model.</span></span>

<span data-ttu-id="5bb35-110">Die meisten Geschäftsobjekte werden auf der Unternehmens Dienst Ebene erstellt, wobei die restlichen Objekte zwischen den Benutzeroberflächen Objekten in der Benutzer Dienst Ebene und den Datenobjekten auf der Ebene der Datendienste aufgeteilt werden.</span><span class="sxs-lookup"><span data-stu-id="5bb35-110">Most of the business objects will end up in the business services tier, with the remaining objects divided between user interface objects in the user services tier and data objects in the data services tier.</span></span> <span data-ttu-id="5bb35-111">Beachten Sie beim Erstellen eines logischen Modells mit der dreistufigen Architektur Folgendes:</span><span class="sxs-lookup"><span data-stu-id="5bb35-111">When creating a logical model using the three-tier architecture, keep the following in mind:</span></span>

-   <span data-ttu-id="5bb35-112">Einige der Nomen im konzeptionellen Entwurf stellen nicht die eigentlichen physischen Datenelemente dar, können aber eine Aktion im System oder in der Rolle eines Geschäftsobjekts im System darstellen.</span><span class="sxs-lookup"><span data-stu-id="5bb35-112">Some of the nouns in the conceptual design will not represent actual physical pieces of data but may represent an action on the system or a business object's role on the system.</span></span> <span data-ttu-id="5bb35-113">Ein Geschäftsobjekt kann auch ein Dienst sein, der eine bestimmte Art von Systemsteuerung ausführt, z. b. eine Anmelde-ID für die Sicherheit.</span><span class="sxs-lookup"><span data-stu-id="5bb35-113">A business object can also be a service that performs some sort of system control, such as a login ID for security.</span></span>
-   <span data-ttu-id="5bb35-114">Vermeiden Sie es, vage Geschäftsobjekte zu erstellen, indem Sie im [konzeptionellen Modell](the-conceptual-model--application-requirements.md)"zwischen den Zeilen lesen".</span><span class="sxs-lookup"><span data-stu-id="5bb35-114">Avoid creating vague business objects by "reading between the lines" in the [conceptual model](the-conceptual-model--application-requirements.md).</span></span> <span data-ttu-id="5bb35-115">Diese Geschäftsobjekte sind möglicherweise nicht korrekt, und Sie sollten Sie sorgfältig in Erwägung ziehen, bevor Sie Sie in das logische Modell einschließen.</span><span class="sxs-lookup"><span data-stu-id="5bb35-115">Such business objects may not be correct, and you should consider them carefully before including them in the logical model.</span></span> <span data-ttu-id="5bb35-116">Wenn Sie richtig angezeigt werden, fügen Sie Sie explizit dem konzeptionellen Modell hinzu.</span><span class="sxs-lookup"><span data-stu-id="5bb35-116">If they appear correct, add them to the conceptual model explicitly.</span></span>
-   <span data-ttu-id="5bb35-117">Vermeiden Sie das Erstellen von Geschäftsobjekten, die dieselben Informationen oder Funktionen ausdrücken. Duplizierung kann in Bezug auf die Anwendungs Geschwindigkeit und-Leistung kostspielig sein.</span><span class="sxs-lookup"><span data-stu-id="5bb35-117">Avoid creating business objects that express the same information or function; duplication can be costly in terms of application speed and performance.</span></span>
-   <span data-ttu-id="5bb35-118">Bestimmen von Objektabhängigkeiten.</span><span class="sxs-lookup"><span data-stu-id="5bb35-118">Determine object dependencies.</span></span> <span data-ttu-id="5bb35-119">Einige Nomen im konzeptionellen Modell können einfach Attribute anderer Geschäftsobjekte sein.</span><span class="sxs-lookup"><span data-stu-id="5bb35-119">Some nouns in the conceptual model may simply be attributes of other business objects.</span></span> <span data-ttu-id="5bb35-120">Entscheiden Sie, ob das Attribut unabhängig vorhanden sein kann.</span><span class="sxs-lookup"><span data-stu-id="5bb35-120">Decide if the attribute can exist independently.</span></span> <span data-ttu-id="5bb35-121">Wenn dies der Fall ist, sollte es ein eigenes Geschäftsobjekt werden. Wenn dies nicht der Fall ist, sollte Sie mit dem entsprechenden Geschäftsobjekt kombiniert werden.</span><span class="sxs-lookup"><span data-stu-id="5bb35-121">If so, it should become its own business object; if not, it should be combined with the appropriate business object.</span></span>
-   <span data-ttu-id="5bb35-122">Vermeiden Sie das Erstellen von Geschäftsobjekten, die zu viel versuchen.</span><span class="sxs-lookup"><span data-stu-id="5bb35-122">Avoid creating business objects that try to do too much.</span></span> <span data-ttu-id="5bb35-123">Das Überladen von Geschäftsobjekten kann bedeuten, dass mehr Zeit für die Trennung von Code aufgewendet wird und ein Wartungsaufwand entstehen kann.</span><span class="sxs-lookup"><span data-stu-id="5bb35-123">Overloading business objects can mean more time spent separating code later and can be a maintenance headache.</span></span> <span data-ttu-id="5bb35-124">Unterschiedliche Objekte im konzeptionellen Modell sollten nicht kombiniert werden. Sie sollten separate Geschäftsobjekte verbleiben.</span><span class="sxs-lookup"><span data-stu-id="5bb35-124">Distinct objects in the conceptual model should not be combined; they should remain separate business objects.</span></span> <span data-ttu-id="5bb35-125">Sie können alle Ähnlichkeiten behandeln, indem Sie Code verwenden, um die gemeinsamen Funktionen an ein Geschäftsobjekt zu delegieren.</span><span class="sxs-lookup"><span data-stu-id="5bb35-125">You can handle any similarities by using code to delegate their common functions to a business object.</span></span>
-   <span data-ttu-id="5bb35-126">Entfernen Sie alle Geschäftsobjekte, die in keinem Verwendungs Szenario verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="5bb35-126">Remove any business objects that are not used in any usage scenarios.</span></span> <span data-ttu-id="5bb35-127">Wenn die Objekte für zukünftiges Wachstum vorgesehen sind, sollten Sie Sie nach Abschluss der ersten Anwendung implementieren.</span><span class="sxs-lookup"><span data-stu-id="5bb35-127">If the objects are intended to accommodate future growth, consider implementing them after the initial application is completed.</span></span>

<span data-ttu-id="5bb35-128">An diesem Punkt können Sie mit der Betrachtung von Computern und Code beginnen.</span><span class="sxs-lookup"><span data-stu-id="5bb35-128">At this point, you can start thinking in terms of the computers and code.</span></span> <span data-ttu-id="5bb35-129">Bestimmen Sie aus diesen Unternehmens Diensten die Anzeige-und Eingabefunktionen, die Sie als Benutzerdienste bereitstellen müssen.</span><span class="sxs-lookup"><span data-stu-id="5bb35-129">From these business services, determine the display and input functionality you need to provide as user services.</span></span> <span data-ttu-id="5bb35-130">Definieren Sie, welche Tabellen und gespeicherten Prozeduren als Datendienste bereitgestellt werden müssen.</span><span class="sxs-lookup"><span data-stu-id="5bb35-130">Define what tables and stored procedures need to be provided as data services.</span></span> <span data-ttu-id="5bb35-131">Um das logische Modell abzuschließen, definieren Sie die Schnittstellen für jeden Dienst, und schließen Sie die Definitionen der einzelnen Datenfelder und deren Validierungsregeln ein.</span><span class="sxs-lookup"><span data-stu-id="5bb35-131">To complete the logical model, define the interfaces for each service and include definitions of each data field and their validation rules.</span></span> <span data-ttu-id="5bb35-132">Fügen Sie außerdem alle Funktionen, ihre Syntax und ihre Parameter ein.</span><span class="sxs-lookup"><span data-stu-id="5bb35-132">Also include all functions, their syntax, and their parameters.</span></span>

<span data-ttu-id="5bb35-133">Die Dienst-oder Objektdefinition sollte keinen Aspekt der physischen Implementierung enthalten.</span><span class="sxs-lookup"><span data-stu-id="5bb35-133">The service or object definition should not include any aspect of the physical implementation.</span></span> <span data-ttu-id="5bb35-134">Ziel ist es, eine klare Richtlinie bereitzustellen, mit der die logischen Komponenten-Generatoren arbeiten und anderen Programmierern das Programmieren von Anwendungs Teilen ermöglichen können, die die Komponente verwenden können, bevor Sie tatsächlich abgeschlossen ist.</span><span class="sxs-lookup"><span data-stu-id="5bb35-134">The intent is to provide a clear guideline for the logical component builders to work from and to enable other programmers to code pieces of the application that may use the component before it is actually completed.</span></span> <span data-ttu-id="5bb35-135">Im logischen Modellentwurf sollten Sie jeden Bildschirm, jede Funktion und jedes Objekt dokumentieren.</span><span class="sxs-lookup"><span data-stu-id="5bb35-135">In the logical model design, you should document every screen, function, and object.</span></span> <span data-ttu-id="5bb35-136">Fahren Sie erst mit dem physischen Modell oder der Implementierung fort, wenn Sie diese Kriterien erfüllen.</span><span class="sxs-lookup"><span data-stu-id="5bb35-136">Do not proceed to the physical model or implementation until you meet these criteria.</span></span> <span data-ttu-id="5bb35-137">Wenn Sie den Vorgang fortsetzen, während sich das konzeptionelle Modell und das logische Modell nicht in der Vereinbarung befinden, treten im Verlauf des Entwicklungs Zeitraums schwerwiegende Probleme auf.</span><span class="sxs-lookup"><span data-stu-id="5bb35-137">If you proceed while the conceptual model and logical model are not in agreement, you will have serious problems later in the development cycle.</span></span>

<span data-ttu-id="5bb35-138">Die inkrementelle Entwicklung einer COM+-Anwendung ist üblich.</span><span class="sxs-lookup"><span data-stu-id="5bb35-138">Incremental development of a COM+ application is common.</span></span> <span data-ttu-id="5bb35-139">Dies umfasst das Erstellen einer Teilmenge der endgültigen, bekannten Komponenten und das Testen der einzelnen Anwendungsebenen: Client, Business und Daten Ebenen – bis zum Datenspeicher.</span><span class="sxs-lookup"><span data-stu-id="5bb35-139">This involves creating a subset of the final, known components and testing them through each layer of the application: client, business, and data tiers—through to data storage.</span></span> <span data-ttu-id="5bb35-140">Dieses Arbeitsmodell bietet Einblick in die zusätzlichen Anforderungen der Kunden-und Implementierungsaspekte.</span><span class="sxs-lookup"><span data-stu-id="5bb35-140">This working model provides insight into additional requirements by the customer and implementation considerations.</span></span> <span data-ttu-id="5bb35-141">Dieses Arbeitsmodell wird häufig auf einem einzelnen Computer getestet.</span><span class="sxs-lookup"><span data-stu-id="5bb35-141">Often this working model is tested on a single computer.</span></span>
