# <a name="optimizing-interactions-with-the-com-business-logic-tier"></a>Optimieren von Interaktionen mit der com+-Geschäftslogik Ebene

Die Architektur für eine verteilte Anwendung teilt häufig das System in drei (oder mehr) Ebenen auf. Interaktionen zwischen diesen Ebenen können sich erheblich auf den Erfolg der Anwendung auswirken. In diesem Abschnitt wird erläutert, wie die Interaktionen zwischen der Geschäftslogik Ebene und der Präsentations-und Datenebene optimiert werden. Weitere Informationen finden Sie in den nachfolgenden Themen:

-   [Optimieren von Interaktionen zwischen der com+-Geschäftslogik Ebene und der Präsentationsebene](optimizing-interactions-between-the-com--business-logic-tier-and-the-presentation-tier.md)
-   [Optimieren von Interaktionen zwischen der com+-Geschäftslogik Ebene und der Datenebene](optimizing-interactions-between-the-com--business-logic-tier-and-the-data-tier.md)
