# <a name="optimizing-the-user-experience"></a>Optimieren der Benutzer Leistung

Wenn Sie eine Light-fähige Benutzeroberfläche implementieren, sollten Sie sorgfältig überlegen, wie Benutzer auf das Programmverhalten reagieren können. Vermeiden Sie jarringübergänge oder häufige Änderungen am Inhalt und an der Benutzeroberfläche des Programms. Reibungslose und schrittweise Übergänge, die nur bei Bedarf stattfinden, sind am besten geeignet. Im Idealfall sollte Ihr Programm mit Benutzern in realen Beleuchtungsbedingungen und in Benutzer Szenarien getestet werden. Zum Schluss sollten Sie einen Mechanismus erstellen, mit dem Benutzer die hellen Optimierungen des Programms manuell ändern oder die Funktionalität deaktivieren können.
