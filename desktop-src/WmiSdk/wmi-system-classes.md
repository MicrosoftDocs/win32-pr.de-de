# <a name="wmi-system-classes"></a>WMI-Systemklassen

Die WMI-Systemklassen sind eine Auflistung vordefinierter Klassen, die auf dem Common Information Model [*(CIM) basieren.*](gloss-c.md) Im Gegensatz zu Klassen, die von Anbietern bereitgestellt werden, werden die Systemklassen nicht in Managed Object Format [*MOF-Datei deklariert.*](gloss-m.md) WMI erstellt einen Satz dieser Klassen, [](gloss-n.md) wenn ein neuer WMI-Namespace erstellt wird.

Objekte aus den Systemklassen werden verwendet, um WMI-Aktivitäten zu unterstützen, z. B.: Ereignis- und Anbieterregistrierung, Sicherheit und Ereignisbenachrichtigung. Einige Objekte sind temporär, und einige werden im Repository als Instanzen der Systemklassen gespeichert.

Systemklassen folgen einer Benennungskonvention, die aus einem doppelten Unterstrich ( ) gefolgt \_ \_ vom Klassennamen besteht. Wenn Sie eine MOF-Datei schreiben, [](gloss-p.md)um Klassen für einen WMI-Anbieter zu definieren, kompiliert [**Mofcomp.exe**](mofcomp.md) keine Klasse mit einem anfänglichen doppelten Unterstrich ( ), da dies für \_ WMI-Systemklassennamen reserviert \_ ist.

Die Dokumentation für die Systemklassen enthält nur die lokalen Nichtsystemeigenschaften. Links werden in Klassendefinitionen bereitgestellt, damit Sie schnell und einfach durch die Klassenhierarchie navigieren können.
