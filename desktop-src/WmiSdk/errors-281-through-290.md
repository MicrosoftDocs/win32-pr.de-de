# <a name="errors-281-through-290"></a>Fehler 281 bis 290

Beschreibt die WMI-SNMP-Anbieter Fehler 281 bis 290.

[Schwerwiegender Fehler 282](#fatal-error-282)

[Schwerwiegender Fehler 283](#fatal-error-283)

## <a name="fatal-error-282"></a>Schwerwiegender Fehler 282

<dl> <dt>

<span id="_282__Fatal_____fileName___line____Syntax_error._Last_token_read_is__token__"></span><span id="_282__fatal_____filename___line____syntax_error._last_token_read_is__token__"></span><span id="_282__FATAL_____FILENAME___LINE____SYNTAX_ERROR._LAST_TOKEN_READ_IS__TOKEN__"></span>**<282, schwerwiegender>: " <fileName> : <Zeilen \#>: Syntax Fehler. Letzter gelesener Token ist <token> "**
</dt> <dd>

Dieser Fehler tritt auf, wenn der Compiler weiß, dass ein Syntax Fehler aufgetreten ist, aber keine spezifischeren Informationen über den Syntax Fehler erhalten kann.

</dd> </dl>

## <a name="fatal-error-283"></a>Schwerwiegender Fehler 283

<dl> <dt>

<span id="_283__Fatal_____fileName___line_____Unrecognized_character__character__in_input_"></span><span id="_283__fatal_____filename___line_____unrecognized_character__character__in_input_"></span><span id="_283__FATAL_____FILENAME___LINE_____UNRECOGNIZED_CHARACTER__CHARACTER__IN_INPUT_"></span>**<283, schwerwiegender>: " <fileName> : <Zeilen \# \#>: Unbekanntes Zeichen <character> in Eingabe"**
</dt> <dd>

In der Eingabe ist ein unbekanntes Zeichen vorhanden. In der Regel werden unbekannte Zeichen ignoriert, und diese Fehlermeldung meldet das erste unbekannte Zeichen.

</dd> </dl>
