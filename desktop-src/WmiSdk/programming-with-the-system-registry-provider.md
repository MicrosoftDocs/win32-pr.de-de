# <a name="programming-with-the-system-registry-provider"></a>Programmieren mit dem Systemregistrierungsanbieter

In den folgenden Themen wird beschrieben, wie Anwendungen den Systemregistrierungsanbieter verwenden können, um Daten für klassen bereitzustellen, die Sie erstellen.



| Thema                                                                                                                      | Beschreibung                                                                                                                                                                                                                    |
|----------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [Definieren von Klassen für den Systemregistrierungsanbieter](defining-classes-for-the-system-registry-provider.md)                 | Für den Zugriff auf die Systemregistrierung müssen Sie Windows Klassen erstellen, die Registrierungsinformationen im WMI-Repository (Windows Management Instrumentation) speichern.                                                                     |
| [Verwenden des Systemregistrierungsanbieters als Eigenschaftenanbieter](using-the-system-registry-provider-as-a-property-provider.md) | Der Systemregistrierungsanbieter erfordert, dass Sie mehrere Qualifizierer verwenden, um Registrierungseigenschaften anzuzeigen.                                                                                                                             |
| [Beschreiben einer Ressource für die Registrierung](describing-a-resource-for-the-registry.md)                                       | Sie können auf Daten in der Registrierung zugreifen, die eine Systemressource beschreiben. Um die Informationen zu ändern, benötigt der Systemregistrierungsanbieter eine bestimmte Syntax.                                                                      |
| [Zugreifen auf einen unbenannten Registrierungswert](accessing-an-unnamed-registry-value.md)                                             | Um auf die meisten Registrierungswerte zuzugreifen, erstellen Sie eine Klasse, um die Werte in WMI zu empfangen, oder Sie können bestimmte Registrierungseigenschaften untersuchen. Der Systemregistrierungsanbieter erfordert jedoch eine spezielle Syntax, um unbenannte Registrierungswerte anzuzeigen. |
