# <a name="s-wmi"></a>S (WMI)

[A](gloss-a.md) B [C](gloss-c.md) [D](gloss-d.md) [E](gloss-e.md) [F G](gloss-f.md) [H](gloss-h.md) [I](gloss-i.md) J [K](gloss-k.md) [L](gloss-l.md) [M](gloss-m.md) [N](gloss-n.md) [O](gloss-o.md) [P](gloss-p.md) [Q](gloss-q.md) [R](gloss-r.md) S T [U](gloss-t.md) V [W](gloss-w.md) X Y Z

<dl> <dt>

<span id="wmi.gloss_schema"></span><span id="WMI.GLOSS_SCHEMA"></span>**Schema**
</dt> <dd>

Eine Auflistung von Klassendefinitionen, die [*verwaltete Objekte*](gloss-m.md) in einer bestimmten Umgebung beschreiben.

</dd> <dt>

<span id="wmi.gloss_security_descriptor"></span><span id="WMI.GLOSS_SECURITY_DESCRIPTOR"></span>**Sicherheitsbeschreibung**
</dt> <dd>

Eine Datenstruktur, die die Sicherheitsinformationen für ein sicherungsfähiges Objekt enthält, z. B. eine Freigabe, Datei, Senke oder einen Ereignisfilter.

</dd> <dt>

<span id="wmi.gloss_security_identifier"></span><span id="WMI.GLOSS_SECURITY_IDENTIFIER"></span>**Sicherheits-ID (SID)**
</dt> <dd>

Eine Datenstruktur, die Benutzer-, Gruppen- und Computerkonten identifiziert.

</dd> <dt>

<span id="wmi.gloss_semi_synchronous"></span><span id="WMI.GLOSS_SEMI_SYNCHRONOUS"></span>**semisynchron**
</dt> <dd>

Siehe *semisynchrone Methode*.

</dd> <dt>

<span id="wmi.gloss_semi__synchronous"></span><span id="WMI.GLOSS_SEMI__SYNCHRONOUS"></span>**semisynchron**
</dt> <dd>

Siehe *semisynchrone Methode*.

</dd> <dt>

<span id="wmi.gloss_semisynchronous"></span><span id="WMI.GLOSS_SEMISYNCHRONOUS"></span>**semisynchrone Methode**
</dt> <dd>

Ein Methodenaufruf, der sofort zurückgegeben wird und es der Anwendung oder dem Skript ermöglicht, die zurückgegebenen Objekte als Auflistung aufzählen.

</dd> <dt>

<span id="gloss_servicesid"></span><span id="GLOSS_SERVICESID"></span>**Dienst-SID**
</dt> <dd>

Eine eindeutige *SID,* die für jeden Sicherheitskontext erstellt wird. das heißt, eine für "LocalService" und eine für "NetworkService". Nur die *Dienst-SID* verfügt über Berechtigungen für Ressourcen wie Threads und Ereignisse, die vom WMI-Anbieterhostprozess (WMI Provider Host Process, wmiprvse.exe) erstellt werden.

</dd> <dt>

<span id="wmi.gloss_sid"></span><span id="WMI.GLOSS_SID"></span>**Sid**
</dt> <dd>

Weitere Informationen *finden Sie unter Sicherheits-ID.*

</dd> <dt>

<span id="wmi.gloss_singleton_class"></span><span id="WMI.GLOSS_SINGLETON_CLASS"></span>**Singleton-Klasse**
</dt> <dd>

Eine WMI-Klasse, die eine einzelne Instanz unterstützt.

</dd> <dt>

<span id="wmi.gloss_sink"></span><span id="WMI.GLOSS_SINK"></span>**Waschbecken**
</dt> <dd>

Ein COM-Objekt, das als physisches Übermittlungsziel für die Ergebnisse eines asynchronen Vorgangs oder einer Ereignisbenachrichtigung fungiert. Eine von einem permanenten Consumer [*implementierte Senke*](gloss-p.md) unterstützt die [**IWbemUnboundObjectSink-Schnittstelle.**](/windows/desktop/api/Wbemprov/nn-wbemprov-iwbemunboundobjectsink) Eine Senke, die von einem [*temporären Consumer oder*](gloss-t.md) von Anwendungen implementiert wird, die asynchrone Aufrufe ausführen, unterstützt die [**IWbemObjectSink-Schnittstelle.**](iwbemobjectsink.md)

Skriptclients können das [**SWbemSink-Objekt**](swbemsink.md) und Ereignisse wie [**OnObjectReady**](swbemsink-onobjectready.md)verwenden, um Rückrufe zu empfangen, die sich aus asynchronen Aufrufen ergeben.

</dd> <dt>

<span id="wmi.gloss_standard_consumer"></span><span id="WMI.GLOSS_STANDARD_CONSUMER"></span>**Standard-Consumer**
</dt> <dd>

Ein vorinstallierter permanenter Consumers, der eine Aktion ausführt, z. B. das Senden einer E-Mail oder das Schreiben in ein Protokoll, wenn dies durch eine [*Managed Object Format-Datei (MOF)*](gloss-m.md) oder ein Skript konfiguriert wird. [](gloss-p.md)

</dd> <dt>

<span id="wmi.gloss_standard_provider"></span><span id="WMI.GLOSS_STANDARD_PROVIDER"></span>**Standardanbieter**
</dt> <dd>

Ein [*in*](gloss-p.md) WMI integrierter Anbieter, der sich von einem Drittanbieter oder benutzerdefinierten Anbieter unterscheiden kann.

</dd> <dt>

<span id="wmi.gloss_standard_qualifier"></span><span id="WMI.GLOSS_STANDARD_QUALIFIER"></span>**Standardqualifizierer**
</dt> <dd>

Ein [*Qualifizierer,*](gloss-q.md) [*CIM-Objekt-Manager*](gloss-c.md) automatisch an eine Klasse, Instanz, Eigenschaft, Methode oder einen Parameter einer Methode angefügt wird.

</dd> <dt>

<span id="wmi.gloss_standard_schema"></span><span id="WMI.GLOSS_STANDARD_SCHEMA"></span>**Standardschema**
</dt> <dd>

Ein allgemeines konzeptionelles Framework, das die Klassen organisiert und verbindet, die den aktuellen Betriebszustand eines Systems, Netzwerks oder einer Anwendung darstellen. Die [*Distributed Management Task Force (DMTF)*](gloss-d.md) definiert das Standardschema im Common Information Model [*(CIM).*](gloss-c.md)

</dd> <dt>

<span id="wmi.gloss_static_class"></span><span id="WMI.GLOSS_STATIC_CLASS"></span>**statische Klasse**
</dt> <dd>

Eine CIM-Klassendefinition, die sich selten ändert und im [*CIM-Repository*](gloss-c.md) gespeichert wird, bis sie explizit gelöscht wird. Die [*CIM-Objekt-Manager*](gloss-c.md) kann eine Definition einer statischen Klasse ohne anbieter [*bereitstellen.*](gloss-p.md) Eine statische Klasse kann statische *oder* dynamische [*Instanzen unterstützen.*](gloss-d.md)

</dd> <dt>

<span id="wmi.gloss_static_instance"></span><span id="WMI.GLOSS_STATIC_INSTANCE"></span>**Statische Instanz**
</dt> <dd>

Eine Instanz einer CIM-Klasse, die dauerhaft im [*CIM-Repository*](gloss-c.md) gespeichert wird und bis zum expliziten Löschen gültig bleibt, auch bei Systemneustarts.

</dd> <dt>

<span id="wmi.gloss_static_method"></span><span id="WMI.GLOSS_STATIC_METHOD"></span>**statische Methode**
</dt> <dd>

Eine Methode, die in einer CIM-Klasse definiert ist, die durch Abrufen der Klassendefinition statt durch Abrufen einer Instanz der -Klasse ausgeführt wird.

</dd> <dt>

<span id="wmi.gloss_subschema"></span><span id="WMI.GLOSS_SUBSCHEMA"></span>**subschema**
</dt> <dd>

Ein Teil eines *Schemas,* das sich im Besitz einer bestimmten Organisation befindet. Das [*Win32schema*](gloss-w.md) ist ein Beispiel für ein Unterschema.

</dd> <dt>

<span id="wmi.gloss_system_class"></span><span id="WMI.GLOSS_SYSTEM_CLASS"></span>**Systemklasse**
</dt> <dd>

Eine Klasse, die vom [*CIM-Objekt-Manager*](gloss-c.md) definiert wird, um Kernfeatures wie Ereignisbenachrichtigung, Sicherheit und Lokalisierung zu unterstützen. Eine Systemklasse wird automatisch in jedem Namespace [*definiert.*](gloss-n.md) Eine Systemklasse wird von der abstrakten Basisklasse [**\_ \_ SystemClass ableiten.**](--systemclass.md)

</dd> <dt>

<span id="wmi.gloss_system_monitor"></span><span id="WMI.GLOSS_SYSTEM_MONITOR"></span>**Systemmonitor**
</dt> <dd>

Ein (CIM)-Hilfsprogramm, das die grafische Benutzeroberfläche für die Leistungsüberwachung ist.

</dd> <dt>

<span id="wmi.gloss_system_property"></span><span id="WMI.GLOSS_SYSTEM_PROPERTY"></span>**Systemeigenschaft**
</dt> <dd>

Eine Eigenschaft, die [*CIM-Objekt-Manager*](gloss-c.md) definiert, um Informationen für die einzelnen Klassen zu liefern, z. B. Name, Ableitung und [*Namespace.*](gloss-n.md)

</dd> </dl>
