# <a name="s-wmi"></a>S (WMI)

<dl> <dt>

<span id="wmi.gloss_schema"></span><span id="WMI.GLOSS_SCHEMA"></span>**Chaos**
</dt> <dd>

Eine Auflistung von Klassendefinitionen, die [*verwaltete Objekte*](gloss-m.md) in einer bestimmten Umgebung beschreiben.

</dd> <dt>

<span id="wmi.gloss_security_descriptor"></span><span id="WMI.GLOSS_SECURITY_DESCRIPTOR"></span>**Sicherheits Beschreibung**
</dt> <dd>

Eine Datenstruktur, die die Sicherheitsinformationen für ein Sicherungs fähiges Objekt enthält, z. b. eine Freigabe, eine Datei, eine Senke oder ein Ereignis Filter.

</dd> <dt>

<span id="wmi.gloss_security_identifier"></span><span id="WMI.GLOSS_SECURITY_IDENTIFIER"></span>**Sicherheits-ID (SID)**
</dt> <dd>

Eine Datenstruktur, mit der Benutzer-, Gruppen-und Computer Konten identifiziert werden.

</dd> <dt>

<span id="wmi.gloss_semi_synchronous"></span><span id="WMI.GLOSS_SEMI_SYNCHRONOUS"></span>**SemiSynchron**
</dt> <dd>

Siehe *semisynchrone Methode*.

</dd> <dt>

<span id="wmi.gloss_semi__synchronous"></span><span id="WMI.GLOSS_SEMI__SYNCHRONOUS"></span>**SemiSynchron**
</dt> <dd>

Siehe *semisynchrone Methode*.

</dd> <dt>

<span id="wmi.gloss_semisynchronous"></span><span id="WMI.GLOSS_SEMISYNCHRONOUS"></span>**semisynchrone Methode**
</dt> <dd>

Ein Methodenaufrufe, der sofort zurückgegeben wird und der Anwendung oder dem Skript das Auflisten der zurückgegebenen Objekte als Auflistung ermöglicht.

</dd> <dt>

<span id="gloss_servicesid"></span><span id="GLOSS_SERVICESID"></span>**Dienst-SID**
</dt> <dd>

Eine eindeutige *sid* , die für jeden Sicherheitskontext erstellt wird. Das heißt, eine für "LocalService" und eine für "Network Service". Nur die *Dienst-SID* verfügt über Berechtigungen für Ressourcen wie Threads und Ereignisse, die vom WMI-Anbieter Host Prozess (wmiprvse.exe) erstellt werden.

</dd> <dt>

<span id="wmi.gloss_sid"></span><span id="WMI.GLOSS_SID"></span>**SID**
</dt> <dd>

Siehe *Sicherheits*-ID.

</dd> <dt>

<span id="wmi.gloss_singleton_class"></span><span id="WMI.GLOSS_SINGLETON_CLASS"></span>**Singleton-Klasse**
</dt> <dd>

Eine WMI-Klasse, die eine einzelne-Instanz unterstützt.

</dd> <dt>

<span id="wmi.gloss_sink"></span><span id="WMI.GLOSS_SINK"></span>**Körper**
</dt> <dd>

Ein COM-Objekt, das als Ziel für die physische Übermittlung für die Ergebnisse eines asynchronen Vorgangs oder einer Ereignis Benachrichtigung fungiert. Eine von einem [*permanenten Consumer*](gloss-p.md) implementierte Senke unterstützt die [**iwbemunboundobjectsink**](/windows/desktop/api/Wbemprov/nn-wbemprov-iwbemunboundobjectsink) -Schnittstelle. Eine von einem [*temporären Consumer*](gloss-t.md) oder Anwendungen implementierte Senke, die asynchrone Aufrufe durchführen, unterstützt die [**iwbebobjectsink**](iwbemobjectsink.md) -Schnittstelle.

Skript Erstellungs Clients können [**das-**](swbemsink.md) Objekt und Ereignisse (z. b. [**onobjectready**](swbemsink-onobjectready.md)) von-Skripts verwenden, um Rückrufe zu empfangen, die durch asynchrone Aufrufe entstehen.

</dd> <dt>

<span id="wmi.gloss_standard_consumer"></span><span id="WMI.GLOSS_STANDARD_CONSUMER"></span>**Standard Consumer**
</dt> <dd>

Ein vorinstallierter [*dauerhafter*](gloss-p.md) Consumer, der eine Aktion ausführt, z. b. das Senden einer e-Mail-Nachricht oder das Schreiben in ein Protokoll, wenn diese von einer [*Managed Object Format Datei (MOF)*](gloss-m.md) oder einem Skript konfiguriert werden

</dd> <dt>

<span id="wmi.gloss_standard_provider"></span><span id="WMI.GLOSS_STANDARD_PROVIDER"></span>**Standardanbieter**
</dt> <dd>

Ein [*Anbieter*](gloss-p.md) , der in WMI integriert ist, der sich von einem Drittanbieter oder einem benutzerdefinierten Anbieter unterscheidet.

</dd> <dt>

<span id="wmi.gloss_standard_qualifier"></span><span id="WMI.GLOSS_STANDARD_QUALIFIER"></span>**Standard Qualifizierer**
</dt> <dd>

Ein [*Qualifizierer*](gloss-q.md) , der vom [*CIM-Objekt-Manager*](gloss-c.md) automatisch an eine Klasse, eine Instanz, eine Eigenschaft, eine Methode oder einen Parameter einer Methode angefügt wird.

</dd> <dt>

<span id="wmi.gloss_standard_schema"></span><span id="WMI.GLOSS_STANDARD_SCHEMA"></span>**Standardschema**
</dt> <dd>

Ein gängiges konzeptionelles Framework, mit dem die Klassen organisiert und verknüpft werden, die den aktuellen Betriebszustand eines Systems, Netzwerks oder einer Anwendung darstellen. Die [*verteilte Verwaltungs Task Force (DMTF)*](gloss-d.md) definiert das Standardschema im [*Common Information Model (CIM)*](gloss-c.md).

</dd> <dt>

<span id="wmi.gloss_static_class"></span><span id="WMI.GLOSS_STATIC_CLASS"></span>**statische Klasse**
</dt> <dd>

Eine CIM-Klassendefinition, die selten geändert wird und im [*CIM-Repository*](gloss-c.md) gespeichert wird, bis Sie explizit gelöscht wird. Der [*CIM-Objekt-Manager*](gloss-c.md) kann eine Definition einer statischen Klasse ohne [*Anbieter*](gloss-p.md)bereitstellen. Eine statische Klasse kann entweder *statische* oder [*dynamische Instanzen*](gloss-d.md)unterstützen.

</dd> <dt>

<span id="wmi.gloss_static_instance"></span><span id="WMI.GLOSS_STATIC_INSTANCE"></span>**statische Instanz**
</dt> <dd>

Eine Instanz einer CIM-Klasse, die dauerhaft im CIM- [*Repository*](gloss-c.md) gespeichert wird und bis zum expliziten Löschen gültig bleibt, auch über Systemneustarts.

</dd> <dt>

<span id="wmi.gloss_static_method"></span><span id="WMI.GLOSS_STATIC_METHOD"></span>**statische Methode**
</dt> <dd>

Eine Methode, die in einer CIM-Klasse definiert wird, die durch Abrufen der Klassendefinition statt durch Abrufen einer Instanz der-Klasse ausgeführt wird.

</dd> <dt>

<span id="wmi.gloss_subschema"></span><span id="WMI.GLOSS_SUBSCHEMA"></span>**unter Schema**
</dt> <dd>

Ein Teil eines *Schemas* , das im Besitz einer bestimmten Organisation ist. Der [*Win32schema*](gloss-w.md) ist ein Beispiel für ein unter Schema.

</dd> <dt>

<span id="wmi.gloss_system_class"></span><span id="WMI.GLOSS_SYSTEM_CLASS"></span>**System Klasse**
</dt> <dd>

Eine Klasse, die der [*CIM-Objekt-Manager*](gloss-c.md) definiert, um Kernfunktionen zu unterstützen, z. b. Ereignis Benachrichtigung, Sicherheit und Lokalisierung. Eine System Klasse wird automatisch in jedem [*Namespace*](gloss-n.md)definiert. Eine System Klasse wird von der abstrakten Basisklasse [**\_ \_ System Class**](--systemclass.md)abgeleitet.

</dd> <dt>

<span id="wmi.gloss_system_monitor"></span><span id="WMI.GLOSS_SYSTEM_MONITOR"></span>**System Monitor**
</dt> <dd>

Ein (CIM)-Hilfsprogramm, das die GUI für die Leistungsüberwachung ist.

</dd> <dt>

<span id="wmi.gloss_system_property"></span><span id="WMI.GLOSS_SYSTEM_PROPERTY"></span>**System Eigenschaft**
</dt> <dd>

Eine Eigenschaft, die von der [*CIM-Objekt-Manager*](gloss-c.md) definiert wird, um Informationen bereitzustellen, die für die einzelnen Klassen gelten, z. b. Name, Ableitung und [*Namespace*](gloss-n.md).

</dd> </dl>
