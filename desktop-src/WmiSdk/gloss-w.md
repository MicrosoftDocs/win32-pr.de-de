# <a name="w-wmi"></a>W (WMI)

[A](gloss-a.md) B [C](gloss-c.md) [D](gloss-d.md) [E](gloss-e.md) [F](gloss-f.md) G [H](gloss-h.md) [I](gloss-i.md) J [K](gloss-k.md) [L](gloss-l.md) [M](gloss-m.md) [N](gloss-n.md) [O](gloss-o.md) [P](gloss-p.md) [Q](gloss-q.md) [R](gloss-r.md) [S](gloss-s.md) [T](gloss-t.md) U V W X Y Z

<dl> <dt>

<span id="wmi.gloss_win32_schema"></span><span id="WMI.GLOSS_WIN32_SCHEMA"></span>**Win32-Schema**
</dt> <dd>

Eine Microsoft-Erweiterung des [*CIM-Schemas,*](gloss-c.md) die Definitionen von Klassen und Instanzen enthält, um [*verwaltete Objekte*](gloss-m.md) darzustellen, die in einer typischen Windows-Umgebung vorhanden sind.

</dd> <dt>

<span id="wmi.gloss_wmi_class"></span><span id="WMI.GLOSS_WMI_CLASS"></span>**WMI-Klasse**
</dt> <dd>

Eine Vorlage für einen Typ von [*verwaltetem Objekt.*](gloss-m.md) WMI-Klassen definieren die Grundeinheiten der Verwaltung.

</dd> <dt>

<span id="wmi.gloss_wmi_class_provider"></span><span id="WMI.GLOSS_WMI_CLASS_PROVIDER"></span>**WMI-Klassenanbieter**
</dt> <dd>

Ein COM-Server, der Klassendefinitionen zur Verfügung stellt. Klassenanbieter implementieren Methoden in der [**IWbemServices-Schnittstelle,**](/windows/desktop/api/WbemCli/nn-wbemcli-iwbemservices) um das Abrufen, Ändern, Löschen, Aufzählen und Abfragen von Daten zu unterstützen.

</dd> <dt>

<span id="wmi.gloss_wmi_control"></span><span id="WMI.GLOSS_WMI_CONTROL"></span>**WMI-Steuerelement**
</dt> <dd>

Ein mmc-Snap-In (Microsoft Management Console), mit dem ein Benutzer WMI-Einstellungen auf einem lokalen oder Remotecomputer konfigurieren kann.

</dd> <dt>

<span id="wmi.gloss_wmi_for_windows_driver_model"></span><span id="WMI.GLOSS_WMI_FOR_WINDOWS_DRIVER_MODEL"></span>**WMI für Windows-Treibermodell (WDM)**
</dt> <dd>

Eine Komponente auf niedriger Ebene, die Zugriff zwischen WMI und Hardwaregeräten bietet.

</dd> <dt>

<span id="wmi.gloss_wmi_odbc_adapter"></span><span id="WMI.GLOSS_WMI_ODBC_ADAPTER"></span>**WMI ODBC-Adapter**
</dt> <dd>

Eine Schnittstelle, die es einer Anwendung ermöglicht, Mithilfe von WMI Query Language Informationen aus *Windows* Management abzurufen. Dies bietet eine Standard-API, mit der ODBC-basierte Anwendungen die Daten im [*CIM-Repository*](gloss-c.md) wie eine Datenbank verwenden können.

</dd> <dt>

<span id="wmi.gloss_wmi_query_language"></span><span id="WMI.GLOSS_WMI_QUERY_LANGUAGE"></span>**WMI Query Language (WQL)**
</dt> <dd>

Eine Teilmenge der strukturierten Abfragesprache (SQL) mit Erweiterungen zur Unterstützung von WMI-Ereignisbenachrichtigungen und anderen WMI-spezifischen Features.

</dd> <dt>

<span id="wmi.gloss_wmi_method"></span><span id="WMI.GLOSS_WMI_METHOD"></span>**WMI-Methode**
</dt> <dd>

Eine Methode, die von der [WMI-COM-API](com-api-for-wmi.md) oder der [Skript-API](scripting-api-for-wmi.md)implementiert wird. Zwei Beispiele sind die COM-Methode [**IWbemServices::ExecMethod**](/windows/desktop/api/WbemCli/nf-wbemcli-iwbemservices-execmethod) und die ähnliche Skript-API-Methode [**SWbemServices.ExecMethod**](swbemservices-execmethod.md).

</dd> <dt>

<span id="wmi.gloss_wmi_repository"></span><span id="WMI.GLOSS_WMI_REPOSITORY"></span>**WMI-Repository**
</dt> <dd>

Ein zentraler Speicherbereich, der vom [*CIM-Objekt-Manager*](gloss-c.md)verwaltet wird.

</dd> <dt>

<span id="wmi.gloss_wql"></span><span id="WMI.GLOSS_WQL"></span>**Wql**
</dt> <dd>

Weitere Informationen finden Sie *unter WMI Query Language (WQL).*

</dd> </dl>
