# <a name="graphics-low-level-client-support"></a>Unterstützung von Grafiken Low-Level Clients

\[Diese Funktionen können bei jeder Betriebssystem Revision geändert werden. Verwenden Sie stattdessen Microsoft DirectDraw und Microsoft Direct3DAPIs; Diese APIs isolieren Anwendungen vor solchen Betriebssystem Änderungen und verbergen viele andere Schwierigkeiten bei der direkten Interaktion mit Anzeige Treibern. Weitere Informationen finden Sie unter [Einführung in die Anzeige](/windows-hardware/drivers/display/introduction-to-display--windows-2000-model-).\]

DirectDraw und Direct3D verwenden einige kernelmodusroutinen, um mit dem Betriebssystem und dem Anzeigetreiber zu kommunizieren.

Der Betriebssystem-Kernel scheint ein DirectDraw/Direct3D-Anzeigetreiber aus der Sicht der Benutzermoduskomponenten zu sein. Es gibt jedoch einige Unterschiede zwischen Kernel-und Anzeige Treibern. Die Kernelmoduskomponenten bewahren eine Kopie von Strukturen auf, die DirectDraw/Direct3D-Objekte darstellen, z. b. Geräte und Oberflächen. Die Benutzermoduskomponente von DirectDraw/Direct3D fordert die Erstellung dieser Strukturen an, die mit Handles bezeichnet werden, die vom Kernel Modus zurückgegeben werden. Folglich sind die Unterschiede größtenteils in den Argumenten, die an die Routinen übergeben werden: der Kernel akzeptiert Handles, wohingegen Treiber normalerweise Zeiger akzeptieren.

## <a name="evolution-of-kernel-mode-entry-points"></a>Weiterentwicklung von kernelmoduseinstiegs Punkten

DirectDraw stellt alle seine Verbindungen mit dem Kernel Modus über den Gdi32.dll mithilfe mehrerer Einstiegspunkte her. Diese Einstiegspunkte Mars Hallen in der Regel nur Parameter. Im Rahmen der Geräte Erstellung gibt der Gdi32.dll Zeiger auf Routinen innerhalb von sich selbst zurück, die dann von DirectDraw und Direct3D 7,0 verwendet werden.

Direct3D 8. *x* sorgt für eine saubere Unterbrechung von Gdi32.dll und ruft den kernelmoduseinstiegs Punkt direkt auf.

## <a name="calling-the-kernel"></a>Aufrufen des Kernels

Diese Einstiegspunkte sind ähnlich und in vielen Fällen identisch mit den Einstiegspunkten im Anzeigetreiber selbst. Daher ist ein Verständnis der Informationen zum Treiber Development Kit (DDK) für DirectDraw und Direct3D eine wesentliche Voraussetzung für die Verwendung dieser Funktionen.

Das Aufrufen dieser Funktionen unterscheidet sich zwischen Betriebssystemen. Unter Windows 2000, W2KUMODE. Die lib-Bibliothek enthält Routinen, mit denen der aufrufenden Thread den Übergang zum Kernel Modus durchführen kann. Diese Routinen machen keinen Parameter Marshalling, und sind der ideale Aufruf Mechanismus unter Windows 2000. Der Mechanismus, der den Übergang zum Kernel Modus durchführt, basiert auf der Reihenfolge einer Tabelle. Diese Reihenfolge ändert sich bei der Überarbeitung von Betriebssystemen. Daher empfiehlt es sich nicht, eine Anwendung zu erstellen, die auf W2KUMODE basiert. LIB für Windows 2000 und erwarten, dass Sie unter Windows XP unverändert ausgeführt wird. Im folgenden Abschnitt wird ein Mechanismus zum Erstellen der Portabilität von Betriebssystemen beschrieben.

Unter Windows XP exportiert D3D8THK.DLL alle kernelmodusfunktionen, jedoch mit einem leicht ergänzten Namen. Im folgenden Beispiel wird gezeigt, wie Anwendungen die Low-Level-Client Funktionen mithilfe des Präprozessors Alias ausführen können. Bevor Sie dies tun, verknüpfen Sie dynamisch mit D3D8THK.DLL.


```

#define NtGdiDdUnlock OsThunkDdUnlock
#include "ntgdi.h"      // Brings in the definition of "NtGdiDdUnlock", 
.                       //   which is now seen as OsThunkDdUnlock.
.

OsThunkDdUnlock(hSurface, puUnlockData);
```



## <a name="writing-portable-applications"></a>Schreiben von portablen Anwendungen

Es stehen mehrere Verfahren zur Verfügung, mit denen eine Anwendung über Betriebssysteme hinweg portabel ist. Die stabilste besteht darin, das Redistributable-Paket Microsoft DirectX 8. x unter Windows 2000 als Teil des Installationsprozesses ihrer eigenen Anwendung zu installieren. Dadurch wird sichergestellt, dass D3D8THK.DLL auf Windows 2000-Systemen verfügbar ist und einen einzelnen Codepfad in der Anwendung zulässt, da Sie das gleiche Verfahren wie oben beschrieben für Windows XP verwenden können.

Anwendungen können auch Ihre eigene isolierende dll erstellen und zwei Versionen dieser DLL implementieren, eine für Windows 2000 mithilfe der oben gezeigten Methode und eine für Windows XP, die D3D8THK.DLL verwendet.

## <a name="device-creation"></a>Geräte Erstellung

DirectDraw und Direct3D müssen zunächst eine Instanz der Abstraktion des Treibers des Treiber Objekts erstellen. In Gdi32.dll und Ddraw.dll wird dies erreicht, indem [**ddcreatedirectdrawobject**](/windows/desktop/api/Ddrawgdi/nf-ddrawgdi-ddcreatedirectdrawobject), [**ddquerydirectdrawobject**](/windows/desktop/api/Ddrawgdi/nf-ddrawgdi-ddquerydirectdrawobject) und [**ddreenabledirectdrawobject**](/windows/desktop/api/Ddrawgdi/nf-ddrawgdi-ddreenabledirectdrawobject)aufgerufen werden. In Direct3D 8. *x*, dies wird durch Aufrufen von [**ntgdiddcreatedirectdrawobject**](-dxgkernel-ntgdiddcreatedirectdrawobject.md), [**ntgdiddquerydirectdrawobject**](-dxgkernel-ntgdiddquerydirectdrawobject.md) und [**ntgdiddreenabledirectdrawobject**](-dxgkernel-ntgdiddreenabledirectdrawobject.md)erreicht. Beachten Sie, dass die gdientry \* -Einstiegspunkte nur Marshallern für die kernelmoduseinstiegs Punkte sind.

Nachdem diese Funktionen aufgerufen wurden, wird ein Handle zurückgegeben, das die Abstraktion des Kernel Objekts eines DirectDraw-Geräts darstellt. Dieses Kernel Objekt stellt eine Treiber Instanz dar, die im Besitz des Anzeige Treibers ist. Dieses Handle kann in nachfolgenden Aufrufen verwendet werden, wie z. b. [**ddkreatesurfaceobject**](/windows/desktop/api/Ddrawgdi/nf-ddrawgdi-ddcreatesurfaceobject) (oder [**ntgdiddkreatesurfaceobject**](-dxgkernel-ntgdiddcreatesurfaceobject.md) in Direct3D 8).*x*), um weitere Objekte zu erstellen.

## <a name="kernel-objects"></a>Kernel Objekte

Einige Einstiegspunkte werden verwendet, um die Kernel Modus-Darstellungen bestimmter Objekte zu verwalten. [**Ddkreatesurfaceobject**](/windows/desktop/api/Ddrawgdi/nf-ddrawgdi-ddcreatesurfaceobject) (oder [**ntgdiddkreatesurfaceobject**](-dxgkernel-ntgdiddcreatesurfaceobject.md) in Direct3D 8.*x*) ist ein gutes Beispiel. Dieser Einstiegspunkt erstellt ein kernelmodusobjekt, das einen Teil des System Speichers darstellt. Der Einstiegspunkt gibt ein Handle zurück, und dieses Handle kann in nachfolgenden Zeichnungs Aufrufen verwendet werden. Der Kernel leitet einen Zeiger auf seine eigene Struktur von diesen Handles ab und übergibt diesen Zeiger an den Anzeigetreiber, der dann den Vorgang ausführt.
