# <a name="address-windowing-extensions"></a><span data-ttu-id="842ce-103">Address windowingextensions</span><span class="sxs-lookup"><span data-stu-id="842ce-103">Address Windowing Extensions</span></span>

<span data-ttu-id="842ce-104">Bei Address windowingextensions (AWE) handelt es sich um einen Satz von Erweiterungen, mit denen eine Anwendung den physischen Speicher, der größer als 4 GB ist, schnell bearbeiten kann</span><span class="sxs-lookup"><span data-stu-id="842ce-104">Address Windowing Extensions (AWE) is a set of extensions that allows an application to quickly manipulate physical memory greater than 4GB.</span></span> <span data-ttu-id="842ce-105">Bestimmte datenintensive Anwendungen, wie z. b. Datenbankverwaltungssysteme und wissenschaftliche und technische Software, benötigen Zugriff auf sehr große Daten Caches.</span><span class="sxs-lookup"><span data-stu-id="842ce-105">Certain data-intensive applications, such as database management systems and scientific and engineering software, need access to very large caches of data.</span></span> <span data-ttu-id="842ce-106">Im Fall von sehr großen Datasets ist das Einschränken des Caches auf den Zeitraum von 2 GB des Benutzer Adressraums einer Anwendung eine schwerwiegende Einschränkung.</span><span class="sxs-lookup"><span data-stu-id="842ce-106">In the case of very large data sets, restricting the cache to fit within an application's 2GB of user address space is a severe restriction.</span></span> <span data-ttu-id="842ce-107">In diesen Fällen ist der Cache zu klein, um die Anwendung ordnungsgemäß zu unterstützen.</span><span class="sxs-lookup"><span data-stu-id="842ce-107">In these situations, the cache is too small to properly support the application.</span></span>

<span data-ttu-id="842ce-108">AWE löst dieses Problem, indem es Anwendungen ermöglicht, große Mengen an Arbeitsspeicher direkt zu adressieren und dabei weiterhin 32-Bit-Zeiger zu verwenden.</span><span class="sxs-lookup"><span data-stu-id="842ce-108">AWE solves this problem by allowing applications to directly address huge amounts of memory while continuing to use 32-bit pointers.</span></span> <span data-ttu-id="842ce-109">AWE ermöglicht es Anwendungen, Daten Caches mit einer Größe von mehr als 4 GB zu haben (wobei ausreichend physischer Arbeitsspeicher vorhanden ist).</span><span class="sxs-lookup"><span data-stu-id="842ce-109">AWE allows applications to have data caches larger than 4GB (where sufficient physical memory is present).</span></span> <span data-ttu-id="842ce-110">AWE verwendet physische nicht auslagerbare Speicher-und Fenster Sichten verschiedener Teile dieses physischen Speichers innerhalb eines virtuellen 32-Bit-Adressraums.</span><span class="sxs-lookup"><span data-stu-id="842ce-110">AWE uses physical nonpaged memory and window views of various portions of this physical memory within a 32-bit virtual address space.</span></span>

<span data-ttu-id="842ce-111">AWE weist einige Einschränkungen hinsichtlich der Verwendung dieses Arbeitsspeichers auf, primär weil diese Einschränkungen eine extrem schnelle Zuordnung, Neuzuordnung und Freigabe ermöglichen.</span><span class="sxs-lookup"><span data-stu-id="842ce-111">AWE places a few restrictions on how this memory may be used, primarily because these restrictions allow extremely fast mapping, remapping, and freeing.</span></span> <span data-ttu-id="842ce-112">Die schnelle Speicherverwaltung ist für diese potenziell riesigen Adressräume von Bedeutung.</span><span class="sxs-lookup"><span data-stu-id="842ce-112">Fast memory management is important for these potentially enormous address spaces.</span></span>

-   <span data-ttu-id="842ce-113">Virtuelle Adressbereiche, die für AWE reserviert sind, können nicht mit anderen Prozessen verwendet werden (und sind daher nicht vererbbar).</span><span class="sxs-lookup"><span data-stu-id="842ce-113">Virtual address ranges allocated for the AWE are not sharable with other processes (and therefore not inheritable).</span></span> <span data-ttu-id="842ce-114">Tatsächlich sind zwei verschiedene virtuelle AWE-Adressen innerhalb desselben Prozesses nicht berechtigt, dieselbe physische Seite zuzuordnen.</span><span class="sxs-lookup"><span data-stu-id="842ce-114">In fact, two different AWE virtual addresses within the same process are not allowed to map the same physical page.</span></span> <span data-ttu-id="842ce-115">Diese Einschränkungen ermöglichen eine schnelle Neuzuordnung und Bereinigung, wenn der Arbeitsspeicher freigegeben wird.</span><span class="sxs-lookup"><span data-stu-id="842ce-115">These restrictions provide fast remapping and cleanup when memory is freed.</span></span>
-   <span data-ttu-id="842ce-116">Die physischen Seiten, die für eine AWE-Region zugeordnet werden können, werden durch die Anzahl der auf dem Computer vorhandenen physischen Seiten beschränkt, da dieser Arbeitsspeicher nie ausgelagert wird – er wird gesperrt, bis die Anwendung ihn explizit freigibt oder beendet.</span><span class="sxs-lookup"><span data-stu-id="842ce-116">The physical pages that can be allocated for an AWE region are limited by the number of physical pages present in the machine, since this memory is never paged – it is locked down until the application explicitly frees it or exits.</span></span> <span data-ttu-id="842ce-117">Die physischen Seiten, die für einen bestimmten Prozess reserviert werden, können in einer beliebigen virtuellen AWE-Region innerhalb desselben Prozesses zugeordnet werden.</span><span class="sxs-lookup"><span data-stu-id="842ce-117">The physical pages allocated for a given process can be mapped into any AWE virtual region within the same process.</span></span> <span data-ttu-id="842ce-118">Anwendungen, die AWE verwenden, müssen darauf achten, dass Sie nicht so viel physischen Arbeitsspeicher beanspruchen, weil Sie dazu führen, dass andere Anwendungen übermäßig viel oder weniger Ressourcen erstellt werden.</span><span class="sxs-lookup"><span data-stu-id="842ce-118">Applications that use AWE must be careful not to take so much physical memory that they cause other applications to page excessively or prevent creation of new processes or threads due to lack of resources.</span></span> <span data-ttu-id="842ce-119">Verwenden Sie die [**GlobalMemoryStatus usex**](/windows/win32/api/sysinfoapi/nf-sysinfoapi-globalmemorystatusex) -Funktion, um die Verwendung des physischen Speichers zu überwachen.</span><span class="sxs-lookup"><span data-stu-id="842ce-119">Use the [**GlobalMemoryStatusEx**](/windows/win32/api/sysinfoapi/nf-sysinfoapi-globalmemorystatusex) function to monitor physical memory use.</span></span>
-   <span data-ttu-id="842ce-120">Virtuelle AWE-Adressen sind immer Lese-/Schreibzugriff und können nicht über Aufrufe von [**VirtualProtect**](/windows/win32/api/memoryapi/nf-memoryapi-virtualprotect) geschützt werden (d. h. kein Schreib geschützter Speicher, NoAccess-Speicher, Wächter Seiten und die like-Angabe).</span><span class="sxs-lookup"><span data-stu-id="842ce-120">AWE virtual addresses are always read/write and cannot be protected via calls to [**VirtualProtect**](/windows/win32/api/memoryapi/nf-memoryapi-virtualprotect) (that is, no read-only memory, noaccess memory, guard pages, and the like can be specified).</span></span>
-   <span data-ttu-id="842ce-121">AWE-Adressbereiche können nicht verwendet werden, um Daten für Grafiken oder Videoaufrufe zu puffern.</span><span class="sxs-lookup"><span data-stu-id="842ce-121">AWE address ranges cannot be used to buffer data for graphics or video calls.</span></span>
-   <span data-ttu-id="842ce-122">Ein AWE-Speicherbereich kann nicht aufgeteilt werden und kann auch nicht gelöscht werden.</span><span class="sxs-lookup"><span data-stu-id="842ce-122">An AWE memory range cannot be split, nor can pieces of it be deleted.</span></span> <span data-ttu-id="842ce-123">Stattdessen muss der gesamte virtuelle Adressbereich als Einheit gelöscht werden, wenn ein Löschvorgang erforderlich ist.</span><span class="sxs-lookup"><span data-stu-id="842ce-123">Instead, the entire virtual address range must be deleted as a unit when deletion is required.</span></span> <span data-ttu-id="842ce-124">Dies bedeutet, dass Sie beim Aufrufen von [**VirtualFree**](/windows/win32/api/memoryapi/nf-memoryapi-virtualfree)eine Arbeitsspeicher **\_ Freigabe** angeben müssen.</span><span class="sxs-lookup"><span data-stu-id="842ce-124">This means you must specify **MEM\_RELEASE** when calling [**VirtualFree**](/windows/win32/api/memoryapi/nf-memoryapi-virtualfree).</span></span>

<span data-ttu-id="842ce-128">Diese Lösung behandelt die Probleme mit dem physischen Speicher auf sehr allgemeine, weit verbreitete Weise.</span><span class="sxs-lookup"><span data-stu-id="842ce-128">This solution addresses the physical memory issues in a very general, widely applicable manner.</span></span> <span data-ttu-id="842ce-129">Zu den Vorteilen von AWE gehören:</span><span class="sxs-lookup"><span data-stu-id="842ce-129">Some of the benefits of AWE are:</span></span>
