### <a name="expanded-api-support-for-cloud-fonts-and-custom-font-sets"></a>Erweiterte API-Unterstützung für cloudschriftarten und benutzerdefinierte Schriftart Sätze

Windows 10 enthielt APIs, mit denen apps problemlos über einen Windows-Schriftart Dienst auf Schriftarten zugreifen können. Im Windows 10 Creators Update werden APIs für Remote Schriftarten erweitert, um den einfachen Zugriff auf Schriftarten aus anderen Quellen im Web zu ermöglichen, auf die über HTTP oder HTTPS zugegriffen werden kann. 

Die neuen Remote Schriftart-APIs können mit öffentlichen oder privaten Webdiensten verwendet werden. Darüber hinaus können Sie für den Zugriff auf unformatierte, OpenType-Schriftart Dateien (. ttf,. OTF.,. TTC,. OTC) oder Schriftarten verwendet werden, die in [WOFF](https://www.w3.org/TR/WOFF/) -oder [WOFF2](https://www.w3.org/TR/WOFF2/) -Containerformaten verpackt sind. Die neuen APIs werden zusammen mit vorhandenen APIs für Warteschlangen Anforderungen zum Herunterladen von Remote Schriftart Daten und zum Verarbeiten des eigentlichen Downloadvorgangs verwendet.

Andere neue APIs vereinfachen apps das Arbeiten mit benutzerdefinierten Schriftarten, die im lokalen Dateisystem gespeichert sind oder in einen Arbeitsspeicher Puffer geladen werden.

Weitere Informationen zu neuen APIs für das Arbeiten mit Remote Schriftarten, benutzerdefinierten Schriftart Sätzen oder WOFF/WOFF2-Containerformaten finden Sie im folgenden Thema:

[Benutzerdefinierte Schriftartenkombinationen](custom-font-sets-win10.md)

Weitere Informationen finden Sie auch in den Links zu API-Referenz Themen in diesem Thema.  Die Verwendung neuer und vorhandener APIs für die Arbeit mit benutzerdefinierten Schriftarten wird auch im [Beispiel für benutzerdefinierte DirectWrite-Schriftart Sätze](https://github.com/Microsoft/Windows-classic-samples/tree/master/Samples/DirectWriteCustomFontSets/)veranschaulicht. Dieses Beispiel veranschaulicht die Code Implementierung für verschiedene verschiedene Szenarios, z. WOFF2. lokale Schriftarten auf einem Datenträger, Remote Schriftarten im Web, Schriftart Daten im Arbeitsspeicher und Schriftarten in gepackten WOFF-oder-Formaten.

### <a name="improved-support-for-color-fonts"></a>Verbesserte Unterstützung für farbige Schriftarten

Ab Windows 10 Anniversary Update bietet DirectWrite integrierte Unterstützung für eine größere Anzahl von Farb Schriftart Formaten, sodass Entwickler in ihren DirectWrite-gestützten apps mehr Arten von Schriftarten verwenden können als je zuvor. Unter anderem werden folgende Elemente unterstützt:

-   Die OpenType-Tabelle "colr", die kompakten Vektor Inhalte in Schriftarten ermöglicht. (Wird seit Windows 8.1 unterstützt.)
-   Die OpenType-Tabelle "SVG", die SVG-Inhalte in Schriftarten ermöglicht.
-   Die Tabelle "cbdt" OpenType, die Farb Bitmapinhalt in Schriftarten ermöglicht.
-   Die OpenType-Tabelle "sbix", die Farb Bitmapinhalt in Schriftarten ermöglicht.

[Direct2D](../direct2d/direct2d-portal.md), das DirectWrite für das Text Rendering verwendet, unterstützt diese Farb Schriftart Formate automatisch, wenn die [**D2D1 \_ Draw \_ Text \_ options-Option \_ \_ Color \_ Font**](/windows/win32/api/d2d1/ne-d2d1-d2d1_draw_text_options) -Flag aktivieren aktiviert ist. Weitere Informationen finden Sie unter den folgenden Themen:

-   [Farbige Schriftarten](color-fonts.md)
-   [Beispiel für DirectWrite-Farb Glyphe](https://github.com/Microsoft/Windows-universal-samples/tree/master/Samples/DWriteColorGlyph)

### <a name="support-for-adobe-typekit-and-other-font-service-clients"></a>Unterstützung für Adobe typekit und andere Schriftart Dienst Clients

Einige Schriftart Dienste, wie z. b. Adobe typekit, verfügen über Client seitige Hilfsprogramme, mit denen Benutzer Schriftarten aus dem Dienst laden und in verschiedenen Anwendungen auf Ihrem Windows-Computer verwenden können. Diese Hilfsprogramme funktionieren in der Regel durch Lauf Zeit Aufrufe an GDI zum Laden zusätzlicher Schriftarten, anstatt die Schriftarten im System dauerhaft zu installieren. Aufgrund dieses Entwurfs können die Schriftarten in früheren Windows-Versionen in GDI-basierten Anwendungen verwendet werden, aber nicht in DirectWrite-Anwendungen. Ab dem Windows 10 Anniversary Update sind Schriftarten, die von solchen Dienstprogrammen geladen werden, auch in DirectWrite und in GDI verfügbar.

Schriftarten, die von einem Dienstprogramm für Schriftart geladen werden, sind in der System Schriftart Auflistung sichtbar, die durch Aufrufen der [**idwrite-Factory:: getsystemfontcollection**](/windows/win32/api/dwrite/nf-dwrite-idwritefactory-getsystemfontcollection) -Methode abgerufen wird. Da Schriftart Dienste in der Regel einem pro-Benutzer-Lizenzierungsmodell folgen, werden die von diesen Dienstprogrammen geladenen Schriftarten auf Benutzerbasis verwaltet. Demzufolge können vorhandene DirectWrite-Anwendungen Schriftarten verwenden, die Endbenutzer mithilfe solcher Dienste abgerufen haben, ohne dass Codeänderungen in der Anwendung erforderlich sind, was für Benutzer eine nahtlose Benutzerfunktion bietet.

### <a name="support-for-opentype-collections-using-cff-outlines"></a>Unterstützung für OpenType-Auflistungen mithilfe von CFF

Die Schriftart Formate OpenType und TrueType haben lange unterstützt, dass mehrere Schriftarten in einer einzelnen Schriftart Datei zusammengefasst werden können, die als "Schriftart Sammlung" bezeichnet wird. Die OpenType-Spezifikation hat den Schriftarten die Verwendung von TrueType-oder CFF-Formaten für Glyphe-Daten immer gestattet. Bis vor kurzem hat die Spezifikation jedoch nur für Auflistungen zulässig, in denen Symbol Gliederungen das TrueType-Format verwenden. OpenType Version 1,7 ermöglicht es Auflistungen jetzt, TrueType-oder CFF-Formate für Symbol Gliederungs Daten zu verwenden. Ab dem Windows 10 Anniversary Update unterstützt DirectWrite OpenType-Auflistungen mithilfe von CFF-Gliederungs Daten.

### <a name="windows-font-service-integration"></a>Windows-Schriftart Dienst Integration

Dies gilt für alle Windows 10-Editionen, einschließlich Windows 10 Mobile, Xbox und hololens sowie den Desktop Client. ### <a name="font-set-apis"></a>Schriftart Satz-APIs

Die Schnittstellen der Schriftart Auflistung von DirectWrite stellen eine Ansicht für eine Auflistung von Schriftarten bereit, die nach Schriftfamilien organisiert ist. dabei werden Gewichtung, Streckung und Stil als untergeordnete Familien Attribute verwendet. Intern implementiert DirectWrite die Schriftart Sammlungs Schnittstelle mithilfe einer flachen Liste von Schriftarten mit verschiedenen Attributen. Dieser Ansatz ist flexibler, da in die Aufzählung von Gewichtungs-/streckungs-Familien unterstützen kann, aber auch das Abfragen und Filtern mit anderen Schriftart Attributen unterstützen kann.

In Windows 10 wird dieser flexiblere Schriftart Behandlungs Mechanismus für Anwendungen über das idwrite-FONTSET und verwandte APIs zur Verfügung gestellt. Die Schriftart Satz-APIs können z. b. verwendet werden, um eine benutzerdefinierte Schriftart Auswahl-Benutzeroberfläche zu erstellen, die benutzerdefinierte Schriftart Eigenschaften in einem benutzerdefinierten Schriftart Satz nutzt.

Weitere Informationen finden Sie unter den folgenden Themen:

In Windows 10 wird ein zusätzlicher "proportionaler" Zeilen Abstands Modus unterstützt, der Anwendungen mehr Optionen für das Verhalten von Zeilen Abstände bietet. ### <a name="chinese-japanese-and-korean-extensions-c--d"></a>Chinesisch, Japanisch und Koreanisch Erweiterungen C & D

Alle paar Jahre gibt das Unicode-Konsortium eine standardisierte Liste von Ergänzungen für den chinesischen, japanischen und koreanischen vereinheitlichten ideograph-Block frei. Bei der Unicode 6,0-Revision haben Sie die Erweiterungs Blöcke C und D freigegeben. Diese Blöcke von Ideogramme finden Sie in der Unicode-Website [Erweiterung C](https://www.unicode.org/charts/PDF/U2A700.pdf) und der [Erweiterung D](https://www.unicode.org/charts/PDF/U2B740.pdf).

Ab Windows 8 unterstützt [DirectWrite](direct-write-portal.md) die Unicode-Code Punkte für diese neuen Blöcke standardisierter cjk-Ideographen, sodass Sie Sie in DirectWrite-Apps verwenden können.

### <a name="indian-rupee-symbol"></a>Indische Rupie (Symbol)

Im März 2005 kündigte die indische Regierung einen Wettbewerb an, um ein Symbol für die Indische Rupie-Währung auszuwählen. Nach vielen Wettbewerben hat sich der indische Regierungsdienst am 15. Juli 2010 den von D. udaya Kumar erstellten Entwurf entschieden, und [DirectWrite](direct-write-portal.md) enthält Unterstützung für den Unicode-Codepunkt, der an das Symbol gebunden ist. DirectWrite-apps unterstützen nun dieses Währungssymbol.

### <a name="emoji"></a>Emoji

[DirectWrite](direct-write-portal.md) unterstützt jetzt die Verwendung von emoji in-apps. Vorherige Versionen von DirectWrite, die mit einem fehlenden Symbolfeld angezeigt werden, wenn Sie versucht haben, ein Emoji ideograph zu erzeugen. Ab Windows 8 unterstützt DirectWrite den Unicode-Codeblock, der Emoji zugeordnet ist, sodass die entsprechenden Symbole angezeigt werden, wenn Ihre APP die Unicode-Standard Codepunkte für Emoji verwendet.

### <a name="myanmar-tiffinagh-and-old-hangul-languages"></a>Myanmar, tiffinagh und alte Hangul-Sprachen

Ab Windows 8 unterstützt [DirectWrite](direct-write-portal.md) den Block von Unicode-Code Punkten, die den Symbolen in den Sprachen Myanmar, tiffinagh und Old Hangul entsprechen, sodass Sie Apps erstellen können, die Text aus diesen drei Sprachen enthalten. Zusätzlich zur Unterstützung dieser Zeichen unterstützt DirectWrite die einzigartige Methode, mit der alte Hangul den Zeilenumbruch verarbeitet.
