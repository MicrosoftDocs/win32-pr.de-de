# <a name="inputtype-complex-type"></a>Komplexer InputType-Typ

Definiert einen Eingabe Datentyp.

``` syntax
<xs:complexType name="InputType">
    <xs:sequence>
        <xs:element name="description"
            type="string"
         />
        <xs:element name="outType"
            type="OutputType"
            minOccurs="0"
            maxOccurs="unbounded"
         />
        <xs:any
            processContents="lax"
            namespace="##other"
         />
    </xs:sequence>
    <xs:attribute name="name"
        type="QName"
        use="required"
     />
    <xs:attribute name="value"
        type="string"
        use="required"
     />
    <xs:attribute name="symbol"
        type="CSymbolType"
        use="required"
     />
</xs:complexType>
```

## <a name="child-elements"></a>Untergeordnete Elemente



| Element                                                                  | type                                                             | BESCHREIBUNG                                                                                                                                                                                                                                                                                                                                                                                                   |
|--------------------------------------------------------------------------|------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**Beschreibung**](eventmanifestschema-description-inputtype-element.md) | Zeichenfolge                                                           | Eine Beschreibung des Eingabetyps.<br/>                                                                                                                                                                                                                                                                                                                                                                   |
| [**outtype**](eventmanifestschema-outtype-inputtype-element.md)         | [**OutputType**](eventmanifestschema-outputtype-complextype.md) | Eine Liste der Ausgabe Datentypen, die bestimmen, wie die Daten gerendert werden. Sie müssen mindestens einen Ausgabetyp angeben. Geben Sie eine Liste an, wenn der Eingabetyp als verschiedene Ausgabetypen gerendert werden kann. Beispielsweise kann der Eingabetyp Win: UInt16 als unsignierter Kurzwert, Portnummer oder hexadezimal Zahl gerendert werden. Für einen der Ausgabetypen, die Sie angeben, muss das default-Attribut auf "true" festgelegt sein.<br/> |

| Name   | type                                                              | BESCHREIBUNG                                                                                                                                                                                                                                              |

Die Typen werden in der \\ include- \\Winmeta.xml Datei definiert, die in der Windows SDK enthalten ist. Die Spalte "Standard-XML-Format" gibt den kanonischen Typ für den jeweiligen Eingabetyp an.
