# <a name="printticket-syntax-and-schema"></a>Print Ticket-Syntax und-Schema

Dieses Thema ist nicht aktuell. Die aktuellsten Informationen finden Sie in der [PrintSchema-Spezifikation](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Print Tickets werden in der Syntax des XML-basierten Print Schema-Frameworks ausgedrückt. Daher sollte der Reader die [Zusammenfassung der Element Typen](summary-of-element-types.md) für die erforderliche Syntax überprüfen.

Das PrintTicket-Schema ist in Bezug auf die verwendeten Elementtypen und in der allgemeinen Struktur, die durch übergeordnete und untergeordnete Elemente ausgedrückt wird, eng mit dem Druck Schema Framework vergleichbar. Beachten Sie, dass ParameterDef-Elemente in einem PrintTicket nicht zulässig sind und dass parameterinit-Elemente nur in einem PrintTicket zulässig sind.
