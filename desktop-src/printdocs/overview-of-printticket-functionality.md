# <a name="overview-of-printticket-functionality"></a>Übersicht über die PrintTicket-Funktionalität

Dieses Thema ist nicht aktuell. Die aktuellsten Informationen finden Sie in der [PrintSchema-Spezifikation](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Print Ticket verwendet ein XML-basiertes Format zum Ausdrücken von Geräte Konfigurationsinformationen mithilfe der Feature/Option und Parameterkonstrukte des Print Schema-Frameworks. Dies schließt alle Standardelement Typen sowie die Erweiterbarkeits Funktionen des Druck Schema-Frameworks ein. Bei einem PrintTicket wird davon ausgegangen, dass es sich um eine eigenständige Beschreibung der Verarbeitung einer einzelnen Seite handelt. Die Schritte zum Extrahieren und Konstruieren eines solchen PrintTicket aus einem bestimmten Dokumentformat werden hier nicht behandelt.

Ein Print Ticket kann verwendet werden, um ein Printworks-Dokument zu erhalten, in dem die Merkmale des Geräts für eine bestimmte Konfiguration beschrieben werden. Alternativ kann das PrintTicket mit einem Satz von Renderinganweisungen übermittelt werden, um eine Ausgabe Seite zu entwickeln, die entsprechend der in PrintTicket angegebenen Konfiguration gerendert und verarbeitet wird.
