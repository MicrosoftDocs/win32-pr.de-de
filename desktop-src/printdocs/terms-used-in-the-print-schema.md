# <a name="terms-used-in-the-print-schema"></a>Im Druckschema verwendete Begriffe

Dieses Thema ist nicht aktuell. Die aktuellsten Informationen finden Sie unter Print Schema Specification (Spezifikation des [Druckschemas).](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip)

Die folgende Liste enthält Begriffe, die für das Druckschema und verwandte Technologien spezifisch sind.



| Begriff                                                                                                                                                                 | BESCHREIBUNG                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="attribute_name"></span><span id="ATTRIBUTE_NAME"></span>Attributname<br/>                                                                          | Ein Attributname ist das Token auf der linken Seite des Gleichheitszeichens in einem XML-Standardattribut. Die Standarddarstellung eines XML-Attributs ist *Attributname*="*Attributwert*". Obwohl der Begriff "Attributname" ähnlich ist, darf er nicht mit dem namensattribut verwechselt werden, das im Druckschemaframework definiert ist. Das im Druckschemaframework definierte Namensattribut ist eine bestimmte Instanz eines Attributnamens. <br/>                                                                                                        |
| <span id="candidate"></span><span id="CANDIDATE"></span>Kandidat<br/>                                                                                         | Option. Jede Option, die zu der Liste gehört, die während des Bewertungsprozesses ausgewertet wird. Sehen Sie sich die Begriffe, Bewertungs- oder Zuordnungsoption an, und verweisen Sie auf Option.<br/>                                                                                                                                                                                                                                                                                                                                                                                |
| <span id="Configuration"></span><span id="configuration"></span><span id="CONFIGURATION"></span>Konfiguration<br/>                                             | Beschreibt einen der Zustände, die ein Gerät annehmen kann oder in dem es platziert werden kann. Das Gerät führt seine Aufgabe in jedem Zustand unterschiedlich aus. Es gibt einen eindeutigen und identifizierbaren Unterschied zwischen dem Verhalten des Geräts in jedem dieser Zustände. Beispielsweise wird das Medium in einem Zustand aus einer Taskleiste gezogen und in einer anderen Taskleiste angezeigt. Eine Änderung in einer oder beiden Dieser Taskleisten bestimmt einen anderen Zustand.<br/>                                                                                                                               |
| <span id="Constraint"></span><span id="constraint"></span><span id="CONSTRAINT"></span>Einschränkung<br/>                                                         | Informationen, die zum Bestimmen der Gültigkeit einer Konfiguration erforderlich sind. Eine Einschränkung hat häufig folgendes Format: FeatureA set to Option1 is not compatible with FeatureB set to Option2.<br/>                                                                                                                                                                                                                                                                                                                                                            |
| <span id="corresponding_elements"></span><span id="CORRESPONDING_ELEMENTS"></span>entsprechende Elemente<br/>                                                  | Zwei Elemente, die relativ zu einem angegebenen Ursprungspaar die gleiche Position aufweisen. Die Ursprungselemente sind häufig zwei Option-Elemente, die verglichen werden.<br/>                                                                                                                                                                                                                                                                                                                                                              |
| <span id="default"></span><span id="DEFAULT"></span>Standard<br/>                                                                                               | Option. Die Option, die entweder durch das Druckschema oder den PrintCapabilities-Anbieter als Option festgelegt wird, die einem Feature zugewiesen ist, wenn printTicket keine explizite Einstellung für dieses Feature enthält. Jedes Feature verfügt über eine festgelegte Standardoption.<br/>                                                                                                                                                                                                                                                               |
| <span id="device_configuration_attribute"></span><span id="DEVICE_CONFIGURATION_ATTRIBUTE"></span>Gerätekonfigurationsattribut<br/>                          | Ein Geräteattribut, das zur Definition der Gerätekonfiguration beiträgt. Ein Gerätekonfigurationsattribut wird im Druckschemaframework entweder durch Feature-/Options- oder Parameterkonstrukte dargestellt. Beachten Sie, dass das in diesem Kontext verwendete Attribut weder mit XML-Attributen noch mit durch das Print Schema Framework definierten Attribute-Elementen in Beziehung ist.<br/>                                                                                                                                                                          |
| <span id="element_type"></span><span id="ELEMENT_TYPE"></span>Elementtyp<br/>                                                                                | Der Typ der einzelnen XML-Elemente, die im Print Schema Framework definiert sind, wird durch das XML-Elementtag identifiziert.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="elements_in_common"></span><span id="ELEMENTS_IN_COMMON"></span>Allgemeine Elemente<br/>                                                              | Siehe entsprechende Elemente.<br/> Feature. Ein vom Druckschemaframework definiertes Konstrukt, das ein bestimmtes Gerätekonfigurationsattribut darstellt. Jede Featureinstanz stellt ein Geräteattribut dar. Eine Funktionsinstanz wird definiert, indem jeder ihrer Zustände explizit definiert wird.<br/> Identity-Option. Die Option, die bei Auswahl keine beobachtbaren Auswirkungen hat. in der Tat ein "no-op". Das Ergebnis ist immer gleich, unabhängig davon, wie oft diese Option ausgewählt wird.<br/>                                               |
| <span id="instance_name"></span><span id="INSTANCE_NAME"></span>Instanzname<br/>                                                                             | Ein XML-Attribut, das die spezifische Instanz eines Elementtyps identifiziert. Weitere Informationen finden Sie unter Name-Attribut.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="interface_client"></span><span id="INTERFACE_CLIENT"></span>Schnittstellenclient<br/>                                                                    | Die Entität, die PrintCapabilities-Dokumente verwendet und PrintTickets erstellt und nutzt. Häufige Beispiele für Schnittstellenclients sind Benutzeroberflächenmodule und Renderingmodule.<br/>                                                                                                                                                                                                                                                                                                                                   |
| <span id="interface_provider"></span><span id="INTERFACE_PROVIDER"></span>Schnittstellenanbieter<br/>                                                              | Die Entität, die Funktionen über eine Schnittstelle implementiert und verfügbar macht. Im Druckschema bedeutet dies das Erstellen von PrintCapabilities-Dokumenten und das Überprüfen von PrintTickets. <br/>                                                                                                                                                                                                                                                                                                                                                 |
| <span id="location_of_an_element"></span><span id="LOCATION_OF_AN_ELEMENT"></span>Position eines Elements<br/>                                                  | Die Position einer bestimmten Instanz eines Elements innerhalb der Elementhierarchie. Der Speicherort wird sowohl vom jeweiligen Element als auch von allen Vorgängern dieses Elements definiert. Anders ausgedrückt: Location entspricht einem vollqualifizierten Pfad in einem Dateisystem. <br/>                                                                                                                                                                                                                                                                    |
| <span id="name_attribute"></span><span id="NAME_ATTRIBUTE"></span>name-Attribut<br/>                                                                          | Die Dem XML-Attribut zugewiesene Bezeichnung mit dem Attributnamen "name". Dieses Attribut wird in den meisten Elementtypen angezeigt, die im Druckschemaframework definiert sind. Sie identifiziert und definiert Instanzen jedes Elementtyps. Die folgenden beiden Begriffe beziehen sich auf Namensattribute: die Nebengeordnete Eindeutigkeitsregel und die Namespacekonvention.<br/>                                                                                                                                                                                         |
| <span id="Namespace_Convention"></span><span id="namespace_convention"></span><span id="NAMESPACE_CONVENTION"></span>Namespacekonvention<br/>                 | Gibt an, dass schemadefinierte Namensattribute sowie jedes schemadefinierte Objekt in einem dedizierten Namespace, im öffentlichen Namespace und in einem anderen Namespace vorhanden sein müssen, der als zu der definierenden Partei gehörend identifiziert werden kann.<br/> Option. Ein Druckschema-Frameworkkonstrukt, das zum Darstellen von Zuständen verwendet wird, in denen jedes Feature annehmen oder platziert werden kann. Wenn das Feature beispielsweise die Mediengröße darstellt, können typische Optionswerte unter anderem Letter, Legal, A4 sein.<br/> |
| <span id="Parameter"></span><span id="parameter"></span><span id="PARAMETER"></span>Parameter<br/>                                                             | Das Druckschema unterstützt die Möglichkeit, parametrisierte Daten für Benutzereingaben zu beschreiben. Es gibt drei Print Schema Framework-Elementtypen, die in parametrischen Eingaben Rollen spielen: ParameterDef, ParameterInit und ParameterRef. Ein ParameterDef-Element kennzeichnet einen Parameter (seinen Wertebereich, den Typ seiner Daten und andere Merkmale). Ein ParameterInit-Element wird verwendet, um einen Parameter zu initialisieren. Ein ParameterRef-Element wird verwendet, um auf eine bestimmte Instanz eines ParameterInit-Elements zu verweisen.<br/>         |
| <span id="parameterized"></span><span id="PARAMETERIZED"></span>Parametrisierte<br/>                                                                             | Option. Eine Option-Instanz, die mindestens ein ScoredProperty-Element enthält, das wiederum ein ParameterRef-Element anstelle eines Value-Elements enthält.<br/>                                                                                                                                                                                                                                                                                                                                                                                 |
| <span id="perfectly-matching"></span><span id="PERFECTLY-MATCHING"></span>perfekt übereinstimmend<br/>                                                              | Optionen. Zwei Optionsinstanzen werden als perfekt übereinstimmend betrachtet, wenn für jedes ScoredProperty-Element in einer Option-Instanz eine entsprechende ScoredProperty-Instanz in der anderen Option-Instanz vorhanden ist und beide Value-Elemente der entsprechenden ScoredProperty-Instanzen identisch sind.<br/>                                                                                                                                                                                                                                        |
| <span id="PrintCapabilities_Schema"></span><span id="printcapabilities_schema"></span><span id="PRINTCAPABILITIES_SCHEMA"></span>PrintCapabilities-Schema<br/> | Das Schema, das die Syntax und Semantik definiert, die von einem PrintCapabilities-Dokument verwendet werden. Das PrintCapabilities-Schema und das PrintTicket-Schema sind Teilmengen des Druckschemas. Siehe auch Druckschemaframework.<br/>                                                                                                                                                                                                                                                                                                                       |
| <span id="PrintTicket_Schema"></span><span id="printticket_schema"></span><span id="PRINTTICKET_SCHEMA"></span>PrintTicket-Schema<br/>                         | Das Schema, das die Syntax und Semantik definiert, die vom PrintTicket verwendet werden. Das PrintTicket-Schema und das PrintCapabilities-Schema sind Teilmengen des Druckschemas. Siehe auch Druckschemaframework.<br/>                                                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Framework"></span><span id="print_schema_framework"></span><span id="PRINT_SCHEMA_FRAMEWORK"></span>Druckschemaframework<br/>         | Das Schema, das die Syntax und Semantik definiert, die von Druckschematechnologien verwendet werden. Beispielsweise definiert das Druckschemaframework eine Auflistung von Elementtypen und deren Verwendung, führt aber nicht alle Elementinstanzen auf.<br/>                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Keywords"></span><span id="print_schema_keywords"></span><span id="PRINT_SCHEMA_KEYWORDS"></span>Druckschemaschlüsselwörter<br/>             | Das Schema, das einzelne Instanzen von Property-, Feature- und Option-Elementen definiert, die häufig zum Beschreiben von Geräten und der Auftragsformatierung verwendet werden. Dieses Schema definiert eine gemeinsame Sprache, die PrintCapabilities-Dokumente und PrintTickets portierbar macht. Wird auch als öffentliches Schema bezeichnet.<br/>                                                                                                                                                                                                                                         |
| <span id="privately-defined"></span><span id="PRIVATELY-DEFINED"></span>privat definiert<br/>                                                                 | Alle Instanzen von Elementen, die in einem PrintCapabilities-Dokument oder PrintTicket angezeigt werden und nicht explizit in den Schlüsselwörtern für das Druckschema definiert sind. Diese Elementinstanzen müssen explizit mit einem privaten Namespace pro Anbieter qualifiziert werden.<br/>                                                                                                                                                                                                                                                                                           |
| <span id="reference"></span><span id="REFERENCE"></span>Verweis<br/>                                                                                         | Option. Die Option, mit der jede kandidatende Option verglichen wird. Eine Referenzoption stammt in der Regel aus printTicket. Sehen Sie sich die Begriffe, die Bewertungs- oder Zuordnungsoption und die Kandidatenoption an.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="root_level"></span><span id="ROOT_LEVEL"></span>Stammebene<br/>                                                                                      | Die Stammebene ist die Basis der hierarchischen Struktur des Druckschemaframeworks. Ein Element auf Stammebene wird als untergeordnetes Element eines PrintCapabilities-Elements oder eines PrintTicket-Elements definiert.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="scoping_prefix"></span><span id="SCOPING_PREFIX"></span>Bereichspräfix<br/>                                                                          | Eine Textbezeichnung, die an ein Schemaschlüsselwort angefügt ist, um kontextbezogenen Bereich zu ermöglichen. Die zulässigen Bereichspräfixe für Das Druckschema sind Job, Doc und Page. Das Bereichspräfix für JobInputBin ist beispielsweise "Job".<br/>                                                                                                                                                                                                                                                                                                                               |
| <span id="ScoredProperty"></span><span id="scoredproperty"></span><span id="SCOREDPROPERTY"></span>ScoredProperty<br/>                                         | Ein Print Schema Framework-Konstrukt, das verwendet wird, um die Qualitäten zu darstellen, die eine Optionsinstanz von einer anderen unterscheiden oder unterscheiden. Ein Beispiel hierfür ist der Unterschied zwischen papierformatiertem Letter -Dokument und Legal-Size-Papier.<br/>                                                                                                                                                                                                                                                                                                                              |
| <span id="scoring_or_mapping"></span><span id="SCORING_OR_MAPPING"></span>Bewertung oder Zuordnung<br/>                                                              | Option. Der Prozess der Auswahl der (Kandidaten-)Option-Instanz aus einer Liste, die am besten mit einer angegebenen Option -Instanz (Verweisinstanz) wählt. Eng verwandte Begriffe sind Kandidatenoption und Referenzoption. <br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="Sibling_Uniqueness_Rule"></span><span id="sibling_uniqueness_rule"></span><span id="SIBLING_UNIQUENESS_RULE"></span>Gleichgeordnete Eindeutigkeitsregel<br/>     | Gibt an, dass alle gleichgeordneten Elemente desselben Elementtyps eindeutige Namensattribute haben müssen. Einige Elementtypen sind von dieser Regel ausgenommen. Diese Ausnahmen werden im Druckschemaframework definiert.<br/>                                                                                                                                                                                                                                                                                                                                          |
| <span id="single-valued_versus_multivalued"></span><span id="SINGLE-VALUED_VERSUS_MULTIVALUED"></span>Einwertige im Vergleich zu mehrwertigen<br/>                    | Eigenschaft. Eine Eigenschaft, deren Wert invariant ist (hängt nicht von der Konfiguration ab) wird als einwertig betrachtet. Eine solche Eigenschaft verfügt unter jeder möglichen Konfiguration nur über einen einzelnen Wert. Eine Eigenschaft, die von der Konfiguration abhängt, wird als mehrwertige Eigenschaft betrachtet.<br/>                                                                                                                                                                                                                                                        |
| <span id="suffix_suffixes"></span><span id="SUFFIX_SUFFIXES"></span>Suffixe/Suffixe<br/>                                                                       | Ein Schemaschlüsselwort, das den Feature- oder Parameternamen darstellt. Das Suffix für JobInputBin ist beispielsweise "InputBin".<br/>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| <span id="Validation"></span><span id="validation"></span><span id="VALIDATION"></span>Validierung<br/>                                                         | Der Prozess der Überprüfung, ob ein PrintTicket dem Schema entspricht und eine Konfiguration darstellt, die von einem bestimmten Gerät ausgeführt werden kann.<br/>                                                                                                                                                                                                                                                                                                                                                                                           |
| <span id="_Undefined_"></span><span id="_undefined_"></span><span id="_UNDEFINED_"></span>\_Undefined\_<br/>                                                   | Die hier dargestellten spezifischen Daten sind ein Platzhalter für den spezifischen Treiber, das Gerät oder die Anwendung, der bzw. die erzeugt werden soll. Es wird nicht vermittelt, dass die Daten eine Zeichenfolge sein sollten.<br/>                                                                                                                                                                                                                                                                                                                                                            |
