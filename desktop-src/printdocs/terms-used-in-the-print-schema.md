# <a name="terms-used-in-the-print-schema"></a>Begriffe, die im Druck Schema verwendet werden

Dieses Thema ist nicht aktuell. Die aktuellsten Informationen finden Sie in der [PrintSchema-Spezifikation](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Die folgende Liste enthält Begriffe, die für das Druck Schema und zugehörige Technologien spezifisch sind.



| Begriff                                                                                                                                                                 | BESCHREIBUNG                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="attribute_name"></span><span id="ATTRIBUTE_NAME"></span>Attribut Name<br/>                                                                          | Ein Attribut Name ist das Token auf der linken Seite des Gleichheitszeichens in einem Standard-XML-Attribut. Die Standarddarstellung eines XML-Attributs ist *Attribute Name*= "*Attribut Wert*". Obwohl der Begriff "Attribut Name" ähnlich ist, darf er nicht mit dem Namensattribut verwechselt werden, das im Druck Schema Framework definiert ist. Das im Druck Schema Framework definierte Name-Attribut ist eine bestimmte Instanz eines Attribut namens. <br/>                                                                                                        |
| <span id="candidate"></span><span id="CANDIDATE"></span>Anwärter<br/>                                                                                         | Andere. Jede Option, die zur Liste gehört, die während der Bewertung ausgewertet wird. Weitere Informationen finden Sie unter Begriffe, Bewertung oder Zuordnung und Verweis Option.<br/>                                                                                                                                                                                                                                                                                                                                                                                |
| <span id="Configuration"></span><span id="configuration"></span><span id="CONFIGURATION"></span>Konfiguri<br/>                                             | Beschreibt einen der Zustände, die ein Gerät annehmen oder in das er eingefügt werden kann. Das Gerät führt seine Aufgabe in jedem Status unterschiedlich aus. Es gibt einen eindeutigen und identifizierbaren Unterschied zwischen dem Verhalten des Geräts in den einzelnen Zuständen. Beispielsweise wird in einem Zustand das Medium aus einer Leiste herausgezogen und am Ende in einer anderen Info Leiste angezeigt. Eine Änderung in einem oder beiden dieser Tabletts bestimmt einen anderen Zustand.<br/>                                                                                                                               |
| <span id="Constraint"></span><span id="constraint"></span><span id="CONSTRAINT"></span>Einschränkung<br/>                                                         | Informationen, die erforderlich sind, um die Gültigkeit einer Konfiguration zu bestimmen. Eine Einschränkung hat häufig folgende Form: featurefest gelegt auf Option1 ist nicht kompatibel mit featureb ist auf Option2 festgelegt.<br/>                                                                                                                                                                                                                                                                                                                                                            |
| <span id="corresponding_elements"></span><span id="CORRESPONDING_ELEMENTS"></span>entsprechende Elemente<br/>                                                  | Zwei Elemente, die denselben Speicherort haben, relativ zu einem angegebenen paar von Ursprungs Elementen. Die Ursprungs Elemente sind häufig zwei Options Elemente, die verglichen werden.<br/>                                                                                                                                                                                                                                                                                                                                                              |
| <span id="default"></span><span id="DEFAULT"></span>vorgegebene<br/>                                                                                               | Andere. Die Option, die entweder durch das Druck Schema oder den printfunktionsanbieter als Option festgelegt wird, die einer Funktion zugewiesen wird, wenn das PrintTicket keine explizite Einstellung für diese Funktion enthält. Jede Funktion verfügt über eine festgelegte Standard Option.<br/>                                                                                                                                                                                                                                                               |
| <span id="device_configuration_attribute"></span><span id="DEVICE_CONFIGURATION_ATTRIBUTE"></span>Geräte Konfigurations Attribut<br/>                          | Ein Geräte Attribut, das zur Definition der Gerätekonfiguration beiträgt. Ein Geräte Konfigurations Attribut wird im Druck Schema Framework entweder durch Feature/Option oder Parameterkonstrukte dargestellt. Beachten Sie, dass in diesem Kontext verwendetes Attribut keine Beziehung zu XML-Attributen oder zum Drucken von Schema Framework-definierten Attribut Elementen hat.<br/>                                                                                                                                                                          |
| <span id="element_type"></span><span id="ELEMENT_TYPE"></span>Elementtyp<br/>                                                                                | Der Typ jedes XML-Elements, das im Druck Schema Framework definiert ist, wird durch sein XML-Elementtag identifiziert.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="elements_in_common"></span><span id="ELEMENTS_IN_COMMON"></span>allgemeine Elemente<br/>                                                              | Siehe entsprechende-Elemente.<br/> Befinden. Ein vom Druck Schema Framework definiertes Konstrukt, das ein bestimmtes Geräte Konfigurations Attribut darstellt. Jede Funktions Instanz stellt ein Geräte Attribut dar. Eine Funktions Instanz wird definiert, indem jeder ihrer Zustände explizit definiert wird.<br/> Identity-Option. Die Option, die keinen wahrnehmbaren Effekt hat, wenn Sie ausgewählt ist. tatsächlich ein "No-op". Das Ergebnis ist immer gleich, unabhängig davon, wie oft diese Option ausgewählt ist.<br/>                                               |
| <span id="instance_name"></span><span id="INSTANCE_NAME"></span>Instanzname<br/>                                                                             | Ein XML-Attribut, das die bestimmte Instanz eines Elementtyps identifiziert. Siehe Name-Attribut.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="interface_client"></span><span id="INTERFACE_CLIENT"></span>Schnittstellen Client<br/>                                                                    | Die Entität, die Printworks-Dokumente verarbeitet, und die PrintTickets erstellt und verarbeitet. Gängige Beispiele für Schnittstellen Clients sind Module der Benutzeroberfläche (UI) und renderingmodule.<br/>                                                                                                                                                                                                                                                                                                                                   |
| <span id="interface_provider"></span><span id="INTERFACE_PROVIDER"></span>Schnittstellen Anbieter<br/>                                                              | Die Entität, die implementiert und Funktionen über eine Schnittstelle verfügbar macht. Im Druck Schema ist dies das Erstellen von Printworks-Dokumenten und das Validieren von PrintTickets. <br/>                                                                                                                                                                                                                                                                                                                                                 |
| <span id="location_of_an_element"></span><span id="LOCATION_OF_AN_ELEMENT"></span>Speicherort eines Elements<br/>                                                  | Die Position einer bestimmten Instanz eines Elements in der Element Hierarchie. Der Speicherort wird sowohl vom jeweiligen Element als auch von allen übergeordneten Elementen dieses Elements definiert. Anders ausgedrückt: Location ist das Äquivalent eines voll qualifizierten Pfads in einem Dateisystem. <br/>                                                                                                                                                                                                                                                                    |
| <span id="name_attribute"></span><span id="NAME_ATTRIBUTE"></span>Name-Attribut<br/>                                                                          | Die dem XML-Attribut mit dem Attributnamen "Name" angegebene Bezeichnung. Dieses Attribut wird in den meisten Elementtypen angezeigt, die im Druck Schema Framework definiert sind. Es identifiziert und definiert Instanzen der einzelnen Elementtypen. Die folgenden zwei Begriffe verweisen auf namens Attribute: die gleich geordnete Eindeutigkeits Regel und die-Namespace Konvention.<br/>                                                                                                                                                                                         |
| <span id="Namespace_Convention"></span><span id="namespace_convention"></span><span id="NAMESPACE_CONVENTION"></span>Namespace Konvention<br/>                 | Gibt an, dass Schema definierte namens Attribute sowie beliebige Schema definierte Objekte in einem dedizierten Namespace, dem öffentlichen Namespace und in einem anderen Namespace vorhanden sein müssen, der als Mitglied der definierenden Partei angegeben werden kann.<br/> Andere. Ein Druck Schema-Framework-Konstrukt, mit dem Zustände dargestellt werden, die von den einzelnen Features angenommen oder in platziert werden können. Wenn das Feature z. b. die Mediengröße darstellt, können typische Optionswerte Buchstabe, Legal, A4 und andere sein.<br/> |
| <span id="Parameter"></span><span id="parameter"></span><span id="PARAMETER"></span>Parame<br/>                                                             | Das Druck Schema unterstützt die Möglichkeit, parametrisierte Daten für den Zweck der Benutzereingabe zu beschreiben. Es gibt drei Typen von Druck Schema Framework-Elementen, die Rollen in parametrischer Eingabe wiedergeben: ParameterDef, parameterinit und ParameterRef. Ein ParameterDef-Element kennzeichnet einen Parameter (seinen Wertebereich, den Datentyp und andere Merkmale). Ein parameterinit-Element wird verwendet, um einen Parameter zu initialisieren. Ein ParameterRef-Element wird verwendet, um auf eine bestimmte Instanz eines parameterinit-Elements zu verweisen.<br/>         |
| <span id="parameterized"></span><span id="PARAMETERIZED"></span>parametrisierte<br/>                                                                             | Andere. Eine Options Instanz, die mindestens ein ScoredProperty-Element enthält, das wiederum ein ParameterRef-Element anstelle eines Value-Elements enthält.<br/>                                                                                                                                                                                                                                                                                                                                                                                 |
| <span id="perfectly-matching"></span><span id="PERFECTLY-MATCHING"></span>vollständig übereinstimmend<br/>                                                              | Optionen. Zwei Options Instanzen werden als Übereinstimmung betrachtet, wenn für jedes ScoredProperty-Element in einer Options Instanz eine entsprechende ScoredProperty-Instanz in der anderen Options Instanz vorhanden ist und beide Wert Elemente der entsprechenden ScoredProperty-Instanzen identisch sind.<br/>                                                                                                                                                                                                                                        |
| <span id="PrintCapabilities_Schema"></span><span id="printcapabilities_schema"></span><span id="PRINTCAPABILITIES_SCHEMA"></span>Printfunktionalitäten-Schema<br/> | Das Schema, das die Syntax und die Semantik definiert, die von einem printfunktionalitäten-Dokument verwendet werden. Das printfunktionalitäten-Schema und das PrintTicket-Schema sind Teilmengen des Druck Schemas. Siehe auch Print Schema Framework.<br/>                                                                                                                                                                                                                                                                                                                       |
| <span id="PrintTicket_Schema"></span><span id="printticket_schema"></span><span id="PRINTTICKET_SCHEMA"></span>PrintTicket-Schema<br/>                         | Das Schema, das die Syntax und die Semantik definiert, die von PrintTicket verwendet werden. Das PrintTicket-Schema und das printfunktionalitäten-Schema sind Teilmengen des Druck Schemas. Siehe auch Print Schema Framework.<br/>                                                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Framework"></span><span id="print_schema_framework"></span><span id="PRINT_SCHEMA_FRAMEWORK"></span>Print Schema Framework<br/>         | Das Schema, das die Syntax und die Semantik definiert, die von Druck Schema Technologien verwendet werden. Das Print Schema-Framework definiert z. b. eine Auflistung von Elementtypen und deren Verwendungsmöglichkeiten, listet aber nicht alle Element Instanzen auf.<br/>                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Keywords"></span><span id="print_schema_keywords"></span><span id="PRINT_SCHEMA_KEYWORDS"></span>Schema Schlüsselwörter drucken<br/>             | Das Schema, das einzelne Instanzen von Eigenschaften, Features und Options Elementen definiert, die häufig zum Beschreiben von Geräten und Auftrags Formatierung verwendet werden. Dieses Schema definiert eine gemeinsame Sprache, mit der Printworks-Dokumente und PrintTickets portabel werden. Wird auch als öffentliches Schema bezeichnet.<br/>                                                                                                                                                                                                                                         |
| <span id="privately-defined"></span><span id="PRIVATELY-DEFINED"></span>Privat definiert<br/>                                                                 | Alle Instanzen von Elementen, die in einem Printworks-Dokument oder in PrintTicket angezeigt werden und nicht explizit in den Schlüsselwörtern für den Druck Schema definiert sind. Diese Element Instanzen müssen explizit mit einem privaten Namespace pro Anbieter qualifiziert werden.<br/>                                                                                                                                                                                                                                                                                           |
| <span id="reference"></span><span id="REFERENCE"></span>Angabe<br/>                                                                                         | Andere. Die Option, mit der die einzelnen Kandidaten Optionen verglichen werden. Eine Verweis Option stammt in der Regel aus Print Ticket. Weitere Informationen finden Sie unter den Nutzungsbedingungen, der Bewertungsoption und der Kandidaten Option.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="root_level"></span><span id="ROOT_LEVEL"></span>Stamm Ebene<br/>                                                                                      | Die Stamm Ebene ist die Basis der hierarchischen Struktur des Print Schema-Frameworks. Ein Element auf Stamm Ebene ist als untergeordnetes Element eines Print-Funktionen-oder PrintTicket-Elements definiert.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="scoping_prefix"></span><span id="SCOPING_PREFIX"></span>Bereichs Präfix<br/>                                                                          | Eine Text Bezeichnung, die an ein Schema Schlüsselwort angefügt wird, um den Kontext Bereich bereitzustellen. Die Präfixe für das Druck Schema sind zulässig: Job, doc und Page. Beispielsweise ist das Bereichs Präfix für JobInputBin "Job".<br/>                                                                                                                                                                                                                                                                                                                               |
| <span id="ScoredProperty"></span><span id="scoredproperty"></span><span id="SCOREDPROPERTY"></span>ScoredProperty<br/>                                         | Ein Druck Schema-Framework-Konstrukt, mit dem die Qualitäten dargestellt werden, die eine Options Instanz von einer anderen unterscheiden oder unterscheiden. Ein Beispiel hierfür ist der Unterschied zwischen Buchstaben und Papier (rechts).<br/>                                                                                                                                                                                                                                                                                                                              |
| <span id="scoring_or_mapping"></span><span id="SCORING_OR_MAPPING"></span>Bewertung oder Zuordnung<br/>                                                              | Andere. Der Prozess der Auswahl aus einer Liste der (Kandidaten-) Options Instanz, die am besten mit einer bestimmten (Verweis-) Options Instanz übereinstimmt. Eng verwandte Begriffe sind eine Kandidaten Option und Verweis Option. <br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="Sibling_Uniqueness_Rule"></span><span id="sibling_uniqueness_rule"></span><span id="SIBLING_UNIQUENESS_RULE"></span>Gleich geordnete Eindeutigkeits Regel<br/>     | Gibt an, dass alle gleich geordneten Elemente desselben Elementtyps über eindeutige namens Attribute verfügen müssen. Einige Elementtypen sind von dieser Regel ausgenommen. Diese Ausnahmen werden im Druck Schema Framework definiert.<br/>                                                                                                                                                                                                                                                                                                                                          |
| <span id="single-valued_versus_multivalued"></span><span id="SINGLE-VALUED_VERSUS_MULTIVALUED"></span>einwertig im Vergleich zu mehrwertigen Werten<br/>                    | Eigenschaft. Eine Eigenschaft, deren Wert invariante ist (hängt nicht von der Konfiguration ab), wird als einwertig betrachtet. Eine solche Eigenschaft hat nur einen einzelnen Wert in einer möglichen Konfiguration. Eine Eigenschaft, die von der Konfiguration abhängt, wird als mehr wertig betrachtet.<br/>                                                                                                                                                                                                                                                        |
| <span id="suffix_suffixes"></span><span id="SUFFIX_SUFFIXES"></span>Suffix/Suffixe<br/>                                                                       | Ein Schema Schlüsselwort, das den Funktions-oder Parameternamen darstellt. Das Suffix für JobInputBin beispielsweise ist "InputBin".<br/>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| <span id="Validation"></span><span id="validation"></span><span id="VALIDATION"></span>Validierungs<br/>                                                         | Der Prozess, bei dem überprüft wird, ob ein Print Ticket dem Schema entspricht, und stellt eine Konfiguration dar, die von einem bestimmten Gerät ausgeführt werden kann.<br/>                                                                                                                                                                                                                                                                                                                                                                                           |
| <span id="_Undefined_"></span><span id="_undefined_"></span><span id="_UNDEFINED_"></span>\_Definiert\_<br/>                                                   | Bei den hier dargestellten Daten handelt es sich um einen Platzhalter für den spezifischen Treiber, das Gerät oder die Anwendung. Es wird nicht angezeigt, dass die Daten eine Zeichenfolge sein sollten.<br/>                                                                                                                                                                                                                                                                                                                                                            |
