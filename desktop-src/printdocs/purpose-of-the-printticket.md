# <a name="purpose-of-the-printticket"></a>Zweck des PrintTickets

Dieses Thema ist nicht aktuell. Die aktuellen Informationen finden Sie unter [Print Schema Specification (Spezifikation des Druckschemas).](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip)

Der Hauptzweck von PrintTicket ist das Ausdrücken von Gerätekonfigurationen. Im Gegensatz zum PrintCapabilities-Dokument, das alle möglichen Konfigurationen definiert, die das Gerät annehmen kann (konfigurationsraum des Geräts), dient das PrintTicket dazu, eine einzelne spezifische Konfiguration zu definieren, die auf das Auftragsobjekt (in diesem Fall die Seite) angewendet wird, das verarbeitet wird. Um die Portabilität und Konsistenz von Konfigurationen zu fördern, werden die spezifischen Schlüsselwörter und die Hierarchie, die im Abschnitt Print Schema Keywords (Schlüsselwörter des Druckschemas) dargestellt werden, sowohl von der PrintCapabilities- als auch der PrintTicket-Technologie verwendet. Darüber hinaus [](print-schema-framework.md) definiert das Druckschemaframework die grundlegenden Konstrukte beider Technologien, um eine eindeutige, offene und erweiterbare Kommunikation von Auftragsformatierung und PrintCapabilities zu fördern.
