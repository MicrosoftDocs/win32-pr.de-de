    -   <span data-ttu-id="7a6a4-112">Identifizieren Sie für mit Schlüsselwörtern für Druckschemas definierte Featureinstanzen jeden verfügbaren Status, auf den dieses Feature festgelegt werden kann.</span><span class="sxs-lookup"><span data-stu-id="7a6a4-112">For Print Schema Keywords-defined Feature instances, identify each of the available states that this Feature can be set to.</span></span> <span data-ttu-id="7a6a4-113">Jeder Zustand entspricht einer Option der Funktionsinstanz.</span><span class="sxs-lookup"><span data-stu-id="7a6a4-113">Each state corresponds to an Option of the Feature instance.</span></span> <span data-ttu-id="7a6a4-114">Bestimmen Sie, welcher dieser Zustände den Mit diesem Feature verknüpften Vom Schema definierten Optionsinstanzen drucken entspricht und welche Zustände eine benutzerdefinierte Optionsinstanz erfordern.</span><span class="sxs-lookup"><span data-stu-id="7a6a4-114">Determine which of these states correspond to Print Schema-defined Option instances associated with this Feature, and which states require a customized Option instance.</span></span> <span data-ttu-id="7a6a4-115">Das Thema [Optionsdefinitionen](option-definitions.md) enthält Informationen zum Erstellen neuer Optionsinstanzen und zum Ableiten neuer Optionsinstanzen von vorhandenen Optionsinstanzen.</span><span class="sxs-lookup"><span data-stu-id="7a6a4-115">The [Option Definitions](option-definitions.md) topic presents information on how to construct new Option instances, and how to derive new Option instances from existing Option instances.</span></span>

<span data-ttu-id="7a6a4-138">Eine Liste zusätzlicher Eigenschafteninstanzen, die bereitgestellt werden müssen, um Funktions-/Optionskonstrukte und Parameter vollständig zu definieren, finden Sie unter [ParameterDef](parameterdef.md) und [ParameterInit.](parameterinit.md)</span><span class="sxs-lookup"><span data-stu-id="7a6a4-138">For a list of additional Property instances that must be supplied to fully define Feature/Option constructs and parameters, see [ParameterDef](parameterdef.md) and [ParameterInit](parameterinit.md).</span></span> <span data-ttu-id="7a6a4-139">Beispielsweise muss jedes Feature sein Benutzeroberflächenverhalten angeben, insbesondere ob genau eine oder mehrere Optionsinstanzen für jedes Feature gleichzeitig ausgewählt werden können.</span><span class="sxs-lookup"><span data-stu-id="7a6a4-139">For example, each Feature must specify its user interface (UI) behavior, specifically whether exactly one or multiple Option instances may be selected for each Feature at one time.</span></span> <span data-ttu-id="7a6a4-140">Das Dokument Druckschemaschlüsselwörter definiert diese Eigenschafteninstanzen, in denen sie im PrintCapabilities-Dokument angezeigt werden müssen und welche in den Schlüsselwörtern für Druckschemas definierten Wertinstanzen verfügbar sind.</span><span class="sxs-lookup"><span data-stu-id="7a6a4-140">The Print Schema Keywords document defines these Property instances, where they must appear within the PrintCapabilities document, and which Value instances defined in the Print Schema Keywords are available.</span></span>

<span data-ttu-id="7a6a4-141">Der PrintCapabilities-Anbieter ist für die Ausgabe des entsprechenden Werts für alle konfigurationsabhängigen Eigenschafteninstanzen verantwortlich.</span><span class="sxs-lookup"><span data-stu-id="7a6a4-141">The PrintCapabilities provider is responsible for emitting the appropriate Value for all configuration-dependent Property instances.</span></span> <span data-ttu-id="7a6a4-142">Wenn z. <span data-ttu-id="7a6a4-147">Beachten Sie, dass ein funktionaler Konflikt zwischen Einstellungen als zwei nicht in Konflikt stehende Druckschemaelemente definiert ist, die eine ähnliche Funktion aufweisen, aber unterschiedliche Features sind.</span><span class="sxs-lookup"><span data-stu-id="7a6a4-147">Note that a functional conflict between settings is defined as two, non-conflicting Print Schema elements that have similar function but are different features.</span></span> <span data-ttu-id="7a6a4-148">Ein Beispiel wären JobDuplexAllDocumentsContiguously und DocumentDuplex. beide stellen die Duplexfunktion des Geräts dar, unterscheiden sich jedoch in der Anwendung der Funktion, eine anwendung auf den gesamten Auftrag zusammenhängend und eine für Dokumente.</span><span class="sxs-lookup"><span data-stu-id="7a6a4-148">An example would be JobDuplexAllDocumentsContiguously and DocumentDuplex;
