## <a name="join-the-windows-desktop-application-program"></a>Treten Sie dem Windows-Desktop Anwendungsprogramm bei

**Wenn Ihr Unternehmen bereits über ein Partner Center-Konto verfügt**: Melden Sie sich bei Ihrem Partner Center-Konto an (mit dem Microsoft-Konto, das dem Konto Besitzer zugeordnet ist), und navigieren Sie zur Seite **Programme** (entweder in den **Kontoeinstellungen** **oder im linken** Navigationsmenü). Klicken Sie unter **Windows-Desktop Anwendung** **auf starten,** um dem Programm ohne zusätzliche Kosten beizutreten. Wenn Sie über einen Azure AD Mandanten verfügen, der Ihrem Partner Center-Konto zugeordnet ist, können Benutzer, die Sie hinzugefügt haben, auf das Windows-Desktop Anwendungsprogramm zugreifen. In Kürze können Sie einen präziseteren Zugriff auf dieses Programm festlegen.

> [!Tip]  
> Wenn Ihr Unternehmen über ein Partner Center-Konto verfügt, aber keinen Zugriff darauf hat, bitten Sie den Administrator, [Sie als Benutzer hinzuzufügen](/windows/uwp/publish/add-users-groups-and-azure-ad-applications). Beachten Sie, dass nur der Konto Besitzer dem Windows-Desktop Anwendungsprogramm beitreten kann.

 

**Wenn Ihr Unternehmen nicht über ein Partner Center-Konto verfügt**: Sie können [sich direkt für das Windows-Desktop Anwendungsprogramm registrieren](https://login.microsoftonline.com/common/oauth2/authorize?client_id=4990cffe-04e8-4e8b-808a-1175604b879f&response_mode=form_post&response_type=code+id_token&scope=openid+profile&state=OpenIdConnect.AuthenticationProperties%3dWc5R_wIKVD0EbOy2UUxS0_0GQJnIAbD-eisMn7Gb4cJL18fRdelvbtj5_R0zoGlsebcnAxIvwKS5kx4Ma4mLMbU4l9ULsE9ajiZU4wtchLJXyJGsPCjCBUNV7TY1SzwXAI-LepSoXkqa8xSywVb7JZ3Xed-Lcw-kwEShFOwt0SdSdc1nNevHbPOhotOeFQcqbo0HESVYXk6pZORJ_OYimG99onp_zSTyludOvvaTd9GYKUgX9exCU5IHReP7MzJDHOgqTg&nonce=637177463071243612.NDU4MjE2ZTMtNmVkMi00YWNiLWEzZGEtMjYyNDRkODI0M2FmOTM3MmE1NzgtMzQ1OC00M2ZkLWJhMDktYzI4YTNhNzdiYTk0&redirect_uri=https%3a%2f%2fpartner.microsoft.com%2faad%2fauthPostGateway&resource=797f4846-ba00-4fd7-ba43-dac1f8f63013&mkt=en-US) . In Kürze wird die Option zum [Zuordnen eines Azure AD Mandanten mit Ihrem Konto](/windows/uwp/publish/add-users-groups-and-azure-ad-applications) bereitgestellt, damit andere Personen in Ihrem Unternehmen sich auch anmelden können.

## <a name="add-your-desktop-applications"></a>Hinzufügen Ihrer Desktop Anwendungen

Nachdem Sie das Programm verknüpft haben, müssen Sie Ihre Windows-Desktop Anwendungen Ihrem Dashboard hinzufügen, damit wir Ihnen Ihre Analyseberichte zeigen können.

Wir verwenden die Code Signatur zum Einrichten der Identität Ihres Unternehmens und zum Abrufen von Analysen für apps, die Sie veröffentlichen.

Wir stellen Ihnen eine Datei bereit und bitten Sie, Sie mit denselben gültigen, nicht abgelaufenen, nicht gesperrten Code Signatur Zertifikaten zu signieren, die Sie zum Signieren Ihrer Desktop Anwendungen verwenden. Danach laden Sie die signierte Datei in das Dashboard hoch. Dadurch können wir wissen, dass alle Desktop Anwendungen, die mit demselben Zertifikat signiert sind, zu Ihrem Konto gehören. Ihre Zertifikat Informationen werden nicht für andere Zwecke verwendet.

> [!Important]  
> Sie müssen diesen Vorgang nicht wiederholen, wenn Sie eine neue Desktop Anwendung freigeben. Nachdem Sie die signierte Datei hochgeladen haben, werden automatisch alle neuen Anwendungen identifiziert, die mit demselben Zertifikat signiert sind, und wir werden automatisch Analysen für diese Produkte abrufen. Außerdem ist es nicht erforderlich, die bereitgestellte Datei in Ihren Anwendungen zu verteilen oder irgendeine Art von Zuordnung für Ihre Produkte zu übermitteln.

 

**So fügen Sie eine oder mehrere Desktop Anwendungen hinzu**

1.  Wählen Sie in Ihrem Dashboard **Desktop Anwendungen hinzufügen** aus.
2.  Laden Sie auf der nächsten Seite die Registrierungsdatei herunter, indem **Sie Datei herunterladen** auswählen, und speichern Sie die Datei auf Ihrem Computer.
3.  Signieren Sie die soeben heruntergeladene Datei mit demselben Code Signaturzertifikat, das Sie zum Authentifizieren Ihrer Desktop Anwendung (en) verwenden. Sie können SignTool.exe (verfügbar in Microsoft Visual Studio und als Teil des [Windows SDK](https://developer.microsoft.com/windows/downloads/windows-10-sdk)) verwenden, um diese Datei zu signieren. Weitere Informationen zu diesem Prozess werden unten beschrieben.
4.  Laden Sie die soeben signierte Datei hoch, indem Sie Sie in das Feld ziehen (oder klicken Sie, um die Dateien zu durchsuchen).
5.  Wählen Sie über **Mitteln** aus, um den Vorgang abzuschließen.

![Schritte zum Hinzufügen von Desktop Anwendungen](images/uploadcert.png)

Wenn Sie mehr als ein Code Signaturzertifikat verwenden, können Sie die obigen Schritte für jedes Ihrer Zertifikate wiederholen. Sie können für jedes aktuelle Zertifikat, das Sie zum Signieren Ihrer Anwendungen verwenden, eine Datei herunterladen, Signieren und hochladen. Allerdings können Sie nur ein Zertifikat pro heruntergeladener Datei verwenden.

Nachdem Sie diese Schritte ausgeführt haben, werden Sie feststellen, welche Windows-Desktop Anwendungen mit demselben Zertifikat signiert sind, das Sie zum Signieren der Datei verwendet haben. In den meisten Fällen beginnen wir mit der Anzeige analytischer Berichte innerhalb von 48 Stunden, obwohl dies gelegentlich etwas länger dauern kann.

## <a name="use-signtoolexe-to-sign-the-downloaded-file"></a>Verwenden von signtool.exe zum Signieren der heruntergeladenen Datei

Microsoft stellt ein Tool zum Signieren von Dateien, SignTool.exe, mit Visual Studio und im [Windows SDK](https://developer.microsoft.com/windows/downloads/windows-10-sdk)bereit. Sie können dieses Tool verwenden, um den Code Signatur Prozess auszuführen und zu überprüfen. Weitere Informationen zu SignTool.exe finden Sie [hier](/dotnet/framework/tools/signtool-exe).

Im folgenden finden Sie zwei der gängigsten Möglichkeiten, dieses Tool zum Signieren der signalisierbaren Datei zu verwenden.

-   Wenn Sie Zugriff auf das Code Signaturzertifikat als [PFX-Datei (Personal Information Exchange)](/windows-hardware/drivers/install/personal-information-exchange---pfx--files) haben:

    ``` syntax
    signtool sign /f MyCert.pfx /p MyCertPassword /v SignableFile.bin
    ```

    ![Screenshot, der ein Eingabe Aufforderungs Fenster mit dem Befehl "SignTool Sign/f mycert. pfx/p mycertpassword/v signablefile. bin" anzeigt.](images/signtool2.png)

-   Wenn das Code Signaturzertifikat im lokalen Zertifikat Speicher verfügbar ist:

    ``` syntax
    Signtool sign /v /s MY /n CertSubjectName SignableFile.bin
    ```

    ![Eingabe Aufforderungs Fenster mit diesem Befehl](images/signtool.png)

Nachdem Sie die Datei signiert haben, können Sie überprüfen, ob Sie erfolgreich mit einem gültigen Zertifikat mit folgendem Zertifikat signiert wurde:

``` syntax
signtool verify /a SignableFile.bin
```

## <a name="viewing-your-analytic-data"></a>Anzeigen der analytischen Daten

Nachdem die signierten Dateien hochgeladen wurden und Sie Ihre Desktop Anwendungen identifiziert haben, zeigt Ihr Dashboard eine Übersicht über Ihre Anwendungen sowie wichtige Metriken an.

Unsere Telemetriedaten zeigen Integritäts Informationen, wie z. b. Abstürze, für jede Anwendung, die Ihrem Zertifikat zugeordnet ist. Ihr Dashboard zeigt eine Übersicht der Anwendungen sowie Schlüsselmetriken. Sie können eine beliebige Anwendung auswählen, um den Integritäts [Bericht](#health-report)anzuzeigen, den [Bericht zu installieren](#installs-report)und den Bericht im Dashboard zu [blockieren](#application-blocks-report) . Sie können [analytische Daten auch Programm gesteuert abrufen, indem Sie die Microsoft Store Analytics-API verwenden](#retrieve-analytic-data-using-the-microsoft-store-analytics-api).

> [!Note]  
> Wenn Sie feststellen, dass die Metadaten einer Anwendung so aktualisiert wurden, dass ein neuer Name verwendet wird, beginnen wir damit, neue Daten unter dem neuen Namen zu melden. Vergangenheits Daten, die dem alten Namen zugeordnet sind, werden 30 Tage lang aufbewahrt.
> 
> Analytics ist erst dann für eine Anwendung verfügbar, wenn Sie auf mindestens 100 Geräten installiert wurde.


### <a name="health-report"></a>Integritätsbericht

Der  Integritäts Bericht ermöglicht es Ihnen, Daten im Zusammenhang mit der Leistung und Qualität Ihrer APP zu erhalten, einschließlich abstürzen und nicht reagierender Ereignisse. Gegebenenfalls können Sie Stapel Überwachungen und/oder CAB-Dateien zum weiteren Debuggen anzeigen.

![Integritäts Bericht-Windows-Desktop Anwendungsprogramm](images/health.png)

Sie können die Daten auf verschiedene Weise filtern, um Folgendes zu ermöglichen:

-   Anzeigen einer Zusammenfassung aller Fehlertypen, sortiert nach Anzahl von Treffern
-   Drilldown zu einem bestimmten Fehler und Herunterladen von Stapel Überwachungen zum schnelleren Debuggen des Problems
-   Vergleichen einer neuen Version der Anwendung mit den vorherigen Releases
-   Anzeigen von Integritäts Daten in Aggregaten oder nach Regionen, sodass Sie Probleme, die spezifisch für eine Region sind, isolieren können
-   Vergleichen Sie die Leistung Ihrer Desktop Anwendungen in Windows-Versionen oder einer bestimmten Version, z. b. der neuesten Version von Windows 10.
-   Anzeigen von Integritäts Informationen für eine bestimmte ausführbare Datei, die in der Anwendung enthalten ist

Wählen Sie oben in der **Fehler** Tabelle **Symbole hochladen** aus, um eine ZIP-Datei mit den [Symbol Dateien](/windows-hardware/drivers/debugger/symbols-and-symbol-files)Ihrer Anwendung hochzuladen. Diese Symbol Dateien werden indiziert und verwendet, um genauere Stapel Überwachungen zu liefern. Symbol Dateitypen in der ZIP-Datei sollten PDB, dll oder exe sein. Nachdem Sie die ZIP-Datei erfolgreich hochgeladen haben, sollte weniger angezeigt werden **. Unbekannte** Werte für neue Fehler in der Fehlerliste der Anwendung in ungefähr 5 Tagen.

### <a name="installs-report"></a>Installiert den Bericht.

Mit dem Bericht " **Installationen** " können Sie sehen, auf wie vielen Geräten eine Anwendung für einen beliebigen Tag installiert wurde und wie viele Geräte in den letzten 30 Tagen von den einzelnen Anwendungsversionen installiert wurden.

Sie können die Daten auf verschiedene Weise filtern, um Folgendes zu ermöglichen:

-   Anzeigen einer Zusammenfassung ihrer Installationen, sortiert nach Beliebtheit
-   Vergleichen einer neuen Version der Anwendung mit den vorherigen Releases
-   Anzeigen der Installationsdaten in Aggregat oder nach Region
-   Vergleichen Sie die Leistung Ihrer Desktop Anwendungen in Windows-Versionen oder in einer bestimmten Version, z. b. der neuesten Windows 10-Version oder Windows Insider fast und langsamen Releases.

## <a name="managing-your-desktop-application-metadata"></a>Verwalten von Metadaten der Desktop Anwendung

Wir verwenden die Metadaten Dateiname, Dateiversion, Produktname und Produktversion in Ihren ausführbaren Dateien, um die logischen Gruppierungen der ausführbaren Dateien in Anwendungen zu ableiten. Wenn ausführbare Dateien keine genauen Metadaten aufweisen, werden Sie möglicherweise unter einem **unbekannten** Anwendungsnamen angezeigt, oder der Anwendungsname wird standardmäßig auf den Namen der einzelnen ausführbaren Dateien angewendet.

Wenn Sie die Metadaten Ihrer Apps und Dateien auf dem aktuellen standhalten, müssen Sie sicherstellen, dass Sie in Ihrem Dashboard ordnungsgemäß dargestellt werden. Hier sind einige Vorschläge:

-   Verwenden Sie Ihr Zertifikat, um jede ausführbare Datei zu signieren, die Sie im Analytics-Bericht anzeigen möchten, und nicht nur die ausführbaren Dateien für das Setup.
-   Geben Sie konsistente Informationen zu Produktname und Produktversion für alle ausführbaren Dateien an, die in derselben Anwendung (d. h. *meiner Anwendung*) enthalten sind. Wenn einige Ihrer ausführbaren Dateien mit mehreren Anwendungen verteilt werden, weisen Sie ihnen eindeutige Namen (d. h. frei *gegebene Komponenten*) zu, damit Sie die Analyse für diese ausführbaren Dateien getrennt von den Anwendungen sehen können, mit denen Sie verteilt wurden.
-   Jedes Mal, wenn Sie Änderungen an Ihren Metadaten vornehmen, wird möglicherweise ein neuer Eintrag für Ihre Anwendung in Ihrem Dashboard angezeigt. Wenn Sie eine Änderung vornehmen, werden die Änderungen von neuen eingehenden Telemetriedaten widerspiegelt, Ihre alten Telemetriedaten werden jedoch weiterhin als **unbekannte** Anwendung angezeigt.
-   Wenn Sie eine Datei überarbeiten, stellen Sie sicher, dass Sie die Anwendungs Version und die Produkt Versionsnummern aktualisieren.
    > [!Tip]  
    > Verwenden [**Sie VERSIONINFO**](/windows/desktop/menurc/versioninfo-resource) -Ressourcen, um **FileDescription**, **File Version**, **ProductName** und **ProductVersion** für Ihre Dateien und Anwendungen festzulegen. Im folgenden Beispiel wird eine **VERSIONINFO** -Ressource definiert:

## <a name="add-and-manage-account-users"></a>Hinzufügen und Verwalten von Kontobenutzern

Sie können Azure Active Directory verwenden, um weitere Benutzer in Ihrem Windows-Desktop-Anwendungsprogramm Konto hinzuzufügen und zu verwalten. Sie können einzelne Benutzer, Gruppen von Benutzern oder Azure AD Anwendungen hinzufügen und jeweils eine vordefinierte Rolle (Manager oder Entwickler) erteilen.

### <a name="associate-azure-active-directory-with-your-account"></a>Ihrem Konto Azure Active Directory zuordnen

Um Kontobenutzer hinzuzufügen und zu verwalten, müssen Sie Ihr Konto zunächst dem Azure Active Directory Ihrer Organisation zuordnen. Wenn in Ihrer Organisation bereits mit Office 365 oder anderen Unternehmensdiensten von Microsoft gearbeitet wird, verfügen Sie bereits über Azure AD. Andernfalls können Sie ohne zusätzliche Kosten einen neuen Azure AD Mandanten erstellen.

Weitere Informationen finden [Sie unterzuordnen von Azure Active Directory zu Ihrem Partner Center-Konto](/windows/uwp/publish/associate-azure-ad-with-dev-center) . Während sich das Thema auf das Entwicklerprogramm von Windows-apps konzentriert, funktioniert die Zuordnung eines Mandanten auf die gleiche Weise für das Windows-Desktop Anwendungsprogramm.

### <a name="add-users-groups-and-azure-ad-applications-to-your-account"></a>Hinzufügen von Benutzern, Gruppen und Azure AD Anwendungen zu Ihrem Konto

Nachdem Sie die Azure AD Zuordnung eingerichtet haben, können Sie Benutzer hinzufügen, indem Sie im Abschnitt "Benutzer" unter "Kontoeinstellungen" auf "Benutzer" klicken. Jedem Benutzer wird eine Rolle zugewiesen, die den Zugriff auf das Konto definiert. Sie können auch Gruppen von Benutzern und Azure AD Anwendungen hinzufügen, um Ihnen Zugriff auf Ihr Partner Center-Konto zu gewähren. Weitere Informationen zum Hinzufügen von Benutzern finden [Sie unter Hinzufügen von Benutzern, Gruppen und Azure AD Anwendungen](/windows/uwp/publish/add-users-groups-and-azure-ad-applications).

Jeder Benutzer-, Gruppen-oder Azure AD Anwendung, die Sie Ihrem Konto hinzufügen, muss eine Rolle zugewiesen werden. Dieser Vorgang wird unter [Festlegen von Rollen oder benutzerdefinierten Berechtigungen für Kontobenutzer](/windows/uwp/publish/set-custom-permissions-for-account-users)beschrieben. Beachten Sie jedoch, dass es für das Windows-Desktop Anwendungsprogramm nicht möglich ist, benutzerdefinierte Berechtigungen zuzuweisen oder den Zugriff nach Produkt einzuschränken. Stattdessen muss jedem Benutzer eine der folgenden Standardrollen zugewiesen werden.



| Role      | Beschreibung                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
|-----------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Manager   | Kann Zertifikate hochladen und entfernen und alle analytischen Daten anzeigen. Hat vollen Zugriff auf das Konto, mit Ausnahme von Änderungen an Finanzinformationen. Dies umfasst das Verwalten von Benutzern, aber beachten Sie, dass die Möglichkeit, Benutzer im Azure AD Mandanten zu erstellen und zu löschen, von der Berechtigung des Kontos in Azure AD abhängig ist. Das heißt, wenn einem Benutzer die Rolle "Manager" zugewiesen ist, aber keine globalen Administrator Berechtigungen im Azure Ad der Organisation vorhanden sind, können Sie keine neuen Benutzer erstellen oder Benutzer aus dem Verzeichnis löschen (obwohl Sie die Konto Rolle eines Benutzers ändern können).<br/> Beachten Sie Folgendes: Wenn Ihr Konto mehr als einem Azure AD Mandanten zugeordnet ist, kann ein Manager keine kompletten Details für einen Benutzer anzeigen (einschließlich Vorname, Nachname, e-Mail-Wiederherstellungs-e-Mail und ob es sich um einen Azure AD globalen Administrator handelt), es sei denn, Sie sind beim gleichen Mandanten wie dieser Benutzer mit einem Konto mit globalen Administrator Berechtigungen für diesen Mandanten angemeldet. Allerdings können Sie Benutzer in allen Mandanten hinzufügen und entfernen, die dem Konto zugeordnet sind. <br/> |
| Entwickler | Können die dem Konto zugeordneten Anwendungen und Zertifikat Details anzeigen **und den Bericht** "Integrität und **Installation** " anzeigen. Finanzinformationen oder Kontoeinstellungen können nicht angezeigt werden.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |



 

## <a name="faq"></a>Häufig gestellte Fragen

-   **Warum sehe ich keine Daten für eine Anwendung?** Es werden keine Daten angezeigt, bis wir genug Benutzer erkennen, um sinnvolle Informationen zu sammeln. Wenn Sie Ihre Anwendung soeben veröffentlicht haben, kann es einige Zeit dauern, bis dieser minimale Akzeptanz Schwellenwert erreicht wird. Ein weiterer Grund, warum Daten möglicherweise nicht angezeigt werden, ist, wenn Sie keine Datei mit dem Zertifikat für eine bestimmte Anwendung signiert haben. Stellen Sie sicher, dass Sie Dateien hochladen, die mit jedem Zertifikat signiert sind, mit dem Sie Ihre Anwendungen signieren.
-   **Kann ich über eine API auf diese Daten zugreifen?** Ja, die Daten werden über eine öffentliche API zur Verfügung gestellt, wenn das Programm für alle Entwickler verfügbar ist.
-   **Wie sieht es mit Anwendungen mit älteren Zertifikaten aus?** Leider wird das Übermitteln von Zertifikaten, die abgelaufen sind oder gesperrt sind, nicht unterstützt, auch wenn Sie diese mit demselben Schlüssel erneuern.
-   **Warum wird eine Anwendung angezeigt, die ich nicht erkenne?** Wenn das Zertifikat, das Sie zum Signieren von Dateien in der Anwendung verwenden, auch von einer anderen Person in Ihrem Unternehmen verwendet wird, um eine andere Anwendung zu signieren, sehen Sie auch Telemetriedaten für diese Anwendung In Zukunft wird eine Option zum Ausblenden von Anwendungen aus Ihrem Dashboard bereitgestellt. Wenn Ihr Unternehmens Konto an einen Azure AD Mandanten angefügt ist, können Sie Ihren Administrator bitten, Benutzerberechtigungen zu ändern, sodass nur bestimmte Anwendungen für Sie sichtbar sind.
-   **Wie kann ich Feedback zur Verfügung stellen oder Support erhalten?** Wenn Sie Unterstützung benötigen, können Sie [hier](https://developer.microsoft.com/windows/support/)eine Supportanfrage erstellen. Zum Freigeben Ihres Feedbacks verwenden Sie den Link **Feedback** (unter **Kontoeinstellungen**), und wählen Sie den Bereich **Analyse** aus, um uns mitzuteilen, was Sie denken.
