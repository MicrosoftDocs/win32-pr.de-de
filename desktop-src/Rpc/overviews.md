# <a name="overviews"></a>Übersichten

Dieser Teil des Remote Prozedur Aufrufs (RPC)-Programmier Handbuch und-Referenz besteht aus einer Sequenz von Themen, die Ihnen das Verständnis verteilter Anwendungsprogrammierung und RPC wie folgt erleichtern:

-   Das [Microsoft RPC-Modell](microsoft-rpc-model.md) bietet eine Übersicht über das Client/Server-Programmiermodell, die Standards für die Programmierung verteilter Anwendungen und eine Beschreibung der Funktionsweise von Microsoft RPC.
-   [Die Installation der RPC-Programmierumgebung erläutert](installing-the-rpc-programming-environment.md) , wie Sie die Dateien und Tools installieren, die für die Entwicklung verteilter Anwendungen mit Microsoft RPC erforderlich sind.
-   Beim Entwickeln von [RPC-Anwendungen](building-rpc-applications.md) werden der mittlere Compiler und die erforderliche Umgebung zum entwickeln verteilter Anwendungen mit Microsoft RPC beschrieben.
-   Eine [Verbindung zwischen dem Client und dem Server](connecting-the-client-and-the-server.md) bietet einen Überblick über den Prozess der Initialisierung und Ausführung verteilter Anwendungen.
-   Das [Tutorial](tutorial.md) bietet einen Überblick über die Entwicklung einer kleinen verteilten Anwendung. In diesem Beispiel werden alle Schritte der Entwicklung einer verteilten Anwendung, die verwendeten Tools und die Komponenten veranschaulicht, die die ausführbaren Programme bilden.
-   [IDL-und ACF-Dateien](the-idl-and-acf-files.md) beschreiben die IDL-und ACF-Dateien, die verwendet werden, um die Schnittstelle für den Remote Prozedur aufzurufen und die Mittelwert-Compilerschalter anzugeben, die die Verarbeitung dieser Dateien steuern.
-   Die [Daten-und sprach Features](data-and-language-features.md) veranschaulichen die Verwendung von Standard Datentypen.
-   [Arrays und Zeiger](arrays-and-pointers.md) erläutert, wie Array Zeiger als Parameter übergeben werden.
-   [Pipes](pipes.md) beschreibt, wie Named Pipes als Transportmechanismus für Remote Prozedur Aufrufe verwendet werden.
-   [Bindung und Handles](binding-and-handles.md) beschreibt das Bindungs handle – die Datenstruktur, die es dem Entwickler ermöglicht, die aufrufende Anwendung an die Remote Prozedur zu binden.
-   Die [Speicherverwaltung](memory-management.md) bietet Ideen zum Verwalten des Arbeitsspeichers auf dem Client und dem Server beim Ausführen von Remote Prozedur aufrufen.
-   [Serialisierungsdienste](serialization-services.md) beschreiben die Methoden zum Codieren oder Decodieren von Daten.
-   [Sicherheit](security.md) beschreibt die Methoden zum Implementieren von Sicherheitsfeatures in ihren verteilten Anwendungen.
-   Beim [Installieren und Konfigurieren von RPC-Anwendungen](installing-and-configuring-rpc-applications.md) wird die Installation von Client-und Server Anwendungen erläutert. hier wird beschrieben, wie Sie den Namen Dienstanbieter und den Sicherheitsdienst konfigurieren. In diesem Abschnitt sind auch Netzwerk Transportinformationen für RPC enthalten.
-   [Asynchroner RPC](asynchronous-rpc.md) stellt Informationen zu den asynchronen Microsoft-Erweiterungen für die RPC-Definition dar. Asynchrone Remote Prozedur Aufrufe geben sofort zurück, ohne auf die Ausgabe zu warten. Wenn die Ausführung der Remote Prozedur auf dem Server abgeschlossen ist, werden die Rückgabe Daten an den Client übertragen.
-   [RPC-Message Queuing](rpc-message-queuing.md) beschreibt die Verwendung des Message Queuing Dienstanbieter (MSMQ), mit dem Benutzer unabhängig vom aktuellen Status der Kommunikationsanwendungen und Systeme über Netzwerke und Systeme hinweg kommunizieren können.
-   [Remote Prozedur Aufrufe mithilfe von RPC über HTTP](remote-procedure-calls-using-rpc-over-http.md) bieten RPC-Clients die Möglichkeit, eine sichere Verbindung über das Internet mit RPC-Serverprogrammen herzustellen und Remote Prozedur Aufrufe auszuführen.
-   Der [RPC-Lastenausgleich](rpc-load-balancing.md) beschreibt die Verteilung von hohem RPC-über-HTTP-Datenverkehr auf zahlreiche RPC-Server innerhalb einer Serverfarm.
-   Die [Beispiele](examples.md) enthalten eine Beschreibung der RPC-Beispiel Programme, die mit dem Microsoft Platform Software Developer-Kit ausgeliefert werden.
