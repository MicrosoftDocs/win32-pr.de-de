# <a name="remote-procedure-call"></a>Remoteprozeduraufruf

## <a name="purpose"></a>Zweck

Der Microsoft-Remote Prozedur Aufruf (RPC) definiert eine leistungsstarke Technologie zum Erstellen verteilter Client-/Serverprogramme. Die RPC-laufzeitstubvorgänge und-Bibliotheken verwalten die meisten Prozesse im Zusammenhang mit Netzwerkprotokollen und-Kommunikation. Dies ermöglicht es Ihnen, sich auf die Details der Anwendung und nicht auf die Details des Netzwerks zu konzentrieren.

## <a name="where-applicable"></a>Anwendungsbereich

RPC kann in allen Client/Server-Anwendungen verwendet werden, die auf Windows-Betriebssystemen basieren. Sie kann auch zum Erstellen von Client-und Serverprogrammen für heterogene Netzwerkumgebungen verwendet werden, die solche Betriebssysteme wie UNIX und Apple enthalten.

## <a name="developer-audience"></a>Entwicklergruppe

RPC ist für die Verwendung durch C/C++-Programmierer konzipiert. Vertrautheit mit dem Microsoft Interface Definition Language ("Mittel l") und dem Mittell-Compiler sind erforderlich.

## <a name="run-time-requirements"></a>Laufzeitanforderungen

Die RPC-Laufzeitbibliotheken sind in Windows enthalten. Die Komponenten der RPC-Entwicklungsumgebung werden installiert, wenn Sie das Microsoft Windows Software Development Kit (SDK) installieren. Weitere Informationen finden Sie unter [Installieren der RPC-Programmierumgebung](installing-the-rpc-programming-environment.md).
