# <a name="security-rpc"></a>Sicherheit (RPC)

Mit der zunehmenden Verwendung verteilter Anwendungen ist die Notwendigkeit einer sicheren Kommunikation zwischen den Client- und Serverabschnitten von Anwendungen von entscheidender Bedeutung. Die RPC-Laufzeitbibliothek (Remote Procedure Call) stellt eine standardisierte Schnittstelle für Authentifizierungsdienste für Clients und Server bereit. Die Authentifizierungsdienste auf dem Serverhostsystem bieten RPC-Authentifizierung. Anwendungen verwenden authentifizierte Remoteprozeduraufrufe, um sicherzustellen, dass alle Aufrufe von autorisierten Clients stammen. Sie können auch dazu beitragen, sicherzustellen, dass alle Serverantworten von authentifizierten Servern stammen.

Dieser Abschnitt enthält eine Übersicht über authentifizierten RPC in den folgenden Abschnitten:

-   [RPC Security Essentials](rpc-security-essentials.md)
-   [Sicherheitsmethoden](security-methods.md)
