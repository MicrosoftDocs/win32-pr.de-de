# <a name="security-rpc"></a>Sicherheit (RPC)

Dank der verstärkten Nutzung verteilter Anwendungen ist die Notwendigkeit der sicheren Kommunikation zwischen den Client-und Server Teilen von Anwendungen von größter Wichtigkeit. Die Remote Prozedur Aufruf (RPC)-Lauf Zeit Bibliothek stellt eine standardisierte Schnittstelle für Authentifizierungsdienste für Clients und Server bereit. Die Authentifizierungsdienste auf dem Server Host System bieten eine RPC-Authentifizierung. Anwendungen verwenden authentifizierte Remote Prozedur Aufrufe, um sicherzustellen, dass alle Aufrufe von autorisierten Clients stammen. Sie können auch sicherstellen, dass alle Server Antworten von authentifizierten Servern stammen.

Dieser Abschnitt enthält eine Übersicht über die authentifizierte RPC-Authentifizierung in den folgenden Abschnitten:

-   [Grundlagen zu RPC](rpc-security-essentials.md)
-   [Sicherheitsmethoden](security-methods.md)
