# <a name="n-rpc"></a>N (RPC)

[a](a-glos.md) [B](b-glos.md) [C](c-glos.md) [D](d-glos.md) [E](e-glos.md) [F](f-glos.md) G H [I](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) N [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_name_service_glos"></span><span id="_RPC_NAME_SERVICE_GLOS"></span>**Namensdienst**
</dt> <dd>

Dienst, der-Objekten Namen zuordnet und die Name-Objekt-Paare in einer-Datenbank speichert. Beispielsweise ordnet der RPC-Namensdienst einem [*Bindungs handle*](b-glos.md) einen logischen Namen zu, sodass Client Anwendungen auf diesen logischen Namen und nicht auf eine Protokoll Sequenz und eine Netzwerkadresse verweisen können. Siehe auch Name Service – Interface Daemon ([nsid)](/windows), Client Verzeichnisdienst ([*CDs)*](c-glos.md), [*Locator*](l-glos.md).

</dd> <dt>

<span id="_rpc_nca_glos"></span><span id="_RPC_NCA_GLOS"></span>**Netzwerk Computing-Architektur (NCA)**
</dt> <dd>

Sammlung von Richtlinien für die verteilte Verarbeitung. Die RPC-Kommunikationsprotokolle befolgen diese Richtlinien.

</dd> <dt>

<span id="_rpc_nsi_glos"></span><span id="_RPC_NSI_GLOS"></span>**Name Service Independent (NSI)**
</dt> <dd>

Standard für API-Funktionen, die einer verteilten Anwendung den Zugriff auf RPC Name-Service-Datenbankelemente über verschiedene Name-Service-Anbieter ermöglichen, wie z. b. OSF-DCE-Zell Verzeichnisdienst oder Microsoft Locator. Siehe auch [– Interface Daemon](/windows) (nsid).

</dd> <dt>

<span id="_rpc_nsid_glos"></span><span id="_RPC_NSID_GLOS"></span>**Name Service – Interface Daemon (nsid)**
</dt> <dd>

Dienst, der eine Schnittstelle zwischen Microsoft [*Locator*](l-glos.md) und den OSF-DCE-Dienstnamen Dienst-Datenbanken für RPC-Name-Service-Funktionen bereitstellt.

</dd> <dt>

<span id="_rpc_named_pipe_glos"></span><span id="_RPC_NAMED_PIPE_GLOS"></span>**Named Pipe**
</dt> <dd>

Verbindungs orientiertes Protokoll basierend auf Server Message Blocks (smsb) und [NetBIOS](/windows), die für die Kommunikation zwischen einem Server Prozess und einem oder mehreren Client Prozessen verwendet werden.

</dd> <dt>

<span id="_rpc_netbeui_glos"></span><span id="_RPC_NETBEUI_GLOS"></span>**Erweiterte NetBIOS-Benutzeroberfläche (NetBEUI)**
</dt> <dd>

LAN-Manager-System eigenes Transportprotokoll und Netzwerkgerätetreiber. Siehe auch [NetBIOS](/windows).

</dd> <dt>

<span id="_rpc_netbios_glos"></span><span id="_RPC_NETBIOS_GLOS"></span>**Network Basic Input/Output System (NetBIOS)**
</dt> <dd>

Software Schnittstelle zwischen dem Microsoft MS-DOS-Betriebssystem, dem e/a-Bus und einem lokalen Netzwerk.

</dd> <dt>

<span id="_rpc_ndr_glos"></span><span id="_RPC_NDR_GLOS"></span>**Netzwerkdaten Darstellung (NDR)**
</dt> <dd>

Das Standard Format, das während der Netzwerkübertragung verwendet wird, das unabhängig vom Datentyp Format auf einer bestimmten Computerarchitektur ist. Zu den übertragenen Daten gehören Informationen, die das dazugehörige NDR-Format angeben.

</dd> <dt>

<span id="_rpc_network_address_glos"></span><span id="_RPC_NETWORK_ADDRESS_GLOS"></span>**Netzwerkadresse**
</dt> <dd>

Adresse, mit der ein Server in einem Netzwerk identifiziert wird.

</dd> <dt>

<span id="_rpc_nonencapsulated_union_glos"></span><span id="_RPC_NONENCAPSULATED_UNION_GLOS"></span>**nicht gekapselt Union**
</dt> <dd>

Unterscheidungs- [*Union*](d-glos.md) , die weniger restriktiv ist als eine gekapselte Union, da der diskriminant und die Union nicht eng gebunden sind. Wenn die Union ein Parameter ist, ist der diskriminant ein weiterer Parameter. Wenn die Union ein Strukturfeld ist, ist die Diskriminante ein weiteres Strukturfeld. Der Schalter für die IDL-Schlüsselwörter \[ [**\_ ist**](/windows/desktop/Midl/switch-is) , \] und \[ der [**\_ Switchtyp**](/windows/desktop/Midl/switch-type) \] identifiziert den Diskriminanten und seinen Typ. Siehe auch [*gekapselte Union*](e-glos.md).

</dd> <dt>

<span id="_rpc_nonidempotent_glos"></span><span id="_RPC_NONIDEMPOTENT_GLOS"></span>**nonidempotent**
</dt> <dd>

Gibt an, dass ein Remote Prozedur aufruter nicht mehr als einmal ausgeführt werden kann, da er einen anderen Wert zurückgibt oder einen Zustand ändert.

</dd> </dl>
