# <a name="n-rpc"></a>N (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) [D](d-glos.md) [E](e-glos.md) F [G](f-glos.md) H [I](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) N [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) T [](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_name_service_glos"></span><span id="_RPC_NAME_SERVICE_GLOS"></span>**Name-Dienst**
</dt> <dd>

Dienst, der Namen Objekten zu ordnet und die Name-Objekt-Paare in einer Datenbank speichert. Beispielsweise ordnet der RPC-Namensdienst einem [](b-glos.md) Bindungshand handle einen logischen Namen zu, sodass Clientanwendungen auf diesen logischen Namen verweisen können, anstatt auf eine Protokollsequenz und Netzwerkadresse. Siehe auch name service–interface daemon ([nsid)](/windows), Client Directory Service ([*CDS)*](c-glos.md), [*Locator*](l-glos.md).

</dd> <dt>

<span id="_rpc_nca_glos"></span><span id="_RPC_NCA_GLOS"></span>**Netzwerkcomputingarchitektur (NCA)**
</dt> <dd>

Sammlung von Richtlinien für verteiltes Computing. Die RPC-Kommunikationsprotokolle befolgen diese Richtlinien.

</dd> <dt>

<span id="_rpc_nsi_glos"></span><span id="_RPC_NSI_GLOS"></span>**Name Service Independent (NSI)**
</dt> <dd>

Standard für API-Funktionen, die einer verteilten Anwendung den Zugriff auf RPC-Name-Service-Datenbankelemente über verschiedene Name-Service-Anbieter wie OSF-DCE Cell Directory Service oder Microsoft Locator ermöglichen. Siehe auch [–Schnittstellen-Daemon](/windows) (nsid).

</dd> <dt>

<span id="_rpc_nsid_glos"></span><span id="_RPC_NSID_GLOS"></span>**name service–interface daemon (nsid)**
</dt> <dd>

Dienst, der eine Schnittstelle zwischen Microsoft [*Locator und*](l-glos.md) den Dienstdatenbanken des OSF-DCE Cell Directory-Dienstnamens für RPC-Namensdienstfunktionen bietet.

</dd> <dt>

<span id="_rpc_named_pipe_glos"></span><span id="_RPC_NAMED_PIPE_GLOS"></span>**Named Pipe**
</dt> <dd>

Verbindungsorientiertes Protokoll, das auf Server Message Blocks (SMBs) und [NetBIOS](/windows)basiert und für die Kommunikation zwischen einem Serverprozess und mindestens einem Clientprozess verwendet wird.

</dd> <dt>

<span id="_rpc_netbeui_glos"></span><span id="_RPC_NETBEUI_GLOS"></span>**NetBIOS Extended Benutzeroberfläche (NetBEUI)**
</dt> <dd>

Natives LAN Manager-Transportprotokoll und Netzwerkgerätetreiber. Siehe auch [NetBIOS](/windows).

</dd> <dt>

<span id="_rpc_netbios_glos"></span><span id="_RPC_NETBIOS_GLOS"></span>**Network Basic Input/Output System (NetBIOS)**
</dt> <dd>

Softwareschnittstelle zwischen dem Microsoft MS-DOS-Betriebssystem, dem E/A-Bus und einem lokalen Netzwerk.

</dd> <dt>

<span id="_rpc_ndr_glos"></span><span id="_RPC_NDR_GLOS"></span>**Netzwerkdatendarstellung (NDR)**
</dt> <dd>

Standardformat, das während der Netzwerkübertragung verwendet wird und unabhängig vom Datentypformat auf einer bestimmten Computerarchitektur ist. Übertragene Daten enthalten Informationen, die das NDR-Format angibt.

</dd> <dt>

<span id="_rpc_network_address_glos"></span><span id="_RPC_NETWORK_ADDRESS_GLOS"></span>**Netzwerkadresse**
</dt> <dd>

Adresse, die einen Server in einem Netzwerk identifiziert.

</dd> <dt>

<span id="_rpc_nonencapsulated_union_glos"></span><span id="_RPC_NONENCAPSULATED_UNION_GLOS"></span>**nicht kapselte Union**
</dt> <dd>

[*Unterscheidungs-Union,*](d-glos.md) die weniger restriktiv als eine gekapselte Union ist, da diskriminant und Union nicht eng gebunden sind. Wenn die Union ein Parameter ist, ist die Diskriminanz ein anderer Parameter. Wenn die Union ein Strukturfeld ist, ist die Diskriminanz ein anderes Strukturfeld. Der Schalter für \[ [**IDL-Schlüsselwörter \_ ist ,**](/windows/desktop/Midl/switch-is) \] und der \[ [**\_ Switchtyp**](/windows/desktop/Midl/switch-type) \] identifiziert den Diskriminanten und seinen Typ. Siehe auch [*gekapselte Union.*](e-glos.md)

</dd> <dt>

<span id="_rpc_nonidempotent_glos"></span><span id="_RPC_NONIDEMPOTENT_GLOS"></span>**nicht idempotent**
</dt> <dd>

Gibt an, dass ein Remoteprozeduraufruf nicht mehr als einmal ausgeführt werden kann, da er einen anderen Wert zurück gibt oder einen Zustand ändert.

</dd> </dl>
