# <a name="asynchronous-io-and-asynchronous-rpc"></a><span data-ttu-id="a8111-103">Asynchrone e/a-und asynchrone RPC</span><span class="sxs-lookup"><span data-stu-id="a8111-103">Asynchronous I/O and Asynchronous RPC</span></span>

<span data-ttu-id="a8111-104">Asynchrone e/a ist ein effizientes Mittel für einen einzelnen Thread zur gleichzeitigen Verwaltung mehrerer e/a-Anforderungen.</span><span class="sxs-lookup"><span data-stu-id="a8111-104">Asynchronous I/O is an efficient means for a single thread to manage multiple I/O requests simultaneously.</span></span> <span data-ttu-id="a8111-105">Der asynchrone RPC auf dem Server erreicht einen ähnlichen Zweck für RPC-Anforderungen.</span><span class="sxs-lookup"><span data-stu-id="a8111-105">Asynchronous RPC on the server accomplishes a similar purpose for RPC requests.</span></span> <span data-ttu-id="a8111-106">In Windows-Versionen vor Windows Vista wird von der Veröffentlichung von asynchronen e/a-Anforderungen von Server Prozeduren mithilfe von asynchronem RPC abgeraten.</span><span class="sxs-lookup"><span data-stu-id="a8111-106">In versions of Windows prior to Windows Vista, posting asynchronous I/O requests from server procedures using asynchronous RPC is discouraged.</span></span> <span data-ttu-id="a8111-107">In Windows Vista und höheren Versionen von Windows werden asynchrone e/a-Anforderungen, die einem e/a-Abschlussport zugeordnet sind, jedoch von asynchronem RPC unterstützt.</span><span class="sxs-lookup"><span data-stu-id="a8111-107">However, in Windows Vista and later versions of Windows, asynchronous I/O requests that are associated with an I/O completion port are supported by asynchronous RPC.</span></span>

<span data-ttu-id="a8111-108">Vor Windows Vista kann ein asynchroner Remote Prozedur Aufrufvorgang abgeschlossen werden, bevor die asynchrone e/a-Anforderung abgeschlossen ist.</span><span class="sxs-lookup"><span data-stu-id="a8111-108">Prior to Windows Vista, an asynchronous remote procedure call may complete before the asynchronous I/O request completes.</span></span> <span data-ttu-id="a8111-109">Wenn der asynchrone Aufruf abgeschlossen ist, kann der zugehörige Thread beendet werden, wenn die RPC-Laufzeit entscheidet, dass genügend Threads für die erwartete Arbeitsauslastung zur Verfügung stehen.</span><span class="sxs-lookup"><span data-stu-id="a8111-109">When the asynchronous call completes, its thread may terminate if the RPC runtime decides it has enough threads available to service the expected workload.</span></span> <span data-ttu-id="a8111-110">Das System bindet alle e/a-Anforderungen an den Thread, der Sie initiiert.</span><span class="sxs-lookup"><span data-stu-id="a8111-110">The system binds all I/O requests to the thread that initiates them.</span></span> <span data-ttu-id="a8111-111">Wenn der Thread beendet wird, werden alle für diesen Thread ausstehenden e/a-Anforderungen abgebrochen.</span><span class="sxs-lookup"><span data-stu-id="a8111-111">If the thread terminates, any I/O requests pending on that thread are aborted.</span></span> <span data-ttu-id="a8111-112">Ausstehende e/a-Anforderungen können nicht in einen anderen Thread verschoben werden.</span><span class="sxs-lookup"><span data-stu-id="a8111-112">Pending I/O requests cannot be moved to another thread.</span></span>
