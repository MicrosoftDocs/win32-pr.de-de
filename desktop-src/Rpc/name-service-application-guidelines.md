# <a name="name-service-application-guidelines"></a><span data-ttu-id="bd970-103">Richtlinien für die Namensdienst Anwendung</span><span class="sxs-lookup"><span data-stu-id="bd970-103">Name Service Application Guidelines</span></span>

<span data-ttu-id="bd970-104">Wenn Sie die verteilte Anwendung entwickeln, müssen Sie den Anwendungs Benutzern eine Methode zur Angabe des Namens bereitstellen, unter dem Sie die Anwendung in der Name Service-Datenbank registrieren können.</span><span class="sxs-lookup"><span data-stu-id="bd970-104">When you develop your distributed application, you need to provide the application users with a method for specifying the name under which they can register the application in the name service database.</span></span> <span data-ttu-id="bd970-105">Diese Methode kann aus einer Datendatei, einer Befehlszeilen Eingabe oder einem Dialogfeld bestehen.</span><span class="sxs-lookup"><span data-stu-id="bd970-105">This method can consist of a data file, command-line input, or dialog box.</span></span>

<span data-ttu-id="bd970-106">Obwohl die RPC-Namensdienst Architektur verschiedene Methoden zum Organisieren der Server Einträge einer Anwendung unterstützt, ist Sie für Suchvorgänge optimiert.</span><span class="sxs-lookup"><span data-stu-id="bd970-106">Although the RPC name service architecture supports various methods for organizing an application's server entries, it is optimized for lookups.</span></span> <span data-ttu-id="bd970-107">Demzufolge kann die Leistung des Namens Dienstanbieter und der Anwendung durch häufige Aktualisierungen beeinträchtigt werden.</span><span class="sxs-lookup"><span data-stu-id="bd970-107">As a result, frequent updates can hinder the performance of both the name service and the application.</span></span> <span data-ttu-id="bd970-108">Um zu vermeiden, dass Informationen unnötig exportiert werden, wählen Sie einen Entwurf aus, mit dem der Server feststellen kann, ob seine Informationen in der Name Service-Datenbank</span><span class="sxs-lookup"><span data-stu-id="bd970-108">To avoid exporting information unnecessarily, choose a design that lets the server determine whether its information is in the name service database.</span></span> <span data-ttu-id="bd970-109">Außerdem sollte jede Serverinstanz in ihren eigenen Eintrags Namen exportieren.</span><span class="sxs-lookup"><span data-stu-id="bd970-109">In addition, each server instance should export to its own entry name.</span></span> <span data-ttu-id="bd970-110">Andernfalls ist es für eine Instanz schwierig, ihre unterstützten Objekt-UUIDs oder Protokoll Sequenzen zu ändern, ohne dass die Informationen einer anderen Instanz gestört werden.</span><span class="sxs-lookup"><span data-stu-id="bd970-110">Otherwise, it will be difficult for an instance to change its supported object UUIDs or protocol sequences without disturbing another instance's information.</span></span>

<span data-ttu-id="bd970-111">Mit der folgenden Methode werden diese Fehlerquellen vermieden und eine gute Leistung erzielt, unabhängig davon, welchen Namen Dienst Ihr Netzwerk verwendet.</span><span class="sxs-lookup"><span data-stu-id="bd970-111">The following method avoids these pitfalls and provides good performance, no matter what name service your network uses.</span></span>

<span data-ttu-id="bd970-112">Entwerfen Sie zunächst Ihre Anwendung so, dass beim ersten Start einer bestimmten Serverinstanz ein eindeutiger Name für den Server Eintrag ausgewählt und dieser Name zusammen mit den anderen Konfigurationsinformationen der Anwendung in der Registrierung gespeichert wird.</span><span class="sxs-lookup"><span data-stu-id="bd970-112">To begin with, design your application so that the first time a given server instance starts, it picks a unique server-entry name and saves this name in the registry along with the application's other configuration information.</span></span> <span data-ttu-id="bd970-113">Exportieren Sie dann die Bindungs Handles und Objekt-UUIDs (sofern vorhanden) in den Namen des Namens Diensts.</span><span class="sxs-lookup"><span data-stu-id="bd970-113">Then, have it export its binding handles and object UUIDs, if any, to its name service entry.</span></span>

<span data-ttu-id="bd970-114">Bei nachfolgenden Aufrufen der Serverinstanz sollte überprüft werden, ob der Name-Dienst Eintrag vorhanden ist und den richtigen Satz von Objekt-UUIDs und Bindungs Handles enthält.</span><span class="sxs-lookup"><span data-stu-id="bd970-114">Subsequent invocations of the server instance should check that the name service entry is present and contains the correct set of object UUIDs and binding handles.</span></span> <span data-ttu-id="bd970-115">Ein fehlender Eintrag kann darauf hinweisen, dass ein Administrator ihn entfernt hat oder dass ein Stromausfall dazu geführt hat, dass die Namen Dienst Informationen verloren gegangen sind.</span><span class="sxs-lookup"><span data-stu-id="bd970-115">A missing entry may mean that an administrator removed it, or that a power outage caused the name service information to be lost.</span></span> <span data-ttu-id="bd970-116">Es ist wichtig, zu überprüfen, ob die Bindungs Handles im Eintrag korrekt sind. Wenn ein Administrator einem Computer TCP/IP-Unterstützung hinzufügt, lauschen die RPC-Server z. b. auf diese Protokoll Sequenz, wenn Sie [**rpcserveruseallprotsqs**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcserveruseallprotseqs)aufrufen.</span><span class="sxs-lookup"><span data-stu-id="bd970-116">It is important to verify that the binding handles in the entry are correct; if an administrator adds TCP/IP support to a computer, for example, RPC servers will listen on that protocol sequence when they call [**RpcServerUseAllProtseqs**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcserveruseallprotseqs).</span></span> <span data-ttu-id="bd970-117">Wenn der Server den Namensdienst Eintrag jedoch nicht aktualisiert, werden Clients nicht darüber informiert, dass TCP unterstützt wird.</span><span class="sxs-lookup"><span data-stu-id="bd970-117">However, if the server does not update the name service entry, clients will not be informed that TCP is supported.</span></span>

<span data-ttu-id="bd970-118">Wenn der Client importiert, sollte **null** als Eintrags Name angegeben werden.</span><span class="sxs-lookup"><span data-stu-id="bd970-118">When the client imports, it should specify **NULL** as the entry name.</span></span> <span data-ttu-id="bd970-119">Das Angeben von **null** bewirkt, dass die Microsoft RPC-Bibliotheksfunktionen in allen Namensdienst Einträgen in der Domäne oder Arbeitsgruppe des Client Computers nach der Schnittstelle suchen, sodass die Informationen für jede Instanz gefunden werden.</span><span class="sxs-lookup"><span data-stu-id="bd970-119">Specifying **NULL** causes the Microsoft RPC library functions to search for the interface in all name service entries in the client computer's domain or workgroup, thus finding the information for every instance.</span></span>

<span data-ttu-id="bd970-120">Wenn Sie Objekt-UUIDs verwenden, um bekannte Objekte (z. b. Drucker) darzustellen, können Sie eine Variation dieser Methode verwenden.</span><span class="sxs-lookup"><span data-stu-id="bd970-120">If you use object UUIDs to represent well-known objects such as printers, you can use a variation of this method.</span></span> <span data-ttu-id="bd970-121">Anstatt Bindungen in einen Eintrag zu exportieren, entwerfen Sie Ihre Anwendung so, dass jede Instanz einen Eintrag für jedes unterstützte Objekt erstellt, z. b. "/.:/ Printer/laser1 "and"/.:/ Drucker/Laser2. "</span><span class="sxs-lookup"><span data-stu-id="bd970-121">Instead of exporting bindings to one entry, design your application so that each instance creates an entry for each supported object, such as "/.:/printers/Laser1" and "/.:/printers/Laser2."</span></span> <span data-ttu-id="bd970-122">Dann soll der Server seine Bindungs Handles in jeden Server Eintrag Exportieren, zusammen mit dem für diesen Eintrag relevanten Objekt UUID.</span><span class="sxs-lookup"><span data-stu-id="bd970-122">Then, have the server export its binding handles to each server entry, along with the object UUID relevant to that entry.</span></span>

<span data-ttu-id="bd970-123">In diesem Fall kann ein Client eine Ressource anhand des Namens suchen, indem Sie den entsprechenden Server Eintrag importiert. die Objekt-UUID der Ressource ist nicht erforderlich.</span><span class="sxs-lookup"><span data-stu-id="bd970-123">In this case, a client can look up a resource by name by importing from the relevant server entry; it does not require the object UUID of the resource.</span></span> <span data-ttu-id="bd970-124">Wenn Sie über die Ressourcen-UUID, aber nicht über den Namen verfügt, kann Sie aus dem **null** -Eintrag importiert werden.</span><span class="sxs-lookup"><span data-stu-id="bd970-124">If it has the resource UUID but not the name, it can import from the **null** entry.</span></span>
