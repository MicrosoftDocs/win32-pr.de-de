# <a name="asynchronous-rpc"></a><span data-ttu-id="f7ee9-105">Asynchroner RPC</span><span class="sxs-lookup"><span data-stu-id="f7ee9-105">Asynchronous RPC</span></span>

<span data-ttu-id="f7ee9-106">Bei asynchronem Remote Prozedur Aufruf (RPC) handelt es sich um eine Microsoft-Erweiterung, die verschiedene Einschränkungen des herkömmlichen RPC-Modells behandelt, wie von der Open Software Foundation – verteilten Computing-Umgebung (OSF-DCE) definiert.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-106">Asynchronous Remote Procedure Call (RPC) is a Microsoft extension that addresses several limitations of the traditional RPC model as defined by the Open Software Foundation–Distributed Computing Environment (OSF-DCE).</span></span> <span data-ttu-id="f7ee9-107">Der asynchrone RPC trennt einen Remote Prozedur Aufruf von seinem Rückgabewert, der die folgenden Einschränkungen von herkömmlichem, synchronem RPC auflöst:</span><span class="sxs-lookup"><span data-stu-id="f7ee9-107">Asynchronous RPC separates a remote procedure call from its return value, which resolves the following limitations of traditional, synchronous RPC:</span></span>

-   <span data-ttu-id="f7ee9-108">**Mehrere ausstehende Aufrufe von einem Single Thread-Client.**</span><span class="sxs-lookup"><span data-stu-id="f7ee9-108">**Multiple outstanding calls from a single-threaded client.**</span></span> <span data-ttu-id="f7ee9-109">Im herkömmlichen RPC-Modell wird ein Client in einem Remote Prozedur Aufruf blockiert, bis der Aufruf zurückgegeben wird.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-109">In the traditional RPC model, a client is blocked in a remote procedure call until the call returns.</span></span> <span data-ttu-id="f7ee9-110">Dadurch wird verhindert, dass ein Client mehrere ausstehende Aufrufe hat, während der Thread weiterhin für andere Aufgaben verfügbar ist.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-110">This prevents a client from having multiple outstanding calls, while still having its thread available to do other work.</span></span>
-   <span data-ttu-id="f7ee9-111">**Langsame oder verzögerte Clients.**</span><span class="sxs-lookup"><span data-stu-id="f7ee9-111">**Slow or delayed clients.**</span></span> <span data-ttu-id="f7ee9-112">Ein Client, der langsam Daten erzeugt, möchte möglicherweise einen Remote Prozedur Aufrufvorgang mit anfänglichen Daten ausführen und dann bei der Erstellung zusätzliche Daten bereitstellen.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-112">A client that is slow to produce data may want to make a remote procedure call with initial data and then supply additional data as it is produced.</span></span> <span data-ttu-id="f7ee9-113">Dies ist bei herkömmlichem (synchronen) RPC nicht möglich.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-113">This is not possible with conventional (synchronous) RPC.</span></span>
-   <span data-ttu-id="f7ee9-114">**Langsame oder verzögerte Server.**</span><span class="sxs-lookup"><span data-stu-id="f7ee9-114">**Slow or delayed servers.**</span></span> <span data-ttu-id="f7ee9-115">Ein Remote Prozedur Aufruf, der lange Zeit in Anspruch nimmt, bindet den dispatchthread für die Dauer der Aufgabe ein.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-115">A remote procedure call that takes a long time to complete will tie up the dispatch thread for the duration of the task.</span></span> <span data-ttu-id="f7ee9-116">Mit dem asynchronen RPC kann der Server einen separaten (asynchronen) Vorgang starten, um die Anforderung zu verarbeiten und die Antwort zurücksenden, wenn Sie verfügbar ist.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-116">With asynchronous RPC, the server can start a separate (asynchronous) operation to process the request and send back the reply when it is available.</span></span> <span data-ttu-id="f7ee9-117">Der Server kann die Antwort auch inkrementell senden, wenn die Ergebnisse verfügbar werden, ohne dass ein Verteilungs Thread für die Dauer des Remote Aufrufes gebunden werden muss.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-117">The server can also send the reply incrementally as results become available without having to tie up a dispatch thread for the duration of the remote call.</span></span> <span data-ttu-id="f7ee9-118">Durch die asynchrone Client Anwendung können Sie verhindern, dass ein langsamer Server eine Client Anwendung unnötig bindet.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-118">By making the client application asynchronous, you can prevent a slow server from unnecessarily tying up a client application.</span></span>
-   <span data-ttu-id="f7ee9-119">**Übertragung großer Datenmengen.**</span><span class="sxs-lookup"><span data-stu-id="f7ee9-119">**Transfer of large amounts of data.**</span></span> <span data-ttu-id="f7ee9-120">Bei der Übertragung großer Datenmengen zwischen Client und Server (vor allem über langsame Verbindungen) werden sowohl der Client Thread als auch der Server-Manager-Thread für die Dauer der Übertragung verbunden.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-120">Transferring large amounts of data between the client and server, especially over slow links, ties up both the client thread and the server manager thread for the duration of the transfer.</span></span> <span data-ttu-id="f7ee9-121">Bei asynchronem RPC und Pipes kann die Datenübertragung inkrementell erfolgen, ohne dass der Client oder der Server andere Tasks ausführen muss.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-121">With asynchronous RPC and pipes, data transfer can take place incrementally, and without blocking the client or server from performing other tasks.</span></span>

<span data-ttu-id="f7ee9-122">Sie nutzen asynchrone RPC-Mechanismen, indem Sie Funktionen mit dem \[ [**Async**](/windows/desktop/Midl/async) -Attribut deklarieren \] .</span><span class="sxs-lookup"><span data-stu-id="f7ee9-122">You take advantage of asynchronous RPC mechanisms by declaring functions with the \[[**async**](/windows/desktop/Midl/async)\] attribute.</span></span> <span data-ttu-id="f7ee9-123">Da Sie diese Deklaration in einer Attribut Konfigurationsdatei (ACF) vornehmen, müssen Sie keine Änderungen an der IDL-Datei (Interface Definition Language) vornehmen. asynchronen RPC hat keine Auswirkung auf das Wire-Protokoll (wie die Daten zwischen Client und Server übertragen werden).</span><span class="sxs-lookup"><span data-stu-id="f7ee9-123">Because you make this declaration in an attribute configuration file (ACF), you do not have to make any changes to the Interface Definition Language (IDL) file; asynchronous RPC has no effect on the wire protocol (how the data is transmitted between client and server).</span></span> <span data-ttu-id="f7ee9-124">Dies bedeutet, dass sowohl synchrone als auch asynchrone Clients mit einer asynchronen Serveranwendung kommunizieren können.</span><span class="sxs-lookup"><span data-stu-id="f7ee9-124">This means that both synchronous and asynchronous clients can communicate with an asynchronous server application.</span></span>
