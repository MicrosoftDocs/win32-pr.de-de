# <a name="configuring-the-name-service-provider"></a>Konfigurieren des Name Service-Anbieters

Wenn Ihre verteilte Anwendung ihre Schnittstelle bei einem Namensdienst registriert, müssen sowohl der Client als auch der Server den gleichen Namensdienst verwenden. Microsoft RPC interagiert mit Microsoft Locator und einem beliebigen namens Dienstanbieter (NSP), der der Microsoft RPC Name Service Interface (NSI) entspricht – beispielsweise der DCE-Zell Verzeichnisdienst, auf den über den Namensdienst Daemon (nsid) der Digital Equipment Corporation zugegriffen wird. Der Microsoft-Locator, der für die Verwendung in Windows-Umgebungen entwickelt wurde, ist der Standardname Service-Anbieter.

In diesem Abschnitt wird erläutert, wie Sie namens Dienstanbieter installieren und konfigurieren:

-   [Konfigurieren des Namens Dienstanbieter](configuring-the-name-service-for-windows-xp-windows-2000-or-windows-nt.md)
-   [Konfigurieren des Namens Dienstanbieter für Windows 95](configuring-the-name-service-for-windows-95.md)
-   [Konfigurieren des Namens Dienstanbieter für Windows 3. x oder MS-DOS](configuring-the-name-service-for-windows-3-x-or-ms-dos.md)
-   [Starten und Beenden von Microsoft Locator](starting-and-stopping-microsoft-locator.md)
