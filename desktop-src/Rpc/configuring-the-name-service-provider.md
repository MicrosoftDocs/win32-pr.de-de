# <a name="configuring-the-name-service-provider"></a>Konfigurieren des Namensdienstanbieters

Wenn Ihre verteilte Anwendung ihre Schnittstelle bei einem Namensdienst registriert, müssen sowohl der Client als auch der Server denselben Namensdienst verwenden. Microsoft RPC interagiert mit Microsoft Locator und allen Namendienstanbietern (Name Service Provider, NSP), die der Microsoft RPC Name Service Interface (NSI) folgen, z. B. dem DCE Cell Directory-Dienst, auf den über den Namensdienst-Daemon (NSID) der Digital Equipment Corporation zugegriffen wird. Der Microsoft Locator, der für die Verwendung in Windows ist, ist der Standardnamendienstanbieter.

In diesem Abschnitt wird das Installieren und Konfigurieren von Namensdienstanbietern erläutert:

-   [Konfigurieren des Namensdiensts](configuring-the-name-service-for-windows-xp-windows-2000-or-windows-nt.md)
-   [Konfigurieren des Namensdiensts für Windows 95](configuring-the-name-service-for-windows-95.md)
-   [Konfigurieren des Namensdiensts für Windows 3.x oder MS-DOS](configuring-the-name-service-for-windows-3-x-or-ms-dos.md)
-   [Starten und Beenden des Microsoft-Locators](starting-and-stopping-microsoft-locator.md)
