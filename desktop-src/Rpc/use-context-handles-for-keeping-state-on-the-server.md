# <a name="use-context-handles-for-keeping-state-on-the-server"></a>Verwenden von Kontexthandles zum Behalten des Zustands auf dem Server

**Bewährte Methode:** Verwenden Sie Kontexthandles, wenn der Zustand zwischen Aufrufen auf dem Server beibehalten wird.

Kontexthandles sind für neue RPC-Programmierer oft einschüchternd, und der Mehraufwand von Lernkontexthandles scheint sich zunächst nicht lohnt. Es lohnt sich, da Kontexthandles integrierte Lösungen für viele häufige Fehler in der Netzwerkprogrammierung aufweisen, die andernfalls von Grund auf neu implementiert werden müssten. Das Verstehen von Kontexthandles lohnt sich später.
