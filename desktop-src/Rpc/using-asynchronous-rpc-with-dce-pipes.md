# <a name="using-asynchronous-rpc-with-dce-pipes"></a>Verwenden von asynchronen RPC mit DCE Pipes

Microsoft RPC unterstützt die Verwendung von DCE-Pipes. Obwohl sie einen ähnlichen Namen haben, stehen DCE-Pipes nicht mit [Named Pipes](asynchronous-rpc-over-the-named-pipe-protocol.md)in Zusammenhang. Named Pipes sind ein Transportprotokoll. DCE-Pipes sind eine protokollunabhängige Methode der Client-/Serverkommunikation.

In diesem Abschnitt wird RPC mit asynchronen DCE-Pipes erläutert. Es ist in die folgenden Themen unterteilt:

-   [Asynchrone Pipes](asynchronous-pipes.md)
-   [Deklarieren asynchroner Pipes](declaring-asynchronous-pipes.md)
-   [Clientseitige asynchrone Pipebehandlung](client-side-asynchronous-pipe-handling.md)
-   [Serverseitige asynchrone Pipebehandlung](server-side-asynchronous-pipe-handling.md)

Ein Beispiel für eine asynchrone Pipe-RPC-Anwendung finden Sie im FileRep-Beispiel im Platform Software Development Kit (SDK).
