# <a name="using-asynchronous-rpc-with-dce-pipes"></a>Verwenden von asynchronem RPC mit DCE-Pipes

Microsoft RPC unterstützt die Verwendung von DCE-Pipes. Obwohl Sie einen ähnlichen Namen haben, sind DCE-Pipes nicht mit [Named Pipes](asynchronous-rpc-over-the-named-pipe-protocol.md)verknüpft. Named Pipes sind ein Transportprotokoll. DCE-Pipes sind eine Protokoll unabhängige Methode der Client/Server-Kommunikation.

In diesem Abschnitt wird die RPC-Verwendung von asynchronen DCE-Pipes erläutert. Es ist in die folgenden Themen unterteilt:

-   [Asynchrone Pipes](asynchronous-pipes.md)
-   [Deklarieren von asynchronen Pipes](declaring-asynchronous-pipes.md)
-   [Client seitige asynchrone Pipe-Verarbeitung](client-side-asynchronous-pipe-handling.md)
-   [Server seitige asynchrone Pipe-Verarbeitung](server-side-asynchronous-pipe-handling.md)

Eine Beispiel-RPC-Anwendung für eine asynchrone Pipe finden Sie im filerep-Beispiel im Platform Software Development Kit (SDK).
