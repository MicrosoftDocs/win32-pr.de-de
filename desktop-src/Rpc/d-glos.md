# <a name="d-rpc"></a>D (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) D [E](e-glos.md) [F](f-glos.md) G H [I](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_datagram_glos"></span><span id="_RPC_DATAGRAM_GLOS"></span>**Datagramm**
</dt> <dd>

Kommunikationsprotokoll oder Transport, in dem Datenpakete unabhängig voneinander weitergeleitet werden. Sie können verschiedenen Routen folgen und in einer anderen Reihenfolge eintreffen, aus der sie gesendet wurden. [*UDP*](u-glos.md) und [*IPX*](i-glos.md) sind Beispiele für Transport layer-datagram-Protokolle. Siehe auch [*verbindungsorientierte*](c-glos.md).

</dd> <dt>

<span id="_rpc_discriminant_glos"></span><span id="_RPC_DISCRIMINANT_GLOS"></span>**diskriminant**
</dt> <dd>

Variable, die die Datentypen angibt, die in einer Union gespeichert werden können.

</dd> <dt>

<span id="_rpc_discriminated_union_glos"></span><span id="_RPC_DISCRIMINATED_UNION_GLOS"></span>**Unterscheidungsgewerkschaft**
</dt> <dd>

(oder *Variantendatensatz*) Union, die einen Diskriminator als Teil der Datenstruktur enthält, sodass der derzeit gültige Datentyp zusammen mit der Union übertragen wird. Siehe auch [*gekapselte Union*](e-glos.md), [*nicht gekapselte Union*](n-glos.md).

</dd> <dt>

<span id="_rpc_dce_glos"></span><span id="_RPC_DCE_GLOS"></span>**Distributed Computing Environment (DCE)**
</dt> <dd>

Spezifikation der Open Software Foundation für eine Reihe integrierter Dienste, einschließlich Remoteprozeduraufrufen, verteilten Dateisystemen und Sicherheitsdiensten. Der OSF-DCE RPC-Standard ist die Grundlage für Microsoft RPC.

</dd> <dt>

<span id="_rpc_dynamic_endpoint_glos"></span><span id="_RPC_DYNAMIC_ENDPOINT_GLOS"></span>**Dynamischer Endpunkt**
</dt> <dd>

Netzwerkspezifische Serveradresse, die zur Laufzeit angefordert und zugewiesen wird. Siehe auch [*den bekannten Endpunkt*](w-glos.md).

</dd> <dt>

<span id="_rpc_dynamic_identity_tracking_glos"></span><span id="_RPC_DYNAMIC_IDENTITY_TRACKING_GLOS"></span>**Dynamische Identitätsnachverfolgung**
</dt> <dd>

Spezifikation, dass die RPC-Laufzeitbibliothek die Anmeldeinformationen des aufrufenden Threads anstelle des Bindungshandle für die Authentifizierung jedes Mal verwendet, wenn der Client eine Remoteprozedur aufruft. Siehe auch [*statische Identitätsnachverfolgung.*](s-glos.md)

</dd> </dl>
