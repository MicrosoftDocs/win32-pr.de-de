# <a name="d-rpc"></a>D (RPC)

[a](a-glos.md) [B](b-glos.md) [C](c-glos.md) D [E](e-glos.md) [F](f-glos.md) G H [I](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_datagram_glos"></span><span id="_RPC_DATAGRAM_GLOS"></span>**Datagram**
</dt> <dd>

Kommunikationsprotokoll oder Transport, in dem Datenpakete unabhängig weitergeleitet werden. Sie können unterschiedliche Routen befolgen und in einer anderen Reihenfolge eintreffen, in der Sie gesendet wurden. [*UDP*](u-glos.md) und [*IPX*](i-glos.md) sind Beispiele für – Datagramm-Protokolle der Transportschicht. Siehe auch [*verbindungsorientiert*](c-glos.md).

</dd> <dt>

<span id="_rpc_discriminant_glos"></span><span id="_RPC_DISCRIMINANT_GLOS"></span>**Discriminant**
</dt> <dd>

Eine Variable, die die Datentypen angibt, die in einer Union gespeichert werden können.

</dd> <dt>

<span id="_rpc_discriminated_union_glos"></span><span id="_RPC_DISCRIMINATED_UNION_GLOS"></span>**Unterscheidungs-Union**
</dt> <dd>

(oder *Variant-Datensatz*) Union, die einen Diskriminator als Teil der Datenstruktur enthält, sodass der aktuell gültige Datentyp zusammen mit der Union übertragen wird. Siehe auch [*kapselte Union*](e-glos.md), [*nicht gekapselte Union*](n-glos.md).

</dd> <dt>

<span id="_rpc_dce_glos"></span><span id="_RPC_DCE_GLOS"></span>**Verteilte Computerumgebung (DCE)**
</dt> <dd>

Die Spezifikation der Open Software Foundation für eine Reihe integrierter Dienste, einschließlich Remote Prozedur aufrufen, verteilter Dateisysteme und Sicherheitsdiensten. Der RPC-Standard OSF-DCE ist die Grundlage für Microsoft RPC.

</dd> <dt>

<span id="_rpc_dynamic_endpoint_glos"></span><span id="_RPC_DYNAMIC_ENDPOINT_GLOS"></span>**dynamischer Endpunkt**
</dt> <dd>

Netzwerk spezifische –-Server Adresse, die zur Laufzeit angefordert und zugewiesen wird. Siehe auch [*bekannten Endpunkt*](w-glos.md).

</dd> <dt>

<span id="_rpc_dynamic_identity_tracking_glos"></span><span id="_RPC_DYNAMIC_IDENTITY_TRACKING_GLOS"></span>**dynamische Identitäts Verfolgung**
</dt> <dd>

Angabe, dass die RPC-Lauf Zeit Bibliothek die Anmelde Informationen des aufrufenden Threads anstelle des Bindungs Handles für die Authentifizierung verwendet, wenn der Client eine Remote Prozedur aufruft. Siehe auch [*statische Identitäts Überwachung*](s-glos.md).

</dd> </dl>
