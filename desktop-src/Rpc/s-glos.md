# <a name="s-rpc"></a>S (RPC)

<dl> <dt>

<span id="rpc.s_glos_1_gly"></span><span id="RPC.S_GLOS_1_GLY"></span>**Skalierbarkeit**
</dt> <dd>

Die Möglichkeit, die zusätzliche Verarbeitungsleistung auf einem Multiprozessorsystem (2, 4, 8, 32 oder mehr Prozessoren) vollständig zu nutzen. Skalierbarkeit ist auch die Möglichkeit, eine große Anzahl von Clients zu bedienen.

</dd> <dt>

<span id="_rpcl_spp_glos"></span><span id="_RPCL_SPP_GLOS"></span>**Sequenziertes Paket Protokoll (SPP)**
</dt> <dd>

[*Verbindungs orientiertes*](c-glos.md) Kommunikationsprotokoll von Banyan Vines für das Routing von Informationspaketen über lokale Netzwerke.

</dd> <dt>

<span id="_rpc_spx_glos"></span><span id="_RPC_SPX_GLOS"></span>**Sequenzierter Paket Austausch (SPX)**
</dt> <dd>

[*Verbindungs orientiertes*](c-glos.md) Novell NetWare-Kommunikationsprotokoll für das Routing von Informationspaketen über lokale Netzwerke und Wide Area Networks.

</dd> <dt>

<span id="_rpc_serialization_glos"></span><span id="_RPC_SERIALIZATION_GLOS"></span>**Serialisierung**
</dt> <dd>

Prozess des Marshalling von Daten zu (Encoding) und zum abmarshalling von Daten aus (decodierten) Puffern, die Sie steuern. Dies steht im Gegensatz zur herkömmlichen RPC-Verwendung, bei der die stubvorgänge und die RPC-Laufzeit die marshallingpuffer steuern. Wird auch als Pickeln bezeichnet. Siehe auch [*Procedure Serialisierung*](p-glos.md), [*Type Serialisierung*](t-glos.md).

</dd> <dt>

<span id="_rpc_server_stub_glos"></span><span id="_RPC_SERVER_STUB_GLOS"></span>**Server-Stub**
</dt> <dd>

Mittlerer l-generierter Quellcode in C-Sprache, der alle Funktionen enthält, die für die Serveranwendung erforderlich sind, um Remote Anforderungen mithilfe von lokalen Prozedur aufrufen verarbeiten zu können. Siehe auch [*Client-Stub*](c-glos.md).

</dd> <dt>

<span id="_rpc_session_glos"></span><span id="_RPC_SESSION_GLOS"></span>**Sitzung**
</dt> <dd>

Es wurde eine Beziehung zwischen einer Client Anwendung und einer Serveranwendung hergestellt. Siehe auch [*binden*](b-glos.md), [*Bindungs handle*](b-glos.md).

</dd> <dt>

<span id="_rpc_static_callback_function_glos"></span><span id="_RPC_STATIC_CALLBACK_FUNCTION_GLOS"></span>**statische Rückruffunktion**
</dt> <dd>

Remote Prozedur, die Teil der Clientseite einer verteilten Anwendung ist, die ein Server zum Abrufen von Informationen vom Client abrufen kann. Das \[ [Callback](/windows/desktop/Midl/callback) - \] Attribut bezeichnet eine statische Rückruffunktion.

</dd> <dt>

<span id="_rpc_static_identity_tracking_glos"></span><span id="_RPC_STATIC_IDENTITY_TRACKING_GLOS"></span>**statische Identitäts Überwachung**
</dt> <dd>

Die statische Identitäts Verfolgung gibt an, dass die RPC-Laufzeit die Sicherheits Anmelde Informationen im Bindungs Handle des Clients für alle RPC-Aufrufe verwendet. Siehe auch [*dynamische Identitäts Überwachung*](d-glos.md).

</dd> <dt>

<span id="_rpc_string_binding_glos"></span><span id="_RPC_STRING_BINDING_GLOS"></span>**Zeichen folgen Bindung**
</dt> <dd>

Zeichenfolge, die aus den Optionen für die Objekt- [*UUID*](u-glos.md), die [*Protokoll Sequenz*](p-glos.md), die [*Netzwerkadresse*](n-glos.md), den [*Endpunkt*](e-glos.md)und den Endpunkt besteht, die alle zum Erstellen eines [*Bindungs Handles*](b-glos.md) für den angegebenen Server verwendet werden können.

</dd> <dt>

<span id="_rpc_strong_typing_glos"></span><span id="_RPC_STRONG_TYPING_GLOS"></span>**starke Typisierung**
</dt> <dd>

Compilererzwingung der strikten Kontrolle über Datentypen. In mittlerer l und RPC wird die starke Typisierung verwendet, um sicherzustellen, dass Daten von verschiedenen Computern in einer verteilten Umgebung konsistent interpretiert werden.

</dd> </dl>
