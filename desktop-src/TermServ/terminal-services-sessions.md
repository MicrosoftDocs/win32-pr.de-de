# <a name="remote-desktop-sessions"></a><span data-ttu-id="b947f-103">Remotedesktop Sitzungen</span><span class="sxs-lookup"><span data-stu-id="b947f-103">Remote Desktop Sessions</span></span>

<span data-ttu-id="b947f-104">Wenn sich ein Benutzer an einem Remotedesktopdienste – aktivierten Computer anmeldet, wird eine Sitzung für den Benutzer gestartet.</span><span class="sxs-lookup"><span data-stu-id="b947f-104">When a user logs on to a Remote Desktop Services–enabled computer, a session is started for the user.</span></span> <span data-ttu-id="b947f-105">Jede Sitzung wird durch eine eindeutige Sitzungs-ID identifiziert.</span><span class="sxs-lookup"><span data-stu-id="b947f-105">Each session is identified by a unique session ID.</span></span> <span data-ttu-id="b947f-106">Da jede Anmeldung bei einem Remotedesktopverbindung-Client (RDC) eine separate Sitzungs-ID erhält, ist die Benutzer Darstellung ähnlich wie bei mehreren Computern gleichzeitig. beispielsweise ein Office-Computer und ein Heimcomputer.</span><span class="sxs-lookup"><span data-stu-id="b947f-106">Because each logon to a Remote Desktop Connection (RDC) client receives a separate session ID, the user-experience is similar to being logged on to multiple computers at the same time; for example, an office computer and a home computer.</span></span>

<span data-ttu-id="b947f-107">Jede Remote Desktop Sitzung ist einer interaktiven Fenster Station zugeordnet.</span><span class="sxs-lookup"><span data-stu-id="b947f-107">Each remote desktop session is associated with an interactive window station.</span></span> <span data-ttu-id="b947f-108">Der einzige unterstützte Fenster Stationsname für eine interaktive Fenster Station ist "WinSta0"; Daher ist jede Sitzung ihrer eigenen "WinSta0"-Fenster Station zugeordnet.</span><span class="sxs-lookup"><span data-stu-id="b947f-108">The only supported window station name for an interactive window station is "WinSta0"; therefore each session is associated with its own "WinSta0" window station.</span></span> <span data-ttu-id="b947f-109">Es gibt drei Standard Desktops für jede Fenster Station: den Desktop Winlogon, den Bildschirmschoner Desktop und den interaktiven Desktop.</span><span class="sxs-lookup"><span data-stu-id="b947f-109">There are three standard desktops for each window station: the Winlogon desktop, the screen saver desktop, and the interactive desktop.</span></span>

<span data-ttu-id="b947f-110">Der Benutzer, der der interaktiven Fenster Station für eine Sitzung zugeordnet ist, wird als *interaktiver Benutzer* bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="b947f-110">The user associated with the interactive window station for a session is known as the *interactive user*.</span></span> <span data-ttu-id="b947f-111">Auf einem Remotedesktopverbindung-Client (RDC) können neben dem interaktiven Benutzer in der Remotedesktopdienste Konsole mehrere interaktive Benutzer vorhanden sein.</span><span class="sxs-lookup"><span data-stu-id="b947f-111">On a Remote Desktop Connection (RDC) client there can be multiple interactive users in addition to the interactive user on the Remote Desktop Services console.</span></span> <span data-ttu-id="b947f-112">Um den Bezeichner der aktuell an die Konsole angefügten Sitzung abzurufen, verwenden Sie die [**wzgetactiveconsolesessionid**](/windows/desktop/api/Winbase/nf-winbase-wtsgetactiveconsolesessionid) -Funktion.</span><span class="sxs-lookup"><span data-stu-id="b947f-112">To retrieve the identifier of the session currently attached to the console, use the [**WTSGetActiveConsoleSessionId**](/windows/desktop/api/Winbase/nf-winbase-wtsgetactiveconsolesessionid) function.</span></span>

<span data-ttu-id="b947f-113">Wenn ein Benutzer sich von einem Remotedesktopverbindung-Client (RDC) abmeldet, wird die Sitzung, die der Client auf dem Remotedesktop-Sitzungshost (RD-Sitzungshost)-Server (früher als Terminal Server bezeichnet) hat, gelöscht, und die dieser Sitzung zugeordneten Fenster Stationen und Desktops werden entfernt.</span><span class="sxs-lookup"><span data-stu-id="b947f-113">When a user logs off from a Remote Desktop Connection (RDC) client, the session that the client has on the Remote Desktop Session Host (RD Session Host) server (formerly known as a terminal server) is deleted and the window stations and desktops associated with that session are removed.</span></span> <span data-ttu-id="b947f-114">Da die Remotedesktopdienste Konsolen Sitzung jedoch nie gelöscht wird, werden die der Konsolen Sitzung zugeordneten Fenster Stationen nicht gelöscht.</span><span class="sxs-lookup"><span data-stu-id="b947f-114">However, because the Remote Desktop Services console session is never deleted, the window stations associated with the console session are not deleted.</span></span> <span data-ttu-id="b947f-115">Dies wirkt sich darauf aus, wie sich Anwendungen in einer Remotedesktopdienste Umgebung Verhalten, wenn Sie für die Ausführung im Sicherheitskontext des interaktiven Benutzers konfiguriert sind, der auch als "runas Interactive User"-Objekt Aktivierungs Modus bezeichnet wird.</span><span class="sxs-lookup"><span data-stu-id="b947f-115">This affects how applications behave in a Remote Desktop Services environment when they are configured to run in the security context of the interactive user, also known as the "RunAs Interactive User" object activation mode.</span></span>

<span data-ttu-id="b947f-116">Weitere Informationen zum Starten eines lokalen Server Prozesses für eine bestimmte Sitzung finden Sie unter [Sitzungs-zu-Sitzung-Aktivierung mit einem sitzungsmoniker](session-to-session-activation-with-a-session-moniker.md) und [Verwenden eines sitzungsmonikers](using-a-session-moniker.md).</span><span class="sxs-lookup"><span data-stu-id="b947f-116">For more information about starting a local server process on a specified session, see [Session-to-Session Activation with a Session Moniker](session-to-session-activation-with-a-session-moniker.md) and [Using a Session Moniker](using-a-session-moniker.md).</span></span> <span data-ttu-id="b947f-117">Weitere Informationen zu Sicherheits Kontexten finden Sie [im Sicherheitskontext des Clients](/windows/desktop/SecAuthZ/the-client-security-context).</span><span class="sxs-lookup"><span data-stu-id="b947f-117">For more information about security contexts, see [The Client's Security Context](/windows/desktop/SecAuthZ/the-client-security-context).</span></span>
