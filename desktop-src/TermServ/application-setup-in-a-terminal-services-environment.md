# <a name="application-setup"></a><span data-ttu-id="fb6ea-103">Anwendungseinrichtung</span><span class="sxs-lookup"><span data-stu-id="fb6ea-103">Application setup</span></span>

<span data-ttu-id="fb6ea-104">Das automatisierte Setup Verfahren für viele vorhandene Anwendungen setzt voraus, dass die Anwendung für einen einzelnen Benutzer installiert wird.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-104">The automated setup procedure for many existing applications assumes that the application is being installed for a single user.</span></span> <span data-ttu-id="fb6ea-105">In einer mehr Benutzer Remotedesktopdienste Umgebung kann diese Annahme folgende Probleme verursachen:</span><span class="sxs-lookup"><span data-stu-id="fb6ea-105">In a multiuser Remote Desktop Services environment, this assumption can create the following problems:</span></span>

-   <span data-ttu-id="fb6ea-106">Wenn das Setup Verfahren die Registrierung und die Desktopumgebung nur für einen Benutzer aktualisiert, müssen zusätzliche Benutzer das gesamte Paket neu installieren, oder ein Administrator muss manuell Informationen aus der Registrierung und dem Desktop eines Benutzers an die anderen Benutzer kopieren.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-106">If the setup procedure updates the registry and desktop environment for just one user, additional users must reinstall the entire package or an administrator must manually copy information from the registry and desktop of one user to the other users.</span></span>
-   <span data-ttu-id="fb6ea-107">Mit einigen Setup Verfahren können Sie die Anwendung zum Zeitpunkt der Installation anpassen, indem Sie Features ausschließen.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-107">With some setup procedures, you can customize the application at installation time by excluding features.</span></span> <span data-ttu-id="fb6ea-108">Wenn der anfängliche Installer einen Teil der Anwendung ausschließt, müssen zusätzliche Benutzer die Anwendung neu installieren, um die ausgeschlossenen Features zu erhalten.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-108">If the initial installer excludes part of the application, additional users must reinstall the application to get the excluded features.</span></span>

<span data-ttu-id="fb6ea-109">Um diese Probleme zu vermeiden, sollten beim Installieren einer Anwendung auf einem Remotedesktop-Sitzungshost-Server (RD-Sitzungshost) die folgenden Richtlinien beachtet werden:</span><span class="sxs-lookup"><span data-stu-id="fb6ea-109">To avoid these problems, setup procedures should use the following guidelines when installing an application on a Remote Desktop Session Host (RD Session Host) server:</span></span>

-   <span data-ttu-id="fb6ea-110">Installieren Sie Anwendungen in der Standardbenutzer Umgebung, die allen Benutzern gemeinsam ist.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-110">Install applications into the default user environment common to all users.</span></span> <span data-ttu-id="fb6ea-111">Führen Sie vor dem Installieren der Anwendung den Befehl **Change user/install** Console aus, und führen Sie nach Abschluss der Installation den Befehl **Change user/execute** Console aus.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-111">Before installing the application, execute the **change user /install** console command, and after installation is complete, execute the **change user /execute** console command.</span></span> <span data-ttu-id="fb6ea-112">Verwenden Sie ein Remotedesktopdienste Kompatibilitäts Skript für die-Installation.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-112">Use a Remote Desktop Services compatibility script for the installation.</span></span>
-   <span data-ttu-id="fb6ea-113">Unterstützung der benutzerspezifischen Anpassung durch die Verwendung von Benutzerprofilen.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-113">Support user-specific customization through the use of user profiles.</span></span> <span data-ttu-id="fb6ea-114">Erstellen Sie hierzu ein Format für [Administrative Vorlagen Dateien](/previous-versions/windows/desktop/Policy/administrative-template-file-format) , damit ein Administrator die Registrierung so konfigurieren kann, dass die für die einzelnen Benutzer verfügbaren Features angezeigt werden.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-114">To do this, create an [Administrative Template File Format](/previous-versions/windows/desktop/Policy/administrative-template-file-format) so an administrator can configure the registry to indicate the features available to each user.</span></span> <span data-ttu-id="fb6ea-115">Die Anwendung kann dann zur Laufzeit Features aktivieren oder deaktivieren, abhängig von den Einstellungen in den Registrierungs Einstellungen des aktuellen Benutzers.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-115">Then, at run time, the application can enable or disable features depending on the settings in the current user's registry settings.</span></span> <span data-ttu-id="fb6ea-116">Die Anwendung kann die Konfiguration pro Benutzer in der Registrierungs Struktur des **aktuellen HKEY-Benutzers** speichern und jedem Benutzer die Konfiguration der Anwendung entsprechend Ihren Einstellungen gestatten.</span><span class="sxs-lookup"><span data-stu-id="fb6ea-116">The application can store the per user configuration in the **HKEY CURRENT USER** registry hive and let every user configure the application according to their preferences.</span></span>
