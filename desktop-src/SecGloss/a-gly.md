<span id="_security_access_control_list_gly"></span><span id="_SECURITY_ACCESS_CONTROL_LIST_GLY"></span>**Zugriffs Steuerungs Liste**
</dt> <dd>

ACL Eine Liste von Sicherheitsmaßnahmen, die auf ein Objekt angewendet werden. (Bei einem Objekt kann es sich um eine Datei, einen Prozess, ein Ereignis oder eine beliebige andere Sicherheits Beschreibung handeln.) Ein Eintrag in einer Zugriffs Steuerungs Liste (Access Control List, ACL) ist ein Zugriffs Steuerungs Eintrag (ACE). Es gibt zwei Typen von Zugriffs Steuerungs Listen, "diskreten" und "System".

Siehe auch *Zugriffs Steuerungs Eintrag*, freigegebene [*Zugriffs Steuerungs Liste*](d-gly.md), [*Sicherheits Beschreibung*](s-gly.md)und [*System Zugriffs Steuerungs Liste*](s-gly.md).

<span id="_security_access_mask_gly"></span><span id="_SECURITY_ACCESS_MASK_GLY"></span>**Zugriffs Maske**

Siehe auch *Zugriffs Steuerungs Eintrag*.

</dd> <dt>

<span id="_security_access_token_gly"></span><span id="_SECURITY_ACCESS_TOKEN_GLY"></span>**Zugriffs Token**
</dt> <dd>

Ein Zugriffstoken enthält die Sicherheitsinformationen für eine Anmeldesitzung. Wenn sich ein Benutzer anmeldet, wird vom System ein Zugriffstoken erstellt. Alle Prozesse, die vom Benutzer ausgelöst werden, weisen dann eine Kopie des Tokens auf. Das Token identifiziert den Benutzer, die Gruppen des Benutzers und die Rechte des Benutzers. Mithilfe des Tokens steuert das System den Zugriff auf sicherungsfähige Objekte sowie die Durchführung von Systemvorgängen auf dem lokalen Computer durch den Benutzer. Es gibt zwei Arten von Zugriffs Token: Primär und Identitätswechsel.

Siehe auch Identitätswechsel [*Token*](i-gly.md), [*primäres Token*](p-gly.md), [*Privileg*](p-gly.md), [*Prozess*](p-gly.md)und [*Sicherheits*](s-gly.md)-ID.

<span id="_security_asymmetric_key_gly"></span><span id="_SECURITY_ASYMMETRIC_KEY_GLY"></span>**asymmetrischer Schlüssel**
</dt> <dd>

Eines der Schlüsselpaare, die mit einem asymmetrischen kryptografischen Algorithmus verwendet werden. Ein solcher Algorithmus verwendet zwei kryptografische Schlüssel: einen "öffentlichen Schlüssel" für die Verschlüsselung und einen "privaten Schlüssel" für die Entschlüsselung. Bei der Signatur und Überprüfung werden die Rollen umgekehrt: der öffentliche Schlüssel wird zur Überprüfung verwendet, und der private Schlüssel wird für die Signatur Generierung verwendet. Die wichtigste Funktion solcher Algorithmen besteht darin, dass Ihre Sicherheit nicht von der Beibehaltung des geheimen Schlüssels des öffentlichen Schlüssels abhängt (es kann jedoch eine gewisse Gewissheit der Authentizität öffentlicher Schlüssel erforderlich sein, z. b., dass Sie von einer vertrauenswürdigen Quelle abgerufen werden). Der private Schlüssel muss geheim geheim gehalten werden. Beispiele für Algorithmen mit öffentlichem Schlüssel sind [*DSA (Digital Signature Algorithmus)*](d-gly.md), ECDSA (Elliptic Curve Digital Signature Algorithmus) und die Familie Rivest-Shamir-Adleman (RSA) der Algorithmen.

</dd> <dt>

Das Authentifizierungs Paket überprüft dann die Anmelde Informationen und authentifiziert oder lehnt den Anmeldeversuch des Benutzers ab.

</dd> <dt>

<span id="_security_authenticode_gly"></span><span id="_SECURITY_AUTHENTICODE_GLY"></span>**Authenticode**

Eine Sicherheitsfunktion von Internet Explorer. Mit Authenticode können Anbieter von herunter ladbarem ausführbarem Code (z. b. Plug-ins oder ActiveX-Steuerelemente) digitale Zertifikate an Ihre Produkte anfügen, um Endbenutzern zu ermöglichen, dass der Code vom ursprünglichen Entwickler ist und nicht geändert wurde. Mit Authenticode können sich Endbenutzer selbst entscheiden, ob Softwarekomponenten, die vor dem Herunterladen im Internet bereitgestellt werden, angenommen oder abgelehnt werden sollen.
