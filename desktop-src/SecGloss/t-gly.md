<dl> <dt>

<span id="_security_t_0_protocol_gly"></span><span id="_SECURITY_T_0_PROTOCOL_GLY"></span>**T = 0 Protokoll**
</dt> <dd>

Ein asynchrones, Zeichen orientiertes, Halbduplex Übertragungsprotokoll.

</dd> <dt>

<span id="_security_t_1_protocol_gly"></span><span id="_SECURITY_T_1_PROTOCOL_GLY"></span>**T = 1 Protokoll**
</dt> <dd>

Ein asynchrones, Block orientiertes Halbduplex-Übertragungsprotokoll.

</dd> <dt>

<span id="_security_terminal_gly"></span><span id="_SECURITY_TERMINAL_GLY"></span>**Vergütungen**
</dt> <dd>

Eine Kombination aus Monitor-, Tastatur-, Maus-und zusammengestellten Peripheriegeräten, z. b. Smartcard-Lesern. Mehrere Prozesse können einem einzelnen Terminal zugeordnet werden, aber nur ein Prozess steuert das Terminal zu einem beliebigen Zeitpunkt.

</dd> <dt>

<span id="_security_three_phase_key_exchange_protocol_gly"></span><span id="_SECURITY_THREE_PHASE_KEY_EXCHANGE_PROTOCOL_GLY"></span>**dreiphasige Schlüsselaustausch Protokoll**
</dt> <dd>

Ein Protokoll, das verwendet wird, um eine authentifizierte und verschlüsselte Verbindung zwischen zwei Benutzern in einem nicht sicheren Netzwerk zu generieren. Benutzer tauschen eine Reihe von Nachrichten aus, um ein paar von Verschlüsselungsschlüsseln auszuhandeln. Der Absender verwendet einen Schlüssel zum Verschlüsseln von Nachrichten, die an den Empfänger gesendet werden, und der andere wird vom Empfänger verwendet, um die an den Absender gesendeten Nachrichten zu verschlüsseln. Mit diesem Protokoll wird sichergestellt, dass beide Benutzer aktiv sind und Nachrichten direkt an einander senden.

</dd> <dt>

<span id="_security_thumbprint_gly"></span><span id="_SECURITY_THUMBPRINT_GLY"></span>**Fingerabdruck**
</dt> <dd>

Der [*SHA-1-*](s-gly.md) Hash eines Zertifikats.

</dd> <dt>

<span id="_security_tls_gly"></span><span id="_SECURITY_TLS_GLY"></span>**TLS**
</dt> <dd>

Siehe *Transport Layer Security Protokoll*.

</dd> <dt>

<span id="_security_token_gly"></span><span id="_SECURITY_TOKEN_GLY"></span>**toben**
</dt> <dd>

Siehe [*Zugriffs Token*](a-gly.md).

</dd> <dt>

<span id="_security_transaction_gly"></span><span id="_SECURITY_TRANSACTION_GLY"></span>**Geschäfte**
</dt> <dd>

Ein Vorgang, mit dem mehrere Interaktionen als einzelner Vorgang durchgeführt werden können. Zwischenschritte werden erst ausgeführt, wenn alle Interaktionen erfolgreich abgeschlossen wurden. Wenn eine Interaktion fehlschlägt, werden alle Schritte an die ursprüngliche Einstellung zurückgegeben.

</dd> <dt>

<span id="_security_transport_layer_gly"></span><span id="_SECURITY_TRANSPORT_LAYER_GLY"></span>**Transport Ebene**
</dt> <dd>

Die Netzwerkebene für die Servicequalität und die exakte Informationsübermittlung. Auf dieser Ebene werden u. a. Fehler erkannt und korrigiert.

</dd> <dt>

<span id="_security_transport_layer_security_protocol_gly"></span><span id="_SECURITY_TRANSPORT_LAYER_SECURITY_PROTOCOL_GLY"></span>**Transport Layer Security-Protokoll**
</dt> <dd>

TLS Ein Protokoll, das den Datenschutz und die Sicherheit zwischen zwei Anwendungen bereitstellt, die über ein Netzwerk kommunizieren.

</dd> <dt>

<span id="_security_triple_des_gly"></span><span id="_SECURITY_TRIPLE_DES_GLY"></span>**Dreifache des**
</dt> <dd>

Eine Variation des des-Blockchiffre Algorithmus, der Klartext mit einem Schlüssel verschlüsselt, den resultierenden Chiffre Text mit einem zweiten Schlüssel verschlüsselt und schließlich das Ergebnis der zweiten Verschlüsselung mit einem dritten Schlüssel verschlüsselt. Triple des ist ein symmetrischer Algorithmus, der den gleichen Algorithmus und die gleichen Schlüssel für die Verschlüsselung und Entschlüsselung verwendet.

</dd> <dt>

<span id="_security_trustee_gly"></span><span id="_SECURITY_TRUSTEE_GLY"></span>**Stiftungs**
</dt> <dd>

Das Benutzerkonto, das Gruppenkonto oder die Anmelde Sitzung, auf die ein Zugriffs Steuerungs Eintrag angewendet wird. Jeder ACE in einer Zugriffs Steuerungs Liste (Access Control List, ACL) gilt für einen Vertrauens nehmer.

Siehe auch [*Zugriffs Steuerungs Eintrag*](a-gly.md) und [*Zugriffs Steuerungs Liste*](a-gly.md).

</dd> <dt>

<span id="_security_trust_list_gly"></span><span id="_SECURITY_TRUST_LIST_GLY"></span>**Vertrauens Liste**
</dt> <dd>

Siehe [*Zertifikat Vertrauens Liste*](c-gly.md).

</dd> <dt>

<span id="_security_trust_provider_gly"></span><span id="_SECURITY_TRUST_PROVIDER_GLY"></span>**Vertrauens Anbieter**
</dt> <dd>

Die Software, die entscheidet, ob eine Datei als vertrauenswürdig eingestuft wird. Diese Entscheidung basiert auf dem Zertifikat, das der Datei zugeordnet ist.

</dd> </dl>
