<span id="_security_backup_authority_gly"></span><span id="_SECURITY_BACKUP_AUTHORITY_GLY"></span>**Sicherungs Autorität**

Ein Typ von Daten, die in einer PKCS 7-Meldung enthalten sind \# . Basis Inhaltstypen enthalten nur Daten, keine kryptografischen Erweiterungen wie Hashes oder Signaturen. Derzeit ist der einzige Basis Inhaltstyp der Daten Inhaltstyp.

<span id="_security_base_cryptographic_functions_gly"></span><span id="_SECURITY_BASE_CRYPTOGRAPHIC_FUNCTIONS_GLY"></span>**grundlegende Kryptografiefunktionen**
</dt> <dd>

Die niedrigste Funktionsebene in der CryptoAPI-Architektur. Sie werden von Anwendungen und anderen kryptoapi-Funktionen auf hoher Ebene verwendet, um den Zugriff auf vom CSP bereitgestellte Kryptografiealgorithmen, eine sichere Schlüsselgenerierung und eine sichere Speicherung von geheimen Schlüsseln bereitzustellen.

Siehe auch [*Kryptografiedienstanbieter*](c-gly.md).

</dd> <dt>

<span id="_security_basic_encoding_rules_gly"></span><span id="_SECURITY_BASIC_ENCODING_RULES_GLY"></span>**Grundlegende Codierungsregeln**
</dt> <dd>

Mini Der Regelsatz, der zum Codieren von ASN. 1-Daten in einen Stream von Bits (Nullen oder solche) für die externe Speicherung oder Übertragung verwendet wird. Ein einzelnes ASN. 1-Objekt kann über mehrere äquivalente ber-Codierungen verfügen. BER wird in der CCITT-Empfehlung X. 209 definiert. Dies ist eine der beiden Codierungs Methoden, die derzeit von CryptoAPI verwendet werden.
