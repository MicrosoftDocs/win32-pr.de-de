# <a name="h-security-glossary"></a>H (Sicherheits Glossar)

<dl> <dt>

<span id="_security_handle_gly"></span><span id="_SECURITY_HANDLE_GLY"></span>**bewältigen**
</dt> <dd>

Ein Token, das verwendet wird, um ein Objekt zu identifizieren oder darauf zuzugreifen, z. b. das Handle eines Kryptografieanbieters, eines Zertifikat Speicher, einer Nachricht oder eines Schlüssel Paars.

</dd> <dt>

<span id="_security_hash_gly"></span><span id="_SECURITY_HASH_GLY"></span>**Hash**
</dt> <dd>

Ein Ergebnis mit fester Größe, das durch Anwenden einer mathematischen Funktion (des *Hash Algorithmus*) auf eine beliebige Datenmenge erzielt wird. (Wird auch als "Nachrichten Digest" bezeichnet.)

Siehe auch *Hash Funktionen*.

</dd> <dt>

<span id="_security_hash_object_gly"></span><span id="_SECURITY_HASH_OBJECT_GLY"></span>**Hash Objekt**
</dt> <dd>

Ein Objekt, das zum Hash von Nachrichten oder Sitzungs Schlüsseln verwendet wird. Das Hash Objekt wird durch einen-Befehl von " **cryptkreatehash**" erstellt. Die Definition des-Objekts wird durch den im-Befehl angegebenen CSP definiert.

</dd> <dt>

<span id="_security_hashing_algorithm_gly"></span><span id="_SECURITY_HASHING_ALGORITHM_GLY"></span>**Hash Algorithmus**
</dt> <dd>

Ein Algorithmus zur Erzeugung eines Hashwerts aus Daten, beispielsweise aus einer Nachricht oder aus einem Sitzungsschlüssel. Typische Hashalgorithmen sind MD2, MD4, MD5 und SHA-1.

</dd> <dt>

<span id="_security_hashing_functions_gly"></span><span id="_SECURITY_HASHING_FUNCTIONS_GLY"></span>**Hash Funktionen**
</dt> <dd>

Eine Reihe von Funktionen, die verwendet werden, um Hash Objekte zu erstellen und zu zerstören, die Parameter eines Hash Objekts sowie Hashdaten und Sitzungsschlüssel zu erhalten oder festzulegen.

</dd> <dt>

<span id="_security_hash_based_message_authentication_code_gly"></span><span id="_SECURITY_HASH_BASED_MESSAGE_AUTHENTICATION_CODE_GLY"></span>**Hash basiertes Nachrichtenauthentifizierungscode**
</dt> <dd>

HMAC Ein symmetrischer verschlüsselter Hash Algorithmus, der von Microsoft-Kryptografiedienstanbietern implementiert wird. Ein HMAC wird verwendet, um die Integrität der Daten zu überprüfen, um sicherzustellen, dass Sie während der Speicherung oder Übertragung nicht geändert wurden. Sie kann mit einem beliebigen iterierten kryptografischen Hash Algorithmus verwendet werden, z. b. MD5 oder SHA-1. CryptoAPI verweist auf diesen Algorithmus durch den Algorithmusbezeichner (calg \_ HMAC) und die Klasse (ALG- \_ Klassen \_ Hash).

Siehe auch [*Nachrichtenauthentifizierungscode*](m-gly.md).

</dd> <dt>

<span id="_security_hcsbc_gly"></span><span id="_SECURITY_HCSBC_GLY"></span>**Hcsbc**
</dt> <dd>

Datentyp, der als Handle für einen Sicherungs Kontext für Zertifikat Dienste fungiert. Die Rolle besteht darin, den Kontext Zustand zwischen dem Server und den Backup-APIs beizubehalten, wenn eine Sicherung durchgeführt wird.

</dd> <dt>

<span id="_security_hmac_gly"></span><span id="_SECURITY_HMAC_GLY"></span>**HMAC**
</dt> <dd>

Siehe *Hash basiertes Nachrichtenauthentifizierungscode*.

</dd> </dl>
