# <a name="d3dpresent"></a>D3DPRESENT

Diese Werte dienen auch als Flagwerte für das Feld PresentationIntervals von [**D3DCAPS9.**](/windows/desktop/api/D3D9Caps/ns-d3d9caps-d3dcaps9)

## <a name="remarks"></a>Hinweise

Der Fenstermodus unterstützt D3DPRESENT \_ INTERVAL \_ DEFAULT, D3DPRESENT \_ INTERVAL IMMEDIATE und \_ D3DPRESENT \_ INTERVAL \_ ONE. D3DPRESENT \_ INTERVAL DEFAULT und \_ D3DPRESENT \_ INTERVAL ONE sind nahezu gleichwertig \_ (weitere Informationen zur Timerauflösung finden Sie weiter unten). Sie führen eine ähnliche Leistung wie COPY \_ VSYNC aus, da pro Frame nur eine vorhanden ist, und verhindern das Löschen mit Balkenfolge. Im Gegensatz dazu versucht D3DPRESENT \_ INTERVAL \_ IMMEDIATE, eine unbegrenzte Präsentationsrate bereitzustellen.

Der Vollbildmodus unterstützt eine ähnliche Verwendung wie der Fenstermodus, indem D3DPRESENT \_ INTERVAL IMMEDIATE unabhängig von der \_ Aktualisierungsrate oder dem Austauscheffekt unterstützt wird. D3DPRESENT \_ INTERVAL DEFAULT verwendet die \_ Standardmäßige Auflösung des Systemzeitgebers, während D3DPRESENT \_ INTERVAL ONE \_ [**timeBeginPeriod**](/windows/win32/api/timeapi/nf-timeapi-timebeginperiod) aufruft, um die Auflösung des Systemtimers zu verbessern. Dies verbessert die Qualität der vertikalen Synchronisierung, beansprucht aber etwas mehr Verarbeitungszeit. Beide Parameter versuchen, vertikal zu synchronisieren.
