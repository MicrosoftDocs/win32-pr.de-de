# <a name="built-in-provider-context-identifiers"></a>Integrierte Anbieter Kontext Bezeichner

Die Bezeichner für die Anbieter Kontexte, die in die Windows-Filter Plattform (WFP) integriert sind, werden jeweils durch eine GUID dargestellt. Diese integrierten Anbieter Kontexte identifizieren die Standardrichtlinien für die Verwendung mit Secure Sockets.

Diese Bezeichner werden wie folgt definiert.

<dl> <dt>

<span id="FWPM_PROVIDER_CONTEXT_SECURE_SOCKET_AUTHIP"></span><span id="fwpm_provider_context_secure_socket_authip"></span>**swpm- \_ Anbieter \_ Kontext \_ Secure \_ Socket \_ AuthIP**
</dt> <dd> <dl> <dt>



Authentifiziertes Internetprotokoll-Hauptmodus-Standard Richtlinie (AuthIP) für Secure Sockets.


</dt> </dl> </dd> <dt>

<span id="FWPM_PROVIDER_CONTEXT_SECURE_SOCKET_IPSEC"></span><span id="fwpm_provider_context_secure_socket_ipsec"></span>**swpm- \_ Anbieter \_ Kontext \_ Secure \_ Socket \_ IPSec**
</dt> <dd> <dl> <dt>



Internet Protokoll Sicherheit (IPSec)-Schnellmodus-Standard Richtlinie für sichere Sockets.


</dt> </dl> </dd> </dl>
