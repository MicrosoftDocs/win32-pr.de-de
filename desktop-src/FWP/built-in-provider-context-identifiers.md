# <a name="built-in-provider-context-identifiers"></a>Integrierte Anbieterkontextbezeichner

Die Bezeichner für die Anbieterkontexte, die in die Windows Filtering Platform (WFP) integriert sind, werden jeweils durch eine GUID dargestellt. Diese integrierten Anbieterkontexte identifizieren die Standardrichtlinien, die mit Secure Sockets verwendet werden.

Diese Bezeichner sind wie folgt definiert.

<dl> <dt>

<span id="FWPM_PROVIDER_CONTEXT_SECURE_SOCKET_AUTHIP"></span><span id="fwpm_provider_context_secure_socket_authip"></span>**SECURE \_ SOCKET \_ AUTHIP DES \_ FWPM-ANBIETERKONTEXTS \_ \_**
</dt> <dd> <dl> <dt>



Authentifiziertes Internetprotokoll (AuthIP)-Standardmodus-Standardrichtlinie für Secure Sockets.


</dt> </dl> </dd> <dt>

<span id="FWPM_PROVIDER_CONTEXT_SECURE_SOCKET_IPSEC"></span><span id="fwpm_provider_context_secure_socket_ipsec"></span>**FWPM \_ PROVIDER \_ CONTEXT \_ SECURE \_ SOCKET \_ IPSEC**
</dt> <dd> <dl> <dt>



Standardrichtlinie für den IPsec-Schnellmodus (Internet Protocol Security) für Secure Sockets.


</dt> </dl> </dd> </dl>
