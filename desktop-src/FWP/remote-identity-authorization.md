# <a name="remote-identity-authorization"></a><span data-ttu-id="da2ff-103">Remote Identitäts Autorisierung</span><span class="sxs-lookup"><span data-stu-id="da2ff-103">Remote Identity Authorization</span></span>

<span data-ttu-id="da2ff-104">Das Szenario der IPSec-Richtlinie für die Remote Identitäts Autorisierung erfordert, dass eingehende Verbindungen von einem bestimmten Satz von Remote Sicherheits Prinzipale stammen, die in einer Zugriffs Steuerungs Liste (ACL) der Windows-Sicherheits Beschreibung (SD) angegeben sind.</span><span class="sxs-lookup"><span data-stu-id="da2ff-104">The Remote Identity Authorization IPsec policy scenario requires that inbound connections come from a specific set of remote security principals which are specified in a Windows security descriptor (SD) access control list (ACL).</span></span> <span data-ttu-id="da2ff-105">Wenn die Remote Identität (wie von IPSec festgelegt) nicht mit der Gruppe zulässiger Identitäten identisch ist, wird die Verbindung gelöscht.</span><span class="sxs-lookup"><span data-stu-id="da2ff-105">If the remote identity (as determined by IPsec) does not match the set of allowed identities, the connection will be dropped.</span></span> <span data-ttu-id="da2ff-106">Diese Richtlinie muss in Verbindung mit einer der Richtlinien Optionen für den Transportmodus angegeben werden.</span><span class="sxs-lookup"><span data-stu-id="da2ff-106">This policy must be specified in conjunction with one of the transport mode policy options.</span></span>

<span data-ttu-id="da2ff-107">Wenn AuthIP aktiviert ist, können zwei Sicherheits Deskriptoren angegeben werden: eine, die dem AuthIP-Hauptmodus entspricht, und die andere, die dem erweiterten AuthIP-Modus entspricht.</span><span class="sxs-lookup"><span data-stu-id="da2ff-107">If AuthIP is enabled, two security descriptors can be specified, one corresponding to AuthIP main mode, and the other corresponding to AuthIP extended mode.</span></span>
