# <a name="performance-and-memory-consumption-under-wow64"></a><span data-ttu-id="f3dbd-107">Leistung und Arbeitsspeicher Verbrauch unter WOW64</span><span class="sxs-lookup"><span data-stu-id="f3dbd-107">Performance and Memory Consumption Under WOW64</span></span>

<span data-ttu-id="f3dbd-108">Der Leistungs-und Arbeitsspeicher Verbrauch unter WOW64 wird durch die folgenden Faktoren bestimmt:</span><span class="sxs-lookup"><span data-stu-id="f3dbd-108">Performance and memory consumption under WOW64 are determined by the following factors:</span></span>

-   <span data-ttu-id="f3dbd-109">Prozessor Hardware.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-109">Processor hardware.</span></span> <span data-ttu-id="f3dbd-110">Die Anweisungs Emulation wird auf dem Chip ausgeführt.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-110">Instruction emulation is performed on the chip.</span></span> <span data-ttu-id="f3dbd-111">Auf dem x64-Prozessor werden x86-Anweisungen nativ vom Prozessor ausgeführt.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-111">On the x64 processor, x86 instructions are executed natively by the processor.</span></span> <span data-ttu-id="f3dbd-112">Daher ist die Ausführungsgeschwindigkeit unter WOW64 auf x64 vergleichbar mit der Geschwindigkeit unter 32-Bit-Windows.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-112">Therefore, execution speed under WOW64 on x64 is similar to its speed under 32-bit Windows.</span></span> <span data-ttu-id="f3dbd-113">Auf dem Intel Itanium-Prozessor und allen ARM64-Prozessoren ist mehr Software an der Emulation beteiligt, und die Leistung wird dadurch beeinträchtigt.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-113">On the Intel Itanium processor and any ARM64 processors, more software is involved in the emulation, and performance suffers as a result.</span></span>
-   <span data-ttu-id="f3dbd-114">API-Thunk-Aufwand.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-114">API thunk overhead.</span></span> <span data-ttu-id="f3dbd-115">Dieser Aufwand ist im Vergleich zu Systemaufrufen an den NT-Kernel gering.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-115">This overhead is small compared to system calls to the NT kernel.</span></span> <span data-ttu-id="f3dbd-116">NT-Kernelfunktionen sollen nur selten aufgerufen werden.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-116">NT kernel functions are intended to be called infrequently.</span></span>
-   <span data-ttu-id="f3dbd-117">Größe des virtuellen Arbeitsspeichers.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-117">Virtual memory size.</span></span> <span data-ttu-id="f3dbd-118">Auf dem Intel Itanium-Prozessor erhöht WOW64 einen erheblichen mehr Aufwand, wenn mindestens zwei Instanzen der gleichen 32-Bit-Anwendung gleichzeitig ausgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-118">On the Intel Itanium processor, WOW64 adds significant overhead if two or more instances of the same 32-bit application are running concurrently.</span></span> <span data-ttu-id="f3dbd-119">Dies liegt an den nativen 8-KB-Seiten auf dem Intel Itanium, der die Emulation der systemeigenen 4-KB-Seiten in der x86-Architektur erschwert (mehr Seiten werden als beschreibbar gekennzeichnet; Alle beschreibbaren Seiten sind für den Prozess privat).</span><span class="sxs-lookup"><span data-stu-id="f3dbd-119">This is due to the native 8 KB pages on the Intel Itanium, which complicates the emulation of the native 4 KB pages on the x86 architecture (more pages are marked as writable; all writable pages are private to the process).</span></span> <span data-ttu-id="f3dbd-120">Dies kann sich negativ auf die Skalierbarkeit von Terminal Diensten auf bestimmten Prozessoren auswirken.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-120">This can adversely affect the scalability of Terminal Services on certain processors.</span></span> <span data-ttu-id="f3dbd-121">Dies ist für den x64-Prozessor nicht der Fall.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-121">This is not the case for the x64 processor.</span></span>
-   <span data-ttu-id="f3dbd-122">Workingset.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-122">Working set.</span></span> <span data-ttu-id="f3dbd-123">WOW64 vergrößert die Größe des Workingsets der Anwendung.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-123">WOW64 increases the size of the application's working set.</span></span>

<span data-ttu-id="f3dbd-124">WOW64 ermöglicht es 32-Bit-Anwendungen, den 64-Bit-Kernel zu nutzen.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-124">WOW64 enables 32-bit applications to take advantage of the 64-bit kernel.</span></span> <span data-ttu-id="f3dbd-125">Daher können 32-Bit-Anwendungen eine größere Anzahl von Kernel Handles und Fenster Handles verwenden.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-125">Therefore, 32-bit applications can use a larger number of kernel handles and window handles.</span></span> <span data-ttu-id="f3dbd-126">32-Bit-Anwendungen können jedoch möglicherweise nicht so viele Threads unter WOW64 erstellen, wie Sie bei der nativen Ausführung auf x86-basierten Systemen möglich sind, da WOW64 einen zusätzlichen 64-Bit-Stapel (in der Regel 512 KB) für jeden Thread zuordnet.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-126">However, 32-bit applications may not be able to create as many threads under WOW64 as they can when running natively on x86-based systems because WOW64 allocates an additional 64-bit stack (usually 512 KB) for each thread.</span></span> <span data-ttu-id="f3dbd-127">Außerdem ist eine bestimmte Menge an Adressraum für WOW64 und die verwendeten Datenstrukturen reserviert.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-127">In addition, some amount of address space is reserved for WOW64 itself and the data structures it uses.</span></span> <span data-ttu-id="f3dbd-128">Der reservierte Betrag hängt vom Prozessor ab. mehr ist auf dem Intel Itanium als auf den x64-oder ARM64-Prozessoren reserviert.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-128">The amount reserved depends on the processor; more is reserved on the Intel Itanium than on the x64 or ARM64 processors.</span></span>

<span data-ttu-id="f3dbd-129">Wenn für die Anwendung das Flag für die [**Image \_ Datei mit \_ großen \_ Adress \_**](/windows/desktop/api/dbghelp/ns-dbghelp-loaded_image) Werten im Image Header festgelegt ist, erhält jede 32-Bit-Anwendung 4 GB virtuellen Adressraum in der WOW64-Umgebung.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-129">If the application has the [**IMAGE\_FILE\_LARGE\_ADDRESS\_AWARE**](/windows/desktop/api/dbghelp/ns-dbghelp-loaded_image) flag set in the image header, each 32-bit application receives 4 GB of virtual address space in the WOW64 environment.</span></span> <span data-ttu-id="f3dbd-130">Wenn das Flag zum **\_ \_ Erkennen von großen \_ Adressen \_ für die Image Datei** nicht festgelegt ist, erhält jede 32-Bit-Anwendung 2 GB virtuellen Adressraum in der WOW64-Umgebung.</span><span class="sxs-lookup"><span data-stu-id="f3dbd-130">If the **IMAGE\_FILE\_LARGE\_ADDRESS\_AWARE** flag is not set, each 32-bit application receives 2 GB of virtual address space in the WOW64 environment.</span></span>
