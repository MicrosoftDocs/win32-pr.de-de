# <a name="implementing-reference-counting"></a><span data-ttu-id="d6818-103">Implementieren der Verweis Zählung</span><span class="sxs-lookup"><span data-stu-id="d6818-103">Implementing Reference Counting</span></span>

<span data-ttu-id="d6818-104">Die Verweis Zählung erfordert die Arbeit an einem Teil der Implementierung einer Klasse und den Clients, die Objekte dieser Klasse verwenden.</span><span class="sxs-lookup"><span data-stu-id="d6818-104">Reference counting requires work on the part of both the implementor of a class and the clients who use objects of that class.</span></span> <span data-ttu-id="d6818-105">Wenn Sie eine Klasse implementieren, müssen Sie die [**adressf**](/windows/win32/api/unknwn/nf-unknwn-iunknown-addref) -und [**releasemethoden**](/windows/win32/api/unknwn/nf-unknwn-iunknown-release) als Teil der [**IUnknown**](/windows/desktop/api/Unknwn/nn-unknwn-iunknown) -Schnittstelle implementieren.</span><span class="sxs-lookup"><span data-stu-id="d6818-105">When you implement a class, you must implement the [**AddRef**](/windows/win32/api/unknwn/nf-unknwn-iunknown-addref) and [**Release**](/windows/win32/api/unknwn/nf-unknwn-iunknown-release) methods as part of the [**IUnknown**](/windows/desktop/api/Unknwn/nn-unknwn-iunknown) interface.</span></span> <span data-ttu-id="d6818-106">Diese beiden Methoden verfügen über die folgenden einfachen Implementierungen:</span><span class="sxs-lookup"><span data-stu-id="d6818-106">These two methods have the following simple implementations:</span></span>

-   <span data-ttu-id="d6818-107">[**Adressf**](/windows/win32/api/unknwn/nf-unknwn-iunknown-addref) Inkremente den internen Verweis Zähler des Objekts.</span><span class="sxs-lookup"><span data-stu-id="d6818-107">[**AddRef**](/windows/win32/api/unknwn/nf-unknwn-iunknown-addref) increments the object's internal reference count.</span></span>
-   <span data-ttu-id="d6818-108">[**Release**](/windows/win32/api/unknwn/nf-unknwn-iunknown-release) Dekremente zuerst den internen Verweis Zähler des Objekts und dann überprüft, ob der Verweis Zähler auf null gefallen ist.</span><span class="sxs-lookup"><span data-stu-id="d6818-108">[**Release**](/windows/win32/api/unknwn/nf-unknwn-iunknown-release) first decrements the object's internal reference count, and then it checks whether the reference count has fallen to zero.</span></span> <span data-ttu-id="d6818-109">Wenn dies der Fall ist, bedeutet dies, dass das Objekt nicht mehr verwendet wird, sodass die Freigabe Funktion das Objekt **freigibt** .</span><span class="sxs-lookup"><span data-stu-id="d6818-109">If it has, that means no one is using the object any longer, so the **Release** function deallocates the object.</span></span>

<span data-ttu-id="d6818-110">Ein gängiger Implementierungs Ansatz für die meisten Objekte besteht darin, dass nur eine Implementierung dieser Methoden (zusammen mit [**QueryInterface**](/windows/desktop/api/Unknwn/nf-unknwn-iunknown-queryinterface(q))) vorhanden ist, die von allen Schnittstellen gemeinsam genutzt wird, und somit ein Verweis Zähler, der auf das gesamte Objekt angewendet wird.</span><span class="sxs-lookup"><span data-stu-id="d6818-110">A common implementation approach for most objects is to have only one implementation of these methods (along with [**QueryInterface**](/windows/desktop/api/Unknwn/nf-unknwn-iunknown-queryinterface(q))), which is shared between all interfaces, and therefore a reference count that applies to the entire object.</span></span> <span data-ttu-id="d6818-111">Allerdings ist die Verweis Zählung aus Sicht eines Clients streng und eindeutig ein Schnittstellen Zeiger Konzept. Daher können Objekte, die diese Funktion nutzen, durch dynamisches erstellen, zerstören, laden und Entladen von Teilen ihrer Funktionalität basierend auf den aktuell ausgewertenden Schnittstellen Zeigern implementiert werden.</span><span class="sxs-lookup"><span data-stu-id="d6818-111">However, from a client's perspective, reference counting is strictly and clearly a per-interface-pointer notion, and therefore objects that take advantage of this capability by dynamically constructing, destroying, loading, or unloading portions of their functionality based on the currently extant interface pointers may be implemented.</span></span> <span data-ttu-id="d6818-112">Diese werden als " *abtrennbare Schnittstellen*" bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="d6818-112">These are colloquially called *tear-off interfaces*.</span></span>

<span data-ttu-id="d6818-113">Wenn ein Client eine Methode (oder API-Funktion) aufruft, wie z. b. [**QueryInterface**](/windows/desktop/api/Unknwn/nf-unknwn-iunknown-queryinterface(q)), die einen neuen Schnittstellen Zeiger zurückgibt, ist die aufgerufene Methode für das Inkrementieren des Verweis zähgers über den zurückgegebenen Zeiger verantwortlich.</span><span class="sxs-lookup"><span data-stu-id="d6818-113">Whenever a client calls a method (or API function), such as [**QueryInterface**](/windows/desktop/api/Unknwn/nf-unknwn-iunknown-queryinterface(q)), that returns a new interface pointer, the method being called is responsible for incrementing the reference count through the returned pointer.</span></span> <span data-ttu-id="d6818-114">Wenn ein Client z. b. zuerst ein-Objekt erstellt, empfängt er einen Schnittstellen Zeiger auf ein Objekt, das aus Sicht des Clients einen Verweis Zähler von 1 hat.</span><span class="sxs-lookup"><span data-stu-id="d6818-114">For example, when a client first creates an object, it receives an interface pointer to an object that, from the client's point of view, has a reference count of one.</span></span> <span data-ttu-id="d6818-115">Wenn der Client dann die [**adressf**](/windows/win32/api/unknwn/nf-unknwn-iunknown-addref) für den Schnittstellen Zeiger aufruft, wird der Verweis Zähler zu zwei.</span><span class="sxs-lookup"><span data-stu-id="d6818-115">If the client then calls [**AddRef**](/windows/win32/api/unknwn/nf-unknwn-iunknown-addref) on the interface pointer, the reference count becomes two.</span></span> <span data-ttu-id="d6818-116">Der Client muss [**Release**](/windows/win32/api/unknwn/nf-unknwn-iunknown-release) zweimal für den Schnittstellen Zeiger abrufen, um alle Verweise auf das Objekt zu löschen.</span><span class="sxs-lookup"><span data-stu-id="d6818-116">The client must call [**Release**](/windows/win32/api/unknwn/nf-unknwn-iunknown-release) twice on the interface pointer to drop all of its references to the object.</span></span>

<span data-ttu-id="d6818-117">Ein Beispiel dafür, wie Verweis Zähler ausschließlich pro Schnittstelle sind, tritt auf, wenn ein Client [**QueryInterface**](/windows/desktop/api/Unknwn/nf-unknwn-iunknown-queryinterface(q)) für den ersten Zeiger für eine neue Schnittstelle oder die gleiche Schnittstelle aufruft.</span><span class="sxs-lookup"><span data-stu-id="d6818-117">An example of how reference counts are strictly per-interface-pointer occurs when a client calls [**QueryInterface**](/windows/desktop/api/Unknwn/nf-unknwn-iunknown-queryinterface(q)) on the first pointer for either a new interface or the same interface.</span></span> <span data-ttu-id="d6818-118">In jedem dieser Fälle ist es erforderlich, dass der Client die [**Freigabe**](/windows/win32/api/unknwn/nf-unknwn-iunknown-release) für jeden Zeiger einmal aufruft.</span><span class="sxs-lookup"><span data-stu-id="d6818-118">In either of these cases, the client is required to call [**Release**](/windows/win32/api/unknwn/nf-unknwn-iunknown-release) once for each pointer.</span></span> <span data-ttu-id="d6818-119">COM erfordert nicht, dass ein Objekt denselben Zeiger zurückgibt, wenn er mehrmals zur gleichen Schnittstelle aufgefordert wird.</span><span class="sxs-lookup"><span data-stu-id="d6818-119">COM does not require that an object return the same pointer when asked for the same interface multiple times.</span></span> <span data-ttu-id="d6818-120">(Die einzige Ausnahme hierbei ist eine [**IUnknown**](/windows/desktop/api/Unknwn/nn-unknwn-iunknown)-Abfrage, die ein Objekt für com identifiziert.) Dies ermöglicht es der Objekt Implementierung, Ressourcen effizient zu verwalten.</span><span class="sxs-lookup"><span data-stu-id="d6818-120">(The only exception to this is a query to [**IUnknown**](/windows/desktop/api/Unknwn/nn-unknwn-iunknown), which identifies an object to COM.) This allows the object implementation to manage resources efficiently.</span></span>

<span data-ttu-id="d6818-121">Thread Sicherheit ist auch ein wichtiges Problem beim Implementieren von [**Adressaten**](/windows/win32/api/unknwn/nf-unknwn-iunknown-addref) und [**Releases**](/windows/win32/api/unknwn/nf-unknwn-iunknown-release).</span><span class="sxs-lookup"><span data-stu-id="d6818-121">Thread-safety is also an important issue in implementing [**AddRef**](/windows/win32/api/unknwn/nf-unknwn-iunknown-addref) and [**Release**](/windows/win32/api/unknwn/nf-unknwn-iunknown-release).</span></span> <span data-ttu-id="d6818-122">Weitere Informationen finden Sie unter [Prozesse, Threads und Apartments](processes--threads--and-apartments.md).</span><span class="sxs-lookup"><span data-stu-id="d6818-122">For more information, see [Processes, Threads, and Apartments](processes--threads--and-apartments.md).</span></span>
