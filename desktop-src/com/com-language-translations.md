# <a name="com-language-translations"></a>COM-Sprachübersetzungen

Mit dem Component Object Model (com) erstellte Komponenten können in Anwendungen wieder verwendet werden, die in einer beliebigen Programmiersprache geschrieben wurden, die com unterstützt. Dies liegt daran, dass com ein binärer Standard ist und daher sprachunabhängig ist.

COM-Objekte sind in der relevantesten Programmiersprache oder in den relevantesten Sprachen dokumentiert. Beispielsweise werden Objekte, die zur Verwendung in Webseiten erstellt werden, in der Regel im Microsoft Visual Basic-Entwicklungssystem dokumentiert, wohingegen Objekte auf Systemebene in der Regel in C++ dokumentiert werden. Da com jedoch sprachneutral ist, sind Sie nicht auf die Verwendung eines Objekts in derselben Sprache beschränkt, in der es geschrieben oder dokumentiert ist. Beispielsweise können Sie eine Anwendung in JScript schreiben, die ein in C++ erstelltes Steuerelement verwendet und in Visual Basic dokumentiert ist.

In den folgenden Themen werden die Unterschiede zwischen Programmiersprachen erläutert und beschrieben, wie COM-Objekt Syntax von einer Sprache in eine andere übersetzt wird. In weiteren Themen wird beschrieben, wie COM-Objekte in verschiedenen Skriptsprachen und Umgebungen verwendet werden.

-   [Syntaxunterschiede](syntax-differences.md)
-   [Datentyp Konvertierungen](data-type-conversions.md)
-   [IDL-Dateien](idl-files.md)
-   [Übersetzen der COM-Objekt Syntax für Programmiersprachen](translating-com-object-syntax-for-programming-languages.md)
-   [Skripterstellung mit COM-Objekten](scripting-with-com-objects.md)

Die Absicht besteht darin, die häufigsten sprach Übersetzungsprobleme zu beheben, die bei der Verwendung von COM-Objekten auftreten. Die beschriebenen Techniken und Prinzipien gelten für jede Programmiersprache oder Skriptsprache, die com unterstützt. Da Skriptsprachen und Programmiersprachen unterschiedliche Programmierparadigmen darstellen, wird die Übersetzung zwischen Skriptsprachen und Programmiersprachen nicht behandelt.
