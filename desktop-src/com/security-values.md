# <a name="security-values"></a>Sicherheitswerte

Mehrere Werte wirken sich auf die Sicherheit für COM-Anwendungen aus, einschließlich Authentifizierungs Stufe, Identitätswechsel Ebene, Cloaking, Anwendungs Identität, Konfiguration der Software Einschränkungs Richtlinie und Verweis Nachverfolgung.

Weitere Informationen finden Sie unter den folgenden Themen:

-   Auf der [Authentifizierungs Ebene](authentication-level.md) wird beschrieben, wie der Schutz für die Kommunikation zwischen COM-Objekten gesteuert werden kann.
-   [Delegierung und](delegation-and-impersonation.md) Identitätswechsel erläutert, wie sich das Cloaking auf die Identität auswirkt, die dem Server während des Identitäts Wechsels angezeigt wird, und wie der Client dem Server die Berechtigung erteilt, in seinem Namen zu agieren.
-   Die [Anwendungs Identität](application-identity.md) identifiziert und erläutert die Identitäten, die Ihre Anwendung verwenden kann.
-   Die [Richtlinie für Software Einschränkung](software-restriction-policy.md) erläutert, wie Sie Ihr System vor den Gefahren beim Ausführen von unbekanntem Code schützen können.
-   Die [Verweis Nachverfolgung](reference-tracking.md) erläutert, wie Sie sicherstellen können, dass Objekte nicht zu früh freigegeben werden.
