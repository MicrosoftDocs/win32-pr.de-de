# <a name="security-values"></a>Sicherheitswerte

Mehrere Werte wirken sich auf die Sicherheit für COM-Anwendungen aus, einschließlich Authentifizierungsebene, Identitätswechselebene, Verleumdung, Anwendungsidentität, Konfiguration der Softwareeinschränkungsrichtlinie und Verweisnachverfolgung.

Weitere Informationen finden Sie in den folgenden Themen:

-   [Die Authentifizierungsebene](authentication-level.md) beschreibt, wie Sie den Umfang des Schutzes für die Kommunikation zwischen COM-Objekten steuern können.
-   [Delegierung und Identitätswechsel](delegation-and-impersonation.md) erläutert, wie sich das Tarnen auf die Identität auswirkt, die dem Server während des Identitätswechsels angezeigt wird, und wie der Client dem Server die Berechtigung erteilt, in seinem Namen zu handeln.
-   [Die Anwendungsidentität](application-identity.md) identifiziert und erläutert die Identitäten, die Ihre Anwendung verwenden kann.
-   In der [Richtlinie für Softwareeinschränkung](software-restriction-policy.md) wird erläutert, wie Sie Ihr System vor der Gefahr schützen, dass unbekannter Code ausgeführt wird.
-   [In der Referenznachverfolgung](reference-tracking.md) wird erläutert, wie Sie sicherstellen können, dass Objekte nicht zu früh freigegeben werden.
