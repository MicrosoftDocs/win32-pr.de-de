# <a name="standard-font-object"></a><span data-ttu-id="a343a-103">Standard Schriftart Objekt</span><span class="sxs-lookup"><span data-stu-id="a343a-103">Standard Font Object</span></span>

<span data-ttu-id="a343a-104">Die standardmäßige ambient-Schriftart Eigenschaft, die vom Container bereitgestellt wird, und die Standard Schriftart Eigenschaft, die vom-Steuerelement bereitgestellt wird, stellen ein Standardschrift</span><span class="sxs-lookup"><span data-stu-id="a343a-104">The standard ambient font property supplied by the container and the standard font property supplied by the control both provide a standard font object.</span></span> <span data-ttu-id="a343a-105">Das heißt, dass diese Standard Schriftarten einen **IDispatch** -Zeiger auf ein Standard schriftobjekt bereitstellen.</span><span class="sxs-lookup"><span data-stu-id="a343a-105">That is, these standard fonts supply an **IDispatch** pointer to a standard font object.</span></span>

<span data-ttu-id="a343a-106">Das Font-Objekt ist eine vom System bereitgestellte Implementierung eines Satzes von Schnittstellen oberhalb der zugrunde liegenden GDI-Schriftart Unterstützung.</span><span class="sxs-lookup"><span data-stu-id="a343a-106">The font object is a system-provided implementation of a set of interfaces on top of the underlying GDI font support.</span></span> <span data-ttu-id="a343a-107">Ein Schriftart Objekt wird über die API-Funktion [**olecreatefontindirekt**](/windows/desktop/api/OleCtl/nf-olectl-olecreatefontindirect) mit einer data-ttu-id="a343a-108">Das Font-Objekt unterstützt eine Reihe von Lese-/Schreibeigenschaften sowie benutzerdefinierte Methoden über die Schnittstelle [**IFont**](/windows/desktop/api/OCIdl/nn-ocidl-ifont)und unterstützt denselben Satz von Eigenschaften (aber nicht die Methoden) über eine "dispinterface"- [**IFontDisp**](/windows/win32/api/ocidl/nn-ocidl-ifontdisp).</span><span class="sxs-lookup"><span data-stu-id="a343a-108">The font object supports a number of read/write properties as well as custom [**IFontDisp**](/windows/win32/api/ocidl/nn-ocidl-ifontdisp).</span></span> <span data-ttu-id="a343a-109">Die dispinterface wird für die zuvor erwähnten Schriftart Eigenschaften verwendet.</span><span correspond to the GDI font attributes that are described in the [**LOGFONT**](/windows/win32/api/dimm/ns-dimm-logfonta) structure.</span></span>

<span data-ttu-id="a343a-111">Das Schriftart Objekt unterstützt auch die ausgehende Schnittstelle [**IPropertyNotifySink**](/windows/desktop/api/OCIdl/nn-ocidl-ipropertynotifysink) , damit ein Client ermitteln kann, wann die Schriftart Eigenschaften geändert werden.</span><span class="sxs-lookup"><span data-stu-id="a343a-111">The font object also supports the outgoing interface [**IPropertyNotifySink**](/windows/desktop/api/OCIdl/nn-ocidl-ipropertynotifysink) so that a client can determine when font properties change.</span></span> <span data-ttu-id="a343a-112">Da das Font-Objekt mindestens eine ausgehende Schnittstelle unterstützt, implementiert es zu diesem Zweck auch [**IConnectionPointContainer**](/windows/desktop/api/OCIdl/nn-ocidl-iconnectionpointcontainer) und einen Verbindungspunkt für **IPropertyNotifySink** .</span><span class="sxs-lookup"><span data-stu-id="a343a-112">Since the font object supports at least one outgoing interface, it also implements [**IConnectionPointContainer**](/windows/desktop/api/OCIdl/nn-ocidl-iconnectionpointcontainer) and one connection point for **IPropertyNotifySink** for this purpose.</span></span>

<span data-ttu-id="a343a-113">Das Font-Objekt stellt eine hFont-Eigenschaft bereit, bei der es sich um ein Windows-Schriftart Handle handelt, das den anderen für die Schriftart angegebenen Attributen entspricht.</span><span class="sxs-lookup"><span data-stu-id="a343a-113">The font object provides an hFont property that is a Windows font handle that conforms to the other attributes specified for the font.</span></span> <span data-ttu-id="a343a-114">Das Schriftart Objekt verzögert das Erkennen dieser Schriftart, wenn möglich, sodass das Festlegen von zwei Eigenschaften für eine Schriftart nicht dazu führt, dass eine zwischen Schriftart realisiert wird.</span><span class="sxs-lookup"><span data-stu-id="a343a-114">The font object delays realizing this font when possible, so consecutively setting two properties on a font won't cause an intermediate font to be realized.</span></span> <span data-ttu-id="a343a-115">Außerdem verwaltet das Standard Schriftart Objekt als Optimierung einen Cache von Schriftart Handles.</span><span class="sxs-lookup"><span data-stu-id="a343a-115">In addition, as an optimization, the standard font object maintains a cache of font handles.</span></span> <span data-ttu-id="a343a-116">Zwei Schriftart Objekte im gleichen Prozess mit identischen Eigenschaften geben das gleiche Schriftart Handle zurück.</span><span class="sxs-lookup"><span data-stu-id="a343a-116">Two font objects in the same process that have identical properties will return the same font handle.</span></span> <span data-ttu-id="a343a-117">Mit dem Font-Objekt können Schriftarten aus diesem Cache nach Bedarf entfernt werden. Dies führt zu speziellen Überlegungen für Clients, die diese hFont-Eigenschaft verwenden.</span><span class="sxs-lookup"><span data-stu-id="a343a-117">The font object can remove fonts from this cache at will, which introduces special considerations for clients using this hFont property.</span></span> <span data-ttu-id="a343a-118">Weitere Informationen finden [**Sie unter IFont:: get \_ hFont**](/windows/desktop/api/OCIdl/nf-ocidl-ifont-get_hfont) .</span><span class="sxs-lookup"><span data-stu-id="a343a-118">See [**IFont::get\_hFont**](/windows/desktop/api/OCIdl/nf-ocidl-ifont-get_hfont) for more details.</span></span>

<span data-ttu-id="a343a-119">Das Schriftart Objekt unterstützt auch [**IPersistStream**](/windows/desktop/api/ObjIdl/nn-objidl-ipersiststream) , sodass es sich selbst speichern und aus einer Instanz von [**IStream**](/windows/desktop/api/objidl/nn-objidl-istream)laden kann.</span><span class="sxs-lookup"><span data-stu-id="a343a-119">The font object also supports [**IPersistStream**](/windows/desktop/api/ObjIdl/nn-objidl-ipersiststream) such that it can save and load itself from an instance of [**IStream**](/windows/desktop/api/objidl/nn-objidl-istream).</span></span> <span data-ttu-id="a343a-120">Alle anderen Objekte, die intern ein Schriftart Objekt verwenden, speichern und laden die Schriftart normalerweise als Teil der eigenen persistenzverarbeitung des Objekts.</span><span class="sxs-lookup"><span data-stu-id="a343a-120">Any other object that uses a font object internally would normally save and load the font as part of the object's own persistence handling.</span></span>

<span data-ttu-id="a343a-121">Außerdem unterstützt das Font-Objekt [**IDataObject**](/windows/desktop/api/ObjIdl/nn-objidl-idataobject) , über das es einen Eigenschaften Satz mit typisierten Werten für jede Schriftart Eigenschaft bereitstellt.</span><span class="sxs-lookup"><span data-stu-id="a343a-121">In addition, the font object supports [**IDataObject**](/windows/desktop/api/ObjIdl/nn-objidl-idataobject) through which it provides a property set containing typed values for each font property.</span></span>
