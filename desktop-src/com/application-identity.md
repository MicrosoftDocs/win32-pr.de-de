# <a name="application-identity"></a>Anwendungsidentität

Die Identität einer Anwendung ist das Konto, das zum Ausführen der Anwendung verwendet wird. Die Identität kann die identität des benutzers sein, der derzeit angemeldet ist (der interaktive Benutzer), des Benutzers, der den Server gestartet hat, eines angegebenen Benutzers oder eines Diensts. Informationen zum Festlegen der Anwendungsidentität mithilfe von Dcomcnfg.exe finden Sie unter Festlegen der [prozessweiten Sicherheit mithilfe von DCOMCNFG.](setting-processwide-security-using-dcomcnfg.md)

In den folgenden Themen werden die vier Arten der Anwendungsidentität beschrieben und die Einschränkungen der einzelnen Typen erläutert:

-   [Interaktiver Benutzer](interactive-user.md)
-   [Starten des Benutzers](launching-user.md)
-   [Angegebener Benutzer](specified-user.md)
-   [Dienstidentität](service-identity.md)
