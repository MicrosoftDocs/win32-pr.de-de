# <a name="application-identity"></a>Anwendungsidentität

Die Identität einer Anwendung ist das Konto, das zum Ausführen der Anwendung verwendet wird. Dabei kann es sich um die Identität des aktuell angemeldeten Benutzers (interaktiver Benutzer), den Benutzer, der den Server gestartet hat, einen angegebenen Benutzer oder einen Dienst handeln. Informationen zum Festlegen der Anwendungs Identität mithilfe von Dcomcnfg.exe finden Sie unter [Festlegen der Processwide Security mithilfe von DCOMCNFG](setting-processwide-security-using-dcomcnfg.md).

In den folgenden Themen werden die einzelnen vier Arten von Anwendungs Identitäten beschrieben und die Einschränkungen der einzelnen Typen erläutert:

-   [Interaktiver Benutzer](interactive-user.md)
-   [Benutzer wird gestartet](launching-user.md)
-   [Angegebener Benutzer](specified-user.md)
-   [Dienst Identität](service-identity.md)
