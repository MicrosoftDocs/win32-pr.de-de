# <a name="linked-objects-and-monikers"></a><span data-ttu-id="9b14b-103">Verknüpfte Objekte und Moniker</span><span class="sxs-lookup"><span data-stu-id="9b14b-103">Linked Objects and Monikers</span></span>

<span data-ttu-id="9b14b-104">Verknüpfte Objekte, wie eingebettete Objekte, verlassen sich auf einen Objekt Handler, um mit Server Anwendungen zu kommunizieren.</span><span class="sxs-lookup"><span data-stu-id="9b14b-104">Linked objects, like embedded objects, rely on an object handler to communicate with server applications.</span></span> <span data-ttu-id="9b14b-105">Das verknüpfte Objekt selbst verwaltet jedoch die Benennung und Nachverfolgung von Verknüpfungs Quellen.</span><span class="sxs-lookup"><span data-stu-id="9b14b-105">The linked object itself, however, manages the naming and tracking of link sources.</span></span> <span data-ttu-id="9b14b-106">Das verknüpfte Objekt verhält sich wie ein Prozess interner Server.</span><span class="sxs-lookup"><span data-stu-id="9b14b-106">The linked object acts like an in-process server.</span></span> <span data-ttu-id="9b14b-107">Wenn beispielsweise ein verknüpftes Objekt aktiviert ist, wird die OLE-Serveranwendung, die die Verknüpfungs Quelle ist, gesucht und gestartet.</span><span class="sxs-lookup"><span data-stu-id="9b14b-107">For example, when activated, a linked object locates and launches the OLE server application that is the link source.</span></span>

<span data-ttu-id="9b14b-108">Der Handler eines verknüpften Objekts besteht aus zwei Hauptkomponenten: der Handlerkomponente und der Verknüpfungs Komponente.</span><span class="sxs-lookup"><span data-stu-id="9b14b-108">A linked object's handler is made up of two main components: the handler component and the linking component.</span></span> <span data-ttu-id="9b14b-109">Die Handlerkomponente enthält die Steuerungs-und Remoting-Teile und-Funktionen, ähnlich wie ein Handler für ein eingebettetes Objekt.</span><span class="sxs-lookup"><span data-stu-id="9b14b-109">The handler component contains the controlling and remoting pieces and functions much like a handler for an embedded object.</span></span> <span data-ttu-id="9b14b-110">Die Verknüpfungs Komponente verfügt über einen eigenen Controller und Cache und ermöglicht den Zugriff auf den strukturierten Speicher des Objekts.</span><span class="sxs-lookup"><span data-stu-id="9b14b-110">The linking component has its own controller and cache and provides access to the object's structured storage.</span></span> <span data-ttu-id="9b14b-111">Der Verknüpfungs Komponenten Controller unterstützt die Quell Benennung durch die Verwendung von Monikern und die Bindung, das Auffinden und Ausführen der Link Quelle.</span><span class="sxs-lookup"><span data-stu-id="9b14b-111">The linking components controller supports source naming through the use of monikers, and binding, the process of locating and running the link source.</span></span> <span data-ttu-id="9b14b-112">(Weitere Informationen zu Monikern und Bindung finden Sie [im Component Object Model](the-component-object-model.md).)</span><span class="sxs-lookup"><span data-stu-id="9b14b-112">(For more information on monikers and binding, see [The Component Object Model](the-component-object-model.md).)</span></span>

<span data-ttu-id="9b14b-113">Wenn ein Benutzer anfänglich ein verknüpftes Objekt erstellt oder ein vorhandenes Objekt aus dem Speicher lädt, lädt der Container zusammen mit dem Objekt Handler eine Instanz der Verknüpfungs Komponente in den Arbeitsspeicher.</span><span class="sxs-lookup"><span data-stu-id="9b14b-113">When a user initially creates a linked object or loads an existing one from storage, the container loads an instance of the linking component into memory, along with the object handler.</span></span> <span data-ttu-id="9b14b-114">Die Verknüpfungs Komponente stellt Schnittstellen – insbesondere [**iolelink**](/windows/desktop/api/OleIdl/nn-oleidl-iolelink)– bereit, die das Objekt als Link bezeichnen und es Ihnen ermöglichen, die Benennung, Nachverfolgung und Aktualisierung der zugehörigen Verknüpfungs Quelle zu verwalten.</span><span class="sxs-lookup"><span data-stu-id="9b14b-114">The linking component supplies interfaces — most notably [**IOleLink**](/windows/desktop/api/OleIdl/nn-oleidl-iolelink)— that identify the object as a link and enable it to manage the naming, tracking, and updating of its link source.</span></span>

<span data-ttu-id="9b14b-115">Durch Implementieren der [**iolelink**](/windows/desktop/api/OleIdl/nn-oleidl-iolelink) -Schnittstelle stellt ein verknüpftes Objekt seinen Container mit Funktionen bereit, die das Verknüpfen unterstützen.</span><span class="sxs-lookup"><span data-stu-id="9b14b-115">By implementing the [**IOleLink**](/windows/desktop/api/OleIdl/nn-oleidl-iolelink) interface, a linked object provides its container with functions that support linking.</span></span> <span data-ttu-id="9b14b-116">**Iolelink** wird nur von verknüpften Objekten implementiert, und durch Abfragen für diese Schnittstelle kann ein Container ermitteln, ob ein bestimmtes Objekt eingebettet oder verknüpft ist.</span><span class="sxs-lookup"><span data-stu-id="9b14b-116">Only linked objects implement **IOleLink**, and by querying for this interface a container can determine whether a given object is embedded or linked.</span></span> <span data-ttu-id="9b14b-117">Die wichtigste Funktion von **iolelink** ermöglicht einem Container das Binden an die Quelle des verknüpften Objekts, d. h., um die Verbindung mit dem Dokument zu aktivieren, in dem die systemeigenen Daten des verknüpften Objekts gespeichert werden.</span><span class="sxs-lookup"><span data-stu-id="9b14b-117">The most important function provided by **IOleLink** enables a container to binding to the source of the linked object, that is, to activate the connection to the document that stores the linked object's native data.</span></span> <span data-ttu-id="9b14b-118">**Iolelink** definiert auch Funktionen zum Verwalten von Informationen über das verknüpfte Objekt, z. b. zwischengespeicherte Präsentationsdaten und den Speicherort der Verknüpfungs Quelle.</span><span class="sxs-lookup"><span data-stu-id="9b14b-118">**IOleLink** also defines functions for managing information about the linked object, such as cached presentation data and the location of the link source.</span></span>

<span data-ttu-id="9b14b-119">Wenn ein Verbund Dokument gespeichert wird, das ein verknüpftes Objekt enthält, werden die Daten des Links mit der Link Quelle, nicht mit dem Container gespeichert.</span><span class="sxs-lookup"><span data-stu-id="9b14b-119">When a compound document containing a linked object is saved, the link's data is saved with the link source, not with the container.</span></span> <span data-ttu-id="9b14b-120">Nur Informationen zu Name und Speicherort werden zusammen mit dem Verbund Dokument gespeichert.</span><span class="sxs-lookup"><span data-stu-id="9b14b-120">Only information about its name and location is saved along with the compound document.</span></span> <span data-ttu-id="9b14b-121">Dieses Verhalten steht im Gegensatz zu dem eines eingebetteten Objekts, dessen Daten zusammen mit dem des zugehörigen Containers gespeichert werden.</span><span class="sxs-lookup"><span data-stu-id="9b14b-121">This behavior is in contrast to that of an embedded object, whose data is stored along with that of its container.</span></span>

<span data-ttu-id="9b14b-122">Container Anwendungen können Informationen zu ihren eingebetteten Objekten bereitstellen, sodass die letzteren oder Teile davon als Verknüpfungs Quellen fungieren können.</span><span class="sxs-lookup"><span data-stu-id="9b14b-122">Container applications can provide information about their embedded objects such that the latter, or portions thereof, can act as link sources.</span></span> <span data-ttu-id="9b14b-123">Durch Implementieren der Unterstützung für das Verknüpfen mit den eingebetteten Objekten des Containers können Sie geschiebte Einbettungen ermöglichen, sodass der Benutzer die Originale der einzelnen Einbettungs Objekte, auf die ein Link gewünscht ist, nicht mehr nachverfolgen muss.</span><span class="sxs-lookup"><span data-stu-id="9b14b-123">By implementing support for linking to your container's embedded objects, you make nested embeddings possible, relieving the user of having to track down the originals of every embedding object to which a link is desired.</span></span> <span data-ttu-id="9b14b-124">Wenn ein Benutzer beispielsweise ein Microsoft Excel-Arbeitsblatt in Microsoft Word einbetten möchte und das Arbeitsblatt eine in Pinsel erstellte Bitmap enthält, kann der Benutzer eine Verknüpfung mit einer Kopie der Bitmap herstellen, die im Arbeitsblatt enthalten ist.</span><span class="sxs-lookup"><span data-stu-id="9b14b-124">For example, if a user wants to embed a Microsoft Excel worksheet in Microsoft Word, and the worksheet contains a bitmap created in Paintbrush, the user can link to a copy of the bitmap contained in the worksheet rather than the original.</span></span>
