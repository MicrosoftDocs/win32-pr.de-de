# <a name="windowless-controls"></a>Fensterlose Steuerelemente

Die ActiveX Controls 96-Spezifikation enthält eine Definition für fensterlose Steuerelemente. Solche Steuerelemente werden nicht in ihrem eigenen Fenster ausgeführt und erfordern einen Container, um ein freigegebenes Fenster anzubieten, in das das Steuerelement gezeichnet werden kann. Weitere Informationen finden Sie im ActiveX SDK. Fensterlose Steuerelemente sind so konzipiert, dass sie mit älteren Steuerelementcontainern kompatibel sind, indem sie in dieser Situation ein eigenes Fenster erstellen. Fensterlose Steuerelementcontainer sollten fensterlose Steuerelemente auf herkömmliche Weise ohne Probleme hosten. Es kann jedoch nützlich sein, dass ein Container die Steuerelemente unterscheidet, die im fensterlosen Modus ausgeführt werden können, sodass eine entsprechende Komponentenkategorie definiert wird.
