# <a name="data-types-hlsl"></a>Datentypen (HLSL)

HLSL unterstützt viele verschiedene systeminterne Datentypen. Diese Tabelle zeigt, welche Typen zum Definieren von shadervariablen verwendet werden müssen.



|                                                                                                                         |                                            |
|-------------------------------------------------------------------------------------------------------------------------|--------------------------------------------|
| Diesen systeminternen Typ verwenden                                                                                                 | So definieren Sie diese Shader-Variable             |
| [Skalare](dx-graphics-hlsl-scalar.md)                                                                                   | Skalar mit einer Komponente                       |
| [Vektor](dx-graphics-hlsl-vector.md), [Matrix](dx-graphics-hlsl-matrix.md)                                            | Vektor oder Matrix mit mehreren Komponenten        |
| [Sampler](dx-graphics-hlsl-sampler.md), [Textur](dx-graphics-hlsl-texture.md) oder [Puffer](dx-graphics-hlsl-buffer.md)   | Sampler, Textur oder Puffer Objekt         |
| [Struktur](dx-graphics-hlsl-struct.md), [Benutzer definiert](dx-graphics-hlsl-user-defined.md)                                | Benutzerdefinierte Struktur oder TypeDef                |
| Array                                                                                   | Literale skalare Ausdrücke, die mit den meisten anderen Typen deklariert sind                       |
| [State-Objekt](dx-graphics-hlsl-state-object.md) | HLSL-Darstellungen von Zustands Objekten |


 

Damit Sie besser verstehen können, wie Vektoren und Matrizen in HLSL verwendet werden, sollten Sie diese Hintergrundinformationen darüber lesen, wie HLSL [pro-Komponenten-](dx-graphics-hlsl-per-component-math.md) Math verwendet.
