# <a name="data-types-hlsl"></a>Datentypen (HLSL)

HLSL unterstützt viele verschiedene systeminterne Datentypen. Diese Tabelle zeigt, welche Typen zum Definieren von Shadervariablen verwendet werden sollen.



| Verwenden dieses systeminternen Typs                                                                                                                         | So definieren Sie diese Shadervariable                                            |
|-------------------------------------------------------------------------------------------------------------------------|--------------------------------------------|
| [Skalar](dx-graphics-hlsl-scalar.md)                                                                                   | Einkomponentenskalar                       |
| [Vektor,](dx-graphics-hlsl-vector.md) [Matrix](dx-graphics-hlsl-matrix.md)                                            | Vektor oder Matrix mit mehreren Komponenten        |
| [Sampler,](dx-graphics-hlsl-sampler.md) [Textur](dx-graphics-hlsl-texture.md) oder [Puffer](dx-graphics-hlsl-buffer.md)   | Sampler, Textur oder Pufferobjekt         |
| [Struktur](dx-graphics-hlsl-struct.md), [benutzerdefiniert](dx-graphics-hlsl-user-defined.md)                                | Benutzerdefinierte Struktur oder Typedef                |
| Array                                                                                   | Deklarierte literale Skalarausdrücke, die die meisten anderen Typen enthalten                       |
| [State-Objekt](dx-graphics-hlsl-state-object.md) | HLSL-Darstellungen von Zustandsobjekten |


 

Damit Sie besser verstehen können, wie Vektoren und Matrizen in HLSL verwendet werden, können Sie diese Hintergrundinformationen dazu lesen, wie HLSL mathematische Berechnungen [pro Komponente](dx-graphics-hlsl-per-component-math.md) verwendet.
