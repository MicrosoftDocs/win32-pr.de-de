# <a name="d-isolated-applications-and-side-by-side-assemblies"></a>D (isolierte Anwendungen und parallele Assemblys)

<dl> <dt>

<span id="_win32_def_context_assemblyidentity_gly"></span><span id="_WIN32_DEF_CONTEXT_ASSEMBLYIDENTITY_GLY"></span>**DEF-Kontext assemblyIdentity**
</dt> <dd>

Ein **assemblyIdentity** -Unterelement, das die parallele Assembly definiert, die durch das Manifest beschrieben wird. Das unter **geordnete Element "** DEF-Context assemblyIdentity" ist immer das erste Unterelement eines **Assemblyelements** .

</dd> <dt>

<span id="_win32_dll_versioning_conflict_gly"></span><span id="_WIN32_DLL_VERSIONING_CONFLICT_GLY"></span>**DLL-Versions Verwaltungs Konflikt**
</dt> <dd>

Kompatibilitätsproblem. Assemblyfreigabe-Probleme treten auf, wenn eine Anwendung eine Version einer freigegebenen Assembly installiert, die nicht abwärts kompatibel mit der zuvor installierten Version ist. Eine Lösung für Konflikte bei der dll-Versionsverwaltung besteht darin, eine parallele assemblyfreigabe und isolierte Anwendungen zu verwenden. Bei der parallelen assemblyfreigabe können mehrere Versionen derselben Windows-Assembly gleichzeitig ausgeführt werden. Entwickler können auswählen, welche parallele Assembly verwendet werden soll.

</dd> </dl>
