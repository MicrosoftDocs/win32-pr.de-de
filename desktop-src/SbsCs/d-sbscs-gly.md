<dl> <dt>

<span id="_win32_def_context_assemblyidentity_gly"></span><span id="_WIN32_DEF_CONTEXT_ASSEMBLYIDENTITY_GLY"></span>**DEF-context assemblyIdentity**
</dt> <dd>

Ein **assemblyIdentity-Unterelement,** das die durch das Manifest beschriebene nebenseitige Assembly definiert. Ein DEF-context **assemblyIdentity-Unterelement** ist immer das erste Unterelement eines **Assemblyelements.**

</dd> <dt>

<span id="_win32_dll_versioning_conflict_gly"></span><span id="_WIN32_DLL_VERSIONING_CONFLICT_GLY"></span>**Dll-Versionskonflikt**
</dt> <dd>

Kompatibilitätsproblem. Probleme bei der Assemblyfreigabe treten auf, wenn eine Anwendung eine Version einer freigegebenen Assembly installiert, die nicht abwärtskompatibel mit der zuvor installierten Version ist. Eine Lösung für Dll-Versionskonflikte besteht darin, die gemeinsame Nutzung von Assemblys und isolierte Anwendungen zu verwenden. Bei paralleler Assemblyfreigabe können mehrere Versionen derselben Windows Assembly gleichzeitig ausgeführt werden. Entwickler können auswählen, welche assembly nebeneinander verwendet werden soll.

</dd> </dl>
