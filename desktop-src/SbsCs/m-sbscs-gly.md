# <a name="m-isolated-applications-and-side-by-side-assemblies"></a>M (isolierte Anwendungen und nebeneinanderstehende Assemblys)

[A](a-sbscs-gly.md) B C [D](d-sbscs-gly.md) E F [G](g-sbscs-gly.md) H [I](i-sbscs-gly.md) J K L M N O [P](p-sbscs-gly.md) Q [R](r-sbscs-gly.md) [S](s-sbscs-gly.md) T U V W X Y Z

<dl> <dt>

<span id="_win32_manifest_file_gly"></span><span id="_WIN32_MANIFEST_FILE_GLY"></span>**Manifestdatei**
</dt> <dd>

XML-Datei, die eine nebeneinander oder isolierte Anwendung beschreibt. Eine Manifestdatei kann Bindungs- und Aktivierungsmetadaten enthalten, die üblicherweise in der Registrierung gespeichert sind, z. B. COM-Klassen, Schnittstellen und Typbibliotheken. Jede isolierte Anwendung oder nebeneinander stehende Assembly wird von einer Manifestdatei begleitet. Ein Anwendungsmanifest beschreibt eine vollständig oder teilweise isolierte Anwendung und ein Assemblymanifest eine nebeneinanderstehende Assembly.

</dd> <dt>

<span id="_win32_metadata_gly"></span><span id="_WIN32_METADATA_GLY"></span>**Metadaten**
</dt> <dd>

In einer Manifestdatei gespeicherte Informationen zur Bindung oder Aktivierung einer isolierten Anwendung oder parallelen Assembly. Beispielsweise können Informationen zu COM-Klassen, Schnittstellen und Typbibliotheken, die traditionell in der Registrierung gespeichert sind, als Metadaten in einem Manifest gespeichert werden. Metadaten beschreiben die Elemente, aus denen eine Assembly besteht, und ihre Abhängigkeiten von anderen nebenseitigen Assemblys. Die im Anwendungsmanifest und abhängigen Assemblys angegebenen Metadaten können verwendet werden, um den Aktivierungskontext aufzufüllen.

</dd> </dl>
