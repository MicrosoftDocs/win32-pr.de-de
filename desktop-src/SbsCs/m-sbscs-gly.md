# <a name="m-isolated-applications-and-side-by-side-assemblies"></a>M (isolierte Anwendungen und parallele Assemblys)

[a](a-sbscs-gly.md) B C [D](d-sbscs-gly.md) E F [G](g-sbscs-gly.md) H [I](i-sbscs-gly.md) J K L M N O [P](p-sbscs-gly.md) Q [R](r-sbscs-gly.md) [S](s-sbscs-gly.md) T U V W X Y Z

<dl> <dt>

<span id="_win32_manifest_file_gly"></span><span id="_WIN32_MANIFEST_FILE_GLY"></span>**Manifest-Datei**
</dt> <dd>

XML-Datei, die eine parallele Assembly oder eine isolierte Anwendung beschreibt. Eine Manifest-Datei kann Bindungs-und Aktivierungs Metadaten enthalten, die normalerweise in der Registrierung gespeichert sind, z. b. com-Klassen, Schnittstellen und Typbibliotheken. Jede isolierte Anwendung oder parallele Assembly wird von einer Manifest-Datei begleitet. Ein Anwendungs Manifest beschreibt eine vollständig oder teilweise isolierte Anwendung, und ein Assemblymanifest beschreibt eine parallele Assembly.

</dd> <dt>

<span id="_win32_metadata_gly"></span><span id="_WIN32_METADATA_GLY"></span>**benötigten**
</dt> <dd>

Informationen, die in einer Manifest-Datei zur Bindung oder Aktivierung einer isolierten Anwendung oder einer parallelen Assembly gespeichert sind. Beispielsweise können Informationen zu COM-Klassen,-Schnittstellen und-Typbibliotheken, die normalerweise in der Registrierung gespeichert sind, als Metadaten in einem Manifest gespeichert werden. In den Metadaten werden die Elemente beschrieben, aus denen eine Assembly besteht, sowie deren Abhängigkeiten von anderen parallelen Assemblys. Die im Anwendungs Manifest angegebenen Metadaten und abhängigen Assemblys können verwendet werden, um den Aktivierungs Kontext aufzufüllen.

</dd> </dl>
