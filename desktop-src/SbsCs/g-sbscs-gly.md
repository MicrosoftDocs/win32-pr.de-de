# <a name="g-isolated-applications-and-side-by-side-assemblies"></a>G (Isolierte Anwendungen und nebeneinanderstehende Assemblys)

[A](a-sbscs-gly.md) B C [D](d-sbscs-gly.md) E F G H [I](i-sbscs-gly.md) J K L [M](m-sbscs-gly.md) N O [P](p-sbscs-gly.md) Q [R](r-sbscs-gly.md) [S](s-sbscs-gly.md) T U V W X Y Z

<dl> <dt>

<span id="_win32_global_application_configuration_gly"></span><span id="_WIN32_GLOBAL_APPLICATION_CONFIGURATION_GLY"></span>**Globale Anwendungskonfiguration**
</dt> <dd>

Spezifikation, dass alle Anwendungen im System dieselbe parallele Assemblyversion verwenden. Die globale Konfiguration wird pro Assembly in einer globalen Assemblymanifestdatei angegeben, die im Winsxs-Ordner installiert ist. Die globale Konfiguration kann verwendet werden, wenn ein Assemblyentwickler oder -administrator alle Anwendungen auf dem System so konfigurieren muss, dass eine neuere Assemblyversion verwendet wird. In diesem Fall muss die neue Assembly abwärtskompatibel sein. Eine Anwendungsspezifische Konfiguration überschreibt die Standardkonfiguration oder globale Anwendungskonfiguration für bestimmte Anwendungen.

</dd> <dt>

<span id="_win32_global_assembly_manifest_gly"></span><span id="_WIN32_GLOBAL_ASSEMBLY_MANIFEST_GLY"></span>**Globales Assemblymanifest**
</dt> <dd>

Datei, die die globale Anwendungskonfiguration einer side-by-side-Assembly definiert. Globale Assemblymanifestdateien werden im Systemassemblyspeicher im Ordner Winsxs installiert.

</dd> </dl>
