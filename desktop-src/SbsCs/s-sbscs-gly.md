# <a name="s-isolated-applications-and-side-by-side-assemblies"></a>S (Isolierte Anwendungen und side-by-side-Assemblys)

[A](a-sbscs-gly.md) B C [D](d-sbscs-gly.md) E F [G](g-sbscs-gly.md) H [I](i-sbscs-gly.md) J K L [M](m-sbscs-gly.md) N O [P](p-sbscs-gly.md) Q [R](r-sbscs-gly.md) S T U V W X Y Z

<dl> <dt>

<span id="_win32_shared_side_by_side_assembly_gly"></span><span id="_WIN32_SHARED_SIDE_BY_SIDE_ASSEMBLY_GLY"></span>**Freigegebene, nebeneinander seitige Assembly**
</dt> <dd>

Eine freigegebene, gleichzeitige Assembly ist für die Verwendung durch mehrere Anwendungen auf dem Computer verfügbar. Sie wird im Ordner Winsxs des Verzeichnisses Windows installiert. Anwendungen und Administratoren können die Version einer freigegebenen Assembly verwalten, die verwendet werden soll, indem sie die Anwendungskonfiguration angeben. Nebengeordnete Assemblys werden immer von Manifestdateien begleitet, die dem Betriebssystem Informationen über die Assembly bereitstellen.

</dd> <dt>

<span id="_win32_side_by_side_assembly_gly"></span><span id="_WIN32_SIDE_BY_SIDE_ASSEMBLY_GLY"></span>**Side-by-Side-Assembly**
</dt> <dd>

Eine nebeneinander seitige Assembly ist eine Win32-Assembly, die von Manifesten begleitet wird. Eine nebenseitige Assembly enthält eine Sammlung von Ressourcen – eine Gruppe von DLLs, Windows-Klassen, COM-Servern, Typbibliotheken oder Schnittstellen –, die anwendungen immer zusammen zur Verfügung gestellt werden.

</dd> <dt>

<span id="_win32_side_by_side_assembly_sharing_gly"></span><span id="_WIN32_SIDE_BY_SIDE_ASSEMBLY_SHARING_GLY"></span>**Side-by-Side-Assemblyfreigabe**
</dt> <dd>

Verwendung von gleichzeitigen Assemblys zum sicheren Freigeben von Assemblys zwischen mehreren Anwendungen und zum Ausgleichen der negativen Auswirkungen der Freigabe, z. B. DLL-Konflikte. Anstatt eine einzelne Version einer Assembly zu verwenden, die Abwärtskompatibilität mit allen Anwendungen voraus setzt, ermöglicht die parallele Assemblyfreigabe die gleichzeitige Ausführung mehrerer Versionen einer Win32-Assembly auf dem System. Neben assemblyseitige Assemblys werden immer von Assemblymanifestdateien begleitet, die Informationen über die Assembly für das Betriebssystem bereitstellen. Anwendungen und Administratoren können die gemeinsame Nutzung von Assemblys nach der Bereitstellung verwalten, indem sie die Anwendungskonfiguration entweder global oder pro Anwendung aktualisieren.

</dd> </dl>
