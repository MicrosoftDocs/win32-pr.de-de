# <a name="i-isolated-applications-and-side-by-side-assemblies"></a>I (isolierte Anwendungen und parallele Assemblys)

<dl> <dt>

<span id="_win32_isolated_application_gly"></span><span id="_WIN32_ISOLATED_APPLICATION_GLY"></span>**isolierte Anwendung**
</dt> <dd>

Anwendung, die parallele Assemblys verwendet, aber nicht die zugehörigen Assemblys freigibt. Eine isolierte Anwendung wird immer von einer Anwendungs Manifest-Datei begleitet. Das Manifest gibt die Versionen der freigegebenen parallelen Assemblys an, an die die Anwendung zur Laufzeit gebunden wird, und kann einige Metadaten für private parallele Assemblys enthalten. Das Manifest enthält Informationen, die normalerweise in der Registrierung gespeichert sind. eine vollständig isolierte Anwendung ist möglicherweise unabhängig von der Registrierung.

</dd> </dl>
