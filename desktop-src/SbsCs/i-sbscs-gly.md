# <a name="i-isolated-applications-and-side-by-side-assemblies"></a>I (Isolierte Anwendungen und side-by-side-Assemblys)

<dl> <dt>

<span id="_win32_isolated_application_gly"></span><span id="_WIN32_ISOLATED_APPLICATION_GLY"></span>**Isolierte Anwendung**
</dt> <dd>

Anwendung, die gleichzeitige Assemblys verwendet, aber nicht deren Assemblys gemeinsam verwendet. Eine isolierte Anwendung wird immer von einer Anwendungsmanifestdatei begleitet. Das Manifest gibt die Versionen der freigegebenen, nebeneinander ausgeführten Assemblys an, an die die Anwendung zur Laufzeit gebunden wird, und enthält möglicherweise einige Metadaten für private, nebeneinander ausgeführte Assemblys. Das Manifest enthält Informationen, die traditionell in der Registrierung gespeichert sind. eine vollständig isolierte Anwendung kann unabhängig von der Registrierung sein.

</dd> </dl>
