# <a name="locale_name-constants"></a>Gebiets Schema \_ Namen \* Konstanten

In diesem Thema werden die \_ \* von NLS verwendeten Namen Konstanten für den Gebiets Schema definiert.



<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr class="header">
<th>Wert</th>
<th>Bedeutung</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td>LOCALE_NAME_INVARIANT</td>
<td>Name eines invarianten Gebiets Schemas, das stabile Gebiets Schema-und Kalenderdaten bereitstellt.</td>
</tr>
<tr class="even">
<td>LOCALE_NAME_MAX_LENGTH</td>
<td>Maximale Länge eines Gebiets Schema namens. Die maximale Anzahl von Zeichen, die für diese Zeichenfolge zulässig ist, ist 85, einschließlich eines abschließenden NULL-Zeichens.
<blockquote>
[!Note]<br />
Die Anwendung muss die Konstante für die maximale Länge des Gebiets Schema namens verwenden, anstatt den Wert 85 hart codieren zu müssen &quot; &quot; .
</blockquote>
<br/></td>
</tr>
<tr class="odd">
<td>LOCALE_NAME_SYSTEM_DEFAULT</td>
<td>Der Name des aktuellen Gebiets Schemas des Betriebssystems.</td>
</tr>
<tr class="even">
<td>LOCALE_NAME_USER_DEFAULT</td>
<td>Der Name des Gebiets Schemas des aktuellen Benutzers, der der im Bereich Regions-und Sprachoptionen in der Systemsteuerung festgelegten Einstellung entspricht. Dieses Gebiets Schema kann sich vom Gebiets Schema für die aktuelle Benutzeroberflächen Sprache unterscheiden.</td>
</tr>
</tbody>
</table>
