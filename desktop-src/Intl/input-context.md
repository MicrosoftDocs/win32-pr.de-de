# <a name="input-context"></a><span data-ttu-id="a9fe4-103">Eingabe Kontext</span><span class="sxs-lookup"><span data-stu-id="a9fe4-103">Input Context</span></span>

<span data-ttu-id="a9fe4-104">Ein "Eingabe Kontext" ist eine interne Struktur, die vom imm verwaltet wird.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-104">An "input context" is an internal structure maintained by the IMM.</span></span> <span data-ttu-id="a9fe4-105">Sie enthält Informationen zum Status von IME und wird von IME-Fenstern verwendet.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-105">It contains information about the status of the IME and is used by IME windows.</span></span> <span data-ttu-id="a9fe4-106">Standardmäßig erstellt das Betriebssystem jedem Thread einen Eingabe Kontext und weist ihn zu.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-106">By default, the operating system creates and assigns an input context to each thread.</span></span> <span data-ttu-id="a9fe4-107">Innerhalb des Threads handelt es sich bei diesem Standardeingabe Kontext um eine freigegebene Ressource, die mit jedem neu erstellten Fenster verknüpft ist.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-107">Within the thread, this default input context is a shared resource and is associated with each newly created window.</span></span>

<span data-ttu-id="a9fe4-108">Zum Abrufen oder Festlegen von Informationen im IME muss eine IME-fähige Anwendung zunächst ein Handle für den Eingabe Kontext abrufen, der einem angegebenen Fenster zugeordnet ist.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-108">To retrieve or set information in the IME, an IME-aware application must first retrieve a handle to the input context associated with a specified window.</span></span> <span data-ttu-id="a9fe4-109">Die Anwendung ruft das Handle mithilfe der [**immgetcontext**](/windows/desktop/api/Imm/nf-imm-immgetcontext) -Funktion ab.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-109">The application retrieves the handle by using the [**ImmGetContext**](/windows/desktop/api/Imm/nf-imm-immgetcontext) function.</span></span> <span data-ttu-id="a9fe4-110">Sie kann das abgerufene Handle in nachfolgenden Aufrufen der imm-Funktionen verwenden, um IME-Werte abzurufen und festzulegen, z. b. den Stil des Kompositions Fensters, den Kompositionsstil und die Statusfenster Position.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-110">It can use the retrieved handle in subsequent calls to the IMM functions to retrieve and set IME values, such as the composition window style, the composition style, and the status window position.</span></span> <span data-ttu-id="a9fe4-111">Nachdem die Anwendung die Verwendung des Kontexts abgeschlossen hat, muss Sie den Kontext mithilfe der [**immreleasecontext**](/windows/desktop/api/Imm/nf-imm-immreleasecontext) -Funktion freigeben.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-111">Once the application has finished using the context, it must release the context using the [**ImmReleaseContext**](/windows/desktop/api/Imm/nf-imm-immreleasecontext) function.</span></span>

<span data-ttu-id="a9fe4-112">Da der Standardeingabe Kontext eine freigegebene Ressource ist, gelten alle Änderungen, die die Anwendung daran vornimmt, für alle Fenster im Thread.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-112">Because the default input context is a shared resource, any changes the application makes to it apply to all windows in the thread.</span></span> <span data-ttu-id="a9fe4-113">Die Anwendung kann dieses Standardverhalten jedoch überschreiben, indem Sie Ihren eigenen Eingabe Kontext erstellt und einem oder mehreren Fenstern des Threads zuordnet.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-113">However, the application can override this default behavior by creating its own input context and associating it with one or more windows of the thread.</span></span> <span data-ttu-id="a9fe4-114">Die an einem anwendungsspezifischen Eingabe Kontext vorgenommenen Änderungen gelten nur für die Fenster, die dem Kontext zugeordnet sind.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-114">The changes made to an application-specific input context apply only to the windows associated with the context.</span></span>

<span data-ttu-id="a9fe4-115">Die Anwendung kann mithilfe der [**immkreatecontext**](/windows/desktop/api/Imm/nf-imm-immcreatecontext) -Funktion einen Eingabe Kontext erstellen.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-115">Your application can create an input context by using the [**ImmCreateContext**](/windows/desktop/api/Imm/nf-imm-immcreatecontext) function.</span></span> <span data-ttu-id="a9fe4-116">Um den Kontext einem Fenster zuzuweisen, ruft die Anwendung die [**immassociatecontext**](/windows/desktop/api/Imm/nf-imm-immassociatecontext) -Funktion auf.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-116">To assign the context to a window, the application calls the [**ImmAssociateContext**](/windows/desktop/api/Imm/nf-imm-immassociatecontext) function.</span></span> <span data-ttu-id="a9fe4-117">Diese Funktion gibt ein Handle für den zuvor zugeordneten Eingabe Kontext zurück.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-117">This function returns a handle to the previously associated input context.</span></span> <span data-ttu-id="a9fe4-118">Wenn die Anwendung nicht bereits einen Eingabe Kontext mit dem Fenster verknüpft hat, ist das zurückgegebene Handle für den Standardeingabe Kontext.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-118">If the application has not already associated an input context with the window, the returned handle is for the default input context.</span></span> <span data-ttu-id="a9fe4-119">In der Regel speichert die Anwendung dieses Handle und ordnet sie später dem Fenster zu, wenn der angepasste Eingabe Kontext nicht mehr benötigt wird.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-119">Typically, the application saves this handle and later reassociates it with the window when the customized input context is no longer needed.</span></span>

<span data-ttu-id="a9fe4-120">Sobald ein Eingabe Kontext einem Fenster zugeordnet ist, wählt das Betriebssystem automatisch diesen Kontext aus, wenn das Fenster aktiviert wird, und empfängt den Eingabefokus.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-120">Once an input context is associated with a window, the operating system automatically selects that context when the window is activated and receives the input focus.</span></span> <span data-ttu-id="a9fe4-121">Der Stil und andere Informationen im Eingabe Kontext beeinflussen die nachfolgende Tastatureingabe für dieses Fenster, um zu bestimmen, wie der IME funktioniert.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-121">The style and other information in the input context affects subsequent keyboard input for that window, determining how the IME operates.</span></span>

<span data-ttu-id="a9fe4-122">Die Anwendung muss alle benutzerdefinierten Eingabe Kontexte zerstören, bevor Sie beendet wird.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-122">Your application must destroy any customized input context before it terminates.</span></span> <span data-ttu-id="a9fe4-123">Zuerst entfernt die Anwendung den Eingabe Kontext aus jeder Zuordnung, die Sie mit Windows im Thread erstellt hat, mithilfe der [**immassociatecontext**](/windows/desktop/api/Imm/nf-imm-immassociatecontext) -Funktion.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-123">First, the application removes the input context from any association it has made with windows in the thread by using the [**ImmAssociateContext**](/windows/desktop/api/Imm/nf-imm-immassociatecontext) function.</span></span> <span data-ttu-id="a9fe4-124">Anschließend wird die Funktion " [**immdestroycontext**](/windows/desktop/api/Imm/nf-imm-immdestroycontext) " aufgerufen.</span><span class="sxs-lookup"><span data-stu-id="a9fe4-124">Then, it calls the [**ImmDestroyContext**](/windows/desktop/api/Imm/nf-imm-immdestroycontext) function.</span></span>
