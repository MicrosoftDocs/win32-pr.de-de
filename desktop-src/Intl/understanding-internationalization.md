# <a name="understanding-internationalization"></a><span data-ttu-id="bd5ff-103">Grundlegendes zur Internationalisierung</span><span class="sxs-lookup"><span data-stu-id="bd5ff-103">Understanding Internationalization</span></span>

<span data-ttu-id="bd5ff-104">Die Entwicklung internationalisierter Softwareprodukte erfordert nicht nur die Produktion von korrekter Code, sondern auch einen Entwurf, der es dem Produkt ermöglicht, viele Gebiets Schemas zu unterstützen.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-104">Developing internationalized software products requires not only the production of correct code, but also a design that permits the product to support many locales.</span></span> <span data-ttu-id="bd5ff-105">Entwickler und ihre Vorgesetzten sollten sich mit dem Aufwand und der Aufmerksamkeit befassen, die für die Erstellung weltweit Einsatz fähiger Anwendungen erforderlich sind, und zwar unabhängig davon, ob Sie als einzelne Binärdateien bereitgestellt werden, die für die Verwendung in vielen verschiedenen Märkten bereit stehen, oder als mehrere Binärdateien, die jeweils für ein einzelnes Gebiets Schema spezifisch sind.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-105">Developers and their managers should be aware of the level of effort and attention to detail required to create world-ready applications, whether they deliver them as single binaries ready for use in many different markets, or as multiple binaries that are each specific to one locale.</span></span> <span data-ttu-id="bd5ff-106">Als Entwickler müssen Sie sicherstellen, dass ihre Verwaltung weiß, dass die Internationalisierung mehr als nur die Code Entwicklung umfasst.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-106">As a developer, make sure your management understands that internationalization involves more than just code development.</span></span> <span data-ttu-id="bd5ff-107">Der Entwurf für die Internationalisierung am Anfang des Produktzyklen spart Zeit, Geld und Aufwand.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-107">Designing for internationalization at the beginning of your product cycle saves you time, money, and effort.</span></span>

<span data-ttu-id="bd5ff-108">**Internationalisierung** ist die Erstellung von Software, die sich an Gebiets Schemas auf der ganzen Welt anpasst.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-108">**Internationalization** is the creation of software that adapts to locales around the world.</span></span> <span data-ttu-id="bd5ff-109">Bei einem Gebiets **Schema handelt es sich um eine Sammlung** von sprachbezogenen Benutzereinstellungen.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-109">A **locale** is a collection of language-related user preferences.</span></span> <span data-ttu-id="bd5ff-110">Ein Gebiets Schema wird durch die Kopplung einer Sprache und einer geografischen Region angegeben.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-110">A locale is specified by the pairing of a language and a geographical region.</span></span> <span data-ttu-id="bd5ff-111">Beispielsweise sind Englisch (USA) und Englisch (Vereinigtes Königreich) zwei verschiedene Gebiets Schemas, wie z. b. Englisch (Kanada) und Französisch (Kanada).</span><span class="sxs-lookup"><span data-stu-id="bd5ff-111">For example, English (United States) and English (United Kingdom) are two different locales, as are English (Canada) and French (Canada).</span></span>

<span data-ttu-id="bd5ff-112">Internationalisierte Software weist folgende Attribute auf:</span><span class="sxs-lookup"><span data-stu-id="bd5ff-112">Internationalized software has these attributes:</span></span>

-   <span data-ttu-id="bd5ff-113">Die **weltweite Bereitschaft** deckt den Entwurf und die Implementierung ab, die Gebiets Schema unabhängigen Code von Gebiets Schema abhängigen Ressourcen trennen und zwei Hauptbereiche umfasst:</span><span class="sxs-lookup"><span data-stu-id="bd5ff-113">**World-readiness** covers design and implementation that separate locale-independent code from locale-dependent resources, and comprises two major areas:</span></span>
    -   <span data-ttu-id="bd5ff-114">Bei der **Globalisierung** handelt es sich um den Prozess der Entwicklung eines Programm Kerns mit Features und Code Entwürfen, die unabhängig von Sprache oder Gebiets Schema sind.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-114">**Globalization** is the process of developing a program core with features and code design that are independent of language or locale.</span></span> <span data-ttu-id="bd5ff-115">Stattdessen unterstützt der Entwurf die Eingabe, Anzeige und Ausgabe von Unicode-unterstützten sprach Skripts sowie Daten, die sich auf bestimmte Gebiets Schemas beziehen.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-115">Instead, the design supports the input, display, and output of Unicode-supported language scripts, as well as data related to specific locales.</span></span>
    -   <span data-ttu-id="bd5ff-116">**Lokalisier barkeit** ist der Entwurf der Software Codebasis und der Ressourcen, sodass ein Programm, wie unten definiert, in verschiedene sprach Editionen ohne Änderungen am Quellcode lokalisiert werden kann.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-116">**Localizability** is the design of the software code base and resources such that a program can be localized, as defined below, into different language editions without any changes to the source code.</span></span> <span data-ttu-id="bd5ff-117">Beispielsweise müssen Zeichen folgen Puffer im Code und die Textfelder in der Benutzeroberfläche groß genug sein, um längere Text Zeichenfolgen in Sprachen wie Deutsch oder Niederländisch aufnehmen zu können.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-117">For example, string buffers in the code, and text boxes in the UI, must be large enough to accommodate longer text strings in languages such as German or Dutch.</span></span>
-   <span data-ttu-id="bd5ff-118">**Lokalisierung** Dies umfasst das Übersetzen und Anpassen eines Produkts für einen bestimmten Markt, und es geht hauptsächlich um das Erstellen von Ressourcen Dateien, anstatt Quellcode zu schreiben.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-118">**Localization** This involves translating and customizing a product for a specific market, and it is mainly about creating resource files rather than writing source code.</span></span>

<span data-ttu-id="bd5ff-119">Beispielsweise ist die Verwendung der von der Microsoft Win32-API bereitgestellten NLS (National Language Support) ein Softwareentwicklungsprozess, der die weltweite Bereitschaft unterstützt, während das Ändern der Benutzeroberflächen Elemente, die Übersetzung von Text und die Standardisierung der Terminologie in der Regel durch eine Person, die ein Sprachspezialist ist (z. b. ein Konvertierungsprogramm)</span><span class="sxs-lookup"><span data-stu-id="bd5ff-119">For example, using the National Language Support (NLS) supplied by the Microsoft Win32 API is a software development process that supports world-readiness, whereas modifying the UI elements, translating text, and standardizing terminology are localization steps usually performed by someone who is a language specialist, such as a translator.</span></span> <span data-ttu-id="bd5ff-120">Selbst wenn Ihre Code Entwicklung die Welt Bereitschaft hervorgehoben hat, müssen Sie die Lokalisierung verstehen, da sich Ihre Entwurfsentscheidungen auf die Arbeit des Konvertierungs Programms auswirken.</span><span class="sxs-lookup"><span data-stu-id="bd5ff-120">Even when your code development emphasizes world-readiness, you need to understand localization because your design decisions affect the work of the translator.</span></span>
