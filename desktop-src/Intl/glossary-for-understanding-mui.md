# <a name="glossary-for-understanding-mui"></a>Glossar zum Verstehen von GLOSSAR

-   [Globalisierung](#globalization)
-   [Lokalisierbarkeit](#localizability)
-   [Lokalisierung](#localization)
-   [Internationalisierung](#internationalization)

## <a name="globalization"></a>Globalisierung

Der Begriff *Globalisierung* gibt an, dass die Plattform überall einsatzbereit sein muss. In der Praxis wird diese Vision innerhalb Windows realisiert, indem eine einzelne globale Binärdatei für das Windows Betriebssystem verwendet wird, beginnend mit dem release Windows Vista, durch die Unterstützung der Bereitstellung einzelner Images in einem mehrsprachigen Kontext und durch die Bereitstellung von NLS-APIs zum Abrufen gebietsschemaspezifischer Daten.

## <a name="localizability"></a>Lokalisierbarkeit

Der Begriff *Lokalisierbarkeit* bedeutet, dass das Produkt einfacher lokalisiert werden kann, was wiederum die Lokalisierungskosten senken kann. Dies kann durch die effiziente Verwendung von Ressourcen-APIs, die Steuerung des Umfangs und der Anzahl der dem System hinzugefügten Ressourcen und die Verbesserung der Tools erreicht werden, die für die Arbeit mit diesen Ressourcen verwendet werden.

## <a name="localization"></a>Lokalisierung

*Lokalisierung* ist der Prozess der Anpassung eines Programms für einen bestimmten lokalen Markt. Dazu gehören die Übersetzung der Benutzeroberfläche, das Ändern der Größe von Dialogfeldern, das Anpassen von Features (falls erforderlich) und das Testen der Ergebnisse, um sicherzustellen, dass das Programm weiterhin funktioniert. Im am häufigsten verwendeten Kontext konzentriert sich die Lokalisierung auf die Übersetzung der Benutzeroberfläche.

## <a name="internationalization"></a>Internationalisierung

Die *Internationalisierung* wird an einigen Stellen als Synonym für die Globalisierung oder als Globalisierung und Lokalisierung an anderen Stellen verwendet, um alle Aktivitäten zu beschreiben, die ein Produkt im globalen Kontext verbessern.
