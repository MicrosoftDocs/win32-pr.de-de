## <a name="design-concepts"></a><span data-ttu-id="1ff55-152">Entwurfskonzepte</span><span class="sxs-lookup"><span data-stu-id="1ff55-152">Design concepts</span></span>

<span data-ttu-id="1ff55-153">Strukturen dienen dazu, Daten zu organisieren und die Suche zu erleichtern, aber es ist schwierig, Daten innerhalb einer Struktur leicht auffindbar zu machen.</span><span class="sxs-lookup"><span data-stu-id="1ff55-153">Trees are intended to organize data and make it easy to find, yet it's difficult to make data within a tree easily discoverable.</span></span> <span data-ttu-id="1ff55-154">Beachten Sie bei der Entscheidung über Strukturansichten und deren Organisation die folgenden Prinzipien.</span><span class="sxs-lookup"><span data-stu-id="1ff55-154">Keep the following principles in mind when deciding about tree views and their organization.</span></span>

### <a name="predictability-and-discoverability"></a><span data-ttu-id="1ff55-155">Vorhersagbarkeit und Auffindbarkeit</span><span class="sxs-lookup"><span data-stu-id="1ff55-155">Predictability and discoverability</span></span>

<span data-ttu-id="1ff55-156">**Eine Strukturansicht basiert auf den Beziehungen zwischen Objekten.**</span><span class="sxs-lookup"><span data-stu-id="1ff55-156">**A tree view is based on the relationships between objects.**</span></span> <span data-ttu-id="1ff55-157">Strukturen funktionieren am besten, wenn die Objekte eine klare, bekannte, sich gegenseitig ausschließende Beziehung bilden, in der jedes Objekt einem einzelnen, leicht zu bestimmenden Container zugeordnet ist.</span><span class="sxs-lookup"><span data-stu-id="1ff55-157">Trees work best when the objects form a clear, well known, mutually exclusive relationship in which every object maps to a single, easy-to-determine container.</span></span>

<span data-ttu-id="1ff55-158">**Ein erhebliches Problem besteht darin, dass ein Objekt auf verschiedenen Knoten angezeigt werden kann.**</span><span class="sxs-lookup"><span data-stu-id="1ff55-158">**A significant problem is that an object can appear in different nodes.**</span></span> <span data-ttu-id="1ff55-159">Wo würden Benutzer beispielsweise erwarten, ein Hardwaregerät zu finden, das Musik spielt, über eine große Festplatte verfügt und einen USB-Anschluss verwendet?</span><span class="sxs-lookup"><span data-stu-id="1ff55-159">For example, where would users expect to find a hardware device that plays music, has a large hard disk, and uses a USB port?</span></span> <span data-ttu-id="1ff55-160">Möglicherweise in einem von mehreren verschiedenen Containerknoten, z. B. Multimedia, Speicher, USB und möglicherweise in Hardwareressourcen.</span><span class="sxs-lookup"><span data-stu-id="1ff55-160">Perhaps in any of several different container nodes, such as Multimedia, Storage, USB, and possibly in Hardware Resources.</span></span> <span data-ttu-id="1ff55-161">Eine Lösung besteht darin, jedes Objekt unabhängig von den Umständen unter dem jeweils am besten geeigneten Container zu platzieren. Ein anderer Ansatz besteht darin, jedes Objekt unter allen containern zu platzieren, die angewendet werden.</span><span class="sxs-lookup"><span data-stu-id="1ff55-161">One solution is to place each object under the single most appropriate container regardless of the circumstances; another approach is to place each object under all containers that apply.</span></span> <span data-ttu-id="1ff55-162">Erstere höher gestuft eine einfache, saubere Hierarchie und die zweite höher erfindbarkeit jeder hat Vorteile und potenzielle Probleme.</span><span class="sxs-lookup"><span data-stu-id="1ff55-162">The former promotes a simple, clean hierarchy and the latter promotes discoverability each has advantages and potential problems.</span></span>

<span data-ttu-id="1ff55-163">**Benutzer verstehen das Layout der Struktur möglicherweise nicht vollständig, bilden aber ein mentales Modell der Beziehungen, nachdem sie eine Weile mit der Struktur interagiert haben.**</span><span class="sxs-lookup"><span data-stu-id="1ff55-163">**Users may not completely understand the layout of the tree, but they will form a mental model of the relationships after interacting with the tree for a while.**</span></span> <span data-ttu-id="1ff55-164">Wenn dieses mentale Modell falsch ist, führt dies zu Verwirrung.</span><span class="sxs-lookup"><span data-stu-id="1ff55-164">If that mental model is incorrect, it leads to confusion.</span></span> <span data-ttu-id="1ff55-165">Angenommen, ein Musikplayer ist in den Containern Multimedia, Storage und USB zu finden.</span><span class="sxs-lookup"><span data-stu-id="1ff55-165">For example, suppose a music player can be found in the Multimedia, Storage, and USB containers.</span></span> <span data-ttu-id="1ff55-166">Diese Anordnung verbessert die Auffindbarkeit.</span><span class="sxs-lookup"><span data-stu-id="1ff55-166">This arrangement improves discoverability.</span></span> <span data-ttu-id="1ff55-167">Wenn ein Benutzer das Gerät zum ersten Mal in Multimedia findet, kann der Benutzer daraus schließen, dass alle Geräte wie Musikplayer im Multimediacontainer angezeigt werden.</span><span class="sxs-lookup"><span data-stu-id="1ff55-167">If a user first finds the device in Multimedia, the user may conclude that all devices like music players appear in the Multimedia container.</span></span> <span data-ttu-id="1ff55-168">Der Benutzer erwartet dann, dass ähnliche Geräte, z. <span data-ttu-id="1ff55-169">**Die Herausforderung beim Entwerfen einer Struktur besteht darin, die Auffindbarkeit mit einem vorhersagbaren Benutzermodell in Einklang zu bringen, das Verwechslungen minimiert.**</span><span class="sxs-lookup"><span data-stu-id="1ff55-169">**The challenge when designing a tree is to balance discoverability with a predictable user model that minimizes confusion.**</span></span>

### <a name="breadth-vs-depth"></a><span data-ttu-id="1ff55-170">Breite und Tiefe</span><span class="sxs-lookup"><span data-stu-id="1ff55-170">Breadth vs. depth</span></span>

<span data-ttu-id="1ff55-171">Benutzerfreundlichkeitsstudien haben gezeigt, dass **Benutzer objekte in einer Struktur, die breit ist, erfolgreicher finden als in einer Struktur, die tief ist.** Wenn Sie also eine Struktur entwerfen, sollten Sie die Breite über die Tiefe bevorzugen.</span><span class="sxs-lookup"><span data-stu-id="1ff55-171">Usability studies have shown that **users are more successful at finding objects in a tree that is broad than in a tree that is deep**, so when designing a tree you should prefer breadth over depth.</span></span> <span data-ttu-id="1ff55-172">Im Idealfall sollte eine Struktur nicht mehr als vier Ebenen (ohne Zählung des Stammknotens) aufweisen, und die am häufigsten verwendeten Objekte sollten in den ersten beiden Ebenen angezeigt werden.</span><span class="sxs-lookup"><span data-stu-id="1ff55-172">Ideally, a tree should have no more than four levels (not counting the root node) and the most commonly accessed objects should appear in the first two levels.</span></span>
