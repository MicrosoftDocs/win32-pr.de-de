## <a name="design-concepts"></a><span data-ttu-id="bb3d3-152">Entwurfskonzepte</span><span class="sxs-lookup"><span data-stu-id="bb3d3-152">Design concepts</span></span>

<span data-ttu-id="bb3d3-153">Strukturen sind zum Organisieren von Daten gedacht und können problemlos gefunden werden, aber es ist schwierig, die Daten in einer Struktur leicht auffindbar zu machen.</span><span class="sxs-lookup"><span data-stu-id="bb3d3-153">Trees are intended to organize data and make it easy to find, yet it's difficult to make data within a tree easily discoverable.</span></span> <span data-ttu-id="bb3d3-154">Beachten Sie bei der Entscheidung über Struktur Ansichten und deren Organisation die folgenden Prinzipien.</span><span class="sxs-lookup"><span data-stu-id="bb3d3-154">Keep the following principles in mind when deciding about tree views and their organization.</span></span>

### <a name="predictability-and-discoverability"></a><span data-ttu-id="bb3d3-155">Vorhersagbarkeit und Auffindbarkeit</span><span class="sxs-lookup"><span data-stu-id="bb3d3-155">Predictability and discoverability</span></span>

<span data-ttu-id="bb3d3-156">**Eine Strukturansicht basiert auf den Beziehungen zwischen Objekten.**</span><span class="sxs-lookup"><span data-stu-id="bb3d3-156">**A tree view is based on the relationships between objects.**</span></span> <span data-ttu-id="bb3d3-157">Strukturen funktionieren am besten, wenn die Objekte eine klare, bekannte, sich gegenseitig ausschließende Beziehung bilden, in der jedes Objekt einem einzelnen, leicht zu erführenden Container zugeordnet ist.</span><span class="sxs-lookup"><span data-stu-id="bb3d3-157">Trees work best when the objects form a clear, well known, mutually exclusive relationship in which every object maps to a single, easy-to-determine container.</span></span>

<span data-ttu-id="bb3d3-158">**Ein erhebliches Problem ist, dass ein Objekt in verschiedenen Knoten vorkommen kann.**</span><span class="sxs-lookup"><span data-stu-id="bb3d3-158">**A significant problem is that an object can appear in different nodes.**</span></span> <span data-ttu-id="bb3d3-159">Wenn z. b. die Benutzer ein Hardware Gerät finden, das Musik spielt, eine große Festplatte hat und einen USB-Anschluss verwendet?</span><span class="sxs-lookup"><span data-stu-id="bb3d3-159">For example, where would users expect to find a hardware device that plays music, has a large hard disk, and uses a USB port?</span></span> <span data-ttu-id="bb3d3-160">Vielleicht in einem von mehreren unterschiedlichen Container Knoten wie Multimedia, Storage, USB und möglicherweise in Hardware Ressourcen.</span><span class="sxs-lookup"><span data-stu-id="bb3d3-160">Perhaps in any of several different container nodes, such as Multimedia, Storage, USB, and possibly in Hardware Resources.</span></span> <span data-ttu-id="bb3d3-161">Eine Lösung besteht darin, jedes Objekt unabhängig von den Bedingungen unter dem jeweils am besten geeigneten Container zu platzieren. ein anderer Ansatz besteht darin, jedes Objekt in alle Container zu platzieren, die angewendet werden.</span><span class="sxs-lookup"><span data-stu-id="bb3d3-161">One solution is to place each object under the single most appropriate container regardless of the circumstances; another approach is to place each object under all containers that apply.</span></span> <span data-ttu-id="bb3d3-162">Der erste fördert eine einfache, saubere Hierarchie, und letztere stuft die Auffindbarkeit herauf, die jeweils vor-und potenziellen Problemen stehen.</span><span class="sxs-lookup"><span data-stu-id="bb3d3-162">The former promotes a simple, clean hierarchy and the latter promotes discoverability each has advantages and potential problems.</span></span>

<span data-ttu-id="bb3d3-163">**Benutzer können das Layout der Struktur nicht vollständig verstehen, Sie bilden jedoch ein geistiges Modell der Beziehungen, nachdem Sie eine Weile mit der Struktur interagieren.**</span><span class="sxs-lookup"><span data-stu-id="bb3d3-163">**Users may not completely understand the layout of the tree, but they will form a mental model of the relationships after data-stu-id="bb3d3-164">If that mental model is incorrect, it leads to confusion.</span></span> <span data-ttu-id="bb3d3-165">Nehmen wir beispielsweise an, dass sich ein Musikplayer in Multimedia-, Speicher-und USB-Containern befindet.</span><span class="sxs-lookup"><span data-stu-id="bb3d3-165">For example, suppose a music player can be found in the Multimedia, Storage, and USB discoverability.</span></span> <span data-ttu-id="bb3d3-167">Wenn ein Benutzer das Gerät zum ersten Mal in Multimedia findet, wird der Benutzer möglicherweise davon ausgehen, dass alle Geräte wie <span data-ttu-id="bb3d3-169">**Die Herausforderung beim Entwerfen einer Struktur besteht darin, die Auffindbarkeit mit einem vorhersagbaren Benutzer Modell auszugleichen, das Verwirrung minimiert.**</span><span class="sxs-lookup"><span data-stu-id="bb3d3-169">**The challenge when designing a tree is to balance discoverability with a predictable user model that minimizes confusion.**</span></span>

### <a name="breadth-vs-depth"></a><span data-ttu-id="bb3d3-170">Breite und Tiefe</span><span class="sxs-lookup"><span data-stu-id="bb3d3-170">Breadth vs. depth</span></span>

<span data-ttu-id="bb3d3-171">Nutzbarkeits Studien haben ergeben, dass die **Benutzer bei der Suche nach Objekten in einer Struktur, die weitgehend stark ist, nicht in einer tiefgreifenden Struktur erfolgreich sind**.
