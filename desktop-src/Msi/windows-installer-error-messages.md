# <a name="windows-installer-error-messages-for-developers"></a>Windows Installer von Fehlermeldungen (für Entwickler)

> [!NOTE]  
> Dieser Artikel richtet sich an Softwareentwickler, die Windows Installer verwenden, um Installer-Pakete für Ihre Anwendungen zu erstellen. Wenn ein Benutzer während oder nach dem Installieren oder Deinstallieren einer Anwendung Probleme mit dem Computer hat, wenden Sie sich an den Kundensupport für die Software, die Sie installieren oder entfernen möchten. Wenn Sie der Meinung sind, dass Sie Unterstützung für ein Microsoft-Produkt benötigen, besuchen Sie unsere technische Support Website unter [Support.Microsoft.com](https://support.microsoft.com).

Die in diesem Thema beschriebenen Fehlercodes werden vom Windows Installer zurückgegeben und weisen die Fehlercodes 1000 oder höher auf. Die Fehlercodes 1000 bis 1999 sind Versand Fehler und müssen in der [Fehler Tabelle](error-table.md)erstellt werden. Die mehr als 2000 nummerierten Fehlercodes sind interne Fehler und verfügen nicht über erstellte Zeichen folgen. Diese können jedoch auftreten, wenn das Installationspaket falsch erstellt wurde. Informationen zu Fehlercodes, die für die Windows Installer Funktionen **MsiExec.exe** und **InstMsi.exe** spezifisch sind, finden Sie unter [MsiExec.exe und InstMsi.exe Fehlermeldungen](error-codes.md). Eine Liste der reservierten Fehlercodes finden Sie unter [Error Table](error-table.md). Sie können das Internet oder die [Microsoft Support-Website](https://support.microsoft.com/) nach Lösungen für viele der Nachrichten in der folgenden Tabelle durchsuchen.

Entwickler von Installationspaketen können auch die interne Konsistenz Ihrer Pakete mithilfe interner Konsistenz Auswertungen testen. Weitere Informationen finden Sie unter [internal Konsistenz Evaluators-ICES](internal-consistency-evaluators-ices.md).
