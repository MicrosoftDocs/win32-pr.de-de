# <a name="windows-installer-error-messages-for-developers"></a>Windows Installer-Fehlermeldungen (für Entwickler)

> [!NOTE]  
> Dieser Artikel richtet sich an Softwareentwickler, die Windows Installer verwenden, um Installationspakete für ihre Anwendungen zu erstellen. Wenn Benutzer Probleme mit Ihrem Computer haben, während oder nach der Installation oder Deinstallation einer Anwendung, wenden Sie sich an den Kundensupport für die Software, die Sie installieren oder entfernen möchten. Wenn Sie der Meinung sind, dass Sie Support für ein Microsoft-Produkt benötigen, besuchen Sie unsere Website für technischen Support [unter support.microsoft.com.](https://support.microsoft.com)

Die in diesem Thema beschriebenen Fehlercodes werden vom Windows Installer zurückgegeben und enthalten Fehlercodes von 1000 oder höher. Die Fehlercodes 1000 bis 1999 sind Versandfehler und müssen in der [Tabelle Error (Fehler) verfasst werden.](error-table.md) Die Fehlercodes, die größer als 2000 sind, sind interne Fehler und enthalten keine erstellten Zeichenfolgen. Diese können jedoch auftreten, wenn das Installationspaket falsch erstellt wurde. Spezifische Fehlercodes für die funktionen Windows Installer **MsiExec.exe** **und** InstMsi.exefinden Sie unterMsiExec.exe und [InstMsi.exe Fehlermeldungen.](error-codes.md) Eine Liste der reservierten Fehlercodes finden Sie in der [Fehlertabelle](error-table.md). Sie können im Internet oder auf der [Microsoft-Supportwebsite](https://support.microsoft.com/) nach Lösungen für viele der Nachrichten in der folgenden Tabelle suchen.

Entwickler von Installationspaketen können auch die interne Konsistenz ihrer Pakete mithilfe von internen Konsistenzauswertungen testen. Weitere Informationen finden Sie unter [Interne Konsistenzauswertungen – ICEs](internal-consistency-evaluators-ices.md).
