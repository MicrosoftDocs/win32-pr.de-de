<span id="_msi_source_list_gly"></span><span id="_MSI_SOURCE_LIST_GLY"></span>**Quell Liste**

</dd> <dt>

<span id="_msi_standard_action_gly"></span><span id="_MSI_STANDARD_ACTION_GLY"></span>**Standardaktion**

</dd> <dt>

<span id="_msi_standard_user_gly"></span><span id="_MSI_STANDARD_USER_GLY"></span>**Standardbenutzer**
</dt> <dd>

Benutzer ohne Administratorrechte.

</dd> <dt>

<span id="_msi_structured_query_language_gly"></span><span id="_MSI_STRUCTURED_QUERY_LANGUAGE_GLY"></span>**Structured Query Language (SQL)**
</dt> <dd>

Sprache, die zum Verarbeiten von Daten in einer relationalen Datenbank verwendet wird. Sie können SQL verwenden, um Abfragen an die Installer-Datenbank vorzunehmen. Weitere Informationen finden Sie unter [Arbeiten mit Abfragen](working-with-queries.md).

</dd> <dt>

<span id="_msi_summary_information_stream_gly"></span><span id="_MSI_SUMMARY_INFORMATION_STREAM_GLY"></span>**Zusammenfassungs Informationsdaten Strom**
</dt> <dd>

Speicherort in der [*MSI-Datei*](m-gly.md) , um Informationen zu erhalten, die mit Microsoft Windows-Explorer angezeigt werden können. Weitere Informationen finden Sie unter Informationen [zum Zusammenfassungs Informationsdaten Strom](about-the-summary-information-stream.md).

</dd> <dt>

<span id="_msi_synchronous_execution_gly"></span><span id="_MSI_SYNCHRONOUS_EXECUTION_GLY"></span>**synchrone Ausführung**
</dt> <dd>

Wenn das Installationsprogramm darauf wartet, dass der separate Thread einer benutzerdefinierten Aktion beendet wird, bevor der Thread der aktuellen Installation fortgesetzt wird. Weitere Informationen finden Sie unter [synchrone und asynchrone benutzerdefinierte Aktionen](synchronous-and-asynchronous-custom-actions.md) .

</dd> <dt>

<span id="_msi_system_policy_using_windows_installer__gly"></span><span id="_MSI_SYSTEM_POLICY_USING_WINDOWS_INSTALLER__GLY"></span>**System Richtlinie**
</dt> <dd>

Wird von Systemadministratoren verwendet, um Benutzer-und Computerkonfigurationen von einem einzigen Speicherort in einem Netzwerk zu steuern. System Richtlinien übertragen Registrierungs Einstellungen an eine große Anzahl von Computern, ohne dass der Administrator über detaillierte Kenntnisse der Registrierung verfügen muss. Die System Richtlinie des Installers kann mithilfe des Gruppenrichtlinie-Editors (gpe) konfiguriert werden, der in Windows 2000 enthalten ist. Weitere Informationen finden Sie unter [System Richtlinie](system-policy.md).

</dd> </dl>
