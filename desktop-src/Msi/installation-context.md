# <a name="installation-context"></a>Installations Kontext

Windows Installer können ein Paket auf einem Computer in zwei Installations Kontexten installieren: pro Computer und pro Benutzer. Eine Computer spezifische Installation des Pakets ist erforderlich, damit alle Benutzer des Computers auf die Anwendung zugreifen und diese verwenden können. Da eine Computer spezifische Installation Änderungen am System vornimmt, die alle Benutzer betreffen, können Standardbenutzer mit eingeschränkten Berechtigungen daran gehindert werden, ein Paket in den kontextbezogenen Kontext zu installieren, ohne zuvor die Berechtigung zu erhalten.

Sie können den Installations Kontext angeben, indem Sie das Paket für die Installation pro Benutzer oder pro Computer erstellen und die Eigenschaften " [**ALLUSERS**](allusers.md) " und " [**msiinstallperuser**](msiinstallperuser.md) " verwenden. Basierend auf diesen Eigenschaften werden die Werte von Ordnereigenschaften und Registrierungen von Windows Installer automatisch an Positionen für den Kontext pro Benutzer oder Computer bezogen umgeleitet.

> [!Note]  
> Die [**msiinstallperuser**](msiinstallperuser.md) -Eigenschaft, die ab Windows Installer 5,0 und Windows Server 2008 R2 und Windows 7 verfügbar ist, kann die Entwicklung eines einzelnen Pakets vereinfachen, das entweder im Computer-oder benutzerspezifischen Kontext installiert werden kann. Weitere Informationen zum Entwickeln eines Pakets mit zwei Zweck, mit dem der Benutzer den Installations Kontext zum Installations Zeitpunkt auswählen kann, finden Sie unter [Erstellen eines einzelnen Pakets](single-package-authoring.md). Windows Installer ignoriert die **msiinstallperuser** -Eigenschaft, wenn der Wert von [**ALLUSERS**](allusers.md) nicht 2 ist. Windows Installer setzt den Wert von **ALLUSERS** immer auf 1 zurück, wenn er im computerspezifischen Kontext installiert wird, und setzt den Wert von **ALLUSERS** auf eine leere Zeichenfolge ("") zurück, wenn er im kontextbezogenen Kontext installiert wird.

| Verknüpfungen werden im Profil "alle Benutzer" installiert.<br/>                                              | Verknüpfungen werden nur für das Profil dieses Benutzers installiert.<br/>                                                                 |

> Ordner werden durch ihre [**KNOWNFOLDERID**](../shell/knownfolderid.md) -und [**CSIDL**](/windows/win32/api/shlobj_core/nf-shlobj_core-shgetfolderpatha) -Konstanten identifiziert.
