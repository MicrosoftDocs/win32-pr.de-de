<dl> <dt>

<span id="_msi_.idt_file_gly"></span><span id="_MSI_.IDT_FILE_GLY"></span>**IDT-Datei**
</dt> <dd>

Datenbanktabelle des exportierten Installers. Weitere Informationen finden Sie unter [importieren und exportieren](importing-and-exporting.md) und [Windows Installer Dateierweiterungen](windows-installer-file-extensions.md).

</dd> <dt>

<span id="_msi_import_address_table_gly"></span><span id="_MSI_IMPORT_ADDRESS_TABLE_GLY"></span>**Importieren der Adress Tabelle (IAT)**
</dt> <dd>

Dabei wird die berechnete virtuelle Adresse aus jeder aus allen DLLs importierten Funktion gespeichert.

</dd> <dt>

<span id="_msi_internal_user_interface_gly"></span><span id="_MSI_INTERNAL_USER_INTERFACE_GLY"></span>**interne Benutzeroberfläche**
</dt> <dd>

Integrierte Funktionen des Installers, mit denen eine grafische Benutzeroberfläche für die Installation erstellt werden kann. Ein Autor kann eine [*externe Benutzeroberfläche*](e-gly.md)auswählen. Weitere Informationen finden Sie unter Informationen zur [Benutzeroberfläche](about-the-user-interface.md).

</dd> <dt>

<span id="_msi_install_level_gly"></span><span id="_MSI_INSTALL_LEVEL_GLY"></span>**Installationsstufe**
</dt> <dd>

Der angegebene Installations Wert. Eine Anwendung wird nur installiert, wenn die eigene Ebene kleiner oder gleich der Installations Ebene ist. Der Satz von Anwendungen, der für die Installation ausgewählt wurde, kann daher durch Ändern der Installations Ebene geändert werden. Weitere Informationen finden Sie unter [Funktions Tabelle](feature-table.md).

</dd> <dt>

<span id="_msi_install_on_demand_gly"></span><span id="_MSI_INSTALL_ON_DEMAND_GLY"></span>**Installation bei Bedarf**
</dt> <dd>

Dienst, mit dem Anwendungen oder Features installiert werden, die vom Benutzer oder von einer anderen Anwendung angefordert werden. Durch die [*Werbung*](a-gly.md) wird ein Feature oder eine Anwendung für die Installation bei Bedarf bereitgestellt. Weitere Informationen finden Sie unter: [Installation nach Bedarf](installation-on-demand.md).

</dd> <dt>

<span id="_msi_installation_context_gly"></span><span id="_MSI_INSTALLATION_CONTEXT_GLY"></span>**Installations Kontext**
</dt> <dd>

Die Kombination von Installations rechten und Installationstypen erzeugt drei verschiedene Installations Kontexte: Benutzer, die nicht verwaltet werden, pro Benutzer verwaltet und pro Computer verwaltet werden. Es gibt keinen Computer, der nicht verwaltet wird.

</dd> <dt>

<span id="_msi_installer_gly"></span><span id="_MSI_INSTALLER_GLY"></span>**ations**
</dt> <dd>

Der [*Windows Installer*](m-gly.md).

</dd> <dt>

<span id="_msi_installer_database_gly"></span><span id="_MSI_INSTALLER_DATABASE_GLY"></span>**Installer-Datenbank**
</dt> <dd>

Eine relationale Datenbank, die Setup Anweisungen für eine-Installation enthält. Die Installer-Datenbank wird in der [*MSI-Datei*](m-gly.md)gespeichert. Weitere Informationen finden Sie unter [Installer Database](installer-database.md).

</dd> <dt>

<span id="_msi_installer_function_gly"></span><span id="_MSI_INSTALLER_FUNCTION_GLY"></span>**Installer-Funktion**
</dt> <dd>

Wird von einem Benutzer oder einer Anwendung aufgerufen, um Installer-Dienste und-Funktionen zu erhalten. Dies ist die Anwendungsprogrammierschnittstelle des Installers. Weitere Informationen finden Sie unter [installationsfunktionsreferenz](installer-function-reference.md).

</dd> <dt>

<span id="_msi_installer_package_authoring_tool_gly"></span><span id="_MSI_INSTALLER_PACKAGE_AUTHORING_TOOL_GLY"></span>**Installer-Paket Erstellungs Tool**
</dt> <dd>

Software, die es Entwicklern ermöglicht, eine MSI- [*Datei*](m-gly.md)zu erstellen und zu bearbeiten. Dies umfasst zusammen mit [*externen Quelldateien*](e-gly.md) ein [*Paket*](p-gly.md) , das alle für eine Installation erforderlichen Informationen enthält.

</dd> <dt>

<span id="_msi_internal_source_files_gly"></span><span id="_MSI_INTERNAL_SOURCE_FILES_GLY"></span>**interne Quelldateien**
</dt> <dd>

In der [*MSI-Datei*](m-gly.md)gespeichert. Mehrere interne Quelldateien können komprimiert und zusammen in einer CAB- [*Datei*](c-gly.md) gespeichert werden, die in einer MSI-Datei gespeichert ist.

</dd> </dl>
