</dd> <dt>

<span id="_msi_import_address_table_gly"></span><span id="_MSI_IMPORT_ADDRESS_TABLE_GLY"></span>**Importieren der Adresstabelle (IAT)**

Hier wird die berechnete virtuelle Adresse aus jeder Funktion gespeichert, die aus allen DLLs importiert wurde.

</dd> <dt>

<span id="_msi_internal_user_interface_gly"></span><span id="_MSI_INTERNAL_USER_INTERFACE_GLY"></span>**Interne Benutzeroberfläche**
</dt> <dd>

Integrierte Funktionen des Installationsprogramms, die zum Erstellen einer grafischen Benutzeroberfläche für die Installation verwendet werden können. Ein Autor kann eine externe [*Benutzeroberfläche auswählen.*](e-gly.md) Weitere Informationen finden Sie unter [Informationen zum Benutzeroberfläche](about-the-user-interface.md).

</dd> <dt>

<span id="_msi_install_level_gly"></span><span id="_MSI_INSTALL_LEVEL_GLY"></span>**Installationsebene**
</dt> <dd>

Angegebener Installationswert. Eine Anwendung wird nur installiert, wenn ihre eigene Ebene kleiner oder gleich der Installationsebene ist. Der für die Installation ausgewählte Satz von Anwendungen kann daher durch Ändern der Installationsebene geändert werden. Weitere Informationen finden Sie unter [FeatureTabelle](feature-table.md).

</dd> <dt>

<span id="_msi_install_on_demand_gly"></span><span id="_MSI_INSTALL_ON_DEMAND_GLY"></span>**Installation bei Bedarf**
</dt> <dd>

Dienst, der Anwendungen oder Features wie vom Benutzer oder einer anderen Anwendung angefordert installiert. [*Die*](a-gly.md) Werbung macht ein Feature oder eine Anwendung für die Installation bei Bedarf verfügbar. Weitere Informationen finden Sie unter Installation [on Demand](installation-on-demand.md).

</dd> <dt>

<span id="_msi_installation_context_gly"></span><span id="_MSI_INSTALLATION_CONTEXT_GLY"></span>**Installationskontext**
</dt> <dd>

Die Kombination aus Installationsrechten und Installationstypen erzeugt drei verschiedene Installationskontexte: nicht verwaltet pro Benutzer, benutzer- und computerver verwaltet. Es gibt keine Computer, die nicht verwaltet werden.

</dd> <dt>

<span id="_msi_installer_gly"></span><span id="_MSI_INSTALLER_GLY"></span>**Installer**
</dt> <dd>

Der [*Windows Installer*](m-gly.md).

</dd> <dt>

<span id="_msi_installer_database_gly"></span><span id="_MSI_INSTALLER_DATABASE_GLY"></span>**Installer-Datenbank**
</dt> <dd>

Relationale Datenbank mit Setupanweisungen für eine Installation. Die Installer-Datenbank wird in der.msi [*gespeichert.*](m-gly.md) Weitere Informationen finden Sie unter [Installer Database](installer-database.md).

</dd> <dt>

<span id="_msi_installer_function_gly"></span><span id="_MSI_INSTALLER_FUNCTION_GLY"></span>**Installer-Funktion**
</dt> <dd>

Wird von einem Benutzer oder einer Anwendung aufgerufen, um Installationsdienste und -funktionen zu erhalten. Dies ist die Anwendungsprogrammierschnittstelle des Installationsprogramms. Weitere Informationen finden Sie unter [Installer Function Reference (Referenz zur Installerfunktion).](installer-function-reference.md)

</dd> <dt>

<span id="_msi_installer_package_authoring_tool_gly"></span><span id="_MSI_INSTALLER_PACKAGE_AUTHORING_TOOL_GLY"></span>**Installer-Paketerstellungstool**
</dt> <dd>

Software, die einem Entwickler das Erstellen und Bearbeiten einer.msi [*ermöglicht.*](m-gly.md) Dies umfasst zusammen [*mit allen externen Quelldateien*](e-gly.md) ein [*Paket,*](p-gly.md) das alle informationen enthält, die für eine Installation erforderlich sind.

</dd> <dt>

<span id="_msi_internal_source_files_gly"></span><span id="_MSI_INTERNAL_SOURCE_FILES_GLY"></span>**Interne Quelldateien**
</dt> <dd>

Wird in der.msi [*gespeichert.*](m-gly.md) Mehrere interne Quelldateien können komprimiert und zusammen in einer [*Schränkdatei*](c-gly.md) gespeichert werden, die in einer .msi ist.

</dd> </dl>
