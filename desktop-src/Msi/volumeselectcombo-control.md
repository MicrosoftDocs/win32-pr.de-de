# <a name="volumeselectcombo-control"></a>VolumeSelectCombo-Steuerelement

Mit dem VolumeSelectCombo-Steuerelement kann der Benutzer ein Volume aus einer alphabetischen Liste von Volumes auswählen. Die in der Liste angezeigten Volumetypen werden mithilfe von Bits angegeben, die den Steuerelementattributen [RemovableVolume,](removablevolume-control-attribute.md) [FixedVolume,](fixedvolume-control-attribute.md) [RemoteVolume,](remotevolume-control-attribute.md) [CREDOVolume,](cdromvolume-control-attribute.md) [RAMDiskVolume](ramdiskvolume-control-attribute.md)und [FloppyVolume](floppyvolume-control-attribute.md) zugeordnet sind.

Sie können dieses Steuerelement einer Eigenschaft zuordnen, indem Sie den Namen der Eigenschaft in der Property -Spalte der [Control-Tabelle](control-table.md)eingeben.

## <a name="remarks"></a>Hinweise

Dieses Steuerelement kann mithilfe der [**CreateWindowEx-Funktion**](/windows/win32/api/winuser/nf-winuser-createwindowexa) aus der COMBOBOX-Klasse erstellt werden. Sie verfügt über die Formate **CBS \_ DROPDOWNLIST,** **CBS \_ OWNERDRAWFIXED,** **CBS \_ HASSTRINGS,** **WS \_ VSCROLL,** **WS \_ CHILD,** **WS \_ GROUP,** **WS \_ TABSTOP** und **CBS \_ SORT.** Informationen zum Entwickeln einer Benutzeroberfläche mit Windows finden Sie unter [Benutzeroberfläche Entwurf und Entwicklung.](/previous-versions/aa286531(v=msdn.10))

Aus Gründen der Kompatibilität mit Sprachausgaben müssen Sie beim Erstellen eines Dialogfelds mit einem VolumeSelectCombo-Steuerelement als erstes aktives Steuerelement das Textfeld, das zum Bearbeitungsfeld gehört, zum ersten aktiven Steuerelement in der [Dialogtabelle](dialog-table.md)machen. Da der statische Text nicht den Fokus erhalten kann, erhält das Bearbeitungsfeld beim Erstellen des Dialogfelds zunächst den gewünschten Fokus. Dadurch wird sichergestellt, dass Sprachausgaben die richtigen Informationen anzeigen.
