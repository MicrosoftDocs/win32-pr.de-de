# <a name="ipv6-connectivity-and-teredo"></a>IPv6-Konnektivität und Teredo

Die meisten Netzwerke sind IPv4-und End-to-End-IPv6-Konnektivität können nicht problemlos erreicht werden. Darüber hinaus beschränken die meisten NAT-Geräte auch die End-to-End-IPv4-Konnektivität. Mit Windows Vista ist eine End-to-End-Konnektivität zwischen zwei Hosts möglich. Entwickler müssen sich nicht mehr um die End-to-End-Konnektivität kümmern, sondern vielmehr neue Anwendungsszenarien erstellen, die es Ihnen ermöglichen, diese Konnektivität zu nutzen.

Das Ziel von Teredo, als IPv6-Übergangstechnologie, besteht darin, die End-to-End-IPv6-Konnektivität einfach und automatisch zu gestalten. In vielen Fällen müssen Entwickler möglicherweise nicht einmal wissen, dass Übergangs Technologien wie Teredo in Windows Vista vorhanden sind. Windows Vista bietet die Möglichkeit, Teredo im Hintergrund als nahtlose Methode der IPv6-Konnektivität zu verwenden.
