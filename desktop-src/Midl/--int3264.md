# <a name="__int3264-attribute"></a>\_\_int3264-Attribut

Das Schlüsselwort **\_ \_ int3264** gibt einen integralen Typ an, der über die folgenden Eigenschaften verfügt:

-   Es ist 32-Bit auf 32-Bit-Plattformen
-   Es ist 64-Bit auf 64-Bit-Plattformen
-   Bei der Netzwerkverbindung handelt es sich um einen 32-Bit-Wert für die Abwärtskompatibilität Sie wird auf der sendenden Seite abgeschnitten und entsprechend (signiert oder unsigniert) auf der Empfangsseite erweitert.

``` syntax
[ signed | unsigned ] __int3264 [ int ] declarator-list;
```

## <a name="parameters"></a>Parameter

<dl> <dt>

*Deklarator-List* 
</dt> <dd>

Gibt einen oder mehrere Standard-C-Deklaratoren an, z. b. Bezeichner, zeigerdeklaratoren und Array Deklaratoren. (Funktions Deklaratoren und Bitfeld Deklarationen sind in Strukturen, die in Remote Prozedur aufrufen übertragen werden, nicht zulässig. Diese Deklaratoren sind in Strukturen zulässig, die nicht übertragen werden.) Trennen Sie mehrere Deklaratoren durch Kommas.

</dd> </dl>

## <a name="remarks"></a>Bemerkungen

Dieser Typ wurde speziell eingeführt, um das Portieren von 32-Bit-Schnittstellen auf 64 Bit zu vereinfachen. Es ist in der Regel nur in Fällen nützlich, in denen vorhandene Schnittstellen Zeiger als ganze Zahlen senden oder ähnliche Problem Umgehung Verfahren verwenden. Microsoft empfiehlt nicht, diesen Typ (oder andere von ihm abgeleitete öffentliche Typen wie **int \_ ptr**, **DWORD \_ ptr**) zu verwenden, wo dies vermieden werden kann. Insbesondere können diese Typen nicht in OLE-Automatisierungs Schnittstellen und Typbibliotheken verwendet werden. Die Verwendung dieses Typs oder der davon abgeleiteten Typen ist in neuen Schnittstellen nicht empfehlenswert. Weitere Informationen zu diesen Problemen sowie Kommentare und Ratschläge zu anderen 64-Bit-Problemen finden Sie im Microsoft Interface Definition Language (Mittel l): 64-Bit-Portierungs Handbuch.

Dem int3264-Schlüsselwort \_ \_  kann entweder das Schlüsselwort signiert oder das Schlüsselwort unsigniert vorangestellt werden. Das int-Schlüsselwort ist optional und kann ausgelassen werden. Für den Mittelwert Compiler \_ \_ ist die ganzzahlige **int3264** standardmäßig signiert und mit signed \_ \_ int3264 int Synonym.

Der **\_ \_ int3264** Integer-Typ ist einer der Basis Typen der IDL-Sprache. Der ganzzahlige Typ **\_ \_ int3264** kann als Typspezifizierer in konstanten Deklarationen, typedef-Deklarationen, allgemeinen Deklarationen und Funktions Deklaratoren (als Funktionsrückgabetyp-Spezifizierer und Parametertyp Spezifizierer) angezeigt werden. Informationen zu dem Kontext, in dem typspezifier angezeigt wird, finden Sie unter IDL.
