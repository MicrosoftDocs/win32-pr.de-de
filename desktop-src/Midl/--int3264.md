# <a name="__int3264-attribute"></a>\_\_int3264-Attribut

Das Schlüsselwort **\_ \_ int3264** gibt einen ganzzahligen Typ mit den folgenden Eigenschaften an:

-   Es handelt sich um eine 32-Bit-Plattform auf 32-Bit-Plattformen.
-   Dies ist 64-Bit auf 64-Bit-Plattformen.
-   Es ist aus Gründen der Abwärtskompatibilität 32-Bit-on-the-Wire. Er wird auf der sendenden Seite abgeschnitten und auf der empfangenden Seite entsprechend erweitert (signiert oder nicht signiert).

``` syntax
[ signed | unsigned ] __int3264 [ int ] declarator-list;
```

## <a name="parameters"></a>Parameter

<dl> <dt>

*deklarator-list* 
</dt> <dd>

Gibt einen oder mehrere C-Standarddeklaratoren an, z. B. Bezeichner, Zeigerdeklaratoren und Arraydeklaratoren. (Funktionsdeklaratoren und Bitfelddeklarationen sind in Strukturen, die in Remoteprozeduraufrufen übertragen werden, nicht zulässig. Diese Deklaratoren sind in Strukturen zulässig, die nicht übertragen werden.) Trennen Sie mehrere Deklaratoren durch Kommas.

</dd> </dl>

## <a name="remarks"></a>Hinweise

Dieser Typ wurde speziell eingeführt, um das Portieren von 32-Bit-Schnittstellen auf 64-Bit zu vereinfachen. Dies ist im Allgemeinen nur in Fällen nützlich, in denen vorhandene Schnittstellen Zeiger als ganze Zahlen senden oder ähnliche Problemumgehungstechniken verwenden. Microsoft empfiehlt nicht, diesen Typ (oder andere davon abgeleitete öffentliche Typen wie **INT \_ PTR**, **DWORD \_ PTR**) zu verwenden, wo er vermieden werden kann. Insbesondere können diese Typen nicht in OLE-Automatisierungsschnittstellen und Typbibliotheken verwendet werden. Die Verwendung dieses Typs oder von ihm abgeleiteter Typen in neuen Schnittstellen wird nicht empfohlen. Weitere Informationen zu diesen Problemen sowie Kommentare und Hinweise zu anderen 64-Bit-Problemen finden Sie im 64-Bit-Portierungshandbuch für Microsoft Interface Definition Language (MIDL): 64-Bit.

Dem \_ \_ **Schlüsselwort int3264** kann entweder das Schlüsselwort signed oder das Schlüsselwort unsigned vorangestellt werden. Das Schlüsselwort int ist optional und kann ausgelassen werden. Für den MIDL-Compiler ist die ganze Zahl \_ \_ **int3264** standardmäßig signiert und synonym mit signed \_ \_ int3264 int.

Der ganzzahlige Typ **\_ \_ int3264** ist einer der Basistypen der IDL-Sprache. Der ganzzahlige Typ **\_ \_ int3264** kann als Typspezifizierer in const-Deklarationen, TypeDef-Deklarationen, allgemeinen Deklarationen und Funktionsdeklaratoren (als Funktionsrückgabetypspezifizierer und Parametertypspezifizierer) angezeigt werden. Informationen zum Kontext, in dem Typspezifizierer angezeigt werden, finden Sie unter IDL.
