# <a name="g-opengl"></a>G (OpenGL)

<dl> <dt>

<span id="opengl_gamma_correction"></span><span id="OPENGL_GAMMA_CORRECTION"></span>**Gammakorrektur**
</dt> <dd>

Eine Funktion, die auf Farben angewendet wird, die im Frame Puffer gespeichert werden, um die nichtlineare Reaktion des Auges (und manchmal des Monitors) auf lineare Änderungen in Farb Intensitäts Werten zu korrigieren.

</dd> <dt>

<span id="opengl_geometric_model"></span><span id="OPENGL_GEOMETRIC_MODEL"></span>**geometrisches Modell**
</dt> <dd>

Die Objekt Koordinaten Vertices und Parameter, die ein Objekt beschreiben. Beachten Sie, dass OpenGL keine Syntax für geometrische Modelle definiert, sondern vielmehr eine Syntax und Semantik für das Rendering von geometrischen Modellen.

</dd> <dt>

<span id="opengl_geometric_object"></span><span id="OPENGL_GEOMETRIC_OBJECT"></span>**geometrisches Objekt**
</dt> <dd>

Ein geometrisches Modell.

</dd> <dt>

<span id="opengl_geometric_primitive"></span><span id="OPENGL_GEOMETRIC_PRIMITIVE"></span>**geometrisch primitiv**
</dt> <dd>

Ein Punkt, eine Linie oder ein Polygon.

</dd> <dt>

<span id="opengl_gouraud_shading"></span><span id="OPENGL_GOURAUD_SHADING"></span>**Gouraud-Schattierung**
</dt> <dd>

Glatte interpolung von Farben in einem Polygon-oder Liniensegment. Farben werden bei Scheitel Punkten und linear interpoliert, um eine relativ glatte Variation der Farbe zu schaffen. Wird auch als Smooth Schattierung bezeichnet.

</dd> <dt>

<span id="opengl_group"></span><span id="OPENGL_GROUP"></span>**Kreis**
</dt> <dd>

Eine Gruppe von einem, zwei, drei oder vier Elementen, die jedes Pixel eines Bilds im Client Speicher darstellt. Daher sind eine Gruppe und ein Pixel im Kontext eines Client Speicher Abbilds identisch.

</dd> </dl>
