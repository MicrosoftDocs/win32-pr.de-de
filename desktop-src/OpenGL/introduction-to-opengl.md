# <a name="introduction-to-opengl"></a>Einführung in OpenGL

Als eine Softwareschnittstelle für Grafikhardware besteht der Hauptzweck von OpenGL darin, zwei-und dreidimensionale Objekte in einen Framebuffer zu Renderern. Diese Objekte werden als Sequenzen von Vertices (die geometrische Objekte definieren) oder Pixel (die Bilder definieren) beschrieben. OpenGL führt mehrere Prozesse für diese Daten aus, um Sie in Pixel zu konvertieren, um das endgültige gewünschte Bild im Framebuffer zu bilden.

Die folgenden Themen bieten eine globale Ansicht der Funktionsweise von OpenGL:

-   [Primitive und Befehle](primitives-and-commands.md) erläutert Punkte, Liniensegmente und Polygone als grundlegende Zeichnungs Einheiten. und die Verarbeitung von Befehlen.
-   Das [OpenGL-Grafik Steuer](opengl-graphic-control.md) Element beschreibt, welche Grafik Vorgänge OpenGL-Steuerelemente und welche nicht steuert.
-   Das [Ausführungs Modell](execution-model.md) erläutert das Client/Server-Modell zum Interpretieren von OpenGL-Befehlen.
-   Der [grundlegende OpenGL-Vorgang](basic-opengl-operation.md) bietet eine allgemeine Beschreibung, wie OpenGL Daten verarbeitet, um ein entsprechendes Bild im Framebuffer zu liefern.
-   [OpenGL-Funktionsnamen](opengl-function-names.md) beschreiben die Benennungs Konventionen, die in OpenGL verwendet werden.
