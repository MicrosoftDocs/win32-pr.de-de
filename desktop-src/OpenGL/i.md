# <a name="i-opengl"></a>I (OpenGL)

<dl> <dt>

<span id="opengl_image"></span><span id="OPENGL_IMAGE"></span>**Bild**
</dt> <dd>

Ein rechteckiges Array von Pixeln, entweder im Clientspeicher oder im Framepuffer.

</dd> <dt>

<span id="opengl_image_primitive"></span><span id="OPENGL_IMAGE_PRIMITIVE"></span>**Bildprimitive** 
</dt> <dd>

Eine Bitmap oder ein Bild.

</dd> <dt>

<span id="opengl_immediate_mode"></span><span id="OPENGL_IMMEDIATE_MODE"></span>**Direktmodus**
</dt> <dd>

Modus, in dem ein OpenGL-Befehl direkt und nicht aus einer Anzeigeliste aufgerufen wird. Es ist kein Bit im unmittelbaren Modus vorhanden. der Modus im unmittelbaren Modus bezieht sich auf die Verwendung von OpenGL und nicht auf ein bestimmtes Bit des OpenGL-Zustands.

</dd> <dt>

<span id="opengl_index"></span><span id="OPENGL_INDEX"></span>**Index**
</dt> <dd>

Ein einzelner Wert, der als absoluter Wert und nicht als normalisierter Wert in einem angegebenen Bereich interpretiert wird (wie eine Komponente ist). Farbindizes sind die Namen von Farben, die von der Anzeigehardware mithilfe der Farbzuordnung dereferenziert werden. Indizes werden in der Regel maskiert und nicht klammern, wenn sie außerhalb des Bereichs liegen. Beispielsweise wird der Index 0xf7 maskiert, um 0x7, wenn er in einen 4-Bit-Puffer (Farbe oder Schablone) geschrieben wird. Farbindizes und Schablonenindizes werden immer als Indizes und nie als Komponenten behandelt.

</dd> <dt>

<span id="opengl_iris_gl"></span><span id="OPENGL_IRIS_GL"></span>**IRIS GL**
</dt> <dd>

Die proprietäre Grafikbibliothek von Silicon Graphics, die von 1982 bis 1992 entwickelt wurde. OpenGL wurde mit IRIS GL als Ausgangspunkt entworfen.

</dd> </dl>
