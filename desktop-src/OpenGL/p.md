<dl> <dt>

<span id="opengl_parameter"></span><span id="OPENGL_PARAMETER"></span>**Parame**
</dt> <dd>

Ein Wert, der als Argument an einen OpenGL-Befehl übermittelt wird. Manchmal einer der Werte, die als Verweis an einen OpenGL-Befehl übermittelt werden.

</dd> <dt>

<span id="opengl_perspective_division"></span><span id="OPENGL_PERSPECTIVE_DIVISION"></span>**Perspektiven Division**
</dt> <dd>

Die Division von x, y und z durch w, die in Clip Koordinaten ausgeführt wird. Siehe auch [Clip-Koordinaten](c.md).

</dd> <dt>

<span id="opengl_pixel"></span><span id="OPENGL_PIXEL"></span>**Megapixel**
</dt> <dd>

Bildelement. Die Bits an der Position (x, y) aller bitflächen im Frame Puffer bilden das einzelne Pixel (x, y). In einem Bild im Client Speicher ist ein Pixel eine Gruppe von Elementen. In den OpenGL-Fenster Koordinaten entspricht jedes Pixel einem Bildschirmbereich von 1.0 x 1.0. Die Koordinaten der unteren linken Ecke der Pixel Namen x, y sind (x, y) und die obere rechte Ecke (x + 1, y + 1).

</dd> <dt>

<span id="opengl_point"></span><span id="OPENGL_POINT"></span>**Punkt**
</dt> <dd>

Eine genaue Position im Raum, die als Punkt mit Endpunkten gerendert wird.

</dd> <dt>

<span id="opengl_polygon"></span><span id="OPENGL_POLYGON"></span>**Polygon**
</dt> <dd>

Eine near-planare Oberfläche, die durch von Scheitel Punkten angegebene Kanten begrenzt ist. Jedes Dreieck eines Dreiecks Netzes ist ein Polygon, ebenso wie jedes Viereck eines vierseitigen Mesh. Das von glrect angegebene Rechteck ist ebenfalls ein Polygon.

</dd> <dt>

<span id="opengl_primitive"></span><span id="OPENGL_PRIMITIVE"></span>**Christentum**
</dt> <dd>

Eine Form (z. b. ein Punkt, eine Linie, ein Polygon, eine Bitmap oder ein Bild), die als diskrete Entität gezeichnet, gespeichert und manipuliert werden kann. Elemente, aus denen große Grafik Entwürfe erstellt werden.

</dd> <dt>

<span id="opengl_projection_matrix"></span><span id="OPENGL_PROJECTION_MATRIX"></span>**Projektions Matrix**
</dt> <dd>

Die 4X4-Matrix, die Punkte, Linien, Polygone und Raster Positionen von Augen Koordinaten zu Clip Koordinaten transformiert.

</dd> </dl>
