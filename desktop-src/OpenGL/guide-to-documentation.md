# <a name="guide-to-documentation"></a>Leitfaden zur Dokumentation

Der Dokumentations Satz für OpenGL in Windows umfasst fünf Elemente.

-   Das *OpenGL-Referenzhandbuch* enthält eine Übersicht über die Funktionsweise von OpenGL und einen Satz detaillierter Referenzseiten. Die Referenzseiten decken alle 115-eindeutigen OpenGL-Funktionen sowie die 43-Funktionen in der-Bibliothek des OpenGL-Hilfsprogramms (GLU) ab.

-   Im *OpenGL-Programmier Handbuch* wird erläutert, wie Grafikprogramme mithilfe von OpenGL erstellt werden. Es umfasst Erörterung der folgenden wichtigen Themen:

    -   Zeichnen geometrischer Formen
    -   Pixel, Bitmaps, Schriftarten und Bilder
    -   Anzeigen und Matrix Transformationen
    -   Textur Zuordnung
    -   Anzeigen von Listen
    -   Erweiterte zusammengesetzte Techniken
    -   Color
    -   Evaluatoren und nursb
    -   Beleuchtung
    -   Auswahl und Feedback
    -   Blending, Antialiasing und Nebel
    -   Erweiterte Techniken

    Außerdem enthält das *OpenGL-Programmier Handbuch* Anhänge, die die OpenGL-Hilfsprogrammbibliothek und die Hilfsbibliothek des OpenGL-Programmier Handbuchs erörtern.

-   Das dritte Dokumentations Element ist diese Übersicht. Es beschreibt die Windows-Implementierung von OpenGL und bietet einen Überblick über die zugehörigen Komponenten. Es erläutert die Konzepte von renderingkontexten, Pixel Formaten und Puffern. die WGL-Funktionen, die OpenGL mit den Windows-windowingsystemen verbinden und die Windows-Funktionen, die Pixel Formate pro Fenster und die doppelte Pufferung von Fenstern für OpenGL-Grafikfenster unterstützen. Die WGL-Funktionen und die Funktionen sind spezifisch für die Windows-Implementierung von OpenGL.

-   Das vierte Dokumentations Element ist der Satz von Referenzseiten für die WGL-Funktionen, die soeben erwähnten Windows-Funktionen und die [**Pixel formatdescriptor**](/windows/win32/api/wingdi/ns-wingdi-pixelformatdescriptor) -Datenstruktur.

-   Das fünfte Dokumentations Element ist das Handbuch zum Portieren. Es wird erläutert, wie vorhandener OpenGL-Code aus anderen Umgebungen in Windows verschoben wird.

Die ersten beiden Elemente sind die offiziellen OpenGL-Bücher. Sie sind nicht in der Microsoft Windows SDK enthalten.
