<dl> <dt>

<span id="opengl_aliasing"></span><span id="OPENGL_ALIASING"></span>**Aliasing**
</dt> <dd>

Eine renderingtechnik, die Pixel die Farbe des primitiven, das gerendert wird, zuweist, ob diese primitive den gesamten oder einen Teil des Pixel Bereichs abdeckt. Aliasing führt zu verzweigten Kanten oder Verzweigungen. Siehe auch [](jk.md)Verzweigungen.

</dd> <dt>

<span id="opengl_alpha"></span><span id="OPENGL_ALPHA"></span>**Alpha**
</dt> <dd>

Eine vierte Farbkomponente, die normalerweise verwendet wird, um Farbmischung zu steuern Die Alpha Komponente wird nie direkt angezeigt. Gemäß der Konvention gibt OpenGL Alpha die Deckkraft und Transparenz bei einer Skala von 0,0 bis 1,0 an. Der Alpha-Wert 1,0 impliziert eine komplette Deckkraft. der Alpha Wert 0,0 impliziert eine umfassende Transparenz.

</dd> <dt>

<span id="opengl_animation"></span><span id="OPENGL_ANIMATION"></span>**Animationen**
</dt> <dd>

Die Generierung von wiederholten Renderings einer Szene ist schnell genug, mit reibungslos veränderlichen Standpunkten oder Objektpositionen, dass die Illusion von Bewegung erreicht wird. Die OpenGL-Animation verwendet fast immer doppelte Pufferung.

</dd> <dt>

<span id="opengl_antialiasing"></span><span id="OPENGL_ANTIALIASING"></span>**Antialiasing**
</dt> <dd>

Eine renderingtechnik, die Pixel basierend auf dem Bruchteil des Pixel Bereichs, der durch das primitive, das gerendert wird, abgedeckt wird. Das Rendern mit Antialiasing reduziert oder entfernt die Verzweigungen, die sich aus dem Rendern von Alias ergeben. Siehe auch [](jk.md)Verzweigungen, [Rendering](r.md).

</dd> <dt>

<span id="opengl_application_specific_clipping"></span><span id="OPENGL_APPLICATION_SPECIFIC_CLIPPING"></span>**anwendungsspezifisches Clipping** 
</dt> <dd>

Clipping von primitiven für Flächen in Augen Koordinaten. Die Ebenen werden von der Anwendung mithilfe von [**glclipplane**](glclipplane.md)angegeben. Siehe auch [Eye-Koordinaten](e.md).

</dd> </dl>
