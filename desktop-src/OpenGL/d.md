# <a name="d-opengl"></a>D (OpenGL)

<dl> <dt>

<span id="opengl_depth"></span><span id="OPENGL_DEPTH"></span>**Eingeh**
</dt> <dd>

Bezieht sich im Allgemeinen auf die z-Fenster Koordinate.

</dd> <dt>

<span id="opengl_depth_cueing"></span><span id="OPENGL_DEPTH_CUEING"></span>**tiefen Cueing**
</dt> <dd>

Eine renderingtechnik, die eine Farbe basierend auf der Entfernung vom Standpunkt zuweist.

</dd> <dt>

<span id="opengl_display_list"></span><span id="OPENGL_DISPLAY_LIST"></span>**Liste anzeigen**
</dt> <dd>

Eine benannte Liste von OpenGL-Befehlen. Anzeigelisten werden immer auf dem Server gespeichert, sodass mit Anzeigelisten der Netzwerk Datenverkehr in Client-/Server-Umgebungen reduziert werden kann. Der Inhalt einer Anzeigeliste ist möglicherweise vorverarbeitet und wird daher möglicherweise effizienter ausgeführt als derselbe Satz von OpenGL-Befehlen, die im unmittelbaren Modus ausgeführt werden. Eine solche Vorverarbeitung ist besonders wichtig für die Berechnung intensiver Befehle wie z. b. [**glteximage**](glteximage1d.md).

</dd> <dt>

<span id="opengl_dithering"></span><span id="OPENGL_DITHERING"></span>**Dithering**
</dt> <dd>

Eine Technik, mit der der wahrgenommene Bereich von Farben in einem Bild auf Kosten der räumlichen Auflösung erhöht wird. Angrenzenden Pixeln werden unterschiedliche Farbwerte zugewiesen. Wenn Sie von einer Entfernung aus angezeigt werden, scheinen diese Farben in eine einzelne zwischen Farbe zu mischen. Die Vorgehensweise ähnelt der Hälfte der Verwendung in schwarz-und weißen Veröffentlichungen, um Grauschattierungen zu erzielen.

</dd> <dt>

<span id="opengl_double_buffering"></span><span id="OPENGL_DOUBLE_BUFFERING"></span>**doppelte Pufferung**
</dt> <dd>

Die Verwendung von OpenGL-Kontexten, in denen die Vorder-und Hintergrund Farb Puffer doppelt gepuffert werden. Die Smooth Animation wird durch Rendering in nur den zurück-Puffer (der nicht angezeigt wird) ausgeführt und bewirkt dann, dass die Vorder-und Hintergrund Puffer ausgetauscht werden.

</dd> </dl>
