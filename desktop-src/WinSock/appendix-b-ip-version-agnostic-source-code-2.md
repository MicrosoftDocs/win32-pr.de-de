# <a name="appendix-b-ip-version-agnostic-source-code"></a>Anhang B: Ip-Version Agnostic Source Code

Dieser Anhang veranschaulicht eine umgeschriebene Version der Simplec.c- und Simples.c-Beispielanwendungen, die IPv4 oder IPv6 ordnungsgemäß verarbeitet.

-   [IPv6-fähiger Clientcode](ipv6-enabled-client-code-2.md)
-   [IPv6-fähiger Servercode](ipv6-enabled-server-code-2.md)

Dieser Code veranschaulicht die in diesem IPv6-Handbuch festgelegten Richtlinien und ist enthalten, um Quellcode zur Verfügung zu stellen, der erfolgreich geändert wurde, um Unterstützung für IPv6 hinzuzufügen. Dieses Beispiel ist absichtlich einfach, bietet jedoch ein praktisches Beispiel für Dasbing und Überprüfung. Eine einzige IPv4-Version dieses Quellcodes finden Sie in [Anhang A: Nur IPv4-Quellcode.](appendix-a-ipv4-only-source-code-2.md)

Wenn Sie den Quellcode in Anhang A (nur IPv4) und Anhang B (IP-Version agnostisch) vergleichen, erhalten Sie ein Gefühl für die Änderungen, die erforderlich sind, um Ihre Windows Sockets-Anwendung so zu ändern, dass Unterstützung für IPv6 hinzugefügt wird.
