# <a name="transactional-versus-streaming-applications"></a>Transaktions- im Vergleich zu Streaminganwendungen

Es gibt zwei grundlegende Arten von Netzwerkanwendungen: *Transaktions- und* *Streaminganwendungen.* Diese Anwendungstypen werden auch als *interaktive Anwendungstypen* *bzw.* Batchverarbeitungsanwendungstypen bezeichnet.

Transaktionsanwendungen sind Stop-and-Go-Anwendungen. Sie führen in der Regel Anforderungs-/Antwortvorgänge aus, die häufig geordnet sind. Beispiele für Transaktionsanwendungen sind ein synchroner Remoteprozeduraufruf (RPC) sowie einige IMPLEMENTIERUNGen von HTTP und Domain Name System (DNS).

Streaminganwendungen verschieben Daten. Um Streaminganwendungen mit einem parallelen Begriff zu beschreiben, halten sich Streaminganwendungen an eine Datenübertragungsgedanke vom Datenübertragungs- und Datenübertragungsspektrum, in der Regel mit wenig Bedenken bei der Datenbestellung. Beispiele für Streaminganwendungen sind die Netzwerksicherung und das Dateiübertragungsprotokoll (FTP).

Nachdem der Anwendungstyp bestimmt wurde, werden auch seine Netzwerk- und Protokollmerkmale bestimmt.
