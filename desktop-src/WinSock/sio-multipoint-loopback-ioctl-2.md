# <a name="sio_multipoint_loopback-ioctl"></a>SIO \_ Multipoint \_ Loopback ioctl

Wenn d- \_ Blatt Sockets in einer Datenebene verwendet werden, die nicht in der Stammdatenbank verwendet wird, ist es im allgemeinen wünschenswert, zu steuern, ob der gesendete Datenverkehr auch im selben Socket empfangen wird. Der "sio \_ Multipoint \_ Loopback Command Code" für [**wspioctl**](/previous-versions/windows/hardware/network/ff566296(v=vs.85)) wird zum Aktivieren oder Deaktivieren der Loopback Funktion von Multipoint-Datenverkehr verwendet.
