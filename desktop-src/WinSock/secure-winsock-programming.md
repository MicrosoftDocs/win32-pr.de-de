# <a name="secure-winsock-programming"></a>Sichere Winsock-Programmierung

Es folgt eine Anleitung zum Sichern Windows Sockets-Programmierung. Sie ist so konzipiert, dass sie ein Verständnis der Winsock-Sicherheit und der Optionen bietet, die entwicklern für sichere Netzwerkanwendungen zur Verfügung stehen.

-   [Verwenden von SO \_ REUSEADDR und SO \_ EXCLUSIVEADDRUSE](using-so-reuseaddr-and-so-exclusiveaddruse.md)
-   [Winsock Secure Socket-Erweiterungen](winsock-secure-socket-extensions.md)

Die Kommunikation mit Sockets kann auch mithilfe der SSL/TLS-Standards mithilfe von Secure Channel verschlüsselt werden. Dies wird auch als Schannel-Technologie bezeichnet. Schannel ist ein Sicherheitsunterstützungsanbieter (Security Support Provider, SSP), der eine Reihe von Sicherheitsprotokollen enthält, die Identitätsauthentifizierung und sichere, private Kommunikation durch Verschlüsselung bereitstellen. Schannel wird hauptsächlich für Internetanwendungen verwendet, die eine sichere HTTP-Kommunikation (Hypertext Transfer Protocol) erfordern. Weitere Informationen finden Sie unter [Sicherer Kanal.](../secauthn/secure-channel.md)
