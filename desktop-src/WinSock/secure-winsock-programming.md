# <a name="secure-winsock-programming"></a>Sichere Winsock-Programmierung

Im folgenden finden Sie eine Anleitung zum Sichern der Windows Sockets-Programmierung. Es wurde entwickelt, um ein Verständnis der Winsock-Sicherheit und der Optionen zu bieten, die für den Entwickler der sicheren Netzwerk Anwendung verfügbar sind.

-   [Verwenden von " \_ reuseaddr" und " \_ exclusiveaddruse"](using-so-reuseaddr-and-so-exclusiveaddruse.md)
-   [Winsock Secure Socket Extensions](winsock-secure-socket-extensions.md)

Die Kommunikation mit Sockets kann auch mithilfe der SSL/TLS-Standards mithilfe des sicheren Kanals, auch als SChannel-Technologie bezeichnet, verschlüsselt werden. SChannel ist eine Security Support Provider (SSP), die eine Reihe von Sicherheitsprotokollen enthält, die Identitäts Authentifizierung und eine sichere private Kommunikation durch Verschlüsselung bereitstellen. SChannel wird hauptsächlich für Internet Anwendungen verwendet, die eine HTTP-Kommunikation (Secure Hypertext Transfer Protocol) erfordern. Weitere Informationen finden Sie unter [sicherer Kanal](../secauthn/secure-channel.md).
