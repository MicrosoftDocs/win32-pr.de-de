# <a name="modifying-configuration-information-in-the-database"></a>Ändern von Konfigurationsinformationen in der Datenbank

Mithilfe der Erweiterungs-Snap-in-Erweiterung müssen Benutzer die Konfigurationsinformationen über den Analyse Knoten ändern können. Dadurch können Benutzer Änderungen vornehmen und die Konfiguration dann über einen bestimmten Zeitraum erneut anwenden. Die zu verwendende Logik sollte mit der Änderung von Informationen in den Konfigurationsdateien identisch sein. Die vorgenommenen Änderungen wirken sich auf die gespeicherte Konfiguration in der Sicherheitsdatenbank aus.
