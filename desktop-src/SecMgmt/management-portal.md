# <a name="security-management"></a>Sicherheitsverwaltung

## <a name="purpose"></a>Zweck

Die Sicherheitsverwaltungstechnologien können verwendet werden, um die Richtlinie [](/windows/desktop/SecGloss/p-gly) der lokalen Sicherheitsstelle [*(Local Security Authority,*](/windows/desktop/SecGloss/l-gly) LSA) und die Kennwortfilterrichtlinie zu verwalten, die Fähigkeit von Programmen aus externen Quellen abfragen und Dienstanlagen zu verwenden, die die Funktionalität des Sicherheitskonfigurationstools erweitern.

Microsoft-Sicherheitsverwaltungstechnologien umfassen die LSA-Richtlinien-API, die Kennwortfilter-API, die sicherere API und die Dienstsicherheitsanlagen-API. Mit diesen Technologien können Softwareentwickler Anwendungen erstellen, die Systeme und Anwendungen verwalten.

## <a name="developer-audience"></a>Entwicklergruppe

Die LSA-Richtlinien-API, die Kennwortfilter-API, die sicherere API und die Dienstsicherheitsanlagen-API sind für die Verwendung durch Entwickler von Anwendungen vorgesehen, mit denen Administratoren ihre Systeme verwalten und schützen können. Entwickler sollten mit den Programmiersprachen C und C++ und mit Windows Programmieren vertraut sein.
