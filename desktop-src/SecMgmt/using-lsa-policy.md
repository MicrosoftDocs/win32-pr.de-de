# <a name="using-lsa-policy"></a>Verwenden der LSA-Richtlinie

In den folgenden Themen wird beschrieben, wie die LSA-Richtlinien Funktionen verwendet werden.

-   Die [Verwendung von LSA-Unicode](using-lsa-unicode-strings.md) -Zeichen folgen erläutert, wie Zeichen folgen in das von einigen LSA-Richtlinien Funktionen verwendete Struktur Format konvertiert werden.
-   Beim [Öffnen eines Richtlinien Objekt Handles](opening-a-policy-object-handle.md) wird beschrieben, wie ein Handle für das lokale [**Richtlinien**](policy-object.md) Objekt geöffnet wird. Dieses Handle ist für viele der LSA-Richtlinien Funktionen als Eingabeparameter erforderlich.
-   Informationen zum [Verwalten von Richtlinien Informationen](managing-policy-information.md) finden Sie unter Festlegen und Abfragen von globalen Richtlinien Informationen für das lokale System und die Domäne.
-   Beim [empfangen von Richtlinien Änderungs Ereignissen](receiving-policy-change-events.md) wird beschrieben, wie ein Ereignis erstellt und registriert wird, um Benachrichtigungen zu empfangen, wenn sich Richtlinien Informationen ändern.
-   [Verwalten von Konto Berechtigungen](managing-account-permissions.md) erläutert das Erstellen, löschen und Auflisten von Konten sowie das Hinzufügen und Entfernen von Berechtigungen für diese Konten.
-   [Verwalten von Informationen zu vertrauenswürdigen](managing-trusted-domain-information.md) Domänen: Beschreibt das Erstellen, auflisten und löschen vertrauenswürdiger Domänen sowie das Festlegen und Abrufen von Informationen zu vertrauenswürdigen Domänen.
-   Beim über [setzen von Namen und SIDs](translating-between-names-and-sids.md) wird erläutert, wie SIDs Kontonamen zugeordnet werden und umgekehrt.
-   Die [Zuordnung der POSIX](mapping-posix-identifiers.md) -Bezeichner beschreibt, wie SIDs zu 32-Bit-POSIX-bezeichdern zugeordnet werden.
-   Beim [Speichern privater Daten](storing-private-data.md) wird erläutert, wie private Daten Zeichenfolgen gespeichert und abgerufen werden.

Informationen dazu, wie Sie die LSA zum Anmelden und Authentifizieren von Benutzern verwenden, finden Sie unter [LSA-Authentifizierung](/windows/desktop/SecAuthN/lsa-authentication).
