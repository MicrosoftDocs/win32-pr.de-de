# <a name="d-upnp-apis"></a>D (UPnP-APIs)

Enthält UPnP-bezogene Begriffe, die mit dem Buchstaben D beginnen.

[A](a-gly.md) B [C](c-gly.md) D [E](e-gly.md) F G [H](h-gly.md) I J K L M [N](n-gly.md) O [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) T [U](u-gly.md) V W X Y Z

<dl> <dt>

<span id="upnp.d_1_gly"></span><span id="UPNP.D_1_GLY"></span>**Gerät**
</dt> <dd>

Ein Gerät ist ein Container mit anderen Diensten oder Geräten. Ein Gerät kann andere logische Geräte enthalten.

</dd> <dt>

<span id="upnp.d_2_gly"></span><span id="UPNP.D_2_GLY"></span>**Gerätesteuerungsobjekt**
</dt> <dd>

Ein Gerätesteuerungsobjekt ist der zentrale Verwaltungspunkt für die Dienstobjekte eines Geräts. Wenn ein Gerät registriert ist, wird sein Gerätesteuerungsobjekt an den Gerätehost übergeben.

</dd> <dt>

<span id="upnp.d_3_gly"></span><span id="UPNP.D_3_GLY"></span>**Gerätebeschreibung**
</dt> <dd>

Eine Gerätebeschreibung ist eine Liste der generischen Eigenschaften aller Geräte in einem physischen Gerät, einschließlich der Dienste, der Hierarchie des Geräts und seiner Eigenschaften.

</dd> <dt>

<span id="upnp.d_4_gly"></span><span id="UPNP.D_4_GLY"></span>**Gerätehost**
</dt> <dd>

Der Gerätehost verarbeitet die Ermittlungs-, Beschreibungs-, Steuerungs- und Ereignisfunktionen eines Geräts. Geräte enthalten den Code zum Verarbeiten der gerätespezifischen Funktionalität, z. B. das Zurückspulen eines VCR.

</dd> <dt>

<span id="upnp.d_5_gly"></span><span id="UPNP.D_5_GLY"></span>**Geräteanbieter**
</dt> <dd>

Ein Geräteanbieter ist eine Entität, die beim Gerätehost registriert ist. Dieser Geräteanbieter wird nicht im Netzwerk veröffentlicht. Der Geräteanbieter registriert jedoch andere Geräte und veröffentlicht sie im Netzwerk.

</dd> <dt>

<span id="upnp.d_6_gly"></span><span id="UPNP.D_6_GLY"></span>**Gerätetyp**
</dt> <dd>

Ein Standardgerätetyp wird durch urn:schemas-upnp-org:device gekennzeichnet, gefolgt von einem eindeutigen Namen, der vom UPnP-Forum zugewiesen wird.

Zwischen UPnP-basierten Gerätevorlagen und Gerätetypen besteht eine 1:1-Beziehung. UPnP-Anbieter können zusätzliche Gerätetypen angeben. diese werden durch urn:domain-name:device gekennzeichnet, gefolgt von einem eindeutigen Namen, der vom Anbieter zugewiesen wird, wobei domain-name ein domänenname ist, der beim Anbieter registriert ist.

</dd> </dl>
