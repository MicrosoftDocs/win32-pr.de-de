# <a name="d-upnp-apis"></a>D (UPnP-APIs)

Enthält UPnP-bezogene Begriffe, die mit dem Buchstaben "D" beginnen.

[a](a-gly.md) B [C](c-gly.md) D [E](e-gly.md) F G [H](h-gly.md) I J K L M [N](n-gly.md) O [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) T [U](u-gly.md) V W X Y Z

<dl> <dt>

<span id="upnp.d_1_gly"></span><span id="UPNP.D_1_GLY"></span>**Schutz**
</dt> <dd>

Ein Gerät ist ein Container anderer Dienste oder Geräte. Ein Gerät kann andere logische Geräte enthalten.

</dd> <dt>

<span id="upnp.d_2_gly"></span><span id="UPNP.D_2_GLY"></span>**Geräte Steuerungsobjekt**
</dt> <dd>

Ein Geräte Steuerungsobjekt ist der zentrale Verwaltungspunkt für die Dienst Objekte eines Geräts. Wenn ein Gerät registriert ist, wird das zugehörige Geräte Steuerungsobjekt an den Geräte Host übermittelt.

</dd> <dt>

<span id="upnp.d_3_gly"></span><span id="UPNP.D_3_GLY"></span>**Geräte Beschreibung**
</dt> <dd>

Eine Geräte Beschreibung ist eine Liste der generischen Eigenschaften aller Geräte in einem physischen Gerät, einschließlich der-Dienste, der Hierarchie des Geräts und der zugehörigen Eigenschaften.

</dd> <dt>

<span id="upnp.d_4_gly"></span><span id="UPNP.D_4_GLY"></span>**Geräte Host**
</dt> <dd>

Der Geräte Host verarbeitet die Ermittlungs-, Beschreibungs-, Steuerungs-und Ereignis Teile der Funktionalität eines Geräts. Geräte enthalten den Code, der die gerätespezifischen Funktionen wie das rebuggen eines VCR behandelt.

</dd> <dt>

<span id="upnp.d_5_gly"></span><span id="UPNP.D_5_GLY"></span>**Geräte Anbieter**
</dt> <dd>

Ein Geräte Anbieter ist eine Entität, die beim Geräte Host registriert ist. Dieser Geräte Anbieter wird nicht im Netzwerk veröffentlicht. Der Geräte Anbieter registriert jedoch andere Geräte und veröffentlicht Sie im Netzwerk.

</dd> <dt>

<span id="upnp.d_6_gly"></span><span id="UPNP.D_6_GLY"></span>**Gerätetyp**
</dt> <dd>

Ein Standard Gerätetyp wird durch urn: Schemas-UPnP-org: Device: gefolgt von einem eindeutigen Namen angegeben, der vom UPnP-Forum zugewiesen wird.

Zwischen UPnP-basierten Geräte Vorlagen und Gerätetypen besteht eine eins-zu-eins-Beziehung. UPnP-Anbieter können zusätzliche Gerätetypen angeben. Diese werden durch den urn: Domain-Name: Device: gefolgt von einem eindeutigen Namen angegeben, der vom Hersteller zugewiesen wurde. dabei ist Domain-Name ein Domänen Name, der für den Hersteller registriert ist.

</dd> </dl>
