# <a name="s-upnp-apis"></a>S (UPnP-APIs)

Enthält UPnP-bezogene Begriffe, die mit dem Buchstaben "S" beginnen.

[a](a-gly.md) B [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F G [H](h-gly.md) I J K L M [N](n-gly.md) O [P](p-gly.md) Q [R](r-gly.md) S T [U](u-gly.md) V W X Y Z

<dl> <dt>

<span id="upnp.s_1_gly"></span><span id="UPNP.S_1_GLY"></span>**Leistungs**
</dt> <dd>

Bei einem Dienst handelt es sich um eine logische Funktionseinheit. Dienste machen Aktionen verfügbar und modellieren einige oder alle Zustände eines physischen Geräts mithilfe von Statusvariablen.

</dd> <dt>

<span id="upnp.s_2_gly"></span><span id="UPNP.S_2_GLY"></span>**Dienst Beschreibung**
</dt> <dd>

Eine Dienst Beschreibung ist eine Liste der Aktionen, die ein Dienst bereitstellt, und der Zustandsvariablen, die den Aktionen zugeordnet sind.

</dd> <dt>

<span id="upnp.s_3_gly"></span><span id="UPNP.S_3_GLY"></span>**Dienst Objekt**
</dt> <dd>

Ein Dienst Objekt ist die Steuerungspunkt-API-Darstellung eines Dienstanbieter. Ein Dienst Objekt implementiert die [**iupnpservice**](/windows/desktop/api/Upnp/nn-upnp-iupnpservice) -Schnittstelle.

</dd> <dt>

<span id="upnp.s_4_gly"></span><span id="UPNP.S_4_GLY"></span>**Diensttyp**
</dt> <dd>

Ein Diensttyp ist ein URN, der einen Dienst identifiziert. Zwischen einer UPnP-Dienst Vorlage und einem Diensttyp besteht eine eins-zu-eins-Beziehung. Mehrere Standard Dienst Typen werden von den Arbeitsausschüssen des UPnP-Forums definiert. Die Dienst Typen weisen die Form urn: Schemas-UPnP-org: Service: ServiceType: Version auf. Ein Scanner-Diensttyp könnte z. b. "urn: Schemas-UPnP-org: Service: Scanner: 1" lauten.

UPnP-Anbieter können zusätzliche Dienste angeben. Solche Dienste werden durch den urn: Domain-Name: Service: ServiceType: Version bezeichnet, wobei Domain-Name ein Domänen Name ist, der für den Hersteller registriert ist.

</dd> <dt>

<span id="upnp.s_5_gly"></span><span id="UPNP.S_5_GLY"></span>**Zustands Variable**
</dt> <dd>

Eine Zustands Variable ist ein Datenelement, das einen Teil des Dienst Zustands beschreibt.

</dd> </dl>
