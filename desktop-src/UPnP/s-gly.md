# <a name="s-upnp-apis"></a>S (UPnP-APIs)

Enthält UPnP-bezogene Begriffe, die mit dem Buchstaben S beginnen.

[A](a-gly.md) B [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F G [H](h-gly.md) I J K L M [N](n-gly.md) O [P](p-gly.md) Q [R](r-gly.md) S T [U](u-gly.md) V W X Y Z

<dl> <dt>

<span id="upnp.s_1_gly"></span><span id="UPNP.S_1_GLY"></span>**Service**
</dt> <dd>

Ein Dienst ist eine logische Funktionseinheit. Dienste machen Aktionen verfügbar und modellieren einige oder alle Zustände eines physischen Geräts mithilfe von Zustandsvariablen.

</dd> <dt>

<span id="upnp.s_2_gly"></span><span id="UPNP.S_2_GLY"></span>**Dienstbeschreibung**
</dt> <dd>

Eine Dienstbeschreibung ist eine Liste der Aktionen, die ein Dienst bietet, und der den Aktionen zugeordneten Zustandsvariablen.

</dd> <dt>

<span id="upnp.s_3_gly"></span><span id="UPNP.S_3_GLY"></span>**Dienstobjekt**
</dt> <dd>

Ein Dienstobjekt ist die Steuerungspunkt-API-Darstellung eines Diensts. Ein Dienstobjekt implementiert die [**IUPnPService-Schnittstelle.**](/windows/desktop/api/Upnp/nn-upnp-iupnpservice)

</dd> <dt>

<span id="upnp.s_4_gly"></span><span id="UPNP.S_4_GLY"></span>**Diensttyp**
</dt> <dd>

Ein Diensttyp ist ein URN, der einen Dienst identifiziert. Zwischen einer UPnP-Dienstvorlage und einem Diensttyp besteht eine 1:1-Beziehung. Mehrere Standarddiensttypen werden vom UPnP-Forum definiert. Die Diensttypen haben das Formular urn:schemas-upnp-org:service:serviceType:version. Ein Scannerdiensttyp könnte beispielsweise urn:schemas-upnp-org:service:scanner:1 sein.

UPnP-Anbieter können zusätzliche Dienste angeben. Solche Dienste werden durch urn:domain-name:service:serviceType:version bezeichnet, wobei domain-name ein Domänenname ist, der beim Anbieter registriert ist.

</dd> <dt>

<span id="upnp.s_5_gly"></span><span id="UPNP.S_5_GLY"></span>**Zustandsvariable**
</dt> <dd>

Eine Zustandsvariable ist ein Datenteil, der einen Teil des Zustands eines Diensts beschreibt.

</dd> </dl>
