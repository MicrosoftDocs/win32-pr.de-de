# <a name="control-point-programming-considerations"></a>Überlegungen zur Steuerungspunkt Programmierung

Entwickler, die UPnP-basierte Anwendungen (oder Steuerungs Punkte) erstellen, müssen diese Anwendungen von einem coinit \_ Apartmentthreaded-Client verwenden. Es gibt bekannte Probleme bei der Verwendung der Kontrollpunkt-API von einem coinit- \_ Multithread-Client, der unter hoher Belastung ausgeführt wird.
