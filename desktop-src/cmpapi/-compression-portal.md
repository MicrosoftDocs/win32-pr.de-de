# <a name="compression-api"></a>Komprimierungs-API

## <a name="purpose"></a>Zweck

Die Komprimierungs-API macht die Windows MSZIP-, XPRESS-, XPRESS \_ HUFF- und LZMS-Komprimierungsalgorithmen verfügbar. Dadurch können Entwickler von Windows-Anwendungen Versionen verwalten, Dienste nutzen und die verfügbar gemachten Komprimierungsalgorithmen erweitern.

## <a name="developer-audience"></a>Entwicklergruppe

Die Komprimierungs-API ist für die Verwendung durch professionelle C/C++-Entwickler von Windows-Anwendungen konzipiert. Die Komprimierungs-API macht die verlustfreien Komprimierungsalgorithmen verfügbar, die von Windows über eine öffentliche Schnittstelle und eine Benutzermodus-API verwendet werden. Die Komprimierungs-API ist die empfohlene Methode für Windows-Entwickler, um die Komprimierungsalgorithmen zu steuern. Dieses Feature ist 64-Bit unter 64-Bit-Windows und 32-Bit unter 32-Bit-Windows.

## <a name="run-time-requirements"></a>Laufzeitanforderungen

Die Komprimierungs-API ist ab Windows 8 oder Windows Server 2012 verfügbar.
