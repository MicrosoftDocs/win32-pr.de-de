# <a name="about-native-wifi"></a>Informationen zu nativem WiFi

Die nativen WiFi-API-Funktionen (Application Programming Interface) haben zwei Zwecke: zum Verwalten von Drahtlos Netzwerk Profilen und zum Verwalten von drahtlos Netzwerkverbindungen. Die API-Elemente werden vom Auto Configuration Module (ACM) verfügbar gemacht. Die zur Verfügung gestellten Verbindungs-und Trennung-API-Elemente können verwendet werden, um die automatische Konfigurations Logik zu überschreiben
