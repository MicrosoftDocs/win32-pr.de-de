# <a name="file-terminals"></a>Dateiterminals

In TAPI 3.1 wird das Terminalobjekt um ein Dateiterminal erweitert. Dateiterminals bieten eine einfache Möglichkeit, allgemeine Vorgänge auszuführen, z. B. das Aufzeichnen eines Audiodatenstroms oder eines Streams in einer Datei oder die nachfolgende Wiedergabe aufgezeichneter Audiodaten in einem Stream.

Fünf grundlegende Objekte werden verwendet, um Dateiterminals zu bearbeiten:

-   Terminal für die Dateiwiedergabe
-   Dateiaufzeichnungsterminal
-   Dateiwiedergabespur
-   Dateiaufzeichnungsspur
-   Dateiereignis

Die File Terminal-Objekte können als Abstraktion für den Speicher bezeichnet werden, bei dem es sich in der Regel um eine Datei handelt. Spuren entsprechen den einzelnen Audiodatenströmen in diesem Speicher, z. B. wenn die Stimme aufgezeichnet wird.

Weitere Informationen zu Dateiterminals finden Sie unter:

-   [Terminal für die Dateiwiedergabe und Terminal für die Dateiaufzeichnung](file-playback-terminal-and-file-recording-terminal.md)
-   [Nachverfolgen von Terminals](track-terminals.md)

Weitere Informationen und Codebeispiele und Aufgabenlisten im Zusammenhang mit Dateiterminals finden Sie unter [Verwenden von Dateiterminals.](using-file-terminals.md)
