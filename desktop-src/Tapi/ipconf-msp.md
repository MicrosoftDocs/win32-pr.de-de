# <a name="ipconf-msp"></a>Ipconf-msp

\[Ipconf MSP ist nicht für die Verwendung in Windows Vista, Windows Server 2008 und nachfolgenden Versionen des Betriebssystems verfügbar. Die RTC-Client-API bietet eine ähnliche Funktionalität.\]

Der ipconf-msp (Confmsp.dll) bietet, wenn er mit seinem TSP (ipconf. TSP) gekoppelt ist, Konferenz Zugriff auf IP-Netzwerke (Internet Protocol), z. b. das Internet und viele unternehmensinterne Intranets. Zusätzlich zu den standardmäßigen MSPi-Methoden macht dieses MSP Schnittstellen verfügbar, die die Kontrolle über einzelne Teilnehmer einer Konferenz ermöglichen.

Eine Zusammenfassung der anbieterspezifischen Schnittstellen, die von diesem MSP verfügbar gemacht werden, finden Sie unter [ipconf MSP-Schnittstellen](ipconf-msp-interfaces.md) .

Der ipconf-MSP wurde in Windows 2000 SP1 verbessert, um die Kommunikation zwischen h323-und SDP-Clients zu ermöglichen. Sie macht jetzt zusätzlich die [**imulticastcontrol**](imulticastcontrol.md) -Schnittstelle verfügbar.

Der ipconf-MSP wird mit Windows 2000 installiert.
