<dl> <dt>

<span id="tapi2.integrated_services_digital_network_isdn__tapgloss"></span><span id="TAPI2.INTEGRATED_SERVICES_DIGITAL_NETWORK_ISDN__TAPGLOSS"></span>**Integriertes Services Digital Network (ISDN)**
</dt> <dd>

Ein vollständig digitales Telefon/Telekommunikations Netzwerk für die schnelle Durchführung von Sprache, Daten, Bildern und Videos, indem digital codierte Signale mit einer Geschwindigkeit von bis zu 144.000 Bit/s in den Punkten der Basis Raten Schnittstelle (BRI-ISDN) und viel höher in den PRI-ISDN-Linien (Primary Rate Interface) gesendet werden. ISDN besteht aus mindestens drei Kanälen und bis zu 32 Kanälen für die gleichzeitige, unabhängige Übertragung von Sprache und Daten. Weitere Informationen finden Sie unter [*Basic Rate Interface (BRI-ISDN)*](b-tapgloss.md), [*Primary Rate Interface (PRI-ISDN)*](p-tapgloss.md).

</dd> <dt>

<span id="tapi2.interactive_voice_response_ivr_application_tapgloss"></span><span id="TAPI2.INTERACTIVE_VOICE_RESPONSE_IVR_APPLICATION_TAPGLOSS"></span>**IVR-Anwendung (Interactive Voice Response)**
</dt> <dd>

Eine Telefonieanwendung, die einen Telefon Aufrufer über eine Hierarchie von Menüs führt, sprach Antworten bereitstellt, sprach-und Dateneingaben sammelt und andere Vorgänge im Auftrag des Aufrufers oder des Programm Sponsors ausführt. Siehe [*webtelefoniemethode*](w-tapgloss.md).

</dd> <dt>

<span id="tapi2.inverse_multiplexing_tapgloss"></span><span id="TAPI2.INVERSE_MULTIPLEXING_TAPGLOSS"></span>**umgekehrtes Multiplexing**
</dt> <dd>

Der Prozess, bei dem die Bandbreite einem Aufruf ermöglicht, weitere Informationen gleichzeitig im gleichen Zeitraum zu übertragen, als Informationen von einem einzelnen Signal, das über einen einzigen Kanal übertragen wird. Dies erfolgt durch Aufteilen eines einzelnen hoch Geschwindigkeits Kanals in mehrere Signale, wobei die verschiedenen Signale über mehrere Funktionen übertragen werden, die mit einer niedrigeren Geschwindigkeit als das ursprüngliche Signal betrieben werden, und dann die separat übertragenen Teile in das ursprüngliche Signal mit der ursprünglichen Rate wieder kombiniert werden. Siehe [*Bandbreite*](b-tapgloss.md).

</dd> <dt>

<span id="tapi2.ip_multicast_tapgloss"></span><span id="TAPI2.IP_MULTICAST_TAPGLOSS"></span>**IP-Multicast**
</dt> <dd>

IP-Multicast ist ein Schema, mit dem dieselben Daten gleichzeitig an mehrere Computer gesendet werden. Ein Netzwerk mit aktivierten Multicast fähigen Routern kann diese Informationen viel schneller vermitteln, indem Pakete nur einmal zwischen den Routern gesendet werden. Weitere Informationen finden Sie unter Informationen [zu Rendezvous-IP-telefoniekonferenzen](./about-rendezvous-ip-telephony-conferencing.md).

</dd> <dt>

<span id="tapi2.isdn_tapgloss"></span><span id="TAPI2.ISDN_TAPGLOSS"></span>**Digitalen**
</dt> <dd>

Weitere Informationen finden Sie unter *Integrated Services Digital Network (ISDN)*.

</dd> </dl>
