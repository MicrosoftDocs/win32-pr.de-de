# <a name="multitrack-terminals"></a>Multitrack-Terminals

TapI 3.1 führt das Konzept eines Multitrack-Terminals ein, bei dem es sich im Wesentlichen um eine Sammlung von "Track"-Terminals handelt. Multitrack-Terminals erleichtern die Auslastung des Programmierers in Situationen, in denen mehrere Medienstreams an einer Kommunikationssitzung beteiligt sind.

Das [Terminal für die Dateiaufzeichnung](file-playback-terminal-and-file-recording-terminal.md) ist ein Beispiel für ein Multitrack-Terminal. Sie besteht aus "Spuren", wobei jede "Spur" ein Terminal ist, das einem Stream in der aufgezeichneten Datei entspricht.

Ein [Trackterminal kann](track-terminals.md) ein anderes Terminal mit mehreren Verfolgungen oder ein einspuriges Terminal sein. Ein Einspurterminal ist ein Terminal, das über keine Trackterminals verfügt. Ein einspurig gehaltenes Terminal ist ein Terminal im TAPI 3-Sinn des Worts.
