# <a name="w-telephony-api"></a>W (telefonieapi)

<dl> <dt>

<span id="tapi2.web_telephony_tapgloss"></span><span id="TAPI2.WEB_TELEPHONY_TAPGLOSS"></span>**Webtelefonie**
</dt> <dd>

Eine Anwendungsplattform, die Dual-Access-Websites (Zugriff per Telefon oder Internet Browser) und HTML-basierte IVR-Anwendungen unterstützt. Die Plattform umfasst eine in den Server integrierte Lauf Zeit-Engine, eine COM-basierte Verwaltungsschnittstelle zum Installieren und Konfigurieren von webtelefonieanwendungen und Zugriff auf das TAPI 3. x-Aufruf Objekt und die Skript-API zum Implementieren von telefonievorgängen.

</dd> <dt>

<span id="tapi2.web_telephony_engine_webte__tapgloss"></span><span id="TAPI2.WEB_TELEPHONY_ENGINE_WEBTE__TAPGLOSS"></span>**Webtelefonieengine (WTE)**
</dt> <dd>

Bietet Unterstützung für HTML-basierte IVR-Anwendungen und Dual-Access-Websites (eine Website, auf die sowohl telefonisch als auch über einen grafischen Browser zugegriffen werden kann). Siehe *webtelefoniemethode*.

</dd> <dt>

<span id="tapi2.windows_open_services_architecture_wosa__tapgloss"></span><span id="TAPI2.WINDOWS_OPEN_SERVICES_ARCHITECTURE_WOSA__TAPGLOSS"></span>**Windows Open Services-Architektur (WOSA)**
</dt> <dd>

Software Architektur, die eine einzige Schnittstelle auf Systemebene zum Verbinden von Front-End-Anwendungen mit Back-End-Diensten bereitstellt. Windows-Telefonie ist Teil von WOSA.

</dd> </dl>
