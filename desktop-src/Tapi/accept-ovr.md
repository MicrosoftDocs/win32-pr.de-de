# <a name="accept"></a>Akzeptieren

In einigen Umgebungen wird ein Benutzer nicht automatisch mit einem Angebots Gespräch benachrichtigt, z. b. durch klingeln oder durch eine Meldung auf dem Computerbildschirm. Der Accept-Vorgang startet den Warnungs Prozess (in der Regel klingelt), und der [Antwort](answer-ovr.md) Vorgang erfolgt danach. Die Anwendung kann die Sitzung [Löschen](drop-ovr.md) , [umleiten](redirect-ovr.md) oder akzeptieren.

Wenn der aktuelle Dienstanbieter dies unterstützt, kann die Anwendung Benutzer Benutzerinformationen zum Zeitpunkt der Annahme senden. Es gibt keine Garantie, dass das Netzwerk diese Informationen an die aufrufende Partei übermitteln wird.

Nicht alle Dienstanbieter unterstützen die Verwendung dieses Vorgangs.

**TAPI 2. x:** Siehe [**lineaccept**](/windows/win32/api/tapi/nf-tapi-lineaccept).
