<span data-ttu-id="d979d-104">Wie in der Übersicht über [Sitzungs Bezeichner](./session-identifier-ovr.md) erwähnt, ist ein Rückruf handle das Mittel, mit dem eine TAPI 2,2-Anwendung eine bestimmte Kommunikationssitzung identifiziert.</span><span class="sxs-lookup"><span data-stu-id="d979d-104">As is mentioned in the [Session Identifier](./session-identifier-ovr.md) overview, a call handle is the means by which a TAPI 2.2 application identifies a particular communications session.</span></span> <span data-ttu-id="d979d-105">Wenn eine Anwendung eine Sitzung initiiert, gibt TAPI ein Rückruf Handle zur Verwendung in weiteren Vorgängen oder Abfragen zurück.</span><span class="sxs-lookup"><span data-stu-id="d979d-105">When an application initiates a session, TAPI returns a call handle for use in further operations or queries.</span></span> <span data-ttu-id="d979d-106">Wenn eine Anwendung über eine eingehende Sitzung benachrichtigt wird, übergibt TAPI auch ein "callhandle".</span><span class="sxs-lookup"><span data-stu-id="d979d-106">When an application is notified of an incoming session, TAPI also passes in a call handle.</span></span>

<span data-ttu-id="d979d-107">Nachdem eine Sitzung beendet wurde und sich der Sitzungszustand im Leerlauf befindet, bleibt das Telefon Handle gültig, bis die Anwendung das Handle zuweist oder die Zeile geschlossen wird.</span><span class="sxs-lookup"><span data-stu-id="d979d-107">After a session has ended and the session state is idle, the call handle remains valid until the application deallocates the handle or the line is closed.</span></span> <span data-ttu-id="d979d-108">Die Zeile wird möglicherweise von der Anwendung geschlossen, oder es wird eine Meldung zum [**\_ Schließen**](line-close.md) der Nachricht empfangen.</span><span class="sxs-lookup"><span data-stu-id="d979d-108">The line might be closed by the application, or it may receive a [**LINE\_CLOSE**](line-close.md) message.</span></span> <span data-ttu-id="d979d-109">Wenn eine Zeile geschlossen wird, werden alle Aufruf Handles für Aufrufe in der Zeile sofort ungültig.</span><span class="sxs-lookup"><span data-stu-id="d979d-109">If a line is closed, all call handles to calls on the line instantly become invalid.</span></span>

<span data-ttu-id="d979d-110">Nachdem ein-Rückruf in den *Leerlauf* Zustand wechselt, kann die Anwendung die Informationsstruktur und den Status des Aufrufes weiterhin lesen.</span><span class="sxs-lookup"><span data-stu-id="d979d-110">After a call reverts to the *idle* state, the application is still allowed to read the call's information structure and status.</span></span> <span data-ttu-id="d979d-111">Dies ermöglicht es Anwendungen, Vorgänge wie [**linegetcallinfo**](/windows/desktop/api/Tapi/nf-tapi-linegetcallinfo) zum Abrufen von Aufruf Informationen zu Protokollierungs Zwecken zu verwenden.</span><span class="sxs-lookup"><span data-stu-id="d979d-111">This enables applications to use operations such as [**lineGetCallInfo**](/windows/desktop/api/Tapi/nf-tapi-linegetcallinfo) to retrieve call information for logging purposes.</span></span>

<span data-ttu-id="d979d-112">Wenn die Anwendung nicht mehr für das Handle eines im Leerlauf befindlichen Aufrufes verwendet wird, muss Sie [**linedebelegungecall**](/windows/desktop/api/Tapi/nf-tapi-linedeallocatecall) aufrufen, um den vom-Anruf im Zusammenhang mit dem-Befehl zugeordneten Speicher freizugeben.</span><span class="sxs-lookup"><span data-stu-id="d979d-112">When the application has no further use for the handle of an idle call, it must call [**lineDeallocateCall**](/windows/desktop/api/Tapi/nf-tapi-linedeallocatecall) to free system-allocated memory related to the call.</span></span> <span data-ttu-id="d979d-113">TAPI ordnet für jede Anwendung, die über ein Handle für den-Befehl verfügt, Arbeitsspeicher zu.</span><span class="sxs-lookup"><span data-stu-id="d979d-113">TAPI allocates memory for each call for each application that has a handle to the call.</span></span> <span data-ttu-id="d979d-114">Es ist wahrscheinlich, dass Dienstanbieter auch Speicher zum Speichern von Telefon Informationen zuweisen.</span><span class="sxs-lookup"><span data-stu-id="d979d-114">It is likely that service providers will allocate memory to hold call information as well.</span></span> <span data-ttu-id="d979d-115">Durch die Aufhebung der Zuordnung des-aufrufhandles einer Anwendung können die Bibliothek und der Dienstanbieter diese Speicherressourcen freigeben.</span><span class="sxs-lookup"><span data-stu-id="d979d-115">Deallocation of an application's call handle allows the library and the service provider to reclaim these memory resources.</span></span> <span data-ttu-id="d979d-116">Das Handle einer Anwendung für einen-Aufrufvorgang wird nach einer erfolgreichen Aufhebung der Zuordnung ungültig.</span><span class="sxs-lookup"><span data-stu-id="d979d-116">An application's handle for a call becomes void after a successful deallocation.</span></span>

<span data-ttu-id="d979d-117">Die Anwendung muss selbst Speicher freigeben, der sich auf den-Rückruf bezieht, den Sie für Ihre eigenen Zwecke zugeordnet hat.</span><span class="sxs-lookup"><span data-stu-id="d979d-117">The application must itself free memory related to the call that it allocated for its own purposes.</span></span>
