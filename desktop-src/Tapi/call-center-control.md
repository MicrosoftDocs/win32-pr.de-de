# <a name="call-center-control"></a>Callcenter-Steuerelement

Die grundlegende TAPI 2,2 kann verwendet werden, um Callcenter und andere Elemente der Telefonienetzwerkinfrastruktur (z. b. IVR-und Voicemail-Server) über die Anruf Steuerung von Drittanbietern zu verwalten. Zur Unterstützung der Erstellung von ACD-Proxys, die auf Servern ausgeführt werden, wurden TAPI 2,2 Funktionen hinzugefügt, um Anwendungsentwicklern zusätzliche Unterstützung zu bieten.

In den folgenden Themen werden die TAPI 2,2-Features beschrieben, die Sie zum Implementieren von callcentersteuerelementen verwenden können:

-   [Modellieren eines Callcenter](modeling-of-a-call-center.md)
-   [Station](stations.md)
-   [Vorhersagbare Wahl](predictive-dialing.md)
-   [Callwarteschlangen und Routenpunkte](call-queues-and-route-points.md)
-   [Überwachung und Steuerung des ACD-Agents](acd-agent-monitoring-and-control.md)
-   [Stations Status-Steuerelement](station-status-control.md)
-   [Stations Status-Steuerelement](station-status-control.md)
-   [Timer für den CallState](call-state-timer.md)
-   [Medienereignis Timer](media-event-timers.md)

TAPI 3. x ist die bevorzugte API zum Schreiben von ACD-Agent-Anwendungen. Informationen zur Verwendung dieser Schnittstellen und Methoden finden Sie im Abschnitt " [callcentersteuerelemente](./about-call-center-controls.md) ". TAPI 2,2 bleibt jedoch bei Bedarf für vollständige ACD-Anwendungs Suites verwendbar.
