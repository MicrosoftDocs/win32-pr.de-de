<dl> <dt>

<span id="_dns_fault_tolerance_gly"></span><span id="_DNS_FAULT_TOLERANCE_GLY"></span>**Fehlertoleranz**
</dt> <dd>

Methode, um sicherzustellen, dass ein Netzwerk-oder Computersystem gegen Software-oder Hardwareausfälle resistent ist.

</dd> <dt>

<span id="_dns_flat_name_space_gly"></span><span id="_DNS_FLAT_NAME_SPACE_GLY"></span>**flacher Namespace**
</dt> <dd>

Ein nicht partitionierter Namespace, in dem jeder Eintrag eindeutig sein muss. Flache Namespaces sind aufgrund der Anforderung, dass jeder Eintrag eindeutig ist, nicht gut skalierbar.

</dd> <dt>

<span id="_dns_forwarder_gly"></span><span id="_DNS_FORWARDER_GLY"></span>**Weiterleitung**
</dt> <dd>

Ein DNS-Server, der für die Kommunikation mit Offsite-DNS-Servern vorgesehen ist.

</dd> <dt>

<span id="_dns_fully_qualified_domain_name_fqdn__gly"></span><span id="_DNS_FULLY_QUALIFIED_DOMAIN_NAME_FQDN__GLY"></span>**Vollständig qualifizierter Domänen Name (FQDN)**
</dt> <dd>

Ein Name, der einen Host in der DNS-Hierarchie eindeutig identifiziert, sodass ein Host mit dem Namen Server1 in der Products-Hierarchie bei Microsoft möglicherweise einen FQDN von Server1.Products.Microsoft.com besitzt.

</dd> </dl>
