# <a name="d1201-pushaxisalignedclip-with-antialiased-multisampled-target"></a>D1201: PushAxisAlignedClip With Antialiased Multisampled Target

PushAxisAlignedClip wurde mit einem Multisamplingziel verwendet. Der Aufruf wird weiterhin ausgegeben, aber Artefakte können auftreten, wenn die Begrenzungen des Cliprechtecks nicht pixelbündig ausgerichtet sind. Darüber hinaus stellt das gerenderte Ergebnis möglicherweise nicht das erwartete Multisamplemuster dar.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Die Anwendung hat eine Multisampling-Direct3D-Textur als Direct2D-Ziel interoperiert und pushAxisAlignedClip auf dem Ziel aufgerufen.

## <a name="possible-fixes"></a>Mögliche Fehlerbehebungen

Vermeiden Sie die Interoperabilität von Multisampled-Texturen, wenn clip-Funktionalität wichtig ist. Die Verwendung von Ebenen oder Clips mit diesen Ressourcentypen kann zu Leistungseinbußen oder unerwarteten Renderingergebnissen führt. Dies gilt insbesondere, wenn die Begrenzungen des Clips nicht pixelbündig ausgerichtet sind. Verwenden Sie nach Möglichkeit eine alternative Möglichkeit zum Maskieren von Inhalten anstelle eines Clips, z. B. eine Direct3D-Scissor-Datei. Alternativ können Sie eine Zwischenressource anstelle einer Multisampled-Ressource verwenden. Wenn die Multisamplingressource für das Antialiasing von 2D-Inhalten vorgesehen ist, sollten Sie berücksichtigen, ob das integrierte Pro-Primitive-Antialiasing von Direct2D ausreichend ist.
