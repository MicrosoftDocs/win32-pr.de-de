# <a name="d1201-pushaxisalignedclip-with-antialiased-multisampled-target"></a>D1201: pushaxisalignedclip mit Antialiasing-Multisampling-Ziel

Pushaxisalignedclip wurde mit einem Multisampling-Ziel verwendet. Der-Rückruf wird weiterhin ausgegeben, aber es können Artefakte auftreten, wenn die Begrenzungen des Clip Rechtecks nicht Pixel bündig sind. Außerdem stellt das gerenderte Ergebnis möglicherweise nicht das erwartete Multisampling-Muster dar.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Die Anwendung hat eine Multisampling-Direct3D-Textur als Direct2D-Ziel interagiert und pushaxisalignedclip auf dem Ziel aufgerufen.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Vermeiden Sie Interoperation Multisampling-Texturen, wenn Clip-Funktionalität wichtig ist. Die Verwendung von Ebenen oder Clips mit diesen Arten von Ressourcen kann zu Leistungseinbußen oder unerwarteten renderingergebnissen führen. Dies trifft vor allem dann zu, wenn die Begrenzungen des Clips nicht Pixel orientiert sind. Verwenden Sie nach Möglichkeit eine alternative Methode, um Inhalt anstelle eines Clips (z. b. einem Direct3D Scissor) zu maskieren. Oder verwenden Sie eine zwischengeschaltete Ressource anstelle einer Multisampling-Ressource. Wenn die Multisampling-Ressource für das Antialiasing von 2D-Inhalten vorgesehen ist, sollten Sie überprüfen, ob das integrierte renderingkonventionen-Antialiasing ausreichend ist.
