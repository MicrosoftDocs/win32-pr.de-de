# <a name="d1127-alpha-mode-mismatch"></a>D1127: nicht übereinstimmende Alpha Modus

Der an diese API eingeführte Alpha-Modus ist nicht mit dem Pixel Format der- \[ *Oberfläche* kompatibel \] .

## <a name="placeholders"></a>Platzhalter

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*Oberfläche*
</dt> <dd>

Die Direct3D-Oberfläche, die den Alpha-Modus nicht unterstützt.

</dd> </dl> 

|             |             |
|-------------|-------------|
| Fehlerstufe | Information |



 

## <a name="possible-causes"></a>Mögliche Ursachen

Vorgänge wie ID2D1RenderTarget:: deatebitmapfromwicbitmap oder ID2D1RenderTarget:: roatedxgisurfacerendertarget erfordern sowohl eine Oberfläche als auch einen angegebenen Oberflächen Alpha-Modus. Direct2D überprüft, ob der angegebene Alpha Modus mit der-Oberfläche kompatibel ist. Wenn Sie nicht kompatibel ist, gibt Direct2D diese Meldung aus.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Ändern Sie die Anwendung so, dass Sie einen Alpha-Modus verwendet, der \_ mit dem Alpha Modus der zugrunde liegenden Oberfläche übereinstimmt oder mit diesem kompatibel ist. Weitere Anleitungen zu den Alpha Modi, die von Direct2D verfügbar gemacht werden, finden Sie [unter Unterstützte Pixel Formate und Alpha Modi](supported-pixel-formats-and-alpha-modes.md) .
