# <a name="d1127-alpha-mode-mismatch"></a>D1127: Alphamoduskonflikt

Der an diese API übergebene Alphamodus ist nicht mit dem Pixelformat der \[ *Oberfläche* \] kompatibel.

## <a name="placeholders"></a>Platzhalter

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*Oberfläche*
</dt> <dd>

Die Direct3D-Oberfläche, die den Alphamodus nicht unterstützt.

</dd> </dl> 



 

## <a name="possible-causes"></a>Mögliche Ursachen

Vorgänge wie ID2D1RenderTarget::CreateBitmapFromWicBitmap oder ID2D1RenderTarget::CreateDxgiSurfaceRendertarget erfordern sowohl einen Surface- als auch einen angegebenen Surface-Alphamodus. Direct2D überprüft, ob der angegebene Alphamodus mit der Oberfläche kompatibel ist. Wenn sie nicht kompatibel ist, gibt Direct2D diese Meldung aus.

## <a name="possible-fixes"></a>Mögliche Fehlerbehebungen

Ändern Sie die Anwendung so, dass sie einen \_ ALPHA-MODUS verwendet, der mit dem Alphamodus der zugrunde liegenden Oberfläche übereinstimmt oder mit diesem kompatibel ist. Weitere Informationen zu den von Direct2D verfügbar gemachten Alphamodi finden Sie unter [Unterstützte Pixelformate und Alphamodi.](supported-pixel-formats-and-alpha-modes.md)
