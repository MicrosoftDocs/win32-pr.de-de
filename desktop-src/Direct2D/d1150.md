# <a name="d1150-unsupported-mip-level-selection"></a>D1150: nicht unterstützte MIP-Ebene-Auswahl

Der angegebene IDXGISurface2 verfügt über mehrere MIP-Ebenen, aber auf dieser Geräteebene (d3d9) wird die Auswahl der MIP-Ebene nicht unterstützt.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Interpt Direct3D Texturen können durch MIP zugeordnet werden; Direct2D funktioniert nur jeweils auf einer MIP-Ebene. Andere MIP-Ebenen der übergeordneten Textur werden von Direct2D nicht beeinflusst oder gerendert. Geräte auf Featureebene 9 unterstützen die Auswahl auf MIP-Ebene jedoch nicht und können diese Interoperabilitäts Funktion nicht verwenden.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Für Anwendungen, die erwartungsgemäß auf Featureebene 9-Geräten ausgeführt werden sollen, sollten Sie die Renderinglogik ändern, um das Zusammenwirken von mipzugeordneten Texturen zu vermeiden. Arbeiten Sie, wenn möglich, stattdessen eine zwischengeschaltete zwischen. Um die Auswirkungen der Umgehung dieser Einschränkung zu minimieren, kann die Anwendung dies als einen auf Featureebene 9 spezifischen Fall Back Pfad implementieren.
