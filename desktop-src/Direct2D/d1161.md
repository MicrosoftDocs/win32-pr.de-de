# <a name="d1161-copyfrommemory-requires-cpu-readable-bitmap"></a>D1161: copyfrommemory erfordert CPU-lesbare Bitmap.

Diese Methode ist für eine Bitmap, die mit [**D2D1- \_ \_ bitmapoptionen-CPU- \_ \_ Lese**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options)Vorgang erstellt wurde, unzulässig.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Diese Meldung kann bei einem Aufrufen von [**ID2D1Bitmap:: copyfrommemory**](/windows/win32/api/d2d1/nf-d2d1-id2d1bitmap-copyfrommemory)auftreten. Wenn die Bitmap mithilfe von D2D1 \_ -Bitmap-Optionen CPU-Lesevorgang erstellt wurde \_ \_ \_ , meldet Direct2D einen Fehler, und die debugschicht gibt diese Meldung aus.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Wenn es für eine Anwendung erforderlich ist, [**ID2D1Bitmap:: copyfrommemory**](/windows/win32/api/d2d1/nf-d2d1-id2d1bitmap-copyfrommemory)aufzurufen, stellen Sie sicher, dass die Bitmap nicht als CPU-lesbar erstellt wird. Wenn es nicht möglich ist, die Oberflächen Attribute der Bitmap zu ändern, sollten Sie überprüfen, ob ein entsprechender Renderingvorgang für das gewünschte Ergebnis vorliegt.
