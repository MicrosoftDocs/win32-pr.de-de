# <a name="d1181-enddraw-incorrect-state"></a>D1181: EndDraw (falscher Zustand)

EndDraw wurde aufgerufen, aber das renderTarget befand sich nicht in einem gültigen Zustand. Dies kann daran liegen, dass EndDraw ohne übereinstimmendes beginDraw aufgerufen wird.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Die Anwendung hat mehrere Aufrufe an EndDraw durchgeführt, ohne beginDraw für dasselbe Ziel aufzurufen. Bei der Verwendung von Direct2D sollten Anwendungen den einzelnen Aufrufen von EndDraw mit einem früheren Aufrufen von beginDraw entsprechen. Alternativ kann diese Meldung auftreten, wenn sich das Ziel in einem fehlerhaften Zustand befindet und nicht renderfähig ist.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Stellen Sie sicher, dass jeder Aufrufen von beginDraw mit einem nachfolgenden EndDraw übereinstimmt. Stellen Sie beim Übermitteln von Befehlen an ein Ziel mithilfe von Flush oder EndDraw sicher, dass das zurückgegebene Ergebnis S OK ist, \_ bevor Sie versuchen, das Ziel erneut zu verwenden.
