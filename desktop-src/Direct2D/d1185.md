# <a name="d1185-fillgeometry-opacity-mask-brush-restriction"></a>D1185: FillGeometry Opacity Mask Brush Restriction

Wenn FillGeometry mit einer Deckkraftmaske verwendet wird, muss der primäre Pinsel ein geklammerter Bitmappinsel sein.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Bei Verwendung von ID2D1RenderTarget::FillGeometry war der primäre Pinsel ein anderer Typ als ID2D1BitmapBrush. Oder der primäre Pinsel hatte einen anderen X- oder y-Extend-Modus als D2D1 \_ EXTEND \_ MODE \_ CLAMP.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Wenn die Anwendung FillGeometry mit einer Deckkraftmaske verwendet, stellen Sie sicher, dass der primäre Pinsel (nicht zu verwechseln mit dem Deckkraftpinsel) ein ID2D1BitmapBrush mit geklammerten Erweiterungsmodi ist. Wenn das Durchlässigkeitsmaskierungsverhalten nicht wichtig ist, lassen Sie das Opacity Brush-Argument als NULL, um diese Einschränkungen für den primären Pinsel zu entfernen.
