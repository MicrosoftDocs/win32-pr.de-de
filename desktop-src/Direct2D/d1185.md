# <a name="d1185-fillgeometry-opacity-mask-brush-restriction"></a>D1185: Füll Geometrie mit Füll Geometrie-Pinsel Einschränkung

Wenn fillgeometry mit einer Deck Kraft Maske verwendet wird, muss es sich bei dem primär Pinsel um einen gepinten Bitmap-Pinsel handeln.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Bei Verwendung von ID2D1RenderTarget:: fillgeometry war der primär Pinsel ein anderer Typ als ID2D1BitmapBrush. Oder der primäre Pinsel hatte einen x-oder y-Erweiterungs Modus, der nicht die D2D1 Erweiterungs \_ \_ Modus-Klammer ist \_ .

## <a name="possible-fixes"></a>Mögliche Korrekturen

Wenn die Anwendung fillgeometry mit einer Deck Kraft Maske verwendet, stellen Sie sicher, dass der primär Pinsel (nicht mit dem Deck Kraft Pinsel verwechselt werden muss) ein ID2D1BitmapBrush mit gebundenen Erweiterungs Modi ist. Wenn das Verhalten der Deck Kraft Maskierung nicht wichtig ist, belassen Sie das Deck Kraft Pinsel-Argument als NULL, um diese Einschränkungen für den primär Pinsel zu entfernen.
