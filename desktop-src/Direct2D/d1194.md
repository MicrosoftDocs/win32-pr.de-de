# <a name="d1194-flush-outstanding-dc"></a>D1194: leeren DC leeren

Das Aufrufen von Flush oder EndDraw zwischen einem GetDC/ReleaseDC-Paar ist ungültig.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Bei der Verwendung eines mit GDI interbetriebenen renderTarget kann eine Anwendung den Besitz des DC des Ziels mithilfe von igdiinteroprendertarget:: GetDC oder igdiinteroprendertarget:: ReleaseDC steuern. Es ist jedoch nicht zulässig, ID2D1RenderTarget:: Flush oder ID2D1RenderTarget:: EndDraw aufzurufen, während die Anwendung die Kontrolle über den DC hat. "Flush" und "EndDraw" können dazu führen, dass das Rendering auf der Ziel Ressource erfolgt, und Direct2D sollte nicht versuchen, die Ressource zur gleichen Zeit wie die Anwendung zu ändern.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Um diese Meldung zu beheben, stellen Sie sicher, dass die Anwendung ID2D1RenderTarget:: Flush oder ID2D1RenderTarget:: EndDraw nicht aufruft, während Sie den Besitz eines Domänen Controllers von igdiinteroprendertarget besitzt. Organisieren Sie nach Möglichkeit die Anwendung so, dass das Direct2D-Renderingverhalten von Ihrem GDI-Verhalten getrennt wird, um den DC-Besitz deutlicher zu machen.
