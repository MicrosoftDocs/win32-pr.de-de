# <a name="d1125-dxgi-format-mismatch"></a>D1125: nicht übereinstimmende DXGI-Format

Das an diese API über gegebene Pixel Format ist nicht mit dem Pixel Format der- \[ Oberfläche kompatibel \] .

## <a name="placeholders"></a>Platzhalter

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*Oberfläche*
</dt> <dd>

Das angegebene Renderziel oder die angegebene Bitmap.

</dd> </dl>

## <a name="possible-causes"></a>Mögliche Ursachen

Vorgänge wie ID2D1RenderTarget:: deatebitmapfromwicbitmap oder ID2D1RenderTarget:: roatedxgisurfacerendertarget erfordern sowohl eine Oberfläche als auch ein angegebenes Oberflächen Format. Direct2D überprüft, ob der angegebene Surface-Typ mit der-Oberfläche kompatibel ist. Wenn Sie nicht kompatibel ist, gibt Direct2D diese Meldung aus.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Stellen Sie sicher, dass das angegebene DXGI- \_ Format mit dem Typ der zugrunde liegenden Oberfläche übereinstimmt oder mit diesem kompatibel ist. Verwenden Sie für Direct3D Surface-Ressourcen, die von der Anwendung erstellt wurden, dasselbe DXGI- \_ Format, das zum Erstellen der Ressource verwendet wurde. Weitere Anleitungen finden Sie [unter Unterstützte Pixel Formate und Alpha-Modi](supported-pixel-formats-and-alpha-modes.md) .
