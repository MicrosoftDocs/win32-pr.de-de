# <a name="d1125-dxgi-format-mismatch"></a>D1125: DXGI-Formatkonflikt

Das an diese API übergebene Pixelformat ist nicht mit dem Pixelformat der \[ Oberfläche \] kompatibel.

## <a name="placeholders"></a>Platzhalter

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*Oberfläche*
</dt> <dd>

Das angegebene Renderziel oder die angegebene Bitmap.

</dd> </dl>

## <a name="possible-causes"></a>Mögliche Ursachen

Vorgänge wie ID2D1RenderTarget::CreateBitmapFromWicBitmap oder ID2D1RenderTarget::CreateDxgiSurfaceRendertarget erfordern sowohl eine Oberfläche als auch ein angegebenes Oberflächenformat. Direct2D überprüft, ob der angegebene Oberflächentyp mit der Oberfläche kompatibel ist. Wenn sie nicht kompatibel ist, gibt Direct2D diese Meldung aus.

## <a name="possible-fixes"></a>Mögliche Fehlerbehebungen

Stellen Sie sicher, dass das angegebene DXGI FORMAT mit dem Typ der zugrunde liegenden Oberfläche übereinstimmt oder mit diesem \_ kompatibel ist. Verwenden Sie für Direct3D-Oberflächenressourcen, die von der Anwendung erstellt wurden, dasselbe \_ DXGI-FORMAT, das zum Erstellen der Ressource verwendet wurde. Weitere Informationen finden Sie unter [Unterstützte Pixelformate und Alphamodi.](supported-pixel-formats-and-alpha-modes.md)
