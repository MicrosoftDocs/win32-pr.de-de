# <a name="d1186-dc-render-target-requires-binddc"></a>D1186: das DC-Renderziel erfordert binddc.

Es ist ungültig, ein DC-Renderziel zu verwenden, bevor die binddc-Methode aufgerufen wird.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Die Anwendung hat eine ID2D1DCRenderTarget erstellt und verwendet, um beginDraw aufzurufen, aber noch nicht die binddc-Methode aufzurufen. Oder "binddc" hat einen Fehler verursacht.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Vor der Verwendung eines Renderziels mit DC-interpt sollten Anwendungen binddc aufrufen, um das Ziel einem GDI-Gerätekontext zuzuordnen. Das Ziel wird erst als gültig betrachtet, wenn ein GDI-Gerätekontext zugewiesen wurde. Ein Beispiel finden Sie unter ID2D1DCRenderTarget:: binddc. Wenn die Anwendung binddc bereits aufruft, stellen Sie sicher, dass binddc keinen Fehler zurückgegeben hat.
