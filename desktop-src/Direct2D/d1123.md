# <a name="d1123-dxgi-format-unknown-not-allowed"></a>D1123: DXGI-Format Unbekannt nicht zulässig

DXGI \_ FORMAT UNKNOWN ist als Parameter für diese API nicht \_ zulässig.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Für Direct2D-Funktionen, die ein Oberflächenformat als Parameter akzeptieren, sollte das Oberflächenformat vollständig angegeben werden. In einigen Fällen kann es unbekannt bleiben. Wenn Sie beispielsweise ID2D1RenderTarget:: CreateHwndRenderTarget oder ID2D1RenderTarget::CreateDxgiSurfaceRenderTarget aufrufen, leitet Direct2D möglicherweise ein Oberflächenformat aus dem Auslagerungskettenformat oder dem Direct3D-Oberflächenformat ab.

## <a name="possible-fixes"></a>Mögliche Korrekturen

In Fällen, in denen das Oberflächenformat nicht abgeleitet werden konnte, erstellt Direct2D die Ressource nicht und gibt diese Meldung aus. Um dies zu adressieren, ändern Sie die Anwendung so, dass für den Vorgang, der diese Meldung verursacht, ein anderes Oberflächenformat als UNKNOWN angegeben wird. Ein angemessenes Standardoberflächenformat, das im Zweifelsfall verwendet werden kann, ist DXGI \_ FORMAT \_ B8G8R8A8 \_ UNORM. Wenn die Anwendung eine Direct2D-Ressource erstellen muss, um mit einer Direct3D- oder WIC-Ressource zu zusammenarbeiten, wählen Sie ein Format aus, das der interoperierten Ressource entspricht. Die Semantik für WIC-Ressourcen unterscheidet sich geringfügig von DXGI-Ressourcen. Weitere Informationen finden [Sie unter Unterstützte Pixelformate und Alphamodi.](supported-pixel-formats-and-alpha-modes.md)
