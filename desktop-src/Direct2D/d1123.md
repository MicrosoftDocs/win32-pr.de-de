# <a name="d1123-dxgi-format-unknown-not-allowed"></a>D1123: DXGI-Format unbekannt ist nicht zulässig.

Unbekanntes DXGI- \_ Format \_ ist nicht als Parameter für diese API zulässig.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Für Direct2D-Funktionen, die ein Oberflächen Format als Parameter annehmen, sollte das Oberflächen Format vollständig angegeben werden. In einigen Fällen kann die Datei unbekannt bleiben. Wenn Sie z. b. ID2D1RenderTarget:: createhwndrendertarget oder ID2D1RenderTarget:: createdxgisurfacerendertarget aufrufen, leitet Direct2D möglicherweise ein Surface-Format aus dem SwapChain-Format oder das Backup Direct3D Surface-Format ab, wie dies möglicherweise der Fall ist.

## <a name="possible-fixes"></a>Mögliche Korrekturen

In Fällen, in denen es nicht möglich war, das Oberflächen Format abzuleiten, erstellt Direct2D die Ressource nicht und gibt diese Meldung aus. Um dies zu beheben, ändern Sie die Anwendung so, dass für den Vorgang, der diese Meldung auslöst, ein anderes Oberflächen Format als unbekannt angegeben wird. Ein vernünftiges Standard Oberflächen Format, das im Zweifelsfall verwendet werden kann, ist DXGI- \_ Format \_ B8G8R8A8 \_ unorm. Wenn die Anwendung eine Direct2D-Ressource für die Interaktion mit einer Direct3D-oder WIC-Ressource erstellen muss, wählen Sie ein Format aus, das mit der interbetriebenen Ressource übereinstimmt. Die Semantik für WIC-Ressourcen unterscheidet sich geringfügig von den DXGI-Ressourcen. Weitere Anleitungen finden Sie [unter Unterstützte Pixel Formate und Alpha-Modi](supported-pixel-formats-and-alpha-modes.md) .
