# <a name="d1149-print-detected-an-invalid-unicode-character"></a>D1149: Print hat ein ungültiges Unicode-Zeichen erkannt.

Direct2D Print: Es wurde ein ungültiges Unicode-Zeichen im dwrite- \_ Glyphe- \_ Lauf \_ Description. String erkannt. Sie wurde durch Leerzeichen ersetzt (0x20).






 

## <a name="possible-causes"></a>Mögliche Ursachen

Direct2D Print berücksichtigt ungültige Zeichen, die in der [Unicode-Dokumentation](https://unicode.org/faq/utf_bom.html#utf16-7)notiert sind, sowie bestimmte ASCII-Zeichen außerhalb des sichtbaren Bereichs, die nicht gedruckt werden können. Die Verwendung dieser Zeichen verursacht keinen Fehler. Stattdessen ersetzt Direct2D Print diese Zeichen durch Leerzeichen.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Um diese Benachrichtigung zu vermeiden, verwenden Sie nicht falsch formatierte Unicode-Zeichen oder nicht druckbare ASCII-Zeichen beim Zeichnen von Text für die Druckfunktion.
