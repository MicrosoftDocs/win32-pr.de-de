# <a name="d1116-unexpected-hardware-creation-error"></a>D1116: Unerwarteter Hardwareerstellungsfehler

Beim \[ Erstellen eines Direct3D-Ziels ist ein unerwarteter *Fehlercode* \] aufgetreten.






 

## <a name="placeholders"></a>Platzhalter

<dl> <dt>

<span id="error_code"></span><span id="ERROR_CODE"></span>*Fehlercode*
</dt> <dd>

Der aufgetretene Fehlercode.

</dd> </dl>

## <a name="possible-causes"></a>Mögliche Ursachen

Diese Meldung tritt auf, wenn Direct2D keine Hardwarezielressource mit den von der Anwendung angegebenen Parametern erstellen konnte. Eine wahrscheinliche Ursache ist, dass die Anwendung ein Hardwareziel angefordert hat, bei dem Direct2D den Besitz der Zuweisung der Zieltextur übernehmen muss, z. B. über [**ID2D1Factory::CreateHwndRenderTarget,**](/windows/win32/api/d2d1/nf-d2d1-id2d1factory-createhwndrendertarget(constd2d1_render_target_properties__constd2d1_hwnd_render_target_properties__id2d1hwndrendertarget))und die angeforderte Größe zu groß war. Jede Direct3D-Featureebene erzwingt ein Limit für die Textur dimensionierung. Weitere Informationen finden Sie [unter Direct3D-Featureebenen.](/previous-versions/windows/apps/hh994923(v=win.10))

## <a name="possible-fixes"></a>Mögliche Fehlerbehebungen

Stellen Sie beim Zuordnen eines Ziels sicher, dass die Zielgröße die maximal zulässige Größe auf Featureebene nicht überschreitet. Um Probleme mit der Zielzuordnung genauer zu behandeln, sollten Sie die Anwendung ändern, um die Zieltextur selbst zuzuordnen, und überprüfen Sie, ob die Zuordnung erfolgreich ist. Um diese Textur direkt als Ziel zu verwenden, interoperieren Sie sie als [**ID2D1Bitmap**](/windows/win32/api/d2d1/nn-d2d1-id2d1bitmap) und verwenden **die ID2D1Bitmap** als Argument für [**ID2D1DeviceContext::SetTarget.**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1devicecontext-settarget)
