# <a name="d1116-unexpected-hardware-creation-error"></a>D1116: Unerwarteter Hardware Erstellungs Fehler.

\[  \] Beim Erstellen eines Direct3D-Ziels ist ein unerwarteter Fehlercode aufgetreten.

Der Fehlercode ist aufgetreten.

</dd> </dl>

## <a name="possible-causes"></a>Mögliche Ursachen

Diese Meldung tritt auf, wenn Direct2D keine Hardware Ziel Ressource mit den von der Anwendung angegebenen Parametern erzeugt. Eine wahrscheinliche Ursache ist, dass die Anwendung ein Hardware Ziel angefordert hat, bei dem Direct2D den Besitz der Zuordnung der Ziel Textur übernehmen muss, z. b. durch [**ID2D1Factory:: anatehwndrendertarget**](/windows/win32/api/d2d1/nf-d2d1-id2d1factory-createhwndrendertarget(constd2d1_render_target_properties__constd2d1_hwnd_render_target_properties__id2d1hwndrendertarget)), und die angeforderte Größe zu groß war. Jede Direct3D-Funktionsebene erzwingt einen Grenzwert für die Textur Größenanpassung. Siehe [Direct3D featureebenen](/previous-versions/windows/apps/hh994923(v=win.10)).

## <a name="possible-fixes"></a>Mögliche Korrekturen

Stellen Sie beim Zuordnen eines Ziels sicher, dass die Zielgröße die maximal zulässige Größe für die Featureebene nicht überschreitet. Um die Ziel Zuordnungs Probleme besser zu beheben, können Sie die Anwendung ändern, um die Ziel Textur selbst zuzuordnen, und sicherstellen, dass die Zuordnung erfolgreich ist. Um diese Textur direkt als Ziel zu verwenden, interoperieren Sie Sie als [**ID2D1Bitmap**](/windows/win32/api/d2d1/nn-d2d1-id2d1bitmap) , und verwenden Sie die **ID2D1Bitmap** als Argument für [**ID2D1DeviceContext:: SetTarget**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1devicecontext-settarget).
