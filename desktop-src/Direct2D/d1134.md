# <a name="d1134-buffer-precision-disallowed-by-caps"></a>D1134: Puffer Genauigkeit durch Caps nicht zulässig

Das Gerät verfügt nicht über die Möglichkeit, die angegebene Puffer Genauigkeit zu unterstützen.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Direct2D ermöglicht es einer Anwendung, eine gewünschte Puffer Genauigkeit für einige Vorgänge anzugeben. Beispielsweise wird die Genauigkeit, die für die Imaging-Pipeline verwendet wird, mithilfe von [**ID2D1DeviceContext:: * Trend deringcontrols**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1devicecontext-setrenderingcontrols(constd2d1_rendering_controls))angegeben. Hardware Direct2D Geräte unterstützen jedoch möglicherweise nicht alle Puffer präzigationen, abhängig von ihrer Funktionsebene. Beispielsweise unterstützen Geräte auf Featureebene 9 unter Umständen keine Puffer Genauigkeits Optionen, die nicht 8 Bits pro Kanal sind.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Stellen Sie für Anwendungen, die auf der Hardware ausgeführt werden, sicher, dass die Genauigkeits Spezifizierer für alle Puffer von der Funktionsebene der Hardware unterstützt Verwenden Sie [**ID3D11Device:: checkformatsupport**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkformatsupport) und [**ID3D11Device:: checkfeaturesupport**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkfeaturesupport) , um weitere Informationen darüber zu erhalten, welche Formate von der Hardware unterstützt werden, und um zu bestimmen, welche Oberflächen Genauigkeit zulässig ist. Für einen konsistenten Betrieb mit allen Geräten der Featureebene 9 sollten Sie einen Fall Back Mechanismus entwerfen, um Puffer mit hoher Genauigkeit zu vermeiden.
