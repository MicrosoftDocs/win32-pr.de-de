# <a name="d1134-buffer-precision-disallowed-by-caps"></a>D1134: Puffergenauigkeit durch Obergrenzen nicht zu verdringen

Das Gerät verfügt nicht über die Möglichkeit, die gegebene Puffergenauigkeit zu unterstützen.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Direct2D ermöglicht es einer Anwendung, eine gewünschte Puffergenauigkeit für einige Vorgänge anzugeben. Beispielsweise wird die für die Imagingpipeline verwendete Genauigkeit mit [**id2D1DeviceContext::SetRenderingControls angegeben.**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1devicecontext-setrenderingcontrols(constd2d1_rendering_controls)) Hardware Direct2D-Geräte unterstützen jedoch möglicherweise nicht alle Puffergenauigkeiten, je nach Featureebene. Beispielsweise unterstützen Geräte auf Featureebene 9 möglicherweise keine Puffergenauigkeitsoptionen mit einer Größe von 8 Bits pro Kanal.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Stellen Sie für Anwendungen, die auf Hardware ausgeführt werden, sicher, dass alle Puffergenauigkeitsspezifizierer von der Featureebene der Hardware unterstützt werden. Erwägen Sie die Verwendung von [**ID3D11Device::CheckFormatSupport**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkformatsupport) und [**ID3D11Device::CheckFeatureSupport,**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkfeaturesupport) um weitere Informationen darüber zu erhalten, welche Formate von der Hardware unterstützt werden, und um zu bestimmen, welche Oberflächengenauigkeiten zulässig sind. Entwerfen Sie für einen konsistenten Betrieb mit allen Geräten auf Featureebene 9 einen Fallbackmechanismus, um Puffer mit hoher Genauigkeit zu vermeiden.
