# <a name="d1180-begindraw-incorrect-state"></a>D1180: BeginDraw Incorrect State

BeginDraw wurde mehrmals ohne entsprechenden Aufruf von EndDraw aufgerufen.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Die Anwendung hat mehrere Aufrufe von BeginDraw vorgenommen, ohne EndDraw auf demselben Ziel auf aufruft. Bei Verwendung von Direct2D sollten Anwendungen mit jedem Aufruf von BeginDraw mit einem nachfolgenden EndDraw übereinstimmen. Alternativ kann diese Meldung auftreten, wenn sich das Ziel in einem fehlerhaften Zustand befindet und der BeginDraw-Aufruf das Rendern des Ziels nicht ermöglicht hat.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Stellen Sie sicher, dass jeder Aufruf von BeginDraw mit einem nachfolgenden EndDraw-Aufruf übereinstimmen. Überprüfen Sie beim Übermitteln von Befehlen an ein Ziel mit Flush oder EndDraw, ob das zurückgegebene Ergebnis S OK ist, bevor Sie erneut versuchen, \_ das Ziel zu verwenden.
