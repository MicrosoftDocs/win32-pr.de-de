# <a name="d1180-begindraw-incorrect-state"></a>D1180: beginDraw (falscher Zustand)

BeginDraw wurde mehrmals aufgerufen, ohne dass ein entsprechender Aufruf von EndDraw erfolgt ist.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Die Anwendung hat mehrere Aufrufe an beginDraw durchgeführt, ohne dass EndDraw für dasselbe Ziel aufgerufen wurde. Bei der Verwendung von Direct2D sollten Anwendungen die einzelnen Aufrufe von beginDraw mit einem nachfolgenden EndDraw vergleichen. Diese Meldung kann auch auftreten, wenn sich das Ziel in einem fehlerhaften Zustand befindet und der beginDraw-Aufruf nicht zulässt, dass das Ziel renderfähig ist.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Stellen Sie sicher, dass jeder Aufrufen von beginDraw mit einem nachfolgenden EndDraw übereinstimmt. Stellen Sie beim Übermitteln von Befehlen an ein Ziel mithilfe von Flush oder EndDraw sicher, dass das zurückgegebene Ergebnis S OK ist, \_ bevor Sie versuchen, das Ziel erneut zu verwenden.
