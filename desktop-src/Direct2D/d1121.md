# <a name="d1121-wrong-resource-domain"></a>D1121: falsche Ressourcen Domäne

Fehler bei einem Vorgang, weil eine Geräte abhängige Ressource mit der falschen ID2D1Device (Ressourcen Domäne) verknüpft ist.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Diese Meldung tritt auf, wenn eine Anwendung versucht hat, eine Zeichnungs Operation auszuführen, mit der Ressourcen aus verschiedenen Ressourcen Domänen gemischt werden. Geräte Kontexte, Pinsel und Bitmaps sind z. b. gerätespezifisch. Beim Ausführen eines Vorgangs, wie z. b. ID2D1DeviceContext::D rawbitmap, sollte das Argument Bitmap derselben Ressourcen Domäne wie der Gerätekontext angehören. In diesem Beispiel kann sichergestellt werden, dass die Bitmap mithilfe der Member-Funktion von "Member" des Geräte Kontexts anstelle der eines anderen Geräte Kontexts erstellt wird.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Wenn Gruppen von Ressourcen gleichzeitig verwendet werden sollen, stellen Sie sicher, dass Sie aus demselben ID2D1Device erstellt werden. Nicht Ziel Ressourcen, die direkt aus einem ID2D1Factory erstellt werden, z. b. Zustands Blöcke und Geometrien, werden als Geräte unabhängig angesehen und weisen diese Einschränkung nicht auf.
