# <a name="d1121-wrong-resource-domain"></a>D1121: Falsche Ressourcendomäne

Fehler bei einem Vorgang, weil eine geräteabhängige Ressource der falschen ID2D1Device (Ressourcendomäne) zugeordnet ist.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Diese Meldung tritt auf, wenn eine Anwendung versucht hat, einen Zeichnungsvorgang zu erstellen, bei dem Ressourcen aus verschiedenen Ressourcendomänen kombiniert werden. Gerätekontexte, Pinsel und Bitmaps sind beispielsweise gerätespezifisch. Beim Ausführen eines Vorgangs wie ID2D1DeviceContext::D rawBitmap sollte die Argumentbitmap zur gleichen Ressourcendomäne wie der Gerätekontext gehören. In diesem Beispiel können Sie dies sicherstellen, indem Sie die Bitmap mithilfe der CreateBitmap-Memberfunktion dieses Gerätekontexts und nicht mit der eines anderen Gerätekontexts erstellen.

## <a name="possible-fixes"></a>Mögliche Fehlerbehebungen

Wenn Gruppen von Ressourcen zusammen verwendet werden sollen, stellen Sie sicher, dass sie aus der gleichen ID2D1Device erstellt werden. Nichtzielressourcen, die direkt aus einer ID2D1Factory erstellt werden, z. B. Zustandsblöcke und Geometrien, werden als geräteunabhängig betrachtet und weisen diese Einschränkung nicht auf.
