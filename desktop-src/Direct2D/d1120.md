# <a name="d1120-invalid-attempt-to-draw-bitmap"></a>D1120: Ungültiger Versuch, Bitmap zu zeichnen

Ein Vorgang ist fehlgeschlagen, weil die Bitmap die Option D2D1 \_ BITMAP OPTIONS CANNOT DRAW \_ \_ \_ aufzeichnet.





 

## <a name="possible-causes"></a>Mögliche Ursachen

Diese Meldung tritt auf, wenn eine Anwendung versucht, eine Bitmap zu zeichnen, die mit D2D1 BITMAP OPTIONS CANNOT DRAW gekennzeichnet \_ \_ \_ \_ ist. Dies schließt direkte Aufrufe von DrawBitmap sowie das Zeichnen von Primitiven mithilfe eines Bitmappinsels ein.

## <a name="possible-fixes"></a>Mögliche Fehlerbehebungen

Das FLAG CANNOT \_ DRAW wird zum ordnungsgemäßen Kommentieren von Ressourcen verwendet, die nur als Ziele verwendet werden können, z. B. Swap chain backbuffers. Es ist jedoch auch zulässig, das FLAG CANNOT \_ DRAW für einige Zielbitmaps auszulassen, wenn sie auch als Quelle verwendet werden können. Wenn dies der Fall ist, lassen Sie einfach das FLAG CANNOT \_ DRAW weg. Wenn die Bitmap aufgrund ihrer exklusiven Verwendung als Ziel oder zum Zulassen der CPU-Sperrbarkeit mit CANNOT DRAW versehen werden \_ muss, sollten Sie eine Stagingtextur mit der gleichen Größe wie die gewünschte Quelle verwenden, jedoch ohne DIE \_ NOT DRAW-Bezeichnung. Kopieren Sie mit ID3D11DeviceContext::CopyResource in die Stagingtextur, und verwenden Sie die Stagingtextur als Quelle für den Zeichnungsvorgang. Die Verwendung einer Stagingtextur hat Auswirkungen auf die Leistung. sie sind möglicherweise akzeptabel, wenn sich die Quellbitmap nicht häufig ändert.
