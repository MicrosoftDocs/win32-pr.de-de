# <a name="d1120-invalid-attempt-to-draw-bitmap"></a>D1120: Ungültiger Versuch zum Zeichnen der Bitmap.

Bei einem Vorgang ist ein Fehler aufgetreten, da die \_ Option "D2D1 Bitmap \_ Options" \_ nicht gezeichnet werden kann \_ .



|             |             |
|-------------|-------------|
| Fehlerstufe | Information |



 

## <a name="possible-causes"></a>Mögliche Ursachen

Diese Meldung tritt auf, wenn eine Anwendung versucht, eine Bitmap zu zeichnen, die mit D2D1 \_ bitmapoptionen gekennzeichnet ist, \_ \_ kann nicht \_ gezeichnet werden. Dies umfasst direkte Aufrufe von drawbitmap sowie das Zeichnen von primitiven mithilfe eines Bitmap-Pinsels.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Das Flag zum Zeichnen kann nicht \_ zum ordnungsgemäßen kommentieren von Ressourcen verwendet werden, die nur als Ziele verwendet werden können, wie z. b. Austausch Ketten-Back-Ends. Es ist jedoch auch zulässig, das Flag für das nicht \_ Zeichnen für einige Ziel Bitmaps auszulassen, wenn Sie auch als Quelle verwendet werden können. Wenn dies der Fall ist, lassen Sie einfach das Flag kann nicht \_ gezeichnet werden. Wenn die Bitmap mit Anmerkungen versehen werden muss \_ , da Sie nicht gezeichnet werden kann, weil Sie als Ziel verwendet wird, oder um die CPU-Sperr barkeit zuzulassen, empfiehlt es sich, eine stagingtextur mit der gleichen Größe wie die gewünschte Quelle zu verwenden, ohne dass die Bezeichnung nicht gezeichnet werden kann \_ . Kopieren Sie mithilfe von Verknüpfung id3d11devicecontext aus:: copyresource in die stagingtextur, und verwenden Sie die stagingtextur als Quelle für den Zeichnungs Vorgang. Die Verwendung einer stagingtextur führt zu Leistungseinbußen. Sie sind möglicherweise akzeptabel, wenn die Quell Bitmap nicht häufig geändert wird.
