# <a name="d1120-invalid-attempt-to-draw-bitmap"></a>D1120: Ungültiger Versuch, Bitmap zu zeichnen

Fehler bei einem Vorgang, weil die Bitmap über die Option D2D1 \_ BITMAP OPTIONS CANNOT DRAW \_ \_ \_ verfügt.





 

## <a name="possible-causes"></a>Mögliche Ursachen

Diese Meldung tritt auf, wenn eine Anwendung versucht, eine Bitmap zu zeichnen, die mit D2D1 \_ BITMAP OPTIONS CANNOT DRAW gekennzeichnet \_ \_ \_ ist. Dies schließt direkte Aufrufe von DrawBitmap sowie das Zeichnen von Primitiven mithilfe eines Bitmappinsels ein.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Das FLAG CANNOT DRAW wird zum ordnungsgemäßen Kommentieren von Ressourcen verwendet, die nur als Ziele verwendet werden können, z. B. Swap \_ chain backbuffers. Es ist jedoch auch zulässig, das FLAG CANNOT DRAW für einige Zielbitmaps weglassen, wenn sie auch \_ als Quelle verwendet werden können. Wenn dies der Fall ist, lassen Sie einfach das FLAG CANNOT \_ DRAW weg. Wenn die Bitmap aufgrund ihrer exklusiven Verwendung als Ziel oder zum Zulassen der CPU-Sperrbarkeit mit CANNOT DRAW kommentiert werden muss, sollten Sie die Verwendung einer Stagingtextur mit der gleichen Größe wie die gewünschte Quelle, aber ohne die BEZEICHNUNG CANNOT DRAW in Betracht \_ \_ ziehen. Kopieren Sie mit id3D11DeviceContext::CopyResource in die Stagingtextur, und verwenden Sie die Stagingtextur als Quelle für den Zeichnungsvorgang. Die Verwendung einer Stagingtextur führt zu Leistungsauswirkungen. sie sind möglicherweise akzeptabel, wenn sich die Quellbitmap nicht häufig ändert.
