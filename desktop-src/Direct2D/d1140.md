# <a name="d1140-stroke-transform-type-uses-default-dpi"></a>D1140: der Stroke-Transformationstyp verwendet Standard-dpi

Beim Erweitern eines ID2D1Geometry mit einem fixierten oder einem haarzeiligen Strich geht Direct2D von einem dpi-96 aus. Beim Rendern der erweiterten Geometrie kann sich die Strichbreite ändern.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Wenn Direct2D eine Geometrie in ein Ziel zeichnet, wird die Geometrie erweitert. Allerdings kann eine Anwendung auch ID2D1Geometry-Objekte ohne ein Ziel erstellen und diese mithilfe von Methoden von ID2D1Geometry, wie z. b. "Ausweitung", "strokecontainspoint" und "getwidenedbounds", erweitern. Während dieser Vorgänge verwendet eine Anwendung möglicherweise den D2D1 \_ Stroke \_ Tranform-Typ " \_ \_ haarline" oder "Fixed". Diese Attribute stellen normalerweise einen dpi-Wert in Erwägung, aber wenn kein Ziel vorhanden ist, wird kein dpi-Wert angegeben. Diese Informations Meldung gibt eine Warnung aus, dass Direct2D einen Standard-dpi-Wert von 96,0 berücksichtigt.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Diese Meldung dient nur zu Informationszwecken und deutet nicht unbedingt auf ein Problem mit der Anwendung hin.
