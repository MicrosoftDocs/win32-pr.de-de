# <a name="d1158-pixels-unit-mode-unsupported-with-print"></a>D1158: Pixel Einheits Modus wird mit Druck nicht unterstützt

Direct2D Print: in der Befehlsliste, die zum Drucken gesendet wurde, wurde der nicht unterstützte Befehl "in der Befehlsliste" angezeigt.



|             |             |
|-------------|-------------|
| Fehlerstufe | Information |



 

## <a name="possible-causes"></a>Mögliche Ursachen

Direct2D Print lässt keinen Einheiten Modus von Pixel zu. Der Standard Einheits Modus ist Dips.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Wenn Sie diese Meldung beheben möchten, legen Sie beim Erstellen von Druck barem Inhalt den Einheiten Modus für den Gerätekontext auf Pixel fest. Wenn eine globale Skalierungs Transformation angewendet werden muss, die sich auf alle primitiven Zeichnungsvorgänge auswirkt, sollten Sie überprüfen, ob ein gleichwertiger-aufrufswert für ID2D1RenderTarget:: setdpi oder ID2D1RenderTarget:: setTransform ausreicht.
