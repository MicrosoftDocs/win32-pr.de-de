# <a name="d1151-unsupported-array-selection"></a>D1151: nicht unterstützte Array Auswahl

Der bereitgestellte IDXGISurface2 gehört zu einem Array, aber Array Slices werden auf dieser Geräteebene (d3d9) nicht unterstützt.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Interpt Direct3D Texturen sind möglicherweise in einem solchen zusammenhängen. Direct2D wird nur jeweils auf einem Array Slice ausgeführt. Andere Array Segmente der übergeordneten Textur werden von Direct2D nicht beeinflusst oder gerendert. Allerdings unterstützen Geräte auf Featureebene 9 keine Array Slice-Auswahl und können diese Interoperabilitäts Funktion nicht verwenden.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Für Anwendungen, die erwartungsgemäß auf Featureebene 9-Geräten ausgeführt werden sollen, sollten Sie die Renderinglogik ändern, um zu vermeiden, dass Sie die Verwendung von hoch betriebenen Texturen Arbeiten Sie, wenn möglich, stattdessen eine zwischengeschaltete zwischen. Um die Auswirkungen der Umgehung dieser Einschränkung zu minimieren, kann die Anwendung dies als einen auf Featureebene 9 spezifischen Fall Back Pfad implementieren.
