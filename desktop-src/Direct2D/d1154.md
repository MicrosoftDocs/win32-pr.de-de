# <a name="d1154-invalid-bitmap-input-format"></a>D1154: Ungültiges Bitmap-Eingabe Format

Dieser Vorgang ist nicht kompatibel mit dem Pixel Format der Bitmap.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Direct2D unterstützt nur-Alpha-Oberflächen Formate, wie z. b. das DXGI- \_ Format \_ a8 \_ unorm, aber Sie können nur als Deck Kraft Pinsel oder für die Maskierung verwendet werden. Wenn eine Anwendung einen Bitmap-Pinsel erstellt, dessen Bitmap ein Alpha geschütztes Format aufweist, sollte nicht versucht werden, den Pinsel mit Vorgängen zu verwenden, die einen Farbpinsel erfordern. ID2D1RenderTarget::D rawline und ID2D1RenderTarget:: fillrechteck, z. b. einen Farbpinsel.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Um diese Meldung zu beheben, stellen Sie sicher, dass Bitmaps mit reinen Alpha Formaten nur für die Deckkraft Pinsel oder Maskierung verwendet werden. Wenn eine Bitmap Farbinformationen enthalten soll, erstellen Sie Sie mithilfe eines Oberflächen Formats, das Farbkanäle einschließt.
