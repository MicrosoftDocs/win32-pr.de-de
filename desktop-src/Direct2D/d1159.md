# <a name="d1159-invalid-commandlist-topology"></a>D1159: Ungültige CommandList-Topologie.

Eine Befehlsliste kann nicht auf Effekte verweisen, die Teil von Effekt Diagrammen sind, die die Befehlsliste verwenden.



|             |             |
|-------------|-------------|
| Fehlerstufe | Information |



 

## <a name="possible-causes"></a>Mögliche Ursachen

Beim Auffüllen einer Befehlsliste kann eine Anwendung einen aufrufsvorgang von ID2D1DeviceContext::D rawImage enthalten. ID2D1DeviceContext::D rawImage kann zum Zeichnen von Bitmaps, Effekten oder Befehlslisten verwendet werden. Folglich ist es syntaktisch möglich, dass ein ID2D1DeviceContext::D rawImage-Aufrufe versucht, eine Befehlsliste zu zeichnen, während dieselbe Befehlsliste als Ziel festgelegt ist. Auf diese Weise selbst verweisende Befehlslisten sind jedoch ungültig. Direct2D identifiziert Fälle, in denen dies auftritt, und meldet einen Fehler, und die debugschicht gibt diese Meldung aus.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Beim Auffüllen einer Befehlsliste müssen Sie sicherstellen, dass ein Gerätekontext keinen Vorgang ausgibt, der dieselbe Befehlsliste verwendet. Der offensichtlichste Weg hierfür besteht darin, ID2D1DeviceContext::D rawImage mit derselben Befehlsliste aufzurufen, die als Ziel festgelegt ist. Es gibt jedoch auch weniger offensichtliche Möglichkeiten, wie dies auftreten kann: ein ID2D1DeviceContext::D rawImage verweist möglicherweise auf ein Wirkungs Diagramm, und ein Knoten im Effekt Diagramm kann versuchen, die Ziel Befehlsliste zu verwenden. Stellen Sie sicher, dass Befehlslisten vor der Verwendung richtig aufgefüllt werden und dass Aufrufe von ID2D1DeviceContext:: SetTarget auf das beabsichtigte Ziel verweisen.
