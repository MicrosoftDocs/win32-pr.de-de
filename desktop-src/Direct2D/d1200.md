# <a name="d1200-pushlayer-with-multisampled-target"></a>D1200: pushlayer mit Multisampling-Ziel

Pushlayer/poplayer wurde mit einem Multisampling-Ziel verwendet. Der-Rückruf wird weiterhin ausgegeben, das gerenderte Ergebnis stellt jedoch möglicherweise nicht das erwartete Multisampling-Muster dar.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Die Anwendung hat eine Multisampling-Direct3D-Textur als Direct2D-Ziel interagiert und als pushlayer für das Ziel bezeichnet.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Vermeiden Sie Interoperation Multisampling-Texturen, wenn Ebenenfunktionalität wichtig ist. Die Verwendung von Ebenen oder Clips mit diesen Arten von Ressourcen kann zu Leistungseinbußen oder unerwarteten renderingergebnissen führen. Verwenden Sie nach Möglichkeit eine alternative Methode, um die Deckkraft oder Maskierung anstelle von Ebenen zu steuern. Oder verwenden Sie eine zwischengeschaltete Ressource anstelle einer Multisampling-Ressource. Wenn die Multisampling-Ressource für das Antialiasing von 2D-Inhalten vorgesehen ist, sollten Sie überprüfen, ob das integrierte renderingkonventionen-Antialiasing ausreichend ist.
