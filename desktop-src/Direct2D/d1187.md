# <a name="d1187-settarget-called-on-outstanding-dc"></a>D1187: SetTarget wird für ausstehenden Dc aufgerufen

[**SetTarget wurde**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1devicecontext-settarget) zwischen einem [**GetDC**](/windows/win32/api/d2d1/nf-d2d1-id2d1gdiinteroprendertarget-getdc) / [**ReleaseDC-Paar**](/windows/win32/api/d2d1/nf-d2d1-id2d1gdiinteroprendertarget-releasedc) aufgerufen.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Die Anwendung hat ein [**ID2D1GdiInteropRenderTarget-Objekt**](/windows/win32/api/d2d1/nn-d2d1-id2d1gdiinteroprendertarget) erhalten, die [**GetDC-Methode**](/windows/win32/api/d2d1/nf-d2d1-id2d1gdiinteroprendertarget-getdc) aufgerufen und dann [**Flush**](/windows/win32/api/d2d1/nf-d2d1-id2d1rendertarget-flush) oder [**EndDraw**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1commandsink-enddraw) aufgerufen, bevor [**ReleaseDC aufgerufen wird.**](/windows/win32/api/d2d1/nf-d2d1-id2d1gdiinteroprendertarget-releasedc)

## <a name="possible-fixes"></a>Mögliche Korrekturen

Stellen Sie sicher, dass die Anwendung [**ReleaseDC**](/windows/win32/api/d2d1/nf-d2d1-id2d1gdiinteroprendertarget-releasedc) aufruft, bevor [**Flush**](/windows/win32/api/d2d1/nf-d2d1-id2d1rendertarget-flush) oder [**EndDraw**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1commandsink-enddraw)aufruft, und dass jeder Aufruf von **ReleaseDC** mit einem vorherigen Aufruf von [**GetDC übereinstimmen muss.**](/windows/win32/api/d2d1/nf-d2d1-id2d1gdiinteroprendertarget-getdc) Direct2D versucht nicht, mit einer GDI-Zielressource zu arbeiten, wenn der GDI-Gerätekontext derzeit von der Anwendung verwendet wird.
