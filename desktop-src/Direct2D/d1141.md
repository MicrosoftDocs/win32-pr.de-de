# <a name="d1141-print-requires-srgb-color-space"></a>D1141: für Print ist sRGB-Farbraum erforderlich.

Direct2D Print: benutzerdefinierter Farbraum wird nicht unterstützt. Verwenden Sie stattdessen sRGB.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Beim Erstellen eines Direct2D-Druck Steuer Elements verwendet eine Anwendung die Eigenschaften eines D2D1- \_ Druck \_ Steuer Elements \_ mit einem Color Space-Attribut. Direct2D unterstützt den [**\_ \_ \_ benutzerdefinierten D2D1-Farbraum**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_color_space) für einige Vorgänge, z. b. die Erstellung eines Farb Kontexts mithilfe von Daten in der Beim Erstellen eines Druck Steuer Elements unterstützt Direct2D jedoch nur integrierte Farbraum, wie z. b. sRGB oder ScRGB.

## <a name="possible-fixes"></a>Mögliche Korrekturen

Geben Sie beim Erstellen eines Druck Steuer Elements einen Nichtbenutzer definierten Farbraum wie sRGB oder ScRGB an, um die Nachricht zu beheben. Die Meldung weist nicht auf einen Fehler hin, gibt jedoch an, dass Direct2D den benutzerdefinierten Spezifizierer ignoriert und das Druck Steuerelement mithilfe eines sRGB-Farbraum erstellt.
