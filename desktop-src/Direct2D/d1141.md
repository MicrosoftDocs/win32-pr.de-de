# <a name="d1141-print-requires-srgb-color-space"></a>D1141: Druck erfordert sRGB-Farbraum

Direct2D Print: Benutzerdefinierter Farbraum wird nicht unterstützt. Verwenden Sie stattdessen sRGB.






 

## <a name="possible-causes"></a>Mögliche Ursachen

Beim Erstellen eines Direct2D-Drucksteuerelements verwendet eine Anwendung D2D1 \_ PRINT CONTROL PROPERTIES mit einem \_ \_ Farbraumattribut. Direct2D unterstützt [**D2D1 \_ COLOR SPACE \_ \_ CUSTOM**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_color_space) für einige Vorgänge, z. B. die Erstellung eines Farbkontexts mitHILFE von CSV-Daten. Beim Erstellen eines Drucksteuerelements unterstützt Direct2D jedoch nur integrierte Farbräume wie sRGB oder scRGB.

## <a name="possible-fixes"></a>Mögliche Fehlerbehebungen

Um die Nachricht zu adressieren, geben Sie beim Erstellen eines Drucksteuerelements einen nicht benutzerdefinierten Farbraum wie sRGB oder scRGB an. Die Meldung weist nicht auf einen Fehler hin, weist jedoch darauf hin, dass Direct2D den CUSTOM-Spezifizierer ignoriert und das Drucksteuerelement mithilfe eines sRGB-Farbraums erstellt.
